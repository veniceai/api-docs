تتواجد Venice API لتوفر للمطورين وصولًا خاصًا وغير مقيَّد إلى نماذج بمستوى إنتاجي خالية من المرشحات المخفية أو قرارات الصندوق الأسود.

مع تحسّن النماذج، نتقاعد أحيانًا من النماذج الأقدم لصالح بدائل أذكى أو أسرع أو أكثر قدرة. نُصمِّم هذه التحولات لتكون قابلة للتنبؤ ومنخفضة الاحتكاك.

## إيقافات النماذج

نعلم أن الإيقافات يمكن أن تكون مُعطِّلة. لذلك نهدف إلى الإيقاف فقط عند الضرورة، ونصمم ميزات مثل السمات والنماذج التي تحمل علامة Venice لتقليل التعطيل.

قد نُوقِف نموذجًا عندما:

- يقدم نموذج أحدث تحسينًا واضحًا لنفس حالة الاستخدام
- لم يعد النموذج يلبي معاييرنا للأداء أو الموثوقية
- يشهد استخدامًا منخفضًا باستمرار، واستمرار دعمه سيُجزِّئ التجربة للجميع

## عملية الإيقاف

عندما يستوفي النموذج معايير الإيقاف، نُعلِن عن التغيير بإشعار من 30 إلى 60 يومًا. تُنشَر إشعارات الإيقاف عبر [سجل التغييرات](https://featurebase.venice.ai/changelog) و[خادم Discord الخاص بنا](https://discord.gg/askvenice). عند استدعاء نموذج مُهجَر خلال فترة الإشعار، ستتضمن استجابة API تحذير إيقاف.

خلال فترة الإشعار، يظل النموذج متاحًا، رغم أننا قد نقلّل سعة البنية التحتية في بعض الحالات. نوفّر دائمًا بديلًا مُوصى به، وعند الحاجة، نقدم إرشادات الانتقال للمساعدة في التحول.

بعد تاريخ غروب الشمس، ستتم إعادة توجيه الطلبات إلى النموذج تلقائيًا إلى نموذج بقوة معالجة مماثلة بنفس السعر أو أقل. إذا لم يكن التوجيه ممكنًا لأسباب تقنية أو متعلقة بالسلامة، ستُعيد API استجابة 410 Gone. إذا تم اختيار نموذج مُهجَر عبر سمة (مثل `default_code` أو `default_vision` أو `fastest`)، فسيتم إعادة تعيين تلك السمة إلى بديل متوافق.

لا نُزيل النماذج بصمت ولا نُغيِّر السلوك دون إصدار. ستعرف دائمًا ما يعمل وكيف تستعد لما هو قادم.

<Note>
ترقيات الأداء فقط: قد نطلق تحسينات تحافظ على سلوك النموذج بينما تُحسِّن الأداء أو زمن الاستجابة أو كفاءة التكلفة. هذه التحديثات متوافقة مع الإصدارات السابقة ولا تتطلب أي إجراء من العميل.
</Note>

راجع [متعقب إيقاف النماذج](#model-deprecation-tracker) أدناه. للإعلانات السابقة، راجع [سجل التغييرات](https://featurebase.venice.ai/changelog) و[خادم Discord الخاص بنا](https://discord.gg/askvenice).

## كيف يتم اختيار النماذج لـ Venice API

نختار بعناية أي النماذج نتيحها بناءً على الأداء والموثوقية واحتياجات المطورين في العالم الحقيقي. ليتم تضمين النموذج، يجب أن يُظهِر أداءً قويًا، ويتصرف باتساق ضمن endpoints المتوافقة مع OpenAI، ويقدم تحسينًا واضحًا على الأقل لأحد النماذج التي ندعمها بالفعل.


النماذج التي نُقيِّمها قد تُطلق أولًا في [بيتا](/overview/beta-models) لجمع التعليقات والتحقق من الأداء على نطاق واسع.

لا نعرض نماذج زائدة عن الحاجة أو غير مُثبَتة أو غير جاهزة للاستخدام الإنتاجي المتسق. هدفنا هو إبقاء Venice API نظيفة وقادرة ومُحسَّنة لما يبنيه المطورون فعلًا.

تعرف على المزيد في [إيقافات النماذج](/overview/deprecations#model-deprecations) و<a href="/overview/models" target="_blank" rel="noopener noreferrer">قائمة النماذج الحالية</a>.

## الإصدار والأسماء البديلة

تُعرَّف جميع نماذج Venice بمعرّف فريد ودائم. على سبيل المثال:

```venice-uncensored```
```zai-org-glm-4.7```
```zai-org-glm-5```
```qwen3-vl-235b-a22b```

معرّفات النماذج مستقرة. في حالة وجود تغيير كاسر، سنُصدِر معرّف نموذج جديد (على سبيل المثال، إضافة إصدار مثل v2). إذا لم تكن هناك تغييرات كاسرة، فقد نُحدِّث النموذج الموجود وسنُبلِّغ بالتغييرات المهمة.

لتوفير المرونة، تحافظ Venice أيضًا على أسماء بديلة رمزية، مُنفَّذة عبر السمات، تشير إلى النموذج الافتراضي الموصى به لمهمة معينة:

تقدم السمات تجريدًا مستقرًا لاختيار النماذج بينما تمنح Venice المرونة لتحسين التنفيذ الأساسي. يمكن للمطورين الذين يفضلون الوصول التلقائي إلى أحدث النماذج الموصى بها الاعتماد على الأسماء البديلة القائمة على السمات.

للتطبيقات التي تتطلب اتساقًا صارمًا وسلوكًا قابلًا للتنبؤ، نوصي بالإشارة إلى معرّفات النماذج الثابتة.

## التعليقات

يمكنك إرسال تعليقاتك أو طلباتك عبر [بوابة Featurebase الخاصة بنا](https://featurebase.venice.ai). نحتفظ بـ [سجل تغييرات](https://featurebase.venice.ai/changelog) عام، ومتعقب خارطة طريق، ومنطق شفاف لإضافة أو ترقية أو إزالة النماذج، ونشجع المشاركة المجتمعية المستمرة.


## متعقب إيقاف النماذج

النماذج التالية مُجدوَلة للإيقاف أو تم إيقافها مؤخرًا. نوصي بالانتقال إلى البدائل المقترحة قبل تاريخ الإزالة. تظل النماذج مُدرَجة لمدة 30 يومًا بعد تاريخ إزالتها.

### التحقق من حالة الإيقاف عبر API

يمكنك التحقق مما إذا كان النموذج مُجدوَلًا للتقاعد باستدعاء endpoint [قائمة النماذج](/api-reference/endpoint/models/list). تتضمن النماذج ذات تاريخ التقاعد كائن `deprecation` في `model_spec` الخاص بها:

```json
{
  "id": "some-model-id",
  "model_spec": {
    "name": "Some Model",
    "privacy": "private",
    "deprecation": {
      "date": "2025-03-01T00:00:00.000Z"
    }
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

يظهر كائن `deprecation` فقط عندما يكون النموذج مُجدوَلًا للتقاعد. يمكنك التحقق `if (model.model_spec.deprecation)` لمعرفة ما إذا كان نموذج ما يتم تقاعده، واستخدام تاريخ ISO 8601 لتنبيه المستخدمين أو التخطيط للانتقال.

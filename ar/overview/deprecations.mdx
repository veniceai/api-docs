تأسس Venice API ليمنح المطوّرين وصولًا خاصًا وغير مقيَّد إلى نماذج بمستوى الإنتاج، خالية من فلاتر خفية أو قرارات مغلقة.

ومع تحسّن النماذج، نُوقف أحيانًا النماذج الأقدم لصالح بدائل أذكى أو أسرع أو أكثر قدرة. ونصمّم هذه الانتقالات لتكون قابلة للتنبؤ وبأقل قدر من الاحتكاك.

## إيقاف النماذج

نعلم أن قرارات الإيقاف قد تكون مزعجة. لذلك نسعى إلى الإيقاف فقط عند الضرورة، ونصمّم ميزات مثل الـ traits ونماذج Venice ذات العلامة لتقليل الإزعاج.

قد نُوقف نموذجًا عندما:

- يقدّم نموذج أحدث تحسينًا واضحًا في حالة الاستخدام نفسها
- لم يعد النموذج يلبّي معاييرنا للأداء أو الموثوقية
- يشهد استخدامًا منخفضًا باستمرار، ويؤدي استمرار دعمه إلى تجزئة التجربة على الجميع

## إجراءات الإيقاف

عند استيفاء نموذج لمعايير الإيقاف، نُعلن عن التغيير بمهلة من 30 إلى 60 يومًا. تُنشر إشعارات الإيقاف عبر [سجل التغييرات](https://featurebase.venice.ai/changelog) وعلى [خادم Discord الخاص بنا](https://discord.gg/askvenice). وعند استدعاء نموذج موقوف خلال فترة الإشعار، ستتضمن استجابة الـ API تحذيرًا بالإيقاف.

خلال فترة الإشعار يظل النموذج متاحًا، وإن كنا قد نُقلّل سعة البنية التحتية في بعض الحالات. ونوفّر دائمًا بديلًا موصى به، ونقدّم عند الحاجة إرشادات الترحيل لتسهيل الانتقال.

بعد تاريخ الإيقاف، يُوجَّه الطلب إلى النموذج تلقائيًا إلى نموذج بقوة معالجة مماثلة وبسعر مساوٍ أو أقل. وإذا لم يكن التوجيه ممكنًا لأسباب تقنية أو متعلقة بالسلامة، ستُعيد الـ API استجابة 410 Gone. وإذا كان النموذج الموقوف مُختارًا عبر trait (مثل `default_code` أو `default_vision` أو `fastest`)، فستُعاد تعيين هذا الـ trait إلى بديل متوافق.

لا نزيل النماذج بصمت أبدًا ولا نُغيّر السلوك دون إصدار جديد. ستعرف دائمًا ما يعمل وكيف تستعد للقادم.

<Note>
ترقيات الأداء فقط: قد نطرح تحسينات تحافظ على سلوك النموذج مع تحسين الأداء أو زمن الاستجابة أو كفاءة التكلفة. هذه التحديثات متوافقة مع الإصدارات السابقة ولا تتطلب أي إجراء من العميل.
</Note>

## كيف تُختار النماذج لـ Venice API

نختار بعناية النماذج المتاحة بناءً على الأداء والموثوقية واحتياجات المطوّرين الواقعية. ولكي يُدرَج نموذج، يجب أن يُظهر أداءً قويًا، ويتصرف بثبات تحت نقاط النهاية المتوافقة مع OpenAI، ويقدّم تحسينًا واضحًا على نموذج واحد على الأقل من النماذج المدعومة لدينا بالفعل.


النماذج التي نُقيّمها قد تُطلَق أولًا في [إصدار تجريبي](/ar/overview/beta-models) لجمع الملاحظات والتحقق من الأداء على نطاق واسع. 

لا نُتيح نماذج زائدة عن الحاجة أو غير مُجرَّبة أو غير جاهزة للاستخدام المستمر في الإنتاج. هدفنا أن يبقى Venice API نظيفًا وقادرًا ومُحسَّنًا لما يبنيه المطوّرون فعلًا.

اطّلع على المزيد في [إيقاف النماذج](/ar/overview/deprecations#model-deprecations) و<a href="/ar/overview/models" target="_blank" rel="noopener noreferrer">قائمة النماذج الحالية</a>.

## الإصدارات والأسماء البديلة

تُعرَّف جميع نماذج Venice بمعرّف فريد ودائم. على سبيل المثال:

```venice-uncensored```
```zai-org-glm-4.7```
```zai-org-glm-5```
```qwen3-vl-235b-a22b```

معرّفات النماذج ثابتة. إذا حدث تغيير كاسر للتوافق، فسنُصدر معرّف نموذج جديد (مثل إضافة إصدار v2). وإذا لم تكن هناك تغييرات كاسرة، فقد نُحدّث النموذج الحالي مع الإبلاغ عن التغييرات المهمة.

لتوفير المرونة، يحتفظ Venice أيضًا بأسماء بديلة رمزية، تُنفَّذ عبر الـ traits، وتشير إلى النموذج الافتراضي الموصى به لمهمة معيّنة:

<div id="traits-list-placeholder"></div>

تقدّم الـ traits تجريدًا مستقرًا لاختيار النماذج مع منح Venice المرونة لتحسين التنفيذ الكامن. ويمكن للمطوّرين الذين يفضّلون الوصول التلقائي إلى أحدث النماذج الموصى بها الاعتماد على الأسماء البديلة المعتمدة على الـ traits.

أما للتطبيقات التي تتطلب اتساقًا صارمًا وسلوكًا متوقعًا، فنوصي بالإشارة إلى معرّفات نماذج ثابتة.

## الملاحظات

يمكنك تقديم ملاحظاتك أو طلباتك عبر [بوابة Featurebase](https://featurebase.venice.ai). نحتفظ بـ [سجل تغييرات](https://featurebase.venice.ai/changelog) عام ومتتبّع لخارطة الطريق ومبرّرات شفافة لإضافة النماذج أو ترقيتها أو إزالتها، ونشجّع المشاركة المستمرة من المجتمع. 


## متتبّع إيقاف النماذج

النماذج التالية مجدولة للإيقاف أو أُوقفت مؤخرًا. نوصي بالترحيل إلى البدائل المقترحة قبل تاريخ الإزالة. تبقى النماذج مدرجة لمدة 30 يومًا بعد تاريخ إزالتها.

### التحقق من حالة الإيقاف عبر الـ API

يمكنك التحقق ممّا إذا كان نموذج ما مجدولًا للإيقاف باستدعاء نقطة النهاية [List Models](/ar/api-reference/endpoint/models/list). النماذج التي لها تاريخ إيقاف تتضمّن كائن `deprecation` ضمن `model_spec`:

```json
{
  "id": "some-model-id",
  "model_spec": {
    "name": "Some Model",
    "privacy": "private",
    "deprecation": {
      "date": "2025-03-01T00:00:00.000Z"
    }
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

يظهر كائن `deprecation` فقط عندما يكون النموذج مجدولًا للإيقاف. يمكنك التحقق عبر `if (model.model_spec.deprecation)` لمعرفة ما إذا كان النموذج قيد الإيقاف، واستخدام تاريخ ISO 8601 لتنبيه المستخدمين أو التخطيط للترحيل.

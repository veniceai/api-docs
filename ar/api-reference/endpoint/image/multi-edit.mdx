## فئات الجودة

تقبل بعض نماذج التعديل المتعدد معلمة `quality` اختيارية (`low`, `medium`, `high`) تُوازن بين الدقة البصرية والتكلفة. مدعومة حاليًا من `gpt-image-2-edit`؛ تتجاهل النماذج الأخرى المعلمة.

```json
{
  "modelId": "gpt-image-2-edit",
  "images": ["iVBORw0KGgo...", "iVBORw0KGgo..."],
  "prompt": "blend the two scenes into one cinematic frame",
  "resolution": "2K",
  "quality": "medium"
}
```

عند إغفال `quality`، يستخدم النموذج فئته الافتراضية (`high` لـ `gpt-image-2-edit`). توجد الأسعار لكل فئة (1K/2K/4K × low/medium/high) ضمن `model_spec.pricing.quality` في [نقطة نهاية النماذج](/ar/api-reference/endpoint/models/list) ومُدرجة في [نظرة عامة على التسعير](/ar/overview/pricing).

<Info>
التسعير الواعي بالجودة هو الافتراضي لجميع مستدعي API و SDK — لا حاجة إلى رأس اشتراك. تُحاسَب الطلبات التي تُغفل `quality` بفئة النموذج الافتراضية (`high` لـ `gpt-image-2-edit`).
</Info>

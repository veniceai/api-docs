## خيارات الأبعاد

تستخدم نماذج الصور معلمات أبعاد خاصة بالنموذج:

- النماذج المعتمدة على البكسل تقبل `width` و`height`، مثل `venice-sd35` و`qwen-image`.
- نماذج نسبة العرض إلى الارتفاع تقبل `aspect_ratio`، مثل `qwen-image-2`.
- نماذج فئات الدقة تقبل كلًا من `aspect_ratio` و`resolution`.

على سبيل المثال، تدعم `gpt-image-2` و`nano-banana-2` و`nano-banana-pro` قيم `resolution` التالية: `1K` و`2K` و`4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

ينطبق نمط الأبعاد المعتمد على فئات الدقة نفسه على نماذج Nano Banana:

```json
{
  "model": "nano-banana-pro",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

للنماذج التي تقبل نسبة العرض إلى الارتفاع ولكن لا تكشف عن فئات دقة، أرسل `aspect_ratio` بدون `resolution`:

```json
{
  "model": "qwen-image-2",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9"
}
```

للنماذج التي تستخدم أبعاد بكسل مباشرة، أرسل `width` و`height` بدلًا من ذلك:

```json
{
  "model": "venice-sd35",
  "prompt": "a serene canal in venice at sunset",
  "width": 1024,
  "height": 1024
}
```

راجع [نماذج الصور](/ar/models/image) أو [نقطة نهاية النماذج](/ar/api-reference/endpoint/models/list) لخيارات الأبعاد المدعومة من كل نموذج قبل الخلط بين حقول الأبعاد عبر النماذج.

## فئات الجودة

تقبل بعض النماذج معلمة `quality` اختيارية (`low`, `medium`, `high`) تُوازن بين الدقة البصرية والتكلفة. مدعومة حاليًا من `gpt-image-2`؛ تتجاهل النماذج الأخرى المعلمة.

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K",
  "quality": "medium"
}
```

عند إغفال `quality`، يستخدم النموذج فئته الافتراضية (`high` لـ `gpt-image-2`). يعتمد التسعير على دمج `resolution` و`quality` — الأسعار لكل فئة لـ `gpt-image-2` (1K/2K/4K × low/medium/high) مكشوفة ضمن `model_spec.pricing.quality` في [نقطة نهاية النماذج](/ar/api-reference/endpoint/models/list) ومُدرجة في [نظرة عامة على التسعير](/ar/overview/pricing).

<Info>
التسعير الواعي بالجودة هو الافتراضي لجميع مستدعي API و SDK — لا حاجة إلى رأس اشتراك. تُحاسَب الطلبات التي تُغفل `quality` بفئة النموذج الافتراضية (`high` لـ `gpt-image-2`).
</Info>

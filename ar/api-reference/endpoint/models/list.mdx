## تسعير فئات الجودة

لنماذج الصور التي تقبل معلمة `quality` الاختيارية (حاليًا `gpt-image-2` و`gpt-image-2-edit`)، تكشف الاستجابة عن مصفوفة أسعار لكل جودة ضمن `model_spec.pricing.quality`. كل مفتاح من المستوى الأعلى هو فئة دقة (`1K`, `2K`, `4K`) وكل مفتاح متداخل هو مستوى جودة (`low`, `medium`, `high`) يحمل سعر `usd` و`diem` خاصًا به:

```json
"pricing": {
  "resolutions": {
    "1K": { "usd": 0.27, "diem": 0.27 },
    "2K": { "usd": 0.51, "diem": 0.51 },
    "4K": { "usd": 0.84, "diem": 0.84 }
  },
  "quality": {
    "1K": {
      "low":    { "usd": 0.02, "diem": 0.02 },
      "medium": { "usd": 0.07, "diem": 0.07 },
      "high":   { "usd": 0.26, "diem": 0.26 }
    },
    "2K": {
      "low":    { "usd": 0.03, "diem": 0.03 },
      "medium": { "usd": 0.13, "diem": 0.13 },
      "high":   { "usd": 0.50, "diem": 0.50 }
    },
    "4K": {
      "low":    { "usd": 0.05, "diem": 0.05 },
      "medium": { "usd": 0.21, "diem": 0.21 },
      "high":   { "usd": 0.83, "diem": 0.83 }
    }
  }
}
```

`pricing.resolutions` هو الجدول الزمني القديم لكل صورة المُحتفَظ به للتوافق العكسي. `pricing.quality` هي مصفوفة لكل (resolution, quality) تنطبق متى ما كانت معلمة `quality` مدعومة. يبقى كلا الحقلين في الاستجابة حتى يتمكن العملاء من اكتشاف دعم الجودة وعرض المصفوفة في واجهاتهم.

أرسل طلب JSON-RPC 2.0 (مفرد أو دفعة) إلى عقدة بلوكشين مدعومة. تدعم نقطة النهاية كلًا من مصادقة مفتاح API ومصادقة محفظة x402. تتم الفوترة لكل رصيد ومقومة برصيد Venice الخاص بك — اعتماد واحد، فاتورة واحدة، لكل سلسلة أدناه.

## المصادقة

تدعم نقطة النهاية هذه طريقتين للمصادقة:

- **مفتاح API**: مصادقة Bearer token المعيارية عبر رأس `Authorization: Bearer <key>`.
- **محفظة x402**: الدفع حسب الاستخدام برصيد USDC من محفظة على Base أو Solana. لا يتطلب حساب Venice. راجع [دليل x402](/ar/guides/integrations/x402-venice-api) للإعداد.

تتشارك كلتا الطريقتين في حدود المعدل والفوترة نفسها (أرصدة Venice).

## أشكال الطلب

### طلب مفرد

```json
{
  "jsonrpc": "2.0",
  "method": "eth_chainId",
  "params": [],
  "id": 1
}
```

### طلب دُفعي

مصفوفة بحد أقصى **100** كائن JSON-RPC 2.0. يتم التحقق من صحة كل عنصر بشكل مستقل؛ إذا كانت أي طريقة غير مدعومة، تُرفض الدُفعة بأكملها بـ `400` ويُسرد كل اسم طريقة مخالف في رسالة الخطأ.

```json
[
  { "jsonrpc": "2.0", "method": "eth_chainId", "params": [], "id": 1 },
  { "jsonrpc": "2.0", "method": "eth_blockNumber", "params": [], "id": 2 }
]
```

## الطرق المدعومة وفئات التسعير

تُصنَّف الطرق في ثلاث فئات أرصدة. الأرصدة المُستهلَكة لكل استدعاء = `baseCredits[chain] × methodTier`.

| الفئة | المضاعف | أمثلة على الطرق |
|---|---|---|
| **Standard** | 1× | `eth_call`, `eth_getBalance`, `eth_blockNumber`, `eth_sendRawTransaction`, `eth_getLogs`, `eth_getTransactionReceipt`, `eth_estimateGas`, `net_version`, `web3_clientVersion`، طرق مُجمِّع ERC-4337 (`eth_sendUserOperation`, `eth_estimateUserOperationGas`، وغيرها)، امتدادات عائلة السلاسل (`zks_*`, `linea_*`, `bor_*`, `starknet_*`) |
| **Advanced** | 2× | `trace_block`, `trace_call`, `trace_transaction`, `debug_traceCall`, `debug_traceTransaction`, `debug_traceBlockByHash`, `txpool_inspect`, `txpool_status`, `arbtrace_*` |
| **Large** | 4× | `trace_replayBlockTransactions`, `trace_replayTransaction`, `txpool_content`, `arbtrace_replayTransaction`, `arbtrace_replayBlockTransactions` |

### الأرصدة الأساسية لكل سلسلة

| baseCredits | السلاسل |
|---|---|
| **20** | Ethereum + جميع EVM L2s أعلاه (Base, Optimism, Arbitrum, Polygon, Linea, Avalanche, BSC, Blast) و Starknet |
| **30** | zkSync Era |

### أمثلة على التكلفة

عند سعر Venice البالغ `~$6.25 × 10⁻⁷` لكل رصيد:

| الاستدعاء | الأرصدة | التكلفة بالدولار |
|---|---|---|
| `eth_call` على Ethereum (20 × 1×) | 20 | $0.0000125 |
| `trace_transaction` على Ethereum (20 × 2×) | 40 | $0.0000250 |
| `trace_replayTransaction` على Ethereum (20 × 4×) | 80 | $0.0000500 |
| `eth_call` على zkSync (30 × 1×) | 30 | $0.0000188 |

### غير مدعوم

- **الطرق التي تعمل عبر WebSocket فقط** (`eth_subscribe`, `eth_unsubscribe`) — هذا الوكيل HTTP فقط. استخدم الاستعلام (polling) بدلًا من ذلك، أو انتقل إلى مزود WebSocket مباشر.
- **طرق الفلتر ذات الحالة** (`eth_newFilter`, `eth_getFilterChanges`, `eth_getFilterLogs`, `eth_uninstallFilter`, `eth_newBlockFilter`, `eth_newPendingTransactionFilter`) — حالة الفلتر مُثبَّتة بخلفية upstream واحدة وتنكسر بصمت على وكيل HTTP مُوزَّع التحميل. استخدم `eth_getLogs` (عديم الحالة) بدلًا منها.
- **طرق التعدين / حمل المفاتيح** (`eth_sign`, `eth_accounts`, `eth_mining`, `eth_hashrate`, `eth_getWork`, `eth_submitWork`) — لا تحتفظ نقاط نهاية المزود المُستضاف بالمفاتيح الخاصة للمستخدم، لذا تُعيد هذه الطرق دومًا خطأ. وقّع المعاملات من جانب العميل وأرسلها عبر `eth_sendRawTransaction`.
- **الطرق غير المُعيَّنة** — أي شيء غير مُدرج صراحةً في القائمة المسموح بها يُعيد `400`. تواصل مع الدعم لطلب إضافات.

## فوترة دُفعة لكل عنصر

حتى عندما تكون استجابة HTTP `200`، يمكن لعناصر الدُفعة الفردية أن تعود بحقل `error` بصيغة JSON-RPC (مثل خطأ معلمات سيئة أو طريقة غير مدعومة على السلسلة المستهدفة). تفوتر Venice هذه العناصر بـ **5 أرصدة لكل عنصر** بدلًا من فئة الطريقة الكاملة — تنازل بسيط لأخطاء "استكشاف الـ API" العادية.

```json
// طلب دُفعة:
[
  { "jsonrpc": "2.0", "method": "eth_chainId",   "params": [],         "id": 1 },
  { "jsonrpc": "2.0", "method": "eth_getBalance","params": ["bad"],    "id": 2 }
]

// الاستجابة (HTTP 200, X-Venice-RPC-Credits: 25):
[
  { "jsonrpc": "2.0", "id": 1, "result": "0x1" },
  { "jsonrpc": "2.0", "id": 2, "error": { "code": -32602, "message": "invalid params" } }
]
```

يُحاسَب العنصر الأول (الناجح) بـ 20 رصيدًا، والثاني (خطأ على مستوى RPC) بـ 5، المجموع `= 25`.

## حدود المعدل

سقف الطلبات في الدقيقة لكل مُتصِل مُصدَّق:

| الفئة | الطلبات/الدقيقة |
|---|---|
| Standard | 100 |
| Staff | 1,000 |

عند تجاوز السقف، تُعيد نقطة النهاية `429` مع `customMessage` ورؤوس الاستجابة المعيارية `X-RateLimit-*`.

## إمكانية التكرار (Idempotency)

اضبط رأس الطلب `Idempotency-Key` على أي سلسلة تطابق `[A-Za-z0-9_-]{1,255}` لتمكين إعادة المحاولة الآمنة. يُخزَّن الرد مؤقتًا لمدة 24 ساعة باستخدام مفتاح `(user, idempotency-key)`:

- إعادة تشغيل **المفتاح نفسه مع النص نفسه** يُعيد الاستجابة المُخزَّنة مع رأس الاستجابة `Idempotent-Replayed: true`. لا يُلمَس المصدر العلوي ولا تُحسَب أرصدة جديدة.
- إعادة تشغيل **المفتاح نفسه مع نص مختلف** يُعيد `400` لمنع تلف الحالة الصامت. اختر مفتاحًا جديدًا للطلبات المميزة.

## رؤوس الاستجابة

| الرأس | الوصف |
|---|---|
| `X-Venice-RPC-Credits` | الأرصدة المُحاسَبة لهذا الطلب. في الطلبات الدُفعية، هذا هو المجموع عبر العناصر. |
| `X-Venice-RPC-Cost-USD` | التكلفة بالدولار حتى 8 خانات عشرية. تساوي `X-Venice-RPC-Credits × سعر لكل رصيد`. |
| `X-Request-ID` | معرّف ارتباط مكون من 32 حرفًا. أدرجه في أي تواصل مع الدعم. |
| `Idempotent-Replayed` | يظهر بالقيمة `"true"` عند ورود الاستجابة من ذاكرة الـ idempotency. |
| `X-RateLimit-Limit` / `X-RateLimit-Remaining` / `X-RateLimit-Reset` | تُضبط فقط على استجابات 429. |

## التسجيل الجنائي لتمرير المعاملات

كل استدعاء لـ `eth_sendRawTransaction` يُسجَّل من جانب الخادم بهاش المعاملة (keccak256 للبايتات الخام)، وslug الشبكة، ومعرّف الطلب، ومعرّف المستخدم المُتصِل. **لا** نحتفظ بالحمولة الموقَّعة نفسها — يمكن استرداد الهاش من إيصال السلسلة. يوجد هذا المسار التدقيقي حتى إذا تعرّض مفتاح API لعميل ما للاختراق واستُخدِم لتمرير معاملات غير مشروعة عبر بنيتنا التحتية، يمكننا ربط النشاط على السلسلة بالحساب المسؤول.

## مثال

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/ethereum-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_chainId",
    "params": [],
    "id": 1
  }'
```

```json
{ "jsonrpc": "2.0", "id": 1, "result": "0x1" }
```

رؤوس الاستجابة: `X-Venice-RPC-Credits: 20`, `X-Venice-RPC-Cost-USD: 0.00001250`, `X-Request-ID: <nanoid>`.

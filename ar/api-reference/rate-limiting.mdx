  -H "Authorization: Bearer $VENICE_API_KEY"

## معالجة الأخطاء

يجب إعادة محاولة الطلبات الفاشلة (500, 503, 429) باستخدام التراجع الأسي.

بالنسبة لأخطاء 429 تحديدًا، تحقق من رأس `x-ratelimit-reset-requests` للحصول على طابع Unix الزمني الدقيق لوقت إعادة المحاولة. تحتوي معظم مكتبات HTTP على آليات إعادة محاولة مدمجة تعالج ذلك تلقائيًا.

### الحماية من إساءة الاستخدام

إذا قمت بإنشاء أكثر من 20 طلبًا فاشلًا خلال 30 ثانية، ستحظر واجهة API الطلبات اللاحقة لمدة 30 ثانية:

```
Too many failed attempts (> 20) resulting in a non-success status code. Please wait 30s and try again.
```

## رؤوس الاستجابة

تتضمن كل استجابة هذه الرؤوس:

| الرأس | الوصف |
|:-------|:------------|
| `x-ratelimit-limit-requests` | الحد الأقصى للطلبات المسموح بها في النافذة الحالية |
| `x-ratelimit-remaining-requests` | الطلبات المتبقية في النافذة الحالية |
| `x-ratelimit-reset-requests` | طابع Unix الزمني عند إعادة تعيين النافذة |
| `x-ratelimit-limit-tokens` | الحد الأقصى للرموز المسموح بها في الدقيقة |
| `x-ratelimit-remaining-tokens` | الرموز المتبقية في الدقيقة الحالية |
| `x-ratelimit-reset-tokens` | الثواني حتى إعادة تعيين حد الرموز |

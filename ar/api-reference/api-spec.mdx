توفر واجهة برمجة تطبيقات Venice واجهات REST و streaming قائمة على HTTP لبناء تطبيقات الذكاء الاصطناعي بنماذج غير مُقيَّدة (uncensored) واستدلال خاص. يمكنك الإنشاء باستخدام توليد النص وإنشاء الصور والتضمينات والمزيد، كل ذلك دون سياسات محتوى مُقيِّدة. تتوفر أمثلة التكامل وحزم SDK في [الوثائق](/ar/overview/getting-started). يتوفر مرجع واجهة برمجة التطبيقات الخاص بنا أيضًا في شكل [مواصفات OpenAPI YAML.](https://api.venice.ai/doc/api/swagger.yaml)

## المصادقة

تستخدم واجهة برمجة تطبيقات Venice مفاتيح API للمصادقة. أنشئ مفاتيح API الخاصة بك وأدِرها في [إعدادات API](https://venice.ai/settings/api).


تتطلب جميع طلبات API مصادقة HTTP Bearer:

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
مفتاح API الخاص بك سرّي. لا تشاركه ولا تكشفه في أي كود من جانب العميل.
</Note>

## التوافق مع OpenAI

تطبّق واجهة برمجة تطبيقات Venice مواصفات واجهة OpenAI، مما يضمن التوافق مع عملاء وأدوات OpenAI الحاليين. هذا يسمح لك بالتكامل مع Venice باستخدام واجهة OpenAI المألوفة مع الوصول إلى ميزات Venice الفريدة والنماذج غير المُقيَّدة.

### الإعداد

اضبط عميلك لاستخدام عنوان URL الأساسي لـ Venice (`https://api.venice.ai/api/v1`) وقم بإجراء أول طلب لك:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## ميزات خاصة بـ Venice

### مُوجِّهات النظام (System Prompts)

توفر Venice مُوجِّهات نظام افتراضية مصممة لضمان استجابات نماذج طبيعية وغير مُقيَّدة. لديك خياران للتعامل مع مُوجِّهات النظام:

1. **السلوك الافتراضي**: تُلحَق مُوجِّهات النظام الخاصة بك بمُوجِّهات Venice الافتراضية
2. **السلوك المخصص**: تعطيل مُوجِّهات نظام Venice بالكامل

#### تعطيل مُوجِّهات نظام Venice

استخدم خيار `venice_parameters` لإزالة مُوجِّهات النظام الافتراضية في Venice:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### معلمات Venice

يتيح لك كائن `venice_parameters` الوصول إلى ميزات خاصة بـ Venice غير متوفرة في واجهة OpenAI القياسية:

| المعلمة | النوع | الوصف | الافتراضي |
|-----------|------|-------------|---------|
| `character_slug` | string | السلَق (slug) لشخصية Venice عامة (يمكن اكتشافه كـ "Public ID" في صفحة الشخصية المنشورة) | - |
| `strip_thinking_response` | boolean | إزالة كتل `<think></think>` من الاستجابة (للنماذج التي تستخدم تنسيق وسم `<think>` القديم). راجع [نماذج التفكير](/ar/guides/features/reasoning-models). | `false` |
| `disable_thinking` | boolean | في نماذج التفكير المدعومة، عطّل التفكير وأزل كتل `<think></think>` من الاستجابة | `false` |
| `enable_web_search` | string | تفعيل بحث الويب لهذا الطلب (`off`, `on`, `auto` - يفعّل auto بناءً على تقدير النموذج)<br/>تنطبق رسوم إضافية حسب الاستخدام، راجع [التسعير](/ar/overview/pricing#web-search-and-scraping). | `off` |
| `enable_web_scraping` | boolean | تفعيل استخراج محتوى الويب لما يصل إلى 5 عناوين URL مُكتشفة في رسالة المستخدم. يُعزز المحتوى المُستخرَج الاستجابات ويتجاوز بحث الويب. تُحتسَب فقط عناوين URL التي استُخرِجت بنجاح.<br/>تنطبق رسوم إضافية حسب الاستخدام، راجع [التسعير](/ar/overview/pricing#web-search-and-scraping). | `false` |
| `enable_x_search` | boolean | تفعيل بحث xAI الأصلي (الويب + X/Twitter) لنماذج Grok المدعومة (مثل `grok-4-20-beta`). يوفر نتائج بحث أعلى جودة باستخدام بنية بحث xAI. عند التفعيل، يتم تجاوز بحث الويب القياسي لـ Venice.<br/>تنطبق رسوم إضافية حسب الاستخدام، راجع [التسعير](/ar/overview/pricing#web-search-and-scraping). | `false` |
| `enable_web_citations` | boolean | عند تفعيل بحث الويب، اطلب من LLM الاستشهاد بمصادره بتنسيق `[REF]0[/REF]` | `false` |
| `include_search_results_in_stream` | boolean | تجريبي: تضمين نتائج البحث في البث كأول جزء يُبثّ | `false` |
| `return_search_results_as_documents` | boolean | إظهار نتائج البحث في استدعاء أداة متوافق مع OpenAI باسم `venice_web_search_documents` للتكامل مع LangChain | `false` |
| `include_venice_system_prompt` | boolean | ما إذا كان يجب تضمين مُوجِّهات نظام Venice الافتراضية إلى جانب مُوجِّهات النظام المحددة | `true` |

<Note>
يمكن أيضًا تحديد هذه المعلمات كلواحق نموذج تُلحَق باسم النموذج (مثل `zai-org-glm-5:enable_web_search=auto`). راجع [لواحق ميزات النموذج](/ar/api-reference/endpoint/chat/model_feature_suffix) للتفاصيل.
</Note>

### تخزين المُوجِّهات مؤقتًا (Prompt Caching)

تدعم Venice تخزين المُوجِّهات مؤقتًا على نماذج مختارة لتقليل الكمون والتكاليف للمحتوى المتكرر. بالنسبة للنماذج المدعومة، تقوم Venice تلقائيًا بتخزين مُوجِّهات النظام مؤقتًا — لا حاجة لتغييرات في الكود. يمكنك أيضًا تحديد المحتوى يدويًا للتخزين المؤقت باستخدام خاصية `cache_control` على محتوى الرسالة.

| المعلمة | النوع | الوصف |
|-----------|------|-------------|
| `prompt_cache_key` | string | تلميح توجيه اختياري لتحسين معدلات إصابة الذاكرة المؤقتة. عند توفيره، توجه Venice الطلبات إلى البنية الخلفية نفسها، مما يزيد من احتمال إصابات الذاكرة المؤقتة عبر المحادثات متعددة الأدوار. |

راجع [Prompt Caching](/ar/guides/features/prompt-caching) للتفاصيل حول كيفية عمل التخزين المؤقت والفوترة وأفضل الممارسات.

## مرجع رؤوس الاستجابة

تتضمن جميع استجابات Venice API رؤوس HTTP توفر بيانات وصفية حول الطلب، وحدود المعدل، ومعلومات النموذج، ورصيد الحساب. بالإضافة إلى رموز الأخطاء التي تُعاد من استجابات API، يمكنك فحص هذه الرؤوس للحصول على المعرف الفريد لطلب API معين، ومراقبة حدود المعدل، وتتبع رصيد حسابك.

توصي Venice بتسجيل معرفات الطلب (رأس `CF-RAY`) في عمليات النشر للإنتاج لاستكشاف الأخطاء بكفاءة أكبر مع فريق الدعم لدينا، إذا دعت الحاجة.

يقدم الجدول أدناه مرجعًا شاملًا لجميع الرؤوس التي قد تواجهها:

| الرأس | النوع | الغرض | متى يُعاد |
|--------|------|---------|---------------|
| **رؤوس HTTP القياسية** ||||
| `Content-Type` | string | نوع MIME لجسم الاستجابة (`application/json`, `text/csv`, `image/png`, إلخ) | دائمًا |
| `Content-Encoding` | string | الترميز المستخدم لضغط جسم الاستجابة (`gzip`, `br`) | عند إرسال العميل لرأس `Accept-Encoding` |
| `Content-Disposition` | string | كيف يجب عرض المحتوى (مثل `attachment; filename=export.csv`) | عند تنزيل الملفات أو التصديرات |
| `Date` | string | طابع زمني بتنسيق RFC 7231 عند إنشاء الاستجابة | دائمًا |
| **تعريف الطلب** ||||
| `CF-RAY` | string | معرّف فريد لطلب API هذا، يُستخدم لاستكشاف الأخطاء وطلبات الدعم | دائمًا |
| `x-venice-version` | string | الإصدار/المراجعة الحالية لخدمة Venice API (مثل `20250828.222653`) | دائمًا |
| `x-venice-timestamp` | string | طابع زمني للخادم عند معالجة الطلب (تنسيق ISO 8601) | عند تفعيل تتبع الطابع الزمني |
| `x-venice-host-name` | string | اسم مضيف الخادم الذي عالج الطلب | استجابات الأخطاء وسيناريوهات تصحيح الأخطاء |
| **معلومات النموذج** ||||
| `x-venice-model-id` | string | معرّف فريد لنموذج الذكاء الاصطناعي المستخدم للطلب (مثل `venice-01-lite`) | نقاط نهاية الاستدلال التي تستخدم نماذج الذكاء الاصطناعي |
| `x-venice-model-name` | string | الاسم الودي/المعروض لنموذج الذكاء الاصطناعي المستخدم (مثل `Venice Lite`) | نقاط نهاية الاستدلال التي تستخدم نماذج الذكاء الاصطناعي |
| `x-venice-model-router` | string | خدمة الموجِّه/الخلفية التي عالجت استدلال النموذج | نقاط نهاية الاستدلال عند توفر معلومات التوجيه |
| `x-venice-model-deprecation-warning` | string | رسالة تحذير للنماذج المُجدوَلة للإيقاف | عند استخدام نموذج مُهمَل |
| `x-venice-model-deprecation-date` | string | تاريخ إيقاف النموذج (تنسيق تاريخ ISO 8601) | عند استخدام نموذج مُهمَل |
| **معلومات حد المعدل** ||||
| `x-ratelimit-limit-requests` | number | الحد الأقصى لعدد الطلبات المسموح بها في النافذة الزمنية الحالية | جميع الطلبات المُصدَّقة |
| `x-ratelimit-remaining-requests` | number | عدد الطلبات المتبقية في النافذة الزمنية الحالية | جميع الطلبات المُصدَّقة |
| `x-ratelimit-reset-requests` | number | طابع Unix الزمني عند إعادة تعيين حد معدل الطلبات | جميع الطلبات المُصدَّقة |
| `x-ratelimit-limit-tokens` | number | الحد الأقصى للرموز (prompt + completion) المسموح بها في النافذة الزمنية | جميع الطلبات المُصدَّقة |
| `x-ratelimit-remaining-tokens` | number | عدد الرموز المتبقية في النافذة الزمنية الحالية | جميع الطلبات المُصدَّقة |
| `x-ratelimit-reset-tokens` | number | المدة بالثواني حتى إعادة تعيين حد معدل الرموز | جميع الطلبات المُصدَّقة |
| `x-ratelimit-type` | string | نوع حد المعدل المطبق (`user`, `api_key`, `global`) | عند فرض حد المعدل |
| **رؤوس الترقيم** ||||
| `x-pagination-limit` | number | عدد العناصر في كل صفحة | نقاط النهاية المُقسَّمة إلى صفحات |
| `x-pagination-page` | number | رقم الصفحة الحالية (يبدأ من 1) | نقاط النهاية المُقسَّمة إلى صفحات |
| `x-pagination-total` | number | إجمالي عدد العناصر عبر جميع الصفحات | نقاط النهاية المُقسَّمة إلى صفحات |
| `x-pagination-total-pages` | number | إجمالي عدد الصفحات | نقاط النهاية المُقسَّمة إلى صفحات |
| **معلومات رصيد الحساب** ||||
| `x-venice-balance-diem` | string | رصيد رمز DIEM الخاص بك قبل معالجة الطلب | جميع الطلبات المُصدَّقة |
| `x-venice-balance-usd` | string | رصيد الائتمان بالدولار الأمريكي قبل معالجة الطلب | جميع الطلبات المُصدَّقة |
| **رؤوس سلامة المحتوى** ||||
| `x-venice-is-blurred` | string | يشير إلى ما إذا كانت الصورة المُولَّدة قد تم تشويشها بسبب سياسات المحتوى (`true`/`false`) | توليد الصور مع تفعيل Safe Venice |
| `x-venice-is-content-violation` | string | يشير إلى ما إذا كان المحتوى ينتهك سياسات محتوى Venice (`true`/`false`) | نقاط نهاية توليد المحتوى |
| `x-venice-is-adult-model-content-violation` | string | يشير إلى ما إذا كان المحتوى ينتهك سياسات محتوى النموذج للبالغين (`true`/`false`) | نقاط نهاية توليد الصور |
| `x-venice-contains-minor` | string | يشير إلى ما إذا كانت الصورة تحتوي على قاصرين (`true`/`false`) | نقاط نهاية تحليل الصور مع الكشف عن العمر |
| **معلومات العميل** ||||
| `x-venice-middleface-version` | string | إصدار عميل Venice middleface | الطلبات من عملاء Venice middleface |
| `x-venice-mobile-version` | string | إصدار عميل تطبيق Venice للهاتف المحمول | الطلبات من تطبيقات الهاتف المحمول |
| `x-venice-request-timestamp-ms` | number | طابع زمني للطلب يوفره العميل بالميلي ثانية | عند توفير العميل لطابع زمني في الطلب |
| `x-venice-control-instance` | string | معرّف نسخة التحكم لتصحيح الأخطاء | نقاط نهاية توليد الصور لتصحيح الأخطاء |
| **رؤوس المصادقة** ||||
| `x-auth-refreshed` | string | يشير إلى أنه تم تحديث رمز المصادقة أثناء الطلب (`true`/`false`) | عند التحديث التلقائي لرموز المصادقة |
| `x-retry-count` | number | عدد محاولات إعادة المحاولة للطلب | عند حدوث إعادة محاولة للطلب |

### ملاحظات مهمة

- **حالة اسم الرأس**: رؤوس HTTP غير حساسة للحالة، لكن Venice تستخدم أحرفًا صغيرة مع شرطات للتناسق
- **القيم النصية**: تُعاد القيم المنطقية في الرؤوس كسلاسل نصية (`"true"` أو `"false"`)
- **القيم الرقمية**: قد تُعاد الأرقام الكبيرة وقيم الرصيد كسلاسل نصية لمنع فقدان الدقة
- **الرؤوس الاختيارية**: لا تُعاد جميع الرؤوس في كل استجابة؛ يعتمد وجودها على نقطة النهاية وسياق الطلب
- **الضغط**: استخدم `Accept-Encoding: gzip, br` في الطلبات لاستلام استجابات مضغوطة عند الدعم

### مثال: الوصول إلى رؤوس الاستجابة

```javascript
// بعد إجراء طلب API، يمكنك الوصول إلى الرؤوس من كائن الاستجابة
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// التحقق من تحذيرات إيقاف النموذج
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## أفضل الممارسات

1. **حدود المعدل**: راقب رأسي `x-ratelimit-remaining-requests` و `x-ratelimit-remaining-tokens` ونفّذ التراجع الأسي
2. **مراقبة الرصيد**: تتبع رأسي `x-venice-balance-usd` و `x-venice-balance-diem` لتجنب انقطاع الخدمة
3. **مُوجِّهات النظام**: اختبر مع وبدون مُوجِّهات نظام Venice للعثور على الأنسب لحالة استخدامك
4. **مفاتيح API**: حافظ على أمان مفاتيح API الخاصة بك وقم بتدويرها بانتظام
5. **تسجيل الطلبات**: سجّل قيم رأس `CF-RAY` لاستكشاف الأخطاء مع الدعم
6. **إيقاف النموذج**: تحقق من رؤوس `x-venice-model-deprecation-warning` عند استخدام النماذج

## الفروق عن واجهة OpenAI API

بينما تحافظ Venice على توافق عالٍ مع مواصفات OpenAI API، فثمة بعض الفروق الأساسية:

1. **venice_parameters**: تكوينات إضافية مثل `enable_web_search` و `character_slug` و `strip_thinking_response` لوظائف موسعة
2. **مُوجِّهات النظام**: تُلحِق Venice مُوجِّهات النظام الخاصة بك بالإعدادات الافتراضية المُحسَّنة لاستجابات غير مُقيَّدة (عطِّلها بـ `include_venice_system_prompt: false`)
3. **نظام النماذج**: تقدم Venice [قائمة النماذج](/ar/overview/models) الخاصة بها بما في ذلك نماذج غير مُقيَّدة ونماذج تفكير - استخدم معرّفات نماذج Venice بدلًا من تعيينات OpenAI
4. **رؤوس الاستجابة**: رؤوس فريدة لتتبع الرصيد (`x-venice-balance-usd`, `x-venice-balance-diem`)، وتحذيرات إيقاف النموذج، وعلامات سلامة المحتوى
5. **سياسات المحتوى**: سياسات أكثر تساهلًا مع نماذج غير مُقيَّدة مُخصصة وتصفية محتوى اختيارية

## استقرار API

تحافظ Venice على التوافق العكسي لنقاط نهاية ومعلمات v1. لسياسة دورة حياة النموذج، وإشعارات الإيقاف، وإرشادات الترحيل، راجع [Deprecations](/ar/overview/deprecations).

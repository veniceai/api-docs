تُفكّر بعض النماذج بصوت عالٍ قبل الإجابة. تعالج المشكلات خطوة بخطوة، ثم تقدم لك إجابة نهائية. هذا يجعلها أقوى في الرياضيات والشيفرة والمهام التي تتطلب منطقاً مكثفاً.

<div id="reasoning-models-placeholder"></div>

اطلع على القائمة الكاملة للنماذج والتسعير وحدود السياق على [صفحة النماذج](/ar/overview/models). لا تدعم كل نماذج الاستدلال المعامل [`reasoning_effort`](#reasoning-effort). راجع [دعم النماذج](#model-support) للتفاصيل.

## قراءة الإخراج

تُعيد نماذج الاستدلال تفكيرها في حقل منفصل اسمه `reasoning_content`، مما يحافظ على نظافة `content`:

<CodeGroup>
```python Python
response = client.chat.completions.create(
    model="zai-org-glm-5-1",
    messages=[{"role": "user", "content": "What is 15% of 240?"}]
)

thinking = response.choices[0].message.reasoning_content
answer = response.choices[0].message.content
```

```javascript Node.js
const response = await client.chat.completions.create({
    model: "zai-org-glm-5-1",
    messages: [{ role: "user", content: "What is 15% of 240?" }]
});

const thinking = response.choices[0].message.reasoning_content;
const answer = response.choices[0].message.content;
```

```bash cURL
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "zai-org-glm-5-1",
    "messages": [{"role": "user", "content": "What is 15% of 240?"}]
  }'
```
</CodeGroup>

<Info>
يُعيد بعض المزوّدين (Anthropic، Google، OpenAI، Qwen) رموز استدلال مشفّرة أو ملخّصة. عند حدوث ذلك، يحتوي `reasoning_content` على عنصر نائب `"[Some reasoning content is encrypted]"`.
</Info>

### البث

عند البث، يصل `reasoning_content` في الـ delta قبل الإجابة النهائية:

<CodeGroup>
```python Python
stream = client.chat.completions.create(
    model="zai-org-glm-5-1",
    messages=[{"role": "user", "content": "Explain photosynthesis"}],
    stream=True
)

for chunk in stream:
    if chunk.choices:
        delta = chunk.choices[0].delta
        if delta.reasoning_content:
            print(delta.reasoning_content, end="")
        if delta.content:
            print(delta.content, end="")
```

```javascript Node.js
const stream = await client.chat.completions.create({
    model: "zai-org-glm-5-1",
    messages: [{ role: "user", content: "Explain photosynthesis" }],
    stream: true
});

for await (const chunk of stream) {
    if (chunk.choices?.[0]?.delta) {
        const delta = chunk.choices[0].delta;
        if (delta.reasoning_content) process.stdout.write(delta.reasoning_content);
        if (delta.content) process.stdout.write(delta.content);
    }
}
```

```bash cURL
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "zai-org-glm-5-1",
    "messages": [{"role": "user", "content": "Explain photosynthesis"}],
    "stream": true
  }'
```
</CodeGroup>

## جهد الاستدلال

يتحكم المعامل `reasoning_effort` في مقدار التفكير الذي يقوم به النموذج قبل الاستجابة. الجهد الأعلى يعني استدلالاً أعمق ولكن مزيداً من الرموز وزمن الاستجابة.

### القيم المقبولة

| القيمة | الوصف |
|-------|-------------|
| `none` | تعطيل الاستدلال كلياً |
| `minimal` | استدلال أساسي بأقل جهد |
| `low` | استدلال خفيف للمشكلات البسيطة |
| `medium` | استدلال متوازن لتعقيد متوسط |
| `high` | استدلال عميق للمشكلات المعقدة |
| `xhigh` | عمق استدلال إضافي عالٍ |
| `max` | أقصى قدرة استدلالية |

<Warning>
لا تدعم كل النماذج كل القيم. **لا** تقوم Venice بالتعيين التلقائي لأقرب مستوى مدعوم. القيم غير المدعومة تُعيد خطأ 400 من المزوّد الأعلى. على سبيل المثال، إرسال `xhigh` إلى Claude أو `max` إلى GPT-5.2 سيفشل.

عند الشك، استخدم `low` أو `medium` أو `high`. هذه أكثر القيم دعماً.
</Warning>

### دعم النماذج

#### OpenAI

| النموذج | القيم المدعومة |
|-------|-----------------|
| GPT-5.2 | `none`, `low`, `medium`, `high`, `xhigh` |
| GPT-5.2 Codex, GPT-5.3 Codex | `low`, `medium`, `high`, `xhigh` |

#### Anthropic

| النموذج | القيم المدعومة |
|-------|-----------------|
| Claude Opus 4.6, Opus 4.6 Fast | `low`, `medium`, `high`, `max` |
| Claude Opus 4.5, Sonnet 4.5, Sonnet 4.6 | `low`, `medium`, `high` |

#### Google

| النموذج | القيم المدعومة |
|-------|-----------------|
| Gemini 3 Pro Preview | `low`, `high` |
| Gemini 3.1 Pro Preview | `low`, `medium`, `high` |
| Gemini 3 Flash Preview | `minimal`, `low`, `medium`, `high` |

#### xAI

نماذج Grok (Grok 4.1 Fast، Grok Code Fast) **لا** تدعم `reasoning_effort`. تحديده سينتج خطأً.

#### نماذج أخرى

| النموذج | القيم المدعومة |
|-------|-----------------|
| Qwen 3 235B A22B Thinking, Qwen 3.5 35B A3B | `low`, `medium`, `high` |
| Kimi K2.5 | `low`, `medium`, `high` |
| MiniMax M2.5, M2.1 | `low`, `medium`, `high` |
| سلسلة GLM 5.1 | استدلال مدمج فقط، غير قابل للتهيئة |
| DeepSeek R1 | استدلال مدمج فقط، غير قابل للتهيئة |

### الاستخدام

مرّر `reasoning_effort` كمعامل من المستوى الأعلى أو استخدم التنسيق المتداخل `reasoning.effort`:

<CodeGroup>
```python Python
response = client.chat.completions.create(
    model="minimax-m25",
    messages=[{"role": "user", "content": "Prove that there are infinitely many primes"}],
    extra_body={"reasoning": {"effort": "high"}}
)
```

```javascript Node.js
const response = await client.chat.completions.create({

    reasoning: { effort: "high" }

```

## تعطيل الاستدلال

هناك طريقتان لتعطيل الاستدلال:

| الطريقة | الصيغة | كيف تعمل |
|--------|--------|-------------|
| `reasoning.enabled: false` | `"reasoning": {"enabled": false}` | مفتاح على مستوى Venice يمنع إرسال معاملات الاستدلال إلى المزوّد. **موصى به.** |
| `reasoning.effort: "none"` | `"reasoning": {"effort": "none"}` | يُمرَّر إلى المزوّد الذي يقرر كيفية التعامل معه. يدعمه بعض النماذج فقط (مثل GPT-5.x). |

بالنسبة للنماذج التي تدعمه، فإن `reasoning.enabled: false` هو الخيار الأكثر موثوقية:

| النموذج | يمكن تعطيله؟ |
|-------|-------------|
| GPT-5.2 | نعم |
| GPT-5.2 Codex, GPT-5.3 Codex | نعم (ولكن `none` غير مدعوم كجهد) |
| Qwen 3 235B A22B Thinking, Qwen 3.5 35B A3B | نعم |
| سلسلة GLM 5.1 | نعم |
| Claude Opus 4.5/4.6/4.6 Fast, Sonnet 4.5/4.6 | لا (يستدل دائماً) |
| Gemini 3 Pro, 3.1 Pro, 3 Flash | لا (يستدل دائماً) |
| DeepSeek R1 | لا (يستدل دائماً) |

<CodeGroup>
```python Python
response = client.chat.completions.create(
    model="openai-gpt-52",
    messages=[{"role": "user", "content": "What's the capital of France?"}],
    extra_body={"reasoning": {"enabled": False}}
)
```

```javascript Node.js

    model: "openai-gpt-52",
    messages: [{ role: "user", content: "What's the capital of France?" }],
    reasoning: { enabled: false }

```

## حدود الرموز

تولّد نماذج الاستدلال رموز إجابة مرئية (في `content`) ورموز استدلال (في `reasoning_content`). كلاهما يُحتسبان ضمن ميزانية الرموز.

### تعيين سقف للرموز

استخدم `max_completion_tokens` لتحديد العدد الإجمالي للرموز التي يولّدها النموذج، بما في ذلك الاستدلال:

```json
{
  "model": "deepseek-v4-flash",
  "messages": [...],
  "max_completion_tokens": 500
}
```

كما يُقبل `max_tokens` ويتصرف بنفس الطريقة. إذا تم تعيين كليهما، فإن `max_completion_tokens` له الأسبقية.

للحصول على إخراج مرئي أكبر، ارفع السقف، أو خفّض `reasoning_effort`، أو [عطِّل الاستدلال](#disabling-reasoning).

### قراءة التفصيل

يعرض كائن `usage` كيف أُنفِقت ميزانيتك:

```json
"usage": {
  "completion_tokens": 501,
  "completion_tokens_details": { "reasoning_tokens": 169 },
  "prompt_tokens": 13,
  "total_tokens": 514
}
```

في هذا المثال، أُنفِق 169 رمزاً على الاستدلال و332 على الإجابة المرئية. عند الوصول إلى السقف، يكون `finish_reason` هو `length`.

الحد الأعلى لكل نموذج متاح كـ `maxCompletionTokens` على نقطة النهاية [`/v1/models`](/ar/api-reference/endpoint/models/list).

### النماذج غير الاستدلالية

يتصرف `max_tokens` و`max_completion_tokens` بنفس الطريقة على النماذج غير الاستدلالية، حيث يحدّان الإخراج المرئي مباشرة.

## اكتشاف القدرات

تحقق مما يدعمه النموذج عبر نقطة النهاية [`/v1/models`](/ar/api-reference/endpoint/models/list):

| الحقل | المعنى |
|-------|---------|
| `supportsReasoning` | النموذج يمتلك قدرة الاستدلال (chain-of-thought) |
| `supportsReasoningEffort` | النموذج يقبل المعامل `reasoning_effort` / `reasoning.effort` |

## أفضل الممارسات

- اجعل الإعداد الافتراضي هو `medium` للاستخدام العام
- استخدم `high` أو `xhigh` للمهام المعقدة (الرياضيات، الشيفرة، التحليل)
- استخدم `low` للتطبيقات الحساسة لزمن الاستجابة
- استخدم `reasoning.enabled: false` أو اضبط الجهد على `none` لتعطيل الاستدلال
- عند الشك، استخدم `low` أو `medium` أو `high`. هذه أكثر القيم دعماً

أدرجت Venice الآن المخرجات المنظّمة عبر "response\_format" كحقل متاح في واجهة برمجة التطبيقات. يتيح لك هذا الحقل توليد استجابات لطلباتك تتبع تنسيقاً محدّداً مسبقاً. مع هذه الطريقة الجديدة، تقل احتمالية أن تختلق النماذج مفاتيح أو قيماً غير صحيحة داخل الاستجابة، وهو ما كان أكثر شيوعاً عند محاولة ذلك عبر التلاعب بـ system prompt أو عبر استدعاء الوظائف (function calling).

يستخدم حقل المخرجات المنظّمة "response\_format" تنسيق OpenAI API، وهو موصوف بمزيد من التفصيل في دليل OpenAI [هنا](https://platform.openai.com/docs/guides/structured-outputs). كما نشرت OpenAI مقالاً تعريفياً حول استخدام المخرجات المنظّمة داخل واجهة برمجة التطبيقات تحديداً [هنا](https://openai.com/index/introducing-structured-outputs-in-the-api/). نظراً لأن هذه وظيفة متقدمة، توجد بعض "النقاط الحرجة" في أسفل هذه الصفحة ينبغي اتباعها.

هذه الوظيفة غير متاحة بشكل أصلي لجميع النماذج. يُرجى الرجوع إلى قسم النماذج [هنا](https://docs.venice.ai/api-reference/endpoint/models/list?playground=open)، والبحث عن "supportsResponseSchema" للنماذج المتوافقة.

```json
    {
      "id": "venice-uncensored",
      "type": "text",
      "object": "model",
      "created": 1726869022,
      "owned_by": "venice.ai",
      "model_spec": {
        "availableContextTokens": 32768,
        "capabilities": {
          "supportsFunctionCalling": true,
          "supportsResponseSchema": true,
          "supportsWebSearch": true
        },
```

### كيفية استخدام الاستجابات المنظّمة

لاستخدام "response\_format" بشكل صحيح، يمكنك تعريف مخططك (schema) بعدة "properties" تمثل فئات المخرجات، كل منها مع أنواع بيانات مهيّأة بشكل مستقل. يمكن تضمين هذه الكائنات (nesting) لإنشاء هياكل مخرجات أكثر تقدماً.

في ما يلي مثال على استدعاء API يستخدم response\_format لشرح العملية خطوة بخطوة لحل معادلة رياضية.

يمكنك أن ترى أن الـ properties تمت تهيئتها لتتطلب كلاً من "steps" و"final\_answer" داخل الاستجابة. ضمن التداخل، تتكون فئة steps من "explanation" و"output"، كل منهما كسلسلة (string).

```json
curl --request POST \
  --url https://api.venice.ai/api/v1/chat/completions \
  --header 'Authorization: Bearer <api-key>' \
  --header 'Content-Type: application/json' \
  --data '{
  "model": "venice-uncensored",
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful math tutor."
    },
    {
      "role": "user",
      "content": "solve 8x + 31 = 2"
    }
  ],
  "response_format": {
    "type": "json_schema",
    "json_schema": {
      "name": "math_response",
      "strict": true,
      "schema": {
        "type": "object",
        "properties": {
          "steps": {
            "type": "array",
            "items": {
              "type": "object",
              "properties": {
                "explanation": {
                  "type": "string"
                },
                "output": {
                  "type": "string"
                }
              },
              "required": ["explanation", "output"],
              "additionalProperties": false
            }
          },
          "final_answer": {
            "type": "string"
          }
        },
        "required": ["steps", "final_answer"],
        "additionalProperties": false
      }
    }
  }
}

```

هذه هي الاستجابة التي تم استلامها من النموذج. يمكنك أن ترى أن الهيكل اتبع المتطلبات بأن قدّم أولاً "steps" مع "explanation" و"output" لكل خطوة، ثم "final answer".

```json
{
  "steps": [
    {
      "explanation": "Subtract 31 from both sides to isolate the term with x.",
      "output": "8x + 31 - 31 = 2 - 31"
    },
    {
      "explanation": "This simplifies to 8x = -29.",
      "output": "8x = -29"
    },
    {
      "explanation": "Divide both sides by 8 to solve for x.",
      "output": "x = -29 / 8"
    }
  ],
  "final_answer": "x = -29 / 8"
}

```

على الرغم من أن هذا مثال بسيط، يمكن تعميمه على حالات استخدام أكثر تقدماً مثل: استخراج البيانات، وتمارين سلسلة الأفكار (Chain of Thought)، وتوليد واجهة المستخدم (UI Generation)، وتصنيف البيانات وغيرها الكثير.

### نقاط حرجة

في ما يلي بعض المتطلبات الرئيسية التي يجب مراعاتها عند استخدام المخرجات المنظّمة عبر response\_format:

* قد تستغرق الطلبات الأولية التي تستخدم response\_format وقتاً أطول لتوليد الاستجابة. لن تواجه الطلبات اللاحقة زمن الاستجابة نفسه الذي واجهه الطلب الأولي.
\u0001\u0001\u0001
* بالنسبة للاستعلامات الأكبر، قد يفشل النموذج في الإكمال إذا تم الوصول إلى `max_tokens` أو انتهت مهلة النموذج، أو إذا تم انتهاك أي حدود معدل (rate limits)

* تنسيق المخطط (schema) غير الصحيح سيؤدي إلى أخطاء عند الإكمال، وعادةً بسبب انتهاء المهلة

* رغم أن response\_format يضمن أن يُخرج النموذج بطريقة معينة، فإنه لا يضمن أن النموذج قدّم المعلومات الصحيحة في الداخل. المحتوى يحكمه الطلب وأداء النموذج.

* المخرجات المنظّمة عبر response\_format غير متوافقة مع استدعاءات الوظائف المتوازية

* مهم: يجب أن تتضمن جميع الحقول أو المعاملات وسم `required`. لجعل حقل اختيارياً، تحتاج إلى إضافة خيار `null` ضمن `type` للحقل، هكذا `"type": ["string", "null"]`&#x20;

* من الممكن جعل الحقول اختيارية بإعطاء خيار `null` ضمن حقل required للسماح باستجابة فارغة.

* مهم: يجب ضبط `additionalProperties` على false لكي يعمل response\_format بشكل صحيح

* مهم: يجب ضبط `strict` على true لكي يعمل response\_format بشكل صحيح

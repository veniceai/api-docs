تتيح لك مدخلات الملفات إرفاق المستندات وملفات المصدر مباشرةً بطلب `/chat/completions`. تقوم Venice باستخراج الملف إلى نص قبل إرساله إلى النموذج المختار، بحيث يمكنك طرح الأسئلة أو التلخيص أو المقارنة أو تحويل محتوى الملف دون الحاجة إلى بناء محلِّل خاص بك أولاً.

استخدم مدخلات الملفات عندما يعتمد طلبك (prompt) على محتويات مستند أو جدول بيانات أو ملف markdown أو ملف JSON أو ملف شيفرة. وهي مدخلات تقتصر على نطاق الطلب، وليست تخزيناً دائماً للملفات، لذا أدرج الملف في كل طلب يحتاج إليه.

<Info>
تستخدم مدخلات الملفات مصفوفة محتوى الدردشة المتوافقة مع OpenAI. أضف كتلة محتوى بـ `type: "file"` ووفّر محتوى الملف في `file.file_data`.
</Info>

## أنواع الملفات المدعومة

تقبل واجهة برمجة تطبيقات الدردشة مدخلات الملفات إما كعناوين URL ببيانات base64 أو كعناوين URL متاحة للعموم.

الحد الأقصى لحجم الملف هو **25 ميغابايت لكل ملف**، يُقاس بعد فك ترميز عنوان URL بـ base64 أو بعد جلب عنوان URL.

| الفئة | الصِيَغ |
|----------|---------|
| المستندات | PDF, DOCX, PPTX |
| جداول البيانات | XLSX, XLS, CSV |
| النصوص والبيانات | TXT, Markdown, JSON |
| شيفرة المصدر | معظم ملفات الشيفرة الشائعة، بما في ذلك `.py`, `.js`, `.ts`, `.c`, `.cpp`, `.java`, `.go`, `.rs`, `.ps1`, `.sh`, `.yaml`, و `.sql` |

<Note>
تُستخرج الملفات إلى نص قبل الاستدلال. يُحتسب النص المستخرج ضمن سياق الإدخال الخاص بالنموذج، لذا اختر نموذجاً يتوفر فيه عدد كافٍ من `availableContextTokens` للملف بالإضافة إلى تعليماتك والإجابة المتوقعة.
</Note>

## الاستخدام الأساسي

أرسل مصفوفة `messages` يكون فيها `content` الخاص برسالة المستخدم مصفوفةً من كتل النص والملف:

<CodeGroup>
```python Python
import base64
import os
from pathlib import Path

from openai import OpenAI

client = OpenAI(
    api_key=os.environ["VENICE_API_KEY"],
    base_url="https://api.venice.ai/api/v1",
)

path = Path("q3-report.pdf")
file_data = "data:application/pdf;base64," + base64.b64encode(path.read_bytes()).decode("utf-8")

response = client.chat.completions.create(
    model="openai-gpt-55",
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "text",
                    "text": "Summarize this report in five bullets and list the main risks.",
                },
                {
                    "type": "file",
                    "file": {
                        "file_data": file_data,
                        "filename": "q3-report.pdf",
                    },
                },
            ],
        }
    ],
)

print(response.choices[0].message.content)
```

```javascript Node.js
import OpenAI from "openai";
import { readFile } from "node:fs/promises";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const pdf = await readFile("q3-report.pdf");
const fileData = `data:application/pdf;base64,${pdf.toString("base64")}`;

const response = await client.chat.completions.create({
  model: "openai-gpt-55",
  messages: [
    {
      role: "user",
      content: [
        {
          type: "text",
          text: "Summarize this report in five bullets and list the main risks.",
        },
        {
          type: "file",
          file: {
            file_data: fileData,
            filename: "q3-report.pdf",
          },
        },
      ],
    },
  ],
});

console.log(response.choices[0].message.content);
```

## عناوين URL للملفات

إذا كان الملف مستضافاً بالفعل على عنوان HTTP أو HTTPS عام، فمرّر عنوان URL في `file_data` بدلاً من ترميزه بـ base64:

```json
{
  "model": "openai-gpt-55",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "Identify the governing law, renewal terms, and termination rights in this agreement."
        },
        {
          "type": "file",
          "file": {
            "file_data": "https://example.com/contracts/vendor-agreement.pdf",
            "filename": "vendor-agreement.pdf"
          }
        }
      ]
    }
  ]
}
```

<Warning>
استخدم فقط عناوين URL العامة التي يمكن لـ Venice جلبها دون مصادقة. وللملفات الخاصة، أرسل عنوان URL ببيانات base64.
</Warning>

## ملفات متعددة

يمكنك تضمين أكثر من كتلة ملف في الرسالة نفسها. ضع تعليمة نصية قصيرة قبل الملفات حتى يعرف النموذج كيفية استخدامها.

```json
{
  "model": "openai-gpt-55",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "Compare these two policy drafts. Return the material differences and recommend which version is clearer."
        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,JVBERi0xLjQK...",
            "filename": "policy-v1.pdf"
          }
        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,JVBERi0xLjQK...",
            "filename": "policy-v2.pdf"
          }
        }
      ]
    }
  ]
}
```

للحصول على أفضل النتائج، سمِّ كل ملف بوضوح وأشر إلى هذه الأسماء في طلبك.

## عناوين URL ببيانات (Data URLs)

بالنسبة للملفات المحلية، رمِّز بايتات الملف بـ base64 وضع قبلها نوع MIME الصحيح:

| نوع الملف | بادئة عنوان URL للبيانات |
|-----------|-----------------|
| PDF | `data:application/pdf;base64,` |
| DOCX | `data:application/vnd.openxmlformats-officedocument.wordprocessingml.document;base64,` |
| PPTX | `data:application/vnd.openxmlformats-officedocument.presentationml.presentation;base64,` |
| XLSX | `data:application/vnd.openxmlformats-officedocument.spreadsheetml.sheet;base64,` |
| CSV | `data:text/csv;base64,` |
| Markdown | `data:text/markdown;base64,` |
| نص عادي | `data:text/plain;base64,` |
| JSON | `data:application/json;base64,` |

إذا كنت لا تعرف نوع MIME الدقيق، فاستخدم `application/octet-stream`. إن تضمين `filename` دقيق يساعد Venice على التعرف على الملف وعرضه.

## التعامل مع الملفات الكبيرة

نظراً لأن الملفات تصبح نص الطلب، فإن الملفات الكبيرة يمكن أن تزيد من زمن الاستجابة واستخدام الرموز (tokens) والتكلفة. ضع نافذة سياق النموذج في الاعتبار.

يجب أن يكون الملف الأصلي بحجم 25 ميغابايت أو أقل. يزيد ترميز base64 من حجم الطلب بنحو 33%، لذا فإن الملف القريب من حد 25 ميغابايت سينتج جسم طلب JSON أكبر.

أنماط جيدة للملفات الكبيرة:

- اطلب مهمة محددة بدلاً من طلب عام مثل "حلِّل كل شيء".
- ضمِّن فقط المستندات اللازمة للإجابة الحالية.
- استخدم نماذج ذات `availableContextTokens` أكبر للتقارير الطويلة أو قواعد الشيفرة.
- ضع المستندات الثابتة والمتكررة قبل أسئلة المستخدم الديناميكية إذا كنت تستخدم أيضاً [التخزين المؤقت للطلبات](/ar/guides/features/prompt-caching).
- استخدم `stream: true` عندما تتوقع استجابة طويلة.

## مدخلات الملفات مقابل محلّل النصوص

استخدم مدخلات ملفات الدردشة عندما تريد من النموذج أن يستدل على الملف فوراً.

استخدم [واجهة Text Parser API](/ar/api-reference/endpoint/augment/text-parser) عندما تريد استخراج النص أولاً، أو فحص عدد الرموز، أو تخزين النص المستخرج في نظامك الخاص، أو إرسال النص المستخرج نفسه إلى طلبات متعددة.

| الحاجة | استخدم |
|------|-----|
| سؤال نموذج عن مستند في طلب واحد | مدخل ملف الدردشة |
| استخراج النص دون استدلال النموذج | Text Parser API |
| فحص عدد الرموز المستخرجة قبل الطلب | Text Parser API |
| إعادة استخدام النص المستخرج عبر طلبات كثيرة | Text Parser API، ثم تضمين النص في الطلبات |

## أفضل الممارسات

- ضمِّن `filename` كلما أمكن، خاصةً عند إرسال ملفات متعددة.
- ضع التعليمة قبل كتل الملفات حتى يعرف النموذج المهمة قبل قراءة المحتوى المستخرج.
- استخدم عناوين URL العامة فقط للملفات التي يمكن جلبها دون ملفات تعريف ارتباط أو ترويسات أو حالة جلسة موقّعة.
- فضِّل عناوين URL ببيانات base64 للملفات الخاصة أو الملفات المولّدة داخل تطبيقك.
- اطرح أسئلة مركّزة وحدِّد تنسيق الإخراج الذي تريده.
- للاستخراج المنظّم، اجمع بين مدخلات الملفات و[الاستجابات المنظّمة](/ar/guides/features/structured-responses).

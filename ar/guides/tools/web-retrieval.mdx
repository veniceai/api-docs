يوفّر Venice واجهتي API منفصلتين لاكتشاف صفحات الويب واستخراج محتواها:

- **Web Search** تُعيد نتائج مُنظَّمة تتضمن العناوين وعناوين URL والمقتطفات والتواريخ.
- **Web Scrape** تحوّل صفحة عامة محددة إلى Markdown.

استخدمهما معًا عندما يحتاج تطبيقك إلى مواد مصدرية حديثة وقابلة للفحص قبل الاستدلال.

## البحث في الويب

أرسل استعلامًا إلى Search API:

```bash
curl https://api.venice.ai/api/v1/augment/search \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query":"latest developments in decentralized AI"}'
```

Brave هو المزود الافتراضي ويوفر عدم الاحتفاظ بالبيانات. Google متاح أيضًا مع استعلامات مجهولة الهوية يتم توجيهها عبر Venice. راجع [مرجع Web Search API](/api-reference/endpoint/augment/search) لاختيار المزود وللاطلاع على المخطط الكامل للاستجابة.

## استخراج صفحة

بعد اختيار نتيجة، أرسل عنوان URL العام الخاص بها إلى Scrape API:

```bash
curl https://api.venice.ai/api/v1/augment/scrape \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/article"}'
```

تُعيد نقطة النهاية محتوى الصفحة بصيغة Markdown. تطلب أولاً تمثيل Markdown الأصلي للموقع إن كان متاحًا، وإلا فتستخدم الاستخراج المعتمد على المتصفح.

## سير العمل من البحث إلى الاستخراج

<Steps>
  <Step title="البحث">
    أرسل استعلامًا مركّزًا واجمع النتائج المُنظَّمة.
  </Step>
  <Step title="اختيار المصادر">
    قم بفلترة النتائج حسب الصلة والنطاق والتاريخ قبل جلب محتوى الصفحة.
  </Step>
  <Step title="الاستخراج">
    استخرج Markdown فقط من الصفحات التي ينوي تطبيقك استخدامها.
  </Step>
  <Step title="استخدام المحتوى أو تخزينه">
    ضمّن Markdown في مطالبة، أو أنشئ تضمينات، أو خزّنه مؤقتًا، أو احتفظ بعناوين URL المصدرية للاستشهادات.
  </Step>
</Steps>

## البحث المستقل أم البحث عبر النموذج؟

استخدم واجهات API المستقلة عندما يحتاج تطبيقك إلى التحكم في اختيار المصادر أو إعادة استخدام المحتوى المسترجَع. فعّل البحث في الويب في طلب النموذج عندما تريد أن يتولى Venice عملية الاسترجاع كجزء من طلب استدلال واحد.

| الحاجة | الأسلوب |
|---|---|
| فحص النتائج أو فلترتها قبل الاستدلال | Standalone Web Search |
| استخراج عنوان URL محدد ومعروف | Standalone Web Scrape |
| تخزين المحتوى المسترجَع أو إعادة استخدامه | Standalone APIs |
| السماح للنموذج باسترجاع السياق الحالي تلقائيًا | Model web search |

<Warning>
Web Search وWeb Scrape واجهتا API تجريبيتان. قد تتغيّر أشكال طلباتهما واستجاباتهما دون إشعار مسبق. يقبل Web Scrape الصفحات المتاحة للعموم فقط، وقد يتم رفض المواقع التي تحجب الوصول الآلي.
</Warning>

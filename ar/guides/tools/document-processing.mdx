يستخرج Text Parser API النصوص من المستند دون تشغيل استدلال النموذج. استخدمه عندما يحتاج تطبيقك إلى فحص محتوى المستند أو تخزينه أو فهرسته أو إعادة استخدامه قبل إرساله إلى نموذج.

تشمل المدخلات المدعومة ملفات PDF وDOCX وPPTX وXLSX والنصوص العادية بحجم يصل إلى 25 ميغابايت.

## تحليل مستند

ارفع الملف بصيغة `multipart/form-data`:

```bash
curl https://api.venice.ai/api/v1/augment/text-parser \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "file=@document.pdf" \
  -F "response_format=json"
```

تُعيد صيغة الاستجابة الافتراضية `json` النص المُستخرج وعدد رموزه. استخدم `response_format=text` عندما تحتاج فقط إلى النص المُستخرج.

## Text Parser أم إدخال ملف الدردشة؟

كلتا الميزتين تستخرج محتوى المستند، لكنهما تخدمان سير عمل مختلفًا:

| الحاجة | الاستخدام |
|---|---|
| طرح سؤال على النموذج حول مستند فورًا | [Chat file input](/guides/features/file-inputs) |
| استخراج النص دون استدلال | Text Parser |
| التحقق من عدد الرموز قبل اختيار نموذج | Text Parser |
| تخزين المحتوى المُستخرج أو فهرسته أو إعادة استخدامه | Text Parser |
| استخدام نفس المستند عبر عدة مطالبات | Text Parser |

## خط الأنابيب النموذجي

<Steps>
  <Step title="تحليل الملف">
    ارفع المستند المصدر واطلب استجابة JSON.
  </Step>
  <Step title="فحص المخرجات">
    قارن عدد الرموز بنافذة سياق النموذج المستهدف. قلّص المستندات الكبيرة أو قسّمها قبل الاستدلال.
  </Step>
  <Step title="استخدام النص">
    أضِفه إلى مطالبة دردشة، أو أنشئ تضمينات للاسترجاع، أو احفظه في مخزن البيانات الخاص بك.
  </Step>
</Steps>

<Info>
يعمل التحليل في الذاكرة مع عدم الاحتفاظ بأي بيانات. يعالج Venice المستند المُحمَّل ثم يتخلص منه فورًا دون تخزين محتواه أو تسجيله.
</Info>

<Warning>
Text Parser API تجريبي. قد يتغيّر شكل الطلب والاستجابة دون إشعار مسبق.
</Warning>

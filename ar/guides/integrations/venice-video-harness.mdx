## ما هذا

معظم تكاملات Venice مجرد أغلفة رفيعة حول استدعاءات الـ API. أما Venice Video Harness فهو **الطبقة الأعلى** التي تقع بين وكيلك وVenice API:

- **قواعد التنسيق** في `CLAUDE.md`
- **كتيّبات قابلة لإعادة الاستخدام** في `.claude/commands/` (19 أمر سير عمل)
- **وكلاء متخصّصون** في `.claude/agents/` (art-director, prompt-engineer, cut-qa، وأكثر)
- **مهارات إنتاج Venice** في `.claude/skills/` (متوافقة مع صيغة [Agent Skills](/ar/guides/integrations/venice-skills))
- **طبقة تنفيذ TypeScript** في `src/`
- **سجل نماذج شامل** يغطي أكثر من 50 نموذجًا لفيديو وصور وصوت وموسيقى في Venice

مُصمَّم للمنتجين الذين ينتجون:

- مشاريع فيديو ثابتة الشخصيات (أي نوع، أي طول)
- سلاسل أو حملات بأسلوب بصري مثبَّت
- سير عمل من لوحات قصصية إلى فيديو
- محتوى سردي قصير وطويل
- تتابعات سينمائية ذات علامة، وعروض ترويجية وتشويقية
- سلاسل اجتماعية بشخصيات متكررة

## ما الذي يجعله مُحسَّنًا لـ Venice

- **موجّهات صور مضبوطة لنماذج صور Venice** مثل `seedream-v5-lite` و`nano-banana-pro` و`flux-2-pro/max` وغيرها
- **توليد ألواح بمرحلتين** مع تحسين Venice multi-edit لتصحيح الشخصيات
- **منطق توجيه النماذج** لمستويات الحركة والجو وثبات الشخصية
- **توليد فيديو واعٍ بالمراجع** يستخدم `elements` و`reference_image_urls` و`scene_image_urls` بشكل صحيح لكل نموذج
- **تكيّف الموجّه مع البيئة** للتعامل مع مشاهد النهار مقابل الليل
- **مسارات صوت أصيلة في Venice** لـ TTS (Kokoro، Qwen3، ElevenLabs)، وSFX، والموسيقى
- **تقدير التكلفة** قبل التوليد عبر `/video/quote` و`/audio/quote`
- **بناء المعاملات بوعي النموذج** يتخطى تلقائيًا المعاملات التي لا يدعمها النموذج الهدف

## افتراضات توجيه النماذج

افتراضات الـ harness حازمة لأن الاتساق هو الهدف. التوجيه الحالي (أبريل 2026):

**Seedance 2.0 R2V افتراضيًا. Kling O3 R2V كبديل لمشاهد 3+ شخصيات. Seedance 2.0 i2v للقطات التأسيسية.**

| الدور | النموذج الافتراضي | متى يُستخدم |
|---|---|---|
| لقطات شخصيات (1-2) | `seedance-2-0-reference-to-video` | افتراضي R2V بـ `reference_image_urls` المسطّحة ووسوم `@Image`، حتى 15 ث، صوت ستيريو أصلي |
| لقطات شخصيات (3+) | `kling-o3-standard-reference-to-video` | بديل تلقائي بـ `elements` المُهيكَلة لهوية متعددة الشخصيات |
| تأسيسية / مزاج / حركة | `seedance-2-0-image-to-video` | لا شخصيات؛ جودة سينمائية ملحمية، حتى 15 ث |

يمكن تجاوز ذلك لكل مشروع عبر `series.json → videoDefaults`. لاستهداف عائلة خارج Seedance (مثلًا حسابات لا تملك وصول Seedance)، اضبط `videoDefaults` إلى `kling-o3-standard-reference-to-video` و`veo3.1-fast-image-to-video`.

<Note>
**قاعدة الوجوه في Seedance:** يحظر Seedance 2.0 صور الإدخال التي تحتوي وجوهًا والتي لم يُنتجها `seedream-v5-lite` أو `seedream-v5-lite-edit`. يتعامل الـ harness مع هذا تلقائيًا بتوجيه العمل على الصور التي تحتوي شخصيات عبر Seedream وتشغيل بوابة قبل كل استدعاء Seedance.
</Note>

### الصور والصوت والموسيقى

- **الصور (22+ نموذجًا):** `nano-banana-pro/2`، `gpt-image-2`، `flux-2-pro/max`، `grok-imagine`، `qwen-image-2-pro`، `recraft-v4-pro`، `seedream-v4` / `v5-lite`، `lustify-sdxl/v7`، `wai-Illustrious`، وغيرها
- **التحرير المتعدد:** `qwen-edit`، `flux-2-max-edit`، `nano-banana-pro-edit`، `seedream-v5-lite-edit`، `gpt-image-2-edit`، وغيرها
- **TTS:** `tts-kokoro` (50+ صوتًا)، `tts-qwen3-0-6b/1-7b`، `elevenlabs-tts-v3`، `elevenlabs-tts-multilingual-v2`
- **الموسيقى:** `elevenlabs-music`، `minimax-music-v2`، `ace-step-15`، `stable-audio-25`
- **SFX:** `elevenlabs-sound-effects-v2`، `mmaudio-v2-text-to-audio`

## خطوط الإنتاج

### خط التوليد

فيديو سردي من البداية إلى النهاية (نص → لوحة قصصية → فيديو → صوت → تجميع):

```bash
npm run dev -- produce-episode -p output/my-series -e 1
```

التنفيذ المرجعي في `src/mini-drama/` يغطي:

- إدارة السلاسل / الشخصيات / الحلقات
- ورش كتابة النصوص بمساعدة LLM
- توليد لوحات قصصية بمرحلتين (توليد + تحسين multi-edit)
- ضمان جودة الألواح المعتمد على الرؤية
- توليد الفيديو بتسلسل الإطارات
- إنتاج صوتي متعدد الطبقات
- دمج الترجمات والتجميع النهائي

### خط التحرير

قُصّ وسائط موجودة (لقطات مولَّدة من Venice أو لقطات أولية حقيقية). **النص أولًا**: يقرأ الـ LLM ملفًا مضغوطًا `takes_packed.md` (~12KB لكل 40 دقيقة صوت) بدلًا من رمي الإطارات في السياق.

الخطوات الخمس:

<Steps>
  <Step title="التفريغ النصي">
    whisper.cpp المحلي يُنتج `*.words.json` لكل مصدر + `takes_packed.md`.
  </Step>
  <Step title="قراءة الحزمة">
    يصوغ الـ LLM استراتيجية القص من النص وحده.
  </Step>
  <Step title="التأكيد">
    يقترح الاستراتيجية وينتظر "نعم / راجع / إلغاء".
  </Step>
  <Step title="عرض الـ EDL">
    قائمة قص JSON → ffmpeg concat بتلاشيات صوت 30 مللي ثانية. مبدأ الأرشفة أولًا، فلا تُكتب فوق الأصول الأصلية أبدًا.
  </Step>
  <Step title="التقييم الذاتي">
    يُجري وكيل `cut-qa` 6 فحوصات برمجية عند كل حد قطع؛ بحد أقصى 3 تكرارات إصلاح.
  </Step>
</Steps>

تكتشف فحوصات `cut-qa` تراجع نسبة العرض إلى الارتفاع، وقفزات تجزئة الإطار داخل الكلمة، واقتطاع تعليق صوتي، وعدم استمرارية الإضاءة، وقمم صوتية فوق -6 dBFS، وتداخل التسميات مع نص داخل الإطار.

<Tip>
خط التحرير مستوحى من [browser-use/video-use](https://github.com/browser-use/video-use). الفكرة الجوهرية لديهم *"الـ LLM لا يشاهد الفيديو، بل يقرؤه"*، هي ما يجعل التحرير الموجَّه بالوكلاء يعمل دون الغرق في tokens رمي الإطارات.
</Tip>

## الأوامر والوكلاء والمهارات

يكشف الـ harness 19 أمر سير عمل و10 وكلاء متخصّصين و7 مهارات إنتاج. أبرزها:

| أمر سير العمل | الغرض |
|---|---|
| `new-series` | أنشئ سلسلة جديدة بجماليات مثبَّتة |
| `add-character` / `lock-character` | تثبيت الشخصية + الصوت |
| `workshop-episode` | كتابة حلقة تعاونية |
| `storyboard-episode` | لوحة قصصية لحلقة واحدة |
| `produce-episode` | الخط الكامل بأمر واحد |
| `generate-trailer` | خط عرض ترويجي كامل |
| `edit-footage` | خط تحرير بالنص أولًا لوسائط موجودة |
| `ingest-screenplay` | استيعاب سيناريو Fountain أو PDF |

| الوكيل المتخصّص | الدور |
|---|---|
| `art-director` | قرارات الجمالية واللوحة والإضاءة والتأليف |
| `prompt-engineer` | موجّهات صور Venice وثبات الشخصية |
| `storyboard-qa` | ضمان جودة الألواح للاستمرارية وفحوصات الشخصية |
| `cut-qa` | بوابة الجودة بعد التصيير (6 فحوصات لكل قطع، 3 تكرارات بحد أقصى) |
| `overlay-designer` | رسوميات حركية ذات علامة، وكلاء فرعيون متوازيون |
| `trailer-curator` | اختيار لقطات العرض الترويجي وقواعد منع الحرق |

| مهارة الإنتاج | الغرض |
|---|---|
| `venice-api` | استخدام Venice REST API وافتراضاتها |
| `venice-video-model-routing` | توجيه R2V أولًا، وأشجار قرار |
| `character-consistency` | إرشادات ثبات الشخصية في لقطات متعددة |
| `shot-composition` | إرشادات تأليف اللقطة والكاميرا |
| `screenplay-parsing` | سير عمل تحليل السيناريو |
| `video-editing` | فلسفة التحرير بالنص أولًا، صيغة EDL، حلقة cut-qa |

## دورة الذهاب والإياب مع NLE

بعد التصيير، صدّر الجدول الزمني المُجمَّع كـ XML لتدقيقه في محرّرك المفضّل. كل مقطع فيديو وحوار وSFX ومؤثر موسيقي يهبط على مساره الخاص.

```bash
mini-drama export-timeline -p output/<project> -e 1 --format fcpxml      # Final Cut Pro X
mini-drama export-timeline -p output/<project> -e 1 --format premiere    # Premiere Pro
mini-drama export-timeline -p output/<project> -e 1 --format davinci     # DaVinci Resolve
```

## الاستخدام البرمجي

يمكنك أيضًا استدعاء وحدات الـ harness مباشرة من TypeScript الخاص بك:

```typescript
import { VeniceClient } from './src/venice/client.js';
import { generateVideo, quoteVideo } from './src/venice/video.js';
import { listVideoModels } from './src/venice/models.js';

const client = new VeniceClient();

const quote = await quoteVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  duration: '8s',
  audio: true,
});
console.log(`Estimated cost: $${quote.quote}`);

const result = await generateVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  prompt: 'A slow dolly shot pushes forward...',
  duration: '8s',
  imageUrl: 'data:image/png;base64,...',
  audio: true,
  outputPath: 'output/shot-001.mp4',
});

const longModels = listVideoModels({ minDurationSec: 20 });
```

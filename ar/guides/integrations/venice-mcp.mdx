[خادم Venice MCP](https://github.com/veniceai/venice-mcp-server) هو الخادم الرسمي [لبروتوكول سياق النموذج (Model Context Protocol)](https://modelcontextprotocol.io/) في Venice. يكشف Venice API الكامل (الدردشة، والصور، والفيديو، والصوت، والموسيقى، وembeddings، وweb augment، والشخصيات) على هيئة **31 أداة** يمكن لأي وكيل متوافق مع MCP استدعاؤها.

## بداية سريعة

<Steps>
  <Step title="احصل على مفتاح Venice API">
    أنشئ مفتاحًا من [venice.ai/settings/api](https://venice.ai/settings/api). راجع [دليل مفتاح API](/ar/guides/getting-started/generating-api-key) للحصول على تعليمات تفصيلية.
  </Step>

  <Step title="أضِف Venice إلى إعدادات مضيف MCP">
    أضِف هذا إلى ملف إعدادات مضيف MCP الخاص بك:

    ```json
    {
      "mcpServers": {
        "venice": {
          "command": "npx",
          "args": ["-y", "@veniceai/mcp-server@0.2.0"],
          "env": { "VENICE_API_KEY": "<your-venice-api-key>" }
        }
      }
    }
    ```

    مسارات الإعداد الشائعة:

    | المضيف | المسار |
    |---|---|
    | Claude Desktop (macOS) | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Claude Desktop (Windows) | `%APPDATA%\Claude\claude_desktop_config.json` |
    | Cursor | `~/.cursor/mcp.json` |
    | LM Studio | `mcp.json` (من إعدادات MCP في التطبيق) |
  </Step>

  <Step title="أعِد تشغيل مضيف MCP">
    صار لدى وكيلك الآن دردشة، وصور، وفيديو، وموسيقى، وTTS، وASR، و25 أداة Venice أخرى متاحة.
  </Step>
</Steps>

<Note>
معظم مضيفات MCP لا تُمرّر إلا متغيرات البيئة المُدرَجة **صراحةً** في كتلة `env`. لا تُورَّث متغيرات البيئة على مستوى النظام. إذا رأيت أخطاء 402 مع وجود مفتاح API، فتحقق من أن `VENICE_API_KEY` موجود داخل `env` في إعداداتك.
</Note>

## ماذا تحصل عليه

**31 أداة** تغطّي كل أنماط Venice، و**3 موارد** (`venice://models`, `venice://styles`, `venice://voices`)، و**3 قوالب موجّهات**.

### الدردشة وembeddings

| الأداة | الوصف |
|---|---|
| `venice_chat` | إكمال دردشة متوافق مع OpenAI ضد كتالوج LLM الكامل في Venice. |
| `venice_responses` | Responses API متوافق مع OpenAI بدعم أدوات بدورة واحدة أو متعددة الأدوار. |
| `venice_embeddings` | احسب embeddings لمدخل نصي. |
| `venice_chat_with_character` | دردش مع شخصية Venice بواسطة الـ slug. |

### الصور

| الأداة | الوصف |
|---|---|
| `venice_image_generate` | ولّد صورة (Flux 2، وLustify SDXL، وAnime/WAI، وQwen Image، وGPT Image، وNano Banana Pro، وغيرها). |
| `venice_image_edit` | حرّر صورة عبر موجّه. |
| `venice_image_multi_edit` | حرّر عدة صور معًا بموجّه واحد. |
| `venice_image_upscale` | ارفع دقة صورة حتى 4×. |
| `venice_image_remove_bg` | أزل خلفية صورة. |
| `venice_image_styles` | اعرض قائمة الأنماط الجاهزة للصور. |

### الفيديو

| الأداة | الوصف |
|---|---|
| `venice_video_generate` | ضع توليد فيديو في الطابور (Sora 2، وVeo 3.1، وKling، وWan، وLTX 2، وSeedance، وRunway Gen-4، وغيرها). |
| `venice_video_status` | تحقق من حالة مهمة فيديو في الطابور. |
| `venice_video_complete` | علّم فيديو مكتمل بأنه تم تنزيله؛ يحذف الوسائط من الخادم. |
| `venice_video_transcriptions` | فرّغ نصًا من رابط فيديو YouTube. |
| `venice_video_quote` | احصل على تسعيرة قبل وضع المهمة في الطابور. |

### الصوت (TTS / ASR)

| الأداة | الوصف |
|---|---|
| `venice_tts` | تحويل نص إلى كلام بأصوات مُستنسَخة ووسوم انفعال. |
| `venice_asr` | فرّغ صوتًا من رابط. |
| `venice_voice_clone` | اعرض الأصوات المدمجة أو استنسخ صوتًا من عيّنة. |
| `venice_audio_quote` | احصل على تسعيرة لتوليد الموسيقى. |

### الموسيقى

| الأداة | الوصف |
|---|---|
| `venice_music_generate` | ضع توليد موسيقى في الطابور (`ace-step-15`, `elevenlabs-music`, `minimax-music-v2/v25/v26`, `stable-audio-25`, `mmaudio-v2`, `elevenlabs-sound-effects-v2`). |
| `venice_music_status` | تحقق من حالة مهمة موسيقى في الطابور. |
| `venice_music_complete` | علّم مهمة موسيقى مكتملة بأنها تم تنزيلها. |

### Web augment والكتالوج والكريبتو

| الأداة | الوصف |
|---|---|
| `venice_web_search` | ابحث على الويب (مدعوم بـ Firecrawl). |
| `venice_web_scrape` | استخرج رابطًا واحدًا إلى Markdown. |
| `venice_text_parser` | استخرج النص من PDF/DOCX/EPUB/PPTX/XLSX. |
| `venice_list_models` | اعرض كتالوج النماذج الحيّ مع الأسعار. |
| `venice_list_characters` | اعرض شخصيات Venice العامة. |
| `venice_crypto_rpc` | وكّل استدعاءات JSON-RPC إلى Base وEthereum وPolygon وArbitrum أو Optimism. |

### مساعدو محفظة x402

ذو صلة فقط إذا كنت تصادق بمحفظة عبر [x402](/ar/guides/integrations/x402-venice-api) بدلًا من مفتاح API.

| الأداة | الوصف |
|---|---|
| `venice_x402_balance` | تحقق من رصيد x402 المدفوع مسبقًا لعنوان محفظة EVM أو Solana. |
| `venice_x402_top_up_info` | اجلب متطلبات الشحن (الشبكة، وtoken الـ USDC، والمستقبل، والحد الأدنى للمبلغ). |
| `venice_x402_transactions` | اعرض أحدث معاملات شحن وخصم x402 لعنوان محفظة EVM أو Solana. |

## الإعداد

يُضبط الخادم بالكامل عبر متغيرات البيئة.

| متغير البيئة | الافتراضي | ملاحظات |
|---|---|---|
| `VENICE_API_KEY` | _(لا شيء)_ | مفتاح Venice API الخاص بك. أبسط إعداد. |
| `VENICE_DEFAULT_CHAT_MODEL` | `venice-uncensored` | |
| `VENICE_DEFAULT_IMAGE_MODEL` | `flux-2-pro` | |
| `VENICE_DEFAULT_TTS_MODEL` | `tts-kokoro` | |
| `VENICE_DEFAULT_ASR_MODEL` | `openai/whisper-large-v3` | |
| `VENICE_DISABLE_NSFW` | `0` | اضبطه على `1` لإزالة ملاحظات قدرات NSFW من أوصاف الأدوات. |
| `VENICE_HTTP_TIMEOUT_MS` | `60000` | |
| `VENICE_SIWX_TOKEN` | _(لا شيء)_ | رمز مصادقة وضع محفظة x402. راجع [x402 أدناه](#x402-wallet-mode). |

إذا تم ضبط `VENICE_API_KEY` و`VENICE_SIWX_TOKEN` معًا، يفوز مفتاح API.

## وضع محفظة x402

يدعم Venice المصادقة عبر [رمز Sign-In-With-X للمحفظة](/ar/guides/integrations/x402-venice-api) المدعوم برصيد USDC مدفوع مسبقًا على Base أو Solana، إلى جانب تدفّق مفتاح API العادي. لا حاجة لبريد إلكتروني أو هاتف أو KYC: محفظتك هي الهوية الوحيدة.

```json
{
  "mcpServers": {
    "venice": {
      "command": "npx",
      "args": ["-y", "@veniceai/mcp-server@0.2.0"],
      "env": { "VENICE_SIWX_TOKEN": "<base64 Sign-In-With-X payload>" }
    }
  }
}
```

يُمرّر خادم MCP قيمة `VENICE_SIWX_TOKEN` كرأس `X-Sign-In-With-X` في كل استدعاء Venice API. لا يرى الخادم أبدًا مفتاحك الخاص. يحدث توقيع المحفظة وتفويض شحن USDC داخل محفظتك أنت.

| التدفّق | ما يحدث |
|---|---|
| **إعداد لمرة واحدة** | وقّع رسالة Sign-In-With-X في محفظتك → ينتج رمز SIWX (JSON بـ base64). |
| **الشحن** | يُعيد `POST /api/v1/x402/top-up` استجابة 402 + متطلبات الدفع. وقّع دفعة USDC لأحد خيارات Base أو Solana المُعادة، أعد الإرسال، فيقيّد Venice رصيدك. |
| **كل استدعاء استدلال** | يرسل خادم MCP `X-Sign-In-With-X: <SIWX>`؛ ويخصم Venice من رصيدك المدفوع مسبقًا. |

الحد الأدنى للشحن **\$5 USD**. الحد الأدنى للرصيد لاستدعاء الاستدلال **\$0.10**. بمجرد الشحن، يصبح زمن الاستدعاء أقل من 100 مللي ثانية لأن التسوية تحدث خارج السلسلة في حساب رصيد سريع.

<Tip>
المحافظ المرتبطة بحساب Venice مع DIEM مرهون تستهلك من رصيد الرهن بدلًا من أرصدة USDC، لذا لا حاجة للشحن.
</Tip>

## الاستضافة الذاتية (Streamable HTTP)

لعمليات النشر الجماعية أو على مستوى مساحة العمل، شغّل خادم MCP عبر HTTP بدلًا من stdio:

```bash
docker run -p 3333:3333 \
  -e VENICE_API_KEY=<your-venice-api-key> \
  -e VENICE_MCP_AUTH_TOKEN=<choose-a-long-random-token> \
  ghcr.io/veniceai/venice-mcp-server:latest
```

الخادم متاح الآن على `http://localhost:3333/mcp`. على عملاء HTTP إرسال `Authorization: Bearer <VENICE_MCP_AUTH_TOKEN>`.

<Warning>
`/mcp` نقطة نهاية تنفيذ أدوات مدعومة بالاعتمادات: يستطيع المستدعون إنفاق مفتاح Venice API المضبوط أو رصيد x402. عندما يُربط وضع HTTP خارج الـ loopback، يفشل بدء التشغيل ما لم يُضبط `VENICE_MCP_AUTH_TOKEN`. للإنتاج، ثبّت إصدار حزمة npm صراحةً بدلًا من الاعتماد على `latest`.
</Warning>

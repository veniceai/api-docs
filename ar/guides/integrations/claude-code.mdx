[Claude Code](https://docs.anthropic.com/en/docs/claude-code) هي أداة CLI من Anthropic للبرمجة الوكيلة (agentic coding). يوضّح هذا الدليل كيفية تشغيله عبر Venice AI للوصول إلى Claude Opus 4.5/4.6 وSonnet 4.5/4.6 مع الدفع لكل رمز (token).

## لماذا تحتاج إلى موجّه (Router)

يتصل Claude Code مباشرة بـ Anthropic API افتراضياً. لاستخدامه مع Venice، تحتاج إلى [claude-code-router](https://github.com/musistudio/claude-code-router)، وهو وكيل محلي مفتوح المصدر:

<Steps>
  <Step title="يعترض" icon="hand">
    يلتقط طلبات Claude Code الصادرة قبل وصولها إلى Anthropic
  </Step>
  <Step title="يحوّل" icon="arrows-rotate">
    يحوّل تنسيق الطلب ويعيّن معرّفات النماذج (مثل `claude-opus-4-5`)
  </Step>
  <Step title="يعيد التوجيه" icon="route">
    يحوّل الطلبات إلى Venice على `api.venice.ai/api/v1/chat/completions`
  </Step>
</Steps>

## الإعداد

<Steps>
  <Step title="ثبّت Claude Code">
    إن لم تكن قد فعلت ذلك، ثبّت Claude Code CLI من Anthropic:

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="ثبّت الموجّه (Router)">
    ```bash
    npm install -g @musistudio/claude-code-router
    ```
  </Step>

  <Step title="احصل على مفتاح API الخاص بك">
    أنشئ مفتاحاً من [venice.ai/settings/api](https://venice.ai/settings/api). ستقوم بلصقه مباشرة في ملف الإعدادات في الخطوة التالية.
  </Step>

  <Step title="إنشاء ملف الإعدادات">
    أنشئ مجلد الإعدادات:

    ```bash
    mkdir -p ~/.claude-code-router
    ```

    ثم أنشئ `~/.claude-code-router/config.json` بمحررك المفضل:

    ```bash
    # Using nano
    nano ~/.claude-code-router/config.json

    # Or using VS Code
    code ~/.claude-code-router/config.json
    ```

    الصق التهيئة التالية:

    ```json
    {
      "APIKEY": "",
      "LOG": true,
      "LOG_LEVEL": "info",
      "API_TIMEOUT_MS": 600000,
      "HOST": "127.0.0.1",
      "Providers": [
        {
          "name": "venice",
          "api_base_url": "https://api.venice.ai/api/v1/chat/completions",
          "api_key": "your-venice-api-key-here",
          "models": [
            "claude-opus-4-5",
            "claude-sonnet-4-5",
            "claude-opus-4-6",
            "claude-opus-4-6-fast",
            "claude-sonnet-4-6"
          ],
          "transformer": {
            "use": ["anthropic"]
          }
        }
      ],
      "Router": {
        "default": "venice,claude-opus-4-5",
        "think": "venice,claude-opus-4-5",
        "background": "venice,claude-opus-4-5",
        "longContext": "venice,claude-opus-4-5",
        "longContextThreshold": 100000
      }
    }
    ```

    <Note>
    إذا عدّلت `config.json` أثناء تشغيل الموجّه، أعد تشغيله بـ `ccr restart` لتطبيق التغييرات.
    </Note>
  </Step>

  <Step title="التشغيل">
    شغّل الموجّه، ثم Claude Code:

    ```bash
    ccr start
    ccr code
    ```

    أو استخدم طريقة التفعيل:

    ```bash
    eval "$(ccr activate)" && claude
    ```
  </Step>
</Steps>

## النماذج المدعومة

| النموذج | معرّف Venice | الأنسب لـ |
|-------|-----------|----------|
| Claude Opus 4.5 | `claude-opus-4-5` | الاستدلال المعقد، عمليات إعادة الهيكلة الكبيرة |
| Claude Sonnet 4.5 | `claude-sonnet-4-5` | التكرار السريع، البرمجة اليومية |
| Claude Opus 4.6 | `claude-opus-4-6` | الاستدلال المعقد، عمليات إعادة الهيكلة الكبيرة |
| Claude Opus 4.6 Fast | `claude-opus-4-6-fast` | الاستدلال المعقد مع زمن استجابة أقل |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | التكرار السريع، البرمجة اليومية |

<Info>
Claude Code محسَّن لنماذج Claude. على الرغم من أن النماذج الأخرى المتاحة عبر Venice (GPT، DeepSeek، Grok، إلخ) قد تعمل، لا يمكننا ضمان تجربة مكافئة لأن Claude Code يعتمد على ميزات خاصة بـ Claude مثل التفكير الموسّع. للنماذج الأخرى، فكّر في استخدام [API القياسي](/ar/api-reference/endpoint/chat/completions) من Venice.
</Info>

## ميزات الموجّه

يوفر الموجّه عدة ميزات مفيدة إلى جانب التوجيه الأساسي:

<AccordionGroup>
  <Accordion title="بدّل النماذج فوراً">
    استخدم أمر `/model` داخل Claude Code للتبديل بين النماذج دون إعادة تشغيل:
    
    ```
    /model venice,claude-sonnet-4-5
    ```
    
    مفيد عندما تريد Opus للمهام المعقدة وSonnet للتكرارات السريعة.
  </Accordion>

  <Accordion title="تهيئة بصرية مع وضع UI">
    تفضّل واجهة رسومية؟ شغّل محرر الإعدادات المعتمد على الويب:
    
    ```bash
    ccr ui
    ```
    
    يفتح هذا واجهة متصفح لتحرير `config.json` دون لمس الملف مباشرة.
  </Accordion>

  <Accordion title="شرح سيناريوهات الموجّه">
    يتحكم قسم تهيئة `Router` في النموذج الذي يتعامل مع أنواع المهام المختلفة:
    
    | السيناريو | متى يُستخدم |
    |----------|----------------|
    | `default` | الطلبات العامة |
    | `think` | المهام الكثيفة الاستدلال (وضع التخطيط) |
    | `background` | العمليات الخلفية |
    | `longContext` | عندما يتجاوز السياق عتبة `longContextThreshold` بالرموز |
    
    يمكنك توجيه سيناريوهات مختلفة إلى نماذج مختلفة. على سبيل المثال، استخدم Sonnet للمهام الخلفية لتوفير التكاليف.
  </Accordion>

  <Accordion title="التصحيح بالسجلات">
    إذا لم يعمل شيء ما، تحقق من السجلات:
    
    ```bash
    # Server logs (HTTP, API calls)
    ~/.claude-code-router/logs/ccr-*.log
    
    # Application logs (routing decisions)
    ~/.claude-code-router/claude-code-router.log
    ```
    
    اضبط `"LOG_LEVEL": "debug"` في تهيئتك للحصول على مخرجات أكثر تفصيلاً.
  </Accordion>
</AccordionGroup>

## سلوك التخزين المؤقت

يعمل [التخزين المؤقت للطلبات](/ar/guides/features/prompt-caching) في Venice جنباً إلى جنب مع علامات التخزين المؤقت الأصلية لـ Claude Code. تكتشف Venice تلقائياً عندما يرسل Claude Code حقول `cache_control` وتعدّل استراتيجية التخزين المؤقت وفقاً لذلك.

| السيناريو | عمر التخزين (TTL) | المتحكم |
|----------|-----------|--------------|
| افتراضي (موصى به) | 5 دقائق | Claude Code + Venice |
| مع محوّل `cleancache` | 1 ساعة | Venice فقط |

<AccordionGroup>
  <Accordion title="متى لا تستخدم cleancache (معظم المستخدمين)">
    تتيح التهيئة الافتراضية للنظامين التعاون:
    
    - يرسل Claude Code علامات `cache_control` الأصلية الخاصة به
    - تضيف Venice التخزين المؤقت حولها بعمر 5 دقائق
    - يتقاسم النظامان حد التخزين المؤقت لـ 4 كتل
    
    يعمل هذا جيداً لجلسات البرمجة النشطة حيث تقوم بطلبات متكررة.
  </Accordion>

  <Accordion title="متى تستخدم cleancache">
    أضف `cleancache` إلى المحوّل إذا كنت:
    
    - تواجه أخطاء حد التخزين المؤقت لـ 4 كتل
    - تواجه سلوكاً غريباً للتخزين المؤقت
    - تفضّل عمر Venice البالغ ساعة واحدة للجلسات الأطول
    
    ```json
    "transformer": {
      "use": ["anthropic", "cleancache"]
    }
    ```
    
    هذا يزيل علامات التخزين المؤقت لـ Claude Code، مما يمنح Venice التحكم الكامل بعمر أطول.
  </Accordion>
</AccordionGroup>

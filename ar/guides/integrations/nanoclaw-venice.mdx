[NanoClaw](https://github.com/qwibitai/nanoclaw) هو مساعد ذكاء اصطناعي خفيف الوزن ذاتي الاستضافة يعمل على WhatsApp وTelegram. تضيف هذه النسخة الفرعية (fork) دعم Venice AI بحيث يعمل كل شيء بشكل خاص دون اشتراك Anthropic.

## لماذا Venice AI؟

[Venice](https://venice.ai) منصة ذكاء اصطناعي تعطي الأولوية للخصوصية. [لا تخزّن أو تسجّل أي طلبات أو استجابات](https://venice.ai/privacy) على خوادمها — محادثاتك موجودة فقط على جهازك. الطلبات مشفّرة من طرف إلى طرف عبر وكيلها إلى مزوّدي GPU لامركزيين، مع عدم الاحتفاظ بالبيانات. هذا يعني أن محادثات مساعد الذكاء الاصطناعي لديك تبقى خاصة، حتى من Venice نفسها.

توفر Venice وصولاً مجهّلاً إلى نماذج الحدود (Claude Opus وClaude Sonnet) ووصولاً خاصاً بالكامل إلى النماذج مفتوحة المصدر (GLM وQwen) عبر API واحدة — بدّل بينها في أي وقت.

| | **Venice AI** | **مزوّدو الذكاء الاصطناعي التقليديون** |
|---|---|---|
| **الاحتفاظ بالبيانات** | لا شيء — صفر سجلات | نعم |
| **خصوصية الطلب** | مشفّر، لا يُخزَّن أبداً | يُخزَّن على خوادم المزوّد |
| **النماذج مفتوحة المصدر** | نعم (GLM وQwen وغيرها) | لا |
| **نماذج الحدود** | Claude وGPT وغيرها — بشكل مجهّل | فقط عبر الاشتراكات المباشرة |
| **التسعير** | الدفع لكل رمز، بدون اشتراك. أو ارهن [DIEM](https://venice.ai/lp/diem) للحصول على أرصدة يومية متجددة | اشتراكات $20–200/شهر أو API بالدفع لكل رمز |
| **استدلال بدون رقابة** | نعم (النماذج مفتوحة المصدر) | لا |

## لماذا NanoClaw؟

NanoClaw بديل نظيف وبسيط للمنصات الأكبر مثل OpenClaw. مصمم لشخص واحد يشغل بوتاً واحداً.

| | **NanoClaw (Venice)** | **OpenClaw** |
|---|---|---|
| **قاعدة الشيفرة** | ~2,000 سطر، عدد قليل من الملفات | ~500,000 سطر، 53 ملف تهيئة |
| **التبعيات** | ~15 حزمة | أكثر من 70 حزمة |
| **نموذج الأمان** | عزل حاوية Docker على مستوى نظام التشغيل | قوائم سماح على مستوى التطبيق ورموز اقتران |
| **عزل لكل مجموعة** | كل مجموعة تحصل على حاويتها الخاصة ونظام ملفاتها وذاكرتها | عملية مشتركة، ذاكرة مشتركة |
| **الإعداد** | معالج واحد (`/setup`)، ~10 دقائق | تهيئة يدوية متعددة الخطوات |
| **مزوّد الذكاء الاصطناعي** | Venice AI (خاص، بدون اشتراك) | Anthropic (يتطلب مفتاح API أو اشتراكاً) |
| **التخصيص** | عدّل الشيفرة مباشرة — صغيرة بما يكفي للقراءة | ملفات تهيئة وإضافات |
| **المستخدم المستهدف** | شخص واحد، بوت واحد | منصة متعددة المستخدمين |

## ما تحصل عليه

- مساعد ذكاء اصطناعي شخصي على **Telegram** و/أو **WhatsApp**
- مدعوم بـ **Venice AI** — لا حاجة إلى حساب Anthropic
- يعمل البوت في **حاوية Docker معزولة** (في صندوق رمل، لا يمكنها الوصول إلى نظامك)
- **التبديل بين النماذج** — أخبر البوت "switch to zai-org-glm-5" أو "use opus" في أي وقت
- **مهام مجدولة** — اضبط تذكيرات ومهام متكررة
- **البحث على الويب والتصفح** مدمج
- **تنسيق Markdown** في رسائل Telegram

**لـ Telegram** (موصى به للمستخدمين لأول مرة):
1. افتح Telegram وابحث عن **@BotFather**
2. أرسل `/newbot` واتبع التعليمات
3. احفظ الرمز الذي يعطيك إياه BotFather (يبدو مثل `123456789:ABCdef...`)

<Warning>
**لـ WhatsApp — استخدم رقماً افتراضياً، وليس رقمك الشخصي:**

يتصل NanoClaw كجهاز مرتبط برقم WhatsApp الخاص بك. هذا يعني أن **الوكيل يستطيع رؤية كل رسالة واردة وصادرة** — جميع محادثاتك الشخصية ودردشات المجموعات والصور وكل شيء. لا يزال هاتفك يعمل بشكل طبيعي، لكن البوت لديه رؤية كاملة لحساب WhatsApp بأكمله.

**استخدم رقم هاتف افتراضي بدلاً من ذلك.** تمنحك هذه التطبيقات رقماً ثانياً يمكنك تخصيصه بالكامل للبوت:

| التطبيق | السعر | ملاحظات |
|-----|-------|-------|
| [Hushed](https://hushed.com) | ~$5/شهر | موثوق، يعمل جيداً للتحقق من WhatsApp |
| [Burner](https://www.burnerapp.com) | ~$5/شهر | مشابه لـ Hushed، أرقام للاستخدام لمرة واحدة |
| [Google Voice](https://voice.google.com) | مجاني | متاح في الولايات المتحدة فقط، قد لا يعمل للتحقق من WhatsApp في جميع الحالات |

**كيفية إعداده:**
1. احصل على رقم افتراضي من أحد التطبيقات أعلاه
2. ثبّت WhatsApp على جهاز ثانٍ (هاتف قديم أو جهاز لوحي أو محاكي) باستخدام ذلك الرقم الافتراضي
3. أثناء إعداد NanoClaw، امسح رمز QR بذلك الجهاز الثاني — وليس هاتفك الشخصي
</Warning>

## الإعداد

يستغرق الإعداد حوالي 10 دقائق. تحتاج فقط إلى **نافذة Terminal واحدة**.

<Steps>
  <Step title="استنساخ وتثبيت">
    افتح Terminal وشغّل:

    ```bash
    git clone https://github.com/lorenzovenice/nanoclaw-venice.git
    cd nanoclaw-venice
    npm install
    ```

    انتظر حتى ينتهي `npm install` بدون أخطاء.
  </Step>

  <Step title="تشغيل Claude Code مع Venice">
    استبدل `your-key` بمفتاح Venice API الخاص بك وشغّل:

    ```bash
    VENICE_API_KEY=your-key npm run venice
    ```

    يبدأ هذا وكيل Venice ويشغّل Claude Code من خلاله في أمر واحد.

    <Note>
    يستخدم Claude Code افتراضياً **GLM 5** (`zai-org-glm-5`) للحفاظ على تكاليف الإعداد منخفضة. بعد الإعداد، اكتب `/model` داخل Claude Code للتبديل إلى `claude-sonnet-4-6` أو `claude-opus-4-6` للحصول على أفضل أداء.
    </Note>

    إذا طُلب منك "Do you want to use this API key?" — اختر **Yes**.
  </Step>

  <Step title="تشغيل معالج الإعداد">
    في طرفية Claude Code، اكتب:

    ```
    /setup
    ```

    يأخذك المعالج خطوة بخطوة عبر:

    1. **Bootstrap** — يتحقق من Node.js والتبعيات
    2. **مفتاح Venice API** — يتحقق منه ويحفظه
    3. **اختيار القناة** — اختر WhatsApp أو Telegram أو كليهما
    4. **بناء الحاوية** — يبني حاوية Docker (تستغرق بضع دقائق في المرة الأولى)
    5. **مصادقة WhatsApp** — امسح رمز QR بهاتفك (إن وجد)
    6. **إعداد Telegram** — أرسل رسالة إلى بوتك ليكتشف دردشتك
    7. **كلمة التشغيل** — البادئة التي تنشّط البوت (الافتراضي: `@Andy`)
    8. **مجلدات التحميل** — اختر "No" الآن (يمكنك إضافة وصول الملفات لاحقاً)
    9. **بدء الخدمات** — يبدأ كل من NanoClaw ووكيل Venice كخدمات في الخلفية

    يثبّت معالج الإعداد خدمتين في الخلفية:
    - **NanoClaw** — البوت نفسه
    - **Venice proxy** — خادم محلي صغير (localhost:4001) يترجم بين Claude Code وVenice AI

    يبدآن تلقائياً عند الإقلاع ويعيدان تشغيل نفسيهما إذا تعطلا.

    <Note>
    إذا توقف المعالج بين الخطوات، اكتب "continue" أو "next step" لدفعه إلى الأمام.
    </Note>
  </Step>

  <Step title="ابدأ الدردشة">
    بمجرد اكتمال الإعداد، افتح دردشتك (Telegram أو WhatsApp) وأرسل:

    ```
    @Andy hello, are you there?
    ```

    يجب أن يستجيب البوت في غضون ثوانٍ. في قناتك الرئيسية، يمكنك الكتابة بشكل طبيعي دون بادئة `@Andy`.

    **يمكنك الآن إغلاق نافذة الطرفية.** كل شيء يعمل كخدمات في الخلفية ويبدأ تلقائياً عند إقلاع حاسوبك.
  </Step>
</Steps>

## كيف يعمل

هناك طبقتان لـ NanoClaw:

| الطبقة | ما تفعله |
|-------|-------------|
| **Claude Code CLI** | أداة إدارة للإعداد والتصحيح والتخصيص |
| **البوت** | ذكاء اصطناعي في دردشتك، يعمل داخل حاوية Docker معزولة |

لفتح Claude Code في أي وقت:

```bash
cd nanoclaw-venice
ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
```

استخدمه لتشغيل `/setup` أو `/debug` أو `/customize`، أو إجراء تغييرات على سلوك البوت.

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="الوكيل لا يعمل">
    يعمل وكيل Venice كخدمة في الخلفية ويعيد تشغيل نفسه تلقائياً. إذا لم يعمل:

    **macOS:**
    ```bash
    # Check if it's running
    launchctl list | grep venice-proxy

    # Restart it
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Check logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```

    **Linux:**
    ```bash
    # Check if it's running
    systemctl --user status nanoclaw-venice-proxy

    # Restart it
    systemctl --user restart nanoclaw-venice-proxy

    # Check logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```
  </Accordion>

  <Accordion title="Claude Code يظهر خطأ 403 أو 'Please run /login'">
    هذا يعني أن Claude Code لا يستطيع الاتصال بوكيل Venice.

    1. **تحقق من أن الوكيل يعمل.** راجع خطوة استكشاف الأخطاء أعلاه.
    2. **تأكد من أنك في المجلد الصحيح.** دائماً `cd nanoclaw-venice` أولاً.
    3. **ابدأ من جديد:** أغلق جميع الطرفيات وشغّل:
       ```bash
       cd nanoclaw-venice
       ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
       ```
  </Accordion>

  <Accordion title="أخطاء النموذج ('model does not exist')">
    أعد تشغيل الوكيل والبوت:

    **macOS:**
    ```bash
    # Restart proxy
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Restart bot
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw
    ```

    **Linux:**
    ```bash
    # Restart proxy
    systemctl --user restart nanoclaw-venice-proxy

    # Restart bot
    systemctl --user restart nanoclaw
    ```

    تحقق من النماذج المتاحة في [كتالوج النماذج](/ar/models/text).
  </Accordion>

  <Accordion title="البوت لا يستجيب للرسائل">
    اتبع هذه الخطوات بالترتيب:

    1. **تحقق من كلمة التشغيل.** تأكد من أنك تستخدم البادئة الصحيحة (مثل `@Andy hello`).
    2. **تحقق من أن Docker يعمل.** شغّل `docker info` — إذا أعطى خطأً، افتح Docker Desktop.
    3. **تحقق من أن الوكيل يعمل.** راجع "الوكيل لا يعمل" أعلاه.
    4. **تحقق من السجلات:** `tail -f logs/nanoclaw.log` في مجلد المشروع.
    5. **تحقق من سجلات الحاوية.** افتح مجلد `nanoclaw-venice/groups/main/logs/`. افتح أحدث ملف يبدأ بـ `container-`.
    6. **أعد تشغيل كل شيء:** أعد تشغيل كل من الوكيل والبوت (راجع أعلاه).
  </Accordion>

  <Accordion title="فشل بناء الحاوية أثناء الإعداد">
    تأكد من أن Docker Desktop مفتوح ويعمل. انتظر 10 ثوانٍ لكي يبدأ Docker بالكامل، ثم اكتب `continue` في المعالج لإعادة المحاولة.
  </Accordion>

  <Accordion title="انقطع اتصال WhatsApp">
    قد تنتهي صلاحية جلسة WhatsApp. لإعادة الاتصال:

    ```bash
    cd nanoclaw-venice
    npm run auth
    ```

    امسح رمز QR بـ WhatsApp (Settings → Linked Devices → Link a Device)، ثم أعد تشغيل البوت:
    - macOS: `launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
    - Linux: `systemctl --user restart nanoclaw`
  </Accordion>
</AccordionGroup>

## متقدم

<AccordionGroup>
  <Accordion title="امنح البوت وصولاً إلى الملفات على حاسوبك">
    افتراضياً، البوت معزول تماماً عن حاسوبك — يمكنه رؤية ذاكرته الخاصة وسجل المحادثة فقط.

    - **أثناء الإعداد:** عند السؤال عن وصول المجلدات، اختر "Yes"
    - **بعد الإعداد:** شغّل `/customize` في Claude Code
  </Accordion>

  <Accordion title="بدء/إيقاف البوت يدوياً">
    يشغّل NanoClaw خدمتين في الخلفية تبدآن تلقائياً عند الإقلاع.

    **macOS:**
    | الإجراء | الأمر |
    |--------|---------|
    | بدء البوت | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | إيقاف البوت | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | إعادة تشغيل البوت | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw` |
    | بدء الوكيل | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | إيقاف الوكيل | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | إعادة تشغيل الوكيل | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy` |

    **Linux:**
    | الإجراء | الأمر |
    |--------|---------|
    | بدء البوت | `systemctl --user start nanoclaw` |
    | إيقاف البوت | `systemctl --user stop nanoclaw` |
    | إعادة تشغيل البوت | `systemctl --user restart nanoclaw` |
    | بدء الوكيل | `systemctl --user start nanoclaw-venice-proxy` |
    | إيقاف الوكيل | `systemctl --user stop nanoclaw-venice-proxy` |
    | إعادة تشغيل الوكيل | `systemctl --user restart nanoclaw-venice-proxy` |
  </Accordion>

  <Accordion title="استخدام Claude Code عبر Venice (بدون بوت)">
    إذا كنت تريد فقط Claude Code مع Venice ولا تحتاج إلى WhatsApp/Telegram، فيجب أن تعمل خدمة الوكيل. إذا كنت قد شغّلت `/setup` بالفعل، فإنها تعمل بالفعل كخدمة في الخلفية.

    ```bash
    cd nanoclaw-venice
    ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
    ```

    **نصيحة:** أضف هذا إلى `~/.zshrc` (أو `~/.bashrc`) حتى تتمكن من التبديل بسرعة لأي طرفية إلى Venice:

    ```bash
    alias venice='export ANTHROPIC_BASE_URL=http://localhost:4001 && export ANTHROPIC_API_KEY=venice-proxy && echo "Using Venice API"'
    alias anthropic='unset ANTHROPIC_BASE_URL && unset ANTHROPIC_API_KEY && echo "Using Anthropic API"'
    ```

    ثم فقط اكتب `venice` في أي طرفية قبل تشغيل `claude` لاستخدام Venice، أو `anthropic` للتبديل مرة أخرى.
  </Accordion>

  <Accordion title="تشغيل بوتات متعددة">
    يمكنك تشغيل عدة بوتات NanoClaw على نفس الجهاز (مثل واحد للاستخدام الشخصي وآخر للفريق). فقط استنسخ المستودع في مجلد مختلف وشغّل الإعداد مرة أخرى. ملاحظة: يتشاركون في نفس صورة Docker، لذا فإن إعادة بناء واحد يؤثر على جميعهم.
  </Accordion>

  <Accordion title="أوامر المطورين">
    للأشخاص الذين يريدون تعديل شيفرة NanoClaw:

    ```bash
    npm run dev          # Start proxy + NanoClaw with hot reload
    npm run proxy        # Start just the Venice proxy
    npm run build        # Compile TypeScript
    npm test             # Run tests
    ./container/build.sh # Rebuild agent container
    ```
  </Accordion>
</AccordionGroup>

| `proxy/venice-proxy.ts` | يترجم تنسيق Anthropic إلى تنسيق OpenAI لـ Venice |
| `src/index.ts` | المنسّق الرئيسي — حلقة الرسائل، استدعاء الوكيل |
| `src/channels/whatsapp.ts` | اتصال WhatsApp عبر baileys |
| `src/channels/telegram.ts` | بوت Telegram عبر grammy |
| `src/container-runner.ts` | يولّد حاويات الوكلاء المعزولة |

## الأسئلة الشائعة

<AccordionGroup>
  <Accordion title="لماذا أحتاج إلى وكيل (proxy)؟">
    يتحدث Claude Agent SDK بتنسيق رسائل Anthropic. تتحدث Venice بتنسيق OpenAI. يترجم الوكيل بينهما حتى يعمل كل شيء دون تعديل SDK.
  </Accordion>

  <Accordion title="هل يمكنني استخدام نماذج مفتوحة المصدر؟">
    نعم. تستضيف Venice العديد من النماذج. أخبر البوت "switch to zai-org-glm-5" أو أي معرّف نموذج Venice. راجع [كتالوج النماذج](/ar/models/text).
  </Accordion>

  <Accordion title="هل هو آمن؟">
    تعمل الوكلاء في حاويات Docker مع عزل حقيقي على مستوى نظام التشغيل. يُمرَّر مفتاح Venice API عبر stdin، ولا يُكتب على القرص داخل الحاويات أبداً. تحصل كل مجموعة على بيئتها المعزولة الخاصة.
  </Accordion>

  <Accordion title="هل أحتاج إلى اشتراك Anthropic؟">
    لا. كل شيء يعمل عبر Venice AI. تحتاج فقط إلى مفتاح Venice API.
  </Accordion>

  <Accordion title="هل يمكنني استخدام هذا على خادم؟">
    نعم. يعمل على أي جهاز Linux مع Docker. استخدم خدمة systemd للبدء التلقائي عند الإقلاع.
  </Accordion>
</AccordionGroup>

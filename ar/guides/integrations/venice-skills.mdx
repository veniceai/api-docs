[Venice Skills](https://github.com/veniceai/skills) هي المجموعة الرسمية من [مهارات الوكلاء (Agent Skills)](https://www.anthropic.com/news/agent-skills) لـ Venice API. كل مهارة عبارة عن مجلد قائم بذاته يحتوي على `SKILL.md` يُحمِّله وكيل LLM عند الحاجة للعمل بشكل صحيح ضد جزء محدد من الـ API.

## لماذا المهارات؟

بدون المهارات، يضطر وكيلك إلى اكتشاف خصوصيات Venice بالطريقة الصعبة: `venice_parameters`، وقوائم enum لأنواع النماذج، وتدفّقات 402 (مطلوب الدفع)، ودورة حياة طابور/استرجاع الفيديو، وslugs الشخصيات، وما إلى ذلك. تُجمّع المهارات هذه المعرفة في ملفات مركَّزة وعند الطلب بحيث لا يُحمّل الوكيل إلا ما يحتاجه للمهمة الحالية.

يتضمن كل `SKILL.md`:

- نقطة (نقاط) النهاية التي يغطيها
- الرؤوس والمعاملات وأشكال الاستجابة المطلوبة
- مثال curl إضافة إلى مثال SDK مُبسَّط
- قسم "gotchas" يضم الأشياء التي يتعثّر بها المدمجون فعلًا

## كتالوج المهارات

| المهارة | تغطّي |
|---|---|
| `venice-api-overview` | عنوان URL الأساسي، وأوضاع المصادقة، ورؤوس الاستجابة، ونموذج التسعير، والإصدارات |
| `venice-auth` | مفاتيح API بـ Bearer + مصادقة محفظة Sign-In-With-X / x402 |
| `venice-chat` | `/chat/completions` مع `venice_parameters`، والمتعدد الوسائط، والأدوات، والتفكير، والبث |
| `venice-responses` | `/responses`، الـ Responses API المتوافق مع OpenAI (ألفا) |
| `venice-embeddings` | نماذج `/embeddings`، وصيغ التشفير، والأبعاد |
| `venice-image-generate` | `/image/generate`, `/images/generations`, `/image/styles` |
| `venice-image-edit` | `/image/edit`, `/image/multi-edit`, `/image/upscale`, `/image/background-remove` |
| `venice-audio-speech` | نماذج TTS في `/audio/speech` والأصوات والصيغ والبث |
| `venice-audio-music` | `/audio/quote`, `/audio/queue`, `/audio/retrieve`, `/audio/complete` |
| `venice-audio-transcription` | `/audio/transcriptions` مع Whisper وParakeet وScribe وWizper وxAI STT |
| `venice-video` | توليد ونسخ نصي `/video/*` |
| `venice-models` | `/models`, `/models/traits`, `/models/compatibility_mapping` |
| `venice-characters` | `/characters*` + `venice_parameters.character_slug` |
| `venice-api-keys` | عمليات CRUD على `/api_keys`، وحدود المعدل، وإنشاء مفاتيح Web3 |
| `venice-billing` | `/billing/balance`, `/billing/usage`, `/billing/usage-analytics` |
| `venice-x402` | أرصدة محفظة `/x402/*`، USDC على Base أو Solana |
| `venice-crypto-rpc` | وكيل JSON-RPC في `/crypto/rpc/*` بتسعير 1×/2×/4× |
| `venice-augment` | `/augment/text-parser`, `/augment/scrape`, `/augment/search` |
| `venice-errors` | أشكال الأخطاء، و402 مطلوب الدفع، و422 سياسة المحتوى، و429 حدود المعدل، واستراتيجية إعادة المحاولة |

## التثبيت

كل مهارة هي مجرد مجلد يحتوي على `SKILL.md` يبدأ بصدر YAML:

```yaml
---
name: venice-chat
description: When the agent should load this skill and what's in it
---
```

ضع مجلد `skills/` (أو أي مجموعة فرعية) في المسار الذي يراقبه وقت التشغيل لديك.

<Tabs>
  <Tab title="Claude Code">
    على مستوى المشروع:

    ```bash
    git clone https://github.com/veniceai/skills.git
    cp -r skills/skills/* .claude/skills/
    ```

    أو عالميًا، لكل مشروع على جهازك:

    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.claude/skills/venice
    ```
  </Tab>

  <Tab title="Cursor">
    على مستوى المشروع:

    ```bash
    git clone https://github.com/veniceai/skills.git .cursor/skills-venice
    ```

    أو انسخ مهارات فردية:

    ```bash
    cp -r skills/venice-chat .cursor/skills/
    ```
  </Tab>

  <Tab title="Codex">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.codex/skills/venice
    ```

    للتثبيت على مستوى المشروع، استهدف `.codex/skills/` بدلًا من ذلك.
  </Tab>

  <Tab title="OpenCode">

    يقرأ OpenCode أيضًا `.opencode/skills/` و`.claude/skills/` و`.agents/skills/` من جذر المشروع.
  </Tab>

  <Tab title="Hermes Agent">
    يحتوي Hermes على مثبّت مدمج لمهارات Venice:

### مرجع المسارات

| وقت التشغيل | على مستوى المشروع | عالمي |
|---|---|---|
| Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| Codex | `.codex/skills/` | `~/.codex/skills/` (أو `$CODEX_HOME/skills/`) |
| OpenCode | `.opencode/skills/` (و`.claude/skills/`, `.agents/skills/`) | `~/.config/opencode/skills/` |
| Hermes Agent | `$HERMES_OPTIONAL_SKILLS_DIR` | `~/.hermes/skills/` |
| Cursor | `.cursor/skills/` | `~/.cursor/skills/` |
| Cline | `.clinerules/skills/` | غير متاح |
| أوقات تشغيل أخرى | `.agents/skills/` (تقليد) | `~/.agents/skills/` |

<Tip>
أوقات التشغيل التي تُعرّف حقول صدر إضافية (`version`, `platforms`, `metadata.*`, `compatibility`, …) مُلزمة بالمواصفات بتجاهل الحقول غير المعروفة، لذا فإن ملف المهارة نفسه يعمل في كل مكان دون تفريع.
</Tip>

### كـ git submodule

إذا أردت إصدارات مثبّتة في مستودعك الخاص:

```bash
git submodule add https://github.com/veniceai/skills.git vendor/venice-skills
```

ثم أنشئ symlink أو انسخ المجموعات الفرعية التي تريدها إلى مسار مهارات وكيلك.

## كيف تُحمّلها الوكلاء

يكتشف الوكيل كل `SKILL.md` عبر `name` و`description` في صدره. عندما يسأل المستخدم شيئًا يطابق غرض مهارة، يُحمّل الوكيل ذلك الملف الواحد في السياق (لا الكتالوج بأكمله)، فيبقى الموجّه صغيرًا وتبقى الإجابة دقيقة.

على سبيل المثال، وكيل يحتاج إلى توليد موسيقى سيُحمّل `venice-audio-music` ويعرف فورًا:

- أن الموسيقى تمر عبر دورة حياة الطابور/الاسترجاع/الإكمال، لا نقطة نهاية متزامنة
- ما النماذج المتاحة وتسعيرها للدقيقة
- كيفية استدعاء `/audio/quote` لتقدير التكلفة أولًا
- ما يجب أن يكون عليه backoff الاستعلام

بدون المهارة، قد يحاول الوكيل استدعاء `/audio/speech` للموسيقى ويحصل على استجابة عديمة الفائدة.

## تأليف مهارة جديدة

1. انسخ `template/` إلى `skills/<your-skill>/`.
2. املأ الصدر والمتن. اجعل `description` ملموسًا، لأنه ما يستخدمه الوكيل لتحديد متى يُحمّل المهارة.
3. اربط المهارات ذات الصلة في الأسفل للتنقل المتبادل.
4. افتح PR على [`veniceai/skills`](https://github.com/veniceai/skills).

راجع `CONTRIBUTING.md` في المستودع لتقاليد الأسلوب (فقرة افتتاحية قصيرة، وجداول نقاط نهاية صريحة، وcurl + مثال SDK واحد، وقسم "gotchas"، وحد أقصى 500 سطر).

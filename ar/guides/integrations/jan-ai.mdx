[Jan](https://jan.ai/) هو مساعد ذكاء اصطناعي مفتوح المصدر لتشغيل النماذج المحلية والبعيدة من تطبيق سطح مكتب واحد. يعمل Venice في Jan عبر دعمه لموفّر النموذج البعيد المتوافق مع OpenAI.

## المتطلبات

- مفتاح Venice API من [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) مثبَّت ويعمل على جهازك
- معرّف نموذج نصي Venice من [كتالوج النماذج](/models/text)

---

## الإعداد

<Steps>
  <Step title="احصل على مفتاح Venice API الخاص بك">
    ولِّد أو انسخ مفتاح API من [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="افتح إعدادات موفّر النموذج في Jan">
    في Jan، افتح **Settings** وانتقل إلى **Model Providers**.

    اختر تدفق موفّر Jan المخصص المتوافق مع OpenAI. بناءً على إصدار Jan الخاص بك، قد يظهر هذا كـ **OpenAI** مع تفعيل **Custom Endpoint**، أو كموفّر **Any OpenAI Compatible API**.
  </Step>

  <Step title="قم بتكوين endpoint Venice">
    أدخل إعدادات موفّر Venice:

    | الحقل | القيمة |
    |-------|-------|
    | اسم الموفّر | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | مفتاح API | مفتاح Venice API الخاص بك |
    | نوع API | متوافق مع OpenAI |

    احفظ أو طبّق إعدادات الموفّر قبل إضافة النماذج.
  </Step>

  <Step title="أضف نماذج Venice">
    أضف معرّفات نماذج Venice التي تريد استخدامها. يتطلب Jan أن يطابق معرّف النموذج الموفّر تمامًا.

    خيارات بداية جيدة:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    يمكنك إضافة أي نموذج نصي مدعوم من [كتالوج النماذج](/models/text).
  </Step>

  <Step title="ابدأ محادثة">
    افتح محادثة Jan جديدة أو موجودة، اختر أحد نماذج Venice الخاصة بك من محدد النموذج، وأرسل رسالة اختبار.

    سيرسل Jan الطلب إلى Venice باستخدام مفتاح API الخاص بك و base URL المتوافق مع OpenAI.
  </Step>
</Steps>

## التحقق من الإعداد

أرسل مطالبة قصيرة من Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

إذا كان الموفّر مُكوَّنًا بشكل صحيح، يجب أن يبث Jan ردًا من نموذج Venice المختار.

يمكنك أيضًا التحقق من مفتاح API الخاص بك خارج Jan بـ:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

## ملاحظات التكوين

- استخدم `https://api.venice.ai/api/v1` كـ base URL. لا تُضمِّن `/chat/completions`؛ يضيف Jan مسار الـ endpoint لطلبات المحادثة.
- يجب أن تطابق معرّفات النماذج Venice تمامًا. إذا لم يظهر نموذج في محدد Jan، أضفه يدويًا باستخدام معرّف النموذج من [كتالوج النماذج](/models/text).
- احتفظ بـ Venice مُكوَّنة كموفّر متوافق مع OpenAI. لا تستخدم موفّر Jan الأصلي لـ Anthropic لنماذج Venice Claude.
- تتطلب طلبات Venice البعيدة اتصالًا بالإنترنت وتستهلك اعتمادات Venice API.

<Info>
تظل نماذج Jan المحلية متاحة بعد إضافة Venice. يمكنك التبديل بين النماذج المحلية ونماذج Venice من محدد نموذج المحادثة.
</Info>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="خطأ 401 أو خطأ مصادقة">
    تحقق من أن مفتاح API هو مفتاح Venice من [venice.ai/settings/api](https://venice.ai/settings/api)، بدون مسافات إضافية قبله أو بعده.
  </Accordion>

  <Accordion title="404 أو لم يتم العثور على النموذج">
    تأكد من أن معرّف النموذج في Jan يطابق معرّف نموذج Venice تمامًا. تحقق أيضًا من أن base URL هو `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="لا توجد نماذج Venice في المحدد">
    أضف النموذج يدويًا في إعدادات موفّر Jan. استخدم معرّف النموذج بالضبط، مثل `zai-org-glm-5-1`، واحفظ الموفّر مرة أخرى.
  </Accordion>

  <Accordion title="الطلبات لا تزال تذهب إلى موفّر آخر">
    تأكد من أن المحادثة تستخدم نموذجًا مرتبطًا بموفّر Venice الخاص بك. إذا كان Jan يستخدم إدخال نموذج OpenAI أو Anthropic المدمج، فقم بالتبديل إلى نموذج Venice الذي أضفته.
  </Accordion>
</AccordionGroup>

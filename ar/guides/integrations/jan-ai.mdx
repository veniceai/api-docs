[Jan](https://jan.ai/) هو مساعد ذكاء اصطناعي مفتوح المصدر لتشغيل النماذج المحلية والبعيدة من تطبيق سطح مكتب واحد. تعمل Venice في Jan عبر دعمه لمزوّدي النماذج البعيدة المتوافقين مع OpenAI.

## المتطلبات الأساسية

- مفتاح Venice API من [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) مثبَّت ويعمل على جهازك
- معرّف نموذج نصي Venice من [كتالوج النماذج](/ar/models/text)

---

## الإعداد

<Steps>
  <Step title="احصل على مفتاح Venice API">
    أنشئ أو انسخ مفتاح API من [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="افتح إعدادات مزوّد النموذج في Jan">
    في Jan، افتح **Settings** وانتقل إلى **Model Providers**.

    اختر تدفق مزوّد Jan المخصص المتوافق مع OpenAI. حسب إصدار Jan لديك، قد يظهر هذا كـ **OpenAI** مع تمكين **Custom Endpoint**، أو كمزوّد **Any OpenAI Compatible API**.
  </Step>

  <Step title="هيّئ نقطة نهاية Venice">
    أدخل إعدادات مزوّد Venice:

    | الحقل | القيمة |
    |-------|-------|
    | Provider name | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | مفتاح Venice API الخاص بك |
    | API type | متوافق مع OpenAI |

    احفظ أو طبّق إعدادات المزوّد قبل إضافة النماذج.
  </Step>

  <Step title="أضف نماذج Venice">
    أضف معرّفات نماذج Venice التي تريد استخدامها. يتطلب Jan أن يطابق معرّف النموذج المزوّد تماماً.

    خيارات جيدة للبدء:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    يمكنك إضافة أي نموذج نصي مدعوم من [كتالوج النماذج](/ar/models/text).
  </Step>

  <Step title="ابدأ محادثة">
    افتح محادثة Jan جديدة أو موجودة، اختر أحد نماذج Venice من محدّد النماذج، وأرسل رسالة اختبارية.

    سيرسل Jan الطلب إلى Venice باستخدام مفتاح API الخاص بك وعنوان URL الأساسي المتوافق مع OpenAI.
  </Step>
</Steps>

## التحقق من الإعداد

أرسل طلباً قصيراً من Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

إذا كان المزوّد مهيّأ بشكل صحيح، فيجب أن يقوم Jan ببث استجابة من نموذج Venice المحدّد.

يمكنك أيضاً التحقق من مفتاح API الخاص بك خارج Jan باستخدام:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

## ملاحظات حول التهيئة

- استخدم `https://api.venice.ai/api/v1` كعنوان URL أساسي. لا تضمّن `/chat/completions`؛ يضيف Jan مسار نقطة النهاية لطلبات الدردشة.
- يجب أن تتطابق معرّفات النماذج مع Venice تماماً. إذا لم يظهر نموذج في محدد Jan، أضفه يدوياً باستخدام معرّف النموذج من [كتالوج النماذج](/ar/models/text).
- احتفظ بـ Venice مهيّأة كمزوّد متوافق مع OpenAI. لا تستخدم مزوّد Anthropic الأصلي في Jan لنماذج Venice Claude.
- تتطلب طلبات Venice البعيدة اتصالاً بالإنترنت وتستهلك أرصدة Venice API.

<Info>
تبقى نماذج Jan المحلية متاحة بعد إضافة Venice. يمكنك التبديل بين النماذج المحلية وVenice من محدد نموذج الدردشة.
</Info>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="خطأ 401 أو خطأ مصادقة">
    تحقق من أن مفتاح API هو مفتاح Venice من [venice.ai/settings/api](https://venice.ai/settings/api)، دون مسافات إضافية قبله أو بعده.
  </Accordion>

  <Accordion title="خطأ 404 أو نموذج غير موجود">
    تأكد من أن معرّف النموذج في Jan يطابق معرّف نموذج Venice تماماً. تحقق أيضاً من أن عنوان URL الأساسي هو `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="لا توجد نماذج Venice في المحدد">
    أضف النموذج يدوياً في إعدادات مزوّد Jan. استخدم معرّف النموذج بالضبط، مثل `zai-org-glm-5-1`، واحفظ المزوّد مرة أخرى.
  </Accordion>

  <Accordion title="الطلبات لا تزال تذهب إلى مزوّد آخر">
    تأكد من أن الدردشة تستخدم نموذجاً مرتبطاً بمزوّد Venice الخاص بك. إذا كان Jan يستخدم إدخال نموذج OpenAI أو Anthropic مدمجاً، فبدّل إلى نموذج Venice الذي أضفته.
  </Accordion>
</AccordionGroup>

[Aider](https://aider.chat/) هو أداة برمجة زوجية بالذكاء الاصطناعي تحرّر مستودع git المحلي الخاص بك من الـ terminal. يعمل Venice عبر دعم Aider لـ [API المتوافقة مع OpenAI](https://aider.chat/docs/llms/openai-compat.html) — عيّن base URL الخاص بـ Venice ومفتاح API، ثم أضف البادئة `openai/` لمعرّف النموذج.

## المتطلبات

- مفتاح Venice API من [venice.ai/settings/api](https://venice.ai/settings/api)
- Python 3.9+ متوفّر على جهازك
- مستودع git للعمل فيه

---

## الإعداد

<Steps>
  <Step title="ثبّت Aider">
    ثبّت Aider باستخدام المثبِّت الرسمي:

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    راجع [وثائق تثبيت Aider](https://aider.chat/docs/install.html) إذا كنت تفضّل طريقة تثبيت أخرى.
  </Step>

  <Step title="اضبط Venice كـ endpoint خاص بـ OpenAI">
    وجّه Aider إلى Venice بتعيين base URL المتوافق مع OpenAI ومفتاح Venice API الخاص بك.

    **Mac / Linux:**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell):**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (Command Prompt، بشكل دائم):**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    أعد تشغيل الـ terminal بعد `setx` لتصبح المتغيرات الجديدة متاحة.
  </Step>

  <Step title="ابدأ Aider بنموذج Venice">
    من دليل مشروعك، أضف البادئة `openai/` لمعرّف نموذج Venice:

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    نماذج بداية جيدة أخرى:

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    تصفّح جميع معرّفات النماذج النصية في [كتالوج النماذج](/models/text).
  </Step>
</Steps>

## احفظ التكوين بشكل دائم باستخدام `.env`

للإعداد الخاص بالمشروع، أنشئ ملف `.env` في جذر مستودع git الخاص بك:

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

ثم ابدأ Aider بشكل طبيعي:

```bash
aider
```

يقرأ Aider أيضًا `.env` من الدليل الرئيسي الخاص بك. تحظى قيم جذر المستودع بالأولوية على قيم الدليل الرئيسي. أبقِ هذا الملف خارج git — أضف `.env` إلى `.gitignore` إذا لم يكن مُتجاهَلًا بالفعل.

<Info>
يمكنك أيضًا تعيين `AIDER_OPENAI_API_BASE` و`AIDER_OPENAI_API_KEY`. لا تزال وثائق Aider الرسمية توثّق `OPENAI_API_BASE` و`OPENAI_API_KEY` للموفّرين المتوافقين مع OpenAI.
</Info>

---

## احفظ التكوين بشكل دائم باستخدام YAML

يمكنك تخزين نفس الإعدادات في `.aider.conf.yml` في جذر المستودع أو في الدليل الرئيسي:

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

ثم شغّل:

```bash
aider
```

## التحقق من الإعداد

بعد بدء Aider، اطلب منه شيئًا صغيرًا يُثبت أن النموذج قابل للوصول:

```txt
Reply with one sentence confirming you are running through Venice.
```

إذا كان الـ endpoint والمفتاح صحيحين، سيقوم Aider بـ stream للاستجابة من نموذج Venice المختار.

يمكنك أيضًا تأكيد مفتاح API خارج Aider:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## اختيار النماذج

| حالة الاستخدام              | مثال معرّف النموذج  | علم Aider                          |
| --------------------------- | ------------------- | ---------------------------------- |
| برمجة/إعادة هيكلة قوية      | `zai-org-glm-5-1`   | `--model openai/zai-org-glm-5-1`   |
| تكرار سريع                  | `minimax-m25`       | `--model openai/minimax-m25`       |
| برمجة بسياق طويل            | `kimi-k2-5`         | `--model openai/kimi-k2-5`         |
| محادثة غير خاضعة للرقابة    | `venice-uncensored` | `--model openai/venice-uncensored` |

أضف دائمًا البادئة `openai/` لمعرّفات نماذج Venice عند تشغيل Aider. بدون هذه البادئة، لن يوجّه Aider الطلب عبر مسار الموفّر المتوافق مع OpenAI.

اكتشف معرّفات النماذج الحالية في وقت التشغيل:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

أو راجع [GET /models/traits](/api-reference/endpoint/models/traits) للحصول على الافتراضيات مثل `text:default`.

## ملاحظات التكوين

- احتفظ بـ `OPENAI_API_BASE` مُعيَّنًا إلى `https://api.venice.ai/api/v1`. لا تُلحق `/chat/completions`؛ يضيف Aider مسار المحادثة بنفسه.
- استخدم مفتاح Venice API في `OPENAI_API_KEY`. لا تستخدم مفتاح OpenAI مع base URL الخاص بـ Venice.
- قد يحذّر Aider من أن النموذج يحتوي على نافذة سياق أو تكلفة غير معروفة. هذا التحذير متوقع للعديد من معرّفات نماذج Venice وعادةً ما يكون آمنًا تجاهله. راجع [تحذيرات نماذج Aider](https://aider.chat/docs/llms/warnings.html).
- فضّل نماذج البرمجة/الاستدلال القادرة للتعديلات متعددة الملفات. قد تواجه النماذج الأصغر أو غير المخصصة للبرمجة صعوبة مع تنسيقات تحرير Aider.

<Warning>
تعيين `OPENAI_API_BASE` و`OPENAI_API_KEY` في shell الخاص بك يؤثّر أيضًا على الأدوات الأخرى التي تقرأ هذه المتغيرات. فضّل `.env` خاص بالمشروع أو `.aider.conf.yml` إذا كنت تستخدم OpenAI في مكان آخر ضمن نفس البيئة.
</Warning>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="خطأ 401 أو خطأ مصادقة">
    تأكّد من أن `OPENAI_API_KEY` هو مفتاح Venice من [venice.ai/settings/api](https://venice.ai/settings/api)، بدون أي مسافات بيضاء إضافية. على Windows بعد `setx`، أعد تشغيل الـ terminal.
  </Accordion>
  <Accordion title="النموذج غير موجود أو استجابة فارغة">
    تأكّد من أن معرّف النموذج موجود في [كتالوج النماذج النصية](/models/text) وأنك بدأت Aider بالبادئة `openai/`، على سبيل المثال `openai/zai-org-glm-5-1`.
  </Accordion>
  <Accordion title="لا تزال الطلبات تذهب إلى OpenAI">
    تحقق من أن `OPENAI_API_BASE` هو تمامًا `https://api.venice.ai/api/v1` في نفس جلسة shell. اطبعه باستخدام `echo $OPENAI_API_BASE` (أو `$env:OPENAI_API_BASE` في PowerShell).
  </Accordion>
  <Accordion title="تحذير نافذة سياق / تكاليف غير معروفة">
    متوقع للعديد من نماذج Venice. يعود Aider إلى افتراضيات معقولة. لإسكاته، أضف بيانات وصفية للنموذج كما هو موضح في [الإعدادات المتقدمة لنموذج Aider](https://aider.chat/docs/config/adv-model-settings.html).
  </Accordion>
</AccordionGroup>

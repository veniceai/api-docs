[Cline](https://cline.bot/) هو مساعد برمجة بالذكاء الاصطناعي لـ VS Code يمكنه قراءة الملفات وتحريرها وتشغيل الأوامر والعمل على مهام تطوير متعددة الخطوات. يعمل Venice عبر إعدادات موفّر Cline المتوافق مع OpenAI.

## المتطلبات

- مفتاح Venice API من [venice.ai/settings/api](https://venice.ai/settings/api)
- تثبيت [Visual Studio Code](https://code.visualstudio.com/)
- تثبيت [إضافة Cline](https://marketplace.visualstudio.com/items?itemName=saoudrizwan.claude-dev) في VS Code

## الإعداد

<Steps>
  <Step title="افتح إعدادات Cline">
    حدد أيقونة Cline في شريط النشاط في VS Code، ثم حدد أيقونة الترس لفتح إعدادات Cline.
  </Step>

  <Step title="اختر موفّر API">
    عيّن **API Provider** إلى **OpenAI Compatible**.
  </Step>

  <Step title="اضبط Venice">
    أدخل القيم التالية:

    | الإعداد | القيمة |
    | --- | --- |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API Key | مفتاح Venice API الخاص بك |
    | Model ID | معرّف نموذج من [كتالوج النماذج النصية](/models/text) |

    على سبيل المثال، استخدم `zai-org-glm-5-1` كمعرّف نموذج.

    أدخل مفتاح API نفسه، بدون بادئة `Bearer`. أبقِ Base URL على `/api/v1`؛ يضيف Cline مسار `/chat/completions`.
  </Step>

  <Step title="اضبط قدرات النموذج">
    وسّع **Model Configuration** وعيّن نافذة السياق والحد الأقصى لرموز الإخراج لتتطابق مع النموذج المحدد. فعّل القدرات التي يدعمها النموذج، بما في ذلك إدخال الصور أو استخدام الكمبيوتر عند الاقتضاء.

    راجع النموذج المحدد في [كتالوج النماذج النصية](/models/text) قبل تفعيل القدرات الاختيارية.
  </Step>

  <Step title="احفظ واختبر">
    احفظ إعدادات الموفّر، وافتح مشروعًا، وأعطِ Cline مهمة صغيرة مثل:

    ```txt
    Review the current file and suggest one safe improvement.
    ```

    راجع إذن كل ملف أو أمر قبل الموافقة عليه.
  </Step>
</Steps>

## التحقق من Venice API

إذا لم يتمكن Cline من الاتصال، فتأكّد من مفتاح API ومعرّفات النماذج المتاحة بشكل مستقل:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

انسخ `id` النموذج تمامًا إلى حقل **Model ID** في Cline.

---

## ملاحظات التكوين

- استخدم `https://api.venice.ai/api/v1` كـ Base URL، وليس endpoint `/chat/completions` الكامل.
- قد تتغيّر معرّفات النماذج بمرور الوقت. استخدم [كتالوج النماذج النصية](/models/text) أو `GET /models` بدلًا من الاعتماد على مثال قديم.
- تؤثر إعدادات نافذة السياق ورموز الإخراج في Cline على كيفية تخصيصه لميزانية الطلبات. اجعلها متطابقة مع النموذج المحدد للحصول على أفضل النتائج.
- احتفظ بمفتاح Venice API خاصًا. لا تلصقه في ملفات المصدر أو تلتزم به في git.

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="خطأ 401 أو خطأ مصادقة">
    تأكّد من أن مفتاح API مأخوذ من [venice.ai/settings/api](https://venice.ai/settings/api). الصق المفتاح الخام في Cline بدون `Bearer` أو مسافات إضافية.
  </Accordion>
  <Accordion title="النموذج غير موجود">
    تأكّد من أن معرّف النموذج يظهر في [كتالوج النماذج النصية](/models/text) أو في استجابة `GET /models`، ثم انسخه تمامًا.
  </Accordion>
  <Accordion title="خطأ في الاتصال أو الـ endpoint">
    تأكّد من أن **API Provider** هو **OpenAI Compatible** وأن Base URL هو تمامًا `https://api.venice.ai/api/v1`.
  </Accordion>
  <Accordion title="لا يمكن لـ Cline استخدام الأدوات أو تحرير الملفات">
    افتح **Model Configuration**، وتأكّد من أن النموذج المحدد يدعم القدرات المطلوبة، وفعّل استخدام الكمبيوتر إذا كان Cline يعرض هذا الخيار للنموذج.
  </Accordion>
</AccordionGroup>

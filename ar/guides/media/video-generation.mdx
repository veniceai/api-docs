توليد الفيديو غير متزامن. أرسل مهمة، واحفظ `queue_id`، ثم استعلم من `/video/retrieve` حتى تكون الاستجابة `video/mp4`.

## نقاط النهاية

| نقطة النهاية | الغرض | مطلوبة |
|----------|---------|----------|
| `POST /video/quote` | احصل على سعر بالدولار قبل التوليد | لا |
| `POST /video/queue` | أرسل طلب التوليد | نعم |
| `POST /video/retrieve` | استعلم عن الحالة أو نزّل الفيديو | نعم |
| `POST /video/complete` | احذف الفيديو من التخزين | لا |

## الخطوة 1: وضع التوليد في الطابور

**الطلب:**
```bash
POST https://api.venice.ai/api/v1/video/queue
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "wan-2.5-preview-text-to-video",
  "prompt": "A gondola gliding through Venice canals at sunset",
  "duration": "5s",
  "resolution": "720p",
  "aspect_ratio": "16:9"
}
```

**الاستجابة (200):**
```json
{
  "model": "wan-2.5-preview-text-to-video",
  "queue_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

بالنسبة لنماذج Grok Imagine Private، تتضمّن استجابة الطابور حقلًا إضافيًا `download_url`:

```json
{
  "model": "grok-imagine-text-to-video-private",
  "queue_id": "123e4567-e89b-12d3-a456-426614174000",
  "download_url": "https://private-share.venice.ai/v1/share/read/..."
}
```

`download_url` رابط مُسبق التوقيع تستخدمه لتنزيل الفيديو النهائي بدلًا من قراءته من استجابة retrieve. يُعاد مرة واحدة فقط في استجابة الطابور، لذا احتفظ به مع `queue_id`. ينطبق هذا على كل المتغيرات الأربعة من Grok Imagine Private:

- `grok-imagine-text-to-video-private`
- `grok-imagine-image-to-video-private`
- `grok-imagine-reference-to-video-private`
- `grok-imagine-video-to-video-private`

بخلاف متغيرات `grok-imagine-*-video` العامة، لا تُفوتر نماذج Grok Imagine Private على رفض اعتدال المحتوى، فلا تدفع إلا مقابل التوليدات الناجحة.

احفظ `model` و`queue_id` و`download_url` (إن وُجد) لكل الاستدعاءات اللاحقة.

### روابط التنزيل الخاصة

بالنسبة للنماذج الخاصة، `download_url` هو الطريقة التي تجلب بها الملف النهائي عند اكتمال المهمة. الرابط **قصير العمر ومخصّص لغرض واحد**: هو لتسليم MP4 لك، لا ليكون رابطًا طويل الأمد أو لمشاركة واسعة.

إذا انقطع التنزيل، يمكنك **إعادة محاولة نفس `GET` بضع مرات** من البيئة نفسها حتى يكتمل الملف. هذه الإعادات لاسترداد انقطاعات الشبكة—لا لاستعلام الرابط نفسه إلى ما لا نهاية، أو مشاركته بين عدة عملاء، أو تضمينه كرابط وسائط دائم. مثل هذه الأنماط تظهر غالبًا بصيغة **`429`** أو **`410`**، وهو ما قد يكون مفاجئًا إذا توقّعت أن يتصرّف الرابط كاستضافة ملفات عادية.

للموثوقية، **يجب أن تنشأ طلبات `GET` من شبكة عميل واحدة**. توجد مرونة بعض الشيء إذا تغيّر عنوان IP لديك مرة واحدة (مثلًا فصل VPN وإعادة المحاولة)، لكن التباين الواسع في عناوين IP المصدر لا يعمل عادةً.

يظل الرابط صالحًا حتى **24 ساعة**، أو حتى يُحذف الكائن.

<Note>
إذا احتجت إلى رابط مستقر، أو تشغيل عام، أو وصول متكرر بمرور الوقت، فاحفظ الملف إلى **تخزينك الخاص** أولًا وقدّمه من هناك.
</Note>

**الخصوصية: ألغِ الرابط بـ `DELETE`**

عندما تنتهي من جلب الملف—أو إذا قرّرت عدم الاحتفاظ به—يمكنك استدعاء **`DELETE`** على نفس `download_url`. لا حاجة إلى مفتاح Venice API على ذلك الطلب. هذا اختياري لكن **موصى به عندما تهمّ الخصوصية**، لأن بعض الوكلاء وصناديق وسطية خارج Venice تحتفظ بسجلات للروابط الكاملة، وحذف الرابط هو أبسط طريقة لتضييق النافذة التي يوجد فيها الرابط المُسبق التوقيع.

```bash
curl -X DELETE "$DOWNLOAD_URL"
```

**التدفّق:** استعلم من `/video/retrieve` حتى `COMPLETED` → نفّذ `GET` على `download_url` (أعِد المحاولة بخفّة إن انقطع النقل) → احفظ الملف حيث تحتاجه → نفّذ `DELETE` على `download_url` إن أردت إبطال الرابط → اختياريًا استدعِ `/video/complete` إن كنت لا تزال تستخدم تنظيفًا قائمًا على الطابور.

## الخطوة 2: الاستعلام عن الاكتمال

**الطلب:**
```bash
POST https://api.venice.ai/api/v1/video/retrieve
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "wan-2.5-preview-text-to-video",
  "queue_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

**تعتمد الاستجابة على الحالة:**

| Content-Type | المعنى | الإجراء |
|--------------|---------|--------|
| `application/json` | لا تزال قيد المعالجة | انتظر 5 ث، واستعلم مجددًا |
| `video/mp4` | اكتمل | جسم الاستجابة هو ملف الفيديو |
| `application/json` مع `"COMPLETED"` | اكتمل، والفيديو ليس مضمّنًا | نفّذ `GET` على `download_url` من استجابة الطابور |

**استجابة المعالجة (200, application/json):**
```json
{
  "status": "PROCESSING",
  "average_execution_time": 145000,
  "execution_duration": 53200
}
```

الأوقات بالميلي ثانية. استخدم `average_execution_time` لتقدير الانتظار المتبقي.

**استجابة الاكتمال (200, video/mp4):**
جسم الاستجابة بيانات فيديو ثنائية خام. احفظها إلى ملف.

**استجابة الاكتمال (200, application/json مع `"COMPLETED"`):**
بالنسبة للنماذج التي أعادت `download_url` وقت الطابور، تُعيد retrieve دائمًا JSON. اجلب الفيديو بـ `GET download_url` (بدون رأس مصادقة). راجع [روابط التنزيل الخاصة](#private-download-links) لكيفية عمل هذه الروابط، وإعادات المحاولة، و`DELETE` الاختياري.

## الخطوة 3: التنظيف (اختياري)

إمّا الحذف التلقائي عند الاسترجاع:
```json
{
  "model": "wan-2.5-preview-text-to-video",
  "queue_id": "123e4567-e89b-12d3-a456-426614174000",
  "delete_media_on_completion": true
}
```

أو استدعِ `/video/complete` بعد الحفظ:
```bash
POST https://api.venice.ai/api/v1/video/complete
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "wan-2.5-preview-text-to-video",
  "queue_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

**الاستجابة (200):**
```json
{
  "success": true
}
```

## مثال كامل

<CodeGroup>
```python Python
import os
import time
import requests

API_KEY = os.environ.get("VENICE_API_KEY")
BASE_URL = "https://api.venice.ai/api/v1"
HEADERS = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"}

# Queue
resp = requests.post(f"{BASE_URL}/video/queue", headers=HEADERS, json={
    "model": "wan-2.5-preview-text-to-video",
    "prompt": "A gondola gliding through Venice canals at sunset",
    "duration": "5s",
    "resolution": "720p",
    "aspect_ratio": "16:9"
})
data = resp.json()
model, queue_id = data["model"], data["queue_id"]
download_url = data.get("download_url")

# Poll
while True:
    resp = requests.post(f"{BASE_URL}/video/retrieve", headers=HEADERS,
                         json={"model": model, "queue_id": queue_id})
    if "video/mp4" in resp.headers.get("Content-Type", ""):
        with open("output.mp4", "wb") as f:
            f.write(resp.content)
        break
    if resp.json().get("status") == "COMPLETED" and download_url:
        with open("output.mp4", "wb") as f:
            f.write(requests.get(download_url).content)
        break
    time.sleep(5)

# Cleanup
requests.post(f"{BASE_URL}/video/complete", headers=HEADERS,
              json={"model": model, "queue_id": queue_id})
```

```javascript Node.js
const API_KEY = process.env.VENICE_API_KEY;
const BASE_URL = "https://api.venice.ai/api/v1";
const headers = {"Authorization": `Bearer ${API_KEY}`, "Content-Type": "application/json"};

// Queue
const queueResp = await fetch(`${BASE_URL}/video/queue`, {
    method: "POST", headers,
    body: JSON.stringify({
        model: "wan-2.5-preview-text-to-video",
        prompt: "A gondola gliding through Venice canals at sunset",
        duration: "5s", resolution: "720p", aspect_ratio: "16:9"
    })
});
const {model, queue_id, download_url} = await queueResp.json();

// Poll
while (true) {
    const resp = await fetch(`${BASE_URL}/video/retrieve`, {
        method: "POST", headers,
        body: JSON.stringify({model, queue_id})
    });
    if (resp.headers.get("Content-Type")?.includes("video/mp4")) {
        const fs = await import("fs");
        fs.writeFileSync("output.mp4", Buffer.from(await resp.arrayBuffer()));
        break;
    }
    const status = await resp.json();
    if (status.status === "COMPLETED" && download_url) {
        const fs = await import("fs");
        const video = await fetch(download_url);
        fs.writeFileSync("output.mp4", Buffer.from(await video.arrayBuffer()));
        break;
    }
    await new Promise(r => setTimeout(r, 5000));
}

// Cleanup
await fetch(`${BASE_URL}/video/complete`, {
    method: "POST", headers,
    body: JSON.stringify({model, queue_id})
});
```

## معاملات الطلب

### طلب Queue

| المعامل | النوع | مطلوب | افتراضي | الوصف |
|-----------|------|----------|---------|-------------|
| `model` | string | نعم | - | معرّف النموذج. استخدم `wan-2.5-preview-text-to-video` لـ text-to-video و`wan-2.5-preview-image-to-video` لـ image-to-video |
| `prompt` | string | نعم | - | ما تريد توليده. حد أقصى 2500 حرف |
| `negative_prompt` | string | لا | `"low resolution, error, worst quality, low quality, defects"` | ما تريد تجنّبه |
| `duration` | string | نعم | - | `"5s"` أو `"10s"` |
| `resolution` | string | لا | `"720p"` | `"480p"` أو `"720p"` أو `"1080p"` |
| `aspect_ratio` | string | مشروط | - | يعتمد على النموذج. مطلوب للنماذج التي تكشف خيارات نسبة العرض؛ احذفه للنماذج التي لا تدعم اختيار نسبة العرض |
| `audio` | boolean | مشروط | `true` (عند الدعم) | صالح فقط للنماذج بـ `supportsAudioConfig: true`؛ احذفه للنماذج التي لا تدعم إعداد الصوت |
| `image_url` | string | فقط لـ image-to-video | - | رابط أو رابط بيانات base64 لصورة المصدر |
| `audio_url` | string | مشروط | - | رابط أو رابط بيانات base64 لصوت مرجعي للنماذج التي تدعم إدخال الصوت |

التحقق في الطابور خاص بالنموذج. راجع `/models?type=video` لمعرفة حقول الطلب المدعومة لكل نموذج قبل استدعاء `/video/queue`.

### طلب Quote

| المعامل | النوع | مطلوب | افتراضي | الوصف |
|-----------|------|----------|---------|-------------|
| `model` | string | نعم | - | معرّف النموذج المراد تسعيره |
| `duration` | string | نعم | - | `"5s"` أو `"10s"` |
| `resolution` | string | لا | `"720p"` | `"480p"` أو `"720p"` أو `"1080p"` |
| `aspect_ratio` | string | مشروط | - | أدرجه عندما يدعم النموذج المحدد أو يتطلب اختيار نسبة العرض |

## من صورة إلى فيديو

بالنسبة لنماذج image-to-video، مرّر صورة المصدر عبر `image_url`. ويصف الموجّه الحركة المطلوبة، لا محتوى الصورة.

```json
{
  "model": "wan-2.5-preview-image-to-video",
  "prompt": "Camera slowly zooms in as leaves rustle in the wind",
  "image_url": "https://example.com/image.jpg",
  "duration": "5s"
}
```

أو بـ base64:
```json
{
  "model": "wan-2.5-preview-image-to-video",
  "prompt": "Camera slowly zooms in as leaves rustle in the wind",
  "image_url": "data:image/jpeg;base64,/9j/4AAQ...",
  "duration": "5s"
}
```

## التسعيرة

احصل على التكلفة الدقيقة قبل التوليد. أرسل فقط مدخلات التسعير (`model` و`duration` والاختياريات `resolution` و`aspect_ratio` و`audio`):

**الطلب:**
```json
{
  "model": "wan-2.5-preview-text-to-video",
  "duration": "10s",
  "resolution": "1080p"
}
```

**الاستجابة:**
```json
{
  "quote": 0.085
}
```

التسعيرة بالدولار الأمريكي.

## الأخطاء

| الحالة | تعيدها | المعنى | الإجراء |
|--------|-------------|---------|--------|
| 400 | `queue`, `quote`, `retrieve`, `complete` | معاملات غير صالحة | تحقق من جسم الطلب مقابل المخطط |
| 401 | `queue`, `retrieve`, `complete` | فشلت المصادقة | تحقق من مفتاح API |
| 402 | `queue` | رصيد غير كافٍ | أضف أموالًا |
| 404 | `retrieve`, `download_url` | لم يُعثر على الوسائط (غير صالحة أو منتهية أو محذوفة) | تحقق من `model`/`queue_id` أو أعد وضعها في الطابور |
| 410 | `download_url` | الرابط المُسبق التوقيع منتهٍ أو مستخدم بالكامل أو أُلغي (مثلًا بعد `DELETE`) | ابدأ توليدًا جديدًا إن احتجت إلى ملف آخر؛ كل رابط قصير العمر عمدًا |
| 429 | `download_url` | تجاوز حد المعدل—غالبًا من كثرة إعادة المحاولات أو الجلب المتكرر للرابط نفسه | أنهِ التنزيل (لا بأس ببضع إعادات إذا انقطع الاتصال)، احفظ نسخة محلية، واستخدم `DELETE` إن أردت مسح الرابط؛ احتفظ بالوصول المستمر في تخزينك |
| 413 | `queue` | حمولة كبيرة جدًا | قلِّل حجم الصورة/الصوت |
| 422 | `queue`, `retrieve` | مخالفة محتوى | عدّل الموجّه |
| 500 | `queue`, `retrieve`, `complete` | فشل الاستدلال/المعالجة | أعِد المحاولة مع backoff؛ تواصل مع الدعم إن استمرت |
| 503 | `retrieve` | النموذج عند الطاقة القصوى | أعِد المحاولة مع backoff |

## استراتيجية الاستعلام

1. استعلم من `/video/retrieve` على فترات (مثلًا كل 5 ثوانٍ)
2. إذا كان `Content-Type` هو `application/json` و`status` هي `"PROCESSING"`، انتظر واستعلم مجددًا. استخدم `average_execution_time` و`execution_duration` (بالميلي ثانية) لتقدير الوقت المتبقي
3. إذا كان `Content-Type` هو `video/mp4`، احفظ جسم الاستجابة كملف الإخراج
4. إذا كان `Content-Type` هو `application/json` و`status` هي `"COMPLETED"`، نفّذ `GET` على `download_url` من استجابة الطابور لجلب الفيديو (راجع [روابط التنزيل الخاصة](#private-download-links))
5. إذا استخدمت `download_url`، فكّر في `DELETE` على ذلك الرابط عند الانتهاء لتضييق المدة التي يوجد فيها الرابط المُسبق التوقيع؛ ثم اختياريًا اضبط `delete_media_on_completion: true` على retrieve أو استدعِ `/video/complete` للتنظيف القائم على الطابور
6. عامِل `404` كوسائط غير صالحة أو منتهية أو محذوفة؛ وعامِل `500/503` بإعادات المحاولة/backoff

يتيح لك Reference to Video تثبيت مظهر الشخصيات والأجسام والمشاهد بحيث تبقى فيديوهاتك المولَّدة بالذكاء الاصطناعي متّسقة بصريًا. بدلًا من الأمل في أن يفسّر النموذج موجّهك بشكل صحيح، تقدّم مراسي بصرية — صور مرجعية تخبر النموذج بالضبط كيف يبدو موضوعك.

هذه الميزة متاحة على نماذج **Kling O3** و**Grok Imagine R2V** في [Venice Video Studio](https://venice.ai/video). تستخدم كل عائلة نماذج مقاربة مختلفة للصور المرجعية — راجع الأقسام الخاصة بكل نموذج أدناه.

## متى تستخدم Reference to Video

استخدم Reference to Video عندما تحتاج إلى:

- **ثبات الشخصية** — نفس الشخص أو الشخصية عبر لقطات متعددة
- **دقة المنتج** — منتج حقيقي يجب أن يبدو مطابقًا للأصل
- **استمرارية المشهد** — بيئة أو خلفية محددة عبر التوليدات
- **مشاهد متعددة الشخصيات** — عدة شخصيات متميّزة تتفاعل دون اندماج

للحالات البسيطة مثل text-to-video أو image-to-video حيث الاتساق ليس حرجًا، تعمل النماذج القياسية بشكل جيد دون مراجع.

## النماذج المتاحة

| النموذج | المقاربة | الأفضل لـ |
|---|---|---|
| **Kling O3 Pro R2V** | عناصر + صور مشهد | مشاهد معقّدة متعددة الشخصيات مع تحكّم دقيق بالهوية |
| **Kling O3 Standard R2V** | عناصر + صور مشهد | تكرار أسرع على مشاهد قائمة على العناصر |
| **Grok Imagine R2V** | صور مرجعية مسطّحة | توليد سريع موجَّه بالمراجع يصل إلى 7 صور |

يستخدم **Kling O3** مقاربة مُهيكلة بعناصر (مراسي هوية الشخصية مع صور أمامية + مرجعية) وصور مشهد. ويأخذ **Grok Imagine R2V** مقاربة أبسط — ترفع صورًا مرجعية مباشرة وتشير إليها في موجّهك بـ `@Image1` و`@Image2`، وهكذا.

## Kling O3 Reference to Video

### المفاهيم الأساسية

يستخدم Kling O3 Reference to Video ثلاثة أنواع من المدخلات البصرية تعمل معًا:

| الإدخال | مطلوب | الغرض | كيف يُشار إليه في الموجّه |
|---|---|---|---|
| **Elements** | إدخال بصري واحد على الأقل* | تثبيت هوية شخصية أو جسم | `@Element1` و`@Element2` …إلخ |
| **Scene Reference Images** | إدخال بصري واحد على الأقل* | تحديد البيئة والأسلوب والمزاج | `@Image1` و`@Image2` …إلخ |
| **Start Frame** | إدخال بصري واحد على الأقل* | التحكم بأول إطار في الفيديو | غير مطبق (يُضبط عبر الرفع) |
| **End Frame** | لا | التحكم بآخر إطار في الفيديو | غير مطبق (يُضبط عبر الرفع) |

*يُطلب واحد على الأقل من: start frame أو elements أو scene reference images.

### Elements

العنصر هو شخصية أو جسم تريد إبقاءه ثابتًا بصريًا طوال الفيديو. يتكوّن كل عنصر من:

- **Frontal Image** (مطلوبة لكل عنصر) — صورة واضحة بمنظور أمامي للموضوع. هذه هي مرساة الهوية الأساسية. اعتبرها "صورة جواز السفر" لشخصيتك أو منتجك.
- **Reference Images** (1–3 اختياري) — زوايا إضافية للموضوع نفسه (جانب، 45 درجة، خلف). تساعد النموذج على فهم الموضوع في فضاء ثلاثي الأبعاد. إذا لم تُقدَّم، تُستخدم الصورة الأمامية تلقائيًا كمرجع.

يمكنك إضافة حتى **7 عناصر** لكل توليد (محدودة بالإجمالي المجمَّع). أشِر إليها في موجّهك بـ `@Element1` و`@Element2`، وهكذا.

### Scene Reference Images

تُحدّد مراجع المشهد "خشبة المسرح" التي يحدث عليها الفعل. تؤثر على:

- الإضاءة ولوحة الألوان
- العمارة وتفاصيل البيئة
- الأسلوب البصري العام والمزاج

يمكنك إضافة حتى **4 صور مشهد**. أشِر إليها بـ `@Image1` و`@Image2` …إلخ في موجّهك.

### القيود

العدد الإجمالي للصور عبر جميع أنواع الإدخال محدود:

| الحد | القيمة |
|---|---|
| **الحد الأدنى المطلوب** | إدخال بصري واحد على الأقل (start frame أو element أو صورة مشهد) |
| **الإجمالي المجمَّع** (first frame + last frame + elements + scene images) | **7 كحد أقصى** |
| Elements (بدون start/end frame) | 7 كحد أقصى |
| Elements (مع start أو end frame) | 3 كحد أقصى |
| صور مرجع المشهد | 4 كحد أقصى |
| صور مرجعية لكل عنصر | 1–3 |

**سيناريوهات مثال:**

- 7 elements + 0 صور مشهد = 7 ✓ (بدون frames)
- 5 elements + 2 صور مشهد = 7 ✓ (بدون frames)
- First frame (1) + 3 elements + 3 صور مشهد = 7 ✓
- First frame (1) + last frame (1) + 3 elements + 2 صور مشهد = 7 ✓
- First frame (1) + 4 elements = ✗ (3 elements كحد أقصى مع frame)
- First frame (1) + last frame (1) + 4 elements = ✗ (3 elements كحد أقصى مع frames)

<Note>
يتطلب كل عنصر **frontal image**. إذا لم تُقدّم صورًا مرجعية للعنصر، تُستخدم الصورة الأمامية تلقائيًا كمرجع.
</Note>

### وضع متعدد اللقطات

يتيح لك multi-shot تقسيم توليد واحد إلى مشاهد متعددة، لكل منها موجّهها ومدتها. تنتقل العناصر ومراجع المشهد عبر كل اللقطات، مما يحافظ على الاتساق. لا يمكن أن يتجاوز إجمالي المدة عبر كل اللقطات **15 ثانية**.

### دليل خطوة بخطوة (Video Studio)

#### 1. افتح Video Studio واختر النموذج

اذهب إلى [venice.ai/video](https://venice.ai/video). في Model Browser على اليسار، اختر أحد نماذج **Kling O3 Reference to Video**:

- **Kling O3 Pro R2V** — جودة أعلى، وقت توليد أطول (~6 دقائق)
- **Kling O3 Standard R2V** — أسرع، وأكثر فعالية من حيث التكلفة للتكرار

#### 2. أضِف Visual Inputs (واحد على الأقل مطلوب)

يجب أن تقدّم **مدخلاً بصريًا واحدًا على الأقل** لتوليد فيديو: start frame أو element أو صورة مرجع مشهد. في Input Panel، سترى قسم **Elements**. انقر **Add Element** لإنشاء عنصر للشخصيات أو الأجسام التي تريد إبقاءها متّسقة بصريًا.

لكل عنصر:

1. انقر مربّع **Frontal** لرفع صورة واضحة بمنظور أمامي لشخصيتك أو جسمك
2. اختياريًا انقر **Add** أسفل **Reference Images** لرفع زوايا إضافية (1–3)

كرّر للشخصيات أو الأجسام الإضافية (حتى 7 عناصر إجمالًا، أو 3 إذا استخدمت start/end frames).

<Warning>
لا يمكن أن يتجاوز إجمالي first frame + last frame + elements + صور المشهد **7**. راجع [القيود](#limitations) للتفاصيل.
</Warning>

<Tip>
**أفضل صور مرجعية:** استخدم صورًا مضاءة جيدًا بخلفية نظيفة. قدّم زوايا أمامية وجانبية و45 درجة لأقوى تثبيت للهوية. تأكد أن جميع الصور المرجعية تشترك في نفس الأسلوب البصري (لا تخلط بين الواقعي والأنمي).
</Tip>

#### 3. أضِف Scene Reference Images (اختياري)

أسفل قسم Elements، سترى **Scene Reference Images**. ارفع صورًا تحدّد البيئة التي تريدها — موقع محدد أو إعداد إضاءة أو أسلوب فني.

تُوسم تلقائيًا بـ `@Image1` و`@Image2` …إلخ.

#### 4. ارفع Start Frame (اختياري)

إذا أردت التحكم بأول إطار بالضبط في فيديوك، انتقل إلى نوع إدخال **Image** وارفع start frame. يمكنك اختياريًا تعيين end frame أيضًا.

#### 5. اكتب موجّهك

في حقل الموجّه، صِف الفعل الذي تريده مع الإشارة إلى عناصرك وصور مشهدك باستخدام وسوم `@`:

```
@Element1 walks through the streets of @Image1, looking up at the buildings.
The camera slowly tracks from behind, revealing the city skyline.
```

لـ **المشاهد متعددة الشخصيات**:

```
@Element1 and @Element2 enter the cafe in @Image1 from opposite sides.
@Element1 waves and walks toward @Element2, who is sitting at a corner table.
```

#### 6. اضبط الإعدادات

افتح **Video Settings** لتعديل:

| الإعداد | الخيارات | الافتراضي |
|---|---|---|
| Duration | 3 ث – 15 ث | 5 ث |
| Aspect Ratio | 16:9, 9:16, 1:1 | 16:9 |
| Generate Audio | تشغيل/إيقاف | إيقاف |

<Note>
يضيف توليد الصوت مؤثرات صوتية أصلية وحوارًا وصوتًا محيطًا متزامنًا مع الفيديو. ويزيد التكلفة بنحو 25٪.
</Note>

#### 7. ولّد

انقر **Generate Video**. يستغرق Kling O3 عادةً 4–6 دقائق حسب مستوى النموذج والمدة. يمكنك وضع توليدات متعددة في الطابور وتصفّح النتائج في Video Gallery.

### اللوحات القصصية متعددة اللقطات

للتتابعات السردية، استخدم وضع multi-shot لتعريف مشاهد منفصلة ضمن توليد واحد.

1. في منطقة الموجّه، انقر **Add Shot** لإنشاء لقطات إضافية
2. اكتب موجّهًا منفصلاً لكل لقطة
3. اضبط مدة كل لقطة (3–15 ث لكل واحدة، الإجمالي ≤ 15 ث)

تنتقل العناصر ومراجع المشهد تلقائيًا عبر كل اللقطات:

```
Shot 1 (5s): @Element1 stands at the edge of @Image1, looking out at the horizon.
Slow camera push forward.

Shot 2 (5s): Close-up of @Element1's face as they turn toward the camera.
Soft natural lighting, shallow depth of field.

Shot 3 (5s): @Element1 walks away from camera into the distance.
Wide cinematic shot, golden hour lighting.
```

<Warning>
لا يمكن أن يتجاوز إجمالي مدة multi-shot 15 ثانية. على سبيل المثال، ثلاث لقطات 5 ث = 15 ث كحد أقصى.
</Warning>

### نصائح للموجّهات

#### نظِّم موجّهك

اتبع هذا النمط للحصول على نتائج موثوقة:

```
[الموضوع مع وسم @Element] + [الفعل] + [البيئة مع وسم @Image] + [حركة الكاميرا] + [الإضاءة/الأسلوب]
```

**مثال:**

```
@Element1 hops happily across the candy ground of @Image1, stops to look at a
giant lollipop, tilts its head curiously. Cinematic tracking shot, soft warm lighting.
```

#### اجعل الموجّهات 50–150 كلمة

الموجّهات الأقصر تفتقر للتفاصيل. والأطول تُدخل تناقضات. استهدف النقطة المثلى.

#### استخدم لغة كاميرا بسيطة

يستجيب النموذج بشكل أفضل لتوجيهات كاميرا مباشرة:

| استخدم | تجنّب |
|---|---|
| `slow camera push forward` | `dolly zoom with rack focus transition` |
| `tracking shot from behind` | `complex handheld parallax movement` |
| `close-up` | `extreme macro with tilt-shift bokeh` |
| `wide cinematic shot` | `anamorphic ultra-wide establishing crane shot` |

#### استخدم مفردات متّسقة

إذا وصفت شخصية ترتدي "a red jacket" في موجّه واحد، فلا تغيّر إلى "crimson coat" في التالي. يعامل النموذج الكلمات المختلفة كنوايا مختلفة.

#### ضع تعليمات الكاميرا في وقت مبكر

ضع توجيه الكاميرا قرب بداية الموجّه للحصول على نتائج أكثر موثوقية:

```
Cinematic tracking shot of @Element1 walking through @Image1, leaves
blowing in the wind, golden afternoon light.
```

### استخدام Kling O3 عبر API

Kling O3 Reference to Video متاح أيضًا عبر Venice API. راجع [Video Queue API](https://docs.venice.ai/api-reference/endpoint/video/queue) للتفاصيل الكاملة.

#### Python

```python
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "kling-o3-pro-reference-to-video",
        "prompt": "@Element1 walks through @Image1, camera tracking from behind",
        "duration": "8",
        "aspect_ratio": "16:9",
        "audio": True,
        "elements": [
            {
                "frontal_image_url": "https://example.com/character-front.jpg",
                "reference_image_urls": [
                    "https://example.com/character-side.jpg",
                    "https://example.com/character-angle.jpg"
                ]
            }
        ],
        "image_urls": [
            "https://example.com/scene-background.jpg"
        ]
    }
)

queue_id = response.json()["id"]
```

#### مخطط العنصر

يقبل كل عنصر في مصفوفة `elements`:

| الحقل | النوع | مطلوب | الوصف |
|---|---|---|---|
| `frontal_image_url` | string | **نعم** | رابط صورة واضحة بمنظور أمامي |
| `reference_image_urls` | string[] | لا | روابط زوايا إضافية (1–3). إذا حُذفت، تُستخدم الصورة الأمامية كمرجع. |

<Note>
يدعم الـ API أيضًا `video_url` للعناصر القائمة على الفيديو، لكن هذا غير متاح حاليًا في واجهة Video Studio.
</Note>

### استكشاف أخطاء Kling O3

| المشكلة | السبب المحتمل | الإصلاح |
|---|---|---|
| زر Generate معطّل | لم تُقدَّم مدخلات بصرية | أضف مدخلًا بصريًا واحدًا على الأقل: start frame أو element أو صورة مرجع مشهد |
| خطأ "Number of images exceeds the limit" | عدد كبير من المدخلات المجمَّعة | إجمالي first frame + last frame + elements + صور مشهد يجب أن يكون ≤ 7 |
| وجه الشخصية يتغير بين اللقطات | صورة أمامية مختلفة أو مفقودة | استخدم نفس الصورة الأمامية باتساق، واحفظ الوصف مطابقًا |
| تبدو حركة الكاميرا عشوائية | تعليمات كاميرا متعددة أو متناقضة | استخدم تعليمة كاميرا واحدة، وضعها في وقت مبكر من الموجّه |
| الأسلوب يتحوّل بين التوليدات | مراجع مشهد غير متّسقة أو أساليب مختلطة | أعد استخدام صور المشهد نفسها، واحفظ كلمات الأسلوب متّسقة |
| العناصر تندمج في مشاهد متعددة الشخصيات | تعليمات مكانية مبهمة | كن صريحًا بشأن موقع كل عنصر: "foreground left" و"entering from right" |
| الخلفية تبدو مشوّهة | صورة مرجع مشهد مزدحمة أو معقّدة | استخدم صور مرجع مشهد نظيفة وعالية الجودة |
| تبدو الحركة غير طبيعية | أفعال كثيرة في موجّه واحد | بسّط الفعل، استخدم مدة أقصر، فعل واحد لكل لقطة |

<Tip>
اختبر بفيديو 3–5 ثوانٍ قبل الالتزام بمدد أطول. تحافظ المقاطع الأقصر على اتساق أفضل وتتيح لك التكرار بشكل أسرع.
</Tip>

## Grok Imagine Reference to Video

يأخذ Grok Imagine R2V مقاربة أبسط من Kling O3. بدلًا من Elements مُهيكلة مع فصل الصور الأمامية/المرجعية، ترفع **صورًا مرجعية مسطّحة** وتشير إليها مباشرة في موجّهك بـ `@Image1` و`@Image2`، وهكذا. يدمج النموذج تلك المواضيع في الفيديو المُولَّد.

### كيف يعمل

1. ارفع **1–7 صور مرجعية** — صور لشخصيات أو أجسام أو مشاهد تريدها في الفيديو
2. اكتب موجّهًا يصف الفيديو، باستخدام `@Image1` و`@Image2` …إلخ للإشارة إلى صور محددة
3. يولّد النموذج فيديو يدمج تلك المراجع

إذا لم تُدرج وسوم `@Image` في موجّهك، يُشار تلقائيًا إلى كل الصور المرفوعة.

### الإعدادات

| الإعداد | الخيارات | الافتراضي |
|---|---|---|
| Aspect Ratio | 16:9, 4:3, 3:2, 1:1, 2:3, 3:4, 9:16 | 16:9 |
| Resolution | 480p, 720p | 480p |
| Duration | 5s, 8s, 10s | 8s |

<Note>
لا يدعم Grok Imagine R2V توليد الصوت ولا وضع multi-shot ولا Elements. لهذه الميزات، استخدم Kling O3 R2V.
</Note>

### دليل خطوة بخطوة (Video Studio)

#### 1. اختر النموذج

اذهب إلى [venice.ai/video](https://venice.ai/video). في Model Browser، اختر **Grok Imagine R2V**.

#### 2. ارفع الصور المرجعية

انقر **References** في شريط الإدخال (أو استخدم قائمة +) لفتح لوحة الصور المرجعية. ارفع 1–7 صور للشخصيات أو الأجسام أو المشاهد التي تريدها في الفيديو.

تُوسم كل صورة تلقائيًا بـ `@Image1` و`@Image2` …إلخ بترتيب الرفع (من اليسار إلى اليمين).

#### 3. اكتب موجّهك

صِف الفيديو الذي تريده. استخدم وسوم `@Image` للإشارة إلى صور محددة:

```
@Image1 and @Image2 walking together through a sunlit park,
camera slowly tracking alongside them, warm afternoon light.
```

اكتب `@` في حقل الموجّه لرؤية قائمة إكمال تلقائي لمراجع الصور المتاحة.

<Tip>
إذا حذفت وسوم `@Image` بالكامل، يضيف الخلفي تلقائيًا مراجع لكل الصور المرفوعة. هذا مفيد عندما تريد استخدام كل الصور دون تحديد أيها.
</Tip>

#### 4. اضبط الإعدادات وولّد

افتح **Video Settings** لتعديل نسبة العرض والدقة والمدة. انقر **Generate Video**.

### تسعير Grok Imagine R2V

يستخدم Grok Imagine R2V تسعيرًا قائمًا على المدة والدقة:

| الدقة | لكل ثانية |
|---|---|
| 480p | ~$0.063 |
| 720p | ~$0.088 |

**مثال:** فيديو 8 ثوانٍ بدقة 480p = 8 × $0.063 = **~$0.50**

<Note>
يفرض Grok Imagine رسوم اعتدال محتوى على الفيديوهات المولَّدة، حتى لو رُفض الفيديو. ينعكس هذا في تكلفة الرصيد المعروضة قبل التوليد.
</Note>

#### معاملات الـ API

| الحقل | النوع | مطلوب | الوصف |
|---|---|---|---|
| `model` | string | **نعم** | يجب أن يكون `grok-imagine-reference-to-video` |
| `prompt` | string | **نعم** | موجّه نصي مع مراجع اختيارية `@Image1` و`@Image2` |
| `referenceImageUrls` | string[] | **نعم** | 1–7 روابط صور أو روابط بيانات |
| `duration` | string | لا | `"5"` أو `"8"` (افتراضي) أو `"10"` |
| `aspect_ratio` | string | لا | مثل `"16:9"` (افتراضي) أو `"9:16"` أو `"1:1"` |
| `resolution` | string | لا | `"480p"` (افتراضي) أو `"720p"` |

<Note>
لا يستخدم Grok Imagine R2V حقول `elements` أو `image_urls` أو `imageUrl`. تُمرَّر كل الصور المرجعية عبر `referenceImageUrls`.
</Note>

### استكشاف أخطاء Grok Imagine R2V

| المشكلة | السبب المحتمل | الإصلاح |
|---|---|---|
| زر Generate معطّل | لم تُرفع صور مرجعية | ارفع صورة مرجعية واحدة على الأقل |
| خطأ "At least one reference image is required" | `referenceImageUrls` فارغة أو مفقودة | قدّم رابط صورة واحدة على الأقل في `referenceImageUrls` |
| الصورة الخطأ مرتبطة بوسم `@Image` | ترتيب الصور لا يطابق الوسوم | يطابق `@Image1` أول صورة بترتيب الرفع (من اليسار إلى اليمين). أعِد ترتيب الرفع إذا لزم. |
| الموضوع لا يظهر في الفيديو | مراجع كثيرة بدون وسوم صريحة | استخدم وسوم `@Image` في موجّهك لتكون صريحًا بشأن الصور المستخدمة |
| إخراج رديء الجودة | استخدام دقة 480p | جرّب 720p لجودة أعلى (يكلّف أكثر) |
| فيديو قصير جدًا | المدة الافتراضية 8 ث | اضبط المدة على `"10"` لفيديوهات أطول |

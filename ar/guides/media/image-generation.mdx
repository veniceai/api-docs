توليد الصور في Venice متزامن. أرسل موجّهًا إلى `/image/generate` واستلم صورتك في نفس الاستجابة، إمّا بصيغة base64 داخل JSON أو ثنائية خام عندما تكون `return_binary` بقيمة `true`.

## نقاط النهاية

| نقطة النهاية | الغرض | متى تُستخدم |
|---|---|---|
| `POST /image/generate` | واجهة Venice الأصلية لتوليد الصور | استخدمها لدعم كامل الميزات |
| `GET /image/styles` | اعرض قائمة الأنماط الجاهزة | استخدمها قبل إرسال `style_preset` |
| `POST /images/generations` | واجهة توليد صور متوافقة مع OpenAI | استخدمها عند ترحيل عملاء صور OpenAI الحاليين |

## الخطوة 1: إرسال طلب توليد

تحديد الحجم خاص بالنموذج. بعض النماذج تقبل `width` و`height` صراحة؛ وبعضها يكشف `aspect_ratio`؛ ونماذج المستويات تكشف `aspect_ratio` بالإضافة إلى قيم `resolution` مثل `1K` أو `2K` أو `4K`.

**مثال تحديد الحجم بالبكسلات:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "venice-sd35",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
  "negative_prompt": "blurry, low quality, distorted anatomy, text, watermark",
  "width": 1024,
  "height": 1024,
  "format": "webp"
}
```

**مثال تحديد الحجم بنسبة العرض:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-image-2",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "format": "webp"
}
```

**مثال تحديد الحجم بمستوى الدقة:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "gpt-image-2",
  "prompt": "A cinematic wide shot of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

ينطبق النمط نفسه على بقية نماذج المستويات:

```json
{
  "model": "nano-banana-pro",
  "prompt": "A serene canal in Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

استخدم [نماذج الصور](/ar/models/image) أو [Models API](/ar/api-reference/endpoint/models/list) لتأكيد حقول الأحجام التي يقبلها كل نموذج.

**الاستجابة (200):**
```json
{
  "id": "generate-image-1234567890",
  "images": [
    "UklGRiIAAABXRUJQVlA4IBYAAAAwAQCdASoQABAAPm..."
  ],
  "timing": {
    "inferenceDuration": 1840,
    "inferencePreprocessingTime": 22,
    "inferenceQueueTime": 31,
    "total": 1893
  }
}
```

تحتوي مصفوفة `images` على بيانات صور مُرمَّزة بـ base64. فُك أول عنصر لحفظه أو عرضه. الحقل `timing.total` هو المدة الكاملة للطلب بالميلي ثانية.

## الخطوة 2: فك ترميز الصورة وحفظها

<CodeGroup>
```python Python
import base64
import os
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/image/generate",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "venice-sd35",
        "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
        "width": 1024,
        "height": 1024,
        "format": "webp",
    },
)

data = response.json()
image_bytes = base64.b64decode(data["images"][0])

with open("output.webp", "wb") as f:
    f.write(image_bytes)

print(f"Saved image from request {data['id']}")
```

```javascript Node.js
import fs from "fs";

const response = await fetch("https://api.venice.ai/api/v1/image/generate", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "venice-sd35",
    prompt: "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
    width: 1024,
    height: 1024,
    format: "webp",
  }),
});

const data = await response.json();
const imageBuffer = Buffer.from(data.images[0], "base64");
fs.writeFileSync("output.webp", imageBuffer);

console.log(`Saved image from request ${data.id}`);
```
</CodeGroup>

## الخطوة 3: إعادة بيانات ثنائية بدلًا من JSON (اختياري)

إذا أردت أن يكون جسم الاستجابة هو ملف الصورة نفسه، اضبط `return_binary: true`. هذا مفيد عندما تريد بثّ الصورة أو حفظها مباشرة دون فكّ ترميز base64.

```bash
curl https://api.venice.ai/api/v1/image/generate \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o output.png \
  -d '{
    "model": "qwen-image-2",
    "prompt": "Minimalist poster of a moonlit Venetian bridge in deep blue tones",
    "format": "png",
    "return_binary": true
  }'
```

عندما تكون `return_binary` بقيمة `true`، يكون جسم الاستجابة بيانات `image/jpeg` أو `image/png` أو `image/webp` خام بحسب `format` الذي طلبته.

<Note>
لا يُدعم `variants` إلا عندما تكون `return_binary` بقيمة `false`.
</Note>

## الخطوة 4: عرض قائمة الأنماط المتاحة (اختياري)

إذا أردت استخدام `style_preset`، اجلب أولًا الأنماط المتاحة من `/image/styles`:

```bash
curl https://api.venice.ai/api/v1/image/styles \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

**الاستجابة (200):**
```json
[
  "3D Model",
  "Analog Film",
  "Anime",
  "Cinematic",
  "Digital Art"
]
```

ثم مرّر إحدى تلك القيم في طلب التوليد:

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "Cinematic"
}
```

استخدم نقطة نهاية الأنماط عندما تريد أسماء الإعدادات المسبقة بدقة بدلًا من تخمينها.

## معاملات الطلب

| المعامل | النوع | مطلوب | افتراضي | الوصف |
|---|---|---|---|---|
| `model` | string | نعم | - | معرّف النموذج المستخدم في التوليد |
| `prompt` | string | نعم | - | ما تريد توليده |
| `negative_prompt` | string | لا | - | ما تريد تجنّبه في الصورة |
| `width` | integer | لا | `1024` | عرض الإخراج بالبكسل للنماذج المعتمدة على البكسل مثل `venice-sd35` و`qwen-image` |
| `height` | integer | لا | `1024` | ارتفاع الإخراج بالبكسل للنماذج المعتمدة على البكسل مثل `venice-sd35` و`qwen-image` |
| `format` | string | لا | `webp` | صيغة الإخراج: `jpeg` أو `png` أو `webp` |
| `variants` | integer | لا | `1` | عدد الصور المُولَّدة (`1`-`4`)، فقط عندما تكون `return_binary` بقيمة `false` |
| `return_binary` | boolean | لا | `false` | إعادة بايتات صورة خام بدلًا من base64 في JSON |
| `safe_mode` | boolean | لا | `true` | تمويه محتوى البالغين عند تفعيله |
| `seed` | integer | لا | عشوائي | أعِد استخدام نفس البذرة لتكرارات أكثر اتساقًا |
| `cfg_scale` | number | لا | حسب النموذج | قيم أعلى تدفع النموذج لاتباع الموجّه بدقة أكبر |
| `style_preset` | string | لا | - | طبّق نمطًا جاهزًا من [أنماط الصور](/ar/api-reference/endpoint/image/styles) |
| `aspect_ratio` | string | مشروط | - | تستخدمها النماذج التي تدعم تحديد الحجم بنسبة العرض، مثل `qwen-image-2` و`gpt-image-2` و`nano-banana-2` و`nano-banana-pro` |
| `resolution` | string | مشروط | - | تستخدمها النماذج التي تدعم مستويات دقة مثل `1K` أو `2K` أو `4K` |
| `enable_web_search` | boolean | مشروط | `false` | يسمح للنماذج المدعومة باستخدام معلومات الويب الحالية؛ يضيف تكلفة إضافية |

التحقق خاص بالنموذج. راجع [نماذج الصور](/ar/models/image) و[Models API](/ar/api-reference/endpoint/models/list) قبل الاعتماد على معامل عبر عدة نماذج.

## خيارات خاصة بالنموذج

### التوليد بدقة عالية

تدعم بعض نماذج الصور `aspect_ratio` دون مستوى `resolution` قابل للاختيار. على سبيل المثال، يقبل `qwen-image-2` نسبة العرض ويُحوّلها إلى أبعاد إخراج خاصة بالنموذج:

```json
{
  "model": "qwen-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9"
}
```

نماذج صور أخرى تدعم `aspect_ratio` بالإضافة إلى مستوى `resolution`. على سبيل المثال، يدعم `gpt-image-2` و`nano-banana-2` و`nano-banana-pro` المستويات `1K` و`2K` و`4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "4K"
}
```

```json
{
  "model": "nano-banana-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

استخدم [نماذج الصور](/ar/models/image) لمعرفة النماذج التي تدعم دقة أعلى وكيفية تسعيرها.

### الأنماط الجاهزة

إذا دعم النموذج المختار ذلك، يتيح لك `style_preset` توجيه الإخراج دون إعادة كتابة الموجّه بأكمله. يمكنك جلب أسماء الإعدادات الصحيحة من [أنماط الصور](/ar/api-reference/endpoint/image/styles):

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "3D Model"
}
```

راجع [أنماط الصور](/ar/api-reference/endpoint/image/styles) للقائمة الحالية للأنماط.

## نقطة النهاية المتوافقة مع OpenAI

إذا كنت تستخدم بالفعل SDKs صور OpenAI أو تكاملات DALL-E الحالية، فإن Venice يدعم أيضًا `POST /images/generations`. تقدّم صيغة طلب أبسط لكنها بميزات أقل من نقطة نهاية Venice الأصلية.

**الطلب:**
```json
{
  "model": "qwen-image-2",
  "prompt": "A clean isometric illustration of an AI control room",
  "size": "1024x1024",
  "response_format": "b64_json"
}
```

استخدم المسار المتوافق مع OpenAI لترحيلات أسرع. واستخدم `/image/generate` عندما تحتاج إلى خيارات خاصة بـ Venice مثل `cfg_scale` و`style_preset` و`variants` أو الاستجابات الثنائية.

## نصائح للموجّهات

1. ابدأ بالموضوع، ثم أضف الوسيط والإضاءة والتأليف والمزاج.
2. ضع التفاصيل التي يجب تجنّبها في `negative_prompt` بدلًا من تحميل الموجّه الرئيسي.
3. أعِد استخدام `seed` عند التكرار لمقارنة تغييرات الموجّه دون تغيير التركيب بالكامل.
4. اجعل تحديد الحجم واعيًا بالنموذج. بعض النماذج تستخدم `width`/`height`، والبعض يستخدم `aspect_ratio`، ونماذج المستويات تستخدم `aspect_ratio` بالإضافة إلى `resolution`.
5. استخدم `variants` أثناء الاستكشاف، ثم عُد إلى مخرج واحد بعد تثبيت الاتجاه.

## الأخطاء

| الحالة | المعنى | الإجراء |
|---|---|---|
| `400` | معاملات طلب غير صالحة | تحقق من أسماء الحقول والأنواع والقيود الخاصة بالنموذج |
| `401` | فشلت المصادقة أو يتطلب النموذج مستوى وصول أعلى | تحقق من مفتاح API ومستوى الوصول للنموذج |
| `402` | رصيد غير كافٍ | أضف أرصدة على [venice.ai/settings/api](https://venice.ai/settings/api) |
| `415` | نوع محتوى غير صالح | أرسل JSON مع `Content-Type: application/json` |
| `429` | تجاوز حد المعدل أو النموذج محمَّل بشكل زائد | أعِد المحاولة مع backoff؛ تحقق من رأس `Retry-After` |
| `500` | فشلت معالجة الاستدلال | أعِد محاولة الطلب |
| `503` | النموذج عند الطاقة القصوى | أعِد المحاولة بعد تأخير قصير |

<Note>
عندما يكون Safe Venice مفعَّلًا، افحص رؤوس الاستجابة مثل `x-venice-is-blurred` و`x-venice-is-content-violation` إذا كنت تحتاج إلى الكشف عن نتائج الاعتدال برمجيًا.
</Note>

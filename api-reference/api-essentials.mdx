Learn the fundamentals of working with the Venice API, including authentication and debugging requests.

## Authentication

The Venice API uses API keys for authentication. You can create and manage API keys in your [dashboard](https://venice.ai/settings/api-keys).

<Warning>
Your API key is a secret! Do not share it with others or expose it in any client-side code (browsers, apps). Load API keys from environment variables or a key management service on the server.
</Warning>

### Using Your API Key

Provide your API key via HTTP Bearer authentication:

```
Authorization: Bearer VENICE_API_KEY
```

### Example Request

```bash
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### JavaScript Example

```javascript
const response = await fetch('https://api.venice.ai/api/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.VENICE_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'venice-uncensored',
    messages: [{ role: 'user', content: 'Hello!' }]
  })
});
```

## Response Headers

Every API response includes headers with metadata about your request, rate limits, and debugging information.

### Request Identification

| Header | Example | Purpose |
|--------|---------|---------|
| `CF-RAY` | `988d6fc23956b871-GRU` | Unique request ID for support |
| `x-venice-version` | `outerface-api-server-us-east4-00930` | API server instance |
| `Date` | `Fri, 03 Oct 2025 15:22:51 GMT` | Response timestamp |

### Rate Limiting

| Header | Example | Purpose |
|--------|---------|---------|
| `x-ratelimit-limit-requests` | `75` | Max requests in window |
| `x-ratelimit-remaining-requests` | `74` | Requests remaining |
| `x-ratelimit-reset-requests` | `1759504980000` | Reset time (Unix ms) |
| `x-ratelimit-limit-tokens` | `750000` | Max tokens in window |
| `x-ratelimit-remaining-tokens` | `750000` | Tokens remaining |
| `x-ratelimit-reset-tokens` | `1759504980000` | Reset time (Unix ms) |

### Account Balance

| Header | Example | Purpose |
|--------|---------|---------|
| `x-venice-balance-diem` | `79.90338836` | DIEM balance |
| `x-venice-balance-usd` | `50.50` | USD balance |
| `x-venice-balance-vcu` | `1000` | VCU balance |

## Debugging in Production

Venice recommends logging request IDs for efficient troubleshooting:

```javascript
// Log important request information
const requestId = response.headers.get('CF-RAY');
const remaining = response.headers.get('x-ratelimit-remaining-requests');
const balance = response.headers.get('x-venice-balance-usd');

console.log({
  requestId,
  remainingRequests: remaining,
  usdBalance: balance,
  status: response.status
});

// Use the requestId when contacting support
if (!response.ok) {
  console.error(`Request ${requestId} failed with status ${response.status}`);
}
```

## Best Practices

1. **Store API keys securely** - Use environment variables, never hardcode
2. **Monitor rate limits** - Check remaining requests before making calls
3. **Log request IDs** - Essential for debugging production issues
4. **Handle errors gracefully** - Check response status and headers
5. **Watch your balance** - Monitor balance headers to avoid service interruption

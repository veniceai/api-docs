# Error Codes

When an error occurs, the Venice API returns a consistent error response with an HTTP status code and a descriptive message. Some errors also include additional fields, such as refund status, suggested retry models, validation details, or request IDs.

## Error response format

Most errors return:

```json
{
  "error": "Invalid request parameters"
}
```

Some endpoints return additional fields:

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits have been refunded.",
  "credits_refunded": true
}
```

Validation errors may include a `details` object:

```json
{
  "error": "Invalid request parameters",
  "details": {
    "_errors": [],
    "field": {
      "_errors": ["Field is required"]
    }
  }
}
```

## Content moderation errors

Venice is built for private, zero-retention inference. We do not manually review API prompts or store prompts for moderation review.

Some models and upstream providers apply their own content policies. When a provider or model blocks a request, Venice surfaces that result as a predictable API error instead of a generic server failure.

The most common moderation code is `CONTENT_POLICY_VIOLATION`, which returns HTTP `422`.

For video requests, moderation is usually surfaced when retrieving the queued result. The response may include whether credits were refunded:

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits have been refunded.",
  "credits_refunded": true
}
```

If the upstream provider charges for moderated requests, the response may be:

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits were not refunded as this model charges for content-moderated requests.",
  "credits_refunded": false
}
```

For streaming chat completions, provider moderation may be returned as an OpenAI-compatible stream error:

```json
{
  "error": {
    "message": "Content blocked by provider",
    "type": "content_filter_error",
    "code": "content_blocked_by_provider",
    "param": null
  }
}
```

For image generation, some moderation results return HTTP `200` with headers such as:

```text
x-venice-is-content-violation: true
x-venice-is-blurred: true
```

In those cases, the image may be blurred, blacked out, or replaced with a content-policy placeholder.

For more detail, see the Content Moderation guide.

Venice API Tools are standalone endpoints for preparing data and interacting with external systems. They do not require a model request, so you can use them independently or combine their output with chat, embeddings, and agent workflows.

## Choose a Tool

| Goal | Tool | Endpoint |
|---|---|---|
| Extract text without running inference | Text Parser | `POST /augment/text-parser` |
| Find current sources and structured search results | Web Search | `POST /augment/search` |
| Convert a public web page to Markdown | Web Scrape | `POST /augment/scrape` |
| Discover valid blockchain network slugs | Crypto Networks | `GET /crypto/rpc/networks` |
| Read chain state or relay a signed transaction | Crypto RPC | `POST /crypto/rpc/{network}` |

## Common Workflows

### Search, Scrape, and Prompt

Use Web Search to discover relevant URLs, Web Scrape to extract the selected pages, and then include the Markdown in a chat request. Keeping retrieval separate lets your application inspect, filter, cite, or cache sources before inference.

### Parse, Store, and Reuse

Use Text Parser when you need extracted document text independently of a model. You can inspect its token count, store the text in your own system, create embeddings, or reuse it across multiple prompts.

### Discover a Network and Call RPC

Fetch the current network list instead of hard-coding supported chains. Select a returned slug and use it in the Crypto RPC path to read chain state or submit a transaction signed by your application.

<Note>
Text Parser, Web Search, and Web Scrape are experimental APIs. Their request and response formats may change without notice.
</Note>

## Authentication

The tools support Venice API key authentication. Paid endpoints also support x402 wallet authentication where indicated in their API reference.

```bash
-H "Authorization: Bearer $VENICE_API_KEY"
```

See [x402 Wallet Authentication](/guides/integrations/x402-venice-api) if your application pays with a Base or Solana wallet instead of an API key.

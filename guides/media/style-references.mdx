Style references let you apply a named visual treatment, such as `Cinematic` or `Anime`, without adding all of the style details to your prompt. Choose a value returned by `GET /image/styles`, then pass it as `style_preset` to the native `POST /image/generate` endpoint.

<Note>
Style references are named presets, not uploaded reference images. To transform an existing image, use [Image Editing](/guides/media/image-editing).
</Note>

## Step 1: List available styles

Fetch the current list instead of hardcoding style names:

```bash
curl https://api.venice.ai/api/v1/image/styles \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

The response contains style names in `data`:

```json
{
  "data": [
    "3D Model",
    "Analog Film",
    "Anime",
    "Cinematic",
    "Comic Book"
  ],
  "object": "list"
}
```

Use a value exactly as returned, including its capitalization and spaces. The available list can change, so applications that display a style picker should refresh or periodically cache this endpoint.

## Step 2: Apply a style

Pass the selected name in `style_preset` when generating an image:

```bash
curl https://api.venice.ai/api/v1/image/generate \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o venice-cinematic.webp \
  -d '{
    "model": "qwen-image-2",
    "prompt": "A gondola passing beneath a stone bridge in Venice at blue hour",
    "style_preset": "Cinematic",
    "aspect_ratio": "16:9",
    "format": "webp",
    "return_binary": true
  }'
```

The prompt still controls the subject, composition, lighting, and other requested details. The style preset steers the overall visual treatment.

<CodeGroup>
```python Python
import base64
import os
import requests

api_key = os.environ["VENICE_API_KEY"]
headers = {"Authorization": f"Bearer {api_key}"}

styles_response = requests.get(
    "https://api.venice.ai/api/v1/image/styles",
    headers=headers,
)
styles_response.raise_for_status()
styles = styles_response.json()["data"]

style = "Cinematic"
if style not in styles:
    raise ValueError(f"Unknown style preset: {style}")

response = requests.post(
    "https://api.venice.ai/api/v1/image/generate",
    headers={**headers, "Content-Type": "application/json"},
    json={
        "model": "qwen-image-2",
        "prompt": "A gondola passing beneath a stone bridge in Venice at blue hour",
        "style_preset": style,
        "aspect_ratio": "16:9",
        "format": "webp",
    },
)
response.raise_for_status()

with open("venice-cinematic.webp", "wb") as output:
    output.write(base64.b64decode(response.json()["images"][0]))
```

```javascript Node.js
import fs from "node:fs";

const headers = {
  Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
};

const stylesResponse = await fetch(
  "https://api.venice.ai/api/v1/image/styles",
  { headers },
);
if (!stylesResponse.ok) throw new Error(await stylesResponse.text());

const { data: styles } = await stylesResponse.json();
const style = "Cinematic";

if (!styles.includes(style)) {
  throw new Error(`Unknown style preset: ${style}`);
}

const response = await fetch(
  "https://api.venice.ai/api/v1/image/generate",
  {
    method: "POST",
    headers: { ...headers, "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "qwen-image-2",
      prompt: "A gondola passing beneath a stone bridge in Venice at blue hour",
      style_preset: style,
      aspect_ratio: "16:9",
      format: "webp",
    }),
  },
);
if (!response.ok) throw new Error(await response.text());

const { images } = await response.json();
fs.writeFileSync(
  "venice-cinematic.webp",
  Buffer.from(images[0], "base64"),
);
```

## Prompting with style references

- Keep the subject and composition in the prompt. A style does not replace the prompt.
- Avoid conflicting style instructions. For example, do not select `Cinematic` while asking for a flat vector icon unless you want the model to blend both directions.
- Reuse the same `seed`, model, dimensions, and prompt when comparing presets so the style is the main variable.
- Treat a preset as guidance rather than an exact color or layout constraint.

## Model and endpoint support

`style_preset` is a Venice-specific parameter on `POST /image/generate`. It is not supported by the OpenAI-compatible `POST /images/generations` endpoint, whose `style` field is accepted only for compatibility and is not used by Venice image generation.

Style support is model-specific. If a model ignores or rejects `style_preset`, choose another image generation model or describe the desired visual style directly in the prompt. See [Image Models](/models/image) and the [Models API](/api-reference/endpoint/models/list) for current model information.

## Troubleshooting

| Problem | What to check |
|---|---|
| The API rejects the style | Fetch `/image/styles` again and use an exact value from `data` |
| The output does not reflect the preset | Confirm the selected model supports `style_preset` and remove conflicting style language from the prompt |
| The subject or layout is wrong | Add those details to `prompt`; the preset controls visual treatment, not composition |
| An OpenAI client has no effect | Call the native `/image/generate` endpoint and use `style_preset`, not `style` |

Le [Venice MCP Server](https://github.com/veniceai/venice-mcp-server) est le serveur officiel [Model Context Protocol](https://modelcontextprotocol.io/) pour Venice. Il expose l'intégralité de l'API Venice (chat, image, vidéo, audio, musique, embeddings, augment web et characters) sous forme de **31 outils** que n'importe quel agent compatible MCP peut appeler.

## Démarrage rapide

<Steps>
  <Step title="Obtenez une clé API Venice">
    Générez-en une depuis [venice.ai/settings/api](https://venice.ai/settings/api). Consultez le [guide de la clé API](/fr/guides/getting-started/generating-api-key) pour des instructions pas à pas.
  </Step>

  <Step title="Ajoutez Venice à la config de votre hôte MCP">
    Déposez ceci dans le fichier de config de votre hôte MCP :

    ```json
    {
      "mcpServers": {
        "venice": {
          "command": "npx",
          "args": ["-y", "@veniceai/mcp-server@0.2.0"],
          "env": { "VENICE_API_KEY": "<your-venice-api-key>" }
        }
      }
    }
    ```

    Chemins de config courants :

    | Hôte | Chemin |
    |---|---|
    | Claude Desktop (macOS) | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Claude Desktop (Windows) | `%APPDATA%\Claude\claude_desktop_config.json` |
    | Cursor | `~/.cursor/mcp.json` |
    | LM Studio | `mcp.json` (depuis les paramètres MCP de l'app) |
  </Step>

  <Step title="Redémarrez votre hôte MCP">
    Votre agent dispose maintenant des outils chat, image, vidéo, musique, TTS, ASR et 25 autres outils Venice.
  </Step>
</Steps>

<Note>
La plupart des hôtes MCP ne transmettent que les variables d'environnement **explicitement listées** dans le bloc `env`. Les variables d'env système ne sont pas héritées. Si vous voyez des erreurs 402 avec une clé API définie, vérifiez que `VENICE_API_KEY` est bien à l'intérieur de `env` dans votre config.
</Note>

## Ce que vous obtenez

**31 outils** couvrant chaque modalité Venice, **3 ressources** (`venice://models`, `venice://styles`, `venice://voices`) et **3 templates de prompts**.

### Chat et embeddings

| Outil | Description |
|---|---|
| `venice_chat` | Chat completion compatible OpenAI sur le catalogue LLM complet de Venice. |
| `venice_responses` | API Responses compatible OpenAI avec support outil mono- ou multi-tour. |
| `venice_embeddings` | Calculer des embeddings pour une entrée texte. |
| `venice_chat_with_character` | Chatter avec un character Venice par slug. |

### Image

| Outil | Description |
|---|---|
| `venice_image_generate` | Générer une image (Flux 2, Lustify SDXL, Anime/WAI, Qwen Image, GPT Image, Nano Banana Pro et plus). |
| `venice_image_edit` | Éditer une image avec un prompt. |
| `venice_image_multi_edit` | Éditer plusieurs images ensemble avec un seul prompt. |
| `venice_image_upscale` | Upscaler une image jusqu'à 4×. |
| `venice_image_remove_bg` | Supprimer l'arrière-plan d'une image. |
| `venice_image_styles` | Lister les presets de style d'image. |

### Vidéo

| Outil | Description |
|---|---|
| `venice_video_generate` | Mettre en file une génération vidéo (Sora 2, Veo 3.1, Kling, Wan, LTX 2, Seedance, Runway Gen-4 et plus). |
| `venice_video_status` | Vérifier le statut d'un job vidéo en file. |
| `venice_video_complete` | Marquer une vidéo terminée comme téléchargée ; supprime les médias côté serveur. |
| `venice_video_transcriptions` | Transcrire une URL vidéo YouTube. |
| `venice_video_quote` | Obtenir un devis de prix avant la mise en file. |

### Audio (TTS / ASR)

| Outil | Description |
|---|---|
| `venice_tts` | Synthèse vocale avec voix clonées et tags d'émotion. |
| `venice_asr` | Transcrire de l'audio depuis une URL. |
| `venice_voice_clone` | Lister les voix intégrées ou cloner une voix depuis un échantillon. |
| `venice_audio_quote` | Obtenir un devis de prix pour la génération musicale. |

### Musique

| Outil | Description |
|---|---|
| `venice_music_generate` | Mettre en file une génération musicale (`ace-step-15`, `elevenlabs-music`, `minimax-music-v2/v25/v26`, `stable-audio-25`, `mmaudio-v2`, `elevenlabs-sound-effects-v2`). |
| `venice_music_status` | Vérifier le statut d'un job musical en file. |
| `venice_music_complete` | Marquer un job musical terminé comme téléchargé. |

### Augment web, catalogue et crypto

| Outil | Description |
|---|---|
| `venice_web_search` | Rechercher sur le web (propulsé par Firecrawl). |
| `venice_web_scrape` | Scraper une URL en markdown. |
| `venice_text_parser` | Extraire du texte depuis PDF/DOCX/EPUB/PPTX/XLSX. |
| `venice_list_models` | Lister le catalogue de modèles en direct avec les prix. |
| `venice_list_characters` | Lister les characters Venice publics. |
| `venice_crypto_rpc` | Proxy d'appels JSON-RPC vers Base, Ethereum, Polygon, Arbitrum ou Optimism. |

### Helpers de portefeuille x402

Pertinent uniquement si vous vous authentifiez avec un portefeuille via [x402](/fr/guides/integrations/x402-venice-api) au lieu d'une clé API.

| Outil | Description |
|---|---|
| `venice_x402_balance` | Vérifier le solde de crédits x402 prépayé pour une adresse de portefeuille EVM ou Solana. |
| `venice_x402_top_up_info` | Récupérer les exigences de rechargement (réseau, token USDC, destinataire, montant min). |
| `venice_x402_transactions` | Lister les transactions x402 récentes de rechargement et de débit pour une adresse EVM ou Solana. |

## Configuration

Le serveur est entièrement configuré via des variables d'environnement.

| Variable d'env | Défaut | Notes |
|---|---|---|
| `VENICE_API_KEY` | _(aucune)_ | Votre clé API Venice. Configuration la plus simple. |
| `VENICE_DEFAULT_CHAT_MODEL` | `venice-uncensored` | |
| `VENICE_DEFAULT_IMAGE_MODEL` | `flux-2-pro` | |
| `VENICE_DEFAULT_TTS_MODEL` | `tts-kokoro` | |
| `VENICE_DEFAULT_ASR_MODEL` | `openai/whisper-large-v3` | |
| `VENICE_DISABLE_NSFW` | `0` | Mettre à `1` pour retirer les notes de capacité NSFW des descriptions d'outils. |
| `VENICE_HTTP_TIMEOUT_MS` | `60000` | |
| `VENICE_SIWX_TOKEN` | _(aucun)_ | Token d'auth en mode portefeuille x402. Voir [x402 ci-dessous](#x402-wallet-mode). |

Si `VENICE_API_KEY` et `VENICE_SIWX_TOKEN` sont tous deux définis, la clé API l'emporte.

## Mode portefeuille x402

Venice prend en charge l'authentification avec un [token de portefeuille Sign-In-With-X](/fr/guides/integrations/x402-venice-api) adossé à un crédit USDC prépayé sur Base ou Solana, en plus du flux normal par clé API. Aucun email, téléphone ou KYC requis : votre portefeuille est la seule identité.

```json
{
  "mcpServers": {
    "venice": {
      "command": "npx",
      "args": ["-y", "@veniceai/mcp-server@0.2.0"],
      "env": { "VENICE_SIWX_TOKEN": "<base64 Sign-In-With-X payload>" }
    }
  }
}
```

Le serveur MCP transmet `VENICE_SIWX_TOKEN` comme en-tête `X-Sign-In-With-X` à chaque appel API Venice. Le serveur ne voit jamais votre clé privée. La signature du portefeuille et les autorisations de rechargement USDC se produisent dans votre propre portefeuille.

| Flux | Ce qui se passe |
|---|---|
| **Configuration unique** | Signer un message Sign-In-With-X dans votre portefeuille → produit un token SIWX (JSON base64). |
| **Recharger** | `POST /api/v1/x402/top-up` renvoie 402 + exigences de paiement. Signez un paiement USDC pour l'une des options Base ou Solana renvoyées, resoumettez, et Venice crédite votre solde. |
| **Chaque appel d'inférence** | Le serveur MCP envoie `X-Sign-In-With-X: <SIWX>` ; Venice débite votre solde prépayé. |

Le rechargement minimum est de **5 $ USD**. Le solde minimum pour appeler l'inférence est de **0,10 $**. Une fois rechargé, les appels sont à moins de 100 ms car le règlement se fait hors chaîne sur un compte de crédit rapide.

<Tip>
Les portefeuilles liés à un compte Venice avec du DIEM staké consomment depuis le solde de staking au lieu des crédits USDC, donc aucun rechargement n'est nécessaire.
</Tip>

## Auto-hébergement (Streamable HTTP)

Pour des déploiements équipe ou workspace, exécutez le serveur MCP via HTTP plutôt que stdio :

```bash
docker run -p 3333:3333 \
  -e VENICE_API_KEY=<your-venice-api-key> \
  -e VENICE_MCP_AUTH_TOKEN=<choose-a-long-random-token> \
  ghcr.io/veniceai/venice-mcp-server:latest
```

Le serveur est désormais disponible sur `http://localhost:3333/mcp`. Les clients HTTP doivent envoyer `Authorization: Bearer <VENICE_MCP_AUTH_TOKEN>`.

<Warning>
`/mcp` est un endpoint d'exécution d'outils adossé à des justificatifs : les appelants peuvent dépenser la clé API Venice configurée ou le solde x402. Lorsque le mode HTTP s'attache hors de loopback, le démarrage échoue à moins que `VENICE_MCP_AUTH_TOKEN` ne soit défini. Pour la production, épinglez explicitement la version du package npm plutôt que de vous reposer sur `latest`.
</Warning>

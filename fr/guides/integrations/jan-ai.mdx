[Jan](https://jan.ai/) est un assistant IA open source pour exécuter des modèles locaux et distants depuis une application de bureau unique. Venice fonctionne dans Jan via sa prise en charge des fournisseurs de modèles distants compatibles OpenAI.

## Prérequis

- Une clé API Venice depuis [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) installé et en cours d'exécution sur votre machine
- Un ID de modèle de texte Venice depuis le [catalogue de modèles](/models/text)

---

## Configuration

<Steps>
  <Step title="Obtenez votre clé API Venice">
    Générez ou copiez une clé API depuis [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Ouvrez les paramètres de fournisseur de modèle de Jan">
    Dans Jan, ouvrez **Settings** et allez dans **Model Providers**.

    Choisissez le flux de fournisseur personnalisé compatible OpenAI de Jan. Selon votre version de Jan, cela peut apparaître comme **OpenAI** avec **Custom Endpoint** activé, ou comme un fournisseur **Any OpenAI Compatible API**.
  </Step>

  <Step title="Configurez l'endpoint Venice">
    Saisissez les paramètres du fournisseur Venice :

    | Champ | Valeur |
    |-------|-------|
    | Nom du fournisseur | `Venice AI` |
    | URL de base | `https://api.venice.ai/api/v1` |
    | Clé API | Votre clé API Venice |
    | Type d'API | Compatible OpenAI |

    Enregistrez ou appliquez les paramètres du fournisseur avant d'ajouter des modèles.
  </Step>

  <Step title="Ajoutez des modèles Venice">
    Ajoutez les ID de modèles Venice que vous souhaitez utiliser. Jan exige que l'ID du modèle corresponde exactement à celui du fournisseur.

    Bonnes options de départ :

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Vous pouvez ajouter n'importe quel modèle de texte pris en charge depuis le [catalogue de modèles](/models/text).
  </Step>

  <Step title="Démarrez un chat">
    Ouvrez un chat Jan nouveau ou existant, sélectionnez l'un de vos modèles Venice dans le sélecteur de modèle et envoyez un message test.

    Jan enverra la requête à Venice en utilisant votre clé API et l'URL de base compatible OpenAI.
  </Step>
</Steps>

## Vérifier la configuration

Envoyez un court prompt depuis Jan :

```txt
Reply with one sentence confirming you are running through Venice.
```

Si le fournisseur est configuré correctement, Jan devrait streamer une réponse du modèle Venice sélectionné.

Vous pouvez également vérifier votre clé API en dehors de Jan avec :

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Notes de configuration

- Utilisez `https://api.venice.ai/api/v1` comme URL de base. N'incluez pas `/chat/completions` ; Jan ajoute le chemin de l'endpoint pour les requêtes de chat.
- Les ID de modèles doivent correspondre exactement à ceux de Venice. Si un modèle n'apparaît pas dans le sélecteur de Jan, ajoutez-le manuellement en utilisant l'ID du modèle depuis le [catalogue de modèles](/models/text).
- Conservez Venice configuré en tant que fournisseur compatible OpenAI. N'utilisez pas le fournisseur Anthropic natif de Jan pour les modèles Venice Claude.
- Les requêtes distantes Venice nécessitent une connexion Internet et consomment des crédits API Venice.

<Info>
Les modèles locaux de Jan restent disponibles après l'ajout de Venice. Vous pouvez basculer entre les modèles locaux et Venice depuis le sélecteur de modèle de chat.
</Info>

## Dépannage

<AccordionGroup>
  <Accordion title="Erreur 401 ou d'authentification">
    Vérifiez que la clé API est une clé Venice depuis [venice.ai/settings/api](https://venice.ai/settings/api), sans espaces supplémentaires avant ou après.
  </Accordion>

  <Accordion title="404 ou modèle introuvable">
    Confirmez que l'ID du modèle dans Jan correspond exactement à un ID de modèle Venice. Vérifiez également que l'URL de base est `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="Pas de modèles Venice dans le sélecteur">
    Ajoutez le modèle manuellement dans les paramètres du fournisseur de Jan. Utilisez l'ID exact du modèle, par exemple `zai-org-glm-5-1`, et enregistrez à nouveau le fournisseur.
  </Accordion>

  <Accordion title="Les requêtes vont toujours vers un autre fournisseur">
    Assurez-vous que le chat utilise un modèle rattaché à votre fournisseur Venice. Si Jan utilise une entrée de modèle OpenAI ou Anthropic intégrée, basculez vers le modèle Venice que vous avez ajouté.
  </Accordion>
</AccordionGroup>

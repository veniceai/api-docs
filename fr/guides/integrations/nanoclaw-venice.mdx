[NanoClaw](https://github.com/qwibitai/nanoclaw) est un assistant IA léger et auto-hébergé qui s'exécute sur WhatsApp et Telegram. Ce fork ajoute la prise en charge de Venice AI pour que tout s'exécute en privé sans abonnement Anthropic.

## Pourquoi Venice AI ?

[Venice](https://venice.ai) est une plateforme IA axée sur la confidentialité. Elle [ne stocke ni ne journalise aucun prompt ou réponse](https://venice.ai/privacy) sur ses serveurs — vos conversations n'existent que sur votre appareil. Les requêtes sont chiffrées de bout en bout via leur proxy vers des fournisseurs GPU décentralisés, avec zéro rétention de données. Cela signifie que vos conversations avec votre assistant IA restent privées, même de Venice elle-même.

Venice fournit un accès anonymisé aux modèles frontier (Claude Opus, Claude Sonnet) et un accès entièrement privé aux modèles open source (GLM, Qwen) via une seule API — basculez entre eux à tout moment.

| | **Venice AI** | **Fournisseurs IA traditionnels** |
|---|---|---|
| **Rétention de données** | Aucune — zéro journal | Oui |
| **Confidentialité du prompt** | Chiffré, jamais stocké | Stocké sur les serveurs du fournisseur |
| **Modèles open source** | Oui (GLM, Qwen et d'autres) | Non |
| **Modèles frontier** | Claude, GPT et d'autres — anonymement | Uniquement via abonnements directs |
| **Tarification** | Paiement par token, sans abonnement. Ou stakez [DIEM](https://venice.ai/lp/diem) pour des crédits quotidiens qui se rechargent | Abonnements à 20-200 $/mois ou API à la consommation |
| **Inférence non censurée** | Oui (modèles open source) | Non |

## Pourquoi NanoClaw ?

NanoClaw est une alternative propre et minimale à des plateformes plus grandes comme OpenClaw. Il est conçu pour une personne exécutant un seul bot.

| | **NanoClaw (Venice)** | **OpenClaw** |
|---|---|---|
| **Code base** | ~2 000 lignes, une poignée de fichiers | ~500 000 lignes, 53 fichiers de config |
| **Dépendances** | ~15 packages | 70+ packages |
| **Modèle de sécurité** | Isolation de conteneurs Docker au niveau OS | Allowlists au niveau application et codes d'appairage |
| **Isolation par groupe** | Chaque groupe a son propre conteneur, système de fichiers et mémoire | Processus partagé, mémoire partagée |
| **Configuration** | Un assistant (`/setup`), ~10 minutes | Configuration manuelle en plusieurs étapes |
| **Fournisseur IA** | Venice AI (privé, sans abonnement) | Anthropic (nécessite clé API ou abonnement) |
| **Personnalisation** | Modifiez le code directement — il est assez petit pour être lu | Fichiers de config et plugins |
| **Utilisateur cible** | Une personne, un bot | Plateforme multi-utilisateurs |

## Ce que vous obtenez

- Assistant IA personnel sur **Telegram** et/ou **WhatsApp**
- Propulsé par **Venice AI** — pas de compte Anthropic nécessaire
- Le bot s'exécute dans un **conteneur Docker isolé** (sandbox, ne peut pas accéder à votre système)
- **Changement de modèle** — dites au bot "switch to zai-org-glm-5" ou "use opus" à tout moment
- **Tâches planifiées** — définissez des rappels, des tâches récurrentes
- **Recherche et navigation web** intégrées
- **Formatage Markdown** dans les messages Telegram

**Pour Telegram** (recommandé pour les premiers utilisateurs) :
1. Ouvrez Telegram et recherchez **@BotFather**
2. Envoyez `/newbot` et suivez les instructions
3. Enregistrez le token que BotFather vous donne (ressemble à `123456789:ABCdef...`)

<Warning>
**Pour WhatsApp — utilisez un numéro virtuel, PAS le vôtre :**

NanoClaw se connecte en tant qu'appareil lié sur votre numéro WhatsApp. Cela signifie que **l'agent peut voir chaque message entrant et sortant** — toutes vos conversations personnelles, discussions de groupe, photos, tout. Votre téléphone continue de fonctionner normalement, mais le bot a une visibilité totale sur votre compte WhatsApp.

**Utilisez plutôt un numéro de téléphone virtuel.** Ces applications vous donnent un deuxième numéro que vous pouvez entièrement dédier au bot :

| App | Prix | Notes |
|-----|-------|-------|
| [Hushed](https://hushed.com) | ~5 $/mois | Fiable, fonctionne bien pour la vérification WhatsApp |
| [Burner](https://www.burnerapp.com) | ~5 $/mois | Similaire à Hushed, numéros jetables |
| [Google Voice](https://voice.google.com) | Gratuit | États-Unis uniquement, peut ne pas fonctionner pour la vérification WhatsApp dans tous les cas |

**Comment le configurer :**
1. Obtenez un numéro virtuel depuis l'une des applications ci-dessus
2. Installez WhatsApp sur un deuxième appareil (ancien téléphone, tablette ou émulateur) en utilisant ce numéro virtuel
3. Pendant la configuration de NanoClaw, scannez le QR code avec ce deuxième appareil — pas votre téléphone personnel
</Warning>

## Configuration

La configuration prend environ 10 minutes. Vous n'avez besoin que d'**une seule fenêtre Terminal**.

<Steps>
  <Step title="Cloner et installer">
    Ouvrez Terminal et exécutez :

    ```bash
    git clone https://github.com/lorenzovenice/nanoclaw-venice.git
    cd nanoclaw-venice
    npm install
    ```

    Attendez que `npm install` se termine sans erreur.
  </Step>

  <Step title="Lancer Claude Code avec Venice">
    Remplacez `your-key` par votre clé API Venice et exécutez :

    ```bash
    VENICE_API_KEY=your-key npm run venice
    ```

    Cela démarre le proxy Venice et lance Claude Code à travers lui en une seule commande.

    <Note>
    Claude Code utilise par défaut **GLM 5** (`zai-org-glm-5`) pour maintenir les coûts de configuration bas. Après la configuration, tapez `/model` dans Claude Code pour passer à `claude-sonnet-4-6` ou `claude-opus-4-6` pour de meilleures performances.
    </Note>

    Si on vous demande "Do you want to use this API key?" — sélectionnez **Yes**.
  </Step>

  <Step title="Lancer l'assistant de configuration">
    Dans votre terminal Claude Code, tapez :

    ```
    /setup
    ```

    L'assistant vous guide à travers :

    1. **Bootstrap** — vérifie Node.js et les dépendances
    2. **Clé API Venice** — valide et enregistre votre clé
    3. **Choix du canal** — choisissez WhatsApp, Telegram ou les deux
    4. **Build du conteneur** — construit le conteneur Docker (prend quelques minutes la première fois)
    5. **Auth WhatsApp** — scannez le QR code avec votre téléphone (si applicable)
    6. **Configuration Telegram** — envoyez un message à votre bot pour qu'il détecte votre chat
    7. **Mot déclencheur** — préfixe qui active le bot (par défaut : `@Andy`)
    8. **Répertoires montés** — choisissez "No" pour l'instant (vous pouvez ajouter l'accès aux fichiers plus tard)
    9. **Démarrer les services** — NanoClaw et le proxy Venice démarrent tous deux comme services en arrière-plan

    L'assistant de configuration installe deux services en arrière-plan :
    - **NanoClaw** — le bot lui-même
    - **Proxy Venice** — un petit serveur local (localhost:4001) qui traduit entre Claude Code et Venice AI

    Les deux démarrent automatiquement au boot et se redémarrent en cas de crash.

    <Note>
    Si l'assistant s'arrête entre les étapes, tapez "continue" ou "next step" pour le pousser à avancer.
    </Note>
  </Step>

  <Step title="Commencer à chatter">
    Une fois la configuration terminée, ouvrez votre chat (Telegram ou WhatsApp) et envoyez :

    ```
    @Andy hello, are you there?
    ```

    Le bot devrait répondre en quelques secondes. Dans votre canal principal, vous pouvez taper normalement sans le préfixe `@Andy`.

    **Vous pouvez maintenant fermer la fenêtre du terminal.** Tout s'exécute en services en arrière-plan et démarre automatiquement quand votre ordinateur boote.
  </Step>
</Steps>

## Comment ça marche

Il y a deux couches dans NanoClaw :

| Couche | Ce qu'elle fait |
|-------|-------------|
| **CLI Claude Code** | Outil admin pour la configuration, le débogage et la personnalisation |
| **Le bot** | IA dans votre chat, s'exécutant dans un conteneur Docker isolé |

Pour ouvrir Claude Code à tout moment :

```bash
cd nanoclaw-venice
ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
```

Utilisez-le pour exécuter `/setup`, `/debug`, `/customize`, ou apporter des modifications au comportement du bot.

## Dépannage

<AccordionGroup>
  <Accordion title="Le proxy ne s'exécute pas">
    Le proxy Venice s'exécute comme service en arrière-plan et se redémarre automatiquement. S'il ne fonctionne pas :

    **macOS :**
    ```bash
    # Check if it's running
    launchctl list | grep venice-proxy

    # Restart it
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Check logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```

    **Linux :**
    ```bash
    # Check if it's running
    systemctl --user status nanoclaw-venice-proxy

    # Restart it
    systemctl --user restart nanoclaw-venice-proxy

    # Check logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```
  </Accordion>

  <Accordion title="Claude Code affiche une erreur 403 ou 'Please run /login'">
    Cela signifie que Claude Code ne peut pas se connecter au proxy Venice.

    1. **Vérifiez que le proxy s'exécute.** Voir l'étape de dépannage ci-dessus.
    2. **Assurez-vous d'être dans le bon dossier.** Toujours `cd nanoclaw-venice` d'abord.
    3. **Repartez à neuf :** fermez tous les terminaux et exécutez :
       ```bash
       cd nanoclaw-venice
       ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
       ```
  </Accordion>

  <Accordion title="Erreurs de modèle ('model does not exist')">
    Redémarrez le proxy et le bot :

    **macOS :**
    ```bash
    # Restart proxy
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Restart bot
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw
    ```

    **Linux :**
    ```bash
    # Restart proxy
    systemctl --user restart nanoclaw-venice-proxy

    # Restart bot
    systemctl --user restart nanoclaw
    ```

    Consultez les modèles disponibles dans le [catalogue de modèles](/fr/models/text).
  </Accordion>

  <Accordion title="Le bot ne répond pas aux messages">
    Suivez ces étapes dans l'ordre :

    1. **Vérifiez votre mot déclencheur.** Assurez-vous d'utiliser le bon préfixe (par ex. `@Andy hello`).
    2. **Vérifiez que Docker s'exécute.** Exécutez `docker info` — s'il erre, ouvrez Docker Desktop.
    3. **Vérifiez que le proxy s'exécute.** Voir "Le proxy ne s'exécute pas" ci-dessus.
    4. **Vérifiez les logs :** `tail -f logs/nanoclaw.log` dans le dossier du projet.
    5. **Vérifiez les logs du conteneur.** Ouvrez le dossier `nanoclaw-venice/groups/main/logs/`. Ouvrez le fichier le plus récent qui commence par `container-`.
    6. **Redémarrez tout :** redémarrez à la fois le proxy et le bot (voir ci-dessus).
  </Accordion>

  <Accordion title="Le build du conteneur échoue pendant la configuration">
    Assurez-vous que Docker Desktop est ouvert et en cours d'exécution. Attendez 10 secondes que Docker démarre complètement, puis tapez `continue` dans l'assistant pour réessayer.
  </Accordion>

  <Accordion title="WhatsApp déconnecté">
    Votre session WhatsApp peut expirer. Pour reconnecter :

    ```bash
    cd nanoclaw-venice
    npm run auth
    ```

    Scannez le QR code avec WhatsApp (Settings → Linked Devices → Link a Device), puis redémarrez le bot :
    - macOS : `launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
    - Linux : `systemctl --user restart nanoclaw`
  </Accordion>
</AccordionGroup>

## Avancé

<AccordionGroup>
  <Accordion title="Donner au bot l'accès aux fichiers de votre ordinateur">
    Par défaut, le bot est complètement isolé de votre ordinateur — il ne peut voir que sa propre mémoire et son historique de conversation.

    - **Pendant la configuration :** lorsque l'accès aux répertoires est demandé, choisissez "Yes"
    - **Après la configuration :** exécutez `/customize` dans Claude Code
  </Accordion>

  <Accordion title="Démarrer/arrêter manuellement le bot">
    NanoClaw exécute deux services en arrière-plan qui démarrent automatiquement au boot.

    **macOS :**
    | Action | Commande |
    |--------|---------|
    | Démarrer le bot | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Arrêter le bot | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Redémarrer le bot | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw` |
    | Démarrer le proxy | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Arrêter le proxy | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Redémarrer le proxy | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy` |

    **Linux :**
    | Action | Commande |
    |--------|---------|
    | Démarrer le bot | `systemctl --user start nanoclaw` |
    | Arrêter le bot | `systemctl --user stop nanoclaw` |
    | Redémarrer le bot | `systemctl --user restart nanoclaw` |
    | Démarrer le proxy | `systemctl --user start nanoclaw-venice-proxy` |
    | Arrêter le proxy | `systemctl --user stop nanoclaw-venice-proxy` |
    | Redémarrer le proxy | `systemctl --user restart nanoclaw-venice-proxy` |
  </Accordion>

  <Accordion title="Utiliser Claude Code via Venice (sans bot)">
    Si vous voulez juste Claude Code avec Venice et n'avez pas besoin de WhatsApp/Telegram, le service proxy doit être en cours d'exécution. Si vous avez déjà exécuté `/setup`, il s'exécute déjà comme service en arrière-plan.

    ```bash
    cd nanoclaw-venice
    ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
    ```

    **Astuce :** ajoutez ceci à votre `~/.zshrc` (ou `~/.bashrc`) pour pouvoir rapidement basculer n'importe quel terminal vers Venice :

    ```bash
    alias venice='export ANTHROPIC_BASE_URL=http://localhost:4001 && export ANTHROPIC_API_KEY=venice-proxy && echo "Using Venice API"'
    alias anthropic='unset ANTHROPIC_BASE_URL && unset ANTHROPIC_API_KEY && echo "Using Anthropic API"'
    ```

    Puis tapez simplement `venice` dans n'importe quel terminal avant d'exécuter `claude` pour utiliser Venice, ou `anthropic` pour revenir en arrière.
  </Accordion>

  <Accordion title="Exécuter plusieurs bots">
    Vous pouvez exécuter plusieurs bots NanoClaw sur la même machine (par ex. un pour usage personnel et un pour une équipe). Clonez simplement le repo dans un dossier différent et exécutez à nouveau la configuration. Note : ils partagent la même image Docker, donc reconstruire l'un affecte tous les autres.
  </Accordion>

  <Accordion title="Commandes développeur">
    Pour les personnes qui veulent modifier le code de NanoClaw :

    ```bash
    npm run dev          # Start proxy + NanoClaw with hot reload
    npm run proxy        # Start just the Venice proxy
    npm run build        # Compile TypeScript
    npm test             # Run tests
    ./container/build.sh # Rebuild agent container
    ```
  </Accordion>
</AccordionGroup>

## Architecture

```
You (WhatsApp/Telegram)
        ↓
   NanoClaw (Node.js)
        ↓
   Docker Container (isolated sandbox)
        ↓
   Venice Proxy (localhost:4001)
        ↓
   api.venice.ai (private inference)
```

| Fichier | Objectif |
|------|---------|
| `proxy/venice-proxy.ts` | Traduit le format Anthropic en format OpenAI pour Venice |
| `src/index.ts` | Orchestrateur principal — boucle de messages, invocation d'agent |
| `src/channels/whatsapp.ts` | Connexion WhatsApp via baileys |
| `src/channels/telegram.ts` | Bot Telegram via grammy |
| `src/container-runner.ts` | Spawn des conteneurs d'agent isolés |

## FAQ

<AccordionGroup>
  <Accordion title="Pourquoi ai-je besoin d'un proxy ?">
    Le Claude Agent SDK parle le format de messages d'Anthropic. Venice parle le format d'OpenAI. Le proxy traduit entre eux pour que tout fonctionne sans modifier le SDK.
  </Accordion>

  <Accordion title="Puis-je utiliser des modèles open source ?">
    Oui. Venice héberge de nombreux modèles. Dites au bot "switch to zai-org-glm-5" ou n'importe quel ID de modèle Venice. Voir le [catalogue de modèles](/fr/models/text).
  </Accordion>

  <Accordion title="Est-ce sécurisé ?">
    Les agents s'exécutent dans des conteneurs Docker avec une véritable isolation au niveau OS. La clé API Venice est passée via stdin, jamais écrite sur disque dans les conteneurs. Chaque groupe a son propre environnement isolé.
  </Accordion>

  <Accordion title="Ai-je besoin d'un abonnement Anthropic ?">
    Non. Tout s'exécute via Venice AI. Vous n'avez besoin que d'une clé API Venice.
  </Accordion>

  <Accordion title="Puis-je l'utiliser sur un serveur ?">
    Oui. Cela fonctionne sur n'importe quelle machine Linux avec Docker. Utilisez le service systemd pour le démarrage automatique au boot.
  </Accordion>
</AccordionGroup>

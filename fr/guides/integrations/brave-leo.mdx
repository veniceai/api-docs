[Brave Leo](https://brave.com/leo/) est l'assistant IA intégré au navigateur Brave. La fonctionnalité Bring Your Own Model (BYOM) de Leo peut se connecter directement à l'endpoint de chat completions compatible OpenAI de Venice, vous permettant d'utiliser les modèles Venice pour le chat et l'assistance contextuelle sur les pages sans installer d'extension.

## Prérequis

- Brave Browser 1.69 ou ultérieur sur Windows, macOS ou Linux
- Une clé API Venice depuis [venice.ai/settings/api](https://venice.ai/settings/api)
- Un ID de modèle depuis le [catalogue de modèles de texte Venice](/models/text)

<Note>
La fonctionnalité BYOM de Brave est disponible sur ordinateur. L'ID du modèle doit correspondre exactement à un ID de modèle Venice.
</Note>

## Configuration

<Steps>
  <Step title="Ouvrir les paramètres de Leo">
    Ouvrez `brave://settings/leo-ai`, ou ouvrez Leo depuis la barre latérale de Brave, sélectionnez l'icône des paramètres et accédez à **Leo**.
  </Step>

  <Step title="Ajouter un modèle personnalisé">
    Faites défiler jusqu'à **Bring your own model** et sélectionnez **Add new model**.
  </Step>

  <Step title="Configurer l'endpoint Venice">
    Entrez les valeurs suivantes :

    | Champ | Valeur |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | Un ID de modèle depuis le [catalogue de modèles de texte](/models/text) |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | Votre clé API Venice |

    Si Brave affiche **Context size**, réglez-le sur la fenêtre de contexte prise en charge par votre modèle sélectionné. Vous pouvez également ajouter un prompt système optionnel.

    Entrez la clé API elle-même sans préfixe `Bearer`. Brave ajoute l'en-tête d'autorisation à chaque requête.
  </Step>

  <Step title="Enregistrer et sélectionner Venice">
    Sélectionnez **Add model**, puis définissez **Venice AI** comme modèle par défaut pour les nouvelles conversations ou choisissez-le depuis le sélecteur de modèle de Leo.
  </Step>

  <Step title="Tester la connexion">
    Ouvrez Leo sur une page web et envoyez un prompt tel que :

    ```txt
    Summarize this page in five bullet points.
    ```

    Leo envoie la conversation et le contenu pertinent de la page directement à l'endpoint Venice.
  </Step>
</Steps>

## Activer la recherche web Venice

Brave n'expose pas les paramètres de corps de requête spécifiques à Venice dans BYOM. Pour activer la recherche web Venice, ajoutez un [suffixe de fonctionnalité de modèle](/api-reference/endpoint/chat/model_feature_suffix) à l'ID du modèle dans **Model request name** :

```txt
<model-id>:enable_web_search=auto
```

Pour toujours activer la recherche et inclure des citations, utilisez :

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
La recherche web comporte une tarification additionnelle basée sur l'usage. Consultez la [tarification API](/overview/pricing) pour les tarifs actuels.
</Info>

## Confidentialité

Lorsque vous utilisez un endpoint BYOM distant, Brave se connecte directement à ce fournisseur. Les requêtes contournent les serveurs de Brave, donc les protections de proxy et de rétention côté serveur de Brave ne s'appliquent pas à ces requêtes.

Venice ne stocke ni ne journalise le contenu des prompts et des réponses au niveau de son proxy. Le modèle sélectionné détermine le niveau de confidentialité à l'exécution :

- Les modèles **Private** utilisent une infrastructure sans rétention de données.
- Les modèles **Anonymized** masquent les informations d'identification du fournisseur en amont, mais le fournisseur peut traiter le prompt.
- Les modèles **TEE** et **E2EE** ajoutent une isolation matérielle ou un chiffrement côté client lorsqu'ils sont pris en charge.

Consultez l'étiquette de confidentialité du modèle dans le [catalogue de modèles de texte](/models/text) et lisez la [confidentialité Venice](/overview/privacy) avant d'envoyer des informations sensibles.

Brave peut conserver l'historique du chat de Leo dans le stockage local du navigateur. Vous pouvez effacer les données de Leo, utiliser un chat temporaire ou désactiver l'historique local du chat dans les paramètres Leo de Brave.

## Notes de configuration

- Brave requiert le chemin d'endpoint complet : `https://api.venice.ai/api/v1/chat/completions`.
- Les ID de modèles et leur disponibilité évoluent au fil du temps. Copiez l'ID actuel depuis le [catalogue de modèles de texte](/models/text).
- La taille du contexte ne doit pas dépasser la fenêtre de contexte prise en charge par le modèle sélectionné.
- Les résumés de pages peuvent envoyer le contenu de la page à Venice dans le cadre du prompt.
- Les requêtes Venice distantes nécessitent une connexion internet et consomment des crédits API Venice ou votre allocation DIEM disponible.

## Dépannage

<AccordionGroup>
  <Accordion title="Erreur 401 ou d'authentification">
    Vérifiez que la clé provient de [venice.ai/settings/api](https://venice.ai/settings/api). Collez la clé brute sans `Bearer` et supprimez tout espace en début ou fin.
  </Accordion>
  <Accordion title="Modèle introuvable">
    Copiez l'ID du modèle exactement depuis le [catalogue de modèles de texte](/models/text). Si vous avez ajouté un suffixe de fonctionnalité, testez d'abord le modèle sans le suffixe.
  </Accordion>
  <Accordion title="Erreur réseau ou pas de réponse en streaming">
    Vérifiez que l'endpoint du serveur inclut `/api/v1/chat/completions`. Brave Leo attend une réponse en streaming compatible OpenAI.
  </Accordion>
  <Accordion title="Les résumés de pages manquent de contexte">
    Vérifiez l'autorisation de Leo à utiliser le contenu de la page et assurez-vous que la page actuelle est accessible au navigateur. Les pages internes de Brave et certaines pages protégées ne peuvent pas être résumées.
  </Accordion>
</AccordionGroup>

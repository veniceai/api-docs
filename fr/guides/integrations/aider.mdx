[Aider](https://aider.chat/) est un outil de pair-programming IA qui édite votre dépôt git local depuis le terminal. Venice fonctionne grâce à la prise en charge de l'[API compatible OpenAI](https://aider.chat/docs/llms/openai-compat.html) d'Aider — définissez l'URL de base et la clé API Venice, puis préfixez l'ID du modèle avec `openai/`.

## Prérequis

- Une clé API Venice depuis [venice.ai/settings/api](https://venice.ai/settings/api)
- Python 3.9+ disponible sur votre machine
- Un dépôt git dans lequel travailler

## Configuration

<Steps>
  <Step title="Installez Aider">
    Installez Aider avec l'installateur officiel :

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    Consultez la [documentation d'installation d'Aider](https://aider.chat/docs/install.html) si vous préférez une autre méthode d'installation.
  </Step>

  <Step title="Configurez Venice comme endpoint OpenAI">
    Pointez Aider vers Venice en définissant l'URL de base compatible OpenAI et votre clé API Venice.

    **Mac / Linux :**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell) :**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (invite de commandes, persisté) :**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    Redémarrez le terminal après `setx` pour que les nouvelles variables soient disponibles.
  </Step>

  <Step title="Démarrez Aider avec un modèle Venice">
    Depuis le répertoire de votre projet, préfixez l'ID de modèle Venice avec `openai/` :

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    Autres bons modèles de départ :

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    Parcourez tous les ID de modèles de texte dans le [catalogue de modèles](/models/text).
  </Step>
</Steps>

## Persister la configuration avec `.env`

Pour une configuration locale au projet, créez un fichier `.env` à la racine de votre dépôt git :

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

Puis démarrez Aider normalement :

```bash
aider
```

Aider lit également `.env` depuis votre répertoire personnel. Les valeurs à la racine du dépôt ont priorité sur celles du répertoire personnel. Gardez ce fichier hors de git — ajoutez `.env` à `.gitignore` s'il n'est pas déjà ignoré.

<Info>
Vous pouvez également définir `AIDER_OPENAI_API_BASE` et `AIDER_OPENAI_API_KEY`. La documentation officielle d'Aider documente toujours `OPENAI_API_BASE` et `OPENAI_API_KEY` pour les fournisseurs compatibles OpenAI.
</Info>

---

## Persister la configuration avec YAML

Vous pouvez stocker les mêmes paramètres dans `.aider.conf.yml` à la racine de votre dépôt ou dans votre répertoire personnel :

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

Puis exécutez :

```bash
aider
```

## Vérifier la configuration

Une fois Aider démarré, demandez-lui quelque chose de simple qui prouve que le modèle est joignable :

```txt
Reply with one sentence confirming you are running through Venice.
```

Si l'endpoint et la clé sont corrects, Aider stream une réponse depuis le modèle Venice sélectionné.

Vous pouvez aussi confirmer la clé API en dehors d'Aider :

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## Choisir des modèles

| Cas d'usage                     | Exemple d'ID de modèle | Flag Aider                         |
| ------------------------------- | ---------------------- | ---------------------------------- |
| Codage / refactoring performant | `zai-org-glm-5-1`      | `--model openai/zai-org-glm-5-1`   |
| Itération rapide                | `minimax-m25`          | `--model openai/minimax-m25`       |
| Codage à long contexte          | `kimi-k2-5`            | `--model openai/kimi-k2-5`         |
| Chat non censuré                | `venice-uncensored`    | `--model openai/venice-uncensored` |

Préfixez toujours les ID de modèles Venice avec `openai/` lors du lancement d'Aider. Sans ce préfixe, Aider ne routera pas la requête via le chemin du fournisseur compatible OpenAI.

Découvrez les ID de modèles actuels à l'exécution :

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

Ou consultez [GET /models/traits](/api-reference/endpoint/models/traits) pour les valeurs par défaut telles que `text:default`.

## Notes de configuration

- Gardez `OPENAI_API_BASE` défini à `https://api.venice.ai/api/v1`. N'ajoutez pas `/chat/completions` ; Aider ajoute lui-même le chemin de chat.
- Utilisez une clé API Venice dans `OPENAI_API_KEY`. N'utilisez pas de clé OpenAI avec l'URL de base Venice.
- Aider peut avertir que le modèle a une fenêtre de contexte ou un coût inconnu. Cet avertissement est attendu pour de nombreux ID de modèles Venice et peut généralement être ignoré. Voir [avertissements de modèle Aider](https://aider.chat/docs/llms/warnings.html).
- Préférez des modèles de codage/raisonnement performants pour les éditions multi-fichiers. Les modèles plus petits ou non spécialisés en codage peuvent avoir des difficultés avec les formats d'édition d'Aider.

<Warning>
Définir `OPENAI_API_BASE` et `OPENAI_API_KEY` dans votre shell affecte aussi d'autres outils qui lisent ces variables. Préférez un `.env` de projet ou un `.aider.conf.yml` si vous utilisez OpenAI ailleurs dans le même environnement.
</Warning>

## Dépannage

<AccordionGroup>
  <Accordion title="Erreur 401 ou d'authentification">
    Vérifiez que `OPENAI_API_KEY` est une clé Venice provenant de [venice.ai/settings/api](https://venice.ai/settings/api), sans espace superflu. Sur Windows après `setx`, redémarrez le terminal.
  </Accordion>
  <Accordion title="Modèle introuvable ou réponse vide">
    Vérifiez que l'ID de modèle existe dans le [catalogue de modèles de texte](/models/text) et que vous avez lancé Aider avec le préfixe `openai/`, par exemple `openai/zai-org-glm-5-1`.
  </Accordion>
  <Accordion title="Les requêtes vont toujours vers OpenAI">
    Vérifiez que `OPENAI_API_BASE` vaut exactement `https://api.venice.ai/api/v1` dans la même session shell. Affichez-le avec `echo $OPENAI_API_BASE` (ou `$env:OPENAI_API_BASE` dans PowerShell).
  </Accordion>
  <Accordion title="Avertissement de fenêtre de contexte / coûts inconnus">
    Attendu pour de nombreux modèles Venice. Aider retombe sur des valeurs par défaut raisonnables. Pour le faire taire, ajoutez des métadonnées de modèle comme décrit dans les [paramètres avancés de modèles Aider](https://aider.chat/docs/config/adv-model-settings.html).
  </Accordion>
</AccordionGroup>

Le [Venice Video Harness](https://github.com/jordanurbs/venice-video-harness) est une boîte à outils communautaire, agent-first et optimisée pour Venice pour **la création vidéo orientée cohérence à toute longueur**. Il transforme un agent IDE (Claude Code, Cursor, Codex, etc.) en opérateur d'un système de production Venice réutilisable couvrant plus de 50 modèles vidéo, image, audio et musique de Venice.

## Ce que c'est

La plupart des intégrations Venice sont de minces wrappers autour d'appels API. Le Venice Video Harness est la **couche de niveau supérieur** qui se place entre votre agent et l'API Venice :

- **Règles d'orchestration** dans `CLAUDE.md`
- **Playbooks réutilisables** dans `.claude/commands/` (19 commandes de workflow)
- **Agents spécialisés** dans `.claude/agents/` (art-director, prompt-engineer, cut-qa, et plus)
- **Skills de production Venice** dans `.claude/skills/` (compatibles avec le format [Agent Skills](/fr/guides/integrations/venice-skills))
- **Couche d'exécution TypeScript** dans `src/`
- **Registre de modèles complet** couvrant plus de 50 modèles vidéo, image, audio et musique de Venice

Construit pour les créateurs qui produisent :

- Projets vidéo cohérents en personnages (tout genre, toute longueur)
- Séries ou campagnes au style visuel verrouillé
- Workflows storyboard-to-video
- Contenu narratif court et long format
- Séquences cinématiques de marque, bandes-annonces et teasers
- Séries sociales à personnages récurrents

## Ce qui est optimisé pour Venice

- **Prompts d'image réglés pour les modèles d'image Venice** comme `seedream-v5-lite`, `nano-banana-pro`, `flux-2-pro/max` et plus
- **Génération de panels en deux passes** avec raffinement multi-edit Venice pour la correction des personnages
- **Logique de routage de modèle** pour les paliers action, ambiance et cohérence des personnages
- **Génération vidéo consciente des références** qui utilise `elements`, `reference_image_urls` et `scene_image_urls` correctement selon le modèle
- **Adaptation de prompt consciente de l'environnement** pour la gestion des scènes jour vs nuit
- **Chemins audio natifs Venice** pour TTS (Kokoro, Qwen3, ElevenLabs), SFX et musique
- **Estimation des coûts** avant la génération via `/video/quote` et `/audio/quote`
- **Construction de paramètres consciente du modèle** qui ignore automatiquement les paramètres que le modèle cible ne prend pas en charge

## Valeurs par défaut de routage de modèle

Les valeurs par défaut du harness sont assumées car la cohérence est l'objectif. Le routage actuel (avril 2026) :

**Seedance 2.0 R2V par défaut. Kling O3 R2V en fallback pour les scènes à 3+ personnages. Seedance 2.0 i2v pour les plans d'établissement.**

| Rôle | Modèle par défaut | Quand utilisé |
|---|---|---|
| Plans de personnages (1-2 personnages) | `seedance-2-0-reference-to-video` | R2V par défaut avec `reference_image_urls` plat, tags `@Image`, jusqu'à 15 s, audio stéréo natif |
| Plans de personnages (3+ personnages) | `kling-o3-standard-reference-to-video` | Fallback auto avec `elements` structurés pour l'identité multi-personnages |
| Établissement / ambiance / action | `seedance-2-0-image-to-video` | Pas de personnages ; qualité cinématique épique, jusqu'à 15 s |

Ce sont des valeurs surchargeables par projet via `series.json → videoDefaults`. Pour cibler une famille non-Seedance (par ex. comptes sans accès Seedance), définissez `videoDefaults` sur `kling-o3-standard-reference-to-video` et `veo3.1-fast-image-to-video`.

<Note>
**Règle visage Seedance :** Seedance 2.0 bloque les images d'entrée porteuses de visage qui n'ont pas été produites par `seedream-v5-lite` ou `seedream-v5-lite-edit`. Le harness gère cela automatiquement en routant le travail d'image porteuse de personnage via Seedream et en exécutant une porte pré-vol avant chaque appel Seedance.
</Note>

## Pipelines de production

### Pipeline de génération

Vidéo narrative de bout en bout (script → storyboard → vidéo → audio → assemblage) :

```bash
npm run dev -- produce-episode -p output/my-series -e 1
```

L'implémentation de référence dans `src/mini-drama/` couvre :

- Gestion série / personnage / épisode
- Workshop de script propulsé par LLM
- Génération de storyboard en deux passes (génération + raffinement multi-edit)
- QA des panels basé sur la vision
- Génération vidéo avec chaînage de frames
- Post-production audio en couches
- Burn-in de sous-titres et assemblage final

### Pipeline d'édition

Cut des médias déjà existants (plans générés par Venice ou rushes réels). **Text-first** : le LLM lit un `takes_packed.md` compact (~12 Ko par 40 min d'audio) plutôt que de dump des frames vidéo.

Les cinq étapes :

<Steps>
  <Step title="Transcrire">
    whisper.cpp local produit `*.words.json` par source + `takes_packed.md`.
  </Step>
  <Step title="Lire le pack">
    Le LLM forme une stratégie de coupe à partir du texte seul.
  </Step>
  <Step title="Confirmer">
    Propose la stratégie et attend "yes / revise / cancel".
  </Step>
  <Step title="Rendre l'EDL">
    Liste de coupe JSON → ffmpeg concat avec des fades audio de 30 ms. Archive-first, donc les originaux ne sont jamais écrasés.
  </Step>
  <Step title="Self-eval">
    L'agent `cut-qa` exécute 6 vérifications programmatiques à chaque limite de coupe ; max 3 itérations de correction.
  </Step>
</Steps>

Les vérifications `cut-qa` détectent les régressions de ratio d'aspect, les sauts de hash de frame à l'intérieur d'un mot, la troncation VO, la discontinuité de l'éclairage, les pics audio au-dessus de -6 dBFS et le chevauchement des sous-titres avec le texte en frame.

<Tip>
Le pipeline d'édition est inspiré de [browser-use/video-use](https://github.com/browser-use/video-use). Leur insight clé, *"le LLM ne regarde jamais la vidéo, il la lit"*, est ce qui fait fonctionner l'édition pilotée par agent sans se noyer dans des tokens de frame-dump.
</Tip>

## Commandes, agents et skills

Le harness expose 19 commandes de workflow, 10 agents spécialisés et 7 skills de production. Points saillants :

| Commande de workflow | Objectif |
|---|---|
| `new-series` | Créer une nouvelle série avec esthétiques verrouillées |
| `add-character` / `lock-character` | Verrouillage personnage + voix |
| `workshop-episode` | Scripting d'épisode collaboratif |
| `storyboard-episode` | Storyboard d'un épisode |
| `produce-episode` | Pipeline complet en une commande |
| `generate-trailer` | Pipeline complet de bande-annonce |
| `edit-footage` | Pipeline d'édition text-first pour médias existants |
| `ingest-screenplay` | Ingérer un scénario Fountain ou PDF |

| Agent spécialisé | Rôle |
|---|---|
| `art-director` | Décisions esthétique, palette, éclairage, composition |
| `prompt-engineer` | Prompts d'image Venice, cohérence des personnages |
| `storyboard-qa` | QA de panels pour la continuité et les vérifications de personnages |
| `cut-qa` | Porte de qualité post-rendu (6 vérifications par coupe, max 3 itérations) |
| `overlay-designer` | Motion graphics de marque, sous-agents parallèles |
| `trailer-curator` | Sélection de plans de bande-annonce et règles anti-spoiler |

| Skill de production | Objectif |
|---|---|
| `venice-api` | Utilisation de l'API REST Venice et valeurs par défaut |
| `venice-video-model-routing` | Routage R2V-first, arbres de décision |
| `character-consistency` | Conseils de cohérence des personnages multi-plans |
| `shot-composition` | Conseils de composition de plan et caméra |
| `screenplay-parsing` | Workflows de parsing de scénario |
| `video-editing` | Philosophie d'édition text-first, format EDL, boucle cut-qa |

## Aller-retour NLE

Après le rendu, exportez la timeline assemblée en XML pour des ajustements fins dans votre éditeur de choix. Chaque segment vidéo, clip de dialogue, clip SFX et indicateur musical atterrit sur sa propre piste.

```bash
mini-drama export-timeline -p output/<project> -e 1 --format fcpxml      # Final Cut Pro X
mini-drama export-timeline -p output/<project> -e 1 --format premiere    # Premiere Pro
mini-drama export-timeline -p output/<project> -e 1 --format davinci     # DaVinci Resolve
```

## Utilisation programmatique

Vous pouvez aussi appeler les modules du harness directement depuis votre propre TypeScript :

```typescript
import { VeniceClient } from './src/venice/client.js';
import { generateVideo, quoteVideo } from './src/venice/video.js';
import { listVideoModels } from './src/venice/models.js';

const client = new VeniceClient();

const quote = await quoteVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  duration: '8s',
  audio: true,
});
console.log(`Estimated cost: $${quote.quote}`);

const result = await generateVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  prompt: 'A slow dolly shot pushes forward...',
  duration: '8s',
  imageUrl: 'data:image/png;base64,...',
  audio: true,
  outputPath: 'output/shot-001.mp4',
});

const longModels = listVideoModels({ minDurationSec: 20 });
```

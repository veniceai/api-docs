[Hermes Agent](https://hermes-agent.nousresearch.com) est un agent IA open source auto-hébergé construit par [Nous Research](https://nousresearch.com). Il dispose d'une mémoire persistante, de la création autonome de skills et d'une boucle d'apprentissage intégrée qui devient plus performante plus elle s'exécute longtemps. Pointez-le vers l'API Venice et votre agent obtient l'accès à plus de 230 modèles et outils à travers texte, image, vidéo, audio, embeddings et plus.

## Pourquoi Venice + Hermes Agent ?

L'API Venice donne à votre Hermes Agent un accès à l'ensemble de la plateforme Venice via un seul endpoint compatible OpenAI.

| Capacité | Ce que vous obtenez |
|---|---|
| **Texte et chat** | Modèles privés et anonymisés (GLM, Qwen, Claude, GPT, et plus) |
| **Image** | Génération, édition, agrandissement et suppression d'arrière-plan |
| **Vidéo** | Génération et transcription |
| **Audio** | Synthèse vocale (TTS), génération de musique et transcription |
| **Embeddings** | Embeddings vectoriels pour RAG et recherche sémantique |
| **Outils** | Web scraping, recherche web, analyse de texte et RPC crypto |

## Configuration

### 1. Installer Hermes Agent

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>

  <Tab title="WSL2 (Windows)">
    Installez [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install) d'abord, puis exécutez la même commande :
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>
</Tabs>

Après l'installation, rechargez votre shell :

```bash
source ~/.zshrc   # ou : source ~/.bashrc
```

### 2. Configurer Venice comme votre fournisseur

Exécutez l'assistant de configuration de modèle :

```bash
hermes model
```

Sélectionnez **Custom endpoint (self-hosted / VLLM / etc.)** et entrez les éléments suivants lorsqu'on vous le demande :

| Champ | Valeur |
|---|---|
| **URL de base de l'API** | `https://api.venice.ai/api/v1` |
| **Clé API** | Votre clé API Venice |
| **Nom du modèle** | Un ID de modèle Venice (voir ci-dessous) |

Obtenez une clé API depuis [venice.ai/settings/api](https://venice.ai/settings/api) si vous n'en avez pas encore.

<Tip>
Vous pouvez également configurer Venice directement dans `~/.hermes/config.yaml` :

```yaml
model:
  default: zai-org-glm-5
  provider: custom
  base_url: https://api.venice.ai/api/v1
  api_key: ${VENICE_API_KEY}
```

Et définir la clé dans `~/.hermes/.env` :

```bash
VENICE_API_KEY=your-key-here
```
</Tip>

### 3. Choisir un modèle

Lorsque l'assistant demande un modèle, choisissez-en un en fonction de votre cas d'usage :

| Cas d'usage | Modèle | Confidentialité |
|---|---|---|
| Général | `zai-org-glm-5` | Private |
| Raisonnement | `kimi-k2-6` | Private |
| Codage | `claude-opus-4-7` | Anonymized |
| Vision | `z-ai-glm-5v-turbo` | Anonymized |
| Non censuré | `venice-uncensored-1-2` | Private |

Changez votre modèle à tout moment sans redémarrer :

```bash
hermes model              # assistant complet
```

Ou changez en cours de session :

```text
/model custom:claude-opus-4-7
```

### 4. Commencer à discuter

Ouvrez la CLI classique ou la TUI moderne :

```bash
hermes            # CLI classique
hermes --tui      # TUI moderne (recommandé)
```

Essayez un prompt pour vérifier que tout fonctionne :

```text
Summarize this repo in 5 bullets and tell me what the main entrypoint is.
```

## Connecter des plateformes de messagerie

Une fois la CLI fonctionnelle, connectez vos applications de messagerie via la passerelle :

```bash
hermes gateway setup
```

Cela vous guide pour connecter Telegram, Discord, Slack, WhatsApp, Signal et d'autres plateformes. Votre agent devient accessible depuis n'importe quel canal connecté, le tout alimenté par Venice.

```bash
hermes gateway      # démarrer la passerelle de messagerie
```

## Modes de confidentialité

Les modèles Venice dans Hermes Agent suivent les mêmes [niveaux de confidentialité](/overview/privacy) que l'API Venice :

- Les modèles **Private** (GLM, Qwen, DeepSeek, Llama, Venice Uncensored) s'exécutent sur la flotte GPU de Venice. Les prompts ne sont jamais stockés ni journalisés.
- Les modèles **Anonymized** (Claude, GPT, Gemini, Grok) sont transmis via Venice avec toutes les informations d'identification supprimées. Le fournisseur tiers voit Venice comme le client, pas vous.

## Skills API Venice

Hermes Agent dispose d'un système de skills intégré compatible avec le format [Agent Skills](https://github.com/veniceai/skills). Venice publie des skills officiels qui apprennent à votre agent à utiliser chaque endpoint Venice (chat, génération d'image, vidéo, audio, embeddings, outils augment et plus).

Installez les skills Venice directement depuis GitHub :

```bash
hermes skills install veniceai/skills
```

Ou recherchez des skills individuels :

```bash
hermes skills search venice
```

Hermes découvrira chaque skill par son frontmatter `SKILL.md` et le chargera à la demande.

## Commandes clés

| Commande | Description |
|---|---|
| `hermes` | Démarrer la discussion |
| `hermes model` | Changer de fournisseur ou de modèle |
| `hermes tools` | Configurer les outils activés |
| `hermes gateway` | Démarrer la passerelle de messagerie |
| `hermes --continue` | Reprendre votre dernière session |
| `hermes doctor` | Diagnostiquer les problèmes |
| `hermes update` | Mettre à jour vers la dernière version |

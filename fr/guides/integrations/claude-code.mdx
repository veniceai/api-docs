[Claude Code](https://docs.anthropic.com/en/docs/claude-code) est l'outil CLI d'Anthropic pour le codage agentique. Ce guide vous montre comment l'exécuter via Venice AI pour un accès à la consommation à Claude Opus 4.5/4.6 et Sonnet 4.5/4.6.

## Pourquoi vous avez besoin d'un router

Claude Code se connecte directement à l'API d'Anthropic par défaut. Pour l'utiliser avec Venice, vous avez besoin de [claude-code-router](https://github.com/musistudio/claude-code-router), un proxy local open source qui :

<Steps>
  <Step title="Intercepte" icon="hand">
    Capture les requêtes sortantes de Claude Code avant qu'elles n'atteignent Anthropic
  </Step>
  <Step title="Transforme" icon="arrows-rotate">
    Convertit le format de requête et mappe les IDs de modèle (par ex. `claude-opus-4-5`)
  </Step>
  <Step title="Redirige" icon="route">
    Transfère les requêtes vers Venice à `api.venice.ai/api/v1/chat/completions`
  </Step>
</Steps>

## Configuration

<Steps>
  <Step title="Installez Claude Code">
    Si ce n'est pas déjà fait, installez le CLI Claude Code d'Anthropic :

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="Installez le router">
    ```bash
    npm install -g @musistudio/claude-code-router
    ```
  </Step>

  <Step title="Obtenez votre clé API">
    Générez une clé depuis [venice.ai/settings/api](https://venice.ai/settings/api). Vous la collerez directement dans le fichier de config à l'étape suivante.
  </Step>

  <Step title="Créez la configuration">
    Créez le répertoire de configuration :

    ```bash
    mkdir -p ~/.claude-code-router
    ```

    Ensuite, créez `~/.claude-code-router/config.json` avec votre éditeur préféré :

    ```bash
    # Using nano
    nano ~/.claude-code-router/config.json

    # Or using VS Code
    code ~/.claude-code-router/config.json
    ```

    Collez la configuration suivante :

    ```json
    {
      "APIKEY": "",
      "LOG": true,
      "LOG_LEVEL": "info",
      "API_TIMEOUT_MS": 600000,
      "HOST": "127.0.0.1",
      "Providers": [
        {
          "name": "venice",
          "api_base_url": "https://api.venice.ai/api/v1/chat/completions",
          "api_key": "your-venice-api-key-here",
          "models": [
            "claude-opus-4-5",
            "claude-sonnet-4-5",
            "claude-opus-4-6",
            "claude-opus-4-6-fast",
            "claude-sonnet-4-6"
          ],
          "transformer": {
            "use": ["anthropic"]
          }
        }
      ],
      "Router": {
        "default": "venice,claude-opus-4-5",
        "think": "venice,claude-opus-4-5",
        "background": "venice,claude-opus-4-5",
        "longContext": "venice,claude-opus-4-5",
        "longContextThreshold": 100000
      }
    }
    ```

    <Note>
    Si vous modifiez `config.json` pendant que le router est en cours d'exécution, redémarrez-le avec `ccr restart` pour appliquer les changements.
    </Note>
  </Step>

  <Step title="Lancez">
    Démarrez le router, puis Claude Code :

    ```bash
    ccr start
    ccr code
    ```

    Ou utilisez la méthode d'activation :

    ```bash
    eval "$(ccr activate)" && claude
    ```
  </Step>
</Steps>

## Modèles pris en charge

| Modèle | ID Venice | Idéal pour |
|-------|-----------|----------|
| Claude Opus 4.5 | `claude-opus-4-5` | Raisonnement complexe, gros refactors |
| Claude Sonnet 4.5 | `claude-sonnet-4-5` | Itération rapide, codage quotidien |
| Claude Opus 4.6 | `claude-opus-4-6` | Raisonnement complexe, gros refactors |
| Claude Opus 4.6 Fast | `claude-opus-4-6-fast` | Raisonnement complexe avec latence réduite |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | Itération rapide, codage quotidien |

<Info>
Claude Code est optimisé pour les modèles Claude. Bien que d'autres modèles disponibles via Venice (GPT, DeepSeek, Grok, etc.) puissent fonctionner, nous ne pouvons pas garantir une expérience équivalente car Claude Code s'appuie sur des fonctionnalités spécifiques à Claude comme l'extended thinking. Pour d'autres modèles, envisagez d'utiliser l'[API standard de Venice](/fr/api-reference/endpoint/chat/completions).
</Info>

## Fonctionnalités du router

Le router fournit plusieurs fonctionnalités utiles au-delà du routage basique :

<AccordionGroup>
  <Accordion title="Changer de modèle à la volée">
    Utilisez la commande `/model` dans Claude Code pour changer de modèle sans redémarrer :
    
    ```
    /model venice,claude-sonnet-4-5
    ```
    
    Utile lorsque vous voulez Opus pour les tâches complexes et Sonnet pour les itérations rapides.
  </Accordion>

  <Accordion title="Configuration visuelle avec le mode UI">
    Vous préférez une interface graphique ? Lancez l'éditeur de config basé sur le web :
    
    ```bash
    ccr ui
    ```
    
    Cela ouvre une interface navigateur pour éditer votre `config.json` sans toucher au fichier directement.
  </Accordion>

  <Accordion title="Scénarios du router expliqués">
    La section de config `Router` contrôle quel modèle gère différents types de tâches :
    
    | Scénario | Quand il est utilisé |
    |----------|----------------|
    | `default` | Requêtes générales |
    | `think` | Tâches nécessitant beaucoup de raisonnement (Plan Mode) |
    | `background` | Opérations en arrière-plan |
    | `longContext` | Lorsque le contexte dépasse `longContextThreshold` tokens |
    
    Vous pouvez router différents scénarios vers différents modèles. Par exemple, utilisez Sonnet pour les tâches en arrière-plan afin d'économiser des coûts.
  </Accordion>

  <Accordion title="Déboguer avec les logs">
    Si quelque chose ne fonctionne pas, vérifiez les logs :
    
    ```bash
    # Server logs (HTTP, API calls)
    ~/.claude-code-router/logs/ccr-*.log
    
    # Application logs (routing decisions)
    ~/.claude-code-router/claude-code-router.log
    ```
    
    Définissez `"LOG_LEVEL": "debug"` dans votre config pour une sortie plus verbeuse.
  </Accordion>
</AccordionGroup>

## Comportement du caching

Le [prompt caching](/fr/guides/features/prompt-caching) de Venice fonctionne aux côtés des marqueurs de cache natifs de Claude Code. Venice détecte automatiquement lorsque Claude Code envoie des champs `cache_control` et ajuste sa stratégie de mise en cache en conséquence.

| Scénario | TTL du cache | Qui contrôle |
|----------|-----------|--------------|
| Par défaut (recommandé) | 5 minutes | Claude Code + Venice |
| Avec le transformer `cleancache` | 1 heure | Venice uniquement |

<AccordionGroup>
  <Accordion title="Quand NE PAS utiliser cleancache (la plupart des utilisateurs)">
    La configuration par défaut permet aux deux systèmes de coopérer :
    
    - Claude Code envoie ses marqueurs `cache_control` natifs
    - Venice ajoute du caching autour avec un TTL de 5 minutes
    - Les deux systèmes partagent la limite de cache à 4 blocs
    
    Cela fonctionne bien pour les sessions de codage actives où vous faites des requêtes fréquentes.
  </Accordion>

  <Accordion title="Quand utiliser cleancache">
    Ajoutez `cleancache` au transformer si vous :
    
    - Rencontrez des erreurs de limite de cache à 4 blocs
    - Constatez un comportement de cache étrange
    - Préférez le TTL d'une heure de Venice pour des sessions plus longues
    
    ```json
    "transformer": {
      "use": ["anthropic", "cleancache"]
    }
    ```
    
    Cela supprime les marqueurs de cache de Claude Code, donnant à Venice le contrôle complet avec un TTL plus long.
  </Accordion>
</AccordionGroup>

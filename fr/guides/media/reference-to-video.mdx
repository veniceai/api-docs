Reference to Video vous permet de verrouiller l'apparence des personnages, objets et scènes pour que vos vidéos générées par IA restent visuellement cohérentes. Au lieu d'espérer que le modèle interprète votre prompt correctement, vous fournissez des ancrages visuels — des images de référence qui indiquent au modèle exactement à quoi ressemble votre sujet.

Cette fonctionnalité est disponible sur les modèles **Kling O3** et **Grok Imagine R2V** dans le [Venice Video Studio](https://venice.ai/video). Chaque famille de modèles utilise une approche différente des images de référence — voir les sections spécifiques au modèle ci-dessous.

## Quand utiliser Reference to Video

Utilisez Reference to Video lorsque vous avez besoin de :

- **Cohérence des personnages** — la même personne ou le même personnage sur plusieurs plans
- **Précision produit** — un vrai produit qui doit être identique à l'original
- **Continuité de scène** — un environnement ou arrière-plan spécifique à travers les générations
- **Scènes multi-personnages** — plusieurs personnages distincts interagissant sans se mélanger

Pour le simple text-to-video ou image-to-video où la cohérence n'est pas critique, les modèles standards fonctionnent bien sans références.

## Modèles disponibles

| Modèle | Approche | Idéal pour |
|---|---|---|
| **Kling O3 Pro R2V** | Éléments + images de scène | Scènes multi-personnages complexes avec contrôle d'identité précis |
| **Kling O3 Standard R2V** | Éléments + images de scène | Itération plus rapide sur des scènes basées sur des éléments |
| **Grok Imagine R2V** | Images de référence plates | Génération rapide pilotée par référence avec jusqu'à 7 images |

**Kling O3** utilise une approche structurée avec des Éléments (ancrages d'identité de personnage avec images frontales + de référence) et des Images de scène. **Grok Imagine R2V** adopte une approche plus simple — vous téléversez des images de référence directement et y faites référence dans votre prompt avec `@Image1`, `@Image2`, etc.

## Kling O3 Reference to Video

### Concepts clés

Kling O3 Reference to Video utilise trois types d'entrée visuelle qui fonctionnent ensemble :

| Entrée | Requise | Objectif | Comment référencer dans le prompt |
|---|---|---|---|
| **Éléments** | Au moins une entrée visuelle* | Verrouiller l'identité d'un personnage ou d'un objet | `@Element1`, `@Element2`, etc. |
| **Images de référence de scène** | Au moins une entrée visuelle* | Définir l'environnement, le style et l'ambiance | `@Image1`, `@Image2`, etc. |
| **Frame de départ** | Au moins une entrée visuelle* | Contrôler la première frame de la vidéo | N/A (défini via upload) |
| **Frame de fin** | Non | Contrôler la dernière frame de la vidéo | N/A (défini via upload) |

*Au moins une de : frame de départ, éléments ou images de référence de scène est requise.

### Éléments

Un Élément est un personnage ou objet que vous voulez garder visuellement stable tout au long de la vidéo. Chaque élément consiste en :

- **Image frontale** (requise par élément) — une photo claire de face du sujet. C'est l'ancrage d'identité principal. Pensez-y comme à la "photo de passeport" de votre personnage ou produit.
- **Images de référence** (1-3, optionnelles) — angles supplémentaires du même sujet (vue de côté, angle à 45 degrés, dos). Cela aide le modèle à comprendre le sujet dans l'espace 3D. Si non fournies, l'image frontale est utilisée automatiquement comme référence.

Vous pouvez ajouter jusqu'à **7 éléments** par génération (limité par le total combiné). Référencez-les dans votre prompt en utilisant `@Element1`, `@Element2`, etc.

### Images de référence de scène

Les références de scène définissent la "scène" où l'action se déroule. Elles influencent :

- L'éclairage et la palette de couleurs
- Les détails de l'architecture et de l'environnement
- Le style visuel global et l'ambiance

Vous pouvez ajouter jusqu'à **4 images de scène**. Référencez-les comme `@Image1`, `@Image2`, etc. dans votre prompt.

### Limitations

Le nombre total d'images sur tous les types d'entrée est limité :

| Limite | Valeur |
|---|---|
| **Minimum requis** | Au moins 1 entrée visuelle (frame de départ, élément ou image de scène) |
| **Total combiné** (première frame + dernière frame + éléments + images de scène) | **7 maximum** |
| Éléments (sans frame départ/fin) | 7 maximum |
| Éléments (avec frame de départ ou de fin) | 3 maximum |
| Images de référence de scène | 4 maximum |
| Images de référence par élément | 1-3 |

**Scénarios d'exemple :**

- 7 éléments + 0 images de scène = 7 ✓ (pas de frames)
- 5 éléments + 2 images de scène = 7 ✓ (pas de frames)
- Première frame (1) + 3 éléments + 3 images de scène = 7 ✓
- Première frame (1) + dernière frame (1) + 3 éléments + 2 images de scène = 7 ✓
- Première frame (1) + 4 éléments = ✗ (max 3 éléments avec frame)
- Première frame (1) + dernière frame (1) + 4 éléments = ✗ (max 3 éléments avec frames)

<Note>
Chaque élément nécessite une **image frontale**. Si vous ne fournissez pas d'images de référence pour un élément, l'image frontale est utilisée automatiquement comme référence.
</Note>

### Mode multi-plans

Le mode multi-plans vous permet de découper une seule génération en plusieurs scènes, chacune avec son propre prompt et sa propre durée. Les éléments et références de scène se transmettent à travers tous les plans, maintenant la cohérence. La durée totale sur tous les plans ne peut pas dépasser **15 secondes**.

### Guide étape par étape (Video Studio)

#### 1. Ouvrir Video Studio et sélectionner le modèle

Allez sur [venice.ai/video](https://venice.ai/video). Dans le Model Browser à gauche, sélectionnez l'un des modèles **Kling O3 Reference to Video** :

- **Kling O3 Pro R2V** — qualité supérieure, temps de génération plus long (~6 min)
- **Kling O3 Standard R2V** — plus rapide, plus rentable pour l'itération

#### 2. Ajouter des entrées visuelles (au moins une requise)

Vous devez fournir **au moins une entrée visuelle** pour générer une vidéo : une frame de départ, un élément ou une image de référence de scène. Dans le panneau d'entrée, vous verrez la section **Éléments**. Cliquez sur **Add Element** pour créer un élément pour les personnages ou objets que vous voulez garder visuellement cohérents.

Pour chaque élément :

1. Cliquez sur la tuile **Frontal** pour téléverser une image claire de face de votre personnage ou objet
2. Cliquez optionnellement sur **Add** sous **Reference Images** pour téléverser des angles supplémentaires (1-3)

Répétez pour des personnages ou objets supplémentaires (jusqu'à 7 éléments au total, ou 3 si vous utilisez des frames de départ/fin).

<Warning>
Le total combiné de première frame, dernière frame, éléments et images de scène ne peut pas dépasser **7**. Voir [Limitations](#limitations) pour les détails.
</Warning>

<Tip>
**Meilleures images de référence :** utilisez des photos bien éclairées avec un arrière-plan propre. Fournissez des vues de face, de côté et à 45 degrés pour le verrouillage d'identité le plus fort. Assurez-vous que toutes les images de référence partagent le même style visuel (ne mélangez pas photoréaliste et anime).
</Tip>

#### 3. Ajouter des images de référence de scène (optionnel)

Sous la section Éléments, vous verrez **Scene Reference Images**. Téléversez des images qui définissent l'environnement que vous voulez — un lieu spécifique, une configuration d'éclairage ou un style artistique.

Celles-ci sont automatiquement marquées comme `@Image1`, `@Image2`, etc.

#### 4. Téléverser une frame de départ (optionnel)

Si vous voulez contrôler la première frame exacte de votre vidéo, passez au type d'entrée **Image** et téléversez une frame de départ. Vous pouvez aussi définir optionnellement une frame de fin.

#### 5. Écrivez votre prompt

Dans le champ de prompt, décrivez l'action que vous voulez tout en référençant vos éléments et images de scène en utilisant les tags `@` :

```
@Element1 walks through the streets of @Image1, looking up at the buildings.
The camera slowly tracks from behind, revealing the city skyline.
```

Pour les **scènes multi-personnages** :

```
@Element1 and @Element2 enter the cafe in @Image1 from opposite sides.
@Element1 waves and walks toward @Element2, who is sitting at a corner table.
```

#### 6. Configurer les paramètres

Ouvrez **Video Settings** pour ajuster :

| Paramètre | Options | Défaut |
|---|---|---|
| Durée | 3s – 15s | 5s |
| Ratio d'aspect | 16:9, 9:16, 1:1 | 16:9 |
| Generate Audio | On/Off | Off |

<Note>
La génération audio ajoute des effets sonores natifs, des dialogues et de l'audio d'ambiance synchronisés à la vidéo. Elle augmente le coût d'environ 25 %.
</Note>

#### 7. Générer

Cliquez sur **Generate Video**. Kling O3 prend généralement 4 à 6 minutes selon le palier du modèle et la durée. Vous pouvez mettre en file plusieurs générations et parcourir les résultats dans la Video Gallery.

### Storyboard multi-plans

Pour les séquences narratives, utilisez le mode multi-plans pour définir des scènes distinctes au sein d'une seule génération.

1. Dans la zone de prompt, cliquez sur **Add Shot** pour créer des plans supplémentaires
2. Écrivez un prompt séparé pour chaque plan
3. Définissez la durée pour chaque plan (3-15s chacun, total ≤ 15s)

Les éléments et références de scène persistent automatiquement sur tous les plans :

```
Shot 1 (5s): @Element1 stands at the edge of @Image1, looking out at the horizon.
Slow camera push forward.

Shot 2 (5s): Close-up of @Element1's face as they turn toward the camera.
Soft natural lighting, shallow depth of field.

Shot 3 (5s): @Element1 walks away from camera into the distance.
Wide cinematic shot, golden hour lighting.
```

<Warning>
La durée totale multi-plans ne peut pas dépasser 15 secondes. Par exemple, trois plans de 5 secondes = 15s maximum.
</Warning>

### Conseils de prompting

#### Structurez votre prompt

Suivez ce schéma pour des résultats fiables :

```
[subject with @Element tag] + [action] + [environment with @Image tag] + [camera movement] + [lighting/style]
```

**Exemple :**

```
@Element1 hops happily across the candy ground of @Image1, stops to look at a
giant lollipop, tilts its head curiously. Cinematic tracking shot, soft warm lighting.
```

#### Gardez les prompts entre 50 et 150 mots

Les prompts plus courts manquent de détail. Les prompts plus longs introduisent des contradictions. Visez le juste milieu.

#### Utilisez un langage de caméra simple

Le modèle répond mieux à des directions de caméra simples :

| Utilisez | Évitez |
|---|---|
| `slow camera push forward` | `dolly zoom with rack focus transition` |
| `tracking shot from behind` | `complex handheld parallax movement` |
| `close-up` | `extreme macro with tilt-shift bokeh` |
| `wide cinematic shot` | `anamorphic ultra-wide establishing crane shot` |

#### Utilisez un vocabulaire cohérent

Si vous décrivez un personnage portant "a red jacket" dans un prompt, ne basculez pas vers "crimson coat" dans le suivant. Le modèle traite les mots différents comme des intentions différentes.

#### Placez les instructions de caméra tôt

Mettez la direction de caméra près du début du prompt pour des résultats plus fiables :

```
Cinematic tracking shot of @Element1 walking through @Image1, leaves
blowing in the wind, golden afternoon light.
```

### Utilisation de l'API Kling O3

Kling O3 Reference to Video est aussi disponible via l'API Venice. Voir l'[API Video Queue](https://docs.venice.ai/api-reference/endpoint/video/queue) pour les détails complets.

#### Python

```python
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "kling-o3-pro-reference-to-video",
        "prompt": "@Element1 walks through @Image1, camera tracking from behind",
        "duration": "8",
        "aspect_ratio": "16:9",
        "audio": True,
        "elements": [
            {
                "frontal_image_url": "https://example.com/character-front.jpg",
                "reference_image_urls": [
                    "https://example.com/character-side.jpg",
                    "https://example.com/character-angle.jpg"
                ]
            }
        ],
        "image_urls": [
            "https://example.com/scene-background.jpg"
        ]
    }
)

queue_id = response.json()["id"]
```

Chaque élément dans le tableau `elements` accepte :

### Dépannage Kling O3

| Problème | Cause probable | Correction |
|---|---|---|
| Le bouton Generate est désactivé | Aucune entrée visuelle fournie | Ajoutez au moins une entrée visuelle : frame de départ, élément ou image de référence de scène |
| Erreur "Number of images exceeds the limit" | Trop d'entrées combinées | Le total de première frame + dernière frame + éléments + images de scène doit être ≤ 7 |
| Le visage du personnage change entre les plans | Image frontale différente ou manquante | Utilisez la même image frontale de manière cohérente, gardez la description identique |
| Le mouvement de caméra semble aléatoire | Instructions de caméra multiples ou conflictuelles | Utilisez une seule instruction de caméra, placez-la tôt dans le prompt |
| Le style change entre les générations | Références de scène incohérentes ou styles mixtes | Réutilisez les mêmes images de scène, gardez les mots-clés de style cohérents |
| Les éléments se fondent dans les scènes multi-personnages | Instructions spatiales vagues | Soyez explicite sur la position de chaque élément : "foreground left", "entering from right" |
| L'arrière-plan a l'air déformé | Image de référence de scène encombrée ou complexe | Utilisez des images de référence de scène propres et de haute qualité |
| Le mouvement semble non naturel | Trop d'actions dans un prompt | Simplifiez l'action, utilisez une durée plus courte, une action par plan |

<Tip>
Testez avec un clip de 3 à 5 secondes avant de vous engager dans des durées plus longues. Les clips plus courts maintiennent une meilleure cohérence et vous permettent d'itérer plus rapidement.
</Tip>

## Grok Imagine Reference to Video

Grok Imagine R2V adopte une approche plus simple que Kling O3. Au lieu d'Éléments structurés avec séparation image frontale/référence, vous téléversez des **images de référence plates** et les référencez directement dans votre prompt en utilisant `@Image1`, `@Image2`, etc. Le modèle incorpore ces sujets dans la vidéo générée.

### Comment ça marche

1. Téléversez **1-7 images de référence** — photos de personnages, objets ou scènes que vous voulez dans la vidéo
2. Écrivez un prompt qui décrit la vidéo, en utilisant `@Image1`, `@Image2`, etc. pour référencer des images spécifiques
3. Le modèle génère une vidéo incorporant ces références

Si vous n'incluez pas de tags `@Image` dans votre prompt, toutes les images téléversées sont référencées automatiquement.

### Paramètres

| Paramètre | Options | Défaut |
|---|---|---|
| Ratio d'aspect | 16:9, 4:3, 3:2, 1:1, 2:3, 3:4, 9:16 | 16:9 |
| Résolution | 480p, 720p | 480p |
| Durée | 5s, 8s, 10s | 8s |

<Note>
Grok Imagine R2V ne prend pas en charge la génération audio, le mode multi-plans ou les Éléments. Pour ces fonctionnalités, utilisez Kling O3 R2V.
</Note>

### Guide étape par étape (Video Studio)

#### 1. Sélectionner le modèle

Allez sur [venice.ai/video](https://venice.ai/video). Dans le Model Browser, sélectionnez **Grok Imagine R2V**.

#### 2. Téléverser les images de référence

Cliquez sur **References** dans la barre d'outils d'entrée (ou utilisez le menu +) pour ouvrir le panneau des images de référence. Téléversez 1-7 images des personnages, objets ou scènes que vous voulez dans la vidéo.

Chaque image est automatiquement marquée comme `@Image1`, `@Image2`, etc. dans l'ordre de téléversement (de gauche à droite).

#### 3. Écrivez votre prompt

Décrivez la vidéo que vous voulez. Utilisez les tags `@Image` pour référencer des images spécifiques :

```
@Image1 and @Image2 walking together through a sunlit park,
camera slowly tracking alongside them, warm afternoon light.
```

Tapez `@` dans le champ de prompt pour voir un menu d'autocomplétion des références d'images disponibles.

<Tip>
Si vous omettez entièrement les tags `@Image`, le backend ajoute automatiquement des références à toutes les images téléversées. C'est utile lorsque vous voulez que toutes les images soient utilisées sans spécifier laquelle est laquelle.
</Tip>

#### 4. Configurer les paramètres et générer

Ouvrez **Video Settings** pour ajuster le ratio d'aspect, la résolution et la durée. Cliquez sur **Generate Video**.

### Tarification Grok Imagine R2V

Grok Imagine R2V utilise une tarification basée sur la durée et la résolution :

| Résolution | Par seconde |
|---|---|
| 480p | ~0,063 $ |
| 720p | ~0,088 $ |

**Exemple :** une vidéo de 8 secondes en 480p = 8 × 0,063 $ = **~0,50 $**

<Note>
Grok Imagine facture des frais de modération de contenu pour les vidéos générées, même si la vidéo est rejetée. Cela est reflété dans le coût de crédit affiché avant la génération.
</Note>

#### Paramètres de l'API

<Note>

### Dépannage Grok Imagine R2V

| Problème | Cause probable | Correction |
|---|---|---|
| Le bouton Generate est désactivé | Aucune image de référence téléversée | Téléversez au moins 1 image de référence |
| Erreur "At least one reference image is required" | `referenceImageUrls` est vide ou manquant | Fournissez au moins une URL d'image dans `referenceImageUrls` |
| Mauvaise image associée au tag `@Image` | L'ordre des images ne correspond pas aux tags | `@Image1` correspond à la première image dans votre ordre de téléversement (de gauche à droite). Réordonnez les téléversements si nécessaire. |
| Le sujet n'apparaît pas dans la vidéo | Trop de références sans tags explicites | Utilisez les tags `@Image` dans votre prompt pour être explicite sur les images à utiliser |
| Sortie de basse qualité | Utilisation de la résolution 480p | Essayez 720p pour une qualité supérieure (coûte plus) |
| Vidéo trop courte | La durée par défaut est 8s | Définissez la durée sur `"10"` pour des vidéos plus longues |

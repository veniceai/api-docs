## Aperçu

Venice fournit une collection Postman complète qui permet aux développeurs d'explorer et de tester toutes les capacités de notre API. Cette collection comprend des requêtes préconfigurées, des exemples et des variables d'environnement pour vous aider à démarrer rapidement avec les services IA de Venice.

## Accéder à la collection

Notre collection Postman officielle est disponible dans le Venice AI Workspace :

* [Venice AI Postman Workspace](https://www.postman.com/veniceai/workspace/venice-ai-workspace)
* [Exemples Venice AI Postman](https://postman.venice.ai/)

## Fonctionnalités de la collection

- **Requêtes prêtes à l'emploi** : appels d'API préconfigurés pour tous les endpoints Venice
- **Templates d'environnement** : variables d'environnement correctement structurées
- **Exemples de requêtes** : exemples d'utilisation réels pour chaque endpoint
- **Échantillons de réponses** : exemples de réponses pour vous aider à comprendre la sortie de l'API
- **Documentation** : documentation intégrée pour chaque requête

## Démarrer

<Steps>
   <Step title="Forker la collection">
      - Rendez-vous dans le Venice AI Workspace
      - Cliquez sur "Fork" pour créer votre propre copie de la collection
      - Choisissez la destination de votre workspace
   </Step>

   <Step title="Configurer votre environnement">
      - Créez un nouvel environnement dans Postman
      - Ajoutez votre clé API Venice
      - Configurez l'URL de base : `https://api.venice.ai/api/v1`
   </Step>

   <Step title="Effectuer votre première requête">
      - Sélectionnez n'importe quelle requête de la collection
      - Assurez-vous que votre environnement est sélectionné
      - Cliquez sur "Send" pour tester l'API
   </Step>
</Steps>

## Endpoints disponibles

La collection inclut des exemples pour tous les endpoints de l'API Venice :

- Génération de texte
- Génération d'images
- Informations sur les modèles
- Upscaling d'image
- Configuration du system prompt

## Bonnes pratiques

- Gardez votre clé API sécurisée et ne la partagez jamais
- Utilisez des variables d'environnement pour les informations sensibles
- Testez les réponses dans la console Postman avant l'implémentation
- Consultez les exemples de réponses pour comprendre les structures de données attendues

<Note>*Remarque : la collection Postman est régulièrement mise à jour pour refléter les derniers changements et fonctionnalités de l'API.*</Note>

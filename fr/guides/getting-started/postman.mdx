## Vue d'ensemble

Venice fournit une collection Postman complète qui permet aux développeurs d'explorer et de tester l'ensemble des capacités de notre API. Cette collection inclut des requêtes préconfigurées, des exemples et des variables d'environnement pour vous aider à démarrer rapidement avec les services IA de Venice.

## Accéder à la collection

Notre collection Postman officielle est disponible dans le Venice AI Workspace :

* [Workspace Postman Venice AI](https://www.postman.com/veniceai/workspace/venice-ai-workspace)
* [Exemples Postman Venice AI](https://postman.venice.ai/)

## Fonctionnalités de la collection

- **Requêtes prêtes à l'emploi** : Appels API préconfigurés pour tous les endpoints Venice
- **Modèles d'environnement** : Variables d'environnement correctement structurées
- **Exemples de requêtes** : Exemples d'utilisation réels pour chaque endpoint
- **Exemples de réponses** : Réponses d'exemple pour vous aider à comprendre la sortie de l'API
- **Documentation** : Documentation en ligne pour chaque requête

## Démarrage

<Steps>
   <Step title="Forker la collection">
      - Accédez au Venice AI Workspace
      - Cliquez sur « Fork » pour créer votre propre copie de la collection
      - Choisissez votre workspace de destination
   </Step>

   <Step title="Configurer votre environnement">
      - Créez un nouvel environnement dans Postman
      - Ajoutez votre clé API Venice
      - Configurez l'URL de base : `https://api.venice.ai/api/v1`
   </Step>

   <Step title="Effectuer votre première requête">
      - Sélectionnez n'importe quelle requête de la collection
      - Assurez-vous que votre environnement est sélectionné
      - Cliquez sur « Send » pour tester l'API
   </Step>
</Steps>

## Endpoints disponibles

La collection inclut des exemples pour tous les endpoints de l'API Venice :

- Génération de texte
- Génération d'image
- Informations sur les modèles
- Agrandissement d'image
- Configuration du prompt système

## Bonnes pratiques

- Gardez votre clé API en sécurité et ne la partagez jamais
- Utilisez des variables d'environnement pour les informations sensibles
- Testez les réponses dans la console Postman avant l'implémentation
- Examinez les réponses d'exemple pour les structures de données attendues

<Note>*Note : La collection Postman est régulièrement mise à jour pour refléter les derniers changements et fonctionnalités de l'API.*</Note>

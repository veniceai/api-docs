Nous publions parfois des modèles en bêta afin de recueillir des retours et de confirmer leurs performances avant un déploiement complet en production. Les modèles bêta sont disponibles pour tous les utilisateurs mais **ne sont pas recommandés pour un usage en production**.

Le statut bêta ne garantit pas une promotion en production. Un modèle bêta peut être retiré s'il est trop coûteux à exécuter, s'il fonctionne mal à grande échelle ou s'il soulève des préoccupations de sécurité. Les modèles bêta peuvent évoluer sans préavis et peuvent disposer d'une documentation ou d'un support limités. Les modèles qui s'avèrent stables, largement utiles et alignés avec nos standards sont promus en disponibilité générale.

## Considérations importantes

Lorsque vous utilisez des modèles bêta, gardez à l'esprit que :

- Ils peuvent être modifiés ou retirés à tout moment sans la période standard de préavis de dépréciation
- Ils ne conviennent pas aux applications de production ou aux workflows critiques
- Leurs performances, leur disponibilité ou leur comportement peuvent être incohérents
- Le support de migration peut être limité ou absent en cas de retrait
- Ils sont à utiliser de préférence pour les tests, l'évaluation et les projets expérimentaux

Pour les applications de production, nous recommandons d'utiliser les modèles stables de notre [gamme principale de modèles](/fr/models/overview).

### Vérifier le statut bêta via l'API

Vous pouvez vérifier si un modèle est en bêta en appelant l'endpoint [List Models](/fr/api-reference/endpoint/models/list). Les modèles bêta incluent un champ `betaModel` défini sur `true` dans leur `model_spec` :

```json
{
  "id": "some-beta-model",
  "model_spec": {
    "name": "Some Beta Model",
    "betaModel": true,
    "privacy": "private"
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

Vous pouvez vérifier `if (model.model_spec.betaModel)` pour identifier les modèles bêta et avertir les utilisateurs ou les traiter différemment dans votre application.

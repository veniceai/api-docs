## Tarification par palier de qualité

Pour les modèles d'image qui acceptent le paramètre optionnel `quality` (actuellement `gpt-image-2` et `gpt-image-2-edit`), la réponse expose une matrice de prix par qualité sous `model_spec.pricing.quality`. Chaque clé de premier niveau est un palier de résolution (`1K`, `2K`, `4K`) et chaque clé imbriquée est un niveau de qualité (`low`, `medium`, `high`) qui porte ses propres prix `usd` et `diem` :

```json
"pricing": {
  "resolutions": {
    "1K": { "usd": 0.27, "diem": 0.27 },
    "2K": { "usd": 0.51, "diem": 0.51 },
    "4K": { "usd": 0.84, "diem": 0.84 }
  },
  "quality": {
    "1K": {
      "low":    { "usd": 0.02, "diem": 0.02 },
      "medium": { "usd": 0.07, "diem": 0.07 },
      "high":   { "usd": 0.26, "diem": 0.26 }
    },
    "2K": {
      "low":    { "usd": 0.03, "diem": 0.03 },
      "medium": { "usd": 0.13, "diem": 0.13 },
      "high":   { "usd": 0.50, "diem": 0.50 }
    },
    "4K": {
      "low":    { "usd": 0.05, "diem": 0.05 },
      "medium": { "usd": 0.21, "diem": 0.21 },
      "high":   { "usd": 0.83, "diem": 0.83 }
    }
  }
}
```

`pricing.resolutions` est le barème historique par image conservé pour la rétrocompatibilité. `pricing.quality` est la matrice par (résolution, qualité) qui s'applique chaque fois que le paramètre `quality` est pris en charge. Les deux champs sont conservés dans la réponse afin que les clients puissent détecter la prise en charge de la qualité et afficher la matrice dans leurs propres interfaces.

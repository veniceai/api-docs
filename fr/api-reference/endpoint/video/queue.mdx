Appelez `/video/quote` pour obtenir une estimation du prix, puis interrogez `/video/retrieve` avec le `queue_id` renvoyé jusqu'à la fin.

Les modèles privés renvoient aussi une `download_url` pour la vidéo terminée. Il s'agit d'une URL de livraison à courte durée de vie (quelques nouveaux essais sont acceptables si un téléchargement échoue) ; voir le [guide de génération vidéo](/fr/guides/media/video-generation#private-download-links) pour plus de détails et le `DELETE` optionnel pour la confidentialité.

### Seedance 2.0

Pour les modèles `seedance-2-0-*` (text-to-video, image-to-video, reference-to-video, plus les variantes `-fast-*`), consultez le [guide Seedance 2.0](/fr/guides/media/seedance-2-0) pour le modèle à quatre workflows (Reference / Edit / Extend / Stitch), les patterns de prompts canoniques, les limites d'entrées multimodales et les détails de tarification.

### Consentement Seedance

Lorsqu'une requête Seedance image- ou reference-to-video contient un visage humain, l'API renvoie un `409 needs_consent` avec le texte de la politique, et vous soumettez à nouveau la même requête avec une attestation `consents.seedance`. Consultez le [guide de consentement aux médias contenant un visage Seedance](/fr/guides/media/seedance-face-consent) pour le flux complet, l'objet de consentement, la déduplication et la révocation.

### Upscaling vidéo

Pour le modèle `topaz-video-upscale`, utilisez `upscale_factor` (1, 2 ou 4) au lieu de `resolution`, et fournissez une `video_url`. La durée et le FPS sont détectés automatiquement depuis le fichier vidéo. Consultez le [guide d'upscaling vidéo](/fr/guides/media/video-upscaling) pour tous les détails et exemples.

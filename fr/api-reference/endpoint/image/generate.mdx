## Options de dimensionnement

Les modèles d'image utilisent des paramètres de dimensionnement spécifiques au modèle :

- Les modèles basés sur les pixels acceptent `width` et `height`, par exemple `venice-sd35` et `qwen-image`.
- Les modèles basés sur le ratio d'aspect acceptent `aspect_ratio`, par exemple `qwen-image-2`.
- Les modèles à paliers de résolution acceptent à la fois `aspect_ratio` et `resolution`.

Par exemple, `gpt-image-2`, `nano-banana-2` et `nano-banana-pro` prennent en charge les valeurs de `resolution` `1K`, `2K` et `4K` :

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

Le même schéma de dimensionnement par paliers de résolution s'applique également aux modèles Nano Banana :

```json
{
  "model": "nano-banana-pro",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Pour les modèles qui acceptent le ratio d'aspect mais n'exposent pas de paliers de résolution, envoyez `aspect_ratio` sans `resolution` :

```json
{
  "model": "qwen-image-2",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9"
}
```

Pour les modèles qui utilisent des dimensions en pixels directes, envoyez plutôt `width` et `height` :

```json
{
  "model": "venice-sd35",
  "prompt": "a serene canal in venice at sunset",
  "width": 1024,
  "height": 1024
}
```

Consultez [Modèles d'image](/fr/models/image) ou l'[endpoint Models](/fr/api-reference/endpoint/models/list) pour connaître les options de dimensionnement prises en charge par chaque modèle avant de mélanger les champs de dimensionnement entre modèles.

## Paliers de qualité

Certains modèles acceptent un paramètre optionnel `quality` (`low`, `medium`, `high`) qui arbitre entre fidélité visuelle et coût. Actuellement pris en charge par `gpt-image-2` ; les autres modèles ignorent ce paramètre.

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K",
  "quality": "medium"
}
```

Lorsque vous omettez `quality`, le modèle utilise son palier par défaut (`high` pour `gpt-image-2`). La tarification dépend de la combinaison de `resolution` et `quality` — les prix par palier pour `gpt-image-2` (1K/2K/4K × low/medium/high) sont exposés sous `model_spec.pricing.quality` dans l'[endpoint Models](/fr/api-reference/endpoint/models/list) et listés dans l'[aperçu de la tarification](/fr/overview/pricing).

<Info>
La tarification tenant compte de la qualité est la valeur par défaut pour tous les appelants API et SDK — aucun en-tête d'opt-in requis. Les requêtes qui omettent `quality` sont facturées au palier par défaut du modèle (`high` pour `gpt-image-2`).
</Info>

VVV 和 DIEM 是 Venice 为 API 提供的链上资金层。质押 DIEM 即可获得固定的每日推理额度 —— **1 DIEM = 每天 $1** 的 Venice API 额度 —— 无需为每次请求购买预付的 USD 额度。

| 代币 | 作用 |
| --- | --- |
| **VVV** | Venice 在 Base 上的基础代币。质押它可以获得收益并铸造 DIEM。 |
| **sVVV** | 质押 VVV 的凭证。按当前[铸造费率](https://diem-calculator.venice.ai/)锁定 sVVV 以铸造 DIEM。 |
| **DIEM** | 代币化的算力单位。质押 DIEM 可解锁永久每日 API 额度,每枚 DIEM 相当于 $1。 |

DIEM 是 Base 上的一种 ERC-20 代币。你可以转账、交易或质押它。与按需付费的 USD 额度不同,质押的 DIEM 是你自己拥有的容量:每日额度会在每个 epoch 刷新,且不会随着网络使用量增长而被稀释。

<Info>
  有关按该额度计费的模型价格,请参阅 [API 价格](/overview/pricing)。有关无需 API key 的按请求钱包支付,请参阅 [x402](/guides/integrations/x402-venice-api)。
</Info>

## 资金如何运作

API key 按以下顺序从关联的 Venice 账户中扣费:**DIEM**,然后是[打包额度](/overview/pricing#bundled-credits),最后是 USD。

当 DIEM 是当前的计费币种时:

* 你的每日额度等于你已质押的 DIEM 数量(1 枚已质押的 DIEM → 该 epoch 内 $1 的 API 额度)。
* 某个 epoch 内未使用的 DIEM 不会滚存。额度在 **00:00 UTC** 刷新。
* 账户至少需要 **0.1** 枚已质押的 DIEM,DIEM 余额才可用于消费。
* 请求费用使用与按需付费相同的 USD 价目表;DIEM 是结算币种,按当日额度以 1:1 的美元比例结算。

在仪表盘 [venice.ai/settings/api](https://venice.ai/settings/api) 中查看当前及下一 epoch 的 DIEM 余额,或通过 [Billing Balance](/api-reference/endpoint/billing/balance) 端点查询。

## 获取并质押 DIEM

使用位于 [venice.ai/token](https://venice.ai/token) 的代币仪表盘。连接一个 Base 钱包 —— 不要将代币直接发送到合约地址。

<Steps>
  <Step title="获取 VVV 或 DIEM">
    在 Base DEX 上购买 VVV 或 DIEM,例如 [Aerodrome](https://aerodrome.finance) 或 [Uniswap](https://app.uniswap.org),或接收来自其他钱包转账的 DIEM。

    | 资产 | 合约地址(Base) |
    | --- | --- |
    | VVV | [`0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf`](https://basescan.org/token/0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf) |
    | DIEM | [`0xF4d97F2da56e8c3098f3a8D538DB630A2606a024`](https://basescan.org/token/0xF4d97F2da56e8c3098f3a8D538DB630A2606a024) |
  </Step>

  <Step title="质押 VVV(若要铸造 DIEM)">
    在 [venice.ai/token](https://venice.ai/token) 上质押 VVV 以获得 sVVV。质押的 VVV 可获得增发收益。质押合约地址为 [`0x321b7ff75154472B18EDb199033fF4D116F340Ff`](https://basescan.org/address/0x321b7ff75154472B18EDb199033fF4D116F340Ff#code)。
  </Step>

  <Step title="从 sVVV 铸造 DIEM(可选)">
    按当前铸造费率锁定 sVVV 以铸造 DIEM。铸造费率表示每铸造 1 枚 DIEM 需要多少 sVVV,并会随着 DIEM 供应量增长而上升 —— 可在 [DIEM 计算器](https://diem-calculator.venice.ai/)中预览。

    铸造的 DIEM 会作为 ERC-20 出现在你的钱包中。被锁定的 sVVV 会一直保持锁定状态,直到你销毁等量的 DIEM 才能解锁。
  </Step>

  <Step title="质押 DIEM 以获取 API 额度">
    在同一个代币仪表盘上质押 DIEM。每枚质押的 DIEM 在质押期间可为 Venice API 提供**每天 $1** 的额度。

    解除 DIEM 质押有 **1 天**的冷却期。解锁支撑已铸 DIEM 的 sVVV 需要销毁对应的 DIEM;之后解除释放出的 sVVV 质押还有 **7 天**冷却期。
  </Step>

  <Step title="创建或使用 API key">
    使用同一个钱包登录 Venice(Sign-In-With-Ethereum),让质押的 DIEM 关联到你的账户,然后在 [API 设置](https://venice.ai/settings/api)中创建 key,或参阅 [API key 指南](/guides/getting-started/generating-api-key)。

    自主代理可以从持有已质押 VVV 的钱包中铸造 key —— 请参阅[自主代理 API Key 创建](/guides/getting-started/generating-api-key-agent)。
  </Step>
</Steps>

<Warning>
  切勿将 VVV 或 DIEM 直接发送到合约地址。以这种方式发送的代币无法找回。请始终通过 [venice.ai/token](https://venice.ai/token) 或已知的质押交易流程进行质押。
</Warning>

## 在 API 中使用 DIEM

一旦 DIEM 被质押并关联到你的账户,即可使用普通的 Bearer key 调用 API。无需额外的 header 或 `venice_parameters` —— 当 `consumptionCurrency` 为 `DIEM` 时,Venice 会自动从 DIEM 中扣费。

```bash
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### 查询 DIEM 余额

```bash
curl https://api.venice.ai/api/v1/billing/balance \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

响应示例结构:

```json
{
  "canConsume": true,
  "consumptionCurrency": "DIEM",
  "balances": {
    "diem": 90.5,
    "usd": 25
  },
  "diemEpochAllocation": 100
}
```

| 字段 | 含义 |
| --- | --- |
| `balances.diem` | 当前 epoch 剩余的 DIEM 额度(若你未质押 DIEM 则为 `null`) |
| `diemEpochAllocation` | 本 epoch 分配的 DIEM 总量(质押数量)。与 `balances.diem` 比较可了解已用量 |
| `consumptionCurrency` | 下一次将扣费的币种(`DIEM`、`USD`、`BUNDLED_CREDITS` 或旧版 `VCU`) |
| `canConsume` | 账户当前是否可以为请求付费 |

列出模型或用量分析可查看每次请求以 USD 和 DIEM 计价的费用 —— 二者以当日额度中 $1 = 1 DIEM 的比例一致。请参阅 [Billing Usage](/api-reference/endpoint/billing/usage) 和 [Usage Analytics](/api-reference/endpoint/billing/usage-analytics)。

### 限制单个 key 的支出

创建 key 时,你可以设置以 `diem` 或 `usd` 计价的每个 epoch 消耗上限,以防某个集成耗尽全部每日额度。在仪表盘上,此设置显示为 **Epoch Consumption Limits**;通过 Web3 铸造的 key 可在 [`POST /api_keys/generate_web3_key`](/api-reference/endpoint/api_keys/generate_web3_key/post) 中接受 `consumptionLimit` 参数。

## DIEM、USD 额度与 x402 对比

| 方式 | 适用场景 | 如何刷新 |
| --- | --- | --- |
| **质押 DIEM** | 可预测的每日容量、代理、固定的 AI 成本 | 只要 DIEM 保持质押,每个 epoch(00:00 UTC)刷新完整额度 |
| **USD / 加密货币额度** | 超出 DIEM 额度的突发用量、简单的预付支出 | 余额随使用量减少;额度永不过期 |
| **x402(USDC)** | 无需 API key 的无头按请求付费 | 钱包通过 `POST /x402/top-up` 充值 USDC;若存在关联账户上的 DIEM,仍会优先扣除 DIEM |

如果某个请求会超过剩余的 DIEM,且没有 USD 或打包额度可以覆盖,API 会返回 `402` 并携带 `INSUFFICIENT_BALANCE`。请质押更多 DIEM、等待下一个 epoch,或在[设置 → API](https://venice.ai/settings/api) 中添加 USD 额度。

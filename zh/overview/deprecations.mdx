Venice API 致力于让开发者以未受限的方式私密访问生产级模型，免受隐藏的过滤器或黑箱式决策影响。

随着模型的进步，我们偶尔会让较老的模型退役，转而采用更聪明、更快或更强大的替代品。我们将这些过渡设计得可预期、低摩擦。

## 模型弃用

我们知道弃用可能带来干扰。因此，我们仅在必要时进行弃用，并通过特性（traits）与 Venice 品牌模型等设计来最小化干扰。

我们可能在以下情况下弃用一个模型：

- 有更新的模型在相同用例上提供明显改进
- 该模型不再达到我们对性能或可靠性的标准
- 它的使用持续偏低，继续支持它会让其他人的体验变得碎片化

## 弃用流程

当某个模型符合弃用标准时，我们会提前 30–60 天发布公告。弃用通知会发布在[更新日志](https://featurebase.venice.ai/changelog)和我们的 [Discord 服务器](https://discord.gg/askvenice)。在公告期内调用已弃用的模型时，API 响应中会包含弃用警告。

在公告期内，模型仍然可用，但我们在某些情况下可能会减少其基础设施容量。我们始终会提供推荐的替代品，并在需要时给出迁移指引，以协助过渡。

到达退役日期后，对该模型的请求会自动路由到处理能力相近、价格相同或更低的模型。如果出于技术或安全原因无法路由，API 将返回 410 Gone。如果某个弃用模型是通过特性（例如 `default_code`、`default_vision` 或 `fastest`）选中的，该特性会被重新指向兼容的替代模型。

我们从不悄无声息地移除模型，也不会在没有版本控制的情况下改变行为。你始终能清楚知道当前在运行什么，以及如何为下一步做准备。

<Note>
仅性能改进的升级：我们可能会推出在不改变模型行为的前提下提升性能、降低延迟或提高成本效率的改进。这些更新向后兼容，无需客户采取任何行动。
</Note>

参见下文的[模型弃用追踪表](#model-deprecation-tracker)。早期公告请查阅[更新日志](https://featurebase.venice.ai/changelog)和我们的 [Discord 服务器](https://discord.gg/askvenice)。

## Venice API 如何选择模型

我们根据性能、可靠性以及实际开发者需求精心挑选要上线的模型。要被纳入，模型必须展现出强劲的性能，在 OpenAI 兼容端点下行为一致，并相对我们已支持的至少一个模型有明显改进。


我们正在评估的模型可能首先以 [beta](/zh/overview/beta-models) 形式发布，以收集反馈并验证规模化下的性能。

我们不会暴露冗余、未经验证或尚未做好稳定生产准备的模型。我们的目标是保持 Venice API 简洁、强大，并针对开发者实际构建的内容进行优化。

了解更多请见[模型弃用](/zh/overview/deprecations#model-deprecations)以及<a href="/zh/overview/models" target="_blank" rel="noopener noreferrer">当前模型列表</a>。

## 版本控制与别名

所有 Venice 模型都由唯一、永久的 ID 标识。例如：

```venice-uncensored```
```zai-org-glm-4.7```
```zai-org-glm-5```
```qwen3-vl-235b-a22b```

模型 ID 是稳定的。如果存在破坏性变更，我们会发布一个新的模型 ID（例如增加 v2 等版本号）。如果没有破坏性变更，我们可能会更新现有模型，并就重要变更进行沟通。

为提供灵活性，Venice 还维护通过特性（traits）实现的符号别名，指向给定任务下推荐的默认模型：

特性为模型选择提供了稳定的抽象层，同时让 Venice 有空间改进底层实现。偏好自动获得最新推荐模型的开发者可以依赖基于特性的别名。

对于需要严格一致性和可预测行为的应用，我们建议引用固定的模型 ID。

### 通过 API 检查弃用状态

你可以通过调用 [List Models](/zh/api-reference/endpoint/models/list) 端点来检查模型是否已计划退役。有退役日期的模型在其 `model_spec` 中会包含 `deprecation` 对象：

```json
{
  "id": "some-model-id",
  "model_spec": {
    "name": "Some Model",
    "privacy": "private",
    "deprecation": {
      "date": "2025-03-01T00:00:00.000Z"
    }
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

`deprecation` 对象仅在模型计划退役时出现。你可以通过判断 `if (model.model_spec.deprecation)` 来识别正在退役的模型，并使用 ISO 8601 日期向用户发出警告或规划迁移。

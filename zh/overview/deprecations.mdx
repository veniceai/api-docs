Venice API 的存在是为了让开发者能够无限制地、私密地访问生产级模型，免受隐藏过滤器或黑盒决策的影响。

随着模型的进步，我们会偶尔淘汰旧模型，转而使用更智能、更快或更强大的替代品。我们将这些过渡设计得可预测且低摩擦。

## 模型弃用

我们知道弃用可能会带来干扰。这就是为什么我们只在必要时才弃用，并设计 traits 和 Venice 品牌模型等功能以最大程度减少干扰。

我们可能会在以下情况下弃用模型：

- 新模型为同一用例提供明显改进
- 该模型不再符合我们对性能或可靠性的标准
- 该模型的使用率持续较低，继续支持它会使其他所有用户的体验碎片化

## 弃用流程

当某个模型符合弃用条件时，我们会在移除前提供 7–15 天的通知。您可以通过以下方式跟踪即将到来的弃用：

- 下方的[模型弃用追踪器](#model-deprecation-tracker)
- [List Models](/api-reference/endpoint/models/list) API 端点（已安排退役的模型包含 `deprecation` 对象）
- 我们的[更新日志](https://featurebase.venice.ai/changelog)
- 邮件通知，如果您在过去 15 天内调用过该模型，将自动发送

在通知期内，对已弃用模型的调用将在 API 响应中包含弃用警告。

在通知期间，模型仍然可用，但在某些情况下我们可能会减少基础设施容量。我们始终提供推荐的替代品，并在需要时提供迁移指导以协助过渡。

在停用日期之后，对该模型的请求将自动路由到处理能力相似、价格相同或更低的模型。如果由于技术或安全原因无法路由，API 将返回 410 Gone 响应。如果某个已弃用的模型是通过 trait（例如 `default_code`、`default_vision` 或 `fastest`）选择的，则该 trait 将重新分配给兼容的替代品。

我们绝不会在不通知的情况下移除模型或在没有版本管理的情况下改变行为。您将始终知道正在运行什么，以及如何为下一步做好准备。

<Note>
仅性能升级：我们可能会推出在保持模型行为的同时改善性能、延迟或成本效率的更新。这些更新向后兼容，无需客户采取任何行动。
</Note>

请参阅下方的[模型弃用追踪器](#model-deprecation-tracker)。如需更早的公告，请查阅[更新日志](https://featurebase.venice.ai/changelog)。

## 模型如何被纳入 Venice API

我们根据性能、可靠性和实际开发者需求精心挑选要提供的模型。要被纳入，模型必须表现出色，在 OpenAI 兼容端点下行为一致，并相对于我们已支持的至少一个模型提供明显改进。


我们正在评估的模型可能首先以 [beta](/overview/beta-models) 形式发布，以收集反馈并在大规模下验证性能。

我们不会公开冗余的、未经验证的或尚未准备好持续生产使用的模型。我们的目标是保持 Venice API 简洁、强大，并针对开发者实际构建的内容进行优化。

详情请参阅[模型弃用](/overview/deprecations#model-deprecations)和<a href="/overview/models" target="_blank" rel="noopener noreferrer">当前模型列表</a>。

## 版本管理与别名

所有 Venice 模型都由唯一、永久的 ID 标识。例如：

```venice-uncensored```
```zai-org-glm-4.7```
```zai-org-glm-5```
```qwen3-vl-235b-a22b```

模型 ID 是稳定的。如果存在破坏性变更，我们将发布新的模型 ID（例如添加 v2 之类的版本）。如果没有破坏性变更，我们可能会更新现有模型，并就重大变更进行沟通。

Traits 为选择模型提供了稳定的抽象层，同时让 Venice 能够灵活改进底层实现。希望自动访问最新推荐模型的开发者可以依赖基于 trait 的别名。

对于需要严格一致性和可预测行为的应用，我们建议引用固定的模型 ID。

### 通过 API 检查弃用状态

您可以通过调用 [List Models](/api-reference/endpoint/models/list) 端点来检查某个模型是否计划停用。带有停用日期的模型在其 `model_spec` 中包含一个 `deprecation` 对象：

```json
{
  "id": "some-model-id",
  "model_spec": {
    "name": "Some Model",
    "privacy": "private",
    "deprecation": {
      "date": "2025-03-01T00:00:00.000Z"
    }
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

`deprecation` 对象仅在模型计划停用时出现。您可以通过 `if (model.model_spec.deprecation)` 检查模型是否被停用，并使用 ISO 8601 日期警告用户或规划迁移。

## 处理错误

失败请求（500、503、429）应使用指数退避重试。

特别是对于 429 错误，请检查 `x-ratelimit-reset-requests` 响应头以获取你可以重试的精确 Unix 时间戳。大多数 HTTP 库都内置了自动处理这一过程的重试机制。

### 滥用防护

如果你在 30 秒内产生超过 20 个失败请求，API 会在接下来的 30 秒内阻止进一步的请求：

```
Too many failed attempts (> 20) resulting in a non-success status code. Please wait 30s and try again.
```

## 响应头

每个响应都会包含以下头部：

| 头部 | 说明 |
|:-------|:------------|
| `x-ratelimit-limit-requests` | 当前窗口允许的最大请求数 |
| `x-ratelimit-remaining-requests` | 当前窗口剩余的请求数 |
| `x-ratelimit-reset-requests` | 窗口重置的 Unix 时间戳 |
| `x-ratelimit-limit-tokens` | 每分钟允许的最大 token 数 |
| `x-ratelimit-remaining-tokens` | 当前分钟剩余的 token 数 |
| `x-ratelimit-reset-tokens` | token 限制重置前的剩余秒数 |

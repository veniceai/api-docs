## 质量层级定价

对于接受可选 `quality` 参数的图像模型（当前为 `gpt-image-2` 和 `gpt-image-2-edit`），响应中会在 `model_spec.pricing.quality` 下暴露按质量分档的价格矩阵。每个顶层键是分辨率层级（`1K`、`2K`、`4K`），每个嵌套键是质量级别（`low`、`medium`、`high`），并带有各自的 `usd` 与 `diem` 价格：

```json
"pricing": {
  "resolutions": {
    "1K": { "usd": 0.27, "diem": 0.27 },
    "2K": { "usd": 0.51, "diem": 0.51 },
    "4K": { "usd": 0.84, "diem": 0.84 }
  },
  "quality": {
    "1K": {
      "low":    { "usd": 0.02, "diem": 0.02 },
      "medium": { "usd": 0.07, "diem": 0.07 },
      "high":   { "usd": 0.26, "diem": 0.26 }
    },
    "2K": {
      "low":    { "usd": 0.03, "diem": 0.03 },
      "medium": { "usd": 0.13, "diem": 0.13 },
      "high":   { "usd": 0.50, "diem": 0.50 }
    },
    "4K": {
      "low":    { "usd": 0.05, "diem": 0.05 },
      "medium": { "usd": 0.21, "diem": 0.21 },
      "high":   { "usd": 0.83, "diem": 0.83 }
    }
  }
}
```

`pricing.resolutions` 是为向后兼容而保留的旧版按图定价表。`pricing.quality` 是按 (resolution, quality) 维度的矩阵，仅在支持 `quality` 参数时生效。两个字段都会保留在响应中，以便客户端检测对质量的支持情况并在自己的 UI 中展示该矩阵。

获取已认证用户的汇总使用分析数据，按日期、模型和 API 密钥进行细分。此端点提供 API 使用数据的摘要视图，可用于构建仪表板和监控消耗情况。数据缓存 10 分钟。

## 查询参数

你可以使用以下任一方式指定分析的时间段：
- **lookback**：相对时间段，如 "7d"（7 天）、"30d"（30 天），最长可至 "90d"（90 天）。
- **startDate** 和 **endDate**：采用 `YYYY-MM-DD` 格式的自定义日期范围。如果提供其中一个，则两者均为必填。

如果未指定任何参数，默认 lookback 时段为 7 天。

## 响应字段

### lookback
查询所用的 lookback 时段。可以是 "Nd" 格式（例如 "7d"）或 "startDate:endDate" 格式。

### byDate
请求时段内的每日使用总量。

- **date**：日期，采用 `YYYY-MM-DD` 格式
- **USD**：当天的美元使用总额
- **DIEM**：当天的 DIEM 使用总额

### byModel
按模型细分的使用数据，按总花费排序（最高在前）。

- **modelName**：模型的显示名（例如 "GLM 5"）
- **unitType**：所消耗单位的类型（tokens、images、chars、minutes、seconds）
- **modelType**：模型类型（LLM、IMAGE、TTS、ASR、VIDEO），或 null
- **totalUsd**：在该模型上花费的美元总额
- **totalDiem**：在该模型上花费的 DIEM 总额
- **totalUnits**：该模型消耗的总单位数
- **breakdown**：按类型细分的使用数组（仅在存在多个类型时出现）。每条记录包含：
  - **type**：token 类型（例如 "Input"、"Output"、"Cache Read"、"Cache Write"）
  - **usd**：该细分项的美元金额
  - **diem**：该细分项的 DIEM 金额
  - **units**：该细分项的单位数量

### byModelDaily
前 8 个模型的每日图表数据。每条记录包含一个 "date"（时间戳），加上以模型名为键、对应 DIEM 使用值为值的字段。

### topModels
按使用量排名前 8 的模型名数组，用于图表图例。

### byKey
按 API 密钥细分的使用数据，按总花费排序（最高在前）。

- **apiKeyId**：API 密钥 ID；如果使用来自 Web 应用则为 null
- **description**：API 密钥描述或 "Web App"
- **totalUsd**：通过该密钥花费的美元总额
- **totalDiem**：通过该密钥花费的 DIEM 总额
- **totalUnits**：通过该密钥消耗的总单位数

### byKeyDaily
前 8 个 API 密钥的每日图表数据。每条记录包含一个 "date"（时间戳），加上以密钥描述为键、对应 DIEM 使用值为值的字段。

### topKeyNames
按使用量排名前 8 的 API 密钥描述数组，用于图表图例。

## 使用示例

```bash
# 获取过去 7 天的使用分析（默认）
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics" \
  -H "Authorization: Bearer YOUR_API_KEY"

# 获取过去 30 天的使用分析
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics?lookback=30d" \
  -H "Authorization: Bearer YOUR_API_KEY"

# 获取特定日期范围的使用分析
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics?startDate=2024-01-01&endDate=2024-01-31" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 尺寸选项

图像模型使用与模型相关的尺寸参数：

- 基于像素的模型接受 `width` 和 `height`，例如 `venice-sd35` 和 `qwen-image`。
- 基于宽高比的模型接受 `aspect_ratio`，例如 `qwen-image-2`。
- 基于分辨率层级的模型同时接受 `aspect_ratio` 和 `resolution`。

例如，`gpt-image-2`、`nano-banana-2` 和 `nano-banana-pro` 支持的 `resolution` 取值为 `1K`、`2K` 和 `4K`：

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

同样的分辨率层级尺寸模式也适用于 Nano Banana 系列模型：

```json
{
  "model": "nano-banana-pro",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

对于接受宽高比但不暴露分辨率层级的模型，发送 `aspect_ratio` 而不带 `resolution`：

```json
{
  "model": "qwen-image-2",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9"
}
```

对于使用直接像素尺寸的模型，则改为发送 `width` 和 `height`：

```json
{
  "model": "venice-sd35",
  "prompt": "a serene canal in venice at sunset",
  "width": 1024,
  "height": 1024
}
```

在跨模型混合使用尺寸字段前，请查看 [图像模型](/zh/models/image) 或 [Models 端点](/zh/api-reference/endpoint/models/list)，了解每个模型支持的尺寸选项。

## 质量层级

部分模型接受可选的 `quality` 参数（`low`、`medium`、`high`），用于在视觉保真度和成本之间权衡。当前由 `gpt-image-2` 支持；其他模型会忽略该参数。

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K",
  "quality": "medium"
}
```

当你省略 `quality` 时，模型使用其默认层级（`gpt-image-2` 为 `high`）。定价取决于 `resolution` 和 `quality` 的组合 —— `gpt-image-2` 各层级（1K/2K/4K × low/medium/high）的价格位于 [Models 端点](/zh/api-reference/endpoint/models/list) 中 `model_spec.pricing.quality` 下，并在[定价概览](/zh/overview/pricing)中列出。

<Info>
质量感知的定价对所有 API 和 SDK 调用方都是默认行为 —— 无需选择加入的请求头。未传 `quality` 的请求按模型的默认层级计费（`gpt-image-2` 为 `high`）。
</Info>

发生错误时，Venice API 会返回 HTTP 状态码和描述性消息。某些错误还包含额外字段，例如退款状态、建议重试模型、验证细节或请求 ID。

## 错误响应格式

大多数错误返回一个带有人类可读消息的 `error` 字段：

```json
{
  "error": "Invalid request parameters"
}
```

某些端点会包含端点特有的额外字段。例如，视频审核错误可能包含是否退还了额度的信息：

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits have been refunded.",
  "credits_refunded": true
}
```

验证错误可能包含带有字段级验证信息的 `details` 和 `issues` 字段：

```json
{
  "error": "Invalid request parameters",
  "details": {
    "_errors": [],
    "field": {
      "_errors": ["Field is required"]
    }
  },
  "issues": []
}
```

某些 OpenAI 兼容端点可能针对特定情况（如上下文长度错误）返回 OpenAI 风格的错误对象：

```json
{
  "error": {
    "message": "Your request exceeds the model's maximum context. Please reduce your prompt or completion length.",
    "type": "invalid_request_error",
    "param": "messages",
    "code": "context_length_exceeded"
  }
}
```

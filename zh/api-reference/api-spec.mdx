Venice API 提供基于 HTTP 的 REST 与流式接口，用于构建采用未审查模型和私有推理的 AI 应用。你可以进行文本生成、图像生成、嵌入等创作，不受严格内容策略的限制。集成示例和 SDK 见[文档](/zh/overview/getting-started)。我们的 API 参考也可作为 [OpenAPI YAML 规范](https://api.venice.ai/doc/api/swagger.yaml)获取。

## 认证

Venice API 使用 API 密钥进行认证。在 [API 设置](https://venice.ai/settings/api) 中创建和管理你的 API 密钥。


所有 API 请求都需要 HTTP Bearer 认证：

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
你的 API 密钥是机密。请勿分享或将其暴露在任何客户端代码中。
</Note>

## OpenAI 兼容性

Venice 的 API 实现了 OpenAI API 规范，从而与现有的 OpenAI 客户端和工具兼容。这使你可以使用熟悉的 OpenAI 接口与 Venice 集成，同时享受 Venice 的独特功能和未审查模型。

### 设置

将客户端配置为使用 Venice 的基础 URL（`https://api.venice.ai/api/v1`）并发出第一个请求：

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## Venice 专属功能

### 系统提示

Venice 提供默认系统提示，旨在确保模型响应未审查且自然。你有两种方式处理系统提示：

1. **默认行为**：你的系统提示会被追加到 Venice 的默认提示之后
2. **自定义行为**：完全禁用 Venice 的系统提示

#### 禁用 Venice 系统提示

使用 `venice_parameters` 选项以移除 Venice 的默认系统提示：

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### Venice 参数

`venice_parameters` 对象让你可以访问标准 OpenAI API 中不可用的 Venice 专属功能：

| 参数 | 类型 | 说明 | 默认值 |
|-----------|------|-------------|---------|
| `character_slug` | string | Venice 公开角色的 slug（可在已发布角色页面上以 "Public ID" 的形式查到） | - |
| `strip_thinking_response` | boolean | 从响应中剥离 `<think></think>` 块（适用于使用旧版 `<think>` 标签格式的模型）。参见[推理模型](/zh/guides/features/reasoning-models)。 | `false` |
| `disable_thinking` | boolean | 在受支持的推理模型上禁用思考，并从响应中剥离 `<think></think>` 块 | `false` |
| `enable_web_search` | string | 为该请求启用网页搜索（`off`、`on`、`auto`，`auto` 表示由模型自主决定）<br/>会按使用量额外计费，参见[定价](/zh/overview/pricing#web-search-and-scraping)。 | `off` |
| `enable_web_scraping` | boolean | 对用户消息中检测到的最多 5 个 URL 启用网页抓取。抓取到的内容用于增强响应并绕过网页搜索。仅成功抓取的 URL 才会计费。<br/>会按使用量额外计费，参见[定价](/zh/overview/pricing#web-search-and-scraping)。 | `false` |
| `enable_x_search` | boolean | 为受支持的 Grok 模型（例如 `grok-4-20-beta`）启用 xAI 原生搜索（网页 + X/Twitter）。通过使用 xAI 的搜索基础设施获得更高质量的搜索结果。启用后，Venice 标准的网页搜索会被跳过。<br/>会按使用量额外计费，参见[定价](/zh/overview/pricing#web-search-and-scraping)。 | `false` |
| `enable_web_citations` | boolean | 启用网页搜索时，请求 LLM 使用 `[REF]0[/REF]` 格式引用其来源 | `false` |
| `include_search_results_in_stream` | boolean | 实验性：将搜索结果作为流中发出的第一个分块包含 | `false` |
| `return_search_results_as_documents` | boolean | 通过名为 `venice_web_search_documents` 的 OpenAI 兼容工具调用呈现搜索结果，方便 LangChain 集成 | `false` |
| `include_venice_system_prompt` | boolean | 是否在指定的系统提示之外再附加 Venice 的默认系统提示 | `true` |

<Note>
这些参数也可以作为模型后缀附加在模型名后（例如 `zai-org-glm-5:enable_web_search=auto`）。详情见[模型特性后缀](/zh/api-reference/endpoint/chat/model_feature_suffix)。
</Note>

### 提示缓存

Venice 在精选模型上支持提示缓存，以降低重复内容的延迟和成本。对于受支持的模型，Venice 会自动缓存系统提示 —— 无需修改代码。你也可以通过消息内容上的 `cache_control` 属性手动标记要缓存的内容。

| 参数 | 类型 | 说明 |
|-----------|------|-------------|
| `prompt_cache_key` | string | 可选的路由提示，用于提高缓存命中率。提供时，Venice 会将请求路由到相同的后端基础设施，提升多轮对话中的缓存命中可能性。 |

关于缓存机制、计费方式和最佳实践，请参见[提示缓存](/zh/guides/features/prompt-caching)。

## 响应头参考

所有 Venice API 响应都会包含 HTTP 头，用于提供请求的元数据、速率限制、模型信息和账户余额。除了 API 响应中返回的错误码外，你还可以检查这些头部以获取特定 API 请求的唯一 ID、监控速率限制并跟踪账户余额。

Venice 建议在生产环境中记录请求 ID（`CF-RAY` 头），以便在需要时与我们的支持团队进行更高效的故障排查。

下表是你可能遇到的所有头部的完整参考：

| 头部 | 类型 | 用途 | 返回时机 |
|--------|------|---------|---------------|
| **标准 HTTP 头** ||||
| `Content-Type` | string | 响应体的 MIME 类型（`application/json`、`text/csv`、`image/png` 等） | 始终 |
| `Content-Encoding` | string | 用于压缩响应体的编码（`gzip`、`br`） | 当客户端发送 `Accept-Encoding` 头时 |
| `Content-Disposition` | string | 内容应如何展示（例如 `attachment; filename=export.csv`） | 下载文件或导出数据时 |
| `Date` | string | 响应生成时的 RFC 7231 格式时间戳 | 始终 |
| **请求标识** ||||
| `CF-RAY` | string | 本次 API 请求的唯一标识，用于故障排查和支持请求 | 始终 |
| `x-venice-version` | string | Venice API 服务的当前版本/修订号（例如 `20250828.222653`） | 始终 |
| `x-venice-timestamp` | string | 处理请求时的服务器时间戳（ISO 8601 格式） | 启用时间戳跟踪时 |
| `x-venice-host-name` | string | 处理请求的服务器主机名 | 错误响应和调试场景 |
| **模型信息** ||||
| `x-venice-model-id` | string | 该请求所使用 AI 模型的唯一标识（例如 `venice-01-lite`） | 使用 AI 模型的推理端点 |
| `x-venice-model-name` | string | 所用 AI 模型的友好/显示名（例如 `Venice Lite`） | 使用 AI 模型的推理端点 |
| `x-venice-model-router` | string | 处理模型推理的路由/后端服务 | 推理端点（在有路由信息时） |
| `x-venice-model-deprecation-warning` | string | 计划弃用模型的警告消息 | 使用已弃用模型时 |
| `x-venice-model-deprecation-date` | string | 模型将被弃用的日期（ISO 8601 日期） | 使用已弃用模型时 |
| **速率限制信息** ||||
| `x-ratelimit-limit-requests` | number | 当前时间窗内允许的最大请求数 | 所有已认证请求 |
| `x-ratelimit-remaining-requests` | number | 当前时间窗内剩余的请求数 | 所有已认证请求 |
| `x-ratelimit-reset-requests` | number | 请求速率限制重置时的 Unix 时间戳 | 所有已认证请求 |
| `x-ratelimit-limit-tokens` | number | 时间窗内允许的最大 token 数（提示 + 补全） | 所有已认证请求 |
| `x-ratelimit-remaining-tokens` | number | 当前时间窗内剩余的 token 数 | 所有已认证请求 |
| `x-ratelimit-reset-tokens` | number | token 速率限制重置前的剩余秒数 | 所有已认证请求 |
| `x-ratelimit-type` | string | 所应用的速率限制类型（`user`、`api_key`、`global`） | 当强制执行速率限制时 |
| **分页头** ||||
| `x-pagination-limit` | number | 每页项目数 | 分页端点 |
| `x-pagination-page` | number | 当前页码（从 1 开始） | 分页端点 |
| `x-pagination-total` | number | 所有页面的项目总数 | 分页端点 |
| `x-pagination-total-pages` | number | 总页数 | 分页端点 |
| **账户余额信息** ||||
| `x-venice-balance-diem` | string | 请求处理前的 DIEM token 余额 | 所有已认证请求 |
| `x-venice-balance-usd` | string | 请求处理前的 USD credit 余额 | 所有已认证请求 |
| **内容安全头** ||||
| `x-venice-is-blurred` | string | 表示生成的图像是否因内容策略而被模糊（`true`/`false`） | 启用 Safe Venice 的图像生成 |
| `x-venice-is-content-violation` | string | 表示内容是否违反 Venice 的内容策略（`true`/`false`） | 内容生成端点 |
| `x-venice-is-adult-model-content-violation` | string | 表示内容是否违反成人模型的内容策略（`true`/`false`） | 图像生成端点 |
| `x-venice-contains-minor` | string | 表示图像中是否含有未成年人（`true`/`false`） | 启用年龄检测的图像分析端点 |
| **客户端信息** ||||
| `x-venice-middleface-version` | string | Venice middleface 客户端版本 | 来自 Venice middleface 客户端的请求 |
| `x-venice-mobile-version` | string | Venice 移动应用客户端版本 | 来自移动应用的请求 |
| `x-venice-request-timestamp-ms` | number | 客户端提供的请求时间戳（毫秒） | 客户端在请求中提供时间戳时 |
| `x-venice-control-instance` | string | 用于调试的控制实例标识 | 用于调试的图像生成端点 |
| **认证头** ||||
| `x-auth-refreshed` | string | 表示请求期间认证 token 是否被刷新（`true`/`false`） | 认证 token 自动刷新时 |
| `x-retry-count` | number | 该请求的重试次数 | 发生请求重试时 |

### 重要说明

- **头名称大小写**：HTTP 头不区分大小写，但 Venice 使用带连字符的小写以保持一致
- **字符串值**：头中的布尔值以字符串形式返回（`"true"` 或 `"false"`）
- **数值**：较大的数字和余额可能以字符串返回，以防止精度丢失
- **可选头**：并非每个响应都会返回所有头；存在与否取决于端点和请求上下文
- **压缩**：在请求中使用 `Accept-Encoding: gzip, br` 可在支持的情况下接收压缩响应

### 示例：访问响应头

```javascript
// 发出 API 请求后，从响应对象访问头部
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// 检查模型弃用警告
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## 最佳实践

1. **速率限制**：监控 `x-ratelimit-remaining-requests` 和 `x-ratelimit-remaining-tokens` 头部并实现指数退避
2. **余额监控**：跟踪 `x-venice-balance-usd` 和 `x-venice-balance-diem` 头部以避免服务中断
3. **系统提示**：分别在带和不带 Venice 系统提示的情况下测试，以为你的用例找到最佳配置
4. **API 密钥**：妥善保管你的 API 密钥并定期轮换
5. **请求日志**：记录 `CF-RAY` 头的值以便与支持团队进行故障排查
6. **模型弃用**：使用模型时检查 `x-venice-model-deprecation-warning` 头

## 与 OpenAI API 的差异

虽然 Venice 与 OpenAI API 规范保持高度兼容，但仍有一些关键差异：

1. **venice_parameters**：额外的配置项，如 `enable_web_search`、`character_slug` 和 `strip_thinking_response`，用于扩展功能
2. **系统提示**：Venice 会将你的系统提示追加到针对未审查响应优化的默认提示之后（可通过 `include_venice_system_prompt: false` 禁用）
3. **模型生态**：Venice 拥有自己的[模型阵容](/zh/overview/models)，包括未审查模型和推理模型 —— 使用 Venice 的模型 ID，而非 OpenAI 映射
4. **响应头**：用于余额跟踪（`x-venice-balance-usd`、`x-venice-balance-diem`）、模型弃用警告和内容安全标记的专属头部
5. **内容策略**：拥有专门的未审查模型并提供可选内容过滤，策略更为宽松

## API 稳定性

Venice 对 v1 端点和参数保持向后兼容。关于模型生命周期策略、弃用通知和迁移指引，请参阅[弃用](/zh/overview/deprecations)。

Venice AI 是 OpenAI 的**即插即用替代品**。相同的 SDK、相同的代码 —— 只需修改两行。获得隐私优先的推理、未审查的模型与具有竞争力的定价。

## 两行式迁移

### Python

```python
# 之前（OpenAI）
from openai import OpenAI
client = OpenAI()

# 之后（Venice）—— 修改 api_key 和 base_url
from openai import OpenAI
client = OpenAI(
    api_key="your-venice-api-key",          # ← 修改 1
    base_url="https://api.venice.ai/api/v1"  # ← 修改 2
)
```

### Node.js

```javascript
// 之前（OpenAI）
import OpenAI from 'openai';
const client = new OpenAI();

// 之后（Venice）
import OpenAI from 'openai';
const client = new OpenAI({
  apiKey: 'your-venice-api-key',
  baseURL: 'https://api.venice.ai/api/v1',
});
```

### cURL

```bash
# 之前
curl https://api.openai.com/v1/chat/completions ...

# 之后 —— 只需更改 URL 和密钥
curl https://api.venice.ai/api/v1/chat/completions ...
```

### 环境变量

```bash
# 之前
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1

# 之后
OPENAI_API_KEY=your-venice-api-key
OPENAI_BASE_URL=https://api.venice.ai/api/v1
```

<Tip>
许多库和工具会自动读取 `OPENAI_API_KEY` 和 `OPENAI_BASE_URL`。只需更新这些环境变量可能就足够了。
</Tip>

## 功能兼容性

| 功能 | OpenAI | Venice | 说明 |
|---------|--------|--------|-------|
| 聊天补全 | ✅ | ✅ | 完全兼容 |
| 流式传输 | ✅ | ✅ | SSE 格式相同 |
| 函数调用 | ✅ | ✅ | 相同的 `tools` 参数 |
| 结构化输出 | ✅ | ✅ | 相同的 `response_format` |
| 视觉 | ✅ | ✅ | 相同的 content 数组格式 |
| 嵌入 | ✅ | ✅ | 相同的 API |
| 图像生成 | ✅ | ✅ | 通过 `/images/generations` 与 OpenAI 兼容* |
| TTS | ✅ | ✅ | 兼容 |
| STT | ✅ | ✅ | 兼容 |
| Assistants API | ✅ | ❌ | 改用 Characters 或 Minds |

| Fine-tuning | ✅ | ❌ | 不提供 |

*Venice 还在 `POST /images/generations` 提供了一个 OpenAI 兼容端点，便于从 DALL-E 迁移。如需 Venice 原生的图像 API 及更多选项，请参见 [图像生成](/zh/api-reference/endpoint/image/generate)。

## Venice 独有功能

Venice 提供 OpenAI 没有的能力：

### 1. 内置网页搜索
```python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Latest AI news today"}],
    extra_body={
        "venice_parameters": {
            "enable_web_search": "auto"
        }
    }
)
```

### 2. 网页抓取
```python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Summarize https://example.com/article"}],
    extra_body={
        "venice_parameters": {
            "enable_web_scraping": True
        }
    }
)
```

### 3. Characters（AI 角色）
```python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Tell me about yourself"}],
    extra_body={
        "venice_parameters": {
            "character_slug": "venice-ai"
        }
    }
)
```

### 4. 未审查模型
Venice 的私有模型不进行内容过滤，适合用于：
- 没有护栏的创意写作
- 安全研究和红队演练
- 没有拒绝模式的诚实分析
- 没有免责声明的医疗/法律信息

### 5. 视频生成
```python
# 排队一个视频生成任务
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"},
    json={
        "model": "wan-2.6-text-to-video",
        "prompt": "A serene lake at sunset with gentle waves",
        "resolution": "720p",
        "duration": 5,
    }
)
job_id = response.json()["id"]
```

## 为什么迁移？

### 隐私
- 私有模型采用**零数据留存** —— 你的提示永远不会被存储
- **绝不**使用你的数据进行训练
- OpenAI 会保留数据 30 天，并可能用于安全研究

### 成本
- 私有模型通常**比 OpenAI 等价模型更便宜**
- `qwen3-4b` 以 $0.05/100 万输入计费，比 gpt-4o-mini 便宜 10 倍
- `venice-uncensored` 输入 $0.20/100 万 vs gpt-4o $2.50/100 万

### 自由
- 未审查模型**无内容过滤**
- 不会因有争议的用例而暂停账户
- 原生 Web3，支持加密货币支付
- 通过质押 DIEM 获取每日 credit

### 模型多样性
- 可访问多家供应商的模型（Qwen、Llama、Mistral、Gemma、Claude、GPT、Grok 等）
- 可在私有模型与匿名模型之间按请求切换
- 定期增加新模型

## 框架迁移

通过更改 base URL，大多数 AI 框架都能与 Venice 配合使用：

| 框架 | 所需更改 |
|-----------|----------------|
| LangChain | `ChatOpenAI` 中的 `base_url` |
| Vercel AI SDK | `createOpenAI` 中的 `baseURL` |
| CrewAI | `OPENAI_API_BASE` 环境变量 |
| LlamaIndex | `OpenAI` 中的 `api_base` |
| AutoGen | 配置中的 `base_url` |
| Haystack | `OpenAIGenerator` 中的 `api_base_url` |
| Claude Code | `--api-base` 标志或环境变量 |
| Cursor | 设置中的自定义 API endpoint |
| Continue.dev | `config.json` 中的 `apiBase` |

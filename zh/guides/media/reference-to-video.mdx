Reference to Video 让你能够锁定角色、物体和场景的外观，让 AI 生成的视频保持视觉一致性。你不再需要寄希望于模型正确解读 prompt，而是提供视觉锚点——告诉模型主体外观的参考图像。

此功能在 [Venice Video Studio](https://venice.ai/video) 中的 **Kling O3** 和 **Grok Imagine R2V** 模型上可用。每个模型家族对参考图像采用不同的方式——请参阅下方按模型划分的章节。

## 何时使用 Reference to Video

当你需要以下功能时，使用 Reference to Video：

- **角色一致性**——多个镜头中保持同一人物或角色
- **产品准确性**——真实产品必须看起来与原版完全一致
- **场景连续性**——多次生成中保持特定的环境或背景
- **多角色场景**——多个不同角色互动且不会相互融合

对于一致性不关键的简单文本生视频或图像生视频，标准模型在没有参考的情况下也能很好工作。

## 可用模型

| 模型 | 方法 | 适用场景 |
|---|---|---|
| **Kling O3 Pro R2V** | Elements + 场景图像 | 需要精确身份控制的复杂多角色场景 |
| **Kling O3 Standard R2V** | Elements + 场景图像 | 基于元素的场景的快速迭代 |
| **Grok Imagine R2V** | 扁平参考图像 | 使用最多 7 张图像的快速参考驱动生成 |

**Kling O3** 采用结构化方式，使用 Elements（带正面图 + 参考图的角色身份锚点）和场景图像。**Grok Imagine R2V** 采用更简单的方式——你直接上传参考图像，并在 prompt 中通过 `@Image1`、`@Image2` 等方式引用。

## Kling O3 Reference to Video

### 核心概念

Kling O3 Reference to Video 使用三种类型的视觉输入协同工作：

| 输入 | 必需 | 用途 | 在 prompt 中如何引用 |
|---|---|---|---|
| **Elements** | 至少一项视觉输入* | 锁定角色或物体的身份 | `@Element1`、`@Element2` 等 |
| **场景参考图像** | 至少一项视觉输入* | 设置环境、风格和氛围 | `@Image1`、`@Image2` 等 |
| **起始帧** | 至少一项视觉输入* | 控制视频的第一帧 | 不适用（通过上传设置） |
| **结束帧** | 否 | 控制视频的最后一帧 | 不适用（通过上传设置） |

*至少需要以下之一：起始帧、elements 或场景参考图像。

### Elements

Element 是你希望在整个视频中保持视觉稳定的角色或物体。每个 element 包含：

- **正面图像**（每个 element 必需）——主体清晰的正面照片。这是主要的身份锚点。可以把它想成你角色或产品的"护照照片"。
- **参考图像**（1–3 张，可选）——同一主体的其他角度（侧视、45 度角、背面）。它们帮助模型在 3D 空间中理解主体。如果未提供，正面图像会自动用作参考。

每次生成最多可添加 **7 个 elements**（受总数限制）。在 prompt 中使用 `@Element1`、`@Element2` 等引用。

### 场景参考图像

场景参考定义动作发生的"舞台"。它们影响：

- 光照和色彩调色板
- 建筑与环境细节
- 整体视觉风格和氛围

你最多可以添加 **4 张场景图像**。在 prompt 中以 `@Image1`、`@Image2` 等形式引用。

### 限制

所有输入类型的图像总数受限：

| 限制 | 数值 |
|---|---|
| **最低要求** | 至少 1 项视觉输入（起始帧、element 或场景图像） |
| **总数合计**（起始帧 + 结束帧 + elements + 场景图像） | **最多 7 张** |
| Elements（不含起始/结束帧） | 最多 7 个 |
| Elements（含起始或结束帧） | 最多 3 个 |
| 场景参考图像 | 最多 4 张 |
| 每个 element 的参考图像 | 1–3 张 |

**示例场景：**

- 7 个 elements + 0 张场景图像 = 7 ✓（无帧）
- 5 个 elements + 2 张场景图像 = 7 ✓（无帧）
- 起始帧 (1) + 3 个 elements + 3 张场景图像 = 7 ✓
- 起始帧 (1) + 结束帧 (1) + 3 个 elements + 2 张场景图像 = 7 ✓
- 起始帧 (1) + 4 个 elements = ✗（带帧时 elements 最多 3 个）
- 起始帧 (1) + 结束帧 (1) + 4 个 elements = ✗（带帧时 elements 最多 3 个）

<Note>
每个 element 都需要一张**正面图像**。如果你没有为某个 element 提供参考图像，正面图像会自动用作参考。
</Note>

### 多镜头模式

多镜头模式允许你将一次生成拆分为多个场景，每个场景有自己的 prompt 和时长。Elements 和场景参考会在所有镜头中延续，保持一致性。所有镜头的总时长不能超过 **15 秒**。

### 分步指南（Video Studio）

#### 1. 打开 Video Studio 并选择模型

前往 [venice.ai/video](https://venice.ai/video)。在左侧的 Model Browser 中，选择 **Kling O3 Reference to Video** 模型之一：

- **Kling O3 Pro R2V**——更高质量，生成时间更长（约 6 分钟）
- **Kling O3 Standard R2V**——更快，更经济，适合迭代

#### 2. 添加视觉输入（至少需要一项）

你必须提供**至少一项视觉输入**才能生成视频：起始帧、element 或场景参考图像。在输入面板中你会看到 **Elements** 部分。点击 **Add Element** 为想要保持视觉一致的角色或物体创建 element。

对每个 element：

1. 点击 **Frontal** 方块上传角色或物体的清晰正面图像
2. 可选地在 **Reference Images** 下点击 **Add** 上传其他角度（1–3 张）

为其他角色或物体重复此操作（最多 7 个 elements，使用起始/结束帧时为 3 个）。

<Warning>
起始帧、结束帧、elements 和场景图像的总和不能超过 **7**。详情请参阅 [限制](#限制)。
</Warning>

<Tip>
**最佳参考图像：** 使用光线充足、背景干净的照片。提供正面、侧面和 45 度角视图以获得最强的身份锁定效果。确保所有参考图像共享相同的视觉风格（不要混用写实和动漫）。
</Tip>

#### 3. 添加场景参考图像（可选）

在 Elements 部分下方，你会看到 **Scene Reference Images**。上传定义所需环境的图像——特定地点、光照设置或艺术风格。

它们会自动被标记为 `@Image1`、`@Image2` 等。

#### 4. 上传起始帧（可选）

如果你想精确控制视频的第一帧，切换到 **Image** 输入类型并上传起始帧。你也可以选择性地设置结束帧。

#### 5. 编写 prompt

在 prompt 字段中，描述你想要的动作，并使用 `@` 标签引用你的 elements 和场景图像：

```
@Element1 walks through the streets of @Image1, looking up at the buildings.
The camera slowly tracks from behind, revealing the city skyline.
```

对于**多角色场景**：

```
@Element1 and @Element2 enter the cafe in @Image1 from opposite sides.
@Element1 waves and walks toward @Element2, who is sitting at a corner table.
```

#### 6. 配置设置

打开 **Video Settings** 调整：

| 设置 | 选项 | 默认值 |
|---|---|---|
| 时长 | 3s – 15s | 5s |
| 宽高比 | 16:9, 9:16, 1:1 | 16:9 |
| 生成音频 | On/Off | Off |

<Note>
音频生成会添加与视频同步的原生音效、对话和环境音频。成本约增加 25%。
</Note>

#### 7. 生成

点击 **Generate Video**。Kling O3 通常需要 4–6 分钟，具体取决于模型档次和时长。你可以排队多次生成，并在 Video Gallery 中浏览结果。

### 多镜头分镜

对于叙事序列，使用多镜头模式在单次生成中定义独立的场景。

1. 在 prompt 区域，点击 **Add Shot** 创建额外镜头
2. 为每个镜头编写单独的 prompt
3. 设置每个镜头的时长（每个 3–15s，总计 ≤ 15s）

Elements 和场景参考会自动在所有镜头中持续：

```
Shot 1 (5s): @Element1 stands at the edge of @Image1, looking out at the horizon.
Slow camera push forward.

Shot 2 (5s): Close-up of @Element1's face as they turn toward the camera.
Soft natural lighting, shallow depth of field.

Shot 3 (5s): @Element1 walks away from camera into the distance.
Wide cinematic shot, golden hour lighting.
```

<Warning>
多镜头总时长不能超过 15 秒。例如三个 5 秒镜头 = 最多 15s。
</Warning>

### Prompt 编写技巧

#### 构造你的 prompt

按以下模式可获得可靠结果：

```
[带 @Element 标签的主体] + [动作] + [带 @Image 标签的环境] + [镜头运动] + [光照/风格]
```

**示例：**

```
@Element1 hops happily across the candy ground of @Image1, stops to look at a
giant lollipop, tilts its head curiously. Cinematic tracking shot, soft warm lighting.
```

#### 保持 prompt 在 50–150 词

过短的 prompt 缺乏细节。过长的 prompt 会引入矛盾。瞄准最佳区间。

#### 使用简单的镜头语言

模型对直接的镜头指令响应最佳：

| 推荐 | 避免 |
|---|---|
| `slow camera push forward` | `dolly zoom with rack focus transition` |
| `tracking shot from behind` | `complex handheld parallax movement` |
| `close-up` | `extreme macro with tilt-shift bokeh` |
| `wide cinematic shot` | `anamorphic ultra-wide establishing crane shot` |

#### 使用一致的词汇

如果你在一个 prompt 中描述角色穿着 "a red jacket"，不要在下一个 prompt 中改为 "crimson coat"。模型会把不同的词当作不同的意图。

#### 把镜头指令放在前面

把镜头方向放在 prompt 开头附近以获得更可靠的结果：

```
Cinematic tracking shot of @Element1 walking through @Image1, leaves
blowing in the wind, golden afternoon light.
```

### Kling O3 API 使用

Kling O3 Reference to Video 也通过 Venice API 提供。完整细节请参阅 [Video Queue API](https://docs.venice.ai/api-reference/endpoint/video/queue)。

#### Python

```python
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "kling-o3-pro-reference-to-video",
        "prompt": "@Element1 walks through @Image1, camera tracking from behind",
        "duration": "8",
        "aspect_ratio": "16:9",
        "audio": True,
        "elements": [
            {
                "frontal_image_url": "https://example.com/character-front.jpg",
                "reference_image_urls": [
                    "https://example.com/character-side.jpg",
                    "https://example.com/character-angle.jpg"
                ]
            }
        ],
        "image_urls": [
            "https://example.com/scene-background.jpg"
        ]
    }
)

queue_id = response.json()["id"]
```

```javascript

  },
```

```bash
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \

    "audio": true,

      {

        ]
      }

      "https://example.com/scene-background.jpg"
```

#### Element 模式

`elements` 数组中的每个 element 接受：

| 字段 | 类型 | 必需 | 说明 |
|---|---|---|---|
| `frontal_image_url` | string | **是** | 清晰的正面图像 URL |
| `reference_image_urls` | string[] | 否 | 其他角度的 URL（1–3 张）。如果省略，正面图像将用作参考。 |

<Note>
API 还支持用于基于视频的 elements 的 `video_url`，但当前在 Video Studio UI 中尚不可用。
</Note>

### Kling O3 故障排查

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| Generate 按钮不可用 | 未提供视觉输入 | 至少添加一项视觉输入：起始帧、element 或场景参考图像 |
| "Number of images exceeds the limit" 错误 | 组合输入过多 | 起始帧 + 结束帧 + elements + 场景图像的总和必须 ≤ 7 |
| 角色面孔在不同镜头间发生变化 | 正面图像不同或缺失 | 始终使用相同的正面图像，保持描述一致 |
| 镜头运动看起来随机 | 多条或冲突的镜头指令 | 使用单一镜头指令，并放在 prompt 开头 |
| 不同生成之间风格漂移 | 场景参考不一致或风格混杂 | 复用相同的场景图像，保持风格关键词一致 |
| 多角色场景中元素融合 | 空间指令模糊 | 明确每个 element 的位置："foreground left"、"entering from right" |
| 背景看起来扭曲 | 场景参考图像杂乱或过于复杂 | 使用干净、高质量的场景参考图像 |
| 动作看起来不自然 | 单个 prompt 中动作过多 | 简化动作，使用更短时长，每个镜头一个动作 |

<Tip>
在使用更长时长之前，先用 3–5 秒的剪辑进行测试。较短的剪辑能保持更好的一致性，让你更快迭代。
</Tip>

## Grok Imagine Reference to Video

Grok Imagine R2V 采用比 Kling O3 更简单的方式。它不使用结构化的 Elements（正面/参考分离），而是上传**扁平参考图像**，并在 prompt 中直接用 `@Image1`、`@Image2` 等引用它们。模型会将这些主体合入生成的视频中。

### 工作方式

1. 上传 **1–7 张参考图像**——你希望出现在视频中的角色、物体或场景的照片
2. 编写描述视频的 prompt，使用 `@Image1`、`@Image2` 等引用特定图像
3. 模型生成包含这些参考的视频

如果你的 prompt 中没有 `@Image` 标签，所有上传的图像会被自动引用。

### 设置

| 设置 | 选项 | 默认值 |
|---|---|---|
| 宽高比 | 16:9, 4:3, 3:2, 1:1, 2:3, 3:4, 9:16 | 16:9 |
| 分辨率 | 480p, 720p | 480p |
| 时长 | 5s, 8s, 10s | 8s |

<Note>
Grok Imagine R2V 不支持音频生成、多镜头模式或 Elements。需要这些功能请使用 Kling O3 R2V。
</Note>

### 分步指南（Video Studio）

#### 1. 选择模型

前往 [venice.ai/video](https://venice.ai/video)。在 Model Browser 中选择 **Grok Imagine R2V**。

#### 2. 上传参考图像

在输入工具栏中点击 **References**（或使用 + 菜单）打开参考图像面板。上传 1–7 张你想要出现在视频中的角色、物体或场景的图像。

每张图像会按上传顺序（从左到右）自动标记为 `@Image1`、`@Image2` 等。

#### 3. 编写 prompt

描述你想要的视频。使用 `@Image` 标签引用特定图像：

```
@Image1 and @Image2 walking together through a sunlit park,
camera slowly tracking alongside them, warm afternoon light.
```

在 prompt 字段中输入 `@` 可看到可用图像引用的自动完成菜单。

<Tip>
如果你完全省略 `@Image` 标签，后端会自动在前面附加对所有已上传图像的引用。当你希望使用所有图像而不必指明哪个对应哪个时，这很有用。
</Tip>

#### 4. 配置设置并生成

打开 **Video Settings** 调整宽高比、分辨率和时长。点击 **Generate Video**。

### Grok Imagine R2V 定价

Grok Imagine R2V 基于时长和分辨率定价：

| 分辨率 | 每秒 |
|---|---|
| 480p | ~$0.063 |
| 720p | ~$0.088 |

**示例：** 480p 的 8 秒视频 = 8 × $0.063 = **~$0.50**

<Note>
Grok Imagine 对生成的视频收取内容审核费用，即使视频被拒绝也会收费。这会反映在生成前显示的积分成本中。
</Note>

### Grok Imagine R2V 故障排查

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| Generate 按钮不可用 | 未上传参考图像 | 至少上传 1 张参考图像 |
| "At least one reference image is required" 错误 | `referenceImageUrls` 为空或缺失 | 在 `referenceImageUrls` 中至少提供一个图像 URL |
| 与 `@Image` 标签关联的图像不对 | 图像顺序与标签不匹配 | `@Image1` 对应上传顺序中的第一张图像（从左到右）。如有需要请重新排列上传顺序。 |
| 主体未出现在视频中 | 参考过多但缺少显式标签 | 在 prompt 中使用 `@Image` 标签明确指出要使用哪些图像 |
| 输出质量低 | 使用了 480p 分辨率 | 尝试 720p 以获得更高质量（成本更高） |
| 视频太短 | 默认时长为 8s | 将时长设置为 `"10"` 以获得更长的视频 |

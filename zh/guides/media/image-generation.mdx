Venice 上的图像生成是同步的。向 `/image/generate` 发送 prompt，并在同一响应中获得图像——要么作为 JSON 内的 base64，要么在 `return_binary` 为 `true` 时作为原始二进制数据。

## Endpoints

| Endpoint | 用途 | 何时使用 |
|---|---|---|
| `POST /image/generate` | Venice 原生图像生成 API | 需要完整功能支持时使用 |
| `GET /image/styles` | 列出可用的样式预设 | 在发送 `style_preset` 前使用 |
| `POST /images/generations` | 与 OpenAI 兼容的图像生成 API | 迁移现有 OpenAI 图像客户端时使用 |

## 第 1 步：发送生成请求

尺寸取决于具体模型。有些模型接受显式的 `width` 和 `height`；有些公开 `aspect_ratio`；分辨率分级模型则公开 `aspect_ratio` 加上 `1K`、`2K` 或 `4K` 等 `resolution` 值。

**基于像素的尺寸示例：**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "venice-sd35",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
  "negative_prompt": "blurry, low quality, distorted anatomy, text, watermark",
  "width": 1024,
  "height": 1024,
  "format": "webp"
}
```

**基于宽高比的尺寸示例：**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-image-2",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "format": "webp"
}
```

**基于分辨率分级的尺寸示例：**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "gpt-image-2",
  "prompt": "A cinematic wide shot of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

同样的模式适用于其他分辨率分级模型：

```json
{
  "model": "nano-banana-pro",
  "prompt": "A serene canal in Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

使用 [图像模型](/zh/models/image) 或 [Models API](/zh/api-reference/endpoint/models/list) 来确认每个模型接受哪些尺寸字段。

**响应 (200)：**
```json
{
  "id": "generate-image-1234567890",
  "images": [
    "UklGRiIAAABXRUJQVlA4IBYAAAAwAQCdASoQABAAPm..."
  ],
  "timing": {
    "inferenceDuration": 1840,
    "inferencePreprocessingTime": 22,
    "inferenceQueueTime": 31,
    "total": 1893
  }
}
```

`images` 数组包含 base64 编码的图像数据。解码第一个元素以保存或显示它。`timing.total` 是请求的完整持续时间（毫秒）。

## 第 2 步：解码并保存图像

<CodeGroup>
```python Python
import base64
import os
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/image/generate",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "venice-sd35",
        "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
        "width": 1024,
        "height": 1024,
        "format": "webp",
    },
)

data = response.json()
image_bytes = base64.b64decode(data["images"][0])

with open("output.webp", "wb") as f:
    f.write(image_bytes)

print(f"Saved image from request {data['id']}")
```

```javascript Node.js
import fs from "fs";

const response = await fetch("https://api.venice.ai/api/v1/image/generate", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "venice-sd35",
    prompt: "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
    width: 1024,
    height: 1024,
    format: "webp",
  }),
});

const data = await response.json();
const imageBuffer = Buffer.from(data.images[0], "base64");
fs.writeFileSync("output.webp", imageBuffer);

console.log(`Saved image from request ${data.id}`);
```
</CodeGroup>

## 第 3 步：返回二进制数据而非 JSON（可选）

如果你希望响应体就是图像文件本身，请设置 `return_binary: true`。这在你想直接流式传输或保存图像而无需 base64 解码时很有用。

```bash
curl https://api.venice.ai/api/v1/image/generate \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o output.png \
  -d '{
    "model": "qwen-image-2",
    "prompt": "Minimalist poster of a moonlit Venetian bridge in deep blue tones",
    "format": "png",
    "return_binary": true
  }'
```

当 `return_binary` 为 `true` 时，响应体是基于你请求的 `format` 的原始 `image/jpeg`、`image/png` 或 `image/webp` 数据。

<Note>
`variants` 仅在 `return_binary` 为 `false` 时支持。
</Note>

## 第 4 步：列出可用的图像样式（可选）

如果你想使用 `style_preset`，请先从 `/image/styles` 获取可用样式：

```bash
curl https://api.venice.ai/api/v1/image/styles \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

**响应 (200)：**
```json
[
  "3D Model",
  "Analog Film",
  "Anime",
  "Cinematic",
  "Digital Art"
]
```

然后在生成请求中传入其中一个值：

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "Cinematic"
}
```

当你想要精确的预设名称而不是猜测时，可使用样式 endpoint。

## 请求参数

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|---|---|---|---|---|
| `model` | string | 是 | - | 用于生成的模型 ID |
| `prompt` | string | 是 | - | 要生成的内容 |
| `negative_prompt` | string | 否 | - | 图像中要避免的内容 |
| `width` | integer | 否 | `1024` | 基于像素的模型（如 `venice-sd35` 和 `qwen-image`）的输出宽度（像素） |
| `height` | integer | 否 | `1024` | 基于像素的模型（如 `venice-sd35` 和 `qwen-image`）的输出高度（像素） |
| `format` | string | 否 | `webp` | 输出格式：`jpeg`、`png` 或 `webp` |
| `variants` | integer | 否 | `1` | 要生成的图像数量（`1`-`4`），仅在 `return_binary` 为 `false` 时有效 |
| `return_binary` | boolean | 否 | `false` | 返回原始图像字节而非 base64 JSON |
| `safe_mode` | boolean | 否 | `true` | 启用时模糊处理成人内容 |
| `seed` | integer | 否 | 随机 | 重复使用同一 seed 以获得更一致的迭代结果 |
| `cfg_scale` | number | 否 | 取决于模型 | 较高的值促使模型更严格地遵循 prompt |
| `style_preset` | string | 否 | - | 应用来自 [图像样式](/zh/api-reference/endpoint/image/styles) 的预设样式 |
| `aspect_ratio` | string | 条件 | - | 由支持基于宽高比尺寸的模型使用，如 `qwen-image-2`、`gpt-image-2`、`nano-banana-2` 和 `nano-banana-pro` |
| `resolution` | string | 条件 | - | 由支持分辨率分级（如 `1K`、`2K`、`4K`）的模型使用 |
| `enable_web_search` | boolean | 条件 | `false` | 允许支持的模型使用当前的网络信息；会增加额外费用 |

校验是模型特定的。在依赖某个参数跨多个模型使用之前，请查阅 [图像模型](/zh/models/image) 和 [Models API](/zh/api-reference/endpoint/models/list)。

## 模型专属选项

### 高分辨率生成

部分图像模型支持 `aspect_ratio`，但没有可选择的 `resolution` 分级。例如 `qwen-image-2` 接受宽高比并将其映射到模型特定的输出尺寸：

```json
{
  "model": "qwen-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9"
}
```

其他图像模型同时支持 `aspect_ratio` 和 `resolution` 分级。例如 `gpt-image-2`、`nano-banana-2` 和 `nano-banana-pro` 支持 `1K`、`2K` 和 `4K`：

```json
{
  "model": "gpt-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "4K"
}
```

```json
{
  "model": "nano-banana-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

使用 [图像模型](/zh/models/image) 查看哪些模型支持更高分辨率以及它们的定价方式。

### 样式预设

如果所选模型支持，`style_preset` 可让你在不重写整个 prompt 的情况下引导输出。你可以从 [图像样式](/zh/api-reference/endpoint/image/styles) 获取有效的预设名称：

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "3D Model"
}
```

查看 [图像样式](/zh/api-reference/endpoint/image/styles) 了解当前的样式列表。

## OpenAI 兼容 endpoint

如果你已在使用 OpenAI 图像 SDK 或现有的 DALL-E 集成，Venice 也支持 `POST /images/generations`。它提供更简单的请求格式，但功能少于 Venice 原生 endpoint。

**请求：**
```json
{
  "model": "qwen-image-2",
  "prompt": "A clean isometric illustration of an AI control room",
  "size": "1024x1024",
  "response_format": "b64_json"
}
```

使用 OpenAI 兼容路径可加快迁移。当你需要 Venice 专属选项（如 `cfg_scale`、`style_preset`、`variants` 或二进制响应）时，请使用 `/image/generate`。

## Prompt 编写技巧

1. 先写主体，再添加媒介、光照、构图和氛围。
2. 把必须避免的细节放在 `negative_prompt` 里，而不是塞进主 prompt。
3. 迭代时重复使用 `seed`，以便在不彻底改变构图的情况下对比 prompt 的变更。
4. 让尺寸适配模型。有些模型用 `width`/`height`，有些用 `aspect_ratio`，分辨率分级模型则用 `aspect_ratio` 加 `resolution`。
5. 探索阶段使用 `variants`，确定方向后再切回单一输出。

## 错误

| 状态 | 含义 | 操作 |
|---|---|---|
| `400` | 请求参数无效 | 检查字段名、类型以及模型特定的约束 |
| `401` | 认证失败或模型需要更高访问等级 | 检查你的 API key 和模型访问权限 |
| `402` | 余额不足 | 在 [venice.ai/settings/api](https://venice.ai/settings/api) 充值 |
| `415` | 内容类型无效 | 使用 `Content-Type: application/json` 发送 JSON |
| `429` | 超出速率限制或模型过载 | 使用退避策略重试；查看 `Retry-After` 响应头 |
| `500` | 推理处理失败 | 重试请求 |
| `503` | 模型已达容量上限 | 短暂延迟后重试 |

<Note>
启用 Safe Venice 时，如果你需要以编程方式检测内容审核结果，请检查响应头，如 `x-venice-is-blurred` 和 `x-venice-is-content-violation`。
</Note>

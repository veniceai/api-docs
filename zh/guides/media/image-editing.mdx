Venice 上的图像编辑是同步的。将源图像发送到 `/image/edit` 或 `/image/multi-edit`，编辑结果会在同一响应中以 PNG 文件形式返回。对于抠图，`/image/background-remove` 返回带透明背景的 PNG。

<Warning>
图像编辑 endpoints 处于实验性阶段，模型特定行为可能随时间变化。
</Warning>

## Endpoints

| Endpoint | 用途 | 最佳用途 |
|---|---|---|
| `POST /image/edit` | 使用 prompt 编辑单张图像 | 通用编辑和基于 prompt 的内绘 |
| `POST /image/multi-edit` | 使用 1-3 张分层图像编辑 | 借助蒙版或叠加层进行更可控的编辑 |
| `POST /image/background-remove` | 去除图像背景 | 用于产品、肖像和素材的透明抠图 |

## 何时使用哪个 endpoint

- 当你有一张源图像，并希望用 prompt 改变、移除或重新风格化其中一部分时，使用 `/image/edit`。
- 当你需要借助蒙版、叠加层或参考层进行额外控制时，使用 `/image/multi-edit`。
- 当你只想要一个干净的前景主体并带透明背景时，使用 `/image/background-remove`。

<Note>
若需内绘，请使用 `/image/edit` 或 `/image/multi-edit`。`/image/generate` 上旧的 `inpaint` 参数已弃用。
</Note>

## 第 1 步：编辑单张图像

单图编辑是最简单的内绘流程。发送一张图像加一个简短的 prompt，例如 "remove the sign"、"change the sky to sunrise" 或 "replace the background with a studio backdrop"。

**请求：**
```bash
POST https://api.venice.ai/api/v1/image/edit
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-edit",
  "prompt": "Replace the cloudy sky with a warm sunrise while preserving the buildings and canal",
  "image": "https://example.com/venice-canal.jpg"
}
```

**响应 (200)：**
响应体是原始 `image/png` 二进制数据。直接保存为文件即可。

<CodeGroup>
```python Python
import base64
import os
import requests

with open("input.jpg", "rb") as f:
    image_base64 = base64.b64encode(f.read()).decode("utf-8")

response = requests.post(
    "https://api.venice.ai/api/v1/image/edit",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "qwen-edit",
        "prompt": "Remove the tourist crowd from the square and keep the architecture intact",
        "image": image_base64,
    },
)

with open("edited.png", "wb") as f:
    f.write(response.content)
```

```javascript Node.js
import fs from "fs";

const imageBase64 = fs.readFileSync("input.jpg").toString("base64");

const response = await fetch("https://api.venice.ai/api/v1/image/edit", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "qwen-edit",
    prompt: "Remove the tourist crowd from the square and keep the architecture intact",
    image: imageBase64,
  }),
});

const editedImage = Buffer.from(await response.arrayBuffer());
fs.writeFileSync("edited.png", editedImage);
```

```bash cURL
curl https://api.venice.ai/api/v1/image/edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o edited.png \
  -d '{
    "model": "qwen-edit",
    "prompt": "Colorize this black and white portrait naturally",
    "image": "https://example.com/portrait-bw.jpg"
  }'
```

## 第 2 步：使用 multi-edit 进行蒙版或分层内绘

`/image/multi-edit` 最多接受三张图像。第一张是基础图像。其余图像作为编辑层或蒙版，比纯 prompt 编辑提供更多控制。

当你希望以下情形时这是更好的选择：

- 用蒙版定位特定区域
- 把现有构图与叠加层结合
- 比单图 prompt 更严格地约束编辑

**JSON 请求：**
```json
{
  "modelId": "qwen-edit",
  "prompt": "Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective",
  "images": [
    "https://example.com/street-scene.png",
    "https://example.com/billboard-mask.png"
  ]
}
```

**Multipart 请求：**
```bash
curl https://api.venice.ai/api/v1/image/multi-edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "modelId=qwen-edit" \
  -F "prompt=Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective" \
  -F "images=@street-scene.png" \
  -F "images=@billboard-mask.png" \
  -o multi-edited.png
```

与 `/image/edit` 相同，响应体是原始 `image/png` 数据。

<Note>
`/image/multi-edit` 当前在请求 schema 中使用 `modelId` 字段而非 `model`。
</Note>

## 内绘技巧

基于 prompt 的内绘在指令简短且局部时效果最佳：

- `remove the tree`
- `change the sky to sunset`
- `replace the logo with a blank sign`
- `restore the torn corner of the photo`

对于更广泛的场景变化，请描述应保持不变的内容：

```text
Replace the background with a modern photo studio backdrop while preserving the subject pose, facial features, and clothing.
```

如果编辑持续影响错误区域，从 `/image/edit` 切换到 `/image/multi-edit`，并提供蒙版或叠加层。

## 第 3 步：去除背景

当你想要将前景主体隔离到透明背景时使用 `/image/background-remove`。该 endpoint 返回带 alpha 透明度的 PNG。

**使用图像 URL：**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o cutout.png \
  -d '{
    "image_url": "https://example.com/product-photo.jpg"
  }'
```

**使用本地文件上传：**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "image=@product-photo.jpg" \
  -o cutout.png
```

背景去除适用于：

- 电商产品照片
- 头像和肖像
- 准备放置于新背景之上的素材

## 请求参数

### `/image/edit`

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|---|---|---|---|---|
| `image` | 文件、base64 字符串或 URL | 是 | - | 要编辑的源图像 |
| `prompt` | string | 是 | - | 编辑的文本指令 |
| `model` | string | 否 | `qwen-edit` | 编辑模型 ID |
| `aspect_ratio` | string | 否 | 模型默认值 | 支持的模型的输出比例 |
| `modelId` | string | 已弃用 | - | `model` 的已弃用别名 |

### `/image/multi-edit`

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|---|---|---|---|---|
| `images` | 1-3 个文件、base64 字符串或 URL 数组 | 是 | - | 第一张为基础图像；其余为编辑层或蒙版 |
| `prompt` | string | 是 | - | 描述如何组合或编辑各层的文本指令 |
| `modelId` | string | 否 | `qwen-edit` | 编辑模型 ID |

### `/image/background-remove`

| 参数 | 类型 | 必需 | 说明 |
|---|---|---|---|
| `image` | 文件或 base64 字符串 | `image` 或 `image_url` 之一 | 要抠出的源图像 |
| `image_url` | string | `image` 或 `image_url` 之一 | 要抠出的公开图像 URL |

---

## 支持的输入格式

| Endpoint | JSON 输入 | Multipart 输入 | 输出 |
|---|---|---|---|
| `/image/edit` | base64 字符串或 URL | 文件上传 | `image/png` |
| `/image/multi-edit` | base64 字符串或 URL | 文件上传 | `image/png` |
| `/image/background-remove` | base64 字符串或 URL | 文件上传 | `image/png` |

对于编辑 endpoints，图像尺寸必须至少为 `65536` 像素且不超过 `33177600` 像素。上传的文件必须小于 `25MB`。

## 模型与定价

默认的编辑模型为 `qwen-edit`，定价为**每次编辑 $0.04**。其他具备编辑能力的模型可能有不同的定价与约束。

参见：

- [图像定价](/zh/overview/pricing)

## 错误

| 状态 | 含义 | 操作 |
|---|---|---|
| `400` | 请求参数无效 | 检查图像数量、字段名和输入格式 |
| `401` | 认证失败 | 检查你的 API key |
| `402` | 余额不足 | 在 [venice.ai/settings/api](https://venice.ai/settings/api) 充值 |
| `415` | 内容类型无效 | 正确使用 JSON 或 multipart form-data |
| `429` | 超出速率限制或模型过载 | 使用退避策略重试；查看 `Retry-After` 响应头 |
| `500` | 推理处理失败 | 重试请求 |
| `503` | 模型已达容量上限 | 短暂延迟后重试 |

<Note>
某些编辑模型的内容策略比图像生成模型更严格。例如，`qwen-edit` 会阻止涉及露骨色情图像、性化未成年人或现实暴力的请求。
</Note>

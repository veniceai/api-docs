Venice Crypto RPC 是一个用于读取区块链状态和转发已签名交易的 HTTP 代理。它使用与其他 Venice 付费端点相同的 Venice API 密钥或 x402 钱包认证，因此你的应用无需再向 RPC 提供商单独注册账户。

Venice 从不持有你的私钥。请在你的应用中签名交易，并通过 `eth_sendRawTransaction` 提交已签名的载荷。

## 发现支持的网络

获取当前的网络标识列表：

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/networks
```

该发现端点是公开的，无需认证。其响应具有权威性；使用返回的标识之一作为 RPC 端点中的 `{network}`。

## 发送 RPC 请求

向所选网络发送 JSON-RPC 2.0 请求：

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/ethereum-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_blockNumber",
    "params": [],
    "id": 1
  }'
```

该代理支持单个请求以及最多 100 个请求的批量调用。支持的方法、定价层级和可用性各不相同；在依赖某个方法之前，请查阅 [Crypto RPC API 参考](/api-reference/endpoint/crypto/rpc)。

## 读取与写入工作流

### 读取链上状态

使用 `eth_blockNumber`、`eth_getBalance`、`eth_call` 和 `eth_getLogs` 等方法查询公开的链上数据。当相关调用面向同一网络时，请进行批量处理。

### 转发交易

<Steps>
  <Step title="构建交易">
    从目标网络读取账户 nonce、费用数据和 gas 估算值。
  </Step>
  <Step title="本地签名">
    使用应用的钱包库进行签名。切勿将私钥或助记词发送给 Venice。
  </Step>
  <Step title="提交已签名的载荷">
    调用 `eth_sendRawTransaction` 并设置 `Idempotency-Key` 请求头，以确保重试不会产生第二次计费的转发。
  </Step>
  <Step title="检查交易回执">
    轮询 `eth_getTransactionReceipt`，直到交易被打包，然后检查其状态。
  </Step>
</Steps>

## 安全重试

对可能被重试的请求（尤其是交易转发）设置幂等键：

```bash
-H "Idempotency-Key: transaction-123"
```

Venice 会将响应缓存 24 小时。使用相同的请求体重复使用该键将返回缓存的响应而不会再次计费；使用不同的请求体重复使用该键将返回错误。

## 重要限制

- 该代理仅支持 HTTP，不支持 WebSocket 订阅。
- 不支持有状态的过滤器方法；请使用无状态方法，如 `eth_getLogs`。
- Venice 不持有密钥，也不暴露签名方法。
- 允许列表之外的方法将返回错误。
- 检查 `X-Venice-RPC-Credits` 和 `X-Venice-RPC-Cost-USD` 以跟踪请求成本。

部分模型会在回答之前进行"出声思考"。它们逐步推导问题，然后给出最终答案。这让它们在数学、代码以及逻辑密集任务上表现更强。

<div id="reasoning-models-placeholder"></div>

完整模型列表、定价与上下文上限请见[模型页面](/zh/overview/models)。并非所有推理模型都支持 [`reasoning_effort`](#reasoning-effort) 参数。详情见[模型支持](#model-support)。

## 读取输出

推理模型会将思考过程放在单独的 `reasoning_content` 字段中，从而保持 `content` 干净：

<CodeGroup>
```python Python
response = client.chat.completions.create(
    model="zai-org-glm-5-1",
    messages=[{"role": "user", "content": "What is 15% of 240?"}]
)

thinking = response.choices[0].message.reasoning_content
answer = response.choices[0].message.content
```

```javascript Node.js
const response = await client.chat.completions.create({
    model: "zai-org-glm-5-1",
    messages: [{ role: "user", content: "What is 15% of 240?" }]
});

const thinking = response.choices[0].message.reasoning_content;
const answer = response.choices[0].message.content;
```

```bash cURL
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "zai-org-glm-5-1",
    "messages": [{"role": "user", "content": "What is 15% of 240?"}]
  }'
```
</CodeGroup>

<Info>
部分提供商（Anthropic、Google、OpenAI、Qwen）会返回加密或摘要后的推理 token。此时 `reasoning_content` 会包含 `"[Some reasoning content is encrypted]"` 占位符。
</Info>

### 流式

在流式响应中，`reasoning_content` 会先于最终答案出现在 delta 中：

<CodeGroup>
```python Python
stream = client.chat.completions.create(
    model="zai-org-glm-5-1",
    messages=[{"role": "user", "content": "Explain photosynthesis"}],
    stream=True
)

for chunk in stream:
    if chunk.choices:
        delta = chunk.choices[0].delta
        if delta.reasoning_content:
            print(delta.reasoning_content, end="")
        if delta.content:
            print(delta.content, end="")
```

```javascript Node.js
const stream = await client.chat.completions.create({
    model: "zai-org-glm-5-1",
    messages: [{ role: "user", content: "Explain photosynthesis" }],
    stream: true
});

for await (const chunk of stream) {
    if (chunk.choices?.[0]?.delta) {
        const delta = chunk.choices[0].delta;
        if (delta.reasoning_content) process.stdout.write(delta.reasoning_content);
        if (delta.content) process.stdout.write(delta.content);
    }
}
```

```bash cURL
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "zai-org-glm-5-1",
    "messages": [{"role": "user", "content": "Explain photosynthesis"}],
    "stream": true
  }'
```
</CodeGroup>

## 推理力度

`reasoning_effort` 参数控制模型在响应前进行多少思考。力度越高意味着推理越深，但 token 与延迟也越多。

### 可选值

| 值 | 说明 |
|-------|-------------|
| `none` | 完全禁用推理 |
| `minimal` | 最小力度的基本推理 |
| `low` | 适用于简单问题的轻量推理 |
| `medium` | 适合中等复杂度的均衡推理 |
| `high` | 适用于复杂问题的深度推理 |
| `xhigh` | 额外提升的推理深度 |
| `max` | 最大推理能力 |

<Warning>
并非所有模型都支持所有值。Venice **不会**自动映射到最接近的支持级别。不受支持的值会从上游提供商返回 400 错误。例如，将 `xhigh` 发给 Claude，或将 `max` 发给 GPT-5.2，都会失败。

不确定时使用 `low`、`medium` 或 `high`。这些是受支持最广的值。
</Warning>

### 模型支持

#### OpenAI

| 模型 | 支持的值 |
|-------|-----------------|
| GPT-5.2 | `none`、`low`、`medium`、`high`、`xhigh` |
| GPT-5.2 Codex、GPT-5.3 Codex | `low`、`medium`、`high`、`xhigh` |

#### Anthropic

| 模型 | 支持的值 |
|-------|-----------------|
| Claude Opus 4.6、Opus 4.6 Fast | `low`、`medium`、`high`、`max` |
| Claude Opus 4.5、Sonnet 4.5、Sonnet 4.6 | `low`、`medium`、`high` |

#### Google

| 模型 | 支持的值 |
|-------|-----------------|
| Gemini 3 Pro Preview | `low`、`high` |
| Gemini 3.1 Pro Preview | `low`、`medium`、`high` |
| Gemini 3 Flash Preview | `minimal`、`low`、`medium`、`high` |

#### xAI

Grok 系列模型（Grok 4.1 Fast、Grok Code Fast）**不**支持 `reasoning_effort`。指定该参数会导致错误。

#### 其他模型

| 模型 | 支持的值 |
|-------|-----------------|
| Qwen 3 235B A22B Thinking、Qwen 3.5 35B A3B | `low`、`medium`、`high` |
| Kimi K2.5 | `low`、`medium`、`high` |
| MiniMax M2.5、M2.1 | `low`、`medium`、`high` |
| GLM 5.1 系列 | 仅内置推理，不可配置 |
| DeepSeek R1 | 仅内置推理，不可配置 |

### 用法

`reasoning_effort` 可作为顶层参数传入，或使用嵌套的 `reasoning.effort` 格式：

<CodeGroup>
```python Python
response = client.chat.completions.create(
    model="minimax-m25",
    messages=[{"role": "user", "content": "Prove that there are infinitely many primes"}],
    extra_body={"reasoning": {"effort": "high"}}
)
```

```javascript Node.js
const response = await client.chat.completions.create({
    model: "minimax-m25",
    messages: [{ role: "user", content: "Prove that there are infinitely many primes" }],
    reasoning: { effort: "high" }
});
```

```bash cURL
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "minimax-m25",
    "messages": [{"role": "user", "content": "Prove that there are infinitely many primes"}],
    "reasoning": {"effort": "high"}
  }'
```
</CodeGroup>

也接受扁平格式 `"reasoning_effort": "high"`。

## 禁用推理

禁用推理有两种方式：

| 方法 | 语法 | 工作方式 |
|--------|--------|-------------|
| `reasoning.enabled: false` | `"reasoning": {"enabled": false}` | Venice 级别的开关，阻止推理参数被发送给提供商。**推荐使用。** |
| `reasoning.effort: "none"` | `"reasoning": {"effort": "none"}` | 传给提供商，由其决定如何处理。仅部分模型支持（例如 GPT-5.x）。 |

对于支持的模型，`reasoning.enabled: false` 更可靠：

| 模型 | 是否可禁用？ |
|-------|-------------|
| GPT-5.2 | 是 |
| GPT-5.2 Codex、GPT-5.3 Codex | 是（但不支持 `none` effort） |
| Qwen 3 235B A22B Thinking、Qwen 3.5 35B A3B | 是 |
| GLM 5.1 系列 | 是 |
| Claude Opus 4.5/4.6/4.6 Fast、Sonnet 4.5/4.6 | 否（始终推理） |
| Gemini 3 Pro、3.1 Pro、3 Flash | 否（始终推理） |
| DeepSeek R1 | 否（始终推理） |

<CodeGroup>
```python Python
response = client.chat.completions.create(
    model="openai-gpt-52",
    messages=[{"role": "user", "content": "What's the capital of France?"}],
    extra_body={"reasoning": {"enabled": False}}
)
```

```javascript Node.js
const response = await client.chat.completions.create({
    model: "openai-gpt-52",
    messages: [{ role: "user", content: "What's the capital of France?" }],
    reasoning: { enabled: false }
});
```

```bash cURL
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai-gpt-52",
    "messages": [{"role": "user", "content": "What is the capital of France?"}],
    "reasoning": {"enabled": false}
  }'
```
</CodeGroup>

## Token 上限

推理模型会生成可见的回答 token（在 `content` 中）和推理 token（在 `reasoning_content` 中）。两者都计入你的 token 预算。

### 设置 token 上限

使用 `max_completion_tokens` 限制模型生成的 token 总数（包括推理）：

```json
{
  "model": "deepseek-v4-flash",
  "messages": [...],
  "max_completion_tokens": 500
}
```

也接受 `max_tokens`，行为相同。如果同时设置，`max_completion_tokens` 优先。

要获得更多可见输出，可以提高上限、降低 `reasoning_effort`，或[禁用推理](#disabling-reasoning)。

### 读取细分

`usage` 对象展示了预算如何被使用：

```json
"usage": {
  "completion_tokens": 501,
  "completion_tokens_details": { "reasoning_tokens": 169 },
  "prompt_tokens": 13,
  "total_tokens": 514
}
```

在此示例中，169 个 token 用于推理，332 个用于可见回答。达到上限时，`finish_reason` 为 `length`。

每个模型的上限以 `maxCompletionTokens` 字段在 [`/v1/models`](/zh/api-reference/endpoint/models/list) 端点上提供。

### 非推理模型

在非推理模型上，`max_tokens` 和 `max_completion_tokens` 行为相同，直接限制可见输出。

## 能力发现

可通过 [`/v1/models`](/zh/api-reference/endpoint/models/list) 端点查看模型支持的能力：

| 字段 | 含义 |
|-------|---------|
| `supportsReasoning` | 模型具备推理（思维链）能力 |
| `supportsReasoningEffort` | 模型接受 `reasoning_effort` / `reasoning.effort` 参数 |

## 最佳实践

- 一般用途默认使用 `medium`
- 复杂任务（数学、代码、分析）使用 `high` 或 `xhigh`
- 对延迟敏感的应用使用 `low`
- 使用 `reasoning.enabled: false` 或将 effort 设为 `none` 来禁用推理
- 不确定时使用 `low`、`medium` 或 `high`。这些是受支持最广的值

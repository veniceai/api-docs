[Venice Skills](https://github.com/veniceai/skills) 是 Venice API 的官方 [Agent Skills](https://www.anthropic.com/news/agent-skills) 集合。每个 skill 都是一个独立的文件夹，包含一个 `SKILL.md`，LLM agent 按需加载它即可正确地针对 API 的某个具体面进行操作。

## 为什么需要 skills？

没有 skills，你的 agent 必须以痛苦的方式发现 Venice 的种种细节：`venice_parameters`、模型类型枚举、402 payment-required 流程、视频 queue/retrieve 生命周期、character slug 等等。Skills 把这些知识打包成按需加载的、专注的文件，让 agent 只加载当前任务所需的内容。

每个 `SKILL.md` 包含：

- 它所覆盖的 endpoint
- 所需的响应头、参数和响应结构
- 一个 curl 示例和一个最小化的 SDK 示例
- 一个"易踩坑"小节，列出真实集成者常遇到的问题

## Skill 目录

| Skill | 覆盖范围 |
|---|---|
| `venice-api-overview` | 基础 URL、认证模式、响应头、定价模型、版本控制 |
| `venice-auth` | Bearer API key + Sign-In-With-X / x402 钱包认证 |
| `venice-chat` | `/chat/completions`，含 `venice_parameters`、多模态、tools、推理、streaming |
| `venice-responses` | `/responses`，OpenAI 兼容的 Responses API（Alpha） |
| `venice-embeddings` | `/embeddings` 模型、编码格式、维度 |
| `venice-image-generate` | `/image/generate`、`/images/generations`、`/image/styles` |
| `venice-image-edit` | `/image/edit`、`/image/multi-edit`、`/image/upscale`、`/image/background-remove` |
| `venice-audio-speech` | `/audio/speech` 的 TTS 模型、音色、格式、streaming |
| `venice-audio-music` | `/audio/quote`、`/audio/queue`、`/audio/retrieve`、`/audio/complete` |
| `venice-audio-transcription` | 使用 Whisper、Parakeet、Scribe、Wizper、xAI STT 的 `/audio/transcriptions` |
| `venice-video` | `/video/*` 生成与转录 |
| `venice-models` | `/models`、`/models/traits`、`/models/compatibility_mapping` |
| `venice-characters` | `/characters*` + `venice_parameters.character_slug` |
| `venice-api-keys` | `/api_keys` 的 CRUD、速率限制、Web3 key 生成 |
| `venice-billing` | `/billing/balance`、`/billing/usage`、`/billing/usage-analytics` |
| `venice-x402` | `/x402/*` 钱包积分，Base 或 Solana 上的 USDC |
| `venice-crypto-rpc` | `/crypto/rpc/*` JSON-RPC 代理，1×/2×/4× 定价 |
| `venice-augment` | `/augment/text-parser`、`/augment/scrape`、`/augment/search` |
| `venice-errors` | 错误结构、402 payment required、422 内容策略、429 速率限制、重试策略 |

## 安装

每个 skill 都只是一个带 `SKILL.md` 的文件夹，文件以 YAML frontmatter 开头：

```yaml
---
name: venice-chat
description: When the agent should load this skill and what's in it
---
```

把 `skills/` 文件夹（或其中一部分）放到你的 runtime 监听的路径下。

<Tabs>
  <Tab title="Claude Code">
    项目本地：

    ```bash
    git clone https://github.com/veniceai/skills.git
    cp -r skills/skills/* .claude/skills/
    ```

    或全局，适用于你机器上的每个项目：

    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.claude/skills/venice
    ```
  </Tab>

  <Tab title="Cursor">
    项目本地：

    ```bash
    git clone https://github.com/veniceai/skills.git .cursor/skills-venice
    ```

    或单独复制 skills：

    ```bash
    cp -r skills/venice-chat .cursor/skills/
    ```
  </Tab>

  <Tab title="Codex">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.codex/skills/venice
    ```

    对于项目本地安装，请改为 `.codex/skills/`。
  </Tab>

  <Tab title="OpenCode">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.config/opencode/skills/venice
    ```

    OpenCode 也会读取项目根目录下的 `.opencode/skills/`、`.claude/skills/` 和 `.agents/skills/`。
  </Tab>

  <Tab title="Hermes Agent">
    Hermes 自带 Venice skills 的安装器：

    ```bash
    hermes skills install veniceai/skills
    ```

    或直接 symlink：

    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.hermes/skills/venice
    ```
  </Tab>

  <Tab title="Cline">
    ```bash
    git clone https://github.com/veniceai/skills.git .clinerules/skills-venice
    ```
  </Tab>
</Tabs>

### 路径参考

| Runtime | 项目本地 | 全局 |
|---|---|---|
| Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| Codex | `.codex/skills/` | `~/.codex/skills/`（或 `$CODEX_HOME/skills/`） |
| OpenCode | `.opencode/skills/`（也含 `.claude/skills/`、`.agents/skills/`） | `~/.config/opencode/skills/` |
| Hermes Agent | `$HERMES_OPTIONAL_SKILLS_DIR` | `~/.hermes/skills/` |
| Cursor | `.cursor/skills/` | `~/.cursor/skills/` |
| Cline | `.clinerules/skills/` | 无 |
| 其他 runtime | `.agents/skills/`（约定） | `~/.agents/skills/` |

<Tip>
定义了额外 frontmatter 字段（`version`、`platforms`、`metadata.*`、`compatibility` 等）的 runtime 按规范必须忽略未知字段，因此同一个 skill 文件可在所有地方使用而无需分叉。
</Tip>

### 作为 git 子模块

如果你想在自己的仓库中固定版本：

```bash
git submodule add https://github.com/veniceai/skills.git vendor/venice-skills
```

然后把你需要的子集 symlink 或复制到 agent 的 skill 路径中。

## Agent 如何加载它们

Agent 通过 `SKILL.md` 的 frontmatter `name` 和 `description` 来发现每个 skill。当用户提问匹配某个 skill 的用途时，agent 仅把那一个文件加载进上下文（而非整个目录），因此 prompt 保持精简，答复保持准确。

例如，需要生成音乐的 agent 会加载 `venice-audio-music`，并立刻知道：

- 音乐走的是 queue/retrieve/complete 生命周期，而非同步 endpoint
- 有哪些可用模型以及它们的每分钟定价
- 如何先调用 `/audio/quote` 估算成本
- 轮询退避应该是什么样

没有这个 skill，agent 可能会尝试调用 `/audio/speech` 来生成音乐，得到无用的响应。

## 编写新 skill

1. 把 `template/` 复制到 `skills/<your-skill>/`。
2. 填写 frontmatter 和正文。`description` 要具体，因为 agent 就靠它来决定何时加载该 skill。
3. 在底部链接相关 skills 以便交叉导航。
4. 向 [`veniceai/skills`](https://github.com/veniceai/skills) 提交 PR。

样式约定见仓库的 `CONTRIBUTING.md`（简短的首段、明确的 endpoint 表格、curl + 一个 SDK 示例、"易踩坑"小节、≤ 500 行）。

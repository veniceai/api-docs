[NanoClaw](https://github.com/qwibitai/nanoclaw) 是一款轻量、自托管的 AI 助手，可运行在 WhatsApp 和 Telegram 上。本 fork 增加了 Venice AI 支持，因此一切都可以在没有 Anthropic 订阅的情况下私密运行。

## 为什么选 Venice AI？

[Venice](https://venice.ai) 是一个隐私优先的 AI 平台。他们[不在自己的服务器上存储或记录任何 prompt 或响应](https://venice.ai/privacy)——你的对话只存在于你的设备上。请求通过他们的代理端到端加密发送到去中心化 GPU 提供商，零数据留存。这意味着你的 AI 助手对话保持私密，即便对 Venice 自身也是如此。

Venice 通过单一 API 提供匿名访问前沿模型（Claude Opus、Claude Sonnet）和完全私密访问开源模型（GLM、Qwen）——你可随时切换。

| | **Venice AI** | **传统 AI 服务商** |
|---|---|---|
| **数据留存** | 无——零日志 | 是 |
| **Prompt 隐私** | 加密、不存储 | 存储在服务商服务器上 |
| **开源模型** | 是（GLM、Qwen 等） | 否 |
| **前沿模型** | Claude、GPT 等——匿名 | 仅通过直接订阅 |
| **定价** | 按 token 付费，无订阅。或质押 [DIEM](https://venice.ai/lp/diem) 获得每日刷新的积分 | $20–200/月订阅，或按 token 付费 API |
| **无审查推理** | 是（开源模型） | 否 |

## 为什么选 NanoClaw？

NanoClaw 是相对 OpenClaw 等更大型平台的一个干净、极简的替代方案。它专为一个人运行一个 bot 而设计。

| | **NanoClaw (Venice)** | **OpenClaw** |
|---|---|---|
| **代码库** | ~2,000 行代码，几个文件 | ~500,000 行代码，53 个配置文件 |
| **依赖** | ~15 个包 | 70+ 个包 |
| **安全模型** | OS 级 Docker 容器隔离 | 应用级的 allowlist 和配对码 |
| **每群隔离** | 每个群拥有独立的容器、文件系统和内存 | 共享进程，共享内存 |
| **安装** | 一个向导（`/setup`），约 10 分钟 | 手动多步配置 |
| **AI 提供方** | Venice AI（私密，无订阅） | Anthropic（需要 API key 或订阅） |
| **可定制性** | 直接修改代码——它小到可读完 | 配置文件和插件 |
| **目标用户** | 一个人，一个 bot | 多用户平台 |

## 你将获得什么

- **Telegram** 和/或 **WhatsApp** 上的个人 AI 助手
- 由 **Venice AI** 驱动——无需 Anthropic 账户
- bot 运行在**隔离的 Docker 容器**中（沙箱化，无法访问你的系统）
- **模型切换**——随时告诉 bot "switch to zai-org-glm-5" 或 "use opus"
- **定时任务**——设置提醒、循环任务
- 内置 **Web 搜索与浏览**
- Telegram 消息中的 **Markdown 格式**

**对于 Telegram**（推荐首次用户使用）：
1. 打开 Telegram 并搜索 **@BotFather**
2. 发送 `/newbot` 并按提示操作
3. 保存 BotFather 给你的 token（形如 `123456789:ABCdef...`）

<Warning>
**对于 WhatsApp——请使用虚拟号码，而不是你的个人号码：**

NanoClaw 以你 WhatsApp 号码上一个关联设备的身份连入。这意味着 **agent 可以看到所有进出消息**——你所有的个人对话、群聊、照片，全部。你的手机仍然正常工作，但 bot 对你整个 WhatsApp 账户拥有完全的可见性。

**请改用虚拟手机号码。** 这些 App 可以给你一个第二号码，可专门用于该 bot：

| App | 价格 | 备注 |
|-----|-------|-------|
| [Hushed](https://hushed.com) | ~$5/月 | 可靠，WhatsApp 验证效果好 |
| [Burner](https://www.burnerapp.com) | ~$5/月 | 与 Hushed 类似，一次性号码 |
| [Google Voice](https://voice.google.com) | 免费 | 仅美国，部分情况下不能用于 WhatsApp 验证 |

**如何设置：**
1. 从上述 App 获取一个虚拟号码
2. 在第二台设备（旧手机、平板或模拟器）上用该虚拟号码安装 WhatsApp
3. 在 NanoClaw 安装期间，用那台第二设备扫描二维码——而非你的个人手机
</Warning>

## 安装

安装大约需要 10 分钟。你只需要**一个 Terminal 窗口**。

<Steps>
  <Step title="Clone 并安装">
    打开 Terminal 并运行：

    ```bash
    git clone https://github.com/lorenzovenice/nanoclaw-venice.git
    cd nanoclaw-venice
    npm install
    ```

    等待 `npm install` 完成且无错误。
  </Step>

  <Step title="用 Venice 启动 Claude Code">
    把 `your-key` 替换为你的 Venice API key 后运行：

    ```bash
    VENICE_API_KEY=your-key npm run venice
    ```

    这条命令一次性启动 Venice 代理并通过它启动 Claude Code。

    <Note>
    Claude Code 默认使用 **GLM 5**（`zai-org-glm-5`）以降低安装阶段的费用。安装完成后，在 Claude Code 中输入 `/model` 切换到 `claude-sonnet-4-6` 或 `claude-opus-4-6` 以获得最佳性能。
    </Note>

    如果提示 "Do you want to use this API key?"——选择 **Yes**。
  </Step>

  <Step title="运行安装向导">
    在 Claude Code 的 terminal 中，输入：

    ```
    /setup
    ```

    向导会引导你完成：

    1. **Bootstrap**——检查 Node.js 和依赖
    2. **Venice API key**——验证并保存你的 key
    3. **频道选择**——选择 WhatsApp、Telegram 或两者
    4. **容器构建**——构建 Docker 容器（首次构建需几分钟）
    5. **WhatsApp 认证**——用你的手机扫码（如适用）
    6. **Telegram 设置**——向你的 bot 发一条消息以便它识别你的聊天
    7. **触发词**——激活 bot 的前缀（默认：`@Andy`）
    8. **挂载目录**——现在选 "No"（之后可以加上文件访问）
    9. **启动服务**——NanoClaw 与 Venice 代理都作为后台服务启动

    安装向导会安装两个后台服务：
    - **NanoClaw**——bot 本身
    - **Venice 代理**——一个本地小型服务器（localhost:4001），在 Claude Code 与 Venice AI 间做格式转换

    两者都会在开机时自动启动，并在崩溃时自动重启。

    <Note>
    如果向导卡在某一步之间，输入 "continue" 或 "next step" 推动它前进。
    </Note>
  </Step>

  <Step title="开始聊天">
    安装完成后，打开你的聊天（Telegram 或 WhatsApp）并发送：

    ```
    @Andy hello, are you there?
    ```

    bot 应该在几秒内响应。在你的主频道里，无需 `@Andy` 前缀也可正常输入。

    **你现在可以关闭 terminal 窗口。** 一切都作为后台服务运行，并在你的电脑启动时自动启动。
  </Step>
</Steps>

## 工作原理

NanoClaw 分为两层：

| 层 | 作用 |
|-------|-------------|
| **Claude Code CLI** | 用于安装、调试和定制的管理工具 |
| **Bot** | 聊天中的 AI，运行在隔离 Docker 容器中 |

随时打开 Claude Code：

```bash
cd nanoclaw-venice
ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
```

用它来运行 `/setup`、`/debug`、`/customize`，或修改 bot 的行为。

## 故障排查

<AccordionGroup>
  <Accordion title="代理没在运行">
    Venice 代理作为后台服务运行并会自动重启。如果它不工作：

    **macOS：**
    ```bash
    # 检查是否在运行
    launchctl list | grep venice-proxy

    # 重启
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # 查看日志
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```

    **Linux：**
    ```bash
    # 检查是否在运行
    systemctl --user status nanoclaw-venice-proxy

    # 重启
    systemctl --user restart nanoclaw-venice-proxy

    # 查看日志
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```
  </Accordion>

  <Accordion title="Claude Code 显示 403 错误或 'Please run /login'">
    这意味着 Claude Code 无法连接到 Venice 代理。

    1. **检查代理是否在运行。** 见上一条故障排查。
    2. **确保你在正确的目录。** 始终先 `cd nanoclaw-venice`。
    3. **从头开始：** 关闭所有 terminal 并运行：
       ```bash
       cd nanoclaw-venice
       ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
       ```
  </Accordion>

  <Accordion title="模型错误（'model does not exist'）">
    重启代理和 bot：

    **macOS：**
    ```bash
    # 重启代理
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # 重启 bot
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw
    ```

    **Linux：**
    ```bash
    # 重启代理
    systemctl --user restart nanoclaw-venice-proxy

    # 重启 bot
    systemctl --user restart nanoclaw
    ```

    在 [模型目录](/zh/models/text) 中查看可用模型。
  </Accordion>

  <Accordion title="Bot 不回复消息">
    按顺序排查：

    1. **检查触发词。** 确保你用的是正确的前缀（例如 `@Andy hello`）。
    2. **检查 Docker 是否在运行。** 运行 `docker info`——若报错，请打开 Docker Desktop。
    3. **检查代理是否在运行。** 见上文 "代理没在运行"。
    4. **查看日志：** 在项目目录运行 `tail -f logs/nanoclaw.log`。
    5. **查看容器日志。** 打开 `nanoclaw-venice/groups/main/logs/` 文件夹，打开最新的以 `container-` 开头的文件。
    6. **重启一切：** 同时重启代理和 bot（见上文）。
  </Accordion>

  <Accordion title="安装期间容器构建失败">
    确保 Docker Desktop 已打开并正在运行。等待 10 秒让 Docker 完全启动，然后在向导中输入 `continue` 重试。
  </Accordion>

  <Accordion title="WhatsApp 已断开连接">
    你的 WhatsApp 会话可能会过期。要重新连接：

    ```bash
    cd nanoclaw-venice
    npm run auth
    ```

    用 WhatsApp 扫描二维码（设置 → 关联设备 → 关联设备），然后重启 bot：
    - macOS：`launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
    - Linux：`systemctl --user restart nanoclaw`
  </Accordion>
</AccordionGroup>

## 进阶

<AccordionGroup>
  <Accordion title="让 bot 访问你电脑上的文件">
    默认情况下，bot 与你的电脑完全隔离——它只能看到自己的记忆和对话历史。

    - **安装期间：** 当询问目录访问时，选择 "Yes"
    - **安装后：** 在 Claude Code 中运行 `/customize`
  </Accordion>

  <Accordion title="手动启动/停止 bot">
    NanoClaw 运行两个后台服务，它们在开机时自动启动。

    **macOS：**
    | 操作 | 命令 |
    |--------|---------|
    | 启动 bot | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | 停止 bot | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | 重启 bot | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw` |
    | 启动代理 | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | 停止代理 | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | 重启代理 | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy` |

    **Linux：**
    | 操作 | 命令 |
    |--------|---------|
    | 启动 bot | `systemctl --user start nanoclaw` |
    | 停止 bot | `systemctl --user stop nanoclaw` |
    | 重启 bot | `systemctl --user restart nanoclaw` |
    | 启动代理 | `systemctl --user start nanoclaw-venice-proxy` |
    | 停止代理 | `systemctl --user stop nanoclaw-venice-proxy` |
    | 重启代理 | `systemctl --user restart nanoclaw-venice-proxy` |
  </Accordion>

  <Accordion title="只用 Claude Code 通过 Venice（不用 bot）">
    如果你只想让 Claude Code 走 Venice，不需要 WhatsApp/Telegram，代理服务需要在运行。如果你已经运行过 `/setup`，它已作为后台服务在跑。

    ```bash
    cd nanoclaw-venice
    ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
    ```

    **提示：** 把以下添加到你的 `~/.zshrc`（或 `~/.bashrc`），让任意 terminal 都能快速切到 Venice：

    ```bash
    alias venice='export ANTHROPIC_BASE_URL=http://localhost:4001 && export ANTHROPIC_API_KEY=venice-proxy && echo "Using Venice API"'
    alias anthropic='unset ANTHROPIC_BASE_URL && unset ANTHROPIC_API_KEY && echo "Using Anthropic API"'
    ```

    然后在任意 terminal 中先输入 `venice` 再运行 `claude` 即可使用 Venice，或者 `anthropic` 切回去。
  </Accordion>

  <Accordion title="运行多个 bot">
    你可以在同一台机器上运行多个 NanoClaw bot（例如一个个人用、一个团队用）。只需把仓库 clone 到不同目录并再次运行安装。注意：它们共享同一个 Docker 镜像，所以重建其中一个会影响所有 bot。
  </Accordion>

  <Accordion title="开发者命令">
    适用于想修改 NanoClaw 代码的人：

    ```bash
    npm run dev          # 启动代理 + NanoClaw 并支持热重载
    npm run proxy        # 只启动 Venice 代理
    npm run build        # 编译 TypeScript
    npm test             # 跑测试
    ./container/build.sh # 重建 agent 容器
    ```
  </Accordion>
</AccordionGroup>

## 架构

```
你 (WhatsApp/Telegram)
        ↓
   NanoClaw (Node.js)
        ↓
   Docker 容器 (隔离沙箱)
        ↓
   Venice 代理 (localhost:4001)
        ↓
   api.venice.ai (私密推理)
```

| 文件 | 作用 |
|------|---------|
| `proxy/venice-proxy.ts` | 把 Anthropic 格式翻译为 OpenAI 格式给 Venice |
| `src/index.ts` | 主编排器——消息循环、agent 调用 |
| `src/channels/whatsapp.ts` | 通过 baileys 连接 WhatsApp |
| `src/channels/telegram.ts` | 通过 grammy 连接 Telegram bot |
| `src/container-runner.ts` | 启动隔离的 agent 容器 |

## FAQ

<AccordionGroup>
  <Accordion title="为什么需要代理？">
    Claude Agent SDK 使用 Anthropic 的消息格式。Venice 使用 OpenAI 的格式。代理在两者间做转换，让一切无需修改 SDK 即可工作。
  </Accordion>

  <Accordion title="我能使用开源模型吗？">
    可以。Venice 托管许多模型。告诉 bot "switch to zai-org-glm-5" 或任何 Venice 模型 ID。参见 [模型目录](/zh/models/text)。
  </Accordion>

  <Accordion title="安全吗？">
    Agent 运行在带真实 OS 级隔离的 Docker 容器中。Venice API key 通过 stdin 传入，永不写入容器内的磁盘。每个群都有自己独立的隔离环境。
  </Accordion>

  <Accordion title="我需要 Anthropic 订阅吗？">
    不需要。一切都通过 Venice AI 运行。你只需要一个 Venice API key。
  </Accordion>

  <Accordion title="可以在服务器上使用吗？">
    可以。它可以在任何带 Docker 的 Linux 机器上运行。使用 systemd 服务即可开机自启。
  </Accordion>
</AccordionGroup>

本指南演示如何通过 Codex 的官方配置路径运行 OpenAI Codex CLI 并接入 Venice：`~/.codex/config.toml`（用户级）或 `.codex/config.toml`（项目级）。

## 前置条件

- 在 [venice.ai/settings/api](https://venice.ai/settings/api) 获取 Venice API 密钥
- 已在本机安装并能正常运行 Codex CLI

---

## 设置

<Steps>
  <Step title="创建项目配置路径">
    在项目根目录下执行：

    ```bash
    mkdir -p .codex
    ```
  </Step>

  <Step title="创建 .codex/config.toml">
    创建该文件并粘贴下面的配置：

    ```toml
    #:schema https://developers.openai.com/codex/config-schema.json

    model = "openai-gpt-54" # 任意 Venice 模型
    model_provider = "venice"
    model_reasoning_effort = "high"
    personality = "pragmatic"
    sandbox_mode = "workspace-write"

    [model_providers.venice]
    name = "Venice"
    base_url = "https://api.venice.ai/api/v1/"
    experimental_bearer_token = "YOUR VENICE API KEY"
    wire_api = "responses"
    ```
  </Step>

  <Step title="替换两个占位符">
    需要更新：

    - `model`：替换为你要使用的 Venice 模型 ID
    - `experimental_bearer_token`：替换为你真实的 Venice API 密钥

    可在[文本模型目录](/zh/models/text)中浏览可用模型 ID。
  </Step>

  <Step title="按正常方式运行 Codex CLI">
    在同一项目中启动 Codex CLI。它会加载 `.codex/config.toml`（对受信任的项目），并通过 Venice 路由请求。
  </Step>
</Steps>

---

## 官方 Codex 配置位置

- **用户默认**：`~/.codex/config.toml`
- **项目覆盖**：`.codex/config.toml`（仅对受信任项目加载）

如果你希望 Venice 设置在任何项目下都生效，可以把同样的配置放进 `~/.codex/config.toml`。

---

## 配置优先级（从高到低）

1. CLI 标志与 `--config` 覆盖
2. Profile 值（`--profile <name>`）
3. 项目级配置层（`.codex/config.toml`，目录越近优先级越高）
4. 用户配置（`~/.codex/config.toml`）
5. 系统配置（`/etc/codex/config.toml`，Unix）
6. 内置默认值

---

## 备注

- 妥善保管 API 密钥，切勿把真实密钥提交到 git。
- 当项目被标记为不受信任时，Codex 会忽略项目级 `.codex/` 配置。
- 切换模型时，只需更新 `model` 字段。
- 该提供者配置需要 `wire_api = "responses"`。

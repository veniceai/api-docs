X402 让你能够通过钱包认证并维持预付 USDC 余额来使用 Venice 的付费 API 路由。无需 API key 或账户。签名一条消息，在 Base 或 Solana 上充值，即可调用任意受支持的路由。

## 什么是 X402？

[X402](https://www.x402.org/) 是一个开放的支付标准，允许应用程序和 agent 通过加密货币以编程方式为服务付费。Venice 实现了 X402，让钱包可以直接在 Base 或 Solana 上用 USDC 进行认证并为推理付费。

## 先决条件

- 在 Base 或 Solana 上的钱包
- 用于所选链 gas 的原生代币，例如 Base 上的 ETH 或 Solana 上的 SOL
- 所选链上的 USDC（或来自关联 Venice 账户的现有 DIEM 支撑余额）

<Tip>
  考虑使用专用钱包来做自动化，而不是你的主金库钱包。
</Tip>

## 快速开始

[`venice-x402-client`](https://github.com/veniceai/x402-client) SDK 为钱包认证、充值和余额跟踪提供了辅助方法。

```bash
npm install venice-x402-client
```

```typescript
import { VeniceClient } from 'venice-x402-client'

const venice = new VeniceClient(process.env.WALLET_KEY)

await venice.topUp(10) // 如果钱包已经有可消费余额，可跳过

const response = await venice.chat({
  model: 'kimi-k2-5',
  messages: [{ role: 'user', content: 'Hello!' }]
})
```

客户端会为每次请求生成新的 `X-Sign-In-With-X` 响应头，并通过 `X-Balance-Remaining` 响应头自动跟踪余额。

### 配合 OpenAI 兼容工具

如果你使用的工具接受自定义 `fetch`，可使用 `createAuthFetch` 为任意请求添加钱包认证：

```typescript
import { createAuthFetch } from 'venice-x402-client'

const authFetch = createAuthFetch(process.env.WALLET_KEY)
```

### 可用辅助方法

SDK 为最常见的 Venice x402 路由提供了一等公民的辅助方法。对于未覆盖的内容，直接使用 `request()` 或 `createAuthFetch()`。

| 类别 | 方法 |
| --- | --- |
| Chat | `chat()`、`chatStream()` |
| Responses | `responses.create()`、`responses.stream()` |
| Images | `images.generate()`、`images.generations()`、`images.upscale()`、`images.edit()`、`images.multiEdit()`、`images.backgroundRemove()` |
| Audio | `audio.speech()`、`audio.transcribe()`、`audio.queue()`、`audio.retrieve()`、`audio.complete()` |
| Video | `video.queue()`、`video.retrieve()`、`video.generate()`、`video.complete()`、`video.transcribe()` |
| Embeddings | `embeddings()` |
| Models | `models()` |
| Wallet | `getBalance()`、`getTransactions()`、`topUp()` |

## 手动流程

如果你不使用 SDK，或想理解底层协议，下面是分步流程。对于新钱包，除非已有可消费的 DIEM 余额，否则可假定需要先充值。

### 第 1 步：构造 X-Sign-In-With-X 响应头

Venice 期望一个 Base64 编码的 JSON 负载，其中包含已签名的 Sign-In-With-X 消息。EVM 钱包在 Base 上签名 EIP-4361 SIWE 消息。Solana 钱包使用 Ed25519 签名 Solana SIWX 消息。请为每次请求流程生成新的 nonce 和时间戳。

对于 Solana，将 `chainId` 设为 `solana:5eykt4UsFv8P8NJdTREpY1vzqKqZKvdp`，在编码的 JSON 负载中加入 `type: "ed25519"`，并以 base58 或 base64 提供 Ed25519 签名。已签名的消息以 `<domain> wants you to sign in with your Solana account:` 开头，后接钱包地址以及标准的 `URI`、`Version`、`Chain ID`、`Nonce`、`Issued At` 和可选的 `Expiration Time` 字段。

```typescript
import { Wallet } from 'ethers'
import { SiweMessage, generateNonce } from 'siwe'

const wallet = new Wallet(process.env.EVM_PRIVATE_KEY)
const now = new Date()
const resourceUrl = 'https://api.venice.ai/api/v1/chat/completions'

const siwe = new SiweMessage({
  domain: 'api.venice.ai',
  address: wallet.address,
  statement: 'Sign in to Venice AI',
  uri: resourceUrl,
  version: '1',
  chainId: 8453,
  nonce: generateNonce(),
  issuedAt: now.toISOString(),
  expirationTime: new Date(now.getTime() + 5 * 60 * 1000).toISOString(),
})

const message = siwe.prepareMessage()
const signature = await wallet.signMessage(message)

const xSignInWithX = Buffer.from(
  JSON.stringify({
    address: wallet.address,
    message,
    signature,
    timestamp: now.getTime(),
    chainId: 8453,
  }),
  'utf8',
).toString('base64')

console.log(xSignInWithX)
```

### 第 2 步：检查余额

在发起付费请求前，先验证钱包是否有可消费余额：

```bash
export WALLET_ADDRESS=0xYOUR_WALLET_ADDRESS_OR_SOLANA_ADDRESS
export X402_AUTH=YOUR_BASE64_SIWX_HEADER

curl --request GET \
  --url "https://api.venice.ai/api/v1/x402/balance/$WALLET_ADDRESS" \
  --header "X-Sign-In-With-X: $X402_AUTH"
```

响应包括：

- `canConsume`：钱包是否可以发起付费请求
- `balanceUsd`：当前可消费余额
- `minimumTopUpUsd` 和 `suggestedTopUpUsd`：充值指导
- `diemBalanceUsd`：DIEM 支撑余额（若有）

`walletAddress` 路径参数同时接受 EVM 地址（`0x...`）和 Solana base58 地址。

### 第 3 步：充值

在 Base 或 Solana 上用 USDC 充值：

```bash
curl --request POST \
  --url https://api.venice.ai/api/v1/x402/top-up
```

首次调用会返回 `402 Payment Required`，并带一个 `PAYMENT-REQUIRED` 响应头，其中包含 `accepts` 数组。每个条目描述一种被接受的支付方式，包括 `network`、`asset`、`payTo` 和 `amount`。选择你希望用 Base 或 Solana 的支付选项，按照这些精确细节构造一个 `X-402-Payment` 响应头，然后重试同一路由。

#### 为 Base 构造 X-402-Payment 响应头

下面的脚本在 Base 上创建一个已签名的 x402 支付并发送充值请求。需要 npm 包 `x402` 和 `viem`。

```bash
export EVM_PRIVATE_KEY=0xYOUR_PRIVATE_KEY
```

```bash
export X402_PAYMENT="$(
node --input-type=module <<'EOF'
import { createPaymentHeader } from "x402/client";
import { privateKeyToAccount } from "viem/accounts";

const signer = privateKeyToAccount(process.env.EVM_PRIVATE_KEY);
const amountUsd = 5;
const amountBaseUnits = String(Math.round(amountUsd * 1e6));

const header = await createPaymentHeader(signer, 2, {
  scheme: "exact",
  network: "base",
  maxAmountRequired: amountBaseUnits,
  resource: "https://api.venice.ai/api/v1/x402/top-up",
  description: "Venice x402 top-up",
  mimeType: "application/json",
  payTo: "0x2670B922ef37C7Df47158725C0CC407b5382293F",
  maxTimeoutSeconds: 300,
  asset: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
  extra: { name: "USD Coin", version: "2" },
});

process.stdout.write(header);
EOF
)"

curl -X POST "https://api.venice.ai/api/v1/x402/top-up" \
  -H "X-402-Payment: $X402_PAYMENT"
```

<Note>
  在生产环境中，请以最新的 `PAYMENT-REQUIRED` / `accepts` 响应作为真相来源，而不是硬编码这些值。对于 Solana 充值，从 `accepts` 返回的 Solana 条目构造支付。Solana 条目使用 `network: "solana"`，`asset` 为 USDC mint，并可能包含网络相关的元数据，如 `extra.feePayer`。
</Note>

### 第 4 步：发起请求

一旦钱包有可消费余额，使用 `X-Sign-In-With-X` 响应头调用任何受支持的 endpoint：

```bash
export X402_AUTH=YOUR_BASE64_SIWX_HEADER

curl --request POST \
  --url https://api.venice.ai/api/v1/chat/completions \
  --header "Content-Type: application/json" \
  --header "X-Sign-In-With-X: $X402_AUTH" \
  --data '{
    "model": "kimi-k2-5",
    "messages": [
      {
        "role": "user",
        "content": "Hello from an x402-authenticated wallet."
      }
    ]
  }'
```

成功的响应可能包含 `X-Balance-Remaining` 响应头。

### 第 5 步：查看交易（可选）

查看钱包的交易历史：

```bash
export WALLET_ADDRESS=0xYOUR_WALLET_ADDRESS_OR_SOLANA_ADDRESS
export X402_AUTH=YOUR_BASE64_SIWX_HEADER

curl --request GET \
  --url "https://api.venice.ai/api/v1/x402/transactions/$WALLET_ADDRESS?limit=10&offset=0" \
  --header "X-Sign-In-With-X: $X402_AUTH"
```

账本包括 `TOP_UP`、`CHARGE` 和 `REFUND` 等条目。

`walletAddress` 路径参数同时接受 EVM 地址（`0x...`）和 Solana base58 地址。

## 支持的路由

### 付费推理路由

以下 Venice 公共付费路由目前支持 x402 钱包认证。

| 类别 | Endpoints |
| --- | --- |
| Chat | `POST /api/v1/chat/completions`、`POST /api/v1/responses` |
| Image | `POST /api/v1/image/generate`、`POST /api/v1/images/generations`、`POST /api/v1/image/upscale`、`POST /api/v1/image/edit`、`POST /api/v1/image/multi-edit`、`POST /api/v1/image/background-remove` |
| Embeddings | `POST /api/v1/embeddings` |
| Audio | `POST /api/v1/audio/speech`、`POST /api/v1/audio/transcriptions`、`POST /api/v1/audio/complete`、`POST /api/v1/audio/queue`、`POST /api/v1/audio/retrieve` |
| Video | `POST /api/v1/video/complete`、`POST /api/v1/video/queue`、`POST /api/v1/video/retrieve`、`POST /api/v1/video/transcriptions` |

### 充值路由

| Endpoint | 认证 | 用途 |
| --- | --- | --- |
| `POST /api/v1/x402/top-up` | 初次请求：无。重试：`X-402-Payment` | 使用受支持的 Base 或 Solana 支付方式向钱包的可消费余额添加 USDC 积分 |

### 仅钱包路由

这些路由使用 `X-Sign-In-With-X` 进行身份识别，但不会扣除余额。

| Endpoint | 用途 |
| --- | --- |
| `GET /api/v1/x402/balance/{walletAddress}` | 查询 EVM 或 Solana 钱包地址的可消费余额 |
| `GET /api/v1/x402/transactions/{walletAddress}` | 查看 EVM 或 Solana 钱包地址的交易历史 |

## 错误

| 状态 | 可能原因 | 处理方式 |
| --- | --- | --- |
| `401` | Sign-In-With-X 负载格式错误或已过期 | 用新的 nonce 和时间戳重建 `X-Sign-In-With-X` |
| 在付费路由上的 `402` | 可消费余额不足 | 充值并重试 |
| 在 `/x402/top-up` 上的 `402` | 预期情况。这是支付发起流程。 | 使用返回的支付细节构造 `X-402-Payment` 并重试 |
| 在 balance 或 transactions 上的 `403` | 钱包不匹配 | 确认认证钱包与 `walletAddress` 路径参数一致 |
| 在 top-up 上的 `400` | 支付响应头格式错误 | 根据最新的 `402` 响应重建 |

---

## 给 agent

流程相同。将私钥保存在环境变量或密钥管理器中，并在请求前先检查余额以避免不必要的 `402` 往返。

[Claude Code](https://docs.anthropic.com/en/docs/claude-code) 是 Anthropic 的 CLI 工具，用于代理式编码。本指南演示如何让它通过 Venice AI 运行，以按 token 付费的方式访问 Claude Opus 4.5/4.6 与 Sonnet 4.5/4.6。

## 为什么需要一个路由器

Claude Code 默认直接连接 Anthropic 的 API。要让它通过 Venice，你需要 [claude-code-router](https://github.com/musistudio/claude-code-router)，一个开源的本地代理，它会：

<Steps>
  <Step title="拦截" icon="hand">
    在请求到达 Anthropic 之前捕获 Claude Code 的出站请求
  </Step>
  <Step title="转换" icon="arrows-rotate">
    转换请求格式并映射模型 ID（例如 `claude-opus-4-5`）
  </Step>
  <Step title="重定向" icon="route">
    将请求转发到 `api.venice.ai/api/v1/chat/completions` 上的 Venice
  </Step>
</Steps>

## 设置

<Steps>
  <Step title="安装 Claude Code">
    如果尚未安装，请先安装 Anthropic 的 Claude Code CLI：

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="安装路由器">
    ```bash
    npm install -g @musistudio/claude-code-router
    ```
  </Step>

  <Step title="获取 API 密钥">
    在 [venice.ai/settings/api](https://venice.ai/settings/api) 生成密钥。下一步会直接将其粘贴到配置文件中。
  </Step>

  <Step title="创建配置">
    创建配置目录：

    ```bash
    mkdir -p ~/.claude-code-router
    ```

    然后使用你偏好的编辑器创建 `~/.claude-code-router/config.json`：

    ```bash
    # 使用 nano
    nano ~/.claude-code-router/config.json

    # 或使用 VS Code
    code ~/.claude-code-router/config.json
    ```

    粘贴以下配置：

    ```json
    {
      "APIKEY": "",
      "LOG": true,
      "LOG_LEVEL": "info",
      "API_TIMEOUT_MS": 600000,
      "HOST": "127.0.0.1",
      "Providers": [
        {
          "name": "venice",
          "api_base_url": "https://api.venice.ai/api/v1/chat/completions",
          "api_key": "your-venice-api-key-here",
          "models": [
            "claude-opus-4-5",
            "claude-sonnet-4-5",
            "claude-opus-4-6",
            "claude-opus-4-6-fast",
            "claude-sonnet-4-6"
          ],
          "transformer": {
            "use": ["anthropic"]
          }
        }
      ],
      "Router": {
        "default": "venice,claude-opus-4-5",
        "think": "venice,claude-opus-4-5",
        "background": "venice,claude-opus-4-5",
        "longContext": "venice,claude-opus-4-5",
        "longContextThreshold": 100000
      }
    }
    ```

    <Note>
    如果在路由器运行期间修改了 `config.json`，需要执行 `ccr restart` 让更改生效。
    </Note>
  </Step>

  <Step title="启动">
    先启动路由器，再启动 Claude Code：

    ```bash
    ccr start
    ccr code
    ```

    或使用激活方式：

    ```bash
    eval "$(ccr activate)" && claude
    ```
  </Step>
</Steps>

## 支持的模型

| 模型 | Venice ID | 最佳用途 |
|-------|-----------|----------|
| Claude Opus 4.5 | `claude-opus-4-5` | 复杂推理、大型重构 |
| Claude Sonnet 4.5 | `claude-sonnet-4-5` | 快速迭代、日常编码 |
| Claude Opus 4.6 | `claude-opus-4-6` | 复杂推理、大型重构 |
| Claude Opus 4.6 Fast | `claude-opus-4-6-fast` | 低延迟的复杂推理 |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | 快速迭代、日常编码 |

<Info>
Claude Code 针对 Claude 模型进行了优化。虽然 Venice 上的其他模型（GPT、DeepSeek、Grok 等）也可能可用，但由于 Claude Code 依赖于 Claude 专属功能（如扩展思考），我们无法保证同等体验。对于其他模型，可考虑使用 Venice 的[标准 API](/zh/api-reference/endpoint/chat/completions)。
</Info>

## 路由器功能

除了基础路由，该路由器还提供多项实用功能：

<AccordionGroup>
  <Accordion title="按需切换模型">
    在 Claude Code 内部使用 `/model` 命令可在不重启的情况下切换模型：
    
    ```
    /model venice,claude-sonnet-4-5
    ```
    
    适用于在复杂任务时使用 Opus，在快速迭代时使用 Sonnet。
  </Accordion>

  <Accordion title="通过 UI 模式可视化配置">
    更喜欢 GUI？启动基于网页的配置编辑器：
    
    ```bash
    ccr ui
    ```
    
    它会打开浏览器界面来编辑 `config.json`，无需直接操作文件。
  </Accordion>

  <Accordion title="路由场景说明">
    `Router` 配置段控制不同任务类型使用哪个模型：
    
    | 场景 | 使用时机 |
    |----------|----------------|
    | `default` | 一般请求 |
    | `think` | 推理密集型任务（Plan Mode） |
    | `background` | 后台操作 |
    | `longContext` | 上下文超过 `longContextThreshold` token 时 |
    
    你可以将不同场景路由到不同模型。例如，让后台任务使用 Sonnet 以节省成本。
  </Accordion>

  <Accordion title="通过日志调试">
    出现问题时，可查看日志：
    
    ```bash
    # 服务端日志（HTTP、API 调用）
    ~/.claude-code-router/logs/ccr-*.log
    
    # 应用日志（路由决策）
    ~/.claude-code-router/claude-code-router.log
    ```
    
    将配置中的 `"LOG_LEVEL"` 设为 `"debug"` 可获得更详尽的输出。
  </Accordion>
</AccordionGroup>

## 缓存行为

Venice [提示缓存](/zh/guides/features/prompt-caching)可与 Claude Code 的原生 cache 标记协同工作。Venice 会自动检测 Claude Code 发送的 `cache_control` 字段，并据此调整其缓存策略。

| 场景 | Cache TTL | 由谁控制 |
|----------|-----------|--------------|
| 默认（推荐） | 5 分钟 | Claude Code + Venice |
| 使用 `cleancache` 转换器 | 1 小时 | 仅 Venice |

<AccordionGroup>
  <Accordion title="何时不要使用 cleancache（大多数用户）">
    默认配置允许两个系统协作：
    
    - Claude Code 发送其原生的 `cache_control` 标记
    - Venice 在其周围添加 5 分钟 TTL 的缓存
    - 两个系统共享 4 块的缓存上限
    
    这非常适合频繁发起请求的活跃编码会话。
  </Accordion>

  <Accordion title="何时使用 cleancache">
    在以下情况下，将 `cleancache` 加入 transformer：
    
    - 遇到 4 块缓存上限错误
    - 出现奇怪的缓存行为
    - 偏好 Venice 1 小时的 TTL 以适配较长会话
    
    ```json
    "transformer": {
      "use": ["anthropic", "cleancache"]
    }
    ```
    
    这会剥离 Claude Code 的 cache 标记，将完整控制权和更长的 TTL 交给 Venice。
  </Accordion>
</AccordionGroup>

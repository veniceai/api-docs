[Brave Leo](https://brave.com/leo/) 是内置于 Brave 浏览器中的 AI 助手。Leo 的 Bring Your Own Model (BYOM) 功能可以直接连接到 Venice 兼容 OpenAI 的聊天补全端点，让您无需安装扩展即可使用 Venice 模型进行聊天和页面感知辅助。

## 前置条件

- Windows、macOS 或 Linux 上的 Brave 浏览器 1.69 或更高版本
- 来自 [venice.ai/settings/api](https://venice.ai/settings/api) 的 Venice API 密钥
- 来自 [Venice 文本模型目录](/models/text) 的模型 ID

<Note>
Brave 的 BYOM 功能在桌面端可用。模型 ID 必须与 Venice 模型 ID 完全一致。
</Note>

## 设置

<Steps>
  <Step title="打开 Leo 设置">
    打开 `brave://settings/leo-ai`，或从 Brave 的侧边栏打开 Leo，选择设置图标，然后进入 **Leo**。
  </Step>

  <Step title="添加自定义模型">
    滚动到 **Bring your own model**，然后选择 **Add new model**。
  </Step>

  <Step title="配置 Venice 端点">
    输入以下值：

    | 字段 | 值 |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | 来自[文本模型目录](/models/text) 的模型 ID |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | 您的 Venice API 密钥 |

    如果 Brave 显示 **Context size**，请将其设置为所选模型支持的上下文窗口。您还可以添加可选的系统提示。

    请直接输入 API 密钥本身，不要带 `Bearer` 前缀。Brave 会为每个请求添加授权头。
  </Step>

  <Step title="保存并选择 Venice">
    选择 **Add model**，然后将 **Venice AI** 设为新会话的默认模型，或从 Leo 的模型选择器中选择它。
  </Step>

  <Step title="测试连接">
    在网页上打开 Leo，发送如下提示：

    ```txt
    Summarize this page in five bullet points.
    ```

    Leo 会将对话和相关页面内容直接发送到 Venice 端点。
  </Step>
</Steps>

## 启用 Venice 网络搜索

Brave 在 BYOM 中不会暴露 Venice 特定的请求体参数。要启用 Venice 网络搜索，请在 **Model request name** 中的模型 ID 后追加一个 [模型特性后缀](/api-reference/endpoint/chat/model_feature_suffix)：

```txt
<model-id>:enable_web_search=auto
```

若要始终启用搜索并包含引用来源，请使用：

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
网络搜索有额外的按用量计费。当前费率请参阅 [API 定价](/overview/pricing)。
</Info>

## 隐私

当您使用远程 BYOM 端点时，Brave 会直接连接到该提供商。请求会绕过 Brave 的服务器，因此 Brave 的代理和服务器端保留保护不会适用于这些请求。

Venice 不会在其代理处存储或记录提示词和响应内容。所选模型决定了运行时的隐私层级：

- **Private** 模型使用零数据保留基础设施。
- **Anonymized** 模型对上游提供商隐藏可识别信息，但提供商可以处理提示。
- **TEE** 和 **E2EE** 模型在支持的情况下增加硬件支持的隔离或客户端加密。

在发送敏感信息前，请查看[文本模型目录](/models/text)中的模型隐私标签，并阅读 [Venice 隐私](/overview/privacy)。

Brave 可能会将 Leo 聊天历史保存在本地浏览器存储中。您可以清除 Leo 数据、使用临时聊天，或在 Brave 的 Leo 设置中禁用本地聊天历史。

## 配置注意事项

- Brave 需要完整的端点路径：`https://api.venice.ai/api/v1/chat/completions`。
- 模型 ID 及其可用性会随时间变化。请从[文本模型目录](/models/text)复制最新 ID。
- Context size 不应超过所选模型支持的上下文窗口。
- 页面摘要可能会将页面内容作为提示的一部分发送到 Venice。
- 远程 Venice 请求需要互联网连接，会消耗 Venice API 额度或您可用的 DIEM 配额。

## 疑难排查

<AccordionGroup>
  <Accordion title="401 或身份验证错误">
    确认密钥来自 [venice.ai/settings/api](https://venice.ai/settings/api)。粘贴原始密钥，不要带 `Bearer`，并去除任何前导或尾随空格。
  </Accordion>
  <Accordion title="找不到模型">
    从[文本模型目录](/models/text)完整复制模型 ID。如果您添加了特性后缀，请先在不带后缀的情况下测试该模型。
  </Accordion>
  <Accordion title="网络错误或没有流式响应">
    确认服务器端点包含 `/api/v1/chat/completions`。Brave Leo 期望的是兼容 OpenAI 的流式响应。
  </Accordion>
  <Accordion title="页面摘要缺乏页面上下文">
    检查 Leo 使用页面内容的权限，并确保当前页面对浏览器可访问。Brave 的内部页面和一些受保护页面无法被摘要。
  </Accordion>
</AccordionGroup>

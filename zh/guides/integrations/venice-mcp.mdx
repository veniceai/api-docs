[Venice MCP Server](https://github.com/veniceai/venice-mcp-server) 是 Venice 官方的 [Model Context Protocol](https://modelcontextprotocol.io/) 服务器。它将整个 Venice API（chat、image、video、audio、music、embeddings、web augment 和 characters）作为 **31 个工具**暴露出来，供任何兼容 MCP 的 agent 调用。

  作为 [`@veniceai/mcp-server`](https://www.npmjs.com/package/@veniceai/mcp-server) 在 npm 上发布。MIT 许可证。
</Card>

## 快速开始

<Steps>
  <Step title="获取 Venice API key">
    在 [venice.ai/settings/api](https://venice.ai/settings/api) 生成一个。逐步说明见 [API key 指南](/zh/guides/getting-started/generating-api-key)。
  </Step>

  <Step title="将 Venice 添加到 MCP 宿主配置">
    将以下内容放入你的 MCP 宿主配置文件：

    ```json
    {
      "mcpServers": {
        "venice": {
          "command": "npx",
          "args": ["-y", "@veniceai/mcp-server@0.2.0"],
          "env": { "VENICE_API_KEY": "<your-venice-api-key>" }
        }
      }
    }
    ```

    常见配置路径：

    | 宿主 | 路径 |
    |---|---|
    | Claude Desktop (macOS) | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Claude Desktop (Windows) | `%APPDATA%\Claude\claude_desktop_config.json` |
    | Cursor | `~/.cursor/mcp.json` |
    | LM Studio | `mcp.json`（来自应用的 MCP 设置） |
  </Step>

  <Step title="重启你的 MCP 宿主">
    现在你的 agent 已可使用 chat、image、video、music、TTS、ASR 以及另外 25 个 Venice 工具。
  </Step>
</Steps>

<Note>
大多数 MCP 宿主只会传递**显式列出**在 `env` 块中的环境变量。系统级 env 变量不会被继承。如果在已设置 API key 的情况下出现 402 错误，请再次确认配置中 `env` 内有 `VENICE_API_KEY`。
</Note>

## 你将获得什么

**31 个工具** 覆盖每一种 Venice 模态，**3 个资源**（`venice://models`、`venice://styles`、`venice://voices`），以及 **3 个 prompt 模板**。

### Chat 与 embeddings

| 工具 | 说明 |
|---|---|
| `venice_chat` | 针对 Venice 完整 LLM 目录的 OpenAI 兼容 chat completion。 |
| `venice_responses` | OpenAI 兼容的 Responses API，支持单轮或多轮工具调用。 |
| `venice_embeddings` | 为文本输入计算 embeddings。 |
| `venice_chat_with_character` | 按 slug 与 Venice character 对话。 |

### Image

| 工具 | 说明 |
|---|---|
| `venice_image_generate` | 生成图像（Flux 2、Lustify SDXL、Anime/WAI、Qwen Image、GPT Image、Nano Banana Pro 等）。 |
| `venice_image_edit` | 用 prompt 编辑图像。 |
| `venice_image_multi_edit` | 用一个 prompt 同时编辑多张图像。 |
| `venice_image_upscale` | 最多 4× 放大图像。 |
| `venice_image_remove_bg` | 去除图像背景。 |
| `venice_image_styles` | 列出图像样式预设。 |

### Video

| 工具 | 说明 |
|---|---|
| `venice_video_generate` | 将视频生成任务入队（Sora 2、Veo 3.1、Kling、Wan、LTX 2、Seedance、Runway Gen-4 等）。 |
| `venice_video_status` | 检查已入队视频任务的状态。 |
| `venice_video_complete` | 把已完成视频标记为已下载；删除服务端媒体。 |
| `venice_video_transcriptions` | 转录 YouTube 视频 URL。 |
| `venice_video_quote` | 入队前获取价格报价。 |

### Audio (TTS / ASR)

| 工具 | 说明 |
|---|---|
| `venice_tts` | 文本转语音，支持克隆音色和情感标签。 |
| `venice_asr` | 从 URL 转录音频。 |
| `venice_voice_clone` | 列出内置音色或从样本克隆音色。 |
| `venice_audio_quote` | 获取音乐生成的价格报价。 |

### Music

| 工具 | 说明 |
|---|---|
| `venice_music_generate` | 将音乐生成任务入队（`ace-step-15`、`elevenlabs-music`、`minimax-music-v2/v25/v26`、`stable-audio-25`、`mmaudio-v2`、`elevenlabs-sound-effects-v2`）。 |
| `venice_music_status` | 检查已入队音乐任务的状态。 |
| `venice_music_complete` | 把已完成的音乐任务标记为已下载。 |

### Web augment、catalog 和 crypto

| 工具 | 说明 |
|---|---|
| `venice_web_search` | 网页搜索（由 Firecrawl 提供）。 |
| `venice_web_scrape` | 把一个 URL 抓取为 markdown。 |
| `venice_text_parser` | 从 PDF/DOCX/EPUB/PPTX/XLSX 提取文本。 |
| `venice_list_models` | 列出实时模型目录和价格。 |
| `venice_list_characters` | 列出公开的 Venice characters。 |
| `venice_crypto_rpc` | 代理对 Base、Ethereum、Polygon、Arbitrum 或 Optimism 的 JSON-RPC 调用。 |

### x402 钱包辅助

仅在你通过 [x402](/zh/guides/integrations/x402-venice-api) 用钱包认证（而非 API key）时相关。

| 工具 | 说明 |
|---|---|
| `venice_x402_balance` | 查询 EVM 或 Solana 钱包地址的预付 x402 积分余额。 |
| `venice_x402_top_up_info` | 获取充值要求（网络、USDC 代币、收款方、最小金额）。 |
| `venice_x402_transactions` | 列出 EVM 或 Solana 钱包地址最近的 x402 充值与扣费记录。 |

## 配置

服务器完全通过环境变量配置。

| 环境变量 | 默认值 | 备注 |
|---|---|---|
| `VENICE_API_KEY` | _(无)_ | 你的 Venice API key。最简单的方式。 |
| `VENICE_DEFAULT_CHAT_MODEL` | `venice-uncensored` | |
| `VENICE_DEFAULT_IMAGE_MODEL` | `flux-2-pro` | |
| `VENICE_DEFAULT_TTS_MODEL` | `tts-kokoro` | |
| `VENICE_DEFAULT_ASR_MODEL` | `openai/whisper-large-v3` | |
| `VENICE_DISABLE_NSFW` | `0` | 设为 `1` 可从工具描述中移除 NSFW 能力提示。 |
| `VENICE_HTTP_TIMEOUT_MS` | `60000` | |
| `VENICE_SIWX_TOKEN` | _(无)_ | x402 钱包模式认证 token。见下文 [x402](#x402-钱包模式)。 |

如果同时设置 `VENICE_API_KEY` 和 `VENICE_SIWX_TOKEN`，以 API key 为准。

## x402 钱包模式

除了常规 API key 流程外，Venice 还支持使用由 Base 或 Solana 上预付 USDC 积分支撑的 [Sign-In-With-X 钱包 token](/zh/guides/integrations/x402-venice-api) 进行认证。无需邮箱、手机号或 KYC：你的钱包就是唯一身份。

```json
{
  "mcpServers": {
    "venice": {
      "command": "npx",
      "args": ["-y", "@veniceai/mcp-server@0.2.0"],
      "env": { "VENICE_SIWX_TOKEN": "<base64 Sign-In-With-X 负载>" }
    }
  }
}
```

MCP 服务器会把 `VENICE_SIWX_TOKEN` 作为 `X-Sign-In-With-X` 响应头转发到每次 Venice API 调用。服务器永远不会看到你的私钥。钱包签名和 USDC 充值授权都在你自己的钱包中完成。

| 流程 | 发生了什么 |
|---|---|
| **一次性设置** | 在钱包中签名一条 Sign-In-With-X 消息 → 生成一个 SIWX token（base64 JSON）。 |
| **充值** | `POST /api/v1/x402/top-up` 返回 402 + 支付要求。在返回的 Base 或 Solana 选项之一上签名一笔 USDC 支付并重新提交，Venice 即为你的余额入账。 |
| **每次推理调用** | MCP 服务器发送 `X-Sign-In-With-X: <SIWX>`；Venice 从你的预付余额中扣费。 |

最小充值额为 **\$5 USD**。可发起推理的最低余额为 **\$0.10**。一旦充值，调用延迟低于 100ms，因为结算在一个快速积分账户上离链完成。

<Tip>
关联到已质押 DIEM 的 Venice 账户的钱包会从质押余额中消费，而不是 USDC 积分，因此无需充值。
</Tip>

## 自托管（Streamable HTTP）

对于团队或工作区部署，可让 MCP 服务器通过 HTTP（而非 stdio）运行：

```bash
docker run -p 3333:3333 \
  -e VENICE_API_KEY=<your-venice-api-key> \
  -e VENICE_MCP_AUTH_TOKEN=<choose-a-long-random-token> \
  ghcr.io/veniceai/venice-mcp-server:latest
```

服务器现在可在 `http://localhost:3333/mcp` 访问。HTTP 客户端必须发送 `Authorization: Bearer <VENICE_MCP_AUTH_TOKEN>`。

<Warning>
`/mcp` 是一个凭证支撑的工具执行 endpoint：调用者可以消耗已配置的 Venice API key 或 x402 余额。当 HTTP 模式绑定到非回环地址时，除非设置了 `VENICE_MCP_AUTH_TOKEN`，否则启动会失败。生产环境请显式锁定 npm 包版本，而不要依赖 `latest`。
</Warning>

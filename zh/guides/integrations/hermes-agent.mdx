[Hermes Agent](https://hermes-agent.nousresearch.com) 是由 [Nous Research](https://nousresearch.com) 打造的开源、自托管 AI 代理。它具备持久记忆、自主创建技能的能力，以及一个内置的学习循环 —— 运行越久，能力越强。将其指向 Venice API，代理就能访问覆盖文本、图像、视频、音频、嵌入等的 230+ 模型与工具。

## 为什么选择 Venice + Hermes Agent？

通过单一的 OpenAI 兼容端点，Venice API 让你的 Hermes Agent 接入整个 Venice 平台。

| 能力 | 你能获得什么 |
|---|---|
| **文本与聊天** | 私有与匿名模型（GLM、Qwen、Claude、GPT 等） |
| **图像** | 生成、编辑、放大与背景移除 |
| **视频** | 生成与转录 |
| **音频** | 语音合成（TTS）、音乐生成与语音转文本 |
| **嵌入** | 用于 RAG 与语义搜索的向量嵌入 |
| **工具** | 网页抓取、网页搜索、文本解析与加密 RPC |

## 设置

### 1. 安装 Hermes Agent

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>

  <Tab title="WSL2（Windows）">
    先安装 [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install)，然后运行同一命令：
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>
</Tabs>

安装完成后重新加载 shell：

```bash
source ~/.zshrc   # 或：source ~/.bashrc
```

### 2. 将 Venice 配置为你的提供者

运行模型设置向导：

```bash
hermes model
```

选择 **Custom endpoint (self-hosted / VLLM / etc.)**，并在提示时输入：

| 字段 | 值 |
|---|---|
| **API base URL** | `https://api.venice.ai/api/v1` |
| **API key** | 你的 Venice API 密钥 |
| **Model name** | Venice 模型 ID（见下） |

如果尚未获取 API 密钥，请前往 [venice.ai/settings/api](https://venice.ai/settings/api)。

<Tip>
你也可以直接在 `~/.hermes/config.yaml` 中配置 Venice：

```yaml
model:
  default: zai-org-glm-5
  provider: custom
  base_url: https://api.venice.ai/api/v1
  api_key: ${VENICE_API_KEY}
```

并在 `~/.hermes/.env` 中设置密钥：

```bash
VENICE_API_KEY=your-key-here
```
</Tip>

### 3. 选择模型

当向导询问模型时，可根据用途选择：

| 用例 | 模型 | 隐私 |
|---|---|---|
| 通用 | `zai-org-glm-5` | Private |
| 推理 | `kimi-k2-6` | Private |
| 编码 | `claude-opus-4-7` | Anonymized |
| 视觉 | `z-ai-glm-5v-turbo` | Anonymized |
| 未审查 | `venice-uncensored-1-2` | Private |

随时无需重启即可更换模型：

```bash
hermes model              # 完整向导
```

或会话中切换：

```text
/model custom:claude-opus-4-7
```

### 4. 开始聊天

打开经典 CLI 或现代 TUI：

```bash
hermes            # 经典 CLI
hermes --tui      # 现代 TUI（推荐）
```

尝试一条提示以验证一切正常：

```text
Summarize this repo in 5 bullets and tell me what the main entrypoint is.
```

## 连接消息平台

CLI 工作正常后，通过 gateway 连接你的消息应用：

```bash
hermes gateway setup
```

它会带你完成 Telegram、Discord、Slack、WhatsApp、Signal 等平台的连接。代理将能在任意已连接渠道接收消息，全部由 Venice 驱动。

```bash
hermes gateway      # 启动消息 gateway
```

## 隐私模式

Hermes Agent 中的 Venice 模型遵循与 Venice API 相同的[隐私层级](/zh/overview/privacy)：

- **Private** 模型（GLM、Qwen、DeepSeek、Llama、Venice Uncensored）运行在 Venice 的 GPU 集群上。提示永不存储或记录。
- **Anonymized** 模型（Claude、GPT、Gemini、Grok）通过 Venice 进行代理，所有可识别信息均被去除。第三方提供商看到的是 Venice 作为客户，而不是你。

## Venice API 技能

Hermes Agent 内置技能系统，兼容 [Agent Skills](https://github.com/veniceai/skills) 格式。Venice 发布了官方技能，教会代理如何使用 Venice 的所有端点（聊天、图像生成、视频、音频、嵌入、增强工具等）。

直接从 GitHub 安装 Venice 技能：

```bash
hermes skills install veniceai/skills
```

或搜索单个技能：

```bash
hermes skills search venice
```

Hermes 会通过每个技能的 `SKILL.md` 前置元数据进行发现，并按需加载。

## 关键命令

| 命令 | 说明 |
|---|---|
| `hermes` | 开始聊天 |
| `hermes model` | 更改提供者或模型 |
| `hermes tools` | 配置启用的工具 |
| `hermes gateway` | 启动消息 gateway |
| `hermes --continue` | 恢复上次会话 |
| `hermes doctor` | 诊断问题 |
| `hermes update` | 升级到最新版本 |

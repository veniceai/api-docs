Venice 通过单一凭证为你的 agent 同时提供推理（超过 230 个模型）和区块链访问（10 条 EVM 链加上 Starknet）。你的 agent 可以思考、签名并发送交易，无需在推理和 RPC 提供商之间分别维护不同的账户。

## 为什么用 Venice 跑链上 agent？

| 能力 | 你的 agent 获得什么 |
|---|---|
| **推理** | 通过单一 OpenAI 兼容 endpoint 使用 230+ 文本、图像、视频、音频和 embedding 模型 |
| **Crypto RPC** | 连接 10 条 EVM 链加 Starknet（主网与测试网）的 JSON-RPC 2.0 代理 |
| **认证** | 标准 API key 或 x402 钱包认证（无需 Venice 账户） |
| **资金注入** | 自主：质押 VVV 获得每日 DIEM。浏览器：通过 dashboard 进行 USD 或加密货币充值 |
| **批量请求** | 每个请求最多 100 个 JSON-RPC 调用，可并行跨多链 |
| **幂等性** | 通过 `Idempotency-Key` 响应头实现安全重试 |

## 认证

根据 agent 的运行方式选择匹配的认证方式。

| 方式 | 适用场景 | 工作方式 |
|---|---|---|
| **API key** | 服务端 agent，固定部署 | `Authorization: Bearer <key>` 响应头。在 [venice.ai/settings/api](https://venice.ai/settings/api) 获取 key。 |
| **x402 钱包** | 自主、加密原生或短生命周期的 agent | 钱包签名一条 Sign-In-With-X 消息，每次请求在 Base 或 Solana 上用 USDC 付费。无需 Venice 账户。参见 [x402 指南](/zh/guides/integrations/x402-venice-api)。 |

两种方式共享相同的速率限制和按 Venice 积分计费。

<Tip>
真正自主的 agent 可以通过在 Base 上质押 VVV 来自行铸造 API key。参见 [自主 Agent 创建 API Key](/zh/guides/getting-started/generating-api-key-agent)。
</Tip>

## Crypto RPC 快速开始

向 `POST /crypto/rpc/{network}` 发送任何 JSON-RPC 2.0 方法。

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/ethereum-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_chainId",
    "params": [],
    "id": 1
  }'
```

响应：

```json
{ "jsonrpc": "2.0", "id": 1, "result": "0x1" }
```

响应头包括 `X-Venice-RPC-Credits`（扣费的积分）、`X-Venice-RPC-Cost-USD`（美元成本）和 `X-Request-ID`（关联 ID）。

### 支持的网络

| 家族 | 主网 | 测试网 |
|---|---|---|
| Ethereum | `ethereum-mainnet` | `ethereum-sepolia`、`ethereum-holesky` |
| Base | `base-mainnet` | `base-sepolia` |
| Arbitrum | `arbitrum-mainnet` | `arbitrum-sepolia` |
| Optimism | `optimism-mainnet` | `optimism-sepolia` |
| Polygon | `polygon-mainnet` | `polygon-amoy` |
| Linea | `linea-mainnet` | `linea-sepolia` |
| Avalanche C-Chain | `avalanche-mainnet` | `avalanche-fuji` |
| BNB Smart Chain | `bsc-mainnet` | `bsc-testnet` |
| Blast | `blast-mainnet` | `blast-sepolia` |
| zkSync Era | `zksync-mainnet` | `zksync-sepolia` |
| Starknet | `starknet-mainnet` | `starknet-sepolia` |

使用 [`GET /crypto/rpc/networks`](/zh/api-reference/endpoint/crypto/networks) 获取实时权威列表。

### 方法分级

方法被分为三档积分等级。总成本 = `baseCredits[chain] × methodTier`。

| 等级 | 倍数 | 示例 |
|---|---|---|
| **Standard** | 1x | `eth_call`、`eth_getBalance`、`eth_blockNumber`、`eth_sendRawTransaction`、`eth_getLogs`、`eth_getTransactionReceipt`、`eth_estimateGas` |
| **Advanced** | 2x | `trace_block`、`trace_call`、`trace_transaction`、`debug_traceCall`、`debug_traceTransaction` |
| **Large** | 4x | `trace_replayBlockTransactions`、`trace_replayTransaction`、`txpool_content` |

完整列表和定价详情见 [Crypto RPC API 参考](/zh/api-reference/endpoint/crypto/rpc)。

## Agent 食谱

需要在链上读写的 AI agent 常用模式。

### 读取钱包的原生余额

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_getBalance",
    "params": ["0xYourWalletAddress", "latest"],
    "id": 1
  }'
```

### 读取 ERC-20 代币余额

使用 `eth_call` 调用 `balanceOf(address)` 选择器。`data` 字段是 4 字节选择器（`0x70a08231`）后接钱包地址（左填充至 32 字节）。最简单的方式是用库来编码：

```typescript
import { encodeFunctionData, parseAbi } from 'viem'

const data = encodeFunctionData({
  abi: parseAbi(['function balanceOf(address) view returns (uint256)']),
  args: ['0xWalletAddress'],
})

const response = await fetch('https://api.venice.ai/api/v1/crypto/rpc/base-mainnet', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    jsonrpc: '2.0',
    method: 'eth_call',
    params: [{ to: '0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf', data }, 'latest'],
    id: 1,
  }),
})
```

上面的合约地址是 Base 上的 VVV。替换为任意 ERC-20 合约即可。

### 发送已签名交易（完整生命周期）

Venice 永远不持有你的私钥。Agent 通过 RPC 读取收集交易参数，使用 [viem](https://viem.sh) 或 [ethers](https://docs.ethers.org) 等库在本地签名，然后通过 Venice 中继原始十六进制。

<Steps>
  <Step title="获取下一个 nonce">
    ```bash
    curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
      -H "Authorization: Bearer $VENICE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"jsonrpc":"2.0","method":"eth_getTransactionCount","params":["0xAgentWallet","pending"],"id":1}'
    ```
    使用 `"pending"`，这样背靠背发送不会冲突。
  </Step>
  <Step title="获取 gas 价格">
    ```bash
    curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
      -H "Authorization: Bearer $VENICE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"jsonrpc":"2.0","method":"eth_gasPrice","params":[],"id":1}'
    ```
    对于 EIP-1559 链，优先使用 `eth_feeHistory` 计算 `maxFeePerGas` 和 `maxPriorityFeePerGas`。
  </Step>
  <Step title="估算 gas">
    ```bash
    curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
      -H "Authorization: Bearer $VENICE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"jsonrpc":"2.0","method":"eth_estimateGas","params":[{"from":"0xAgentWallet","to":"0xRecipient","value":"0x0","data":"0x..."}],"id":1}'
    ```
  </Step>
  <Step title="本地签名">
    ```typescript
    import { privateKeyToAccount } from 'viem/accounts'
    import { base } from 'viem/chains'

    const account = privateKeyToAccount(process.env.AGENT_PRIVATE_KEY)

    const signed = await account.signTransaction({
      chainId: base.id,
      nonce,                  // 来自步骤 1
      gas,                    // 来自步骤 3
      maxFeePerGas,           // 来自步骤 2 (fee history)
      maxPriorityFeePerGas,   // 来自步骤 2 (fee history)
      to: '0xRecipient',
      value: 0n,
      data: '0x...',
    })
    ```
  </Step>
  <Step title="通过 Venice 提交">
    ```bash
    curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
      -H "Authorization: Bearer $VENICE_API_KEY" \
      -H "Idempotency-Key: agent-tx-<id>" \
      -H "Content-Type: application/json" \
      -d '{"jsonrpc":"2.0","method":"eth_sendRawTransaction","params":["0xSignedHex"],"id":1}'
    ```
    在中继时始终设置 `Idempotency-Key`，避免网络抖动造成重复广播。
  </Step>
  <Step title="轮询回执">
    ```bash
    curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
      -H "Authorization: Bearer $VENICE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"jsonrpc":"2.0","method":"eth_getTransactionReceipt","params":["0xTxHash"],"id":1}'
    ```
    每隔几秒轮询一次，直到 `result` 不为 null。检查 `result.status`（`"0x1"` = 成功）。
  </Step>
</Steps>

<Note>
每次 `eth_sendRawTransaction` 调用都会在服务端记录交易哈希、网络、请求 ID 和调用用户 ID。已签名的负载本身不会被保留。这条审计链路的存在是为了：如果有人用被泄露的密钥进行非法中继，可以追溯到对应的责任账户。
</Note>

### 批量调用（多链组合查询）

在一个请求中最多发送 100 个 JSON-RPC 对象。每个对象都会被独立校验并独立计费。

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/ethereum-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '[
    { "jsonrpc": "2.0", "method": "eth_blockNumber", "params": [], "id": 1 },
    { "jsonrpc": "2.0", "method": "eth_getBalance", "params": ["0xWallet", "latest"], "id": 2 },
    { "jsonrpc": "2.0", "method": "eth_gasPrice", "params": [], "id": 3 }
  ]'
```

对于跨链读取（每条链一次调用），向不同的 `{network}` endpoint 并行发起请求。

### 使用幂等性进行安全重试

将 `Idempotency-Key` 响应头设置为任意符合 `[A-Za-z0-9_-]{1,255}` 的字符串。Venice 会以 `(user, key)` 为键缓存响应 24 小时。重放会返回缓存结果并带 `Idempotent-Replayed: true`，且不再扣费。

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/base-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Idempotency-Key: agent-tx-2026-04-21-001" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_sendRawTransaction",
    "params": ["0xSignedRawTxHex"],
    "id": 1
  }'
```

这对交易中继至关重要——网络抖动可能会让 agent 误把同一笔交易广播两次。

## 为 agent 的 API key 注入资金

agent 拿到 Venice API key 之后，底层账户必须有可消费余额，付费 endpoint 才会接受该 key。有两种方式注入余额：

| 路径 | 自主？ | 工作方式 |
|---|---|---|
| **通过质押 VVV 获得 DIEM** | 是 | 在 Base 上的 [Venice 质押智能合约](https://basescan.org/address/0x321b7ff75154472B18EDb199033fF4D116F340Ff#code) 中质押 VVV。钱包的每日 DIEM 分配按其在质押池中的份额比例计算。账户必须累积至少 0.1 DIEM，才有任何 DIEM 可用于消费。DIEM 在 UTC 00:00 刷新。要提高日消费量，请质押更多 VVV。 |
| **通过 dashboard 进行 USD 或加密货币充值** | 否（需浏览器） | 用同一个钱包（Sign-In-With-Ethereum）登录 [venice.ai](https://venice.ai)，然后在 Settings、API 中添加积分。Stripe（信用卡）和 Coinbase（加密货币）都在该页面背后，需要浏览器。积分永不过期。 |

对于无人值守运行的 agent，**通过质押 VVV 获得 DIEM 是目前唯一一个对已铸造 API key 完全无人参与的资金路径**。如果 agent 的日消费超过其 DIEM 分配，现实选择是：质押更多 VVV，或者让运维人员登录并用 USD 或加密货币充值。

### 自主质押 VVV 并生成密钥

真正自主的 agent 可以在 Base 上自管 VVV 钱包、自行质押并自行铸造 Venice API key，全程无需人工参与。完整流程：

<Steps>
  <Step title="获取 VVV 和用于 gas 的 ETH">
    向 agent 的钱包转入 VVV（或让 agent 在 [Aerodrome](https://aerodrome.finance) 或 [Uniswap](https://app.uniswap.org) 上 swap），并准备少量 Base 上的 ETH 用于两笔质押交易。
  </Step>
  <Step title="质押 VVV">
    在 VVV 代币合约上 `approve` 质押合约，然后对 `0x321b7ff75154472B18EDb199033fF4D116F340Ff` 调用 `stake(amount)`。钱包的 sVVV 余额会与质押同步原子更新。
  </Step>
  <Step title="铸造 API key">
    `GET /api/v1/api_keys/generate_web3_key` 返回一个 15 分钟后过期的 JWT。用质押钱包对原始 token 签名，然后将地址、签名和 token 一起 `POST` 回去。Venice 会返回一个绑定到从该钱包派生的用户账户的 API key。
  </Step>
</Steps>

铸造只需要 sVVV 余额非零，因此质押 1 个 VVV 就足以获得 key。**使用** key 进行消费是另一个问题，依照上方的资金表。

完整代码演示与错误参考见 [自主 Agent 创建 API Key](/zh/guides/getting-started/generating-api-key-agent)。

## 30 秒上手 x402 钱包认证

如果你的 agent 已经有 Base 或 Solana 钱包，可以完全跳过 API key。SDK [`venice-x402-client`](https://github.com/veniceai/x402-client) 帮你处理 Sign-In-With-X 签名、充值和余额跟踪。

```bash
npm install venice-x402-client
```

```typescript
import { VeniceClient } from 'venice-x402-client'

const venice = new VeniceClient(process.env.WALLET_KEY)

await venice.topUp(10) // 如果钱包已有余额，可跳过

const response = await venice.chat({
  model: 'kimi-k2-6',
  messages: [{ role: 'user', content: 'What is the latest block on Base?' }]
})
```

同一个钱包认证也可用于 `/crypto/rpc/{network}` 的区块链读写。完整协议细节见 [x402 指南](/zh/guides/integrations/x402-venice-api)。

## 定价

Crypto RPC 按 Venice 积分计费。每个响应都包含 `X-Venice-RPC-Credits`（扣费积分）和 `X-Venice-RPC-Cost-USD`（美元成本），便于你的 agent 按请求跟踪开销。

### 各链基础积分

| 基础积分 | 链 |
|---|---|
| **20** | Ethereum、Base、Optimism、Arbitrum、Polygon、Linea、Avalanche、BSC、Blast、Starknet |
| **30** | zkSync Era |

### 成本示例

观察到的标准、高级和大型方法档次价格：

| 调用 | 积分 | 美元成本 |
|---|---|---|
| 在 Ethereum 上的 `eth_call` (20 × 1x) | 20 | ~$0.0000140 |
| 在 Ethereum 上的 `trace_transaction` (20 × 2x) | 40 | ~$0.0000280 |
| 在 Ethereum 上的 `trace_replayTransaction` (20 × 4x) | 80 | ~$0.0000560 |
| 在 zkSync 上的 `eth_call` (30 × 1x) | 30 | ~$0.0000210 |

权威成本请始终以 `X-Venice-RPC-Cost-USD` 响应头为准。批量请求中出错的条目按每条 5 个积分统一计费。

### 速率限制

| 等级 | 每分钟请求数 |
|---|---|
| 标准 | 100 |
| 员工 | 1,000 |

超出时 endpoint 会返回 `429` 并带标准 `X-RateLimit-*` 响应头。

## 错误处理

你的 agent 应处理的常见 HTTP 响应：

| 状态 | 含义 | 处理方式 |
|---|---|---|
| `400` | 不支持或未映射的 JSON-RPC 方法，或畸形的批量请求 | 对照 [allowlist](/zh/api-reference/endpoint/crypto/rpc) 核对方法。错误正文会指出有问题的方法。 |
| `400` | 用同一个 `Idempotency-Key` 但请求体不同进行重放 | 对不同请求使用新的 key。 |
| `402` | 完全缺少认证响应头（响应正文包含 `authOptions`，列出两种支持的认证方式），或带有有效认证响应头但积分用尽 | 若无认证：附加 `Authorization: Bearer ...` 或 x402 的 `X-Sign-In-With-X` 响应头。若积分用尽：使用 Bearer key 时为账户充值（DIEM、USD 或 dashboard 充值）；使用 x402 认证时直接调用 `POST /api/v1/x402/top-up`。 |
| `429` | 触发速率限制（标准 100 req/min，员工 1,000 req/min） | 遵守 `X-RateLimit-Reset` 并退避。每个请求最多批量 100 个调用以摊薄限制。 |
| `5xx` | 上游 RPC 节点抖动 | 使用相同的 `Idempotency-Key` 重试以避免重复扣费。 |

批量中按条目的错误（例如 N 个调用中某一个参数无效）会在 `200 OK` 响应中返回，并在出错条目上带 JSON-RPC `error` 字段。这些条目按每条 5 个积分计费。

## 不支持

以下类别的方法被有意拒绝：

- **仅 WebSocket**（`eth_subscribe`、`eth_unsubscribe`）：本代理仅支持 HTTP。请改用轮询。
- **有状态过滤器**（`eth_newFilter`、`eth_getFilterChanges` 等）：过滤器状态绑定到单一后端，在负载均衡代理上会失效。请改用 `eth_getLogs`。
- **持有密钥的方法**（`eth_sign`、`eth_accounts`、`eth_mining`）：托管提供商不持有用户密钥。请在客户端签名后通过 `eth_sendRawTransaction` 提交。
- **未映射的方法**：不在白名单内的都会返回 `400`。如需添加请联系支持。

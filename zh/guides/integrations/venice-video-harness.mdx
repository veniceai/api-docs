[Venice Video Harness](https://github.com/jordanurbs/venice-video-harness) 是一个社区维护的、面向 agent、为 Venice 优化的工具集，用于**任意时长的一致性优先视频创作**。它把一个 IDE agent（Claude Code、Cursor、Codex 等）变成可复用的 Venice 生产系统的操作员，覆盖 50+ 个 Venice 视频、图像、音频和音乐模型。

## 这是什么

大多数 Venice 集成都只是对 API 调用的薄包装。Venice Video Harness 是位于你的 agent 与 Venice API 之间的**更高层**：

- 编排规则放在 `CLAUDE.md`
- 可复用的 playbook 放在 `.claude/commands/`（19 个工作流命令）
- 专门的 agent 放在 `.claude/agents/`（art-director、prompt-engineer、cut-qa 等）
- Venice 生产 skills 放在 `.claude/skills/`（兼容 [Agent Skills](/zh/guides/integrations/venice-skills) 格式）
- TypeScript 执行层放在 `src/`
- 覆盖 50+ 个 Venice 视频、图像、音频和音乐模型的全面模型注册表

为以下类型的创作者构建：

- 角色一致的视频项目（任意类型、任意长度）
- 视觉风格锁定的系列或宣传活动
- Storyboard 到视频的工作流
- 短篇与长篇叙事内容
- 品牌化的电影感序列、预告片和短宣
- 含固定角色的常规化社交媒体系列

## Venice 优化体现在哪

- **图像 prompt 针对 Venice 图像模型调优**，如 `seedream-v5-lite`、`nano-banana-pro`、`flux-2-pro/max` 等
- **两轮分镜生成**，借助 Venice multi-edit 精修来修正角色
- 针对动作、氛围与角色一致性等档次的**模型路由逻辑**
- **感知参考**的视频生成，按模型正确使用 `elements`、`reference_image_urls` 和 `scene_image_urls`
- **环境感知的 prompt 调整**，区分白天与夜晚场景的处理
- **Venice 原生的音频路径**用于 TTS（Kokoro、Qwen3、ElevenLabs）、音效和音乐
- 生成前通过 `/video/quote` 和 `/audio/quote` 进行**成本估算**
- **模型感知的参数构造**，自动跳过目标模型不支持的参数

## 模型路由默认值

Harness 的默认值是有偏好的——因为一致性才是重点。当前路由（2026 年 4 月）：

**默认 Seedance 2.0 R2V。3+ 角色场景回退到 Kling O3 R2V。建立镜头使用 Seedance 2.0 i2v。**

| 角色 | 默认模型 | 何时使用 |
|---|---|---|
| 角色镜头（1-2 个角色） | `seedance-2-0-reference-to-video` | 默认 R2V，使用扁平 `reference_image_urls`、`@Image` 标签，最长 15s，原生立体声音频 |
| 角色镜头（3+ 个角色） | `kling-o3-standard-reference-to-video` | 自动回退到结构化 `elements`，用于多角色身份 |
| 建立 / 氛围 / 动作 | `seedance-2-0-image-to-video` | 无角色；史诗般的电影质感，最长 15s |

这些可在每个项目的 `series.json → videoDefaults` 中覆盖。要使用非 Seedance 家族（例如没有 Seedance 访问的账户），把 `videoDefaults` 设为 `kling-o3-standard-reference-to-video` 和 `veo3.1-fast-image-to-video`。

<Note>
**Seedance 面部规则：** Seedance 2.0 会阻止由非 `seedream-v5-lite` 或 `seedream-v5-lite-edit` 生成的、含人脸的输入图像。Harness 会自动处理这一点：把含角色的图像工作通过 Seedream 路由，并在每次 Seedance 调用前运行一次预检门控。
</Note>

| **Kling 2.6 / 2.5 Turbo** | Pro | Pro | 10s | 2.6：是 / 2.5：否 | `end_image_url` |
| **Veo 3.1** | Fast, Full | Fast, Full | 8s | 是 | 最高 4K 分辨率 |
| **Sora 2** | Standard, Pro | Standard, Pro | 12s | 是 | 最高 1080p |

| **LTX Video 2.0** | Fast, Full, v2.3, 19B | Fast, Full, v2.3, 19B | 20s | 是 | 最高 4K，同步时长最长 |
| **Longcat** | Std, Distilled | Std, Distilled | **30s** | 否 | 最长单镜头 |

### 图像、音频与音乐

- **图像（22+ 个模型）：** `nano-banana-pro/2`、`gpt-image-2`、`flux-2-pro/max`、`grok-imagine`、`qwen-image-2-pro`、`recraft-v4-pro`、`seedream-v4` / `v5-lite`、`lustify-sdxl/v7`、`wai-Illustrious` 等
- **Multi-edit：** `qwen-edit`、`flux-2-max-edit`、`nano-banana-pro-edit`、`seedream-v5-lite-edit`、`gpt-image-2-edit` 等
- **TTS：** `tts-kokoro`（50+ 音色）、`tts-qwen3-0-6b/1-7b`、`elevenlabs-tts-v3`、`elevenlabs-tts-multilingual-v2`
- **音乐：** `elevenlabs-music`、`minimax-music-v2`、`ace-step-15`、`stable-audio-25`
- **音效：** `elevenlabs-sound-effects-v2`、`mmaudio-v2-text-to-audio`

## 生产流水线

### 生成流水线

端到端叙事视频（脚本 → storyboard → 视频 → 音频 → 组装）：

```bash
npm run dev -- produce-episode -p output/my-series -e 1
```

`src/mini-drama/` 中的参考实现涵盖：

- 系列 / 角色 / 剧集管理
- 由 LLM 驱动的剧本研讨
- 两轮 storyboard 生成（generate + multi-edit 精修）
- 基于视觉的分镜 QA
- 带帧链的视频生成
- 分层音频后期制作
- 字幕烧录与最终组装

### 剪辑流水线

剪辑既有素材（Venice 生成的镜头或真实的原始素材）。**文本优先**：LLM 读取一个紧凑的 `takes_packed.md`（每 40 分钟音频约 12KB），而不是逐帧导出视频。

五个步骤：

<Steps>
  <Step title="转录">
    本地 whisper.cpp 为每个源生成 `*.words.json` + `takes_packed.md`。
  </Step>
  <Step title="阅读包">
    LLM 仅从文本形成剪辑策略。
  </Step>
  <Step title="确认">
    提出策略，等待 "yes / revise / cancel"。
  </Step>
  <Step title="渲染 EDL">
    JSON 剪辑表 → 用 30ms 音频淡入淡出的 ffmpeg concat。归档优先，原片永不被覆盖。
  </Step>
  <Step title="自我评估">
    `cut-qa` agent 在每个剪辑边界跑 6 项程序化检查；最多迭代 3 次修正。
  </Step>
</Steps>

`cut-qa` 检查会捕捉宽高比回退、单词内部的帧哈希跳变、VO 截断、光照不连续、超过 -6 dBFS 的音频峰值，以及字幕与画面内文字重叠等问题。

<Tip>
剪辑流水线受 [browser-use/video-use](https://github.com/browser-use/video-use) 启发。他们的核心洞见——*"LLM 永远不看视频，它阅读视频"*——正是 agent 驱动剪辑无需淹没在帧导出 token 中的关键。
</Tip>

## 命令、Agent 和 Skills

Harness 暴露了 19 个工作流命令、10 个专门 agent 和 7 个生产 skill。亮点：

| 工作流命令 | 用途 |
|---|---|
| `new-series` | 创建一个具有锁定美学的新系列 |
| `add-character` / `lock-character` | 角色 + 音色锁定 |
| `workshop-episode` | 协作式剧集剧本撰写 |
| `storyboard-episode` | 为一集做 storyboard |
| `produce-episode` | 一条命令跑完整流水线 |
| `generate-trailer` | 完整的预告片流水线 |
| `edit-footage` | 针对既有素材的文本优先剪辑流水线 |
| `ingest-screenplay` | 摄取 Fountain 或 PDF 剧本 |

| 专门 Agent | 角色 |
|---|---|
| `art-director` | 美学、调色板、光照、构图决策 |
| `prompt-engineer` | Venice 图像 prompt，角色一致性 |
| `storyboard-qa` | 分镜面板 QA，做连续性与角色检查 |
| `cut-qa` | 渲染后质量门控（每个剪辑 6 项检查，最多 3 次迭代） |
| `overlay-designer` | 品牌化运动图形，多个并行子 agent |
| `trailer-curator` | 预告片镜头选择与防剧透规则 |

| 生产 Skill | 用途 |
|---|---|
| `venice-api` | Venice REST API 用法与默认值 |
| `venice-video-model-routing` | R2V 优先路由、决策树 |
| `character-consistency` | 多镜头角色一致性指引 |
| `shot-composition` | 镜头构图与镜头机位指引 |
| `screenplay-parsing` | 剧本解析工作流 |
| `video-editing` | 文本优先剪辑哲学、EDL 格式、cut-qa 循环 |

## NLE 往返

渲染完成后，把组装好的时间线导出为 XML，便于在你常用的剪辑软件中精修。每个视频片段、对白片段、音效片段和音乐线索都落在自己的轨道上。

```bash
mini-drama export-timeline -p output/<project> -e 1 --format fcpxml      # Final Cut Pro X
mini-drama export-timeline -p output/<project> -e 1 --format premiere    # Premiere Pro
mini-drama export-timeline -p output/<project> -e 1 --format davinci     # DaVinci Resolve
```

## 编程式使用

你也可以直接在自己的 TypeScript 中调用 harness 的模块：

```typescript
import { VeniceClient } from './src/venice/client.js';
import { generateVideo, quoteVideo } from './src/venice/video.js';
import { listVideoModels } from './src/venice/models.js';

const client = new VeniceClient();

const quote = await quoteVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  duration: '8s',
  audio: true,
});
console.log(`Estimated cost: $${quote.quote}`);

const result = await generateVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  prompt: 'A slow dolly shot pushes forward...',
  duration: '8s',
  imageUrl: 'data:image/png;base64,...',
  audio: true,
  outputPath: 'output/shot-001.mp4',
});

const longModels = listVideoModels({ minDurationSec: 20 });
```

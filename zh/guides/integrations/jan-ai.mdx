[Jan](https://jan.ai/) 是一款开源 AI 助手，能够在同一桌面应用中运行本地与远程模型。Venice 通过其 OpenAI 兼容的远程模型提供者支持在 Jan 中工作。

## 前置条件

- 在 [venice.ai/settings/api](https://venice.ai/settings/api) 获取 Venice API 密钥
- 已在本机安装并运行 [Jan](https://jan.ai/)
- 来自[模型目录](/zh/models/text)的 Venice 文本模型 ID

---

## 设置

<Steps>
  <Step title="获取 Venice API 密钥">
    在 [venice.ai/settings/api](https://venice.ai/settings/api) 生成或复制一个 API 密钥。
  </Step>

  <Step title="打开 Jan 的模型提供者设置">
    在 Jan 中打开 **Settings**，进入 **Model Providers**。

    选择 Jan 的 OpenAI 兼容自定义提供者流程。根据你的 Jan 版本不同，这可能显示为启用了 **Custom Endpoint** 的 **OpenAI**，或显示为 **Any OpenAI Compatible API** 提供者。
  </Step>

  <Step title="配置 Venice 端点">
    填写 Venice 提供者设置：

    | 字段 | 值 |
    |-------|-------|
    | Provider name | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | 你的 Venice API 密钥 |
    | API type | OpenAI-compatible |

    在添加模型之前，先保存或应用该提供者设置。
  </Step>

  <Step title="添加 Venice 模型">
    添加你希望使用的 Venice 模型 ID。Jan 要求模型 ID 与提供者完全匹配。

    可选的入门模型：

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    你可以从[模型目录](/zh/models/text)中添加任意受支持的文本模型。
  </Step>

  <Step title="开始聊天">
    打开一个新建或已有的 Jan 聊天，从模型选择器中选择你的某个 Venice 模型，并发送一条测试消息。

    Jan 会使用你的 API 密钥和 OpenAI 兼容的 base URL，将请求发送到 Venice。
  </Step>
</Steps>

## 验证设置

在 Jan 中发送一条简短提示：

```txt
Reply with one sentence confirming you are running through Venice.
```

如果提供者配置正确，Jan 应当从所选 Venice 模型流式接收响应。

你也可以在 Jan 之外用以下命令验证 API 密钥：

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## 配置说明

- 使用 `https://api.venice.ai/api/v1` 作为 base URL。不要包含 `/chat/completions`；Jan 会自动追加聊天请求的端点路径。
- 模型 ID 必须与 Venice 完全一致。如果 Jan 的选择器中没有出现某个模型，请使用[模型目录](/zh/models/text)中的模型 ID 手动添加。
- 将 Venice 配置为 OpenAI 兼容提供者。对于 Venice 的 Claude 模型，不要使用 Jan 原生的 Anthropic 提供者。
- 远程 Venice 请求需要网络连接，并会消耗 Venice API credit。

<Info>
添加 Venice 后，Jan 的本地模型仍可使用。你可以从聊天的模型选择器中在本地模型与 Venice 模型之间切换。
</Info>

## 故障排查

<AccordionGroup>
  <Accordion title="401 或认证错误">
    确认 API 密钥是来自 [venice.ai/settings/api](https://venice.ai/settings/api) 的 Venice 密钥，且前后没有多余空格。
  </Accordion>

  <Accordion title="404 或找不到模型">
    确认 Jan 中的模型 ID 与某个 Venice 模型 ID 完全匹配。同时检查 base URL 是否为 `https://api.venice.ai/api/v1`。
  </Accordion>

  <Accordion title="选择器中没有 Venice 模型">
    在 Jan 的提供者设置中手动添加该模型。使用完整的模型 ID，例如 `zai-org-glm-5-1`，并再次保存该提供者。
  </Accordion>

  <Accordion title="请求仍发往其他提供者">
    确保该对话使用的是绑定到你 Venice 提供者上的模型。如果 Jan 正在使用某个内置的 OpenAI 或 Anthropic 模型条目，请切换到你添加的 Venice 模型。
  </Accordion>
</AccordionGroup>

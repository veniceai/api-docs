[Jan](https://jan.ai/) 是一款开源 AI 助手，可在一个桌面应用中运行本地和远程模型。Venice 通过 Jan 的 OpenAI 兼容远程模型提供商支持在 Jan 中工作。

## 前置条件

- 来自 [venice.ai/settings/api](https://venice.ai/settings/api) 的 Venice API 密钥
- 已在您的机器上安装并运行的 [Jan](https://jan.ai/)
- 来自[模型目录](/models/text)的 Venice 文本模型 ID

---

## 设置

<Steps>
  <Step title="获取您的 Venice API 密钥">
    从 [venice.ai/settings/api](https://venice.ai/settings/api) 生成或复制 API 密钥。
  </Step>

  <Step title="打开 Jan 的模型提供商设置">
    在 Jan 中打开 **Settings** 并前往 **Model Providers**。

    选择 Jan 的 OpenAI 兼容自定义提供商流程。根据您的 Jan 版本，这可能显示为启用 **Custom Endpoint** 的 **OpenAI**，或者作为 **Any OpenAI Compatible API** 提供商。
  </Step>

  <Step title="配置 Venice 端点">
    输入 Venice 提供商设置：

    | 字段 | 值 |
    |-------|-------|
    | Provider name | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | 您的 Venice API 密钥 |
    | API type | OpenAI 兼容 |

    在添加模型之前保存或应用提供商设置。
  </Step>

  <Step title="添加 Venice 模型">
    添加您想使用的 Venice 模型 ID。Jan 要求模型 ID 与提供商完全匹配。

    良好的入门选项：

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    您可以从[模型目录](/models/text)中添加任何受支持的文本模型。
  </Step>

  <Step title="开始聊天">
    打开新的或现有的 Jan 聊天，从模型选择器中选择您的 Venice 模型之一，并发送测试消息。

    Jan 将使用您的 API 密钥和 OpenAI 兼容的 base URL 将请求发送到 Venice。
  </Step>
</Steps>

## 验证设置

从 Jan 发送一个简短的 prompt：

```txt
Reply with one sentence confirming you are running through Venice.
```

如果提供商配置正确，Jan 应该会从所选的 Venice 模型流式返回响应。

您还可以在 Jan 外部验证您的 API 密钥：

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## 配置说明

- 使用 `https://api.venice.ai/api/v1` 作为 base URL。不要包含 `/chat/completions`；Jan 会为聊天请求附加端点路径。
- 模型 ID 必须与 Venice 完全匹配。如果模型未出现在 Jan 的选择器中，请使用[模型目录](/models/text)中的模型 ID 手动添加。
- 将 Venice 配置为 OpenAI 兼容的提供商。不要为 Venice Claude 模型使用 Jan 原生的 Anthropic 提供商。
- 远程 Venice 请求需要互联网连接，并会消耗 Venice API 额度。

<Info>
添加 Venice 后，Jan 的本地模型仍然可用。您可以从聊天模型选择器中在本地模型和 Venice 模型之间切换。
</Info>

## 故障排查

<AccordionGroup>
  <Accordion title="401 或身份验证错误">
    请检查 API 密钥是否为来自 [venice.ai/settings/api](https://venice.ai/settings/api) 的 Venice 密钥，且前后没有多余空格。
  </Accordion>

  <Accordion title="404 或模型未找到">
    确认 Jan 中的模型 ID 与 Venice 模型 ID 完全匹配。同时检查 base URL 是否为 `https://api.venice.ai/api/v1`。
  </Accordion>

  <Accordion title="选择器中没有 Venice 模型">
    在 Jan 的提供商设置中手动添加模型。使用确切的模型 ID（例如 `zai-org-glm-5-1`），并再次保存提供商。
  </Accordion>

  <Accordion title="请求仍然发送到其他提供商">
    确保聊天使用的是附加到 Venice 提供商的模型。如果 Jan 使用了内置的 OpenAI 或 Anthropic 模型条目，请切换到您添加的 Venice 模型。
  </Accordion>
</AccordionGroup>

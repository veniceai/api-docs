## Opciones de tamaño

Los modelos de imagen utilizan parámetros de dimensionado específicos:

- Los modelos basados en píxeles aceptan `width` y `height`, por ejemplo `venice-sd35` y `qwen-image`.
- Los modelos basados en relación de aspecto aceptan `aspect_ratio`, por ejemplo `qwen-image-2`.
- Los modelos por niveles de resolución aceptan tanto `aspect_ratio` como `resolution`.

Por ejemplo, `gpt-image-2`, `nano-banana-2` y `nano-banana-pro` admiten valores de `resolution` `1K`, `2K` y `4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

El mismo patrón de dimensionado por niveles de resolución también se aplica a los modelos Nano Banana:

```json
{
  "model": "nano-banana-pro",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Para modelos que aceptan relación de aspecto pero no exponen niveles de resolución, envía `aspect_ratio` sin `resolution`:

```json
{
  "model": "qwen-image-2",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9"
}
```

Para modelos que usan dimensiones directas en píxeles, envía `width` y `height` en su lugar:

```json
{
  "model": "venice-sd35",
  "prompt": "a serene canal in venice at sunset",
  "width": 1024,
  "height": 1024
}
```

Consulta [Modelos de imagen](/es/models/image) o el [endpoint de modelos](/es/api-reference/endpoint/models/list) para conocer las opciones de tamaño admitidas por cada modelo antes de mezclar campos de tamaño entre modelos.

## Niveles de calidad

Algunos modelos aceptan un parámetro opcional `quality` (`low`, `medium`, `high`) que intercambia fidelidad visual por coste. Actualmente lo admite `gpt-image-2`; otros modelos ignoran el parámetro.

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K",
  "quality": "medium"
}
```

Si omites `quality`, el modelo usa su nivel predeterminado (`high` para `gpt-image-2`). El precio depende de la combinación de `resolution` y `quality` — los precios por nivel para `gpt-image-2` (1K/2K/4K × low/medium/high) se exponen bajo `model_spec.pricing.quality` en el [endpoint de modelos](/es/api-reference/endpoint/models/list) y se listan en la [visión general de precios](/es/overview/pricing).

<Info>
La facturación según calidad es el comportamiento por defecto para todos los clientes de la API y los SDK — no se requiere ninguna cabecera para activarlo. Las solicitudes que omiten `quality` se cobran al nivel predeterminado del modelo (`high` para `gpt-image-2`).
</Info>

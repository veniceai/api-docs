Obtén analíticas de uso agregadas para el usuario autenticado, con desgloses por fecha, modelo y API key. Este endpoint proporciona vistas resumidas de tus datos de uso de la API para crear paneles y monitorizar el consumo. Los datos se almacenan en caché durante 10 minutos.

## Parámetros de consulta

Puedes especificar el período de tiempo para las analíticas usando:
- **lookback**: un período relativo como "7d" (7 días), "30d" (30 días), hasta "90d" (90 días)
- **startDate** y **endDate**: un rango de fechas personalizado en formato `YYYY-MM-DD`. Ambos son obligatorios si se proporciona alguno.

Si no se especifican parámetros, el período de lookback predeterminado es de 7 días.

## Campos de la respuesta

### lookback
El período de lookback utilizado para la consulta. Puede estar en formato "Nd" (p. ej., "7d") o en formato "startDate:endDate".

### byDate
Totales de uso diario para el período solicitado.

- **date**: la fecha en formato `YYYY-MM-DD`
- **USD**: uso total en USD para ese día
- **DIEM**: uso total en DIEM para ese día

### byModel
Desglose de uso por modelo, ordenado por gasto total (de mayor a menor).

- **modelName**: nombre visible del modelo (p. ej., "GLM 5")
- **unitType**: tipo de unidades consumidas (tokens, imágenes, caracteres, minutos, segundos)
- **modelType**: tipo de modelo (LLM, IMAGE, TTS, ASR, VIDEO), o null
- **totalUsd**: total en USD gastado en este modelo
- **totalDiem**: total en DIEM gastado en este modelo
- **totalUnits**: total de unidades consumidas para este modelo
- **breakdown**: array con desgloses de uso por tipo (solo presente si hay varios tipos). Cada entrada contiene:
  - **type**: tipo de token (p. ej., "Input", "Output", "Cache Read", "Cache Write")
  - **usd**: importe en USD para este desglose
  - **diem**: importe en DIEM para este desglose
  - **units**: número de unidades para este desglose

### byModelDaily
Datos diarios del gráfico para los 8 modelos principales. Cada entrada contiene una "date" (timestamp) más los nombres de los modelos como claves con valores de uso en DIEM.

### topModels
Array con los 8 nombres de modelos con más uso, para las leyendas del gráfico.

### byKey
Desglose de uso por API key, ordenado por gasto total (de mayor a menor).

- **apiKeyId**: el ID de la API key, o null si el uso provino de la app web
- **description**: descripción de la API key o "Web App"
- **totalUsd**: total en USD gastado a través de esta key
- **totalDiem**: total en DIEM gastado a través de esta key
- **totalUnits**: total de unidades consumidas a través de esta key

### byKeyDaily
Datos diarios del gráfico para las 8 API keys principales. Cada entrada contiene una "date" (timestamp) más las descripciones de las keys como claves con valores de uso en DIEM.

### topKeyNames
Array con las 8 descripciones de API keys con más uso, para las leyendas del gráfico.

## Ejemplo de uso

```bash
# Obtener analíticas de uso de los últimos 7 días (predeterminado)
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Obtener analíticas de uso de los últimos 30 días
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics?lookback=30d" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Obtener analíticas de uso para un rango de fechas específico
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics?startDate=2024-01-01&endDate=2024-01-31" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

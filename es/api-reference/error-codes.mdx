---
title: Códigos de error
---

Cuando ocurre un error, la API de Venice devuelve un código de estado HTTP y un mensaje descriptivo. Algunos errores también incluyen campos adicionales, como estado de reembolso, modelos sugeridos para reintentar, detalles de validación o IDs de solicitud.

## Formato de respuesta de error

La mayoría de los errores devuelven un campo `error` con un mensaje legible para humanos:

```json
{
  "error": "Invalid request parameters"
}
```

Algunos endpoints incluyen campos adicionales específicos del endpoint. Por ejemplo, los errores de moderación de vídeo pueden incluir si se reembolsaron los créditos:

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits have been refunded.",
  "credits_refunded": true
}
```

Los errores de validación pueden incluir campos `details` e `issues` con información de validación a nivel de campo:

```json
{
  "error": "Invalid request parameters",
  "details": {
    "_errors": [],
    "field": {
      "_errors": ["Field is required"]
    }
  },
  "issues": []
}
```

Algunos endpoints compatibles con OpenAI pueden devolver objetos de error al estilo de OpenAI para casos específicos, como errores de longitud de contexto:

```json
{
  "error": {
    "message": "Your request exceeds the model's maximum context. Please reduce your prompt or completion length.",
    "type": "invalid_request_error",
    "param": "messages",
    "code": "context_length_exceeded"
  }
}
```

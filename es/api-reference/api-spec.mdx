La API de Venice ofrece interfaces REST y de streaming basadas en HTTP para construir aplicaciones de IA con modelos sin censura e inferencia privada. Puedes crear con generación de texto, generación de imágenes, embeddings y mucho más, sin políticas de contenido restrictivas. Encontrarás ejemplos de integración y SDKs en la [documentación](/es/overview/getting-started). Nuestra referencia de la API también está disponible como [especificación YAML de OpenAPI.](https://api.venice.ai/doc/api/swagger.yaml)

## Autenticación

La API de Venice utiliza API keys para la autenticación. Crea y gestiona tus API keys en tus [ajustes de la API](https://venice.ai/settings/api).


Todas las solicitudes a la API requieren autenticación HTTP Bearer:

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
Tu API key es un secreto. No la compartas ni la expongas en ningún código del lado del cliente.
</Note>

## Compatibilidad con OpenAI

La API de Venice implementa la especificación de la API de OpenAI, lo que garantiza compatibilidad con los clientes y herramientas existentes de OpenAI. Esto te permite integrarte con Venice utilizando la familiar interfaz de OpenAI y al mismo tiempo acceder a las características únicas y a los modelos sin censura de Venice.

### Configuración

Configura tu cliente para que use la URL base de Venice (`https://api.venice.ai/api/v1`) y haz tu primera solicitud:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## Características específicas de Venice

### System prompts

Venice ofrece system prompts por defecto diseñados para garantizar respuestas naturales y sin censura del modelo. Tienes dos opciones para gestionar los system prompts:

1. **Comportamiento por defecto**: tus system prompts se añaden a los predeterminados de Venice
2. **Comportamiento personalizado**: deshabilita por completo los system prompts de Venice

#### Desactivar los system prompts de Venice

Usa la opción `venice_parameters` para eliminar los system prompts por defecto de Venice:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### Parámetros de Venice

El objeto `venice_parameters` te permite acceder a características específicas de Venice que no están disponibles en la API estándar de OpenAI:

| Parámetro | Tipo | Descripción | Valor por defecto |
|-----------|------|-------------|---------|
| `character_slug` | string | El slug de un personaje público de Venice (se ve como "Public ID" en la página publicada del personaje) | - |
| `strip_thinking_response` | boolean | Elimina los bloques `<think></think>` de la respuesta (modelos que usan el formato heredado de etiqueta `<think>`). Consulta [Modelos de razonamiento](/es/guides/features/reasoning-models). | `false` |
| `disable_thinking` | boolean | En modelos de razonamiento compatibles, desactiva el razonamiento y elimina los bloques `<think></think>` de la respuesta | `false` |
| `enable_web_search` | string | Activa la búsqueda web para esta solicitud (`off`, `on`, `auto` — auto activa según el criterio del modelo)<br/>Se aplica un precio adicional basado en uso, consulta [precios](/es/overview/pricing#web-search-and-scraping). | `off` |
| `enable_web_scraping` | boolean | Activa el web scraping de hasta 5 URLs detectadas en el mensaje del usuario. El contenido extraído enriquece las respuestas y evita la búsqueda web. Solo se facturan las URLs scrapeadas con éxito.<br/>Se aplica un precio adicional basado en uso, consulta [precios](/es/overview/pricing#web-search-and-scraping). | `false` |
| `enable_x_search` | boolean | Activa la búsqueda nativa de xAI (web + X/Twitter) para los modelos Grok compatibles (p. ej., `grok-4-20-beta`). Ofrece resultados de búsqueda de mayor calidad usando la infraestructura de búsqueda de xAI. Cuando se activa, se omite la búsqueda web estándar de Venice.<br/>Se aplica un precio adicional basado en uso, consulta [precios](/es/overview/pricing#web-search-and-scraping). | `false` |
| `enable_web_citations` | boolean | Cuando la búsqueda web está activa, solicita que el LLM cite sus fuentes con el formato `[REF]0[/REF]` | `false` |
| `include_search_results_in_stream` | boolean | Experimental: incluye los resultados de búsqueda en el stream como el primer chunk emitido | `false` |
| `return_search_results_as_documents` | boolean | Expone los resultados de búsqueda en una llamada de herramienta compatible con OpenAI llamada `venice_web_search_documents` para integración con LangChain | `false` |
| `include_venice_system_prompt` | boolean | Si se deben incluir los system prompts por defecto de Venice junto con los system prompts especificados | `true` |

<Note>
Estos parámetros también se pueden especificar como sufijos del modelo añadidos al nombre del modelo (p. ej., `zai-org-glm-5:enable_web_search=auto`). Consulta [Sufijos de características de modelo](/es/api-reference/endpoint/chat/model_feature_suffix) para más detalles.
</Note>

### Prompt caching

Venice admite el almacenamiento en caché de prompts en modelos seleccionados para reducir latencia y costes en contenido repetido. Para los modelos compatibles, Venice almacena automáticamente en caché los system prompts: no se requieren cambios en el código. También puedes marcar contenido manualmente para caché usando la propiedad `cache_control` en el contenido del mensaje.

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `prompt_cache_key` | string | Pista de enrutamiento opcional para mejorar la tasa de aciertos de caché. Cuando se proporciona, Venice enruta las solicitudes a la misma infraestructura backend, aumentando la probabilidad de aciertos de caché en conversaciones multi-turno. |

Consulta [Prompt caching](/es/guides/features/prompt-caching) para más detalles sobre cómo funciona la caché, la facturación y las mejores prácticas.

## Referencia de cabeceras de respuesta

Todas las respuestas de la API de Venice incluyen cabeceras HTTP que aportan metadatos sobre la solicitud, los límites de uso, la información del modelo y el saldo de la cuenta. Además de los códigos de error devueltos en las respuestas, puedes inspeccionar estas cabeceras para obtener el ID único de una solicitud, monitorizar los límites de uso y rastrear el saldo de tu cuenta.

Venice recomienda registrar los IDs de solicitud (cabecera `CF-RAY`) en despliegues de producción para una solución de problemas más eficiente con nuestro equipo de soporte si fuera necesario.

La siguiente tabla ofrece una referencia exhaustiva de todas las cabeceras con las que te puedes encontrar:

| Cabecera | Tipo | Propósito | Cuándo se devuelve |
|--------|------|---------|---------------|
| **Cabeceras HTTP estándar** ||||
| `Content-Type` | string | Tipo MIME del cuerpo de la respuesta (`application/json`, `text/csv`, `image/png`, etc.) | Siempre |
| `Content-Encoding` | string | Codificación utilizada para comprimir el cuerpo de la respuesta (`gzip`, `br`) | Cuando el cliente envía la cabecera `Accept-Encoding` |
| `Content-Disposition` | string | Cómo se debe mostrar el contenido (p. ej., `attachment; filename=export.csv`) | Al descargar archivos o exportaciones |
| `Date` | string | Marca de tiempo en formato RFC 7231 de cuándo se generó la respuesta | Siempre |
| **Identificación de la solicitud** ||||
| `CF-RAY` | string | Identificador único para esta solicitud, usado para diagnóstico y soporte | Siempre |
| `x-venice-version` | string | Versión/revisión actual del servicio de la API de Venice (p. ej., `20250828.222653`) | Siempre |
| `x-venice-timestamp` | string | Marca de tiempo del servidor cuando se procesó la solicitud (formato ISO 8601) | Cuando está activado el seguimiento de timestamps |
| `x-venice-host-name` | string | Nombre del servidor que procesó la solicitud | Respuestas de error y escenarios de depuración |
| **Información del modelo** ||||
| `x-venice-model-id` | string | Identificador único del modelo de IA usado en la solicitud (p. ej., `venice-01-lite`) | Endpoints de inferencia que usan modelos de IA |
| `x-venice-model-name` | string | Nombre amigable/mostrado del modelo de IA usado (p. ej., `Venice Lite`) | Endpoints de inferencia que usan modelos de IA |
| `x-venice-model-router` | string | Router/servicio backend que gestionó la inferencia del modelo | Endpoints de inferencia cuando hay información de enrutamiento disponible |
| `x-venice-model-deprecation-warning` | string | Mensaje de advertencia para modelos programados para deprecación | Cuando se usa un modelo deprecado |
| `x-venice-model-deprecation-date` | string | Fecha en la que el modelo será deprecado (fecha ISO 8601) | Cuando se usa un modelo deprecado |

| `x-ratelimit-reset-requests` | number | Marca de tiempo Unix cuando se reinicia el límite de solicitudes | Todas las solicitudes autenticadas |
| `x-ratelimit-limit-tokens` | number | Número máximo de tokens (prompt + completion) permitidos en la ventana | Todas las solicitudes autenticadas |
| `x-ratelimit-remaining-tokens` | number | Tokens restantes en la ventana actual | Todas las solicitudes autenticadas |
| `x-ratelimit-reset-tokens` | number | Duración en segundos hasta que se reinicia el límite de tokens | Todas las solicitudes autenticadas |
| `x-ratelimit-type` | string | Tipo de límite aplicado (`user`, `api_key`, `global`) | Cuando se aplica el límite de uso |

| `x-pagination-limit` | number | Número de elementos por página | Endpoints paginados |
| `x-pagination-page` | number | Número de página actual (base 1) | Endpoints paginados |
| `x-pagination-total` | number | Número total de elementos en todas las páginas | Endpoints paginados |
| `x-pagination-total-pages` | number | Número total de páginas | Endpoints paginados |

| `x-retry-count` | number | Número de reintentos de la solicitud | Cuando hay reintentos de la solicitud |

### Notas importantes

- **Mayúsculas/minúsculas en nombres de cabecera**: las cabeceras HTTP no distinguen mayúsculas y minúsculas, pero Venice usa minúsculas con guiones por coherencia
- **Valores string**: los valores booleanos en cabeceras se devuelven como strings (`"true"` o `"false"`)
- **Valores numéricos**: los números grandes y los valores de saldo pueden devolverse como strings para evitar pérdida de precisión
- **Cabeceras opcionales**: no todas las cabeceras se devuelven en cada respuesta; la presencia depende del endpoint y del contexto de la solicitud
- **Compresión**: usa `Accept-Encoding: gzip, br` en las solicitudes para recibir respuestas comprimidas cuando esté soportado

### Ejemplo: acceder a las cabeceras de respuesta

```javascript
// Después de realizar una solicitud a la API, accede a las cabeceras desde el objeto de respuesta
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// Comprobar avisos de deprecación de modelo
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## Mejores prácticas

1. **Límites de uso**: monitoriza las cabeceras `x-ratelimit-remaining-requests` y `x-ratelimit-remaining-tokens` e implementa retroceso exponencial
2. **Monitorización del saldo**: rastrea las cabeceras `x-venice-balance-usd` y `x-venice-balance-diem` para evitar interrupciones del servicio
3. **System prompts**: prueba con y sin los system prompts de Venice para encontrar el mejor encaje para tu caso de uso
4. **API keys**: mantén tus API keys seguras y rótalas con regularidad
5. **Registro de solicitudes**: registra los valores de la cabecera `CF-RAY` para diagnóstico con soporte
6. **Deprecación de modelos**: comprueba las cabeceras `x-venice-model-deprecation-warning` al usar modelos

## Diferencias con la API de OpenAI

Aunque Venice mantiene una alta compatibilidad con la especificación de la API de OpenAI, existen algunas diferencias clave:

1. **venice_parameters**: configuraciones adicionales como `enable_web_search`, `character_slug` y `strip_thinking_response` para funcionalidad extendida
2. **System prompts**: Venice añade tus system prompts a los predeterminados que optimizan para respuestas sin censura (desactívalo con `include_venice_system_prompt: false`)
3. **Ecosistema de modelos**: Venice ofrece su propia [lista de modelos](/es/overview/models), incluyendo modelos sin censura y de razonamiento — usa los IDs de modelo de Venice en lugar de los mapeos de OpenAI
4. **Cabeceras de respuesta**: cabeceras únicas para seguimiento del saldo (`x-venice-balance-usd`, `x-venice-balance-diem`), avisos de deprecación de modelos y banderas de seguridad de contenido
5. **Políticas de contenido**: políticas más permisivas con modelos dedicados sin censura y filtrado de contenido opcional

## Estabilidad de la API

Venice mantiene retrocompatibilidad para los endpoints y parámetros v1. Para conocer la política del ciclo de vida de los modelos, los avisos de deprecación y la guía de migración, consulta [Deprecaciones](/es/overview/deprecations).

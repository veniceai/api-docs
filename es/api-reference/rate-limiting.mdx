## Gestión de errores

Las solicitudes fallidas (500, 503, 429) deben reintentarse con retroceso exponencial.

Para los errores 429 en concreto, revisa la cabecera `x-ratelimit-reset-requests` para saber la marca de tiempo Unix exacta en la que puedes reintentar. La mayoría de las librerías HTTP tienen mecanismos de reintento integrados que gestionan esto automáticamente.

### Protección contra abuso

Si generas más de 20 solicitudes fallidas en 30 segundos, la API bloqueará nuevas solicitudes durante 30 segundos:

```
Too many failed attempts (> 20) resulting in a non-success status code. Please wait 30s and try again.
```

## Cabeceras de respuesta

Cada respuesta incluye estas cabeceras:

| Cabecera | Descripción |
|:-------|:------------|
| `x-ratelimit-limit-requests` | Máximo de solicitudes permitidas en la ventana actual |
| `x-ratelimit-remaining-requests` | Solicitudes restantes en la ventana actual |
| `x-ratelimit-reset-requests` | Marca de tiempo Unix cuando se reinicia la ventana |
| `x-ratelimit-limit-tokens` | Máximo de tokens permitidos por minuto |
| `x-ratelimit-remaining-tokens` | Tokens restantes en el minuto actual |
| `x-ratelimit-reset-tokens` | Segundos hasta que se reinicia el límite de tokens |

Si alcanzas tus límites de uso de forma constante y tus patrones de uso muestran **demanda sostenida en el tiempo**, contacta para hablar sobre acceso de partner: [api@venice.ai](mailto:api@venice.ai).

Los límites del nivel Partner pueden ajustarse a tus necesidades específicas.

VVV y DIEM son la capa de financiación on-chain de Venice para la API. Haz staking de DIEM para recibir una asignación diaria fija de inferencia — **1 DIEM = $1 por día** de crédito de la API de Venice — sin comprar créditos USD prepagados para cada solicitud.

| Token | Rol |
| --- | --- |
| **VVV** | Token fundacional de Venice en Base. Haz staking para obtener rendimiento y para acuñar DIEM. |
| **sVVV** | Comprobante del VVV en staking. Bloquea sVVV a la [Mint Rate](https://diem-calculator.venice.ai/) actual para acuñar DIEM. |
| **DIEM** | Unidad de cómputo tokenizada. Haz staking de DIEM para desbloquear un crédito diario perpetuo de la API equivalente a $1 por DIEM. |

DIEM es un ERC-20 en Base. Puedes transferirlo, negociarlo o hacer staking con él. A diferencia de los créditos USD de pago por uso, el DIEM en staking es capacidad que te pertenece: la asignación diaria se renueva cada epoch y no se diluye a medida que crece el uso de la red.

<Info>
  Para las tarifas por modelo facturadas contra esa asignación, consulta [Precios de la API](/overview/pricing). Para pago por solicitud desde una wallet sin clave de API, consulta [x402](/guides/integrations/x402-venice-api).
</Info>

## Cómo funciona la financiación

Las claves de API gastan desde la cuenta de Venice vinculada en este orden: **DIEM**, luego créditos agrupados, luego USD.

Cuando DIEM es la moneda activa:

* Tu asignación diaria es igual a la cantidad de DIEM que tienes en staking (1 DIEM en staking → $1 de crédito de API en ese epoch).
* El DIEM no utilizado en un epoch no se acumula. La asignación se renueva a las **00:00 UTC**.
* Las cuentas necesitan al menos **0.1** DIEM en staking antes de que cualquier saldo de DIEM pueda gastarse.
* Los costos de las solicitudes usan la misma hoja de precios en USD que el pago por uso; DIEM es la moneda de liquidación a una tasa 1:1 con el dólar para la asignación de ese día.

Consulta el saldo actual y del próximo epoch de DIEM en el panel en [venice.ai/settings/api](https://venice.ai/settings/api), o a través del endpoint [Billing Balance](/api-reference/endpoint/billing/balance).

## Obtén DIEM y haz staking

Utiliza el panel de tokens en [venice.ai/token](https://venice.ai/token). Conecta una wallet de Base — no envíes tokens directamente a una dirección de contrato.

<Steps>
  <Step title="Adquiere VVV o DIEM">
    Compra VVV o DIEM en un DEX de Base como [Aerodrome](https://aerodrome.finance) o [Uniswap](https://app.uniswap.org), o recibe DIEM transferido desde otra wallet.

    | Activo | Contrato (Base) |
    | --- | --- |
    | VVV | [`0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf`](https://basescan.org/token/0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf) |
    | DIEM | [`0xF4d97F2da56e8c3098f3a8D538DB630A2606a024`](https://basescan.org/token/0xF4d97F2da56e8c3098f3a8D538DB630A2606a024) |
  </Step>

  <Step title="Haz staking de VVV (si vas a acuñar DIEM)">
    En [venice.ai/token](https://venice.ai/token), haz staking de VVV para recibir sVVV. El VVV en staking gana emisiones. El contrato de staking es [`0x321b7ff75154472B18EDb199033fF4D116F340Ff`](https://basescan.org/address/0x321b7ff75154472B18EDb199033fF4D116F340Ff#code).
  </Step>

  <Step title="Acuña DIEM desde sVVV (opcional)">
    Bloquea sVVV a la Mint Rate actual para acuñar DIEM. La Mint Rate es la cantidad de sVVV requerida por DIEM y aumenta a medida que crece el suministro de DIEM — puedes previsualizarla en la [calculadora de DIEM](https://diem-calculator.venice.ai/).

    El DIEM acuñado aparece en tu wallet como un ERC-20. El sVVV bloqueado permanece bloqueado hasta que quemes la misma cantidad de DIEM para desbloquearlo.
  </Step>

  <Step title="Haz staking de DIEM para obtener crédito de API">
    Haz staking de DIEM en el mismo panel de tokens. Cada DIEM en staking otorga **$1 por día** de crédito de la API de Venice mientras permanezca en staking.

    Retirar DIEM del staking tiene un período de enfriamiento de **1 día**. Desbloquear el sVVV que respaldó el DIEM acuñado requiere quemar ese DIEM; retirar del staking el sVVV liberado tiene luego un período de enfriamiento de **7 días**.
  </Step>

  <Step title="Crea o usa una clave de API">
    Inicia sesión en Venice con la misma wallet (Sign-In-With-Ethereum) para que el DIEM en staking se vincule a tu cuenta, luego crea una clave desde [Configuración de API](https://venice.ai/settings/api) o la [guía de claves de API](/guides/getting-started/generating-api-key).

    Los agentes autónomos pueden acuñar una clave desde una wallet que tenga VVV en staking — consulta [Creación de clave de API para agentes autónomos](/guides/getting-started/generating-api-key-agent).
  </Step>
</Steps>

<Warning>
  Nunca envíes VVV o DIEM directamente a una dirección de contrato. Los tokens enviados de esa forma no pueden recuperarse. Siempre haz staking a través de [venice.ai/token](https://venice.ai/token) o un flujo de transacción de staking conocido.
</Warning>

## Usa DIEM con la API

Una vez que el DIEM esté en staking y vinculado a tu cuenta, llama a la API con una clave Bearer normal. No se requieren encabezados adicionales ni `venice_parameters` — Venice deduce del DIEM automáticamente cuando `consumptionCurrency` es `DIEM`.

```bash
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### Consulta el saldo de DIEM

```bash
curl https://api.venice.ai/api/v1/billing/balance \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

Ejemplo de la forma de la respuesta:

```json
{
  "canConsume": true,
  "consumptionCurrency": "DIEM",
  "balances": {
    "diem": 90.5,
    "usd": 25
  },
  "diemEpochAllocation": 100
}
```

| Campo | Significado |
| --- | --- |
| `balances.diem` | Crédito de DIEM restante para el epoch actual (`null` si no estás haciendo staking de DIEM) |
| `diemEpochAllocation` | DIEM total asignado para este epoch (monto en staking). Compáralo con `balances.diem` para ver el uso hasta ahora |
| `consumptionCurrency` | Moneda que se cobrará a continuación (`DIEM`, `USD`, `BUNDLED_CREDITS`, o el heredado `VCU`) |
| `canConsume` | Si la cuenta puede pagar una solicitud en este momento |

Lista los modelos o las analíticas de uso para ver los costos por solicitud tanto en USD como en DIEM — coinciden a la tasa de $1 por DIEM para la asignación del día. Consulta [Billing Usage](/api-reference/endpoint/billing/usage) y [Usage Analytics](/api-reference/endpoint/billing/usage-analytics).

### Limita el gasto por clave

Al crear una clave, puedes establecer un límite de consumo por epoch denominado en `diem` o `usd` para que una sola integración no pueda agotar toda la asignación diaria. Las claves del panel exponen esto como **Epoch Consumption Limits**; las claves acuñadas vía Web3 aceptan `consumptionLimit` en [`POST /api_keys/generate_web3_key`](/api-reference/endpoint/api_keys/generate_web3_key/post).

## DIEM vs. créditos USD vs. x402

| Vía | Ideal para | Cómo se renueva |
| --- | --- | --- |
| **DIEM en staking** | Capacidad diaria predecible, agentes, costo de IA fijo | Asignación completa cada epoch (00:00 UTC) mientras el DIEM permanezca en staking |
| **Créditos USD / cripto** | Picos por encima de tu asignación de DIEM, gasto prepagado simple | El saldo disminuye con el uso; los créditos nunca expiran |
| **x402 (USDC)** | Pago por solicitud sin cabeza y sin clave de API | La wallet recarga USDC vía `POST /x402/top-up`; el DIEM en una cuenta vinculada aún se gasta primero cuando está presente |

Si una solicitud excediera el DIEM restante y no hay créditos en USD o agrupados que la cubran, la API devuelve `402` con `INSUFFICIENT_BALANCE`. Haz staking de más DIEM, espera al próximo epoch, o añade créditos USD en [Configuración → API](https://venice.ai/settings/api).

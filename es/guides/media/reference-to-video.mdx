Reference to Video te permite fijar la apariencia de personajes, objetos y escenas para que tus videos generados por IA mantengan la consistencia visual. En lugar de esperar que el modelo interprete tu prompt correctamente, tú aportas anclas visuales: imágenes de referencia que indican al modelo exactamente cómo es tu sujeto.

Esta función está disponible en los modelos **Kling O3** y **Grok Imagine R2V** dentro del [Venice Video Studio](https://venice.ai/video). Cada familia de modelos usa un enfoque distinto para las imágenes de referencia; consulta las secciones específicas por modelo más abajo.

## Cuándo usar Reference to Video

Utiliza Reference to Video cuando necesites:

- **Consistencia de personaje**: la misma persona o personaje en varias tomas
- **Precisión de producto**: un producto real que debe verse idéntico al original
- **Continuidad de escena**: un entorno o fondo específico entre generaciones
- **Escenas con varios personajes**: múltiples personajes distintos interactuando sin mezclarse

Para texto a video o imagen a video simples, donde la consistencia no es crítica, los modelos estándar funcionan bien sin referencias.

## Modelos disponibles

| Modelo | Enfoque | Ideal para |
|---|---|---|
| **Kling O3 Pro R2V** | Elementos + imágenes de escena | Escenas complejas con varios personajes y control preciso de identidad |
| **Kling O3 Standard R2V** | Elementos + imágenes de escena | Iteración más rápida en escenas basadas en elementos |
| **Grok Imagine R2V** | Imágenes de referencia planas | Generación rápida guiada por referencia con hasta 7 imágenes |

**Kling O3** usa un enfoque estructurado con Elements (anclas de identidad de personaje con imágenes frontales + de referencia) e imágenes de escena. **Grok Imagine R2V** adopta un enfoque más simple: subes imágenes de referencia directamente y las referencias en tu prompt con `@Image1`, `@Image2`, etc.

## Kling O3 Reference to Video

### Conceptos clave

Kling O3 Reference to Video usa tres tipos de entrada visual que funcionan en conjunto:

| Entrada | Requerida | Propósito | Cómo referenciarla en el prompt |
|---|---|---|---|
| **Elements** | Al menos una entrada visual* | Fija la identidad de un personaje u objeto | `@Element1`, `@Element2`, etc. |
| **Imágenes de referencia de escena** | Al menos una entrada visual* | Define el entorno, estilo y atmósfera | `@Image1`, `@Image2`, etc. |
| **Fotograma inicial** | Al menos una entrada visual* | Controla el primer fotograma del video | N/D (se establece al subir) |
| **Fotograma final** | No | Controla el último fotograma del video | N/D (se establece al subir) |

*Se requiere al menos uno de: fotograma inicial, elements o imágenes de referencia de escena.

### Elements

Un Element es un personaje u objeto que quieres mantener visualmente estable a lo largo del video. Cada element consta de:

- **Imagen frontal** (requerida por element): una foto clara y frontal del sujeto. Es el ancla principal de identidad. Piénsalo como la "foto de pasaporte" de tu personaje o producto.
- **Imágenes de referencia** (1–3, opcional): ángulos adicionales del mismo sujeto (vista lateral, ángulo de 45 grados, vista trasera). Ayudan al modelo a entender el sujeto en el espacio 3D. Si no se proveen, la imagen frontal se usa automáticamente como referencia.

Puedes añadir hasta **7 elements** por generación (limitado por el total combinado). Refiérelos en tu prompt usando `@Element1`, `@Element2`, etc.

### Imágenes de referencia de escena

Las referencias de escena definen el "escenario" donde transcurre la acción. Influyen en:

- Iluminación y paleta de colores
- Arquitectura y detalles del entorno
- Estilo visual y atmósfera general

Puedes añadir hasta **4 imágenes de escena**. Refiérelas como `@Image1`, `@Image2`, etc. en tu prompt.

### Limitaciones

El número total de imágenes entre todos los tipos de entrada está limitado:

| Límite | Valor |
|---|---|
| **Mínimo requerido** | Al menos 1 entrada visual (fotograma inicial, element o imagen de escena) |
| **Total combinado** (fotograma inicial + fotograma final + elements + imágenes de escena) | **7 máximo** |
| Elements (sin fotograma inicial/final) | 7 máximo |
| Elements (con fotograma inicial o final) | 3 máximo |
| Imágenes de referencia de escena | 4 máximo |
| Imágenes de referencia por element | 1–3 |

**Escenarios de ejemplo:**

- 7 elements + 0 imágenes de escena = 7 ✓ (sin fotogramas)
- 5 elements + 2 imágenes de escena = 7 ✓ (sin fotogramas)
- Fotograma inicial (1) + 3 elements + 3 imágenes de escena = 7 ✓
- Fotograma inicial (1) + fotograma final (1) + 3 elements + 2 imágenes de escena = 7 ✓
- Fotograma inicial (1) + 4 elements = ✗ (máx. 3 elements con fotograma)
- Fotograma inicial (1) + fotograma final (1) + 4 elements = ✗ (máx. 3 elements con fotogramas)

<Note>
Cada element requiere una **imagen frontal**. Si no proporcionas imágenes de referencia para un element, la imagen frontal se usa automáticamente como referencia.
</Note>

### Modo multi-toma

El modo multi-toma te permite dividir una sola generación en varias escenas, cada una con su propio prompt y duración. Los elements y las referencias de escena se mantienen entre todas las tomas, conservando la consistencia. La duración total de todas las tomas no puede superar los **15 segundos**.

### Guía paso a paso (Video Studio)

#### 1. Abre Video Studio y selecciona el modelo

Ve a [venice.ai/video](https://venice.ai/video). En el Model Browser a la izquierda, selecciona uno de los modelos **Kling O3 Reference to Video**:

- **Kling O3 Pro R2V**: mayor calidad, tiempo de generación más largo (~6 min)
- **Kling O3 Standard R2V**: más rápido, más rentable para iterar

#### 2. Añade entradas visuales (al menos una requerida)

Debes proporcionar **al menos una entrada visual** para generar un video: un fotograma inicial, un element o una imagen de referencia de escena. En el panel de entrada verás la sección **Elements**. Haz clic en **Add Element** para crear un element para personajes u objetos que quieras mantener visualmente consistentes.

Para cada element:

1. Haz clic en el recuadro **Frontal** para subir una imagen clara y frontal de tu personaje u objeto
2. Opcionalmente haz clic en **Add** bajo **Reference Images** para subir ángulos adicionales (1–3)

Repite para personajes u objetos adicionales (hasta 7 elements en total, o 3 si usas fotogramas inicial/final).

<Warning>
El total combinado de fotograma inicial, fotograma final, elements e imágenes de escena no puede superar **7**. Consulta [Limitaciones](#limitaciones) para más detalles.
</Warning>

<Tip>
**Mejores imágenes de referencia:** usa fotos bien iluminadas con fondo limpio. Aporta vistas frontal, lateral y de 45 grados para la mayor fijación de identidad. Asegúrate de que todas las imágenes de referencia compartan el mismo estilo visual (no mezcles fotorrealismo y anime).
</Tip>

#### 3. Añade imágenes de referencia de escena (opcional)

Debajo de la sección Elements verás **Scene Reference Images**. Sube imágenes que definan el entorno que deseas: una ubicación específica, configuración de iluminación o estilo artístico.

Estas se etiquetan automáticamente como `@Image1`, `@Image2`, etc.

#### 4. Sube un fotograma inicial (opcional)

Si quieres controlar el primer fotograma exacto de tu video, cambia al tipo de entrada **Image** y sube un fotograma inicial. También puedes establecer opcionalmente un fotograma final.

#### 5. Escribe tu prompt

En el campo de prompt, describe la acción que quieres mientras referencias tus elements e imágenes de escena con las etiquetas `@`:

```
@Element1 walks through the streets of @Image1, looking up at the buildings.
The camera slowly tracks from behind, revealing the city skyline.
```

Para **escenas con varios personajes**:

```
@Element1 and @Element2 enter the cafe in @Image1 from opposite sides.
@Element1 waves and walks toward @Element2, who is sitting at a corner table.
```

#### 6. Configura los ajustes

Abre **Video Settings** para ajustar:

| Ajuste | Opciones | Por defecto |
|---|---|---|
| Duración | 3s – 15s | 5s |
| Proporción de aspecto | 16:9, 9:16, 1:1 | 16:9 |
| Generar audio | On/Off | Off |

<Note>
La generación de audio añade efectos de sonido nativos, diálogo y audio ambiente sincronizados con el video. Aumenta el coste en ~25%.
</Note>

#### 7. Genera

Haz clic en **Generate Video**. Kling O3 normalmente tarda de 4 a 6 minutos según el nivel del modelo y la duración. Puedes encolar varias generaciones y explorar resultados en la Video Gallery.

### Storyboard multi-toma

Para secuencias narrativas, usa el modo multi-toma para definir escenas separadas dentro de una sola generación.

1. En el área de prompt, haz clic en **Add Shot** para crear tomas adicionales
2. Escribe un prompt separado para cada toma
3. Establece la duración de cada toma (3–15 s cada una, total ≤ 15 s)

Los elements y las referencias de escena persisten en todas las tomas automáticamente:

```
Shot 1 (5s): @Element1 stands at the edge of @Image1, looking out at the horizon.
Slow camera push forward.

Shot 2 (5s): Close-up of @Element1's face as they turn toward the camera.
Soft natural lighting, shallow depth of field.

Shot 3 (5s): @Element1 walks away from camera into the distance.
Wide cinematic shot, golden hour lighting.
```

<Warning>
La duración total multi-toma no puede superar los 15 segundos. Por ejemplo, tres tomas de 5 segundos = 15 s máximo.
</Warning>

### Consejos de prompting

#### Estructura tu prompt

Sigue este patrón para resultados fiables:

```
[sujeto con etiqueta @Element] + [acción] + [entorno con etiqueta @Image] + [movimiento de cámara] + [iluminación/estilo]
```

**Ejemplo:**

```
@Element1 hops happily across the candy ground of @Image1, stops to look at a
giant lollipop, tilts its head curiously. Cinematic tracking shot, soft warm lighting.
```

#### Mantén los prompts entre 50 y 150 palabras

Los prompts más cortos carecen de detalle. Los más largos introducen contradicciones. Apunta al punto óptimo.

#### Usa un lenguaje de cámara simple

El modelo responde mejor a indicaciones de cámara directas:

| Usar | Evitar |
|---|---|
| `slow camera push forward` | `dolly zoom with rack focus transition` |
| `tracking shot from behind` | `complex handheld parallax movement` |
| `close-up` | `extreme macro with tilt-shift bokeh` |
| `wide cinematic shot` | `anamorphic ultra-wide establishing crane shot` |

#### Usa un vocabulario consistente

Si describes a un personaje con "a red jacket" en un prompt, no cambies a "crimson coat" en el siguiente. El modelo trata palabras distintas como intenciones distintas.

#### Coloca las instrucciones de cámara al principio

Pon la dirección de cámara cerca del inicio del prompt para resultados más fiables:

```
Cinematic tracking shot of @Element1 walking through @Image1, leaves
blowing in the wind, golden afternoon light.
```

### Uso de Kling O3 vía API

Kling O3 Reference to Video también está disponible vía la API de Venice. Consulta la [API de Video Queue](https://docs.venice.ai/api-reference/endpoint/video/queue) para todos los detalles.

#### Python

```python
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "kling-o3-pro-reference-to-video",
        "prompt": "@Element1 walks through @Image1, camera tracking from behind",
        "duration": "8",
        "aspect_ratio": "16:9",
        "audio": True,
        "elements": [
            {
                "frontal_image_url": "https://example.com/character-front.jpg",
                "reference_image_urls": [
                    "https://example.com/character-side.jpg",
                    "https://example.com/character-angle.jpg"
                ]
            }
        ],
        "image_urls": [
            "https://example.com/scene-background.jpg"
        ]
    }
)

queue_id = response.json()["id"]
```

#### Esquema de Element

Cada element en el array `elements` acepta:

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `frontal_image_url` | string | **Sí** | URL de imagen frontal clara |
| `reference_image_urls` | string[] | No | URLs de ángulos adicionales (1–3). Si se omite, la imagen frontal se usa como referencia. |

<Note>
La API también soporta `video_url` para elements basados en video, pero esto actualmente no está disponible en la UI de Video Studio.
</Note>

### Solución de problemas de Kling O3

| Problema | Causa probable | Solución |
|---|---|---|
| El botón Generate está deshabilitado | No se proporcionan entradas visuales | Añade al menos una entrada visual: fotograma inicial, element o imagen de referencia de escena |
| Error "Number of images exceeds the limit" | Demasiadas entradas combinadas | El total de fotograma inicial + fotograma final + elements + imágenes de escena debe ser ≤ 7 |
| La cara del personaje cambia entre tomas | Imagen frontal distinta o ausente | Usa la misma imagen frontal de forma consistente, mantén la descripción idéntica |
| El movimiento de cámara parece aleatorio | Instrucciones de cámara múltiples o contradictorias | Usa una sola instrucción de cámara y colócala al inicio del prompt |
| El estilo cambia entre generaciones | Referencias de escena inconsistentes o estilos mezclados | Reutiliza las mismas imágenes de escena, mantén consistentes las palabras clave de estilo |
| Los elements se mezclan en escenas multi-personaje | Indicaciones espaciales vagas | Sé explícito sobre la posición de cada element: "foreground left", "entering from right" |
| El fondo parece distorsionado | Imagen de referencia de escena recargada o compleja | Usa imágenes de referencia de escena limpias y de alta calidad |
| El movimiento parece poco natural | Demasiadas acciones en un solo prompt | Simplifica la acción, usa duración más corta, una acción por toma |

<Tip>
Prueba con un clip de 3–5 segundos antes de comprometerte con duraciones más largas. Los clips cortos mantienen mejor la consistencia y te permiten iterar más rápido.
</Tip>

## Grok Imagine Reference to Video

Grok Imagine R2V adopta un enfoque más simple que Kling O3. En lugar de Elements estructurados con separación frontal/referencia, subes **imágenes de referencia planas** y las refieres directamente en tu prompt con `@Image1`, `@Image2`, etc. El modelo incorpora esos sujetos en el video generado.

### Cómo funciona

1. Sube **1–7 imágenes de referencia**: fotos de personajes, objetos o escenas que quieras en el video
2. Escribe un prompt que describa el video, usando `@Image1`, `@Image2`, etc., para referenciar imágenes específicas
3. El modelo genera un video incorporando esas referencias

Si no incluyes etiquetas `@Image` en tu prompt, todas las imágenes subidas se referencian automáticamente.

### Ajustes

| Ajuste | Opciones | Por defecto |
|---|---|---|
| Proporción de aspecto | 16:9, 4:3, 3:2, 1:1, 2:3, 3:4, 9:16 | 16:9 |
| Resolución | 480p, 720p | 480p |
| Duración | 5s, 8s, 10s | 8s |

<Note>
Grok Imagine R2V no soporta generación de audio, modo multi-toma ni Elements. Para esas funciones, usa Kling O3 R2V.
</Note>

### Guía paso a paso (Video Studio)

#### 1. Selecciona el modelo

Ve a [venice.ai/video](https://venice.ai/video). En el Model Browser, selecciona **Grok Imagine R2V**.

#### 2. Sube imágenes de referencia

Haz clic en **References** en la barra de entrada (o usa el menú +) para abrir el panel de imágenes de referencia. Sube 1–7 imágenes de los personajes, objetos o escenas que quieras en el video.

Cada imagen se etiqueta automáticamente como `@Image1`, `@Image2`, etc., en el orden en que las subes (de izquierda a derecha).

#### 3. Escribe tu prompt

Describe el video que quieres. Usa etiquetas `@Image` para referenciar imágenes específicas:

```
@Image1 and @Image2 walking together through a sunlit park,
camera slowly tracking alongside them, warm afternoon light.
```

Escribe `@` en el campo de prompt para ver un menú de autocompletado con las referencias de imagen disponibles.

<Tip>
Si omites por completo las etiquetas `@Image`, el backend antepone automáticamente referencias a todas las imágenes subidas. Esto es útil cuando quieres usar todas las imágenes sin especificar cuál es cuál.
</Tip>

#### 4. Configura ajustes y genera

Abre **Video Settings** para ajustar la proporción de aspecto, la resolución y la duración. Haz clic en **Generate Video**.

### Precios de Grok Imagine R2V

Grok Imagine R2V usa precios basados en duración y resolución:

| Resolución | Por segundo |
|---|---|
| 480p | ~$0.063 |
| 720p | ~$0.088 |

**Ejemplo:** un video de 8 segundos a 480p = 8 × $0.063 = **~$0.50**

<Note>
Grok Imagine cobra una tarifa de moderación de contenido por los videos generados, incluso si el video es rechazado. Esto se refleja en el coste en créditos mostrado antes de generar.
</Note>

#### Parámetros de la API

<Note>

</Note>

### Solución de problemas de Grok Imagine R2V

| Problema | Causa probable | Solución |
|---|---|---|
| El botón Generate está deshabilitado | No se subieron imágenes de referencia | Sube al menos 1 imagen de referencia |
| Error "At least one reference image is required" | `referenceImageUrls` está vacío o ausente | Proporciona al menos una URL de imagen en `referenceImageUrls` |
| Imagen incorrecta asociada a la etiqueta `@Image` | El orden de las imágenes no coincide con las etiquetas | `@Image1` corresponde a la primera imagen en tu orden de subida (de izquierda a derecha). Reordena las subidas si es necesario. |
| El sujeto no aparece en el video | Demasiadas referencias sin etiquetas explícitas | Usa etiquetas `@Image` en tu prompt para ser explícito sobre qué imágenes usar |
| Salida de baja calidad | Usando resolución 480p | Prueba 720p para mayor calidad (cuesta más) |
| Video demasiado corto | La duración por defecto es 8s | Establece la duración a `"10"` para videos más largos |

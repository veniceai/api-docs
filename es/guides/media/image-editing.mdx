La edición de imágenes en Venice es síncrona. Envía tu imagen fuente a `/image/edit` o `/image/multi-edit` y el resultado editado vuelve en la misma respuesta como archivo PNG. Para recortes, `/image/background-remove` devuelve un PNG transparente.

<Warning>
Los endpoints de edición de imágenes son experimentales y el comportamiento específico de modelo puede cambiar con el tiempo.
</Warning>

## Endpoints

| Endpoint | Propósito | Ideal para |
|---|---|---|
| `POST /image/edit` | Editar una sola imagen con un prompt | Ediciones generales e inpainting dirigido por prompt |
| `POST /image/multi-edit` | Editar usando 1-3 imágenes en capas | Ediciones más controladas con máscaras o superposiciones |
| `POST /image/background-remove` | Quitar el fondo de una imagen | Recortes transparentes para productos, retratos y activos |

## Cuándo usar cada endpoint

- Usa `/image/edit` cuando tengas una imagen fuente y quieras cambiar, eliminar o restyle una parte con un prompt.
- Usa `/image/multi-edit` cuando necesites control extra con máscaras, superposiciones o capas de referencia.
- Usa `/image/background-remove` cuando solo quieras un sujeto en primer plano limpio con transparencia.

<Note>
Para inpainting, usa `/image/edit` o `/image/multi-edit`. El antiguo parámetro `inpaint` de `/image/generate` está deprecado.
</Note>

## Paso 1: editar una sola imagen

La edición de una sola imagen es el flujo de inpainting más simple. Envía una imagen y un prompt corto como "remove the sign", "change the sky to sunrise" o "replace the background with a studio backdrop".

**Solicitud:**
```bash
POST https://api.venice.ai/api/v1/image/edit
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-edit",
  "prompt": "Replace the cloudy sky with a warm sunrise while preserving the buildings and canal",
  "image": "https://example.com/venice-canal.jpg"
}
```

**Respuesta (200):**
El cuerpo de la respuesta es datos binarios `image/png` en bruto. Guárdalo directamente en un archivo.

<CodeGroup>
```python Python
import base64
import os
import requests

with open("input.jpg", "rb") as f:
    image_base64 = base64.b64encode(f.read()).decode("utf-8")

response = requests.post(
    "https://api.venice.ai/api/v1/image/edit",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "qwen-edit",
        "prompt": "Remove the tourist crowd from the square and keep the architecture intact",
        "image": image_base64,
    },
)

with open("edited.png", "wb") as f:
    f.write(response.content)
```

```javascript Node.js
import fs from "fs";

const imageBase64 = fs.readFileSync("input.jpg").toString("base64");

const response = await fetch("https://api.venice.ai/api/v1/image/edit", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "qwen-edit",
    prompt: "Remove the tourist crowd from the square and keep the architecture intact",
    image: imageBase64,
  }),
});

const editedImage = Buffer.from(await response.arrayBuffer());
fs.writeFileSync("edited.png", editedImage);
```

```bash cURL

  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o edited.png \
  -d '{
    "model": "qwen-edit",
    "prompt": "Colorize this black and white portrait naturally",
    "image": "https://example.com/portrait-bw.jpg"
```

## Paso 2: usa multi-edit para máscaras o inpainting en capas

`/image/multi-edit` acepta hasta tres imágenes. La primera imagen es la imagen base. Las restantes se tratan como capas de edición o máscaras, lo que te da más control que la edición solo por prompt.

Es la mejor opción cuando quieres:

- apuntar a una región específica con una máscara
- combinar una composición existente con una superposición
- restringir la edición más estrictamente de lo que un prompt sobre una sola imagen permite

**Solicitud JSON:**
```json
{
  "modelId": "qwen-edit",
  "prompt": "Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective",
  "images": [
    "https://example.com/street-scene.png",
    "https://example.com/billboard-mask.png"
  ]
}
```

**Solicitud multipart:**
```bash
curl https://api.venice.ai/api/v1/image/multi-edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "modelId=qwen-edit" \
  -F "prompt=Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective" \
  -F "images=@street-scene.png" \
  -F "images=@billboard-mask.png" \
  -o multi-edited.png
```

Como en `/image/edit`, el cuerpo de la respuesta es datos `image/png` en bruto.

<Note>
`/image/multi-edit` usa actualmente el campo `modelId` en lugar de `model` en el esquema de solicitud.
</Note>

## Consejos de inpainting

El inpainting basado en prompt funciona mejor cuando la instrucción es corta y local:

- `remove the tree`
- `change the sky to sunset`
- `replace the logo with a blank sign`
- `restore the torn corner of the photo`

Para cambios de escena más amplios, describe lo que debe permanecer igual:

```text
Replace the background with a modern photo studio backdrop while preserving the subject pose, facial features, and clothing.
```

Si la edición sigue afectando a la zona equivocada, cambia de `/image/edit` a `/image/multi-edit` y proporciona una capa de máscara o superposición.

## Paso 3: eliminar el fondo

Usa `/image/background-remove` cuando quieras el sujeto en primer plano aislado sobre un fondo transparente. Este endpoint devuelve un PNG con transparencia alpha.

**Usando una URL de imagen:**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o cutout.png \
  -d '{
    "image_url": "https://example.com/product-photo.jpg"
  }'
```

**Usando la subida de un archivo local:**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "image=@product-photo.jpg" \
  -o cutout.png
```

Usa la eliminación de fondo para:

- fotos de producto de ecommerce
- fotos de perfil y retratos
- activos que planeas colocar sobre un nuevo fondo

## Parámetros de la solicitud

### `/image/edit`

| Parámetro | Tipo | Obligatorio | Predeterminado | Descripción |
|---|---|---|---|---|
| `image` | archivo, base64 o URL | Sí | - | Imagen fuente a editar |
| `prompt` | string | Sí | - | Instrucciones de texto para la edición |
| `model` | string | No | `qwen-edit` | ID del modelo de edición |
| `aspect_ratio` | string | No | predeterminado del modelo | Relación de aspecto de salida para los modelos compatibles |
| `modelId` | string | Deprecado | - | Alias deprecado de `model` |

### `/image/multi-edit`

| Parámetro | Tipo | Obligatorio | Predeterminado | Descripción |
|---|---|---|---|---|
| `images` | array de 1-3 archivos, strings base64 o URLs | Sí | - | La primera imagen es la base; el resto son capas de edición o máscaras |
| `prompt` | string | Sí | - | Instrucciones de texto sobre cómo combinar o editar las capas |
| `modelId` | string | No | `qwen-edit` | ID del modelo de edición |

### `/image/background-remove`

| Parámetro | Tipo | Obligatorio | Descripción |
|---|---|---|---|
| `image` | archivo o string base64 | Uno de `image` o `image_url` | Imagen fuente a recortar |
| `image_url` | string | Uno de `image` o `image_url` | URL pública de imagen a recortar |

---

## Formatos de entrada admitidos

| Endpoint | Entrada JSON | Entrada multipart | Salida |
|---|---|---|---|
| `/image/edit` | String base64 o URL | Subida de archivo | `image/png` |
| `/image/multi-edit` | Strings base64 o URLs | Subidas de archivo | `image/png` |
| `/image/background-remove` | String base64 o URL | Subida de archivo | `image/png` |

Para los endpoints de edición, las dimensiones de imagen deben ser de al menos `65536` píxeles y como máximo `33177600` píxeles. Los archivos subidos deben pesar menos de `25 MB`.

## Modelos y precios

El modelo de edición predeterminado es `qwen-edit`, con un precio de **$0,04 por edición**. Otros modelos con capacidades de edición pueden tener precios y restricciones diferentes.

## Errores

| Estado | Significado | Acción |
|---|---|---|
| `400` | Parámetros de solicitud inválidos | Comprueba el número de imágenes, los nombres de campo y el formato de entrada |
| `401` | Falló la autenticación | Comprueba tu API key |
| `402` | Saldo insuficiente | Añade créditos en [venice.ai/settings/api](https://venice.ai/settings/api) |
| `415` | Content type inválido | Usa JSON o multipart form-data correctamente |
| `429` | Límite de uso superado o modelo sobrecargado | Reintenta con backoff; revisa la cabecera `Retry-After` |
| `500` | Falló el procesamiento de la inferencia | Reintenta la solicitud |
| `503` | Modelo al límite de capacidad | Reintenta tras un breve retardo |

<Note>
Algunos modelos de edición tienen políticas de contenido más estrictas que los modelos de generación. Por ejemplo, `qwen-edit` bloquea solicitudes que impliquen imágenes sexuales explícitas, menores sexualizados o violencia del mundo real.
</Note>

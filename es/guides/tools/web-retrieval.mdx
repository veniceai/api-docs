Venice ofrece APIs independientes para descubrir páginas web y extraer su contenido:

- **Web Search** devuelve resultados estructurados con títulos, URL, fragmentos y fechas.
- **Web Scrape** convierte una página pública seleccionada en Markdown.

Úsalas juntas cuando tu aplicación necesite material fuente actual y verificable antes de la inferencia.

## Busca en la web

Envía una consulta a la API de Search:

```bash
curl https://api.venice.ai/api/v1/augment/search \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query":"latest developments in decentralized AI"}'
```

Brave es el proveedor predeterminado y ofrece Zero Data Retention. Google también está disponible con consultas anonimizadas enviadas mediante proxy a través de Venice. Consulta la [referencia de la API Web Search](/api-reference/endpoint/augment/search) para la selección del proveedor y el esquema completo de la respuesta.

## Extrae una página

Después de seleccionar un resultado, envía su URL pública a la API de Scrape:

```bash
curl https://api.venice.ai/api/v1/augment/scrape \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/article"}'
```

El endpoint devuelve el contenido de la página como Markdown. Primero solicita la representación Markdown nativa del sitio cuando esté disponible y, en caso contrario, utiliza extracción basada en navegador.

## Flujo de búsqueda y extracción

<Steps>
  <Step title="Busca">
    Envía una consulta específica y recopila los resultados estructurados.
  </Step>
  <Step title="Selecciona las fuentes">
    Filtra los resultados por relevancia, dominio y fecha antes de obtener el contenido de las páginas.
  </Step>
  <Step title="Extrae">
    Extrae Markdown solo de las páginas que tu aplicación tenga previsto utilizar.
  </Step>
  <Step title="Usa o almacena el contenido">
    Incluye el Markdown en un prompt, crea embeddings, guárdalo en caché o conserva las URL de origen para las citas.
  </Step>
</Steps>

## ¿Búsqueda independiente o búsqueda web del modelo?

Utiliza las APIs independientes cuando tu aplicación necesite controlar la selección de fuentes o quiera reutilizar el contenido recuperado. Habilita la búsqueda web en una solicitud al modelo cuando quieras que Venice se encargue de la recuperación como parte de una única solicitud de inferencia.

| Necesidad | Enfoque |
|---|---|
| Inspeccionar o filtrar resultados antes de la inferencia | Web Search independiente |
| Extraer una URL específica conocida | Web Scrape independiente |
| Almacenar en caché o reutilizar el contenido recuperado | APIs independientes |
| Permitir que el modelo recupere contexto actual automáticamente | Búsqueda web del modelo |

<Warning>
Web Search y Web Scrape son APIs experimentales. El formato de sus solicitudes y respuestas puede cambiar sin previo aviso. Web Scrape solo acepta páginas de acceso público, y los sitios que bloquean el acceso automatizado pueden ser rechazados.
</Warning>

[Venice Skills](https://github.com/veniceai/skills) es la colección canónica de [Agent Skills](https://www.anthropic.com/news/agent-skills) para la API de Venice. Cada skill es una carpeta autocontenida con un `SKILL.md` que un agente LLM carga bajo demanda para trabajar correctamente sobre un área específica de la API.

## ¿Por qué skills?

Sin skills, tu agente tiene que descubrir las particularidades de Venice por las malas: `venice_parameters`, enums de tipos de modelo, flujos 402 de pago requerido, ciclo de vida queue/retrieve de video, slugs de personajes, etc. Las skills agrupan ese conocimiento en archivos enfocados y bajo demanda para que el agente solo cargue lo que necesita para la tarea actual.

Cada `SKILL.md` incluye:

- Los endpoint(s) que cubre
- Cabeceras, parámetros y formas de respuesta requeridos
- Un ejemplo en curl y un ejemplo mínimo con SDK
- Una sección de "trampas" con las cosas en las que tropiezan los integradores reales

## Instalación

Cada skill es solo una carpeta con un `SKILL.md` que empieza con frontmatter YAML:

```yaml
---
name: venice-chat
description: When the agent should load this skill and what's in it
---
```

Coloca la carpeta `skills/` (o cualquier subconjunto) en la ruta que tu runtime monitoriza.

<Tabs>
  <Tab title="Claude Code">
    A nivel de proyecto:

    ```bash
    git clone https://github.com/veniceai/skills.git
    cp -r skills/skills/* .claude/skills/
    ```

    O global, para todos los proyectos de tu máquina:

    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.claude/skills/venice
    ```
  </Tab>

  <Tab title="Cursor">
    A nivel de proyecto:

    ```bash
    git clone https://github.com/veniceai/skills.git .cursor/skills-venice
    ```

    O copia skills individuales:

    ```bash
    cp -r skills/venice-chat .cursor/skills/
    ```
  </Tab>

  <Tab title="Codex">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.codex/skills/venice
    ```

    Para instalaciones a nivel de proyecto, apunta a `.codex/skills/`.
  </Tab>

  <Tab title="OpenCode">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ```

    OpenCode también lee `.opencode/skills/`, `.claude/skills/` y `.agents/skills/` desde la raíz del proyecto.
  </Tab>

### Referencia de rutas

| Runtime | A nivel de proyecto | Global |
|---|---|---|
| Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| Codex | `.codex/skills/` | `~/.codex/skills/` (o `$CODEX_HOME/skills/`) |
| OpenCode | `.opencode/skills/` (también `.claude/skills/`, `.agents/skills/`) | `~/.config/opencode/skills/` |
| Hermes Agent | `$HERMES_OPTIONAL_SKILLS_DIR` | `~/.hermes/skills/` |
| Cursor | `.cursor/skills/` | `~/.cursor/skills/` |
| Cline | `.clinerules/skills/` | n/a |
| Otros runtimes | `.agents/skills/` (convención) | `~/.agents/skills/` |

<Tip>
Los runtimes que definen campos extra en el frontmatter (`version`, `platforms`, `metadata.*`, `compatibility`, …) están obligados por la spec a ignorar campos desconocidos, por lo que el mismo archivo de skill funciona en todas partes sin forks.
</Tip>

### Como submódulo de git

Si quieres versiones fijadas en tu propio repo:

```bash
git submodule add https://github.com/veniceai/skills.git vendor/venice-skills
```

Luego haz symlink o copia los subconjuntos que quieras a la ruta de skills de tu agente.

## Cómo las cargan los agentes

El agente descubre cada `SKILL.md` por su `name` y `description` en el frontmatter. Cuando el usuario pide algo que coincide con el propósito de una skill, el agente carga ese único archivo en el contexto (no el catálogo completo), de modo que el prompt se mantiene pequeño y la respuesta precisa.

Por ejemplo, un agente que necesite generar música cargará `venice-audio-music` y sabrá inmediatamente:

- Que la música pasa por el ciclo queue/retrieve/complete, no por un endpoint síncrono
- Qué modelos están disponibles y su precio por minuto
- Cómo llamar a `/audio/quote` para estimar el coste primero
- Cómo debería ser el backoff de polling

Sin la skill, el agente podría intentar llamar a `/audio/speech` para música y obtener una respuesta inútil.

## Crear una nueva skill

1. Copia `template/` a `skills/<your-skill>/`.
2. Rellena el frontmatter y el cuerpo. Mantén `description` concreto, ya que es lo que el agente usa para decidir cuándo cargar la skill.
3. Enlaza skills relacionadas al final para navegación cruzada.
4. Abre un PR contra [`veniceai/skills`](https://github.com/veniceai/skills).

Consulta el `CONTRIBUTING.md` del repositorio para las convenciones de estilo (primer párrafo corto, tablas de endpoints explícitas, ejemplo en curl + un SDK, sección de "trampas", ≤ 500 líneas).

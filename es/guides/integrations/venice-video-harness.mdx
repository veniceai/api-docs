El [Venice Video Harness](https://github.com/jordanurbs/venice-video-harness) es un toolkit comunitario, agent-first y optimizado para Venice para la **creación de video con consistencia por delante a cualquier duración**. Convierte un agente de IDE (Claude Code, Cursor, Codex, etc.) en operador de un sistema reutilizable de producción de Venice que cubre más de 50 modelos de video, imagen, audio y música de Venice.

## Qué es esto

La mayoría de integraciones con Venice son envoltorios finos sobre llamadas a la API. El Venice Video Harness es la **capa superior** que se sitúa entre tu agente y la API de Venice:

- **Reglas de orquestación** en `CLAUDE.md`
- **Playbooks reutilizables** en `.claude/commands/` (19 comandos de flujo)
- **Agentes especializados** en `.claude/agents/` (art-director, prompt-engineer, cut-qa y más)
- **Skills de producción de Venice** en `.claude/skills/` (compatible con el formato [Agent Skills](/es/guides/integrations/venice-skills))
- **Capa de ejecución en TypeScript** en `src/`
- **Registro completo de modelos** que abarca más de 50 modelos de video, imagen, audio y música de Venice

Pensado para creadores que producen:

- Proyectos de video con personajes consistentes (cualquier género, cualquier duración)
- Series o campañas con estilo visual fijado
- Flujos de storyboard a video
- Contenido narrativo de formato corto y largo
- Secuencias cinemáticas de marca, trailers y teasers
- Series sociales con personajes recurrentes

## Qué es lo optimizado para Venice

- **Prompts de imagen afinados para los modelos de imagen de Venice** como `seedream-v5-lite`, `nano-banana-pro`, `flux-2-pro/max` y más
- **Generación de paneles en dos pasadas** con refinado por multi-edit de Venice para corrección de personajes
- **Lógica de enrutamiento de modelos** para niveles de acción, atmósfera y consistencia de personajes
- **Generación de video consciente de referencias** que usa correctamente `elements`, `reference_image_urls` y `scene_image_urls` por modelo
- **Adaptación de prompts consciente del entorno** para manejo de escenas de día vs. de noche
- **Rutas de audio nativas de Venice** para TTS (Kokoro, Qwen3, ElevenLabs), SFX y música
- **Estimación de coste** antes de la generación vía `/video/quote` y `/audio/quote`
- **Construcción de parámetros consciente del modelo** que omite automáticamente los parámetros no admitidos por el modelo destino

## Defaults de enrutamiento de modelos

Los defaults del harness son opinados porque la consistencia es el punto. El enrutamiento actual (abril 2026):

**Seedance 2.0 R2V por defecto. Fallback a Kling O3 R2V para escenas de 3+ personajes. Seedance 2.0 i2v para planos de establecimiento.**

| Rol | Modelo por defecto | Cuándo se usa |
|---|---|---|
| Planos de personajes (1-2 personajes) | `seedance-2-0-reference-to-video` | R2V por defecto con `reference_image_urls` planos, tags `@Image`, hasta 15s, audio estéreo nativo |
| Planos de personajes (3+ personajes) | `kling-o3-standard-reference-to-video` | Fallback automático con `elements` estructurados para identidad multi-personaje |
| Establecimiento / mood / acción | `seedance-2-0-image-to-video` | Sin personajes; calidad cinemática épica, hasta 15s |

Son anulables por proyecto vía `series.json → videoDefaults`. Para apuntar a una familia no-Seedance (p. ej., cuentas sin acceso a Seedance), pon `videoDefaults` en `kling-o3-standard-reference-to-video` y `veo3.1-fast-image-to-video`.

<Note>
**Regla de rostros de Seedance:** Seedance 2.0 bloquea imágenes de entrada con rostros que no fueron producidas por `seedream-v5-lite` o `seedream-v5-lite-edit`. El harness lo gestiona automáticamente enrutando el trabajo con personajes a través de Seedream y ejecutando una verificación previa antes de cada llamada a Seedance.
</Note>

## Pipelines de producción

### Pipeline de generación

Video narrativo de extremo a extremo (guion → storyboard → video → audio → montaje):

```bash
npm run dev -- produce-episode -p output/my-series -e 1
```

Implementación de referencia en `src/mini-drama/` cubre:

- Gestión de series / personajes / episodios
- Workshop de guion impulsado por LLM
- Generación de storyboard en dos pasadas (generar + multi-edit refine)
- QA de paneles basada en visión
- Generación de video con encadenado de fotogramas
- Postproducción de audio por capas
- Quemado de subtítulos y montaje final

### Pipeline de edición

Corta medios ya existentes (planos generados con Venice o footage real). **Text-first**: el LLM lee un `takes_packed.md` compacto (~12 KB por 40 min de audio) en lugar de volcar fotogramas.

Los cinco pasos:

<Steps>
  <Step title="Transcribe">
    Whisper.cpp local produce `*.words.json` por fuente + `takes_packed.md`.
  </Step>
  <Step title="Lee el pack">
    El LLM forma una estrategia de corte solo desde el texto.
  </Step>
  <Step title="Confirma">
    Propone la estrategia y espera "yes / revise / cancel".
  </Step>
  <Step title="Renderiza el EDL">
    Lista de cortes JSON → concat de ffmpeg con fades de audio de 30 ms. Archivo primero, los originales nunca se sobrescriben.
  </Step>
  <Step title="Auto-evalúa">
    El agente `cut-qa` ejecuta 6 comprobaciones programáticas en cada corte; máximo 3 iteraciones de arreglo.
  </Step>
</Steps>

Las comprobaciones de `cut-qa` detectan regresiones de aspect ratio, saltos de hash de fotograma dentro de una palabra, truncamiento de VO, discontinuidad de iluminación, picos de audio por encima de -6 dBFS y solapamiento de subtítulos con texto en pantalla.

<Tip>
El pipeline de edición se inspira en [browser-use/video-use](https://github.com/browser-use/video-use). Su idea clave, *"el LLM nunca ve el video, lo lee"*, es lo que hace que la edición dirigida por agente funcione sin ahogarse en tokens de volcado de fotogramas.
</Tip>

## Comandos, agentes y skills

El harness expone 19 comandos de flujo, 10 agentes especializados y 7 skills de producción. Destacados:

| Comando de flujo | Propósito |
|---|---|
| `new-series` | Crea una nueva serie con estética fijada |
| `add-character` / `lock-character` | Fijar personaje + voz |
| `workshop-episode` | Guionización colaborativa de episodio |
| `storyboard-episode` | Storyboard de un episodio |
| `produce-episode` | Pipeline completo en un comando |
| `generate-trailer` | Pipeline completo de trailer |
| `edit-footage` | Pipeline de edición text-first para medios existentes |
| `ingest-screenplay` | Ingiere un guion Fountain o PDF |

| Agente especializado | Rol |
|---|---|
| `art-director` | Decisiones de estética, paleta, iluminación y composición |
| `prompt-engineer` | Prompts de imagen Venice, consistencia de personajes |
| `storyboard-qa` | QA de paneles para continuidad y comprobaciones de personajes |
| `cut-qa` | Puerta de calidad post-render (6 comprobaciones por corte, máx. 3 iteraciones) |
| `overlay-designer` | Gráficos en movimiento de marca, sub-agentes paralelos |
| `trailer-curator` | Selección de planos para trailer y reglas anti-spoiler |

| Skill de producción | Propósito |
|---|---|
| `venice-api` | Uso y defaults de la API REST de Venice |
| `venice-video-model-routing` | Enrutamiento R2V-first, árboles de decisión |
| `character-consistency` | Guía de consistencia de personajes entre planos |
| `shot-composition` | Guía de composición de plano y cámara |
| `screenplay-parsing` | Flujos de parseo de guion |
| `video-editing` | Filosofía text-first de edición, formato EDL, bucle cut-qa |

## Ida y vuelta con NLE

Tras renderizar, exporta el timeline montado como XML para afinar en tu editor preferido. Cada segmento de video, clip de diálogo, clip de SFX y cue musical cae en su propia pista.

```bash
mini-drama export-timeline -p output/<project> -e 1 --format fcpxml      # Final Cut Pro X
mini-drama export-timeline -p output/<project> -e 1 --format premiere    # Premiere Pro
mini-drama export-timeline -p output/<project> -e 1 --format davinci     # DaVinci Resolve
```

## Uso programático

También puedes llamar directamente a los módulos del harness desde tu propio TypeScript:

```typescript
import { VeniceClient } from './src/venice/client.js';
import { generateVideo, quoteVideo } from './src/venice/video.js';
import { listVideoModels } from './src/venice/models.js';

const client = new VeniceClient();

const quote = await quoteVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  duration: '8s',
  audio: true,
});
console.log(`Estimated cost: $${quote.quote}`);

const result = await generateVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  prompt: 'A slow dolly shot pushes forward...',
  duration: '8s',
  imageUrl: 'data:image/png;base64,...',
  audio: true,
  outputPath: 'output/shot-001.mp4',
});

const longModels = listVideoModels({ minDurationSec: 20 });
```

El [Servidor MCP de Venice](https://github.com/veniceai/venice-mcp-server) es el servidor oficial de [Model Context Protocol](https://modelcontextprotocol.io/) para Venice. Expone la API completa de Venice (chat, imagen, video, audio, música, embeddings, augment web y personajes) como **31 herramientas** que cualquier agente compatible con MCP puede invocar.

## Inicio rápido

<Steps>
  <Step title="Obtén una API key de Venice">
    Genera una en [venice.ai/settings/api](https://venice.ai/settings/api). Consulta la [guía de API key](/es/guides/getting-started/generating-api-key) para instrucciones paso a paso.
  </Step>

  <Step title="Añade Venice a la config de tu host MCP">
    Coloca esto en el archivo de configuración de tu host MCP:

    ```json
    {
      "mcpServers": {
        "venice": {
          "command": "npx",
          "args": ["-y", "@veniceai/mcp-server@0.2.0"],
          "env": { "VENICE_API_KEY": "<your-venice-api-key>" }
        }
      }
    }
    ```

    Rutas de configuración comunes:

    | Host | Ruta |
    |---|---|
    | Claude Desktop (macOS) | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Claude Desktop (Windows) | `%APPDATA%\Claude\claude_desktop_config.json` |
    | Cursor | `~/.cursor/mcp.json` |
    | LM Studio | `mcp.json` (desde los ajustes MCP de la app) |
  </Step>

  <Step title="Reinicia tu host MCP">
    Tu agente ahora tiene disponibles chat, imagen, video, música, TTS, ASR y 25 herramientas más de Venice.
  </Step>
</Steps>

<Note>
La mayoría de hosts MCP solo pasan las variables de entorno **listadas explícitamente** en el bloque `env`. Las variables de entorno a nivel de sistema no se heredan. Si ves errores 402 con una API key configurada, comprueba que `VENICE_API_KEY` esté dentro de `env` en tu configuración.
</Note>

## Qué obtienes

**31 herramientas** que abarcan todas las modalidades de Venice, **3 recursos** (`venice://models`, `venice://styles`, `venice://voices`) y **3 plantillas de prompt**.

### Chat y embeddings

| Herramienta | Descripción |
|---|---|
| `venice_chat` | Chat completion compatible con OpenAI contra el catálogo completo de LLMs de Venice. |
| `venice_responses` | API de Responses compatible con OpenAI con soporte de herramientas mono o multi-turno. |
| `venice_embeddings` | Calcula embeddings para texto de entrada. |
| `venice_chat_with_character` | Chatea con un personaje de Venice por slug. |

### Imagen

| Herramienta | Descripción |
|---|---|
| `venice_image_generate` | Genera una imagen (Flux 2, Lustify SDXL, Anime/WAI, Qwen Image, GPT Image, Nano Banana Pro y más). |
| `venice_image_edit` | Edita una imagen con un prompt. |
| `venice_image_multi_edit` | Edita varias imágenes juntas con un único prompt. |
| `venice_image_upscale` | Escala una imagen hasta 4×. |
| `venice_image_remove_bg` | Elimina el fondo de una imagen. |
| `venice_image_styles` | Lista los presets de estilo de imagen. |

### Video

| Herramienta | Descripción |
|---|---|
| `venice_video_generate` | Encola una generación de video (Sora 2, Veo 3.1, Kling, Wan, LTX 2, Seedance, Runway Gen-4 y más). |
| `venice_video_status` | Comprueba el estado de un trabajo de video en cola. |
| `venice_video_complete` | Marca un video terminado como descargado; elimina los medios en servidor. |
| `venice_video_transcriptions` | Transcribe la URL de un video de YouTube. |
| `venice_video_quote` | Obtén una cotización de precio antes de encolar. |

### Audio (TTS / ASR)

| Herramienta | Descripción |
|---|---|
| `venice_tts` | Texto a voz con voces clonadas y etiquetas de emoción. |
| `venice_asr` | Transcribe audio desde una URL. |
| `venice_voice_clone` | Lista las voces integradas o clona una voz a partir de una muestra. |
| `venice_audio_quote` | Obtén una cotización para generación de música. |

### Música

| Herramienta | Descripción |
|---|---|
| `venice_music_generate` | Encola generación de música (`ace-step-15`, `elevenlabs-music`, `minimax-music-v2/v25/v26`, `stable-audio-25`, `mmaudio-v2`, `elevenlabs-sound-effects-v2`). |
| `venice_music_status` | Comprueba el estado de un trabajo de música en cola. |
| `venice_music_complete` | Marca un trabajo de música terminado como descargado. |

### Web augment, catálogo y cripto

| Herramienta | Descripción |
|---|---|
| `venice_web_search` | Busca en la web (con Firecrawl detrás). |
| `venice_web_scrape` | Scrapea una URL a markdown. |
| `venice_text_parser` | Extrae texto de PDF/DOCX/EPUB/PPTX/XLSX. |
| `venice_list_models` | Lista el catálogo de modelos en vivo con precios. |
| `venice_list_characters` | Lista los personajes públicos de Venice. |
| `venice_crypto_rpc` | Proxy de llamadas JSON-RPC a Base, Ethereum, Polygon, Arbitrum u Optimism. |

### Helpers de cartera x402

Solo relevante si autenticas con cartera vía [x402](/es/guides/integrations/x402-venice-api) en lugar de una API key.

| Herramienta | Descripción |
|---|---|
| `venice_x402_balance` | Comprueba el saldo prepagado de x402 para una dirección de cartera EVM o Solana. |
| `venice_x402_top_up_info` | Obtén los requisitos de recarga (red, token USDC, receptor, importe mínimo). |
| `venice_x402_transactions` | Lista las transacciones recientes de recarga y débito x402 para una dirección de cartera EVM o Solana. |

## Configuración

El servidor se configura por completo mediante variables de entorno.

| Variable | Predeterminado | Notas |
|---|---|---|
| `VENICE_API_KEY` | _(ninguno)_ | Tu API key de Venice. La configuración más simple. |
| `VENICE_DEFAULT_CHAT_MODEL` | `venice-uncensored` | |
| `VENICE_DEFAULT_IMAGE_MODEL` | `flux-2-pro` | |
| `VENICE_DEFAULT_TTS_MODEL` | `tts-kokoro` | |
| `VENICE_DEFAULT_ASR_MODEL` | `openai/whisper-large-v3` | |
| `VENICE_DISABLE_NSFW` | `0` | Pon a `1` para eliminar las notas de capacidad NSFW de las descripciones de herramientas. |
| `VENICE_HTTP_TIMEOUT_MS` | `60000` | |
| `VENICE_SIWX_TOKEN` | _(ninguno)_ | Token de auth con cartera x402. Consulta [x402 abajo](#x402-wallet-mode). |

Si se configuran tanto `VENICE_API_KEY` como `VENICE_SIWX_TOKEN`, prevalece la API key.

## Modo cartera x402

Venice admite autenticación con un [token Sign-In-With-X de cartera](/es/guides/integrations/x402-venice-api) respaldado por crédito USDC prepagado en Base o Solana, además del flujo normal de API key. Sin email, teléfono ni KYC: tu cartera es la única identidad.

```json
{
  "mcpServers": {
    "venice": {
      "command": "npx",
      "args": ["-y", "@veniceai/mcp-server@0.2.0"],
      "env": { "VENICE_SIWX_TOKEN": "<base64 Sign-In-With-X payload>" }
    }
  }
}
```

El servidor MCP reenvía `VENICE_SIWX_TOKEN` como cabecera `X-Sign-In-With-X` en cada llamada a la API de Venice. El servidor nunca ve tu clave privada. La firma de la cartera y las autorizaciones de recarga USDC ocurren en tu propia cartera.

| Flujo | Qué pasa |
|---|---|
| **Configuración única** | Firma un mensaje Sign-In-With-X en tu cartera → produce un token SIWX (JSON en base64). |
| **Recarga** | `POST /api/v1/x402/top-up` devuelve 402 + requisitos de pago. Firma un pago USDC para una de las opciones de Base o Solana devueltas, reenvía y Venice acredita tu saldo. |
| **Cada llamada de inferencia** | El servidor MCP envía `X-Sign-In-With-X: <SIWX>`; Venice descuenta de tu saldo prepagado. |

La recarga mínima es de **\$5 USD**. El saldo mínimo para llamar a inferencia es de **\$0,10**. Una vez recargado, las llamadas son sub-100 ms porque la liquidación ocurre off-chain en una cuenta de crédito rápida.

<Tip>
Las carteras vinculadas a una cuenta de Venice con DIEM en staking consumen del saldo de staking en lugar de los créditos USDC, por lo que no se necesita recarga.
</Tip>

## Auto-hospedaje (Streamable HTTP)

Para despliegues de equipo o workspace, ejecuta el servidor MCP sobre HTTP en lugar de stdio:

```bash
docker run -p 3333:3333 \
  -e VENICE_API_KEY=<your-venice-api-key> \
  -e VENICE_MCP_AUTH_TOKEN=<choose-a-long-random-token> \
  ghcr.io/veniceai/venice-mcp-server:latest
```

El servidor estará ahora disponible en `http://localhost:3333/mcp`. Los clientes HTTP deben enviar `Authorization: Bearer <VENICE_MCP_AUTH_TOKEN>`.

<Warning>
`/mcp` es un endpoint de ejecución de herramientas respaldado por credenciales: los llamadores pueden gastar la API key de Venice configurada o el saldo x402. Cuando el modo HTTP se enlaza fuera de loopback, el arranque falla salvo que se configure `VENICE_MCP_AUTH_TOKEN`. Para producción, fija la versión del paquete npm explícitamente en lugar de depender de `latest`.
</Warning>

[Claude Code](https://docs.anthropic.com/en/docs/claude-code) es la herramienta CLI de Anthropic para programación agéntica. Esta guía te muestra cómo ejecutarla a través de Venice AI para acceso pago-por-token a Claude Opus 4.5/4.6 y Sonnet 4.5/4.6.

## Por qué necesitas un router

Claude Code se conecta directamente a la API de Anthropic por defecto. Para usarlo con Venice, necesitas [claude-code-router](https://github.com/musistudio/claude-code-router), un proxy local open-source que:

<Steps>
  <Step title="Intercepta" icon="hand">
    Captura las solicitudes salientes de Claude Code antes de que lleguen a Anthropic
  </Step>
  <Step title="Transforma" icon="arrows-rotate">
    Convierte el formato de la solicitud y mapea los IDs de modelo (p. ej., `claude-opus-4-5`)
  </Step>
  <Step title="Redirige" icon="route">
    Reenvía las solicitudes a Venice en `api.venice.ai/api/v1/chat/completions`
  </Step>
</Steps>

## Configuración

<Steps>
  <Step title="Instala Claude Code">
    Si aún no lo has hecho, instala la CLI de Claude Code de Anthropic:

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="Instala el router">
    ```bash
    npm install -g @musistudio/claude-code-router
    ```
  </Step>

  <Step title="Obtén tu API key">
    Genera una clave desde [venice.ai/settings/api](https://venice.ai/settings/api). La pegarás directamente en el archivo de configuración en el siguiente paso.
  </Step>

  <Step title="Crea la configuración">
    Crea el directorio de configuración:

    ```bash
    mkdir -p ~/.claude-code-router
    ```

    Luego crea `~/.claude-code-router/config.json` con tu editor preferido:

    ```bash
    # Con nano
    nano ~/.claude-code-router/config.json

    # O con VS Code
    code ~/.claude-code-router/config.json
    ```

    Pega la siguiente configuración:

    ```json
    {
      "APIKEY": "",
      "LOG": true,
      "LOG_LEVEL": "info",
      "API_TIMEOUT_MS": 600000,
      "HOST": "127.0.0.1",
      "Providers": [
        {
          "name": "venice",
          "api_base_url": "https://api.venice.ai/api/v1/chat/completions",
          "api_key": "your-venice-api-key-here",
          "models": [
            "claude-opus-4-5",
            "claude-sonnet-4-5",
            "claude-opus-4-6",
            "claude-opus-4-6-fast",
            "claude-sonnet-4-6"
          ],
          "transformer": {
            "use": ["anthropic"]
          }
        }
      ],
      "Router": {
        "default": "venice,claude-opus-4-5",
        "think": "venice,claude-opus-4-5",
        "background": "venice,claude-opus-4-5",
        "longContext": "venice,claude-opus-4-5",
        "longContextThreshold": 100000
      }
    }
    ```

    <Note>
    Si modificas `config.json` mientras el router está ejecutándose, reinícialo con `ccr restart` para aplicar los cambios.
    </Note>
  </Step>

  <Step title="Inicia">
    Arranca el router y luego Claude Code:

    ```bash
    ccr start
    ccr code
    ```

    O usa el método de activación:

    ```bash
    eval "$(ccr activate)" && claude
    ```
  </Step>
</Steps>

## Modelos compatibles

| Modelo | ID en Venice | Ideal para |
|-------|-----------|----------|
| Claude Opus 4.5 | `claude-opus-4-5` | Razonamiento complejo, refactorizaciones grandes |
| Claude Sonnet 4.5 | `claude-sonnet-4-5` | Iteración rápida, programación diaria |
| Claude Opus 4.6 | `claude-opus-4-6` | Razonamiento complejo, refactorizaciones grandes |
| Claude Opus 4.6 Fast | `claude-opus-4-6-fast` | Razonamiento complejo con menor latencia |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | Iteración rápida, programación diaria |

<Info>
Claude Code está optimizado para modelos Claude. Aunque otros modelos disponibles a través de Venice (GPT, DeepSeek, Grok, etc.) pueden funcionar, no podemos garantizar una experiencia equivalente porque Claude Code se apoya en funciones específicas de Claude como el extended thinking. Para otros modelos, considera usar la [API estándar](/es/api-reference/endpoint/chat/completions) de Venice.
</Info>

## Funcionalidades del router

El router proporciona varias funcionalidades útiles más allá del enrutamiento básico:

<AccordionGroup>
  <Accordion title="Cambia de modelo al vuelo">
    Usa el comando `/model` dentro de Claude Code para cambiar de modelo sin reiniciar:
    
    ```
    /model venice,claude-sonnet-4-5
    ```
    
    Útil cuando quieres Opus para tareas complejas y Sonnet para iteraciones rápidas.
  </Accordion>

  <Accordion title="Configuración visual con modo UI">
    ¿Prefieres una GUI? Abre el editor de configuración basado en web:
    
    ```bash
    ccr ui
    ```
    
    Esto abre una interfaz de navegador para editar tu `config.json` sin tocar el archivo directamente.
  </Accordion>

  <Accordion title="Escenarios del router explicados">
    La sección `Router` de la configuración controla qué modelo gestiona distintos tipos de tareas:
    
    | Escenario | Cuándo se usa |
    |----------|----------------|
    | `default` | Solicitudes generales |
    | `think` | Tareas con mucho razonamiento (modo Plan) |
    | `background` | Operaciones en segundo plano |
    | `longContext` | Cuando el contexto supera `longContextThreshold` tokens |
    
    Puedes enrutar escenarios distintos a modelos distintos. Por ejemplo, usar Sonnet para tareas en segundo plano para ahorrar costes.
  </Accordion>

  <Accordion title="Depuración con logs">
    Si algo no funciona, revisa los logs:
    
    ```bash
    # Logs del servidor (HTTP, llamadas API)
    ~/.claude-code-router/logs/ccr-*.log
    
    # Logs de aplicación (decisiones de enrutamiento)
    ~/.claude-code-router/claude-code-router.log
    ```
    
    Establece `"LOG_LEVEL": "debug"` en tu configuración para una salida más detallada.
  </Accordion>
</AccordionGroup>

## Comportamiento de caché

El [prompt caching](/es/guides/features/prompt-caching) de Venice funciona junto con los marcadores de caché nativos de Claude Code. Venice detecta automáticamente cuándo Claude Code envía campos `cache_control` y ajusta su estrategia de caché en consecuencia.

| Escenario | TTL de caché | Quién controla |
|----------|-----------|--------------|
| Predeterminado (recomendado) | 5 minutos | Claude Code + Venice |
| Con transformer `cleancache` | 1 hora | Solo Venice |

<AccordionGroup>
  <Accordion title="Cuándo NO usar cleancache (la mayoría de los usuarios)">
    La configuración predeterminada permite que ambos sistemas cooperen:
    
    - Claude Code envía sus marcadores nativos `cache_control`
    - Venice añade caché a su alrededor con TTL de 5 minutos
    - Ambos sistemas comparten el límite de 4 bloques de caché
    
    Funciona bien para sesiones activas de programación donde haces solicitudes frecuentes.
  </Accordion>

  <Accordion title="Cuándo usar cleancache">
    Añade `cleancache` al transformer si:
    
    - Encuentras errores del límite de 4 bloques de caché
    - Experimentas comportamiento extraño de caché
    - Prefieres el TTL de 1 hora de Venice para sesiones más largas
    
    ```json
    "transformer": {
      "use": ["anthropic", "cleancache"]
    }
    ```
    
    Esto elimina los marcadores de caché de Claude Code, dando a Venice control total con un TTL más largo.
  </Accordion>
</AccordionGroup>

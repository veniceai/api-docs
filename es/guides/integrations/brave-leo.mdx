[Brave Leo](https://brave.com/leo/) es el asistente de IA integrado en Brave Browser. La función Bring Your Own Model (BYOM) de Leo puede conectarse directamente al endpoint de chat completions compatible con OpenAI de Venice, permitiéndote usar modelos de Venice para chat y asistencia contextual con la página sin instalar ninguna extensión.

## Requisitos previos

- Brave Browser 1.69 o posterior en Windows, macOS o Linux
- Una API key de Venice desde [venice.ai/settings/api](https://venice.ai/settings/api)
- Un ID de modelo del [catálogo de modelos de texto de Venice](/models/text)

<Note>
La función BYOM de Brave está disponible en escritorio. El ID del modelo debe coincidir exactamente con un ID de modelo de Venice.
</Note>

## Configuración

<Steps>
  <Step title="Abre la configuración de Leo">
    Abre `brave://settings/leo-ai`, o abre Leo desde la barra lateral de Brave, selecciona el icono de configuración y navega a **Leo**.
  </Step>

  <Step title="Añade un modelo personalizado">
    Desplázate hasta **Bring your own model** y selecciona **Add new model**.
  </Step>

  <Step title="Configura el endpoint de Venice">
    Introduce los siguientes valores:

    | Campo | Valor |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | Un ID de modelo del [catálogo de modelos de texto](/models/text) |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | Tu API key de Venice |

    Si Brave muestra **Context size**, ajústalo a la ventana de contexto soportada por el modelo seleccionado. También puedes añadir un system prompt opcional.

    Introduce la API key directamente, sin el prefijo `Bearer`. Brave añade la cabecera de autorización a cada solicitud.
  </Step>

  <Step title="Guarda y selecciona Venice">
    Selecciona **Add model** y luego establece **Venice AI** como el modelo predeterminado para nuevas conversaciones o elígelo desde el selector de modelos de Leo.
  </Step>

  <Step title="Prueba la conexión">
    Abre Leo en una página web y envía un prompt como:

    ```txt
    Summarize this page in five bullet points.
    ```

    Leo envía la conversación y el contenido relevante de la página directamente al endpoint de Venice.
  </Step>
</Steps>

## Habilita la búsqueda web de Venice

Brave no expone parámetros específicos de Venice en el cuerpo de la solicitud en BYOM. Para habilitar la búsqueda web de Venice, añade un [sufijo de característica de modelo](/api-reference/endpoint/chat/model_feature_suffix) al ID del modelo en **Model request name**:

```txt
<model-id>:enable_web_search=auto
```

Para habilitar siempre la búsqueda e incluir citas, usa:

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
La búsqueda web tiene precios adicionales basados en uso. Consulta [precios de la API](/overview/pricing) para conocer las tarifas actuales.
</Info>

## Privacidad

Cuando usas un endpoint BYOM remoto, Brave se conecta directamente a ese proveedor. Las solicitudes evitan los servidores de Brave, por lo que las protecciones de proxy y retención del lado del servidor de Brave no se aplican a esas solicitudes.

Venice no almacena ni registra el contenido de los prompts y respuestas en su proxy. El modelo seleccionado determina el nivel de privacidad en tiempo de ejecución:

- Los modelos **Private** utilizan infraestructura de retención de datos cero.
- Los modelos **Anonymized** ocultan la información identificativa del proveedor upstream, pero el proveedor puede procesar el prompt.
- Los modelos **TEE** y **E2EE** añaden aislamiento respaldado por hardware o cifrado del lado del cliente donde se admita.

Comprueba la etiqueta de privacidad del modelo en el [catálogo de modelos de texto](/models/text) y lee [privacidad de Venice](/overview/privacy) antes de enviar información sensible.

Brave puede mantener el historial de chat de Leo en el almacenamiento local del navegador. Puedes borrar los datos de Leo, usar un chat temporal o deshabilitar el historial de chat local en la configuración de Leo de Brave.

## Notas de configuración

- Brave requiere la ruta completa del endpoint: `https://api.venice.ai/api/v1/chat/completions`.
- Los IDs y la disponibilidad de los modelos cambian con el tiempo. Copia el ID actual desde el [catálogo de modelos de texto](/models/text).
- El tamaño de contexto no debe exceder la ventana de contexto soportada por el modelo seleccionado.
- Los resúmenes de páginas pueden enviar el contenido de la página a Venice como parte del prompt.
- Las solicitudes remotas a Venice requieren una conexión a internet y consumen créditos de la API de Venice o tu asignación DIEM disponible.

## Solución de problemas

<AccordionGroup>
  <Accordion title="Error 401 o de autenticación">
    Confirma que la clave proviene de [venice.ai/settings/api](https://venice.ai/settings/api). Pega la clave sin procesar, sin `Bearer` y elimina cualquier espacio inicial o final.
  </Accordion>
  <Accordion title="Modelo no encontrado">
    Copia el ID del modelo exactamente desde el [catálogo de modelos de texto](/models/text). Si añadiste un sufijo de característica, primero prueba el modelo sin el sufijo.
  </Accordion>
  <Accordion title="Error de red o sin respuesta transmitida">
    Confirma que el endpoint del servidor incluye `/api/v1/chat/completions`. Brave Leo espera una respuesta de streaming compatible con OpenAI.
  </Accordion>
  <Accordion title="Los resúmenes de página carecen de contexto de la página">
    Comprueba el permiso de Leo para usar el contenido de la página y asegúrate de que la página actual sea accesible para el navegador. Las páginas internas de Brave y algunas páginas protegidas no se pueden resumir.
  </Accordion>
</AccordionGroup>

[Aider](https://aider.chat/) es una herramienta de programación en pareja con IA que edita tu repositorio git local desde la terminal. Venice funciona con el soporte de [API compatible con OpenAI](https://aider.chat/docs/llms/openai-compat.html) de Aider: establece la URL base y la API key de Venice, y luego antepón `openai/` al ID del modelo.

## Requisitos previos

- Una API key de Venice desde [venice.ai/settings/api](https://venice.ai/settings/api)
- Python 3.9+ disponible en tu máquina
- Un repositorio git donde trabajar

---

## Configuración

<Steps>
  <Step title="Instala Aider">
    Instala Aider con el instalador oficial:

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    Consulta la [documentación de instalación de Aider](https://aider.chat/docs/install.html) si prefieres otro método de instalación.
  </Step>

  <Step title="Configura Venice como el endpoint de OpenAI">
    Apunta Aider a Venice estableciendo la URL base compatible con OpenAI y tu API key de Venice.

    **Mac / Linux:**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell):**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (Símbolo del sistema, persistente):**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    Reinicia la terminal después de `setx` para que las nuevas variables estén disponibles.
  </Step>

  <Step title="Inicia Aider con un modelo de Venice">
    Desde el directorio de tu proyecto, antepón `openai/` al ID del modelo de Venice:

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    Otros buenos modelos para empezar:

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    Explora todos los IDs de modelos de texto en el [catálogo de modelos](/models/text).
  </Step>
</Steps>

## Persistir la configuración con `.env`

Para una configuración local del proyecto, crea un archivo `.env` en la raíz de tu repositorio git:

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

Después inicia Aider normalmente:

```bash
aider
```

Aider también lee `.env` desde tu directorio home. Los valores de la raíz del repositorio tienen prioridad sobre los del directorio home. Mantén este archivo fuera de git: añade `.env` a `.gitignore` si aún no está ignorado.

<Info>
También puedes establecer `AIDER_OPENAI_API_BASE` y `AIDER_OPENAI_API_KEY`. La documentación oficial de Aider sigue documentando `OPENAI_API_BASE` y `OPENAI_API_KEY` para proveedores compatibles con OpenAI.
</Info>

---

## Persistir la configuración con YAML

Puedes almacenar la misma configuración en `.aider.conf.yml` en la raíz de tu repositorio o en tu directorio home:

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

Después ejecuta:

```bash
aider
```

## Verifica la configuración

Después de que Aider arranque, pídele algo pequeño que demuestre que el modelo está accesible:

```txt
Reply with one sentence confirming you are running through Venice.
```

Si el endpoint y la clave son correctos, Aider transmitirá una respuesta desde el modelo de Venice seleccionado.

También puedes confirmar la API key fuera de Aider:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## Elección de modelos

| Caso de uso                       | ID de modelo de ejemplo | Flag de Aider                      |
| --------------------------------- | ----------------------- | ---------------------------------- |
| Programación / refactorización potente | `zai-org-glm-5-1`   | `--model openai/zai-org-glm-5-1`   |
| Iteración rápida                  | `minimax-m25`           | `--model openai/minimax-m25`       |
| Programación con contexto largo   | `kimi-k2-5`             | `--model openai/kimi-k2-5`         |
| Chat sin censura                  | `venice-uncensored`     | `--model openai/venice-uncensored` |

Antepón siempre `openai/` a los IDs de modelos de Venice al lanzar Aider. Sin ese prefijo, Aider no enrutará la solicitud por la ruta del proveedor compatible con OpenAI.

Descubre los IDs de modelos actuales en tiempo de ejecución:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

O consulta [GET /models/traits](/api-reference/endpoint/models/traits) para valores por defecto como `text:default`.

## Notas de configuración

- Mantén `OPENAI_API_BASE` establecido en `https://api.venice.ai/api/v1`. No añadas `/chat/completions`; Aider añade la ruta del chat por sí mismo.
- Usa una API key de Venice en `OPENAI_API_KEY`. No uses una clave de OpenAI con la URL base de Venice.
- Aider puede advertir que el modelo tiene una ventana de contexto o coste desconocidos. Esa advertencia es esperada para muchos IDs de modelos de Venice y normalmente se puede ignorar. Consulta [advertencias de modelo de Aider](https://aider.chat/docs/llms/warnings.html).
- Prefiere modelos capaces de programar/razonar para ediciones multiarchivo. Los modelos más pequeños o no orientados a código pueden tener problemas con los formatos de edición de Aider.

<Warning>
Establecer `OPENAI_API_BASE` y `OPENAI_API_KEY` en tu shell también afecta a otras herramientas que leen esas variables. Prefiere un `.env` de proyecto o `.aider.conf.yml` si usas OpenAI en otras partes del mismo entorno.
</Warning>

## Solución de problemas

<AccordionGroup>
  <Accordion title="Error 401 o de autenticación">
    Confirma que `OPENAI_API_KEY` es una clave de Venice desde [venice.ai/settings/api](https://venice.ai/settings/api), sin espacios en blanco adicionales. En Windows, después de `setx`, reinicia la terminal.
  </Accordion>
  <Accordion title="Modelo no encontrado o respuesta vacía">
    Confirma que el ID del modelo existe en el [catálogo de modelos de texto](/models/text) y que has lanzado Aider con el prefijo `openai/`, por ejemplo `openai/zai-org-glm-5-1`.
  </Accordion>
  <Accordion title="Las solicitudes siguen yendo a OpenAI">
    Comprueba que `OPENAI_API_BASE` es exactamente `https://api.venice.ai/api/v1` en la misma sesión de shell. Imprímelo con `echo $OPENAI_API_BASE` (o `$env:OPENAI_API_BASE` en PowerShell).
  </Accordion>
  <Accordion title="Advertencia de ventana de contexto / costes desconocidos">
    Es esperado para muchos modelos de Venice. Aider recurre a valores predeterminados razonables. Para silenciarla, añade metadatos del modelo como se describe en [ajustes avanzados de modelo de Aider](https://aider.chat/docs/config/adv-model-settings.html).
  </Accordion>
</AccordionGroup>

[Jan](https://jan.ai/) es un asistente de IA de código abierto para ejecutar modelos locales y remotos desde una única app de escritorio. Venice funciona en Jan mediante su soporte de proveedores de modelos remotos compatibles con OpenAI.

## Requisitos previos

- Una API key de Venice desde [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) instalado y en funcionamiento en tu máquina
- Un ID de modelo de texto de Venice del [catálogo de modelos](/models/text)

---

## Configuración

<Steps>
  <Step title="Obtén tu API key de Venice">
    Genera o copia una API key desde [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Abre la configuración de proveedores de modelos de Jan">
    En Jan, abre **Settings** y ve a **Model Providers**.

    Elige el flujo de proveedor personalizado compatible con OpenAI de Jan. Dependiendo de tu versión de Jan, esto puede aparecer como **OpenAI** con **Custom Endpoint** activado o como un proveedor **Any OpenAI Compatible API**.
  </Step>

  <Step title="Configura el endpoint de Venice">
    Introduce los ajustes del proveedor de Venice:

    | Campo | Valor |
    |-------|-------|
    | Nombre del proveedor | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | Tu API key de Venice |
    | Tipo de API | Compatible con OpenAI |

    Guarda o aplica los ajustes del proveedor antes de añadir modelos.
  </Step>

  <Step title="Añade modelos de Venice">
    Añade los IDs de modelo de Venice que quieras usar. Jan requiere que el ID del modelo coincida exactamente con el proveedor.

    Buenas opciones para empezar:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Puedes añadir cualquier modelo de texto compatible del [catálogo de modelos](/models/text).
  </Step>

  <Step title="Inicia un chat">
    Abre un chat nuevo o existente en Jan, selecciona uno de tus modelos de Venice en el selector de modelos y envía un mensaje de prueba.

    Jan enviará la solicitud a Venice utilizando tu API key y la URL base compatible con OpenAI.
  </Step>
</Steps>

---

## Verifica la configuración

Envía un prompt corto desde Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

Si el proveedor está configurado correctamente, Jan debería transmitir una respuesta desde el modelo de Venice seleccionado.

También puedes verificar tu API key fuera de Jan con:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Notas de configuración

- Usa `https://api.venice.ai/api/v1` como URL base. No incluyas `/chat/completions`; Jan añade la ruta del endpoint para las solicitudes de chat.
- Los IDs de modelo deben coincidir exactamente con Venice. Si un modelo no aparece en el selector de Jan, añádelo manualmente usando el ID del modelo del [catálogo de modelos](/models/text).
- Mantén Venice configurado como proveedor compatible con OpenAI. No uses el proveedor nativo de Anthropic de Jan para los modelos Claude de Venice.
- Las solicitudes remotas a Venice requieren conexión a internet y consumen créditos de la API de Venice.

<Info>
Los modelos locales de Jan siguen disponibles después de añadir Venice. Puedes alternar entre modelos locales y de Venice desde el selector de modelos del chat.
</Info>

## Resolución de problemas

<AccordionGroup>
  <Accordion title="Error 401 o de autenticación">
    Comprueba que la API key sea una clave de Venice de [venice.ai/settings/api](https://venice.ai/settings/api), sin espacios extra antes o después.
  </Accordion>

  <Accordion title="404 o modelo no encontrado">
    Confirma que el ID del modelo en Jan coincide exactamente con un ID de modelo de Venice. Comprueba también que la URL base sea `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="No aparecen modelos de Venice en el selector">
    Añade el modelo manualmente en la configuración de proveedores de Jan. Usa el ID exacto del modelo, como `zai-org-glm-5-1`, y vuelve a guardar el proveedor.
  </Accordion>

  <Accordion title="Las solicitudes siguen yendo a otro proveedor">
    Asegúrate de que el chat esté usando un modelo asociado a tu proveedor de Venice. Si Jan está usando una entrada de modelo de OpenAI o Anthropic integrada, cambia al modelo de Venice que añadiste.
  </Accordion>
</AccordionGroup>

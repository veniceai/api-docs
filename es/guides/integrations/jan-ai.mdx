[Jan](https://jan.ai/) es un asistente de IA open-source para ejecutar modelos locales y remotos desde una sola app de escritorio. Venice funciona en Jan a través de su soporte de proveedor de modelos remoto compatible con OpenAI.

## Requisitos previos

- Una API key de Venice de [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) instalado y en ejecución en tu máquina
- Un ID de modelo de texto de Venice del [catálogo de modelos](/es/models/text)

---

## Configuración

<Steps>
  <Step title="Obtén tu API key de Venice">
    Genera o copia una API key desde [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Abre los ajustes de proveedor de modelos de Jan">
    En Jan, abre **Settings** y ve a **Model Providers**.

    Elige el flujo de proveedor personalizado compatible con OpenAI de Jan. Según tu versión de Jan, puede aparecer como **OpenAI** con **Custom Endpoint** activado, o como un proveedor **Any OpenAI Compatible API**.
  </Step>

  <Step title="Configura el endpoint de Venice">
    Introduce los ajustes del proveedor Venice:

    | Campo | Valor |
    |-------|-------|
    | Nombre del proveedor | `Venice AI` |
    | URL base | `https://api.venice.ai/api/v1` |
    | API key | Tu API key de Venice |
    | Tipo de API | Compatible con OpenAI |

    Guarda o aplica los ajustes del proveedor antes de añadir modelos.
  </Step>

  <Step title="Añade modelos de Venice">
    Añade los IDs de modelo de Venice que quieras usar. Jan requiere que el ID del modelo coincida exactamente con el proveedor.

    Buenas opciones para empezar:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Puedes añadir cualquier modelo de texto soportado del [catálogo de modelos](/es/models/text).
  </Step>

  <Step title="Inicia un chat">
    Abre un chat nuevo o existente en Jan, selecciona uno de tus modelos de Venice en el selector de modelos y envía un mensaje de prueba.

    Jan enviará la solicitud a Venice usando tu API key y la URL base compatible con OpenAI.
  </Step>
</Steps>

---

## Verifica la configuración

Envía un prompt corto desde Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

Si el proveedor está configurado correctamente, Jan debería transmitir una respuesta del modelo de Venice seleccionado.

También puedes verificar tu API key fuera de Jan con:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Notas de configuración

- Usa `https://api.venice.ai/api/v1` como URL base. No incluyas `/chat/completions`; Jan añade la ruta del endpoint para las solicitudes de chat.
- Los IDs de modelo deben coincidir exactamente con Venice. Si un modelo no aparece en el selector de Jan, añádelo manualmente usando el ID del [catálogo de modelos](/es/models/text).
- Mantén Venice configurado como proveedor compatible con OpenAI. No uses el proveedor nativo de Anthropic de Jan para los modelos Claude de Venice.
- Las solicitudes remotas a Venice requieren conexión a internet y consumen créditos de la API de Venice.

<Info>
Los modelos locales de Jan siguen disponibles después de añadir Venice. Puedes alternar entre modelos locales y de Venice desde el selector de modelos del chat.
</Info>

## Solución de problemas

<AccordionGroup>
  <Accordion title="Error 401 o de autenticación">
    Comprueba que la API key sea una clave de Venice de [venice.ai/settings/api](https://venice.ai/settings/api), sin espacios extra antes o después.
  </Accordion>

  <Accordion title="404 o modelo no encontrado">
    Confirma que el ID del modelo en Jan coincide exactamente con un ID de modelo de Venice. Comprueba también que la URL base es `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="No aparecen modelos de Venice en el selector">
    Añade el modelo manualmente en los ajustes del proveedor en Jan. Usa el ID exacto, como `zai-org-glm-5-1`, y vuelve a guardar el proveedor.
  </Accordion>

  <Accordion title="Las solicitudes siguen yendo a otro proveedor">
    Asegúrate de que el chat está usando un modelo asociado a tu proveedor Venice. Si Jan está usando una entrada integrada de OpenAI o Anthropic, cambia al modelo de Venice que añadiste.
  </Accordion>
</AccordionGroup>

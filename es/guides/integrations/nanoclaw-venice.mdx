[NanoClaw](https://github.com/qwibitai/nanoclaw) es un asistente de IA ligero y auto-hospedado que funciona en WhatsApp y Telegram. Este fork añade soporte para Venice AI para que todo funcione de forma privada sin una suscripción a Anthropic.

## ¿Por qué Venice AI?

[Venice](https://venice.ai) es una plataforma de IA centrada en la privacidad. [No almacena ni registra prompts ni respuestas](https://venice.ai/privacy) en sus servidores — tus conversaciones existen solo en tu dispositivo. Las solicitudes se cifran de extremo a extremo a través de su proxy hacia proveedores GPU descentralizados, con cero retención de datos. Esto significa que las conversaciones de tu asistente de IA permanecen privadas, incluso frente a Venice.

Venice ofrece acceso anonimizado a modelos frontera (Claude Opus, Claude Sonnet) y acceso totalmente privado a modelos open-source (GLM, Qwen) a través de una sola API — cambia entre ellos en cualquier momento.

| | **Venice AI** | **Proveedores de IA tradicionales** |
|---|---|---|
| **Retención de datos** | Ninguna — cero logs | Sí |
| **Privacidad del prompt** | Cifrado, nunca almacenado | Almacenado en los servidores del proveedor |
| **Modelos open-source** | Sí (GLM, Qwen y otros) | No |
| **Modelos frontera** | Claude, GPT y otros — de forma anónima | Solo mediante suscripciones directas |
| **Precio** | Pago por token, sin suscripción. O haz staking de [DIEM](https://venice.ai/lp/diem) para créditos diarios que se renuevan | Suscripciones de $20–200/mes o API de pago por token |
| **Inferencia sin censura** | Sí (modelos open-source) | No |

## ¿Por qué NanoClaw?

NanoClaw es una alternativa limpia y mínima a plataformas más grandes como OpenClaw. Está diseñado para una persona ejecutando un bot.

| | **NanoClaw (Venice)** | **OpenClaw** |
|---|---|---|
| **Codebase** | ~2.000 líneas, un puñado de archivos | ~500.000 líneas, 53 archivos de config |
| **Dependencias** | ~15 paquetes | 70+ paquetes |
| **Modelo de seguridad** | Aislamiento de contenedor Docker a nivel de SO | Allowlists a nivel de aplicación y códigos de emparejamiento |
| **Aislamiento por grupo** | Cada grupo tiene su propio contenedor, sistema de archivos y memoria | Proceso compartido, memoria compartida |
| **Setup** | Un asistente (`/setup`), ~10 minutos | Configuración manual multi-paso |
| **Proveedor de IA** | Venice AI (privado, sin suscripción) | Anthropic (requiere API key o suscripción) |
| **Personalización** | Edita el código directamente — es lo bastante pequeño para leer | Archivos de config y plugins |
| **Usuario objetivo** | Una persona, un bot | Plataforma multi-usuario |

## Qué obtienes

- Asistente personal de IA en **Telegram** y/o **WhatsApp**
- Impulsado por **Venice AI** — sin necesidad de cuenta de Anthropic
- El bot se ejecuta en un **contenedor Docker aislado** (sandboxed, no puede acceder a tu sistema)
- **Cambio de modelo** — di al bot "switch to zai-org-glm-5" o "use opus" en cualquier momento
- **Tareas programadas** — establece recordatorios y tareas recurrentes
- **Búsqueda y navegación web** integradas
- **Formato markdown** en mensajes de Telegram

**Para Telegram** (recomendado para usuarios primerizos):
1. Abre Telegram y busca **@BotFather**
2. Envía `/newbot` y sigue las indicaciones
3. Guarda el token que te dé BotFather (tiene este aspecto: `123456789:ABCdef...`)

<Warning>
**Para WhatsApp — usa un número virtual, NO el personal:**

NanoClaw se conecta como dispositivo vinculado a tu número de WhatsApp. Eso significa que **el agente puede ver cada mensaje entrante y saliente** — todas tus conversaciones personales, chats de grupo, fotos, todo. Tu teléfono sigue funcionando normalmente, pero el bot tiene visibilidad total de tu cuenta de WhatsApp.

**Usa un número de teléfono virtual.** Estas apps te dan un segundo número que puedes dedicar enteramente al bot:

| App | Precio | Notas |
|-----|-------|-------|
| [Hushed](https://hushed.com) | ~$5/mes | Fiable, funciona bien para la verificación de WhatsApp |
| [Burner](https://www.burnerapp.com) | ~$5/mes | Similar a Hushed, números desechables |
| [Google Voice](https://voice.google.com) | Gratis | Solo EE. UU., puede no funcionar para verificar en WhatsApp en todos los casos |

**Cómo configurarlo:**
1. Obtén un número virtual de una de las apps anteriores
2. Instala WhatsApp en un segundo dispositivo (teléfono antiguo, tablet o emulador) usando ese número virtual
3. Durante la configuración de NanoClaw, escanea el código QR con ese segundo dispositivo — no con tu teléfono personal
</Warning>

## Configuración

La configuración tarda unos 10 minutos. Solo necesitas **una ventana de Terminal**.

<Steps>
  <Step title="Clona e instala">
    Abre Terminal y ejecuta:

    ```bash
    git clone https://github.com/lorenzovenice/nanoclaw-venice.git
    cd nanoclaw-venice
    npm install
    ```

    Espera a que `npm install` termine sin errores.
  </Step>

  <Step title="Lanza Claude Code con Venice">
    Sustituye `your-key` con tu API key de Venice y ejecuta:

    ```bash
    VENICE_API_KEY=your-key npm run venice
    ```

    Esto inicia el proxy de Venice y lanza Claude Code a través de él con un solo comando.

    <Note>
    Claude Code usa por defecto **GLM 5** (`zai-org-glm-5`) para mantener bajos los costes de configuración. Tras configurar, escribe `/model` dentro de Claude Code para cambiar a `claude-sonnet-4-6` o `claude-opus-4-6` para mejor rendimiento.
    </Note>

    Si te pregunta "Do you want to use this API key?" — selecciona **Yes**.
  </Step>

  <Step title="Ejecuta el asistente de configuración">
    En tu terminal de Claude Code, escribe:

    ```
    /setup
    ```

    El asistente te guía por:

    1. **Bootstrap** — comprueba Node.js y dependencias
    2. **API key de Venice** — valida y guarda tu clave
    3. **Elección de canal** — elige WhatsApp, Telegram o ambos
    4. **Build del contenedor** — construye el contenedor Docker (tarda unos minutos la primera vez)
    5. **Auth de WhatsApp** — escanea el código QR con tu teléfono (si aplica)
    6. **Setup de Telegram** — envía un mensaje a tu bot para que detecte tu chat
    7. **Palabra de activación** — prefijo que activa el bot (predeterminado: `@Andy`)
    8. **Directorios montados** — elige "No" por ahora (puedes añadir acceso a archivos después)
    9. **Inicio de servicios** — NanoClaw y el proxy de Venice se inician como servicios en segundo plano

    El asistente instala dos servicios en segundo plano:
    - **NanoClaw** — el bot en sí
    - **Proxy de Venice** — un pequeño servidor local (localhost:4001) que traduce entre Claude Code y Venice AI

    Ambos arrancan automáticamente al iniciar y se reinician solos si fallan.

    <Note>
    Si el asistente se detiene entre pasos, escribe "continue" o "next step" para empujarlo.
    </Note>
  </Step>

  <Step title="Empieza a chatear">
    Una vez completada la configuración, abre tu chat (Telegram o WhatsApp) y envía:

    ```
    @Andy hello, are you there?
    ```

    El bot debería responder en segundos. En tu canal principal, puedes escribir normalmente sin el prefijo `@Andy`.

    **Ya puedes cerrar la ventana de la terminal.** Todo se ejecuta como servicios en segundo plano y arranca automáticamente al iniciar tu ordenador.
  </Step>
</Steps>

## Cómo funciona

NanoClaw tiene dos capas:

| Capa | Qué hace |
|-------|-------------|
| **Claude Code CLI** | Herramienta de admin para setup, depuración y personalización |
| **El bot** | IA en tu chat, ejecutándose dentro de un contenedor Docker aislado |

Para abrir Claude Code en cualquier momento:

```bash
cd nanoclaw-venice
ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
```

Úsalo para ejecutar `/setup`, `/debug`, `/customize` o hacer cambios en el comportamiento del bot.

## Solución de problemas

<AccordionGroup>
  <Accordion title="El proxy no está en ejecución">
    El proxy de Venice se ejecuta como servicio en segundo plano y se reinicia solo. Si no funciona:

    **macOS:**
    ```bash
    # Comprueba si está ejecutándose
    launchctl list | grep venice-proxy

    # Reinícialo
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Revisa los logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```

    **Linux:**
    ```bash
    # Comprueba si está ejecutándose
    systemctl --user status nanoclaw-venice-proxy

    # Reinícialo
    systemctl --user restart nanoclaw-venice-proxy

    # Revisa los logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```
  </Accordion>

  <Accordion title="Claude Code muestra error 403 o 'Please run /login'">
    Esto significa que Claude Code no puede conectarse al proxy de Venice.

    1. **Comprueba que el proxy esté ejecutándose.** Consulta el paso anterior.
    2. **Asegúrate de estar en la carpeta correcta.** Haz siempre `cd nanoclaw-venice` primero.
    3. **Empieza de cero:** cierra todas las terminales y ejecuta:
       ```bash
       cd nanoclaw-venice
       ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
       ```
  </Accordion>

  <Accordion title="Errores de modelo ('model does not exist')">
    Reinicia el proxy y el bot:

    **macOS:**
    ```bash
    # Reiniciar proxy
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Reiniciar bot
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw
    ```

    **Linux:**
    ```bash
    # Reiniciar proxy
    systemctl --user restart nanoclaw-venice-proxy

    # Reiniciar bot
    systemctl --user restart nanoclaw
    ```

    Comprueba los modelos disponibles en el [catálogo de modelos](/es/models/text).
  </Accordion>

  <Accordion title="El bot no responde a los mensajes">
    Ve por estos pasos en orden:

    1. **Comprueba tu palabra de activación.** Asegúrate de usar el prefijo correcto (p. ej., `@Andy hello`).
    2. **Comprueba que Docker esté en ejecución.** Ejecuta `docker info` — si da error, abre Docker Desktop.
    3. **Comprueba que el proxy esté ejecutándose.** Consulta "El proxy no está en ejecución" más arriba.
    4. **Revisa los logs:** `tail -f logs/nanoclaw.log` en la carpeta del proyecto.
    5. **Revisa los logs del contenedor.** Abre la carpeta `nanoclaw-venice/groups/main/logs/`. Abre el archivo más reciente que empieza por `container-`.
    6. **Reinicia todo:** reinicia el proxy y el bot (ver arriba).
  </Accordion>

  <Accordion title="El build del contenedor falla durante la configuración">
    Asegúrate de que Docker Desktop esté abierto y en ejecución. Espera 10 segundos a que Docker arranque del todo, luego escribe `continue` en el asistente para reintentar.
  </Accordion>

  <Accordion title="WhatsApp desconectado">
    Tu sesión de WhatsApp puede expirar. Para reconectar:

    ```bash
    cd nanoclaw-venice
    npm run auth
    ```

    Escanea el código QR con WhatsApp (Settings → Linked Devices → Link a Device) y luego reinicia el bot:
    - macOS: `launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
    - Linux: `systemctl --user restart nanoclaw`
  </Accordion>
</AccordionGroup>

## Avanzado

<AccordionGroup>
  <Accordion title="Dar al bot acceso a archivos de tu ordenador">
    Por defecto, el bot está completamente aislado de tu ordenador — solo puede ver su propia memoria e historial de conversación.

    - **Durante la configuración:** cuando te pregunte sobre el acceso a directorios, elige "Yes"
    - **Tras la configuración:** ejecuta `/customize` en Claude Code
  </Accordion>

  <Accordion title="Iniciar/parar el bot manualmente">
    NanoClaw ejecuta dos servicios en segundo plano que arrancan automáticamente.

    **macOS:**
    | Acción | Comando |
    |--------|---------|
    | Iniciar bot | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Parar bot | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Reiniciar bot | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw` |
    | Iniciar proxy | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Parar proxy | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Reiniciar proxy | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy` |

    **Linux:**
    | Acción | Comando |
    |--------|---------|
    | Iniciar bot | `systemctl --user start nanoclaw` |
    | Parar bot | `systemctl --user stop nanoclaw` |
    | Reiniciar bot | `systemctl --user restart nanoclaw` |
    | Iniciar proxy | `systemctl --user start nanoclaw-venice-proxy` |
    | Parar proxy | `systemctl --user stop nanoclaw-venice-proxy` |
    | Reiniciar proxy | `systemctl --user restart nanoclaw-venice-proxy` |
  </Accordion>

  <Accordion title="Usar Claude Code a través de Venice (sin bot)">
    Si solo quieres Claude Code con Venice y no necesitas WhatsApp/Telegram, el servicio proxy debe estar ejecutándose. Si ya ejecutaste `/setup`, ya se está ejecutando como servicio en segundo plano.

    ```bash
    cd nanoclaw-venice
    ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
    ```

    **Tip:** añade esto a tu `~/.zshrc` (o `~/.bashrc`) para cambiar rápidamente cualquier terminal a Venice:

    ```bash
    alias venice='export ANTHROPIC_BASE_URL=http://localhost:4001 && export ANTHROPIC_API_KEY=venice-proxy && echo "Using Venice API"'
    alias anthropic='unset ANTHROPIC_BASE_URL && unset ANTHROPIC_API_KEY && echo "Using Anthropic API"'
    ```

    Después, simplemente escribe `venice` en cualquier terminal antes de ejecutar `claude` para usar Venice, o `anthropic` para volver.
  </Accordion>

  <Accordion title="Ejecutar varios bots">
    Puedes ejecutar varios bots NanoClaw en la misma máquina (p. ej., uno para uso personal y otro para un equipo). Solo clona el repo en una carpeta distinta y vuelve a ejecutar el setup. Nota: comparten la misma imagen Docker, por lo que reconstruir una afecta a todas.
  </Accordion>

  <Accordion title="Comandos para desarrolladores">
    Para quien quiera modificar el código de NanoClaw:

    ```bash
    npm run dev          # Inicia proxy + NanoClaw con hot reload
    npm run proxy        # Inicia solo el proxy de Venice
    npm run build        # Compila TypeScript
    npm test             # Ejecuta tests
    ./container/build.sh # Reconstruye el contenedor del agente
    ```
  </Accordion>
</AccordionGroup>

## Arquitectura

```
Tú (WhatsApp/Telegram)
        ↓
   NanoClaw (Node.js)
        ↓
   Contenedor Docker (sandbox aislado)
        ↓
   Proxy de Venice (localhost:4001)
        ↓
   api.venice.ai (inferencia privada)
```

| Archivo | Propósito |
|------|---------|
| `proxy/venice-proxy.ts` | Traduce el formato Anthropic al formato OpenAI para Venice |
| `src/index.ts` | Orquestador principal — bucle de mensajes, invocación del agente |
| `src/channels/whatsapp.ts` | Conexión a WhatsApp vía baileys |
| `src/channels/telegram.ts` | Bot de Telegram vía grammy |
| `src/container-runner.ts` | Genera contenedores de agente aislados |

## FAQ

<AccordionGroup>
  <Accordion title="¿Por qué necesito un proxy?">
    El Claude Agent SDK habla el formato de mensajes de Anthropic. Venice habla el formato de OpenAI. El proxy traduce entre ambos para que todo funcione sin modificar el SDK.
  </Accordion>

  <Accordion title="¿Puedo usar modelos open-source?">
    Sí. Venice aloja muchos modelos. Di al bot "switch to zai-org-glm-5" o cualquier ID de modelo de Venice. Consulta el [catálogo de modelos](/es/models/text).
  </Accordion>

  <Accordion title="¿Es seguro?">
    Los agentes se ejecutan en contenedores Docker con aislamiento real a nivel de SO. La API key de Venice se pasa por stdin, nunca se escribe en disco dentro de los contenedores. Cada grupo obtiene su propio entorno aislado.
  </Accordion>

  <Accordion title="¿Necesito una suscripción a Anthropic?">
    No. Todo se ejecuta a través de Venice AI. Solo necesitas una API key de Venice.
  </Accordion>

  <Accordion title="¿Puedo usar esto en un servidor?">
    Sí. Funciona en cualquier máquina Linux con Docker. Usa el servicio systemd para autoarranque al inicio.
  </Accordion>
</AccordionGroup>

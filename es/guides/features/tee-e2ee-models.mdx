Venice ofrece modelos con privacidad reforzada que se ejecutan en entornos de ejecución de confianza (TEE) y soportan cifrado de extremo a extremo (E2EE). Estos modelos proporcionan garantías criptográficas de que tus datos permanecen privados, incluso frente a Venice.

## Entendiendo los niveles de privacidad

| Tipo | Prefijo | Qué significa |
|------|--------|---------------|
| **TEE** | `tee-*` | El modelo se ejecuta en un enclave protegido por hardware. Venice no puede acceder a la computación. Puedes verificarlo con attestation. |
| **E2EE** | `e2ee-*` | Cifrado completo de extremo a extremo. Tus prompts se cifran del lado del cliente antes de enviarse. Solo el TEE puede descifrarlos. |

<Info>
Los modelos E2EE incluyen protección TEE más cifrado del lado del cliente. Los modelos TEE proporcionan seguridad de enclave sin requerir cifrado del lado del cliente.
</Info>

## Modelos TEE

Los modelos TEE se ejecutan dentro de enclaves protegidos por hardware (Intel TDX, NVIDIA Confidential Computing). Los pesos del modelo y tus datos están protegidos frente al sistema anfitrión, incluida la infraestructura de Venice.

### Verificando la attestation de TEE

Puedes verificar criptográficamente que un modelo está corriendo en un TEE genuino obteniendo su informe de attestation:

<CodeGroup>
```bash cURL
# Genera un nonce aleatorio (previene ataques de replay)
NONCE=$(openssl rand -hex 16)

# Obtén la attestation
curl "https://api.venice.ai/api/v1/tee/attestation?model=tee-qwen3-5-122b-a10b&nonce=$NONCE" \
  -H "Authorization: Bearer $API_KEY_VENICE"
```

```python Python
import secrets
import requests

nonce = secrets.token_hex(16)

response = requests.get(
    f"https://api.venice.ai/api/v1/tee/attestation",
    params={"model": "tee-qwen3-5-122b-a10b", "nonce": nonce},
    headers={"Authorization": f"Bearer {api_key}"}
)

attestation = response.json()
print(f"Verified: {attestation['verified']}")
print(f"TEE Provider: {attestation['tee_provider']}")
print(f"Model: {attestation['model']}")
```
</CodeGroup>

La respuesta de attestation incluye:

| Campo | Descripción |
|-------|-------------|
| `verified` | Si la attestation pasó la verificación del servidor |
| `nonce` | Tu nonce, confirmando frescura |
| `model` | ID del modelo certificado |
| `tee_provider` | Identificador del proveedor de TEE |
| `intel_quote` | Quote crudo de Intel TDX (base64) para verificación cliente |
| `nvidia_payload` | Datos de attestation de GPU NVIDIA (si aplica) |
| `signing_key` | Clave pública para verificar las firmas de respuesta (típicamente requerida en flujos E2EE; puede omitirse en algunos modelos TEE simples) |
| `signing_address` | Dirección Ethereum derivada de la clave de firma |

<Tip>
Para uso en producción, verifica la attestation del lado del cliente parseando el quote de Intel TDX y comprobando la attestation de NVIDIA.
</Tip>

<Note>
Para la verificación de modelos TEE simples, `signing_address` y los campos de verificación del lado del servidor son suficientes para comprobaciones básicas de attestation. Una `signing_key` es necesaria cuando necesitas acuerdo de claves E2EE del lado del cliente y comprobaciones estrictas de vinculación de claves.
</Note>

## Modelos E2EE

Los modelos E2EE añaden cifrado del lado del cliente sobre la protección TEE. Tus prompts se cifran antes de salir de tu dispositivo, y solo el TEE puede descifrarlos.

Venice E2EE usa:

- **ECDH (Elliptic Curve Diffie-Hellman)** sobre secp256k1 para el intercambio de claves
- **HKDF-SHA256** para la derivación de claves
- **AES-256-GCM** para el cifrado simétrico
- **Attestation de TEE** para verificar que el modelo corre en un enclave seguro

<Warning>
E2EE requiere implementación del lado del cliente. Los ejemplos siguientes muestran el protocolo completo.
</Warning>

### Cómo funciona E2EE

<Steps>
  <Step title="Generar par de claves efímero">
    El cliente genera un par de claves secp256k1 para esta sesión.
  </Step>
  <Step title="Obtener attestation del TEE">
    El cliente solicita `/api/v1/tee/attestation` y recibe la clave pública del modelo, la evidencia de attestation y un nonce.
  </Step>
  <Step title="Verificar attestation">
    El cliente comprueba la coincidencia del nonce, que el modo debug esté deshabilitado y la validez de la attestation.
  </Step>
  <Step title="Cifrar mensajes">
    El cliente cifra los prompts usando el secreto compartido ECDH → HKDF → AES-GCM.
  </Step>
  <Step title="Enviar solicitud">
    El cliente envía la solicitud con cabeceras E2EE (`X-Venice-TEE-Client-Pub-Key`, `X-Venice-TEE-Model-Pub-Key`, `X-Venice-TEE-Signing-Algo`).
  </Step>
  <Step title="Procesamiento del TEE">
    El TEE descifra la solicitud, la procesa y cifra la respuesta.
  </Step>
  <Step title="Descifrar respuesta">
    El cliente recibe trozos cifrados y los descifra con la clave privada.
  </Step>
</Steps>

Genera un nuevo par de claves para cada sesión. La clave privada debería mantenerse solo en memoria y limpiarse de forma segura tras su uso.

### Paso 3: Obtener y verificar la attestation del TEE

La attestation prueba que el modelo se está ejecutando en un TEE genuino. Verifica siempre la attestation antes de confiar en la clave pública del modelo.

<Info>
**Importante: longitud del nonce** - El nonce del cliente debe ser de **32 bytes (64 caracteres hex)**. Algunos proveedores de TEE requieren exactamente 32 bytes y rechazarán nonces más cortos.
</Info>

<CodeGroup>
```javascript JavaScript
import crypto from 'crypto'

async function fetchAndVerifyAttestation(modelId, apiKey) {
  // Genera un nonce de cliente para protección contra replay (32 bytes = 64 hex)
  const clientNonce = crypto.randomBytes(32).toString('hex')

  const response = await fetch(
    `https://api.venice.ai/api/v1/tee/attestation?model=${encodeURIComponent(modelId)}&nonce=${clientNonce}`,
    { headers: { Authorization: `Bearer ${apiKey}` } }
  )

  const attestation = await response.json()

  // Verifica la attestation
  if (attestation.verified !== true) {
    throw new Error('TEE attestation verification failed on server')
  }

  if (attestation.nonce !== clientNonce) {
    throw new Error('Attestation nonce mismatch - possible replay attack')
  }

  // Obtén la clave pública del modelo para el cifrado
  const modelPublicKey = attestation.signing_key || attestation.signing_public_key
  if (!modelPublicKey) {
    throw new Error('No signing key in attestation response')
  }

  return {
    modelPublicKey,
    signingAddress: attestation.signing_address,
    attestation,
  }
}
```

```python Python
import secrets
import requests

def fetch_and_verify_attestation(model_id: str, api_key: str) -> dict:
    """Obtiene y verifica la attestation del TEE para un modelo."""
    # Genera un nonce de cliente para protección contra replay (32 bytes = 64 hex)
    client_nonce = secrets.token_hex(32)

    response = requests.get(
        f'https://api.venice.ai/api/v1/tee/attestation',
        params={'model': model_id, 'nonce': client_nonce},
        headers={'Authorization': f'Bearer {api_key}'}
    )
    attestation = response.json()

    # Verifica la attestation
    if attestation.get('verified') != True:
        raise ValueError('TEE attestation verification failed on server')

    if attestation.get('nonce') != client_nonce:
        raise ValueError('Attestation nonce mismatch - possible replay attack')

    # Obtén la clave pública del modelo para el cifrado
    model_public_key = attestation.get('signing_key') or attestation.get('signing_public_key')
    if not model_public_key:
        raise ValueError('No signing key in attestation response')

    return {
        'model_public_key': model_public_key,
        'signing_address': attestation.get('signing_address'),
        'attestation': attestation
    }
```

### Paso 4: Cifrar mensajes

Cifra los mensajes de usuario y de sistema antes de enviarlos. Solo los mensajes de rol `user` y `system` necesitan cifrado.

<Warning>
Cuando las cabeceras E2EE están presentes, **todos** los mensajes de rol `user` y `system` deben estar cifrados. Enviar cualquier contenido en texto plano en estos roles producirá un error "Encrypted field is not valid hex".
</Warning>

<CodeGroup>
```javascript JavaScript
import { gcm } from '@noble/ciphers/aes.js'
import { hkdf } from '@noble/hashes/hkdf.js'
import { sha256 } from '@noble/hashes/sha2.js'
import { ec as EC } from 'elliptic'
import crypto from 'crypto'

const HKDF_INFO = new TextEncoder().encode('ecdsa_encryption')

function encryptMessage(plaintext, modelPublicKeyHex) {
  const ec = new EC('secp256k1')

  // Normaliza la clave pública (añade el prefijo 04 si es necesario)
  let normalizedKey = modelPublicKeyHex
  if (!normalizedKey.startsWith('04') && normalizedKey.length === 128) {
    normalizedKey = '04' + normalizedKey
  }

  const modelPublicKey = ec.keyFromPublic(normalizedKey, 'hex')

  // Genera par de claves efímero para este mensaje
  const ephemeralKeyPair = ec.genKeyPair()

  // Secreto compartido ECDH
  const sharedSecret = ephemeralKeyPair.derive(modelPublicKey.getPublic())
  const sharedSecretBytes = new Uint8Array(sharedSecret.toArray('be', 32))

  // Deriva la clave AES con HKDF
  const aesKey = hkdf(sha256, sharedSecretBytes, undefined, HKDF_INFO, 32)

  // Genera nonce aleatorio
  const nonce = crypto.randomBytes(12)

  // Cifra con AES-GCM
  const cipher = gcm(aesKey, nonce)
  const encrypted = cipher.encrypt(new TextEncoder().encode(plaintext))

  // Obtén la clave pública efímera (sin comprimir)
  const ephemeralPublic = new Uint8Array(ephemeralKeyPair.getPublic(false, 'array'))

  // Combina: ephemeral_public (65 bytes) + nonce (12 bytes) + ciphertext
  const result = new Uint8Array(65 + 12 + encrypted.length)
  result.set(ephemeralPublic, 0)
  result.set(nonce, 65)
  result.set(encrypted, 65 + 12)

  return Buffer.from(result).toString('hex')
}

function encryptMessagesForE2EE(messages, modelPublicKey) {
  return messages.map(msg => {
    if (msg.role === 'user' || msg.role === 'system') {
      return {
        ...msg,
        content: encryptMessage(msg.content, modelPublicKey),
      }
    }
    return msg
  })
}
```

```python Python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives.kdf.hkdf import HKDF
from cryptography.hazmat.primitives import hashes
from ecdsa import SECP256k1, VerifyingKey, SigningKey
import os

HKDF_INFO = b'ecdsa_encryption'

def encrypt_message(plaintext: str, model_public_key_hex: str) -> str:
    """Cifra un mensaje usando ECDH + HKDF + AES-GCM."""
    # Normaliza la clave pública
    key_hex = model_public_key_hex
    if not key_hex.startswith('04') and len(key_hex) == 128:
        key_hex = '04' + key_hex

    model_public_key_bytes = bytes.fromhex(key_hex)

    # Parsea la clave pública del modelo (omite el prefijo 04)
    model_verifying_key = VerifyingKey.from_string(
        model_public_key_bytes[1:],
        curve=SECP256k1
    )

    # Genera par de claves efímero para este mensaje
    ephemeral_private = SigningKey.generate(curve=SECP256k1)
    ephemeral_public = ephemeral_private.get_verifying_key()

    # ECDH: calcula el secreto compartido
    shared_point = model_verifying_key.pubkey.point * ephemeral_private.privkey.secret_multiplier
    shared_secret = shared_point.x().to_bytes(32, 'big')

    # Deriva la clave AES con HKDF
    hkdf = HKDF(
        algorithm=hashes.SHA256(),
        length=32,
        salt=None,
        info=HKDF_INFO,
    )
    aes_key = hkdf.derive(shared_secret)

    # Genera nonce aleatorio
    nonce = os.urandom(12)

    # Cifra con AES-GCM
    aesgcm = AESGCM(aes_key)
    ciphertext = aesgcm.encrypt(nonce, plaintext.encode('utf-8'), None)

    # Obtén la clave pública efímera (sin comprimir: 04 || x || y)
    ephemeral_public_bytes = b'\x04' + ephemeral_public.to_string()

    # Combina: ephemeral_public (65 bytes) + nonce (12 bytes) + ciphertext
    result = ephemeral_public_bytes + nonce + ciphertext

    return result.hex()

def encrypt_messages_for_e2ee(messages: list, model_public_key: str) -> list:
    """Cifra los mensajes de usuario y de sistema."""
    encrypted_messages = []
    for msg in messages:
        if msg['role'] in ('user', 'system'):
            encrypted_messages.append({
                **msg,
                'content': encrypt_message(msg['content'], model_public_key)
            })
        else:
            encrypted_messages.append(msg)
    return encrypted_messages
```

### Paso 6: Descifrar los trozos de la respuesta

Las respuestas de los modelos E2EE son trozos cifrados codificados en hex. Descifra cada trozo con tu clave privada.

<CodeGroup>
```javascript JavaScript
import { gcm } from '@noble/ciphers/aes.js'
import { hkdf } from '@noble/hashes/hkdf.js'
import { sha256 } from '@noble/hashes/sha2.js'
import { ec as EC } from 'elliptic'

const HKDF_INFO = new TextEncoder().encode('ecdsa_encryption')

function hexToBytes(hex) {
  const h = hex.startsWith('0x') ? hex.slice(2) : hex
  const bytes = new Uint8Array(h.length / 2)
  for (let i = 0; i < bytes.length; i++) {
    bytes[i] = parseInt(h.substring(i * 2, i * 2 + 2), 16)
  }
  return bytes
}

function isHexEncrypted(s) {
  // Mínimo: ephemeral_pub (65) + nonce (12) + tag (16) = 93 bytes = 186 hex
  if (s.length < 186) return false
  return /^[0-9a-fA-F]+$/.test(s)
}

function decryptChunk(ciphertextHex, clientPrivateKey) {
  const raw = hexToBytes(ciphertextHex)

  // Parsea componentes
  const serverEphemeralPubKey = raw.slice(0, 65)
  const nonce = raw.slice(65, 65 + 12)
  const ciphertext = raw.slice(65 + 12)

  // ECDH con la clave efímera del servidor
  const ec = new EC('secp256k1')
  const clientKey = ec.keyFromPrivate(Buffer.from(clientPrivateKey))
  const serverKey = ec.keyFromPublic(Buffer.from(serverEphemeralPubKey))
  const sharedSecret = clientKey.derive(serverKey.getPublic())
  const sharedSecretBytes = new Uint8Array(sharedSecret.toArray('be', 32))

  // Deriva la clave AES
  const aesKey = hkdf(sha256, sharedSecretBytes, undefined, HKDF_INFO, 32)

  // Descifra
  const cipher = gcm(aesKey, nonce)
  const plaintext = cipher.decrypt(ciphertext)

  return new TextDecoder().decode(plaintext)
}

// Procesa la respuesta en streaming
async function processE2EEStream(response, clientPrivateKey) {
  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  let fullContent = ''

  while (true) {
    const { done, value } = await reader.read()
    if (done) break

    const text = decoder.decode(value)
    const lines = text.split('\n')

    for (const line of lines) {
      if (!line.startsWith('data: ')) continue
      const data = line.slice(6)
      if (data === '[DONE]') continue

      try {
        const chunk = JSON.parse(data)
        const content = chunk.choices?.[0]?.delta?.content

        if (content && isHexEncrypted(content)) {
          const decrypted = decryptChunk(content, clientPrivateKey)
          fullContent += decrypted
          process.stdout.write(decrypted) // Salida en tiempo real
        } else if (content) {
          fullContent += content
          process.stdout.write(content)
        }
      } catch (e) {
        // Omite trozos malformados
      }
    }
  }

  return fullContent
}
```

```python Python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives.kdf.hkdf import HKDF
from cryptography.hazmat.primitives import hashes
from ecdsa import SECP256k1, VerifyingKey, SigningKey
import json
import re

HKDF_INFO = b'ecdsa_encryption'

def is_hex_encrypted(s: str) -> bool:
    """Comprueba si la cadena parece contenido cifrado en hex."""
    if len(s) < 186:  # Mínimo: 65 + 12 + 16 = 93 bytes = 186 hex
        return False
    return bool(re.match(r'^[0-9a-fA-F]+$', s))

def decrypt_chunk(ciphertext_hex: str, client_private_key: bytes) -> str:
    """Descifra un trozo de respuesta E2EE."""
    raw = bytes.fromhex(ciphertext_hex)

    # Parsea componentes
    server_ephemeral_pub = raw[:65]
    nonce = raw[65:77]
    ciphertext = raw[77:]

    # Parsea la clave pública efímera del servidor (omite el prefijo 04)
    server_verifying_key = VerifyingKey.from_string(
        server_ephemeral_pub[1:],
        curve=SECP256k1
    )

    # Reconstruye la clave privada del cliente
    client_signing_key = SigningKey.from_string(client_private_key, curve=SECP256k1)

    # ECDH: calcula el secreto compartido
    shared_point = server_verifying_key.pubkey.point * client_signing_key.privkey.secret_multiplier
    shared_secret = shared_point.x().to_bytes(32, 'big')

    # Deriva la clave AES
    hkdf = HKDF(
        algorithm=hashes.SHA256(),
        length=32,
        salt=None,
        info=HKDF_INFO,
    )
    aes_key = hkdf.derive(shared_secret)

    # Descifra
    aesgcm = AESGCM(aes_key)
    plaintext = aesgcm.decrypt(nonce, ciphertext, None)

    return plaintext.decode('utf-8')

def process_e2ee_stream(response, client_private_key: bytes) -> str:
    """Procesa la respuesta E2EE en streaming."""
    full_content = ''

    for line in response.iter_lines():
        if not line:
            continue

        line_str = line.decode('utf-8')
        if not line_str.startswith('data: '):
            continue

        data = line_str[6:]
        if data == '[DONE]':
            continue

        try:
            chunk = json.loads(data)
            content = chunk.get('choices', [{}])[0].get('delta', {}).get('content', '')

            if content and is_hex_encrypted(content):
                decrypted = decrypt_chunk(content, client_private_key)
                full_content += decrypted
                print(decrypted, end='', flush=True)  # Salida en tiempo real
            elif content:
                full_content += content
                print(content, end='', flush=True)
        except json.JSONDecodeError:
            pass

    print()  # Nueva línea final
    return full_content
```

### Limitaciones de E2EE

<Warning>
E2EE tiene algunas restricciones debido a los requisitos de cifrado:
</Warning>

| Característica | Estado |
|---------|--------|
| Streaming | **Requerido** (no streaming no soportado) |
| Búsqueda web | **Deshabilitada** (filtraría contenido) |
| Subida de archivos | **No soportada** |
| Function calling | **No soportado** |
| System prompt de Venice | **Deshabilitado** (debe cifrarse en cliente) |

### Buenas prácticas de seguridad

1. **Genera nuevos pares de claves por sesión** - No reutilices claves efímeras
2. **Limpia las claves privadas con ceros** - Borra los bytes de la clave privada de memoria al terminar
3. **Verifica la attestation** - Comprueba siempre `verified: true` y la coincidencia del nonce
4. **Comprueba el modo debug** - Rechaza attestations de enclaves debug
5. **Usa streaming** - E2EE requiere streaming para el correcto troceado del cifrado
6. **Maneja los errores con cuidado** - No expongas los errores de descifrado a los usuarios
7. **Usa nonces de 32 bytes** - Los proveedores de TEE requieren exactamente 32 bytes

## Buenas prácticas

<AccordionGroup>
  <Accordion title="Verifica siempre la attestation en producción">
    No te limites a confiar en la respuesta `verified: true`. Parsea el quote de Intel TDX en el cliente y verifica que las mediciones coincidan con los valores esperados. Para GPUs NVIDIA, comprueba la attestation a través del servicio de verificación de NVIDIA.
  </Accordion>
  
  <Accordion title="Usa nonces frescos">
    Genera siempre un nuevo nonce aleatorio para cada solicitud de attestation. Esto previene ataques de replay en los que un atacante podría servir una attestation antigua.
  </Accordion>
  
  <Accordion title="Verifica el binding de claves">
    La clave de firma debería estar vinculada al campo REPORTDATA de TDX. Eso prueba que la clave se generó dentro del enclave.
  </Accordion>
  
  <Accordion title="Comprueba el modo debug">
    Verifica que la attestation TDX no tenga flags de debug establecidos. Un enclave en debug puede ser inspeccionado y no debería ser de confianza para producción.
  </Accordion>
  
  <Accordion title="Usa nuestros SDKs para E2EE">
    E2EE requiere una implementación criptográfica cuidadosa. Usa nuestros SDKs oficiales en lugar de implementar el protocolo tú mismo.
  </Accordion>

## Manejo de errores

| Error | Causa | Solución |
|-------|-------|----------|
| `TEE attestation verification failed` | La attestation no pasó la validación | Reintenta o contacta a soporte |
| `Attestation nonce mismatch` | Posible ataque de replay | Genera un nonce nuevo |
| `TDX debug mode detected` | El enclave está en modo debug | No usar en producción |
| `Failed to decrypt field` | El descifrado E2EE falló del lado del servidor | Comprueba tu implementación de cifrado |
| `E2EE requires streaming` | Solicitud no-streaming a modelo E2EE | Establece `stream: true` |
| `Encrypted field is not valid hex` | Se envió texto plano con cabeceras E2EE | Cifra todos los mensajes user/system |
| `Invalid public key` | Formato de clave incorrecto | Usa 130 chars hex empezando por `04` |

## Solución de problemas

<AccordionGroup>
  <Accordion title="502 Bad Gateway o 'Nonce must be exactly 32 bytes'">
    La longitud del nonce es incorrecta. Los proveedores de TEE requieren exactamente **32 bytes (64 caracteres hex)**.
    
    - Usa `crypto.randomBytes(32).toString('hex')` (JS) o `secrets.token_hex(32)` (Python)
    - Error común: `secrets.token_hex(16)` produce 32 chars hex (16 bytes), no 32 bytes
  </Accordion>
  
  <Accordion title="Falló la verificación de la attestation">
    - Comprueba que el modelo soporte E2EE (`supportsE2EE: true`)
    - Verifica que tu API key sea válida y tenga acceso al modelo solicitado
    - Verifica la conectividad de red con la API de Venice
  </Accordion>
  
  <Accordion title="Falló el descifrado">
    - Asegúrate de estar usando la misma clave privada que generó la clave pública enviada en cabeceras
    - Comprueba que el contenido de la respuesta esté realmente codificado en hex (E2EE activo)
    - Verifica que la clave pública del modelo coincida con la usada para cifrar
  </Accordion>
  
  <Accordion title="Encrypted field is not valid hex">
    - Todos los mensajes con rol `user` y `system` deben estar cifrados cuando las cabeceras E2EE están presentes
    - Verifica que tu contenido cifrado pase la validación `isValidEncrypted()` (mínimo 186 chars hex)
    - Comprueba que la salida del cifrado sea hex en minúsculas sin prefijos
  </Accordion>
  
  <Accordion title="Errores de clave pública inválida">
    - La clave pública del cliente debe ser exactamente **130 caracteres hex** empezando por `04`
    - Usa el helper `validateClientPubkey()` para verificar el formato antes de enviar
    - Asegúrate de usar el formato sin comprimir de la clave pública (65 bytes = 130 chars hex)
  </Accordion>
  
  <Accordion title="Modelo no encontrado">
    - Verifica que el ID del modelo sea correcto y que el modelo soporte E2EE
    - Usa el endpoint `/models` para verificar los modelos E2EE disponibles
  </Accordion>
</AccordionGroup>

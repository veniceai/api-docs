JSON-RPC 2.0 요청(단일 또는 일괄)을 지원되는 블록체인 노드로 전달합니다. API 키 및 x402 지갑 인증을 모두 지원합니다. 청구는 크레딧당으로 이루어지며 Venice 잔액으로 표시됩니다 — 하나의 자격 증명, 하나의 청구서로 아래의 모든 체인을 사용할 수 있습니다.

## 인증

이 엔드포인트는 두 가지 인증 방식을 지원합니다.

- **API 키**: `Authorization: Bearer <key>` 헤더를 통한 표준 Bearer 토큰 인증.
- **x402 지갑**: Base 또는 Solana의 지갑에서 USDC 크레딧으로 사용한 만큼 지불하는 방식. Venice 계정이 필요하지 않습니다. 설정 방법은 [x402 가이드](/ko/guides/integrations/x402-venice-api)를 참고하세요.

두 방식 모두 동일한 레이트 리밋과 청구(Venice 크레딧)를 공유합니다.

## 지원 네트워크

라이브로 권위 있는 목록은 [`GET /crypto/rpc/networks`](/ko/api-reference/endpoint/crypto/networks)를 참고하세요. 현재 지원 범위:

| 패밀리 | 메인넷 | 테스트넷 |
|---|---|---|
| Ethereum | `ethereum-mainnet` | `ethereum-sepolia`, `ethereum-holesky` |
| Polygon | `polygon-mainnet` | `polygon-amoy` |
| Arbitrum | `arbitrum-mainnet` | `arbitrum-sepolia` |
| Optimism | `optimism-mainnet` | `optimism-sepolia` |
| Base | `base-mainnet` | `base-sepolia` |
| Linea | `linea-mainnet` | `linea-sepolia` |
| Avalanche C-Chain | `avalanche-mainnet` | `avalanche-fuji` |
| BNB Smart Chain | `bsc-mainnet` | `bsc-testnet` |
| Blast | `blast-mainnet` | `blast-sepolia` |
| zkSync Era | `zksync-mainnet` | `zksync-sepolia` |
| Starknet | `starknet-mainnet` | `starknet-sepolia` |

## 요청 형태

### 단일 요청

```json
{
  "jsonrpc": "2.0",
  "method": "eth_chainId",
  "params": [],
  "id": 1
}
```

### 일괄(batch) 요청

최대 **100**개의 JSON-RPC 2.0 객체 배열. 각 항목은 독립적으로 검증되며, 지원되지 않는 메서드가 하나라도 있으면 전체 배치가 `400`으로 거부되고 위반 메서드 이름이 모두 오류 메시지에 나열됩니다.

```json
[
  { "jsonrpc": "2.0", "method": "eth_chainId", "params": [], "id": 1 },
  { "jsonrpc": "2.0", "method": "eth_blockNumber", "params": [], "id": 2 }
]
```

## 지원 메서드 및 가격 등급

메서드는 세 가지 크레딧 등급으로 분류됩니다. 호출당 소비 크레딧 = `baseCredits[chain] × methodTier`.

| 등급 | 배수 | 예시 메서드 |
|---|---|---|
| **Standard** | 1× | `eth_call`, `eth_getBalance`, `eth_blockNumber`, `eth_sendRawTransaction`, `eth_getLogs`, `eth_getTransactionReceipt`, `eth_estimateGas`, `net_version`, `web3_clientVersion`, ERC-4337 bundler 메서드(`eth_sendUserOperation`, `eth_estimateUserOperationGas` 등), 체인 패밀리 확장(`zks_*`, `linea_*`, `bor_*`, `starknet_*`) |
| **Advanced** | 2× | `trace_block`, `trace_call`, `trace_transaction`, `debug_traceCall`, `debug_traceTransaction`, `debug_traceBlockByHash`, `txpool_inspect`, `txpool_status`, `arbtrace_*` |
| **Large** | 4× | `trace_replayBlockTransactions`, `trace_replayTransaction`, `txpool_content`, `arbtrace_replayTransaction`, `arbtrace_replayBlockTransactions` |

### 체인별 기본 크레딧

| baseCredits | 체인 |
|---|---|
| **20** | Ethereum + 위의 모든 EVM L2(Base, Optimism, Arbitrum, Polygon, Linea, Avalanche, BSC, Blast) 및 Starknet |
| **30** | zkSync Era |

### 비용 예시

Venice의 크레딧당 가격 `~$6.25 × 10⁻⁷` 기준:

| 호출 | 크레딧 | USD 비용 |
|---|---|---|
| Ethereum의 `eth_call` (20 × 1×) | 20 | $0.0000125 |
| Ethereum의 `trace_transaction` (20 × 2×) | 40 | $0.0000250 |
| Ethereum의 `trace_replayTransaction` (20 × 4×) | 80 | $0.0000500 |
| zkSync의 `eth_call` (30 × 1×) | 30 | $0.0000188 |

### 지원되지 않음

- **WebSocket 전용 메서드** (`eth_subscribe`, `eth_unsubscribe`) — 이 프록시는 HTTP 전용입니다. 대신 폴링하거나, 직접 WebSocket 제공자로 업그레이드하세요.
- **상태 보존 필터 메서드** (`eth_newFilter`, `eth_getFilterChanges`, `eth_getFilterLogs`, `eth_uninstallFilter`, `eth_newBlockFilter`, `eth_newPendingTransactionFilter`) — 필터 상태는 단일 업스트림 백엔드에 고정되어 있어 로드 밸런싱된 HTTP 프록시에서 조용히 깨집니다. 대신 `eth_getLogs`(stateless)를 사용하세요.
- **마이너 / 키 보유 메서드** (`eth_sign`, `eth_accounts`, `eth_mining`, `eth_hashrate`, `eth_getWork`, `eth_submitWork`) — 호스팅된 제공자 엔드포인트는 사용자 개인 키를 보유하지 않으므로 항상 오류를 반환합니다. 트랜잭션은 클라이언트 측에서 서명하고 `eth_sendRawTransaction`을 통해 제출하세요.
- **매핑되지 않은 메서드** — 명시적으로 허용 목록에 없는 항목은 `400`을 반환합니다. 추가를 요청하려면 지원팀에 문의하세요.

## 배치 항목별 청구

HTTP 응답이 `200`인 경우에도 개별 배치 항목이 JSON-RPC `error` 필드와 함께 반환될 수 있습니다(예: 잘못된 파라미터 오류 또는 대상 체인에서 지원되지 않는 메서드). Venice는 이러한 항목을 전체 메서드 등급 대신 **항목당 5 크레딧**으로 청구합니다 — 일반적인 "API 탐색 중" 실수에 대한 작은 양보입니다.

```json
// 배치 요청:
[
  { "jsonrpc": "2.0", "method": "eth_chainId",   "params": [],         "id": 1 },
  { "jsonrpc": "2.0", "method": "eth_getBalance","params": ["bad"],    "id": 2 }
]

// 응답 (HTTP 200, X-Venice-RPC-Credits: 25):
[
  { "jsonrpc": "2.0", "id": 1, "result": "0x1" },
  { "jsonrpc": "2.0", "id": 2, "error": { "code": -32602, "message": "invalid params" } }
]
```

첫 번째 항목(성공)은 20 크레딧, 두 번째 항목(RPC 레벨 오류)은 5 크레딧으로 청구되어 합계 `= 25`.

## 레이트 리밋

인증된 호출자당 분당 요청 한도:

| 등급 | 분당 요청 |
|---|---|
| Standard | 100 |
| Staff | 1,000 |

한도를 초과하면 엔드포인트는 `customMessage`와 표준 `X-RateLimit-*` 응답 헤더와 함께 `429`를 반환합니다.

## 멱등성(Idempotency)

`Idempotency-Key` 요청 헤더를 `[A-Za-z0-9_-]{1,255}`와 일치하는 모든 문자열로 설정하여 안전한 재시도를 활성화할 수 있습니다. 응답은 `(user, idempotency-key)`를 키로 24시간 동안 캐시됩니다.

- **동일한 키와 동일한 본문**으로 재생하면 캐시된 응답과 `Idempotent-Replayed: true` 응답 헤더가 반환됩니다. 업스트림이 호출되지 않으며 새 크레딧이 청구되지 않습니다.
- **동일한 키와 다른 본문**으로 재생하면 무음 상태 손상을 방지하기 위해 `400`을 반환합니다. 별개의 요청에는 새 키를 선택하세요.

## 응답 헤더

| 헤더 | 설명 |
|---|---|
| `X-Venice-RPC-Credits` | 이 요청에 청구된 크레딧. 배치 요청의 경우, 항목 합계입니다. |
| `X-Venice-RPC-Cost-USD` | 소수점 8자리까지의 달러 비용. `X-Venice-RPC-Credits × 크레딧당 가격`과 같습니다. |
| `X-Request-ID` | 32자 상관관계 ID. 지원 문의 시 포함하세요. |
| `Idempotent-Replayed` | 응답이 멱등성 캐시에서 온 경우 값 `"true"`로 표시됩니다. |
| `X-RateLimit-Limit` / `X-RateLimit-Remaining` / `X-RateLimit-Reset` | 429 응답에서만 설정됩니다. |

## 트랜잭션 릴레이를 위한 포렌식 로깅

`eth_sendRawTransaction`에 대한 모든 호출은 tx 해시(원시 바이트의 keccak256), 네트워크 slug, 요청 ID 및 호출 사용자 ID와 함께 서버 측에 기록됩니다. 서명된 페이로드 자체는 보관하지 **않습니다** — 해시는 온체인 영수증에서 복구할 수 있습니다. 이 감사 추적은 고객의 API 키가 침해되어 우리 인프라를 통해 불법 거래를 중계하는 데 사용된 경우, 온체인 활동을 책임 있는 계정으로 추적할 수 있도록 존재합니다.

## 예제

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/ethereum-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_chainId",
    "params": [],
    "id": 1
  }'
```

```json
{ "jsonrpc": "2.0", "id": 1, "result": "0x1" }
```

응답 헤더: `X-Venice-RPC-Credits: 20`, `X-Venice-RPC-Cost-USD: 0.00001250`, `X-Request-ID: <nanoid>`.

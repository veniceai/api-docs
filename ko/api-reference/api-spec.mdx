Venice API는 검열되지 않은 모델과 private 추론으로 AI 애플리케이션을 구축하기 위한 HTTP 기반 REST 및 스트리밍 인터페이스를 제공합니다. 제한적인 콘텐츠 정책 없이 텍스트 생성, 이미지 제작, embeddings 등을 만들 수 있습니다. 통합 예제 및 SDK는 [문서](/ko/overview/getting-started)에서 제공됩니다. API 참조는 [OpenAPI YAML 스펙](https://api.venice.ai/doc/api/swagger.yaml)으로도 사용할 수 있습니다.

## 인증

Venice API는 인증에 API 키를 사용합니다. [API 설정](https://venice.ai/settings/api)에서 API 키를 생성하고 관리하세요.


모든 API 요청에는 HTTP Bearer 인증이 필요합니다.

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
API 키는 비밀입니다. 공유하거나 클라이언트 측 코드에 노출하지 마세요.
</Note>

## OpenAI 호환성

Venice의 API는 OpenAI API 사양을 구현하므로 기존 OpenAI 클라이언트 및 도구와의 호환성을 보장합니다. 이를 통해 친숙한 OpenAI 인터페이스를 사용하면서 Venice의 고유 기능과 검열되지 않은 모델에 접근할 수 있습니다.

### 설정

Venice의 베이스 URL(`https://api.venice.ai/api/v1`)을 사용하도록 클라이언트를 구성하고 첫 요청을 보내세요.

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## Venice 전용 기능

### 시스템 프롬프트

Venice는 검열되지 않고 자연스러운 모델 응답을 보장하도록 설계된 기본 시스템 프롬프트를 제공합니다. 시스템 프롬프트 처리에는 두 가지 옵션이 있습니다.

1. **기본 동작**: 시스템 프롬프트가 Venice의 기본값에 추가됩니다.
2. **사용자 지정 동작**: Venice의 시스템 프롬프트를 완전히 비활성화합니다.

#### Venice 시스템 프롬프트 비활성화

Venice의 기본 시스템 프롬프트를 제거하려면 `venice_parameters` 옵션을 사용하세요.

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### Venice 파라미터

`venice_parameters` 객체를 통해 표준 OpenAI API에서 사용할 수 없는 Venice 전용 기능에 접근할 수 있습니다.

| 파라미터 | 타입 | 설명 | 기본값 |
|-----------|------|-------------|---------|
| `character_slug` | string | 공개 Venice 캐릭터의 캐릭터 slug(발행된 캐릭터 페이지에서 "Public ID"로 확인 가능) | - |
| `strip_thinking_response` | boolean | 응답에서 `<think></think>` 블록을 제거(레거시 `<think>` 태그 형식을 사용하는 모델). [추론 모델](/ko/guides/features/reasoning-models)을 참고하세요. | `false` |
| `disable_thinking` | boolean | 지원되는 추론 모델에서 thinking을 비활성화하고 응답에서 `<think></think>` 블록을 제거 | `false` |
| `enable_web_search` | string | 이 요청에 대해 웹 검색 활성화(`off`, `on`, `auto` - auto는 모델 판단에 따라 활성화)<br/>추가 사용 기반 요금이 적용됩니다. [가격](/ko/overview/pricing#web-search-and-scraping)을 참고하세요. | `off` |
| `enable_web_scraping` | boolean | 사용자 메시지에서 감지된 최대 5개 URL의 웹 스크레이핑 활성화. 스크레이프된 콘텐츠는 응답을 보강하고 웹 검색을 우회합니다. 성공적으로 스크레이프된 URL만 청구됩니다.<br/>추가 사용 기반 요금이 적용됩니다. [가격](/ko/overview/pricing#web-search-and-scraping)을 참고하세요. | `false` |
| `enable_x_search` | boolean | 지원되는 Grok 모델(예: `grok-4-20-beta`)에 대해 xAI의 네이티브 검색(web + X/Twitter) 활성화. xAI의 검색 인프라를 사용하여 더 높은 품질의 검색 결과를 제공합니다. 활성화되면 Venice의 표준 웹 검색은 우회됩니다.<br/>추가 사용 기반 요금이 적용됩니다. [가격](/ko/overview/pricing#web-search-and-scraping)을 참고하세요. | `false` |
| `enable_web_citations` | boolean | 웹 검색이 활성화된 경우, LLM이 `[REF]0[/REF]` 형식으로 출처를 인용하도록 요청 | `false` |
| `include_search_results_in_stream` | boolean | 실험적: 스트림의 첫 번째 청크로 검색 결과 포함 | `false` |
| `return_search_results_as_documents` | boolean | LangChain 통합을 위해 OpenAI 호환 tool call `venice_web_search_documents`로 검색 결과 노출 | `false` |
| `include_venice_system_prompt` | boolean | 지정된 시스템 프롬프트와 함께 Venice의 기본 시스템 프롬프트를 포함할지 여부 | `true` |

<Note>
이러한 파라미터는 모델 이름에 추가된 모델 접미사로도 지정할 수 있습니다(예: `zai-org-glm-5:enable_web_search=auto`). 자세한 내용은 [모델 기능 접미사](/ko/api-reference/endpoint/chat/model_feature_suffix)를 참고하세요.
</Note>

### 프롬프트 캐싱

Venice는 반복되는 콘텐츠에 대한 지연 시간과 비용을 줄이기 위해 특정 모델에서 프롬프트 캐싱을 지원합니다. 지원되는 모델의 경우, Venice는 시스템 프롬프트를 자동으로 캐시합니다 — 코드 변경이 필요하지 않습니다. 메시지 콘텐츠의 `cache_control` 속성을 사용해 콘텐츠를 캐싱하도록 수동으로 표시할 수도 있습니다.

| 파라미터 | 타입 | 설명 |
|-----------|------|-------------|
| `prompt_cache_key` | string | 캐시 적중률을 개선하기 위한 선택적 라우팅 힌트. 제공되면 Venice는 요청을 동일한 백엔드 인프라로 라우팅하여 다중 턴 대화 전반에 걸쳐 캐시 적중 가능성을 높입니다. |

캐싱 작동 방식, 청구 및 모범 사례에 대한 자세한 내용은 [프롬프트 캐싱](/ko/guides/features/prompt-caching)을 참고하세요.

## 응답 헤더 참조

모든 Venice API 응답에는 요청, 레이트 리밋, 모델 정보 및 계정 잔액에 대한 메타데이터를 제공하는 HTTP 헤더가 포함됩니다. API 응답에서 반환되는 오류 코드 외에도, 이러한 헤더를 검사하여 특정 API 요청의 고유 ID를 얻고, 레이트 리밋을 모니터링하며, 계정 잔액을 추적할 수 있습니다.

Venice는 필요할 경우 지원팀과의 효율적인 문제 해결을 위해 프로덕션 배포에서 요청 ID(`CF-RAY` 헤더)를 로깅하는 것을 권장합니다.

아래 표는 만날 수 있는 모든 헤더에 대한 종합적인 참조를 제공합니다.

| 헤더 | 타입 | 목적 | 반환 시점 |
|--------|------|---------|---------------|
| **표준 HTTP 헤더** ||||
| `Content-Type` | string | 응답 본문의 MIME 타입(`application/json`, `text/csv`, `image/png` 등) | 항상 |
| `Content-Encoding` | string | 응답 본문을 압축하는 데 사용된 인코딩(`gzip`, `br`) | 클라이언트가 `Accept-Encoding` 헤더를 보낼 때 |
| `Content-Disposition` | string | 콘텐츠 표시 방법(예: `attachment; filename=export.csv`) | 파일 또는 익스포트를 다운로드할 때 |
| `Date` | string | 응답이 생성된 RFC 7231 형식 타임스탬프 | 항상 |
| **요청 식별** ||||
| `CF-RAY` | string | 이 API 요청에 대한 고유 식별자, 문제 해결 및 지원 요청에 사용 | 항상 |
| `x-venice-version` | string | Venice API 서비스의 현재 버전/리비전 (예: `20250828.222653`) | 항상 |
| `x-venice-timestamp` | string | 요청이 처리될 때의 서버 타임스탬프 (ISO 8601 형식) | 타임스탬프 추적이 활성화된 경우 |
| `x-venice-host-name` | string | 요청을 처리한 서버의 호스트 이름 | 오류 응답 및 디버깅 시나리오 |
| **모델 정보** ||||
| `x-venice-model-id` | string | 요청에 사용된 AI 모델의 고유 식별자 (예: `venice-01-lite`) | AI 모델을 사용하는 추론 엔드포인트 |
| `x-venice-model-name` | string | 사용된 AI 모델의 친숙한/표시 이름 (예: `Venice Lite`) | AI 모델을 사용하는 추론 엔드포인트 |
| `x-venice-model-router` | string | 모델 추론을 처리한 라우터/백엔드 서비스 | 라우팅 정보를 사용할 수 있는 추론 엔드포인트 |
| `x-venice-model-deprecation-warning` | string | 폐기 예정인 모델에 대한 경고 메시지 | 사용 중단된 모델을 사용할 때 |
| `x-venice-model-deprecation-date` | string | 모델이 사용 중단되는 날짜 (ISO 8601 날짜) | 사용 중단된 모델을 사용할 때 |
| **레이트 리밋 정보** ||||
| `x-ratelimit-limit-requests` | number | 현재 시간 윈도우에서 허용되는 최대 요청 수 | 모든 인증된 요청 |
| `x-ratelimit-remaining-requests` | number | 현재 시간 윈도우에 남은 요청 수 | 모든 인증된 요청 |
| `x-ratelimit-reset-requests` | number | 요청 레이트 리밋이 리셋되는 Unix 타임스탬프 | 모든 인증된 요청 |
| `x-ratelimit-limit-tokens` | number | 시간 윈도우에서 허용되는 최대 토큰 수(prompt + completion) | 모든 인증된 요청 |
| `x-ratelimit-remaining-tokens` | number | 현재 시간 윈도우에 남은 토큰 수 | 모든 인증된 요청 |
| `x-ratelimit-reset-tokens` | number | 토큰 레이트 리밋이 리셋될 때까지의 초 단위 시간 | 모든 인증된 요청 |
| `x-ratelimit-type` | string | 적용되는 레이트 리밋 유형 (`user`, `api_key`, `global`) | 레이트 리밋이 적용될 때 |
| **페이지네이션 헤더** ||||
| `x-pagination-limit` | number | 페이지당 항목 수 | 페이지네이션된 엔드포인트 |
| `x-pagination-page` | number | 현재 페이지 번호 (1-based) | 페이지네이션된 엔드포인트 |
| `x-pagination-total` | number | 모든 페이지에 걸친 총 항목 수 | 페이지네이션된 엔드포인트 |
| `x-pagination-total-pages` | number | 총 페이지 수 | 페이지네이션된 엔드포인트 |
| **계정 잔액 정보** ||||
| `x-venice-balance-diem` | string | 요청 처리 전 DIEM 토큰 잔액 | 모든 인증된 요청 |
| `x-venice-balance-usd` | string | 요청 처리 전 USD 크레딧 잔액 | 모든 인증된 요청 |
| **콘텐츠 안전 헤더** ||||
| `x-venice-is-blurred` | string | 콘텐츠 정책으로 인해 생성된 이미지가 블러 처리되었는지 표시 (`true`/`false`) | Safe Venice가 활성화된 이미지 생성 |
| `x-venice-is-content-violation` | string | 콘텐츠가 Venice의 콘텐츠 정책을 위반하는지 표시 (`true`/`false`) | 콘텐츠 생성 엔드포인트 |
| `x-venice-is-adult-model-content-violation` | string | 콘텐츠가 성인 모델 콘텐츠 정책을 위반하는지 표시 (`true`/`false`) | 이미지 생성 엔드포인트 |
| `x-venice-contains-minor` | string | 이미지에 미성년자가 포함되어 있는지 표시 (`true`/`false`) | 연령 감지가 있는 이미지 분석 엔드포인트 |
| **클라이언트 정보** ||||
| `x-venice-middleface-version` | string | Venice middleface 클라이언트의 버전 | Venice middleface 클라이언트의 요청 |
| `x-venice-mobile-version` | string | Venice 모바일 앱 클라이언트의 버전 | 모바일 애플리케이션의 요청 |
| `x-venice-request-timestamp-ms` | number | 클라이언트가 제공한 밀리초 단위 요청 타임스탬프 | 클라이언트가 요청에서 타임스탬프를 제공할 때 |
| `x-venice-control-instance` | string | 디버깅을 위한 컨트롤 인스턴스 식별자 | 디버깅을 위한 이미지 생성 엔드포인트 |
| **인증 헤더** ||||
| `x-auth-refreshed` | string | 요청 중에 인증 토큰이 갱신되었는지 표시 (`true`/`false`) | 인증 토큰이 자동 갱신될 때 |
| `x-retry-count` | number | 요청에 대한 재시도 횟수 | 요청 재시도가 발생할 때 |

### 중요 사항

- **헤더 이름 대소문자**: HTTP 헤더는 대소문자를 구분하지 않지만, Venice는 일관성을 위해 소문자와 하이픈을 사용합니다.
- **문자열 값**: 헤더의 boolean 값은 문자열(`"true"` 또는 `"false"`)로 반환됩니다.
- **숫자 값**: 큰 숫자와 잔액 값은 정밀도 손실을 방지하기 위해 문자열로 반환될 수 있습니다.
- **선택적 헤더**: 모든 헤더가 모든 응답에 반환되는 것은 아닙니다. 존재 여부는 엔드포인트와 요청 컨텍스트에 따라 다릅니다.
- **압축**: 지원되는 곳에서 압축된 응답을 받으려면 요청에 `Accept-Encoding: gzip, br`을 사용하세요.

### 예제: 응답 헤더 접근

```javascript
// API 요청 후 응답 객체에서 헤더에 접근
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// 모델 사용 중단 경고 확인
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## 모범 사례

1. **레이트 리밋**: `x-ratelimit-remaining-requests` 및 `x-ratelimit-remaining-tokens` 헤더를 모니터링하고 지수 백오프를 구현하세요.
2. **잔액 모니터링**: 서비스 중단을 피하기 위해 `x-venice-balance-usd` 및 `x-venice-balance-diem` 헤더를 추적하세요.
3. **시스템 프롬프트**: Venice의 시스템 프롬프트를 사용한 경우와 사용하지 않은 경우를 모두 테스트하여 사용 사례에 가장 적합한 것을 찾으세요.
4. **API 키**: API 키를 안전하게 보관하고 정기적으로 교체하세요.
5. **요청 로깅**: 지원팀과의 문제 해결을 위해 `CF-RAY` 헤더 값을 로깅하세요.
6. **모델 사용 중단**: 모델을 사용할 때 `x-venice-model-deprecation-warning` 헤더를 확인하세요.

## OpenAI API와의 차이점

Venice는 OpenAI API 사양과 높은 호환성을 유지하지만 몇 가지 주요 차이점이 있습니다.

1. **venice_parameters**: 확장 기능을 위한 `enable_web_search`, `character_slug`, `strip_thinking_response`와 같은 추가 구성
2. **시스템 프롬프트**: Venice는 검열되지 않은 응답에 최적화된 기본값에 시스템 프롬프트를 추가합니다(`include_venice_system_prompt: false`로 비활성화)
3. **모델 생태계**: Venice는 검열되지 않은 모델과 추론 모델을 포함한 자체 [모델 라인업](/ko/overview/models)을 제공합니다 - OpenAI 매핑 대신 Venice 모델 ID를 사용하세요
4. **응답 헤더**: 잔액 추적(`x-venice-balance-usd`, `x-venice-balance-diem`), 모델 사용 중단 경고 및 콘텐츠 안전 플래그에 대한 고유 헤더
5. **콘텐츠 정책**: 전용 검열되지 않은 모델과 선택적 콘텐츠 필터링이 있는 더 허용적인 정책

## API 안정성

Venice는 v1 엔드포인트와 파라미터에 대해 하위 호환성을 유지합니다. 모델 라이프사이클 정책, 사용 중단 공지 및 마이그레이션 가이드는 [사용 중단](/ko/overview/deprecations)을 참고하세요.

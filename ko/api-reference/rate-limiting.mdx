## 오류 처리

실패한 요청(500, 503, 429)은 지수 백오프(exponential backoff)로 재시도해야 합니다.

특히 429 오류의 경우, 재시도할 수 있는 정확한 Unix 타임스탬프는 `x-ratelimit-reset-requests` 헤더를 확인하세요. 대부분의 HTTP 라이브러리에는 이를 자동으로 처리하는 내장 재시도 메커니즘이 있습니다.

### 남용 방지

30초 이내에 20회 이상의 실패한 요청을 생성하면, API는 추가 요청을 30초 동안 차단합니다.

```
Too many failed attempts (> 20) resulting in a non-success status code. Please wait 30s and try again.
```

## 응답 헤더

모든 응답에는 다음 헤더가 포함됩니다.

| 헤더 | 설명 |
|:-------|:------------|
| `x-ratelimit-limit-requests` | 현재 윈도우에서 허용되는 최대 요청 수 |
| `x-ratelimit-remaining-requests` | 현재 윈도우에 남은 요청 수 |
| `x-ratelimit-reset-requests` | 윈도우가 리셋되는 Unix 타임스탬프 |
| `x-ratelimit-limit-tokens` | 분당 허용되는 최대 토큰 수 |
| `x-ratelimit-remaining-tokens` | 현재 분에 남은 토큰 수 |
| `x-ratelimit-reset-tokens` | 토큰 한도가 리셋될 때까지의 초 |

레이트 리밋에 지속적으로 도달하고 사용 패턴이 **시간이 지남에 따라 지속적인 수요**를 보이는 경우, Partner 접근에 대해 논의하려면 [api@venice.ai](mailto:api@venice.ai)로 연락하세요.

Partner 등급 한도는 특정 요구 사항에 따라 조정될 수 있습니다.

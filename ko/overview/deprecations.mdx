Venice API는 개발자에게 숨겨진 필터나 블랙박스 결정 없이 프로덕션 등급 모델에 대한 무제한 private 접근을 제공하기 위해 존재합니다.

모델이 개선됨에 따라, 우리는 가끔 더 똑똑하거나 빠르거나 능력 있는 대안을 위해 오래된 모델을 폐기합니다. 우리는 이러한 전환을 예측 가능하고 마찰이 적도록 설계합니다.

## 모델 사용 중단

사용 중단이 혼란을 줄 수 있다는 것을 알고 있습니다. 그래서 우리는 필요할 때만 사용 중단을 목표로 하며, 혼란을 최소화하기 위해 traits 및 Venice 브랜드 모델과 같은 기능을 설계합니다.

다음 경우에 모델을 사용 중단할 수 있습니다.

- 새로운 모델이 동일한 사용 사례에 대해 명확한 개선을 제공할 때
- 모델이 더 이상 성능 또는 안정성 기준을 충족하지 못할 때
- 지속적으로 낮은 사용량을 보이고, 계속 지원하면 다른 모든 사용자의 경험이 분산될 때

## 사용 중단 프로세스

모델이 사용 중단 기준을 충족하면 30~60일 전에 변경을 공지합니다. 사용 중단 공지는 [changelog](https://featurebase.venice.ai/changelog)와 [Discord 서버](https://discord.gg/askvenice)를 통해 게시됩니다. 공지 기간 동안 사용 중단된 모델을 호출하면 API 응답에 사용 중단 경고가 포함됩니다.

공지 기간 동안 모델은 계속 사용 가능하지만, 경우에 따라 인프라 용량을 줄일 수 있습니다. 우리는 항상 권장 대체 모델을 제공하며, 필요한 경우 전환을 돕기 위한 마이그레이션 가이드를 제공합니다.

종료일 이후 해당 모델로의 요청은 동일하거나 더 낮은 가격의 비슷한 처리 능력을 가진 모델로 자동 라우팅됩니다. 기술적 또는 안전상의 이유로 라우팅이 불가능한 경우, API는 410 Gone 응답을 반환합니다. trait(예: `default_code`, `default_vision` 또는 `fastest`)을 통해 사용 중단된 모델이 선택된 경우, 해당 trait는 호환 가능한 대체 모델로 재할당됩니다.

우리는 모델을 조용히 제거하거나 버전 관리 없이 동작을 변경하지 않습니다. 무엇이 실행되고 있는지, 다음에 무엇을 준비해야 하는지 항상 알 수 있습니다.

<Note>
성능 전용 업그레이드: 모델 동작을 유지하면서 성능, 지연 시간 또는 비용 효율성을 개선하는 업그레이드를 출시할 수 있습니다. 이러한 업데이트는 하위 호환되며 고객의 조치가 필요하지 않습니다.
</Note>

아래의 [모델 사용 중단 트래커](#model-deprecation-tracker)를 참고하세요. 이전 공지는 [changelog](https://featurebase.venice.ai/changelog)와 [Discord 서버](https://discord.gg/askvenice)를 참고하세요.

## Venice API에서 모델이 선정되는 방식

우리는 성능, 안정성 및 실제 개발자 요구를 기반으로 어떤 모델을 사용 가능하게 할지 신중하게 선택합니다. 포함되려면 모델은 강력한 성능을 입증하고, OpenAI 호환 엔드포인트에서 일관되게 동작하며, 이미 지원하는 모델 중 적어도 하나에 비해 명확한 개선을 제공해야 합니다.


평가 중인 모델은 먼저 피드백을 수집하고 대규모 성능을 검증하기 위해 [베타](/ko/overview/beta-models)로 출시될 수 있습니다.

우리는 중복되거나, 검증되지 않았거나, 일관된 프로덕션 사용에 준비되지 않은 모델을 노출하지 않습니다. 우리의 목표는 Venice API를 깔끔하고, 능력 있고, 개발자가 실제로 구축하는 것에 최적화된 상태로 유지하는 것입니다.

자세한 내용은 [모델 사용 중단](/ko/overview/deprecations#model-deprecations) 및 <a href="/ko/overview/models" target="_blank" rel="noopener noreferrer">현재 모델 목록</a>을 참고하세요.

## 버전 관리 및 별칭

모든 Venice 모델은 고유하고 영구적인 ID로 식별됩니다. 예:

```venice-uncensored```
```zai-org-glm-4.7```
```zai-org-glm-5```
```qwen3-vl-235b-a22b```

모델 ID는 안정적입니다. 호환성을 깨는 변경이 있는 경우, 새 모델 ID를 출시합니다(예: v2와 같은 버전 추가). 호환성을 깨는 변경이 없는 경우, 기존 모델을 업데이트할 수 있으며 중요한 변경 사항을 알려드립니다.

유연성을 제공하기 위해 Venice는 또한 traits를 통해 구현된 심볼릭 별칭을 유지하며, 이는 주어진 작업에 대한 권장 기본 모델을 가리킵니다.

<div id="traits-list-placeholder"></div>

Traits는 모델 선택을 위한 안정적인 추상화를 제공하면서 Venice가 기본 구현을 개선할 수 있는 유연성을 제공합니다. 최신 권장 모델에 자동 접근을 선호하는 개발자는 trait 기반 별칭에 의존할 수 있습니다.

엄격한 일관성과 예측 가능한 동작이 필요한 애플리케이션의 경우, 고정된 모델 ID를 참조하는 것을 권장합니다.

## 피드백

[Featurebase 포털](https://featurebase.venice.ai)을 통해 피드백이나 요청을 제출할 수 있습니다. 우리는 공개 [changelog](https://featurebase.venice.ai/changelog), 로드맵 트래커, 그리고 모델 추가, 업그레이드 또는 제거에 대한 투명한 근거를 유지하며, 지속적인 커뮤니티 참여를 장려합니다.


## 모델 사용 중단 트래커

다음 모델은 사용 중단이 예정되어 있거나 최근 사용 중단되었습니다. 제거 날짜 이전에 권장 대체 모델로 마이그레이션할 것을 권장합니다. 모델은 제거 날짜 이후 30일 동안 목록에 남아 있습니다.

### API를 통한 사용 중단 상태 확인

[모델 목록](/ko/api-reference/endpoint/models/list) 엔드포인트를 호출하여 모델의 폐기 예정 여부를 확인할 수 있습니다. 폐기 예정일이 있는 모델은 `model_spec`에 `deprecation` 객체를 포함합니다.

```json
{
  "id": "some-model-id",
  "model_spec": {
    "name": "Some Model",
    "privacy": "private",
    "deprecation": {
      "date": "2025-03-01T00:00:00.000Z"
    }
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

`deprecation` 객체는 모델이 폐기 예정일 때만 나타납니다. `if (model.model_spec.deprecation)`을 확인하여 모델이 폐기 중인지 알 수 있으며, ISO 8601 날짜를 사용하여 사용자에게 경고하거나 마이그레이션을 계획할 수 있습니다.

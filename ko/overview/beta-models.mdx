피드백을 수집하고 전체 프로덕션 배포 전에 성능을 확인하기 위해 모델을 베타로 출시하기도 합니다. 베타 모델은 모든 사용자가 사용할 수 있지만 **프로덕션 사용에는 권장되지 않습니다**.

베타 상태가 프로덕션 승격을 보장하지는 않습니다. 베타 모델은 실행 비용이 너무 높거나, 대규모에서 성능이 저조하거나, 안전 문제가 제기되는 경우 제거될 수 있습니다. 베타 모델은 사전 공지 없이 변경될 수 있으며 문서나 지원이 제한적일 수 있습니다. 안정적이고 광범위하게 유용하며 우리의 표준에 부합하는 것으로 입증된 모델은 정식 출시(GA)로 승격됩니다.

## 중요 고려 사항

베타 모델을 사용할 때 다음 사항을 염두에 두세요.

- 표준 사용 중단 공지 기간 없이 언제든지 변경되거나 제거될 수 있습니다
- 프로덕션 애플리케이션이나 중요한 워크플로우에 적합하지 않습니다
- 일관되지 않은 성능, 가용성 또는 동작을 보일 수 있습니다
- 제거 시 마이그레이션 지원이 제한적이거나 없습니다
- 테스트, 평가 및 실험 프로젝트에 가장 적합합니다

프로덕션 애플리케이션의 경우, [메인 모델 라인업](/ko/models/overview)의 안정적인 모델을 사용하는 것을 권장합니다.

### API를 통한 베타 상태 확인

[모델 목록](/ko/api-reference/endpoint/models/list) 엔드포인트를 호출하여 모델이 베타인지 확인할 수 있습니다. 베타 모델은 `model_spec`에 `betaModel` 필드가 `true`로 설정되어 있습니다.

```json
{
  "id": "some-beta-model",
  "model_spec": {
    "name": "Some Beta Model",
    "betaModel": true,
    "privacy": "private"
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

`if (model.model_spec.betaModel)`을 확인하여 베타 모델을 식별하고 사용자에게 경고하거나 애플리케이션에서 다르게 처리할 수 있습니다.

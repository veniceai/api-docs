VVV와 DIEM은 Venice API를 위한 온체인 자금 조달 계층입니다. DIEM을 스테이킹하면 매 요청마다 선불 USD 크레딧을 구매하지 않고도 고정된 일일 인퍼런스 할당량을 받을 수 있습니다 — **1 DIEM = 하루 $1** 상당의 Venice API 크레딧입니다.

| 토큰 | 역할 |
| --- | --- |
| **VVV** | Base 상의 Venice 기반 토큰입니다. 스테이킹하여 수익을 얻고 DIEM을 민팅할 수 있습니다. |
| **sVVV** | 스테이킹된 VVV에 대한 영수증입니다. 현재 [Mint Rate](https://diem-calculator.venice.ai/)에 따라 sVVV를 락업하여 DIEM을 민팅합니다. |
| **DIEM** | 토큰화된 컴퓨트 단위입니다. DIEM을 스테이킹하면 DIEM당 하루 $1에 해당하는 영구적인 일일 API 크레딧이 활성화됩니다. |

DIEM은 Base 상의 ERC-20 토큰입니다. 전송하거나 거래하거나 스테이킹할 수 있습니다. 사용한 만큼 지불하는 USD 크레딧과 달리, 스테이킹된 DIEM은 사용자가 소유하는 용량입니다. 일일 할당량은 매 에포크마다 갱신되며 네트워크 사용량이 증가해도 희석되지 않습니다.

<Info>
  해당 할당량에서 청구되는 모델 요율은 [API Pricing](/overview/pricing)을 참고하세요. API 키 없이 지갑으로 요청당 결제하는 방식은 [x402](/guides/integrations/x402-venice-api)를 확인하세요.
</Info>

## 자금 조달 방식

API 키는 연결된 Venice 계정에서 다음 순서로 차감됩니다: **DIEM**, 그다음 번들 크레딧, 그다음 USD.

DIEM이 활성 통화일 때:

* 일일 할당량은 스테이킹한 DIEM 수량과 같습니다 (스테이킹된 1 DIEM → 해당 에포크의 API 크레딧 $1).
* 에포크 내에서 사용하지 않은 DIEM은 이월되지 않습니다. 할당량은 **00:00 UTC**에 갱신됩니다.
* DIEM 잔액을 사용하려면 계정에 최소 **0.1** DIEM이 스테이킹되어 있어야 합니다.
* 요청 비용은 사용한 만큼 지불하는 방식과 동일한 USD 가격표를 사용합니다. DIEM은 그날의 할당량에 대해 1:1 달러 환율로 결제되는 통화입니다.

현재 및 다음 에포크의 DIEM 잔액은 [venice.ai/settings/api](https://venice.ai/settings/api) 대시보드에서 확인하거나 [Billing Balance](/api-reference/endpoint/billing/balance) 엔드포인트를 통해 조회할 수 있습니다.

## DIEM 획득 및 스테이킹

[venice.ai/token](https://venice.ai/token)에서 토큰 대시보드를 사용하세요. Base 지갑을 연결하세요 — 토큰을 컨트랙트 주소로 직접 전송하지 마세요.

<Steps>
  <Step title="VVV 또는 DIEM 획득">
    [Aerodrome](https://aerodrome.finance)이나 [Uniswap](https://app.uniswap.org)과 같은 Base DEX에서 VVV 또는 DIEM을 구매하거나, 다른 지갑에서 전송된 DIEM을 받으세요.

    | 자산 | 컨트랙트 (Base) |
    | --- | --- |
    | VVV | [`0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf`](https://basescan.org/token/0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf) |
    | DIEM | [`0xF4d97F2da56e8c3098f3a8D538DB630A2606a024`](https://basescan.org/token/0xF4d97F2da56e8c3098f3a8D538DB630A2606a024) |
  </Step>

  <Step title="VVV 스테이킹 (DIEM을 민팅하는 경우)">
    [venice.ai/token](https://venice.ai/token)에서 VVV를 스테이킹하여 sVVV를 받으세요. 스테이킹된 VVV는 이미션 보상을 획득합니다. 스테이킹 컨트랙트는 [`0x321b7ff75154472B18EDb199033fF4D116F340Ff`](https://basescan.org/address/0x321b7ff75154472B18EDb199033fF4D116F340Ff#code)입니다.
  </Step>

  <Step title="sVVV로 DIEM 민팅 (선택 사항)">
    현재 Mint Rate에 따라 sVVV를 락업하여 DIEM을 민팅하세요. Mint Rate는 DIEM 하나를 민팅하는 데 필요한 sVVV의 양이며 DIEM 공급량이 증가함에 따라 상승합니다 — [DIEM calculator](https://diem-calculator.venice.ai/)에서 미리 확인할 수 있습니다.

    민팅된 DIEM은 ERC-20으로 지갑에 표시됩니다. 락업된 sVVV는 동일한 양의 DIEM을 소각하여 해제할 때까지 락업된 상태로 유지됩니다.
  </Step>

  <Step title="API 크레딧을 위한 DIEM 스테이킹">
    동일한 토큰 대시보드에서 DIEM을 스테이킹하세요. 스테이킹된 DIEM 하나당 스테이킹된 동안 **하루 $1**의 Venice API 크레딧이 부여됩니다.

    DIEM 언스테이킹에는 **1일**의 쿨다운이 있습니다. 민팅된 DIEM을 뒷받침한 sVVV의 잠금을 해제하려면 해당 DIEM을 소각해야 하며, 해제된 sVVV를 언스테이킹하는 데는 **7일**의 쿨다운이 있습니다.
  </Step>

  <Step title="API 키 생성 또는 사용">
    동일한 지갑으로 Venice에 로그인하여(Sign-In-With-Ethereum) 스테이킹된 DIEM이 계정에 연결되도록 한 다음, [API settings](https://venice.ai/settings/api)에서 또는 [API 키 가이드](/guides/getting-started/generating-api-key)를 참고하여 키를 생성하세요.

    자율 에이전트는 스테이킹된 VVV를 보유한 지갑에서 키를 민팅할 수 있습니다 — [Autonomous Agent API Key Creation](/guides/getting-started/generating-api-key-agent)을 참조하세요.
  </Step>
</Steps>

<Warning>
  VVV 또는 DIEM을 컨트랙트 주소로 직접 전송하지 마세요. 그렇게 전송된 토큰은 복구할 수 없습니다. 항상 [venice.ai/token](https://venice.ai/token) 또는 알려진 스테이킹 트랜잭션 플로우를 통해 스테이킹하세요.
</Warning>

## API와 함께 DIEM 사용하기

DIEM이 스테이킹되고 계정에 연결되면 일반 Bearer 키로 API를 호출하세요. 추가 헤더나 `venice_parameters`가 필요하지 않습니다 — `consumptionCurrency`가 `DIEM`이면 Venice가 자동으로 DIEM에서 차감합니다.

```bash
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### DIEM 잔액 확인

```bash
curl https://api.venice.ai/api/v1/billing/balance \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

응답 형식 예시:

```json
{
  "canConsume": true,
  "consumptionCurrency": "DIEM",
  "balances": {
    "diem": 90.5,
    "usd": 25
  },
  "diemEpochAllocation": 100
}
```

| 필드 | 의미 |
| --- | --- |
| `balances.diem` | 현재 에포크에서 남은 DIEM 크레딧 (DIEM을 스테이킹하지 않은 경우 `null`) |
| `diemEpochAllocation` | 이 에포크에 할당된 총 DIEM (스테이킹된 금액). 지금까지의 사용량을 확인하려면 `balances.diem`과 비교하세요 |
| `consumptionCurrency` | 다음에 청구될 통화 (`DIEM`, `USD`, `BUNDLED_CREDITS`, 또는 레거시 `VCU`) |
| `canConsume` | 계정이 현재 요청 비용을 지불할 수 있는지 여부 |

모델 목록 조회 또는 사용 분석을 통해 요청당 비용을 USD와 DIEM 양쪽으로 확인할 수 있습니다 — 그날의 할당량에 대해 DIEM당 $1 비율로 일치합니다. [Billing Usage](/api-reference/endpoint/billing/usage) 및 [Usage Analytics](/api-reference/endpoint/billing/usage-analytics)를 참조하세요.

### 키별 지출 한도 설정

키를 생성할 때 `diem` 또는 `usd` 단위의 에포크 소비 한도를 설정하여 단일 통합이 하루 전체 할당량을 소진하지 못하도록 할 수 있습니다. 대시보드 키에서는 이를 **Epoch Consumption Limits**로 노출하며, Web3로 민팅된 키는 [`POST /api_keys/generate_web3_key`](/api-reference/endpoint/api_keys/generate_web3_key/post)에서 `consumptionLimit`을 받습니다.

## DIEM vs USD 크레딧 vs x402

| 경로 | 적합한 용도 | 갱신 방식 |
| --- | --- | --- |
| **스테이킹된 DIEM** | 예측 가능한 일일 용량, 에이전트, 고정된 AI 비용 | DIEM이 스테이킹된 동안 매 에포크(00:00 UTC)마다 전체 할당량 |
| **USD / 암호화폐 크레딧** | DIEM 할당량을 초과하는 급증, 간단한 선불 지출 | 사용량에 따라 잔액이 감소하며, 크레딧은 만료되지 않습니다 |
| **x402 (USDC)** | API 키 없이 헤드리스 요청당 결제 | 지갑은 `POST /x402/top-up`을 통해 USDC를 충전합니다. 연결된 계정의 DIEM이 있으면 여전히 먼저 사용됩니다 |

요청이 남은 DIEM을 초과하고 이를 커버할 USD 또는 번들 크레딧이 없으면 API는 `INSUFFICIENT_BALANCE`와 함께 `402`를 반환합니다. DIEM을 더 스테이킹하거나, 다음 에포크를 기다리거나, [Settings → API](https://venice.ai/settings/api)에서 USD 크레딧을 추가하세요.

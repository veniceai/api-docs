[Jan](https://jan.ai/)은 하나의 데스크톱 앱에서 로컬 및 원격 모델을 실행할 수 있는 오픈 소스 AI 어시스턴트입니다. Venice는 OpenAI 호환 원격 모델 제공자 지원을 통해 Jan에서 작동합니다.

## 사전 요구 사항

- [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 Venice API 키
- 설치되어 실행 중인 [Jan](https://jan.ai/)
- [모델 카탈로그](/models/text)의 Venice 텍스트 모델 ID

---

## 설정

<Steps>
  <Step title="Venice API 키 받기">
    [venice.ai/settings/api](https://venice.ai/settings/api)에서 API 키를 생성하거나 복사하세요.
  </Step>

  <Step title="Jan의 모델 제공자 설정 열기">
    Jan에서 **Settings**를 열고 **Model Providers**로 이동합니다.

    Jan의 OpenAI 호환 사용자 지정 제공자 흐름을 선택하세요. Jan 버전에 따라 **Custom Endpoint**가 활성화된 **OpenAI** 또는 **Any OpenAI Compatible API** 제공자로 나타날 수 있습니다.
  </Step>

  <Step title="Venice 엔드포인트 구성">
    Venice 제공자 설정을 입력하세요:

    | 필드 | 값 |
    |-------|-------|
    | 제공자 이름 | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API 키 | Venice API 키 |
    | API 유형 | OpenAI 호환 |

    모델을 추가하기 전에 제공자 설정을 저장하거나 적용하세요.
  </Step>

  <Step title="Venice 모델 추가">
    사용하려는 Venice 모델 ID를 추가합니다. Jan은 모델 ID가 제공자와 정확히 일치해야 합니다.

    좋은 시작 옵션:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    [모델 카탈로그](/models/text)에서 지원되는 텍스트 모델을 추가할 수 있습니다.
  </Step>

  <Step title="채팅 시작">
    새 또는 기존 Jan 채팅을 열고, 모델 선택기에서 Venice 모델 중 하나를 선택한 다음 테스트 메시지를 보내세요.

    Jan은 API 키와 OpenAI 호환 base URL을 사용하여 Venice로 요청을 보냅니다.
  </Step>
</Steps>

## 설정 확인

Jan에서 짧은 프롬프트를 보내세요:

```txt
Reply with one sentence confirming you are running through Venice.
```

제공자가 올바르게 구성되었다면 Jan은 선택한 Venice 모델에서 응답을 스트리밍해야 합니다.

또한 Jan 외부에서 다음과 같이 API 키를 확인할 수 있습니다:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

## 구성 참고 사항

- base URL로 `https://api.venice.ai/api/v1`을 사용하세요. `/chat/completions`를 포함하지 마세요; Jan이 채팅 요청에 대해 엔드포인트 경로를 추가합니다.
- 모델 ID는 Venice와 정확히 일치해야 합니다. Jan의 선택기에 모델이 표시되지 않으면 [모델 카탈로그](/models/text)의 모델 ID를 사용하여 수동으로 추가하세요.
- Venice를 OpenAI 호환 제공자로 구성된 상태로 유지하세요. Venice Claude 모델에 대해 Jan의 네이티브 Anthropic 제공자를 사용하지 마세요.
- 원격 Venice 요청에는 인터넷 연결이 필요하며 Venice API 크레딧을 소비합니다.

<Info>
Venice를 추가한 후에도 Jan의 로컬 모델은 계속 사용할 수 있습니다. 채팅 모델 선택기에서 로컬 모델과 Venice 모델 사이를 전환할 수 있습니다.
</Info>

## 문제 해결

<AccordionGroup>
  <Accordion title="401 또는 인증 오류">
    API 키가 [venice.ai/settings/api](https://venice.ai/settings/api)의 Venice 키이며 앞이나 뒤에 추가 공백이 없는지 확인하세요.
  </Accordion>

  <Accordion title="404 또는 모델을 찾을 수 없음">
    Jan의 모델 ID가 Venice 모델 ID와 정확히 일치하는지 확인하세요. 또한 base URL이 `https://api.venice.ai/api/v1`인지 확인하세요.
  </Accordion>

  <Accordion title="선택기에 Venice 모델이 없음">
    Jan의 제공자 설정에서 모델을 수동으로 추가하세요. `zai-org-glm-5-1`과 같이 정확한 모델 ID를 사용하고 제공자를 다시 저장하세요.
  </Accordion>

  <Accordion title="요청이 여전히 다른 제공자로 감">
    채팅이 Venice 제공자에 연결된 모델을 사용하고 있는지 확인하세요. Jan이 내장된 OpenAI 또는 Anthropic 모델 항목을 사용하는 경우 추가한 Venice 모델로 전환하세요.
  </Accordion>
</AccordionGroup>

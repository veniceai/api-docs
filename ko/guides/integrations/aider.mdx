[Aider](https://aider.chat/)는 터미널에서 로컬 git 저장소를 편집하는 AI 페어 프로그래밍 도구입니다. Venice는 Aider의 [OpenAI 호환 API](https://aider.chat/docs/llms/openai-compat.html) 지원을 통해 작동합니다 — Venice의 base URL과 API 키를 설정한 뒤 모델 ID 앞에 `openai/` 접두사를 붙이세요.

## 사전 요구 사항

- [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 Venice API 키
- 시스템에서 사용 가능한 Python 3.9 이상
- 작업할 git 저장소

---

## 설정

<Steps>
  <Step title="Aider 설치">
    공식 설치 프로그램으로 Aider를 설치하세요:

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    다른 설치 방법을 선호하는 경우 [Aider 설치 문서](https://aider.chat/docs/install.html)를 참조하세요.
  </Step>

  <Step title="OpenAI 엔드포인트로 Venice 구성">
    OpenAI 호환 base URL과 Venice API 키를 설정하여 Aider가 Venice를 향하도록 하세요.

    **Mac / Linux:**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell):**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (명령 프롬프트, 영구 설정):**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    `setx` 이후에는 새 변수를 사용할 수 있도록 터미널을 재시작하세요.
  </Step>

  <Step title="Venice 모델로 Aider 시작">
    프로젝트 디렉터리에서 Venice 모델 ID 앞에 `openai/`를 붙이세요:

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    처음 시작하기 좋은 다른 모델들:

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    [모델 카탈로그](/models/text)에서 모든 텍스트 모델 ID를 확인할 수 있습니다.
  </Step>
</Steps>

## `.env`로 구성 유지

프로젝트 로컬 설정을 위해 git 저장소 루트에 `.env` 파일을 생성하세요:

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

그런 다음 평소처럼 Aider를 시작하세요:

```bash
aider
```

Aider는 홈 디렉터리의 `.env`도 읽습니다. 저장소 루트 값이 홈 디렉터리 값보다 우선합니다. 이 파일을 git에 포함하지 마세요 — 아직 무시되지 않았다면 `.gitignore`에 `.env`를 추가하세요.

<Info>
`AIDER_OPENAI_API_BASE`와 `AIDER_OPENAI_API_KEY`도 설정할 수 있습니다. 공식 Aider 문서는 OpenAI 호환 제공자에 대해 여전히 `OPENAI_API_BASE`와 `OPENAI_API_KEY`를 문서화하고 있습니다.
</Info>

---

## YAML로 구성 유지

저장소 루트나 홈 디렉터리의 `.aider.conf.yml`에 동일한 설정을 저장할 수 있습니다:

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

그런 다음 실행하세요:

```bash
aider
```

## 설정 확인

Aider가 시작된 후, 모델에 도달 가능한지 확인하기 위해 간단한 질문을 던져보세요:

```txt
Reply with one sentence confirming you are running through Venice.
```

엔드포인트와 키가 올바르다면 Aider는 선택한 Venice 모델로부터 응답을 스트리밍합니다.

Aider 외부에서도 API 키를 확인할 수 있습니다:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## 모델 선택

| 사용 사례                | 예시 모델 ID           | Aider 플래그                        |
| ----------------------- | ------------------- | ---------------------------------- |
| 강력한 코딩 / 리팩터링    | `zai-org-glm-5-1`   | `--model openai/zai-org-glm-5-1`   |
| 빠른 반복               | `minimax-m25`       | `--model openai/minimax-m25`       |
| 긴 컨텍스트 코딩         | `kimi-k2-5`         | `--model openai/kimi-k2-5`         |
| 검열되지 않은 채팅       | `venice-uncensored` | `--model openai/venice-uncensored` |

Aider를 시작할 때는 항상 Venice 모델 ID 앞에 `openai/`를 붙이세요. 이 접두사가 없으면 Aider는 요청을 OpenAI 호환 제공자 경로로 라우팅하지 않습니다.

런타임에 현재 모델 ID를 확인하세요:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

또는 `text:default`와 같은 기본값은 [GET /models/traits](/api-reference/endpoint/models/traits)에서 확인하세요.

## 구성 참고 사항

- `OPENAI_API_BASE`는 `https://api.venice.ai/api/v1`로 설정하세요. `/chat/completions`를 추가하지 마세요; Aider가 채팅 경로를 스스로 추가합니다.
- `OPENAI_API_KEY`에는 Venice API 키를 사용하세요. Venice base URL과 함께 OpenAI 키를 사용하지 마세요.
- Aider가 모델의 컨텍스트 창이나 비용을 알 수 없다는 경고를 표시할 수 있습니다. 많은 Venice 모델 ID에서 예상되는 경고이며 일반적으로 무시해도 안전합니다. [Aider 모델 경고](https://aider.chat/docs/llms/warnings.html)를 참조하세요.
- 여러 파일 편집에는 유능한 코딩/추론 모델을 선호하세요. 더 작거나 코딩용이 아닌 모델은 Aider의 편집 형식에 어려움을 겪을 수 있습니다.

<Warning>
셸에서 `OPENAI_API_BASE`와 `OPENAI_API_KEY`를 설정하면 해당 변수를 읽는 다른 도구들에도 영향을 미칩니다. 동일한 환경에서 OpenAI를 다른 곳에도 사용한다면 프로젝트별 `.env` 또는 `.aider.conf.yml`을 선호하세요.
</Warning>

## 문제 해결

<AccordionGroup>
  <Accordion title="401 또는 인증 오류">
    `OPENAI_API_KEY`가 [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 Venice 키이며 앞뒤 공백이 없는지 확인하세요. Windows에서 `setx` 이후에는 터미널을 재시작하세요.
  </Accordion>
  <Accordion title="모델을 찾을 수 없거나 빈 응답">
    [텍스트 모델 카탈로그](/models/text)에 모델 ID가 존재하며 예를 들어 `openai/zai-org-glm-5-1`처럼 `openai/` 접두사를 붙여 Aider를 실행했는지 확인하세요.
  </Accordion>
  <Accordion title="요청이 여전히 OpenAI로 전송됨">
    동일한 셸 세션에서 `OPENAI_API_BASE`가 정확히 `https://api.venice.ai/api/v1`인지 확인하세요. `echo $OPENAI_API_BASE`(또는 PowerShell에서는 `$env:OPENAI_API_BASE`)로 출력해 보세요.
  </Accordion>
  <Accordion title="알 수 없는 컨텍스트 창 / 비용 경고">
    많은 Venice 모델에서 예상되는 사항입니다. Aider는 합리적인 기본값으로 대체합니다. 이를 숨기려면 [Aider 고급 모델 설정](https://aider.chat/docs/config/adv-model-settings.html)에 설명된 대로 모델 메타데이터를 추가하세요.
  </Accordion>
</AccordionGroup>

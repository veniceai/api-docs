[Cline](https://cline.bot/)은 파일을 읽고 편집하며, 명령을 실행하고, 다단계 개발 작업을 수행할 수 있는 VS Code용 AI 코딩 어시스턴트입니다. Venice는 Cline의 OpenAI 호환 제공자 설정을 통해 작동합니다.

<CardGroup cols={3}>
  <Card title="VS Code 통합" icon="code">
    에디터를 벗어나지 않고 Venice 모델을 사용하세요
  </Card>
  <Card title="OpenAI 호환" icon="plug">
    Venice의 `/chat/completions` 엔드포인트를 통해 연결하세요
  </Card>
  <Card title="모델 유연성" icon="microchip">
    호환되는 어떤 Venice 텍스트 모델이든 선택하세요
  </Card>
</CardGroup>

## 사전 요구 사항

- [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 Venice API 키
- [Visual Studio Code](https://code.visualstudio.com/) 설치
- VS Code에 설치된 [Cline 확장 프로그램](https://marketplace.visualstudio.com/items?itemName=saoudrizwan.claude-dev)

---

## 설정

<Steps>
  <Step title="Cline 설정 열기">
    VS Code 활동 표시줄에서 Cline 아이콘을 선택한 다음, 톱니바퀴 아이콘을 선택하여 Cline 설정을 여세요.
  </Step>

  <Step title="API 제공자 선택">
    **API Provider**를 **OpenAI Compatible**로 설정하세요.
  </Step>

  <Step title="Venice 구성">
    다음 값을 입력하세요:

    | 설정 | 값 |
    | --- | --- |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API Key | 사용자의 Venice API 키 |
    | Model ID | [텍스트 모델 카탈로그](/models/text)의 모델 ID |

    예를 들어, Model ID로 `zai-org-glm-5-1`을 사용하세요.

    API 키는 `Bearer` 접두사 없이 그대로 입력하세요. Base URL은 `/api/v1`로 유지하세요; Cline이 `/chat/completions` 경로를 추가합니다.
  </Step>

  <Step title="모델 기능 구성">
    **Model Configuration**을 확장하고 선택한 모델에 맞춰 컨텍스트 창과 최대 출력 토큰을 설정하세요. 이미지 입력이나 컴퓨터 사용 등 모델이 지원하는 기능이 있다면 활성화하세요.

    선택적 기능을 활성화하기 전에 [텍스트 모델 카탈로그](/models/text)에서 선택한 모델을 확인하세요.
  </Step>

  <Step title="저장 및 테스트">
    제공자 설정을 저장하고, 프로젝트를 연 뒤, Cline에 다음과 같은 작은 작업을 부여하세요:

    ```txt
    Review the current file and suggest one safe improvement.
    ```

    각 파일 또는 명령 권한을 승인하기 전에 검토하세요.
  </Step>
</Steps>

## Venice API 확인

Cline이 연결할 수 없는 경우, API 키와 사용 가능한 모델 ID를 독립적으로 확인하세요:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

모델의 `id`를 Cline의 **Model ID** 필드에 정확히 복사하세요.

---

## 구성 참고 사항

- Base URL로는 전체 `/chat/completions` 엔드포인트가 아닌 `https://api.venice.ai/api/v1`을 사용하세요.
- 모델 ID는 시간이 지남에 따라 변경될 수 있습니다. 오래된 예시에 의존하지 말고 [텍스트 모델 카탈로그](/models/text) 또는 `GET /models`를 사용하세요.
- Cline의 컨텍스트 창 및 출력 토큰 설정은 요청 예산 배정 방식에 영향을 줍니다. 최상의 결과를 위해 선택한 모델과 일치시키세요.
- Venice API 키를 비공개로 유지하세요. 소스 파일에 붙여넣거나 git에 커밋하지 마세요.

## 문제 해결

<AccordionGroup>
  <Accordion title="401 또는 인증 오류">
    API 키가 [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 것인지 확인하세요. `Bearer`나 추가 공백 없이 원시 키를 Cline에 붙여넣으세요.
  </Accordion>
  <Accordion title="모델을 찾을 수 없음">
    [텍스트 모델 카탈로그](/models/text) 또는 `GET /models`의 응답에 Model ID가 있는지 확인한 다음, 정확히 복사하세요.
  </Accordion>
  <Accordion title="연결 또는 엔드포인트 오류">
    **API Provider**가 **OpenAI Compatible**이고 Base URL이 정확히 `https://api.venice.ai/api/v1`인지 확인하세요.
  </Accordion>
  <Accordion title="Cline이 도구를 사용하거나 파일을 편집할 수 없음">
    **Model Configuration**을 열고 선택한 모델이 필요한 기능을 지원하는지 확인한 뒤, 해당 모델에 대해 Cline이 컴퓨터 사용 옵션을 노출한다면 이를 활성화하세요.
  </Accordion>
</AccordionGroup>

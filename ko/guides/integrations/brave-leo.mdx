[Brave Leo](https://brave.com/leo/)는 Brave 브라우저에 내장된 AI 어시스턴트입니다. Leo의 Bring Your Own Model(BYOM) 기능은 Venice의 OpenAI 호환 채팅 완성 엔드포인트에 직접 연결할 수 있어, 확장 프로그램을 설치하지 않고도 채팅 및 페이지 기반 어시스턴트에 Venice 모델을 사용할 수 있습니다.

## 사전 요구 사항

- Windows, macOS, 또는 Linux에서 Brave 브라우저 1.69 이상
- [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 Venice API 키
- [Venice 텍스트 모델 카탈로그](/models/text)의 모델 ID

<Note>
Brave의 BYOM 기능은 데스크톱에서 사용할 수 있습니다. 모델 ID는 Venice 모델 ID와 정확히 일치해야 합니다.
</Note>

## 설정

<Steps>
  <Step title="Leo 설정 열기">
    `brave://settings/leo-ai`를 열거나, Brave 사이드바에서 Leo를 열고 설정 아이콘을 선택한 뒤 **Leo**로 이동하세요.
  </Step>

  <Step title="사용자 지정 모델 추가">
    **Bring your own model**로 스크롤하여 **Add new model**을 선택하세요.
  </Step>

  <Step title="Venice 엔드포인트 구성">
    다음 값을 입력하세요:

    | 필드 | 값 |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | [텍스트 모델 카탈로그](/models/text)의 모델 ID |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | 사용자의 Venice API 키 |

    Brave에 **Context size**가 표시되면 선택한 모델이 지원하는 컨텍스트 창으로 설정하세요. 선택적으로 시스템 프롬프트를 추가할 수도 있습니다.

    API 키는 `Bearer` 접두사 없이 그대로 입력하세요. Brave가 각 요청에 인증 헤더를 추가합니다.
  </Step>

  <Step title="저장 후 Venice 선택">
    **Add model**을 선택한 다음, **Venice AI**를 새 대화의 기본 모델로 설정하거나 Leo의 모델 선택기에서 선택하세요.
  </Step>

  <Step title="연결 테스트">
    웹페이지에서 Leo를 열고 다음과 같은 프롬프트를 보내세요:

    ```txt
    Summarize this page in five bullet points.
    ```

    Leo는 대화와 관련 페이지 콘텐츠를 Venice 엔드포인트로 직접 전송합니다.
  </Step>
</Steps>

## Venice 웹 검색 활성화

Brave는 BYOM에서 Venice 전용 요청 본문 매개변수를 노출하지 않습니다. Venice 웹 검색을 활성화하려면 **Model request name**의 모델 ID에 [모델 기능 접미사](/api-reference/endpoint/chat/model_feature_suffix)를 추가하세요:

```txt
<model-id>:enable_web_search=auto
```

항상 검색을 활성화하고 인용을 포함하려면 다음을 사용하세요:

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
웹 검색에는 추가 사용량 기반 요금이 적용됩니다. 현재 요율은 [API 요금](/overview/pricing)을 참조하세요.
</Info>

## 프라이버시

원격 BYOM 엔드포인트를 사용하면 Brave가 해당 제공자에 직접 연결합니다. 요청이 Brave 서버를 우회하므로 Brave의 프록시 및 서버 측 보존 보호는 해당 요청에 적용되지 않습니다.

Venice는 프록시에서 프롬프트 및 응답 콘텐츠를 저장하거나 로깅하지 않습니다. 런타임 프라이버시 계층은 선택한 모델에 의해 결정됩니다:

- **Private** 모델은 데이터 무보존 인프라를 사용합니다.
- **Anonymized** 모델은 상위 제공자로부터 식별 정보를 숨기지만, 제공자는 프롬프트를 처리할 수 있습니다.
- **TEE** 및 **E2EE** 모델은 지원되는 경우 하드웨어 기반 격리 또는 클라이언트 측 암호화를 추가합니다.

민감한 정보를 전송하기 전에 [텍스트 모델 카탈로그](/models/text)에서 모델의 프라이버시 라벨을 확인하고 [Venice 프라이버시](/overview/privacy)를 읽어보세요.

Brave는 Leo 채팅 기록을 로컬 브라우저 저장소에 보관할 수 있습니다. Brave의 Leo 설정에서 Leo 데이터를 삭제하거나, 임시 채팅을 사용하거나, 로컬 채팅 기록을 비활성화할 수 있습니다.

## 구성 참고 사항

- Brave는 전체 엔드포인트 경로 `https://api.venice.ai/api/v1/chat/completions`가 필요합니다.
- 모델 ID와 가용성은 시간이 지남에 따라 변경됩니다. [텍스트 모델 카탈로그](/models/text)에서 현재 ID를 복사하세요.
- 컨텍스트 크기는 선택한 모델이 지원하는 컨텍스트 창을 초과해서는 안 됩니다.
- 페이지 요약은 프롬프트의 일부로 페이지 콘텐츠를 Venice로 전송할 수 있습니다.
- 원격 Venice 요청은 인터넷 연결이 필요하며 Venice API 크레딧이나 사용 가능한 DIEM 할당량을 소모합니다.

## 문제 해결

<AccordionGroup>
  <Accordion title="401 또는 인증 오류">
    키가 [venice.ai/settings/api](https://venice.ai/settings/api)에서 받은 것인지 확인하세요. `Bearer` 없이 원시 키를 붙여넣고 앞뒤의 공백을 제거하세요.
  </Accordion>
  <Accordion title="모델을 찾을 수 없음">
    [텍스트 모델 카탈로그](/models/text)에서 모델 ID를 정확히 복사하세요. 기능 접미사를 추가한 경우, 먼저 접미사 없이 모델을 테스트하세요.
  </Accordion>
  <Accordion title="네트워크 오류 또는 스트리밍 응답 없음">
    서버 엔드포인트에 `/api/v1/chat/completions`가 포함되어 있는지 확인하세요. Brave Leo는 OpenAI 호환 스트리밍 응답을 기대합니다.
  </Accordion>
  <Accordion title="페이지 요약에 페이지 컨텍스트가 없음">
    Leo가 페이지 콘텐츠를 사용할 수 있는 권한이 있는지 확인하고 현재 페이지가 브라우저에서 접근 가능한지 확인하세요. Brave 내부 페이지와 일부 보호된 페이지는 요약할 수 없습니다.
  </Accordion>
</AccordionGroup>

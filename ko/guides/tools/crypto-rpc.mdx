Venice Crypto RPC는 블록체인 상태를 읽고 서명된 트랜잭션을 중계하기 위한 HTTP 프록시입니다. 다른 유료 Venice 엔드포인트와 동일한 Venice API 키 또는 x402 지갑 인증을 사용하므로, 애플리케이션에서 별도의 RPC 제공자 계정을 사용할 필요가 없습니다.

Venice는 사용자의 개인 키를 보관하지 않습니다. 트랜잭션은 애플리케이션에서 서명한 후 `eth_sendRawTransaction`으로 서명된 페이로드를 제출하세요.

## 지원되는 네트워크 확인

현재 네트워크 슬러그 목록을 가져오세요:

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/networks
```

이 디스커버리 엔드포인트는 공개되어 있으며 인증이 필요하지 않습니다. 응답 결과가 기준이 되며, RPC 엔드포인트의 `{network}` 자리에 반환된 슬러그 중 하나를 사용하세요.

## RPC 요청 전송

선택한 네트워크로 JSON-RPC 2.0 요청을 전송하세요:

```bash
curl https://api.venice.ai/api/v1/crypto/rpc/ethereum-mainnet \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_blockNumber",
    "params": [],
    "id": 1
  }'
```

프록시는 단일 요청과 최대 100개까지의 배치 요청을 지원합니다. 지원되는 메서드, 가격 티어, 가용성은 각기 다르므로 특정 메서드에 의존하기 전에 [Crypto RPC API 참조](/api-reference/endpoint/crypto/rpc)를 확인하세요.

## 읽기 및 쓰기 워크플로

### 체인 상태 읽기

`eth_blockNumber`, `eth_getBalance`, `eth_call`, `eth_getLogs`와 같은 메서드를 사용해 공개된 체인 데이터를 조회하세요. 동일한 네트워크를 대상으로 하는 관련 호출은 배치로 묶어 처리하세요.

### 트랜잭션 중계

<Steps>
  <Step title="트랜잭션 구성">
    대상 네트워크에서 계정 논스, 수수료 데이터, 가스 추정치를 읽어오세요.
  </Step>
  <Step title="로컬에서 서명">
    애플리케이션의 지갑 라이브러리로 서명하세요. 개인 키나 시드 문구는 절대 Venice로 전송하지 마세요.
  </Step>
  <Step title="서명된 페이로드 제출">
    `eth_sendRawTransaction`을 호출하고 `Idempotency-Key` 헤더를 설정하여 재시도 시 청구 가능한 중계가 두 번 생성되지 않도록 하세요.
  </Step>
  <Step title="영수증 확인">
    트랜잭션이 포함될 때까지 `eth_getTransactionReceipt`를 폴링한 후 상태를 확인하세요.
  </Step>
</Steps>

## 안전한 재시도

재시도될 수 있는 요청, 특히 트랜잭션 중계에는 멱등성 키를 설정하세요:

```bash
-H "Idempotency-Key: transaction-123"
```

Venice는 응답을 24시간 동안 캐시합니다. 동일한 요청 본문으로 키를 재사용하면 추가 과금 없이 캐시된 응답이 반환되고, 다른 본문으로 재사용하면 오류가 반환됩니다.

## 주요 제약 사항

- 프록시는 HTTP 전용이며 WebSocket 구독은 지원하지 않습니다.
- 상태를 유지하는 필터 메서드는 지원되지 않습니다. `eth_getLogs`와 같은 무상태 메서드를 사용하세요.
- Venice는 키를 보관하거나 서명 메서드를 노출하지 않습니다.
- 허용 목록에 없는 메서드는 오류를 반환합니다.
- 요청 비용을 추적하려면 `X-Venice-RPC-Credits`와 `X-Venice-RPC-Cost-USD`를 확인하세요.

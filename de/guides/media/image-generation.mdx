Die Bilderzeugung bei Venice ist synchron. Senden Sie einen Prompt an `/image/generate` und erhalten Sie Ihr Bild in derselben Antwort — entweder als Base64 innerhalb von JSON oder als rohe Binärdaten, wenn `return_binary` auf `true` steht.

## Endpoints

| Endpoint | Zweck | Wann verwenden |
|---|---|---|
| `POST /image/generate` | Native Venice-Bilderzeugungs-API | Verwenden Sie diesen Endpoint für volle Funktionsunterstützung |
| `GET /image/styles` | Verfügbare Stil-Voreinstellungen auflisten | Vor dem Senden von `style_preset` verwenden |
| `POST /images/generations` | OpenAI-kompatible Bilderzeugungs-API | Verwenden Sie diesen Endpoint, wenn Sie vorhandene OpenAI-Image-Clients migrieren |

## Schritt 1: Eine Generierungsanfrage senden

Die Größenbestimmung ist modellspezifisch. Einige Modelle akzeptieren explizite `width` und `height`; einige bieten `aspect_ratio`; und Modelle mit Auflösungsstufen bieten `aspect_ratio` plus `resolution`-Werte wie `1K`, `2K` oder `4K`.

**Beispiel mit Pixelangaben:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "venice-sd35",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
  "negative_prompt": "blurry, low quality, distorted anatomy, text, watermark",
  "width": 1024,
  "height": 1024,
  "format": "webp"
}
```

**Beispiel mit Seitenverhältnis:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-image-2",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "format": "webp"
}
```

**Beispiel mit Auflösungsstufe:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "gpt-image-2",
  "prompt": "A cinematic wide shot of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

Dasselbe Muster gilt für andere Modelle mit Auflösungsstufen:

```json
{
  "model": "nano-banana-pro",
  "prompt": "A serene canal in Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Verwenden Sie [Bildmodelle](/de/models/image) oder die [Models-API](/de/api-reference/endpoint/models/list), um zu prüfen, welche Größenfelder jedes Modell akzeptiert.

**Antwort (200):**
```json
{
  "id": "generate-image-1234567890",
  "images": [
    "UklGRiIAAABXRUJQVlA4IBYAAAAwAQCdASoQABAAPm..."
  ],
  "timing": {
    "inferenceDuration": 1840,
    "inferencePreprocessingTime": 22,
    "inferenceQueueTime": 31,
    "total": 1893
  }
}
```

Das `images`-Array enthält base64-kodierte Bilddaten. Dekodieren Sie den ersten Eintrag, um es zu speichern oder anzuzeigen. `timing.total` ist die gesamte Request-Dauer in Millisekunden.

## Schritt 2: Bild dekodieren und speichern

<CodeGroup>
```python Python
import base64
import os
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/image/generate",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "venice-sd35",
        "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
        "width": 1024,
        "height": 1024,
        "format": "webp",
    },
)

data = response.json()
image_bytes = base64.b64decode(data["images"][0])

with open("output.webp", "wb") as f:
    f.write(image_bytes)

print(f"Saved image from request {data['id']}")
```

```javascript Node.js
import fs from "fs";

const response = await fetch("https://api.venice.ai/api/v1/image/generate", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "venice-sd35",
    prompt: "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
    width: 1024,
    height: 1024,
    format: "webp",
  }),
});

const data = await response.json();
const imageBuffer = Buffer.from(data.images[0], "base64");
fs.writeFileSync("output.webp", imageBuffer);

console.log(`Saved image from request ${data.id}`);
```
</CodeGroup>

## Schritt 3: Statt JSON Binärdaten zurückgeben (optional)

Wenn der Antwort-Body die Bilddatei selbst sein soll, setzen Sie `return_binary: true`. Das ist nützlich, wenn Sie das Bild direkt streamen oder ohne Base64-Dekodierung speichern wollen.

```bash
curl https://api.venice.ai/api/v1/image/generate \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o output.png \
  -d '{
    "model": "qwen-image-2",
    "prompt": "Minimalist poster of a moonlit Venetian bridge in deep blue tones",
    "format": "png",
    "return_binary": true
  }'
```

Wenn `return_binary` `true` ist, sind die Antwort-Daten rohe `image/jpeg`-, `image/png`- oder `image/webp`-Daten je nach angefordertem `format`.

<Note>
`variants` wird nur unterstützt, wenn `return_binary` `false` ist.
</Note>

## Schritt 4: Verfügbare Bildstile auflisten (optional)

Wenn Sie `style_preset` verwenden möchten, holen Sie zuerst die verfügbaren Stile von `/image/styles`:

```bash
curl https://api.venice.ai/api/v1/image/styles \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

**Antwort (200):**
```json
[
  "3D Model",
  "Analog Film",
  "Anime",
  "Cinematic",
  "Digital Art"
]
```

Übergeben Sie dann einen dieser Werte in Ihrer Generierungsanfrage:

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "Cinematic"
}
```

Verwenden Sie den Styles-Endpoint, wenn Sie exakte Voreinstellungsnamen wollen, statt sie zu erraten.

## Request-Parameter

| Parameter | Typ | Erforderlich | Standard | Beschreibung |
|---|---|---|---|---|
| `model` | string | Ja | - | Zu verwendende Modell-ID |
| `prompt` | string | Ja | - | Was generiert werden soll |
| `negative_prompt` | string | Nein | - | Was im Bild vermieden werden soll |
| `width` | integer | Nein | `1024` | Ausgabebreite in Pixeln für pixelbasierte Modelle wie `venice-sd35` und `qwen-image` |
| `height` | integer | Nein | `1024` | Ausgabehöhe in Pixeln für pixelbasierte Modelle wie `venice-sd35` und `qwen-image` |
| `format` | string | Nein | `webp` | Ausgabeformat: `jpeg`, `png` oder `webp` |
| `variants` | integer | Nein | `1` | Anzahl der zu erzeugenden Bilder (`1`–`4`), nur wenn `return_binary` `false` ist |
| `return_binary` | boolean | Nein | `false` | Rohe Bild-Bytes statt Base64-JSON zurückgeben |
| `safe_mode` | boolean | Nein | `true` | Adult-Content unschärfen, wenn aktiviert |
| `seed` | integer | Nein | zufällig | Denselben Seed wiederverwenden für konsistentere Iterationen |
| `cfg_scale` | number | Nein | modellabhängig | Höhere Werte zwingen das Modell, dem Prompt enger zu folgen |
| `style_preset` | string | Nein | - | Eine Voreinstellung aus [Bildstilen](/de/api-reference/endpoint/image/styles) anwenden |
| `aspect_ratio` | string | Konditional | - | Wird von Modellen mit verhältnisbasierter Größe verwendet, etwa `qwen-image-2`, `gpt-image-2`, `nano-banana-2` und `nano-banana-pro` |
| `resolution` | string | Konditional | - | Wird von Modellen mit Auflösungsstufen wie `1K`, `2K` oder `4K` verwendet |
| `enable_web_search` | boolean | Konditional | `false` | Erlaubt unterstützten Modellen aktuelle Web-Informationen zu nutzen; zusätzliche Kosten |

Die Validierung ist modellspezifisch. Prüfen Sie [Bildmodelle](/de/models/image) und die [Models-API](/de/api-reference/endpoint/models/list), bevor Sie sich auf einen Parameter über mehrere Modelle hinweg verlassen.

## Modellspezifische Optionen

### Hochauflösende Generierung

Einige Bildmodelle unterstützen `aspect_ratio` ohne wählbare `resolution`-Stufe. `qwen-image-2` etwa akzeptiert ein Seitenverhältnis und mappt es auf modellspezifische Ausgabemaße:

```json
{
  "model": "qwen-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9"
}
```

Andere Bildmodelle unterstützen `aspect_ratio` plus eine `resolution`-Stufe. `gpt-image-2`, `nano-banana-2` und `nano-banana-pro` unterstützen z. B. `1K`, `2K` und `4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "4K"
}
```

```json
{
  "model": "nano-banana-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Unter [Bildmodelle](/de/models/image) sehen Sie, welche Modelle höhere Auflösungen unterstützen und wie diese abgerechnet werden.

### Stil-Voreinstellungen

Sofern das gewählte Modell dies unterstützt, lässt `style_preset` die Ausgabe steuern, ohne den gesamten Prompt umzuschreiben. Gültige Voreinstellungsnamen können Sie über [Bildstile](/de/api-reference/endpoint/image/styles) abrufen:

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "3D Model"
}
```

Die aktuelle Stil-Liste finden Sie unter [Bildstile](/de/api-reference/endpoint/image/styles).

## OpenAI-kompatibler Endpoint

Wenn Sie bereits OpenAI-Image-SDKs oder bestehende DALL-E-Integrationen nutzen, unterstützt Venice auch `POST /images/generations`. Es bietet ein einfacheres Request-Format, aber weniger Funktionen als der native Venice-Endpoint.

**Anfrage:**
```json
{
  "model": "qwen-image-2",
  "prompt": "A clean isometric illustration of an AI control room",
  "size": "1024x1024",
  "response_format": "b64_json"
}
```

Verwenden Sie die OpenAI-kompatible Route für schnellere Migrationen. Verwenden Sie `/image/generate`, wenn Sie Venice-spezifische Optionen wie `cfg_scale`, `style_preset`, `variants` oder Binär-Antworten benötigen.

## Prompting-Tipps

1. Beginnen Sie mit dem Motiv, ergänzen Sie dann Medium, Beleuchtung, Komposition und Stimmung.
2. Schreiben Sie zu vermeidende Details in `negative_prompt`, statt den Hauptprompt zu überladen.
3. Wiederverwenden Sie `seed` beim Iterieren, um Prompt-Änderungen vergleichen zu können, ohne die Komposition komplett zu ändern.
4. Bleiben Sie modellbewusst bei der Größenangabe. Einige Modelle nutzen `width`/`height`, einige `aspect_ratio`, Modelle mit Auflösungsstufen verwenden `aspect_ratio` plus `resolution`.
5. Verwenden Sie `variants` zur Exploration, schalten Sie auf eine einzelne Ausgabe zurück, sobald die Richtung steht.

## Fehler

| Status | Bedeutung | Aktion |
|---|---|---|
| `400` | Ungültige Request-Parameter | Feldnamen, Typen und modellspezifische Einschränkungen prüfen |
| `401` | Authentifizierung fehlgeschlagen oder Modell erfordert höhere Zugriffsstufe | API-Schlüssel und Modellzugriff prüfen |
| `402` | Unzureichendes Guthaben | Credits auf [venice.ai/settings/api](https://venice.ai/settings/api) aufladen |
| `415` | Ungültiger Content-Type | JSON mit `Content-Type: application/json` senden |
| `429` | Rate Limit überschritten oder Modell überlastet | Mit Backoff erneut versuchen; `Retry-After`-Header prüfen |
| `500` | Inference-Verarbeitung fehlgeschlagen | Anfrage erneut senden |
| `503` | Modell ausgelastet | Nach kurzer Verzögerung erneut versuchen |

<Note>
Bei aktivem Safe Venice prüfen Sie Response-Header wie `x-venice-is-blurred` und `x-venice-is-content-violation`, wenn Sie Moderationsergebnisse programmatisch erkennen müssen.
</Note>

Mit der Video-Hochskalierung können Sie vorhandene Videos auf höhere Auflösungen anheben und gleichzeitig die visuelle Qualität verbessern. Das **Topaz-Video-Upscale**-Modell nutzt KI-gestützte Hochskalierung, um die Auflösung um den Faktor 2× oder 4× zu erhöhen oder Qualitätsverbesserungen bei Originalauflösung (1×) anzuwenden.

## Funktionsweise

Die Video-Hochskalierung verwendet dasselbe asynchrone Warteschlangensystem wie die Videogenerierung:

1. **Queue** — Senden Sie Ihr Video an `/video/queue` mit dem Modell `topaz-video-upscale`
2. **Pollen** — Prüfen Sie `/video/retrieve` mit der zurückgegebenen `queue_id`, bis der Status `completed` ist
3. **Complete** — Rufen Sie `/video/complete` auf, um abzuschließen und die Ausgabe-URL zu erhalten

Der Server erkennt Dauer, Bildrate und Abmessungen des Eingangsvideos automatisch aus der hochgeladenen Datei. Sie müssen diese Werte nicht angeben — die Abrechnung erfolgt anhand der tatsächlichen Video-Metadaten.

## Upscale-Faktoren

| `upscale_factor` | Ausgabeauflösung | Anwendungsfall |
|---|---|---|
| `1` | Wie Eingang | Reine Qualitätsverbesserung (Entrauschen, Schärfen) |
| `2` (Standard) | 2× der Eingangsabmessungen | Standard-Upscale — 720p-Input wird zu 1440p-Output |
| `4` | 4× der Eingangsabmessungen | Maximales Upscale — 480p-Input wird zu 1920p-Output |

<Note>
Der Parameter `upscale_factor` ersetzt `resolution` für Upscale-Modelle. Die Übergabe von `resolution` führt zu einem Fehler. Grund: Die Ausgabeauflösung hängt von den Eingangsabmessungen ab — ein 2×-Upscale eines 720p-Videos liefert ein anderes Ergebnis als ein 2×-Upscale eines 480p-Videos.
</Note>

## Unterstützte Eingabeformate

- **Formate**: MP4, MOV, WebM
- **Eingabemethoden**: HTTPS-URL oder `data:video/...;base64,...`-Daten-URL
- **Maximale Dauer**: 300 Sekunden (5 Minuten)

## API-Verwendung

### Upscale-Job in die Warteschlange stellen

<CodeGroup>
```bash cURL
curl https://api.venice.ai/api/v1/video/queue \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "topaz-video-upscale",
    "video_url": "https://example.com/input-video.mp4",
    "upscale_factor": 2
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "topaz-video-upscale",
        "video_url": "https://example.com/input-video.mp4",
        "upscale_factor": 2,
    },
)

data = response.json()
queue_id = data["queue_id"]
print(f"Queued: {queue_id}")
```

```javascript Node.js
const response = await fetch("https://api.venice.ai/api/v1/video/queue", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "topaz-video-upscale",
    video_url: "https://example.com/input-video.mp4",
    upscale_factor: 2,
  }),
});

const { queue_id } = await response.json();
console.log(`Queued: ${queue_id}`);
```
</CodeGroup>

Die Antwort enthält eine `queue_id` zum Verfolgen des Jobs:

```json
{
  "model": "topaz-video-upscale",
  "queue_id": "abc123-def456-..."
}
```

### Auf Fertigstellung pollen

<CodeGroup>
```bash cURL
curl https://api.venice.ai/api/v1/video/retrieve \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"queue_id": "abc123-def456-..."}'
```

```python Python
import time

while True:
    result = requests.post(
        "https://api.venice.ai/api/v1/video/retrieve",
        headers={"Authorization": "Bearer YOUR_API_KEY"},
        json={"queue_id": queue_id},
    )
    data = result.json()

    if data.get("status") == "completed":
        print(f"Video URL: {data['url']}")
        break

    time.sleep(5)
```

```javascript Node.js
const poll = async (queueId) => {
  while (true) {
    const res = await fetch("https://api.venice.ai/api/v1/video/retrieve", {
      method: "POST",
      headers: {
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json",
      },
      body: JSON.stringify({ queue_id: queueId }),
    });
    const data = await res.json();

    if (data.status === "completed") {
      console.log(`Video URL: ${data.url}`);
      return data;
    }

    await new Promise((r) => setTimeout(r, 5000));
  }
};

await poll(queue_id);
```
</CodeGroup>

### Mit complete abschließen

Nach dem Abrufen des Ergebnisses rufen Sie `/video/complete` zum Abschluss auf:

```bash
curl https://api.venice.ai/api/v1/video/complete \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"queue_id": "abc123-def456-..."}'
```

## API-Parameter

| Feld | Typ | Erforderlich | Beschreibung |
|---|---|---|---|
| `model` | string | **Ja** | Muss `topaz-video-upscale` sein |
| `video_url` | string | **Ja** | Eingangs-Video-URL oder Daten-URL. Unterstützte Formate: MP4, MOV, WebM. |
| `upscale_factor` | number | Nein | `1`, `2` (Standard) oder `4`. Steuert den Upscale-Multiplikator. |

### Für Upscale-Modelle nicht verwendete Parameter

Die folgenden Parameter werden für `topaz-video-upscale` **nicht akzeptiert** und führen bei Angabe zu einem Fehler:

| Feld | Grund |
|---|---|
| `resolution` | Verwenden Sie stattdessen `upscale_factor`. Die Ausgabeauflösung hängt von den Eingangsabmessungen ab. |
| `prompt` | Upscaling verwendet keine Text-Prompts. Ein leerer String wird automatisch gesetzt. |

Der `duration`-Parameter wird ebenfalls ignoriert — der Server erkennt die Dauer für eine genaue Abrechnung direkt aus der Videodatei.

## Preise

Die Preise basieren auf **Dauer**, **Ausgabeauflösungs-Stufe** und **Bildrate**. Die Ausgabeauflösungs-Stufe wird durch die Höhe des Eingangsvideos multipliziert mit dem Upscale-Faktor bestimmt.

### Ausgabeauflösungs-Stufen

| Stufe | Ausgabehöhe | Preis pro Sekunde |
|---|---|---|
| 720p | ≤ 720 px | ~0,013 $ |
| 1080p | 721–1080 px | ~0,025 $ |
| 4K | > 1080 px | ~0,10 $ |

<Note>
Videos mit Bildraten über 48 fps kosten das 2-fache des Preises pro Sekunde.
</Note>

### Preisbeispiele

| Eingang | Upscale-Faktor | Ausgabe | Dauer | Geschätzte Kosten |
|---|---|---|---|---|
| 480p, 30fps | 2× | 960p (1080p-Stufe) | 10s | ~0,25 $ |
| 720p, 30fps | 2× | 1440p (4K-Stufe) | 10s | ~1,00 $ |
| 1080p, 30fps | 2× | 2160p (4K-Stufe) | 30s | ~3,00 $ |
| 360p, 24fps | 4× | 1440p (4K-Stufe) | 10s | ~1,00 $ |
| 480p, 60fps | 2× | 960p (1080p-Stufe) | 10s | ~0,50 $ |

Verwenden Sie die [Video-Quote-API](/de/api-reference/endpoint/video/quote), um exakte Preise vor dem Absenden zu erhalten.

### Eine Kostenschätzung einholen

```bash
curl https://api.venice.ai/api/v1/video/quote \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "topaz-video-upscale",
    "duration": "10",
    "input_height": 720
  }'
```

Der Quote-Endpoint akzeptiert `input_height`, um die Ausgabeauflösungs-Stufe abzuschätzen. Das ist optional — wird es weggelassen, geht die Schätzung von einer konservativen Annahme aus.

## Fehlersuche

| Problem | Wahrscheinliche Ursache | Lösung |
|---|---|---|
| `"Use upscale_factor instead of resolution"` | `resolution` wurde in der Anfrage übergeben | `resolution` entfernen und stattdessen `upscale_factor` verwenden |
| Höhere Kosten als erwartet | Eingangsvideo hat hohe Auflösung oder hohe FPS | Eingangsabmessungen mit dem Quote-Endpoint prüfen. 720p+-Input mit 2×-Upscale landet in der 4K-Preisstufe. |
| Job dauert lange | Großes oder langes Video | Hochskalierung ist rechenintensiv. Längere Videos und höhere Upscale-Faktoren brauchen proportional länger. |
| `"Insufficient balance"` | Kontoguthaben zu niedrig | Credits auf [venice.ai/settings/api](https://venice.ai/settings/api) aufladen |

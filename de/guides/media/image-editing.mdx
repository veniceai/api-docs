Die Bildbearbeitung bei Venice ist synchron. Senden Sie Ihr Quellbild an `/image/edit` oder `/image/multi-edit`, und das bearbeitete Ergebnis kommt in derselben Antwort als PNG-Datei zurück. Für Freisteller gibt `/image/background-remove` ein PNG mit Transparenz zurück.

<Warning>
Die Bildbearbeitungs-Endpoints sind experimentell, und modellspezifisches Verhalten kann sich im Laufe der Zeit ändern.
</Warning>

## Endpoints

| Endpoint | Zweck | Am besten für |
|---|---|---|
| `POST /image/edit` | Ein einzelnes Bild mit einem Prompt bearbeiten | Allgemeine Edits und prompt-gesteuertes Inpainting |
| `POST /image/multi-edit` | Bearbeiten mit 1–3 geschichteten Bildern | Kontrolliertere Edits mit Masken oder Overlays |
| `POST /image/background-remove` | Hintergrund eines Bildes entfernen | Transparente Freisteller für Produkte, Porträts und Assets |

## Wann welchen Endpoint verwenden

- Verwenden Sie `/image/edit`, wenn Sie ein Quellbild haben und mit einem Prompt einen Teil davon ändern, entfernen oder umgestalten möchten.
- Verwenden Sie `/image/multi-edit`, wenn Sie zusätzliche Kontrolle durch Masken, Overlays oder Referenz-Layer benötigen.
- Verwenden Sie `/image/background-remove`, wenn Sie nur ein sauberes Vordergrundmotiv mit Transparenz wünschen.

<Note>
Für Inpainting verwenden Sie `/image/edit` oder `/image/multi-edit`. Der alte `inpaint`-Parameter an `/image/generate` ist veraltet.
</Note>

## Schritt 1: Ein einzelnes Bild bearbeiten

Single-Image-Edit ist der einfachste Inpainting-Flow. Senden Sie ein Bild plus einen kurzen Prompt wie "remove the sign", "change the sky to sunrise" oder "replace the background with a studio backdrop".

**Request:**
```bash
POST https://api.venice.ai/api/v1/image/edit
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-edit",
  "prompt": "Replace the cloudy sky with a warm sunrise while preserving the buildings and canal",
  "image": "https://example.com/venice-canal.jpg"
}
```

**Antwort (200):**
Der Antwort-Body sind rohe `image/png`-Binärdaten. Speichern Sie sie direkt in eine Datei.

<CodeGroup>
```python Python
import base64
import os
import requests

with open("input.jpg", "rb") as f:
    image_base64 = base64.b64encode(f.read()).decode("utf-8")

response = requests.post(
    "https://api.venice.ai/api/v1/image/edit",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "qwen-edit",
        "prompt": "Remove the tourist crowd from the square and keep the architecture intact",
        "image": image_base64,
    },
)

with open("edited.png", "wb") as f:
    f.write(response.content)
```

```javascript Node.js
import fs from "fs";

const imageBase64 = fs.readFileSync("input.jpg").toString("base64");

const response = await fetch("https://api.venice.ai/api/v1/image/edit", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "qwen-edit",
    prompt: "Remove the tourist crowd from the square and keep the architecture intact",
    image: imageBase64,
  }),
});

const editedImage = Buffer.from(await response.arrayBuffer());
fs.writeFileSync("edited.png", editedImage);
```

```bash cURL
curl https://api.venice.ai/api/v1/image/edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o edited.png \
  -d '{
    "model": "qwen-edit",
    "prompt": "Colorize this black and white portrait naturally",
    "image": "https://example.com/portrait-bw.jpg"
  }'
```

## Schritt 2: Multi-Edit für Masken oder geschichtetes Inpainting verwenden

`/image/multi-edit` akzeptiert bis zu drei Bilder. Das erste Bild ist das Basisbild. Die übrigen Bilder werden als Edit-Layer oder Masken behandelt, was Ihnen mehr Kontrolle gibt als prompt-only-Editing.

Das ist die bessere Wahl, wenn Sie:

- eine bestimmte Region mit einer Maske ansprechen wollen
- eine vorhandene Komposition mit einem Overlay kombinieren möchten
- den Edit enger einschränken wollen, als es ein Single-Image-Prompt erlaubt

**JSON-Request:**
```json
{
  "modelId": "qwen-edit",
  "prompt": "Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective",
  "images": [
    "https://example.com/street-scene.png",
    "https://example.com/billboard-mask.png"
  ]
}
```

**Multipart-Request:**
```bash
curl https://api.venice.ai/api/v1/image/multi-edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "modelId=qwen-edit" \
  -F "prompt=Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective" \
  -F "images=@street-scene.png" \
  -F "images=@billboard-mask.png" \
  -o multi-edited.png
```

Wie bei `/image/edit` ist der Antwort-Body rohe `image/png`-Daten.

<Note>
`/image/multi-edit` verwendet derzeit im Request-Schema das Feld `modelId` statt `model`.
</Note>

## Inpainting-Tipps

Prompt-basiertes Inpainting funktioniert am besten, wenn die Anweisung kurz und lokal ist:

- `remove the tree`
- `change the sky to sunset`
- `replace the logo with a blank sign`
- `restore the torn corner of the photo`

Für umfassendere Szenenänderungen beschreiben Sie, was gleich bleiben soll:

```text
Replace the background with a modern photo studio backdrop while preserving the subject pose, facial features, and clothing.
```

Wenn der Edit immer wieder den falschen Bereich beeinflusst, wechseln Sie von `/image/edit` zu `/image/multi-edit` und liefern Sie eine Masken- oder Overlay-Schicht.

## Schritt 3: Hintergrund entfernen

Verwenden Sie `/image/background-remove`, wenn Sie das Vordergrundmotiv isoliert auf einem transparenten Hintergrund haben möchten. Dieser Endpoint gibt ein PNG mit Alpha-Transparenz zurück.

**Mit einer Bild-URL:**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o cutout.png \
  -d '{
    "image_url": "https://example.com/product-photo.jpg"
  }'
```

**Mit einem lokalen Datei-Upload:**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "image=@product-photo.jpg" \
  -o cutout.png
```

Hintergrund-Entfernung eignet sich für:

- E-Commerce-Produktfotos
- Profilfotos und Porträts
- Assets, die Sie auf einen neuen Hintergrund setzen möchten

## Request-Parameter

### `/image/edit`

| Parameter | Typ | Erforderlich | Standard | Beschreibung |
|---|---|---|---|---|
| `image` | file, base64-String oder URL | Ja | - | Quellbild zum Bearbeiten |
| `prompt` | string | Ja | - | Textanweisungen für den Edit |
| `model` | string | Nein | `qwen-edit` | Edit-Modell-ID |
| `aspect_ratio` | string | Nein | Modell-Standard | Ausgabe-Seitenverhältnis für unterstützte Modelle |
| `modelId` | string | Veraltet | - | Veralteter Alias für `model` |

### `/image/multi-edit`

| Parameter | Typ | Erforderlich | Standard | Beschreibung |
|---|---|---|---|---|
| `images` | Array aus 1–3 Dateien, base64-Strings oder URLs | Ja | - | Erstes Bild ist das Basisbild; der Rest sind Edit-Layer oder Masken |
| `prompt` | string | Ja | - | Textanweisungen, wie die Layer kombiniert oder bearbeitet werden |
| `modelId` | string | Nein | `qwen-edit` | Edit-Modell-ID |

### `/image/background-remove`

| Parameter | Typ | Erforderlich | Beschreibung |
|---|---|---|---|
| `image` | file oder base64-String | Entweder `image` oder `image_url` | Quellbild zum Freistellen |
| `image_url` | string | Entweder `image` oder `image_url` | Öffentliche Bild-URL zum Freistellen |

---

## Unterstützte Eingabeformate

| Endpoint | JSON-Eingabe | Multipart-Eingabe | Ausgabe |
|---|---|---|---|
| `/image/edit` | Base64-String oder URL | Datei-Upload | `image/png` |
| `/image/multi-edit` | Base64-Strings oder URLs | Datei-Uploads | `image/png` |
| `/image/background-remove` | Base64-String oder URL | Datei-Upload | `image/png` |

Für Edit-Endpoints müssen Bildabmessungen mindestens `65536` und höchstens `33177600` Pixel haben. Hochgeladene Dateien müssen unter `25MB` sein.

## Fehler

| Status | Bedeutung | Aktion |
|---|---|---|
| `400` | Ungültige Request-Parameter | Bildanzahl, Feldnamen und Eingabeformat prüfen |
| `401` | Authentifizierung fehlgeschlagen | API-Schlüssel prüfen |
| `402` | Unzureichendes Guthaben | Credits auf [venice.ai/settings/api](https://venice.ai/settings/api) aufladen |
| `415` | Ungültiger Content-Type | JSON oder Multipart-Form-Data korrekt verwenden |
| `429` | Rate Limit überschritten oder Modell überlastet | Mit Backoff erneut versuchen; `Retry-After`-Header prüfen |
| `500` | Inference-Verarbeitung fehlgeschlagen | Anfrage erneut senden |
| `503` | Modell ausgelastet | Nach kurzer Verzögerung erneut versuchen |

<Note>
Einige Edit-Modelle haben strengere Inhaltsrichtlinien als Bildgenerierungs-Modelle. Beispielsweise blockiert `qwen-edit` Anfragen, die explizite sexuelle Bilder, sexualisierte Minderjährige oder reale Gewalt darstellen.
</Note>

Reference to Video ermöglicht es, das Aussehen von Charakteren, Objekten und Szenen festzulegen, sodass Ihre KI-generierten Videos visuell konsistent bleiben. Statt zu hoffen, dass das Modell Ihren Prompt korrekt interpretiert, liefern Sie visuelle Ankerpunkte — Referenzbilder, die dem Modell genau sagen, wie Ihr Motiv aussieht.

Diese Funktion ist auf den Modellen **Kling O3** und **Grok Imagine R2V** im [Venice Video Studio](https://venice.ai/video) verfügbar. Jede Modellfamilie nutzt einen anderen Ansatz für Referenzbilder — siehe die modellspezifischen Abschnitte unten.

## Wann Reference to Video verwenden

Verwenden Sie Reference to Video, wenn Sie benötigen:

- **Charakter-Konsistenz** — dieselbe Person oder dasselbe Wesen über mehrere Shots hinweg
- **Produkt-Genauigkeit** — ein reales Produkt, das identisch zum Original aussehen muss
- **Szenen-Kontinuität** — eine bestimmte Umgebung oder ein Hintergrund über Generierungen hinweg
- **Multi-Charakter-Szenen** — mehrere unterscheidbare Charaktere, die ohne Vermischung interagieren

Für einfaches Text-to-Video oder Image-to-Video, wo Konsistenz nicht kritisch ist, funktionieren die Standardmodelle ohne Referenzen gut.

## Verfügbare Modelle

| Modell | Ansatz | Am besten für |
|---|---|---|
| **Kling O3 Pro R2V** | Elemente + Szenenbilder | Komplexe Multi-Charakter-Szenen mit präziser Identitätssteuerung |
| **Kling O3 Standard R2V** | Elemente + Szenenbilder | Schnellere Iteration auf elementbasierten Szenen |
| **Grok Imagine R2V** | Flache Referenzbilder | Schnelle referenzgesteuerte Generierung mit bis zu 7 Bildern |

**Kling O3** nutzt einen strukturierten Ansatz mit Elementen (Charakter-Identitäts-Anker mit Frontal- + Referenzbildern) und Szenenbildern. **Grok Imagine R2V** verfolgt einen einfacheren Ansatz — Sie laden Referenzbilder direkt hoch und referenzieren sie im Prompt mit `@Image1`, `@Image2` usw.

## Kling O3 Reference to Video

### Kernkonzepte

Kling O3 Reference to Video verwendet drei Arten visueller Eingaben, die zusammenarbeiten:

| Eingabe | Erforderlich | Zweck | Wie im Prompt referenzieren |
|---|---|---|---|
| **Elemente** | Mindestens eine visuelle Eingabe* | Identität eines Charakters oder Objekts festlegen | `@Element1`, `@Element2` usw. |
| **Szenen-Referenzbilder** | Mindestens eine visuelle Eingabe* | Umgebung, Stil und Stimmung festlegen | `@Image1`, `@Image2` usw. |
| **Start-Frame** | Mindestens eine visuelle Eingabe* | Den ersten Frame des Videos kontrollieren | nicht zutreffend (per Upload festgelegt) |
| **End-Frame** | Nein | Den letzten Frame des Videos kontrollieren | nicht zutreffend (per Upload festgelegt) |

*Mindestens eines ist erforderlich: Start-Frame, Elemente oder Szenen-Referenzbilder.

### Elemente

Ein Element ist ein Charakter oder Objekt, dessen Erscheinung im gesamten Video stabil bleiben soll. Jedes Element besteht aus:

- **Frontalbild** (pro Element erforderlich) — ein klares, frontales Foto des Motivs. Dies ist der primäre Identitäts-Anker. Stellen Sie es sich als "Passfoto" Ihres Charakters oder Produkts vor.
- **Referenzbilder** (1–3, optional) — zusätzliche Winkel desselben Motivs (Seitenansicht, 45-Grad-Winkel, Rückseite). Sie helfen dem Modell, das Motiv im 3D-Raum zu verstehen. Werden sie nicht bereitgestellt, wird das Frontalbild automatisch als Referenz verwendet.

Sie können bis zu **7 Elemente** pro Generierung hinzufügen (begrenzt durch die Gesamtsumme). Referenzieren Sie sie im Prompt mit `@Element1`, `@Element2` usw.

### Szenen-Referenzbilder

Szenenreferenzen definieren die "Bühne", auf der die Aktion stattfindet. Sie beeinflussen:

- Beleuchtung und Farbpalette
- Architektur und Umgebungsdetails
- Gesamten visuellen Stil und die Stimmung

Sie können bis zu **4 Szenenbilder** hinzufügen. Referenzieren Sie sie als `@Image1`, `@Image2` usw. im Prompt.

### Einschränkungen

Die Gesamtzahl der Bilder über alle Eingabetypen hinweg ist begrenzt:

| Grenze | Wert |
|---|---|
| **Mindestens erforderlich** | Mindestens 1 visuelle Eingabe (Start-Frame, Element oder Szenenbild) |
| **Gesamtsumme** (First-Frame + Last-Frame + Elemente + Szenenbilder) | **maximal 7** |
| Elemente (ohne Start-/End-Frame) | maximal 7 |
| Elemente (mit Start- oder End-Frame) | maximal 3 |
| Szenen-Referenzbilder | maximal 4 |
| Referenzbilder pro Element | 1–3 |

**Beispielszenarien:**

- 7 Elemente + 0 Szenenbilder = 7 ✓ (keine Frames)
- 5 Elemente + 2 Szenenbilder = 7 ✓ (keine Frames)
- First-Frame (1) + 3 Elemente + 3 Szenenbilder = 7 ✓
- First-Frame (1) + Last-Frame (1) + 3 Elemente + 2 Szenenbilder = 7 ✓
- First-Frame (1) + 4 Elemente = ✗ (max. 3 Elemente mit Frame)
- First-Frame (1) + Last-Frame (1) + 4 Elemente = ✗ (max. 3 Elemente mit Frames)

<Note>
Jedes Element erfordert ein **Frontalbild**. Wenn Sie keine Referenzbilder für ein Element angeben, wird das Frontalbild automatisch als Referenz verwendet.
</Note>

### Multi-Shot-Modus

Multi-Shot ermöglicht es, eine einzelne Generierung in mehrere Szenen aufzuteilen, jede mit eigenem Prompt und eigener Dauer. Elemente und Szenenreferenzen werden über alle Shots hinweg übernommen und sorgen für Konsistenz. Die Gesamtdauer über alle Shots darf **15 Sekunden** nicht überschreiten.

### Schritt-für-Schritt-Anleitung (Video Studio)

#### 1. Video Studio öffnen und das Modell wählen

Gehen Sie zu [venice.ai/video](https://venice.ai/video). Wählen Sie im Modell-Browser links eines der **Kling O3 Reference to Video**-Modelle:

- **Kling O3 Pro R2V** — höhere Qualität, längere Generierungszeit (~6 Min.)
- **Kling O3 Standard R2V** — schneller, kostengünstiger für Iteration

#### 2. Visuelle Eingaben hinzufügen (mindestens eine erforderlich)

Sie müssen **mindestens eine visuelle Eingabe** liefern, um ein Video zu generieren: einen Start-Frame, ein Element oder ein Szenen-Referenzbild. Im Eingabe-Panel sehen Sie den Abschnitt **Elements**. Klicken Sie auf **Add Element**, um ein Element für Charaktere oder Objekte anzulegen, die visuell konsistent bleiben sollen.

Für jedes Element:

1. Klicken Sie auf die **Frontal**-Kachel, um ein klares, frontales Bild Ihres Charakters oder Objekts hochzuladen
2. Optional auf **Add** unter **Reference Images** klicken, um zusätzliche Winkel (1–3) hochzuladen

Für weitere Charaktere oder Objekte wiederholen (insgesamt bis zu 7 Elemente, bzw. 3 bei Verwendung von Start-/End-Frames).

<Warning>
Die Gesamtsumme aus First-Frame, Last-Frame, Elementen und Szenenbildern darf **7** nicht überschreiten. Details siehe [Einschränkungen](#limitations).
</Warning>

<Tip>
**Beste Referenzbilder:** Verwenden Sie gut ausgeleuchtete Fotos mit klarem Hintergrund. Liefern Sie Front-, Seiten- und 45-Grad-Ansichten für den stärksten Identitäts-Lock. Achten Sie darauf, dass alle Referenzbilder denselben visuellen Stil haben (keine Mischung aus Fotorealistisch und Anime).
</Tip>

#### 3. Szenen-Referenzbilder hinzufügen (optional)

Unter dem Bereich "Elements" sehen Sie **Scene Reference Images**. Laden Sie Bilder hoch, die die gewünschte Umgebung definieren — eine bestimmte Location, ein Lichtsetup oder einen Kunststil.

Diese werden automatisch als `@Image1`, `@Image2` usw. getaggt.

#### 4. Start-Frame hochladen (optional)

Wenn Sie den ersten Frame Ihres Videos exakt kontrollieren möchten, wechseln Sie zum Eingabetyp **Image** und laden einen Start-Frame hoch. Optional können Sie auch einen End-Frame festlegen.

#### 5. Prompt schreiben

Beschreiben Sie im Prompt-Feld die gewünschte Aktion und referenzieren Sie Ihre Elemente und Szenenbilder über die `@`-Tags:

```
@Element1 walks through the streets of @Image1, looking up at the buildings.
The camera slowly tracks from behind, revealing the city skyline.
```

Für **Multi-Charakter-Szenen**:

```
@Element1 and @Element2 enter the cafe in @Image1 from opposite sides.
@Element1 waves and walks toward @Element2, who is sitting at a corner table.
```

#### 6. Einstellungen konfigurieren

Öffnen Sie **Video Settings**, um Folgendes anzupassen:

| Einstellung | Optionen | Standard |
|---|---|---|
| Dauer | 3s – 15s | 5s |
| Seitenverhältnis | 16:9, 9:16, 1:1 | 16:9 |
| Generate Audio | An/Aus | Aus |

<Note>
Audio-Generierung ergänzt native Soundeffekte, Dialog und Umgebungsklang synchron zum Video. Sie erhöht die Kosten um ~25 %.
</Note>

#### 7. Generieren

Klicken Sie auf **Generate Video**. Kling O3 benötigt typischerweise 4–6 Minuten, je nach Modellstufe und Dauer. Sie können mehrere Generierungen einreihen und Ergebnisse in der Video-Galerie durchsuchen.

### Multi-Shot-Storyboarding

Für narrative Sequenzen verwenden Sie den Multi-Shot-Modus, um separate Szenen innerhalb einer einzigen Generierung zu definieren.

1. Klicken Sie im Prompt-Bereich auf **Add Shot**, um weitere Shots zu erstellen
2. Schreiben Sie für jeden Shot einen separaten Prompt
3. Stellen Sie die Dauer pro Shot ein (3–15s je, Gesamtsumme ≤ 15s)

Elemente und Szenenreferenzen werden automatisch über alle Shots übernommen:

```
Shot 1 (5s): @Element1 stands at the edge of @Image1, looking out at the horizon.
Slow camera push forward.

Shot 2 (5s): Close-up of @Element1's face as they turn toward the camera.
Soft natural lighting, shallow depth of field.

Shot 3 (5s): @Element1 walks away from camera into the distance.
Wide cinematic shot, golden hour lighting.
```

<Warning>
Die Gesamtdauer im Multi-Shot-Modus darf 15 Sekunden nicht überschreiten. Beispiel: drei 5-Sekunden-Shots = maximal 15s.
</Warning>

### Prompting-Tipps

#### Prompt strukturieren

Folgen Sie diesem Muster für verlässliche Ergebnisse:

```
[Motiv mit @Element-Tag] + [Aktion] + [Umgebung mit @Image-Tag] + [Kamerabewegung] + [Beleuchtung/Stil]
```

**Beispiel:**

```
@Element1 hops happily across the candy ground of @Image1, stops to look at a
giant lollipop, tilts its head curiously. Cinematic tracking shot, soft warm lighting.
```

#### Prompts bei 50–150 Wörtern halten

Zu kurze Prompts haben zu wenig Detail. Zu lange Prompts erzeugen Widersprüche. Streben Sie den Sweet Spot an.

#### Einfache Kamera-Sprache verwenden

Das Modell reagiert am besten auf geradlinige Kameraanweisungen:

| Verwenden | Vermeiden |
|---|---|
| `slow camera push forward` | `dolly zoom with rack focus transition` |
| `tracking shot from behind` | `complex handheld parallax movement` |
| `close-up` | `extreme macro with tilt-shift bokeh` |
| `wide cinematic shot` | `anamorphic ultra-wide establishing crane shot` |

#### Konsistentes Vokabular verwenden

Wenn Sie einen Charakter in einem Prompt mit "a red jacket" beschreiben, wechseln Sie im nächsten nicht zu "crimson coat". Das Modell behandelt unterschiedliche Wörter als unterschiedliche Absicht.

#### Kameraanweisungen früh platzieren

Setzen Sie die Kameraanweisung nahe an den Anfang des Prompts für zuverlässigere Ergebnisse:

```
Cinematic tracking shot of @Element1 walking through @Image1, leaves
blowing in the wind, golden afternoon light.
```

### Kling-O3-API-Verwendung

Kling O3 Reference to Video ist auch über die Venice-API verfügbar. Vollständige Details siehe [Video-Queue-API](https://docs.venice.ai/api-reference/endpoint/video/queue).

#### Python

```python
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "kling-o3-pro-reference-to-video",
        "prompt": "@Element1 walks through @Image1, camera tracking from behind",
        "duration": "8",
        "aspect_ratio": "16:9",
        "audio": True,
        "elements": [
            {
                "frontal_image_url": "https://example.com/character-front.jpg",
                "reference_image_urls": [
                    "https://example.com/character-side.jpg",
                    "https://example.com/character-angle.jpg"
                ]
            }
        ],
        "image_urls": [
            "https://example.com/scene-background.jpg"
        ]
    }
)

queue_id = response.json()["id"]
```

#### Element-Schema

Jedes Element im `elements`-Array akzeptiert:

| Feld | Typ | Erforderlich | Beschreibung |
|---|---|---|---|
| `frontal_image_url` | string | **Ja** | URL eines klaren Frontalbilds |
| `reference_image_urls` | string[] | Nein | URLs zusätzlicher Winkel (1–3). Werden sie weggelassen, wird das Frontalbild als Referenz verwendet. |

<Note>
Die API unterstützt zudem `video_url` für video-basierte Elemente, dies ist derzeit aber nicht im Video Studio verfügbar.
</Note>

### Kling-O3-Fehlersuche

| Problem | Wahrscheinliche Ursache | Lösung |
|---|---|---|
| Generate-Button ist deaktiviert | Keine visuellen Eingaben bereitgestellt | Mindestens eine visuelle Eingabe hinzufügen: Start-Frame, Element oder Szenen-Referenzbild |
| Fehler "Number of images exceeds the limit" | Zu viele kombinierte Eingaben | Gesamtsumme aus First-Frame + Last-Frame + Elementen + Szenenbildern muss ≤ 7 sein |
| Gesicht ändert sich zwischen Shots | Anderes oder fehlendes Frontalbild | Dasselbe Frontalbild konsistent verwenden, Beschreibung identisch halten |
| Kamerabewegung wirkt zufällig | Mehrere oder widersprüchliche Kameraanweisungen | Eine einzelne Kameraanweisung verwenden und früh im Prompt platzieren |
| Stil verschiebt sich zwischen Generierungen | Inkonsistente Szenenreferenzen oder gemischte Stile | Dieselben Szenenbilder wiederverwenden, Stil-Schlagwörter konsistent halten |
| Elemente verschmelzen in Multi-Charakter-Szenen | Vage räumliche Anweisungen | Position jedes Elements explizit angeben: "foreground left", "entering from right" |
| Hintergrund wirkt verzerrt | Überladenes oder komplexes Szenen-Referenzbild | Saubere, hochwertige Szenen-Referenzbilder verwenden |
| Bewegung wirkt unnatürlich | Zu viele Aktionen in einem Prompt | Aktion vereinfachen, kürzere Dauer, eine Aktion pro Shot |

<Tip>
Testen Sie mit einem 3–5 Sekunden langen Clip, bevor Sie sich für längere Dauern entscheiden. Kürzere Clips behalten Konsistenz besser bei und ermöglichen schnellere Iteration.
</Tip>

## Grok Imagine Reference to Video

Grok Imagine R2V verfolgt einen einfacheren Ansatz als Kling O3. Statt strukturierter Elemente mit Trennung in Frontal-/Referenzbild laden Sie **flache Referenzbilder** hoch und referenzieren sie direkt im Prompt mit `@Image1`, `@Image2` usw. Das Modell übernimmt diese Motive in das generierte Video.

### Wie es funktioniert

1. **1–7 Referenzbilder** hochladen — Fotos von Charakteren, Objekten oder Szenen, die im Video erscheinen sollen
2. Einen Prompt schreiben, der das Video beschreibt und `@Image1`, `@Image2` usw. verwendet, um konkrete Bilder zu referenzieren
3. Das Modell erzeugt ein Video, das diese Referenzen einbezieht

Wenn Sie keine `@Image`-Tags in den Prompt aufnehmen, werden alle hochgeladenen Bilder automatisch referenziert.

### Einstellungen

| Einstellung | Optionen | Standard |
|---|---|---|
| Seitenverhältnis | 16:9, 4:3, 3:2, 1:1, 2:3, 3:4, 9:16 | 16:9 |
| Auflösung | 480p, 720p | 480p |
| Dauer | 5s, 8s, 10s | 8s |

<Note>
Grok Imagine R2V unterstützt keine Audio-Generierung, keinen Multi-Shot-Modus und keine Elemente. Für diese Funktionen verwenden Sie Kling O3 R2V.
</Note>

### Schritt-für-Schritt-Anleitung (Video Studio)

#### 1. Modell auswählen

Gehen Sie zu [venice.ai/video](https://venice.ai/video). Wählen Sie im Modell-Browser **Grok Imagine R2V**.

#### 2. Referenzbilder hochladen

Klicken Sie in der Eingabe-Symbolleiste auf **References** (oder über das +-Menü), um das Panel für Referenzbilder zu öffnen. Laden Sie 1–7 Bilder der Charaktere, Objekte oder Szenen hoch, die im Video erscheinen sollen.

Jedes Bild wird in Ihrer Upload-Reihenfolge (von links nach rechts) automatisch als `@Image1`, `@Image2` usw. getaggt.

#### 3. Prompt schreiben

Beschreiben Sie das gewünschte Video. Verwenden Sie `@Image`-Tags, um konkrete Bilder zu referenzieren:

```
@Image1 and @Image2 walking together through a sunlit park,
camera slowly tracking alongside them, warm afternoon light.
```

Geben Sie `@` im Prompt-Feld ein, um ein Autocomplete-Menü der verfügbaren Bildreferenzen zu sehen.

<Tip>
Wenn Sie `@Image`-Tags vollständig weglassen, fügt das Backend automatisch Referenzen auf alle hochgeladenen Bilder hinzu. Das ist nützlich, wenn Sie alle Bilder verwenden möchten, ohne anzugeben, welches welches ist.
</Tip>

#### 4. Einstellungen konfigurieren und generieren

Öffnen Sie **Video Settings**, um Seitenverhältnis, Auflösung und Dauer anzupassen. Klicken Sie auf **Generate Video**.

### Grok-Imagine-R2V-Preise

Grok Imagine R2V wird nach Dauer und Auflösung abgerechnet:

| Auflösung | Pro Sekunde |
|---|---|
| 480p | ~0,063 $ |
| 720p | ~0,088 $ |

**Beispiel:** Ein 8-Sekunden-Video in 480p = 8 × 0,063 $ = **~0,50 $**

<Note>
Grok Imagine berechnet eine Content-Moderationsgebühr für erzeugte Videos, auch wenn das Video abgelehnt wird. Dies ist im vor der Generierung angezeigten Credit-Preis berücksichtigt.
</Note>

#### API-Parameter

| Feld | Typ | Erforderlich | Beschreibung |
|---|---|---|---|
| `model` | string | **Ja** | Muss `grok-imagine-reference-to-video` sein |
| `prompt` | string | **Ja** | Text-Prompt mit optionalen `@Image1`-, `@Image2`-Referenzen |
| `referenceImageUrls` | string[] | **Ja** | 1–7 Bild-URLs oder Daten-URLs |
| `duration` | string | Nein | `"5"`, `"8"` (Standard) oder `"10"` |
| `aspect_ratio` | string | Nein | z. B. `"16:9"` (Standard), `"9:16"`, `"1:1"` |
| `resolution` | string | Nein | `"480p"` (Standard) oder `"720p"` |

<Note>
Grok Imagine R2V verwendet die Felder `elements`, `image_urls` oder `imageUrl` nicht. Alle Referenzbilder werden über `referenceImageUrls` übergeben.
</Note>

### Grok-Imagine-R2V-Fehlersuche

| Problem | Wahrscheinliche Ursache | Lösung |
|---|---|---|
| Generate-Button ist deaktiviert | Keine Referenzbilder hochgeladen | Mindestens 1 Referenzbild hochladen |
| Fehler "At least one reference image is required" | `referenceImageUrls` leer oder fehlt | Mindestens eine Bild-URL in `referenceImageUrls` angeben |
| Falsches Bild ist dem `@Image`-Tag zugeordnet | Bildreihenfolge passt nicht zu den Tags | `@Image1` entspricht dem ersten Bild Ihrer Upload-Reihenfolge (von links nach rechts). Uploads bei Bedarf neu ordnen. |
| Motiv erscheint nicht im Video | Zu viele Referenzen ohne explizite Tags | `@Image`-Tags im Prompt verwenden, um konkret anzugeben, welche Bilder genutzt werden sollen |
| Niedrige Ausgabequalität | 480p-Auflösung genutzt | 720p für höhere Qualität versuchen (höhere Kosten) |
| Video zu kurz | Standarddauer ist 8s | Dauer auf `"10"` setzen für längere Videos |

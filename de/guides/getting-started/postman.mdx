## Überblick

Venice stellt eine umfassende Postman-Collection zur Verfügung, mit der Entwickler die vollen Möglichkeiten unserer API erkunden und testen können. Diese Collection enthält vorkonfigurierte Anfragen, Beispiele und Umgebungsvariablen, um Ihnen den Einstieg in die KI-Dienste von Venice zu erleichtern.

## Zugriff auf die Collection

Unsere offizielle Postman-Collection ist im Venice AI Workspace verfügbar:

* [Venice AI Postman Workspace](https://www.postman.com/veniceai/workspace/venice-ai-workspace)
* [Venice AI Postman Examples](https://postman.venice.ai/)

## Funktionen der Collection

- **Einsatzbereite Anfragen**: Vorkonfigurierte API-Aufrufe für alle Venice-Endpoints
- **Umgebungsvorlagen**: Sauber strukturierte Umgebungsvariablen
- **Anfrage-Beispiele**: Praxisnahe Verwendungsbeispiele für jeden Endpoint
- **Antwort-Beispiele**: Beispielantworten, damit Sie die Ausgabe der API besser verstehen
- **Dokumentation**: Inline-Dokumentation für jede Anfrage

## Erste Schritte

<Steps>
   <Step title="Collection forken">
      - Navigieren Sie zum Venice AI Workspace
      - Klicken Sie auf "Fork", um Ihre eigene Kopie der Collection zu erstellen
      - Wählen Sie Ihren Workspace als Ziel
   </Step>

   <Step title="Umgebung einrichten">
      - Erstellen Sie eine neue Umgebung in Postman
      - Fügen Sie Ihren Venice-API-Schlüssel hinzu
      - Konfigurieren Sie die Base-URL: `https://api.venice.ai/api/v1`
   </Step>

   <Step title="Ihre erste Anfrage senden">
      - Wählen Sie eine beliebige Anfrage aus der Collection
      - Stellen Sie sicher, dass Ihre Umgebung ausgewählt ist
      - Klicken Sie auf "Send", um die API zu testen
   </Step>
</Steps>

## Verfügbare Endpoints

Die Collection enthält Beispiele für alle Venice-API-Endpoints:

- Textgenerierung
- Bildgenerierung
- Modellinformationen
- Bild-Upscaling
- System-Prompt-Konfiguration

## Best Practices

- Halten Sie Ihren API-Schlüssel sicher und geben Sie ihn niemals weiter
- Verwenden Sie Umgebungsvariablen für sensible Informationen
- Testen Sie Antworten in der Postman-Konsole, bevor Sie sie implementieren
- Überprüfen Sie die Beispielantworten auf erwartete Datenstrukturen

<Note>*Hinweis: Die Postman-Collection wird regelmäßig aktualisiert, um die neuesten API-Änderungen und -Funktionen widerzuspiegeln.*</Note>

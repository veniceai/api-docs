Venice AI ist ein **Drop-in-Ersatz** für OpenAI. Dasselbe SDK, derselbe Code — nur zwei Zeilen ändern. Sie erhalten datenschutzorientierte Inference, unbeschränkte Modelle und konkurrenzfähige Preise.

## Die 2-Zeilen-Migration

### Python

```python
# Vorher (OpenAI)
from openai import OpenAI
client = OpenAI()

# Nachher (Venice) — api_key und base_url ändern
from openai import OpenAI
client = OpenAI(
    api_key="your-venice-api-key",          # ← Änderung 1
    base_url="https://api.venice.ai/api/v1"  # ← Änderung 2
)
```

### Node.js

```javascript
// Vorher (OpenAI)
import OpenAI from 'openai';
const client = new OpenAI();

// Nachher (Venice)
import OpenAI from 'openai';
const client = new OpenAI({
  apiKey: 'your-venice-api-key',
  baseURL: 'https://api.venice.ai/api/v1',
});
```

### cURL

```bash
# Vorher
curl https://api.openai.com/v1/chat/completions ...

# Nachher — einfach URL und Schlüssel ändern
curl https://api.venice.ai/api/v1/chat/completions ...
```

### Umgebungsvariablen

```bash
# Vorher
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1

# Nachher
OPENAI_API_KEY=your-venice-api-key
OPENAI_BASE_URL=https://api.venice.ai/api/v1
```

<Tip>
Viele Bibliotheken und Tools lesen `OPENAI_API_KEY` und `OPENAI_BASE_URL` automatisch. Häufig reicht es, diese Umgebungsvariablen zu aktualisieren.
</Tip>

## Funktionskompatibilität

| Funktion | OpenAI | Venice | Hinweise |
|---------|--------|--------|-------|
| Chat Completions | ✅ | ✅ | Vollständig kompatibel |
| Streaming | ✅ | ✅ | SSE-Format identisch |
| Function Calling | ✅ | ✅ | Gleicher `tools`-Parameter |
| Structured Output | ✅ | ✅ | Gleiches `response_format` |
| Vision | ✅ | ✅ | Gleiches Content-Array-Format |
| Embeddings | ✅ | ✅ | Gleiche API |
| Bildgenerierung | ✅ | ✅ | OpenAI-kompatibel via `/images/generations`* |
| TTS | ✅ | ✅ | Kompatibel |
| STT | ✅ | ✅ | Kompatibel |
| Assistants API | ✅ | ❌ | Verwenden Sie stattdessen Characters oder Minds |
| Batch API | ✅ | ❌ | Noch nicht verfügbar |
| Fine-Tuning | ✅ | ❌ | Nicht verfügbar |

*Venice stellt zusätzlich einen OpenAI-kompatiblen Endpoint unter `POST /images/generations` bereit, um die Migration von DALL-E zu erleichtern. Für Venice's native Image-API mit zusätzlichen Optionen siehe [Image Generate](/de/api-reference/endpoint/image/generate).

## Funktionen, die nur Venice bietet

Venice bietet Fähigkeiten, die OpenAI nicht hat:

### 1. Eingebaute Web-Suche
```python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Latest AI news today"}],
    extra_body={
        "venice_parameters": {
            "enable_web_search": "auto"
        }
    }
)
```

### 2. Web-Scraping
```python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Summarize https://example.com/article"}],
    extra_body={
        "venice_parameters": {
            "enable_web_scraping": True
        }
    }
)
```

### 3. Characters (KI-Personas)
```python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Tell me about yourself"}],
    extra_body={
        "venice_parameters": {
            "character_slug": "venice-ai"
        }
    }
)
```

### 4. Unbeschränkte Modelle
Venice's private Modelle haben keine Inhaltsfilterung und eignen sich daher für:
- Kreatives Schreiben ohne Guardrails
- Sicherheitsforschung und Red Teaming
- Ehrliche Analysen ohne Verweigerungsmuster
- Medizinische/rechtliche Informationen ohne Disclaimer

### 5. Videogenerierung
```python
# Einen Videogenerierungs-Job in die Warteschlange stellen
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"},
    json={
        "model": "wan-2.6-text-to-video",
        "prompt": "A serene lake at sunset with gentle waves",
        "resolution": "720p",
        "duration": 5,
    }
)
job_id = response.json()["id"]
```

## Warum migrieren?

### Datenschutz
- **Zero Data Retention** bei privaten Modellen — Ihre Prompts werden nie gespeichert
- **Kein Training auf Ihren Daten** — niemals
- OpenAI bewahrt Daten 30 Tage auf und kann sie zur Sicherheitsforschung verwenden

### Kosten
- Private Modelle sind **oft günstiger** als OpenAI-Äquivalente
- `qwen3-4b` zu $0,05/1M Input ist 10× günstiger als gpt-4o-mini
- `venice-uncensored` zu $0,20/1M Input vs. gpt-4o zu $2,50/1M

### Freiheit
- **Keine Inhaltsfilterung** bei unbeschränkten Modellen
- Keine Kontosperrungen für kontroverse Anwendungsfälle
- Web3-nativ mit Krypto-Zahlungsoptionen
- DIEM-Staking für tägliche Credits

### Modellvielfalt
- Zugriff auf Modelle mehrerer Anbieter (Qwen, Llama, Mistral, Gemma, Claude, GPT, Grok usw.)
- Pro Anfrage zwischen privaten und anonymisierten Modellen wechseln
- Regelmäßig neue Modelle

## Framework-Migration

Die meisten KI-Frameworks funktionieren mit Venice durch Änderung der Base-URL:

| Framework | Erforderliche Änderung |
|-----------|----------------|
| LangChain | `base_url` in `ChatOpenAI` |
| Vercel AI SDK | `baseURL` in `createOpenAI` |
| CrewAI | `OPENAI_API_BASE`-Umgebungsvariable |
| LlamaIndex | `api_base` in `OpenAI` |
| AutoGen | `base_url` in Config |
| Haystack | `api_base_url` in `OpenAIGenerator` |
| Claude Code | `--api-base`-Flag oder Umgebungsvariable |
| Cursor | Custom API Endpoint in den Einstellungen |
| Continue.dev | `apiBase` in `config.json` |

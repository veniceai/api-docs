Datei-Eingaben erlauben es, Dokumente und Quelldateien direkt an einen `/chat/completions`-Request anzuhängen. Venice extrahiert die Datei zu Text, bevor sie an das ausgewählte Modell gesendet wird — Sie können also Fragen stellen, zusammenfassen, vergleichen oder Datei-Inhalte transformieren, ohne zuerst einen eigenen Parser zu bauen.

Verwenden Sie Datei-Eingaben, wenn Ihr Prompt vom Inhalt eines Dokuments, einer Tabelle, einer Markdown-Datei, einer JSON-Datei oder einer Code-Datei abhängt. Es handelt sich um request-scoped Eingaben, nicht um persistenten Datei-Speicher — fügen Sie die Datei daher jedem Request bei, der sie benötigt.

<Info>
Datei-Eingaben verwenden das OpenAI-kompatible Chat-Content-Array. Fügen Sie einen Content-Block mit `type: "file"` hinzu und liefern Sie den Datei-Inhalt in `file.file_data`.
</Info>

## Unterstützte Dateitypen

Die Chat-API akzeptiert Datei-Eingaben entweder als Base64-Daten-URLs oder als öffentlich zugängliche URLs.

Die maximale Dateigröße beträgt **25 MB pro Datei**, gemessen nach dem Dekodieren einer Base64-Daten-URL oder nach dem Abruf einer URL.

| Kategorie | Formate |
|----------|---------|
| Dokumente | PDF, DOCX, PPTX |
| Tabellen | XLSX, XLS, CSV |
| Text und Daten | TXT, Markdown, JSON |
| Quellcode | Die gängigsten Code-Dateien, darunter `.py`, `.js`, `.ts`, `.c`, `.cpp`, `.java`, `.go`, `.rs`, `.ps1`, `.sh`, `.yaml` und `.sql` |

<Note>
Dateien werden vor der Inference zu Text extrahiert. Der extrahierte Text zählt zum Eingabe-Kontext des Modells — wählen Sie daher ein Modell mit ausreichend `availableContextTokens` für die Datei plus Ihre Anweisungen und die erwartete Antwort.
</Note>

## Grundlegende Nutzung

Senden Sie ein `messages`-Array, in dem der `content` der User-Nachricht ein Array aus Text- und File-Blöcken ist:

<CodeGroup>
```python Python
import base64
import os
from pathlib import Path

from openai import OpenAI

client = OpenAI(
    api_key=os.environ["VENICE_API_KEY"],
    base_url="https://api.venice.ai/api/v1",
)

path = Path("q3-report.pdf")
file_data = "data:application/pdf;base64," + base64.b64encode(path.read_bytes()).decode("utf-8")

response = client.chat.completions.create(
    model="openai-gpt-55",
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "text",
                    "text": "Summarize this report in five bullets and list the main risks.",
                },
                {
                    "type": "file",
                    "file": {
                        "file_data": file_data,
                        "filename": "q3-report.pdf",
                    },
                },
            ],
        }
    ],
)

print(response.choices[0].message.content)
```

```javascript Node.js
import OpenAI from "openai";
import { readFile } from "node:fs/promises";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const pdf = await readFile("q3-report.pdf");
const fileData = `data:application/pdf;base64,${pdf.toString("base64")}`;

const response = await client.chat.completions.create({
  model: "openai-gpt-55",
  messages: [
    {
      role: "user",
      content: [
        {
          type: "text",
          text: "Summarize this report in five bullets and list the main risks.",
        },
        {
          type: "file",
          file: {
            file_data: fileData,
            filename: "q3-report.pdf",
          },
        },
      ],
    },
  ],
});

console.log(response.choices[0].message.content);
```

```bash cURL
PDF_BASE64=$(base64 < q3-report.pdf | tr -d '\n')

curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \

{

        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,$PDF_BASE64",
            "filename": "q3-report.pdf"
          }
        }
      ]
    }
  ]
}
EOF
```

## Datei-URLs

Wenn die Datei bereits unter einer öffentlichen HTTP- oder HTTPS-URL gehostet ist, übergeben Sie die URL in `file_data`, statt sie Base64 zu kodieren:

```json
{
  "model": "openai-gpt-55",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "Identify the governing law, renewal terms, and termination rights in this agreement."
        },
        {
          "type": "file",
          "file": {
            "file_data": "https://example.com/contracts/vendor-agreement.pdf",
            "filename": "vendor-agreement.pdf"
          }
        }
      ]
    }
  ]
}
```

<Warning>
Verwenden Sie nur öffentliche URLs, die Venice ohne Authentifizierung abrufen kann. Für private Dateien senden Sie eine Base64-Daten-URL.
</Warning>

## Mehrere Dateien

Sie können in derselben Nachricht mehr als einen File-Block einschließen. Stellen Sie den Dateien eine kurze Textanweisung voran, damit das Modell weiß, wie es sie verwenden soll.

```json
{
  "model": "openai-gpt-55",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "Compare these two policy drafts. Return the material differences and recommend which version is clearer."
        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,JVBERi0xLjQK...",
            "filename": "policy-v1.pdf"
          }
        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,JVBERi0xLjQK...",
            "filename": "policy-v2.pdf"
          }
        }
      ]
    }
  ]
}
```

Für beste Ergebnisse benennen Sie jede Datei eindeutig und referenzieren diese Namen in Ihrem Prompt.

## Daten-URLs

Bei lokalen Dateien kodieren Sie die Datei-Bytes als Base64 und stellen den korrekten MIME-Typ voran:

| Dateityp | Daten-URL-Präfix |
|-----------|-----------------|
| PDF | `data:application/pdf;base64,` |
| DOCX | `data:application/vnd.openxmlformats-officedocument.wordprocessingml.document;base64,` |
| PPTX | `data:application/vnd.openxmlformats-officedocument.presentationml.presentation;base64,` |
| XLSX | `data:application/vnd.openxmlformats-officedocument.spreadsheetml.sheet;base64,` |
| CSV | `data:text/csv;base64,` |
| Markdown | `data:text/markdown;base64,` |
| Klartext | `data:text/plain;base64,` |
| JSON | `data:application/json;base64,` |

Falls Sie den genauen MIME-Typ nicht kennen, verwenden Sie `application/octet-stream`. Ein präziser `filename` hilft Venice trotzdem, die Datei zu erkennen und anzuzeigen.

## Arbeit mit großen Dateien

Da Dateien zu Prompt-Text werden, können große Dateien Latenz, Token-Verbrauch und Kosten erhöhen. Behalten Sie das Kontextfenster des Modells im Auge.

Die Rohdatei muss 25 MB oder kleiner sein. Base64-Kodierung erhöht die Request-Größe um ca. 33 %, sodass eine Datei nahe der 25-MB-Grenze einen größeren JSON-Request-Body erzeugt.

Gute Muster für große Dateien:

- Eine konkrete Aufgabe statt eines breiten "analysiere alles"-Prompts stellen.
- Nur die für die aktuelle Antwort benötigten Dokumente einbinden.
- Modelle mit größeren `availableContextTokens` für lange Berichte oder Codebasen verwenden.
- Stabile, wiederkehrende Dokumente vor dynamische Nutzerfragen setzen, wenn Sie zusätzlich [Prompt-Caching](/de/guides/features/prompt-caching) verwenden.
- `stream: true` verwenden, wenn Sie eine lange Antwort erwarten.

## Datei-Eingaben vs. Text-Parser

Verwenden Sie Chat-Datei-Eingaben, wenn das Modell direkt über die Datei schlussfolgern soll.

Verwenden Sie die [Text-Parser-API](/de/api-reference/endpoint/augment/text-parser), wenn Sie zuerst Text extrahieren, die Token-Anzahl prüfen, den extrahierten Text in Ihrem eigenen System speichern oder denselben extrahierten Text in mehrere Requests einbinden möchten.

| Anforderung | Verwenden |
|------|-----|
| Ein Modell in einer Anfrage zu einem Dokument fragen | Chat-Datei-Eingabe |
| Text ohne Modell-Inference extrahieren | Text-Parser-API |
| Extrahierte Token-Anzahl vor dem Prompting prüfen | Text-Parser-API |
| Extrahierten Text über viele Requests wiederverwenden | Text-Parser-API, danach Text in Prompts einbinden |

## Best Practices

- Geben Sie nach Möglichkeit immer einen `filename` an, insbesondere bei mehreren Dateien.
- Stellen Sie die Anweisung vor die File-Blöcke, damit das Modell die Aufgabe kennt, bevor es den extrahierten Inhalt liest.
- Verwenden Sie öffentliche URLs nur für Dateien, die ohne Cookies, Header oder signierte Sessions abrufbar sind.
- Bevorzugen Sie Base64-Daten-URLs für private oder in Ihrer Anwendung erzeugte Dateien.
- Stellen Sie fokussierte Fragen und geben Sie das gewünschte Ausgabeformat an.
- Für strukturierte Extraktion kombinieren Sie Datei-Eingaben mit [Strukturierten Antworten](/de/guides/features/structured-responses).

## Fehlersuche

<AccordionGroup>
  <Accordion title="Das Modell sagt, es könne nicht auf die Datei zugreifen">
    Stellen Sie sicher, dass der Message-Content ein Array verwendet und einen Block `type: "file"` enthält. Wenn Sie eine URL verwendet haben, prüfen Sie, dass diese öffentlich ohne Authentifizierung erreichbar ist.
  </Accordion>

  <Accordion title="Die Anfrage ist langsam oder teuer">
    Die Datei kann in eine große Textmenge extrahiert werden. Verwenden Sie ein Modell mit größerem Kontext, schränken Sie die Aufgabe ein, senden Sie weniger Dateien oder vorab-extrahieren und kürzen Sie den Text mit der Text-Parser-API.
  </Accordion>

  <Accordion title="Die Antwort ignoriert eine meiner Dateien">
    Geben Sie jeder Datei einen aussagekräftigen `filename` und verweisen Sie in Ihrem Prompt direkt auf die Dateinamen. Beispiel: "Compare `policy-v1.pdf` against `policy-v2.pdf`."
  </Accordion>

  <Accordion title="Ein Modell lehnt den Datei-Inhalt ab">
    Datei-Eingaben sind auf kompatiblen Chat-Modellen verfügbar. Prüfen Sie die [Modelle-Seite](/de/models/overview) auf aktuelle Modell-Fähigkeiten und Kontextlimits oder versuchen Sie ein aktuelles Textmodell mit großem Kontext.
  </Accordion>
</AccordionGroup>

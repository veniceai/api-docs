Venice bietet nun strukturierte Ausgaben über das Feld "response\_format" in der API. Mit diesem Feld können Sie Antworten auf Ihre Prompts erzeugen, die einem festgelegten, vordefinierten Format folgen. Mit dieser neuen Methode ist es weniger wahrscheinlich, dass die Modelle falsche Schlüssel oder Werte in der Antwort halluzinieren — was bei der Steuerung über System-Prompts oder Function Calling häufiger vorkam.

Das Feld "response\_format" für strukturierte Ausgaben nutzt das OpenAI-API-Format und ist im OpenAI-Leitfaden [hier](https://platform.openai.com/docs/guides/structured-outputs) näher beschrieben. OpenAI hat außerdem einen Einführungsartikel zur Verwendung strukturierter Ausgaben in der API speziell [hier](https://openai.com/index/introducing-structured-outputs-in-the-api/) veröffentlicht. Da es sich um eine fortgeschrittene Funktionalität handelt, gibt es am Ende dieser Seite einige "Stolperfallen", die beachtet werden sollten.

Diese Funktionalität ist nicht für alle Modelle nativ verfügbar. Bitte beachten Sie den Modellabschnitt [hier](https://docs.venice.ai/api-reference/endpoint/models/list?playground=open) und suchen Sie nach "supportsResponseSchema" für die zutreffenden Modelle.

```json
    {
      "id": "venice-uncensored",
      "type": "text",
      "object": "model",
      "created": 1726869022,
      "owned_by": "venice.ai",
      "model_spec": {
        "availableContextTokens": 32768,
        "capabilities": {
          "supportsFunctionCalling": true,
          "supportsResponseSchema": true,
          "supportsWebSearch": true
        },
```

### Strukturierte Antworten verwenden

Um "response\_format" richtig zu nutzen, können Sie Ihr Schema mit verschiedenen "properties" definieren, die Kategorien von Ausgaben repräsentieren — jede mit individuell konfigurierten Datentypen. Diese Objekte können verschachtelt werden, um komplexere Ausgabestrukturen zu erstellen.

Hier ein Beispiel für einen API-Aufruf mit response\_format, der den Schritt-für-Schritt-Prozess zum Lösen einer mathematischen Gleichung erklärt.

Sie sehen, dass die Properties so konfiguriert wurden, dass in der Antwort sowohl "steps" als auch "final\_answer" erforderlich sind. In der Verschachtelung besteht die Kategorie "steps" jeweils aus einer "explanation" und einem "output", beides als String.

```json
curl --request POST \
  --url https://api.venice.ai/api/v1/chat/completions \
  --header 'Authorization: Bearer <api-key>' \
  --header 'Content-Type: application/json' \
  --data '{
  "model": "venice-uncensored",
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful math tutor."
    },
    {
      "role": "user",
      "content": "solve 8x + 31 = 2"
    }
  ],
  "response_format": {
    "type": "json_schema",
    "json_schema": {
      "name": "math_response",
      "strict": true,
      "schema": {
        "type": "object",
        "properties": {
          "steps": {
            "type": "array",
            "items": {
              "type": "object",
              "properties": {
                "explanation": {
                  "type": "string"
                },
                "output": {
                  "type": "string"
                }
              },
              "required": ["explanation", "output"],
              "additionalProperties": false
            }
          },
          "final_answer": {
            "type": "string"
          }
        },
        "required": ["steps", "final_answer"],
        "additionalProperties": false
      }
    }
  }
}

```

Hier die Antwort, die vom Modell zurückgegeben wurde. Die Struktur folgt den Vorgaben, indem zuerst die "steps" mit "explanation" und "output" jedes Schritts und anschließend die "final answer" ausgegeben werden.

```json
{
  "steps": [
    {
      "explanation": "Subtract 31 from both sides to isolate the term with x.",
      "output": "8x + 31 - 31 = 2 - 31"
    },
    {
      "explanation": "This simplifies to 8x = -29.",
      "output": "8x = -29"
    },
    {
      "explanation": "Divide both sides by 8 to solve for x.",
      "output": "x = -29 / 8"
    }
  ],
  "final_answer": "x = -29 / 8"
}

```

Obwohl dies ein einfaches Beispiel ist, lässt sich dies auf fortgeschrittenere Anwendungsfälle übertragen wie: Datenextraktion, Chain-of-Thought-Aufgaben, UI-Generierung, Datenkategorisierung und viele weitere.

### Stolperfallen

Hier einige wichtige Anforderungen bei der Verwendung strukturierter Ausgaben über response\_format:

* Initiale Anfragen mit response\_format können länger dauern, bis eine Antwort erzeugt wird. Nachfolgende Anfragen haben nicht dieselbe Latenz wie die erste Anfrage.

* Bei größeren Anfragen kann das Modell scheitern, wenn `max_tokens` oder das Modell-Timeout erreicht oder Rate Limits verletzt werden

* Ein falsches Schemaformat führt beim Abschluss zu Fehlern, üblicherweise durch Timeout

* Auch wenn response\_format sicherstellt, dass das Modell in einer bestimmten Weise ausgibt, garantiert es nicht, dass das Modell inhaltlich korrekte Informationen liefert. Der Inhalt wird vom Prompt und der Modellqualität bestimmt.

* Strukturierte Ausgaben über response\_format sind nicht mit parallelen Function Calls kompatibel

* Wichtig: Alle Felder oder Parameter müssen einen `required`-Tag enthalten. Um ein Feld optional zu machen, müssen Sie eine `null`-Option in der `type`-Definition ergänzen, etwa so: `"type": ["string", "null"]`&#x20;

* Felder lassen sich durch eine `null`-Option im Required-Feld optional machen, um eine leere Antwort zuzulassen.

* Wichtig: `additionalProperties` muss auf false gesetzt sein, damit response\_format korrekt funktioniert

* Wichtig: `strict` muss auf true gesetzt sein, damit response\_format korrekt funktioniert

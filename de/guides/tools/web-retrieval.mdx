Venice stellt separate APIs zum Auffinden von Webseiten und zum Extrahieren ihrer Inhalte bereit:

- **Web Search** liefert strukturierte Ergebnisse mit Titeln, URLs, Snippets und Datumsangaben.
- **Web Scrape** wandelt eine ausgewählte öffentliche Seite in Markdown um.

Verwenden Sie sie zusammen, wenn Ihre Anwendung vor der Inferenz aktuelles, überprüfbares Ausgangsmaterial benötigt.

## Das Web durchsuchen

Senden Sie eine Anfrage an die Search API:

```bash
curl https://api.venice.ai/api/v1/augment/search \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query":"latest developments in decentralized AI"}'
```

Brave ist der Standardanbieter und bietet Zero Data Retention. Google ist ebenfalls verfügbar, wobei Anfragen anonymisiert über Venice geleitet werden. Weitere Informationen zur Anbieterauswahl und zum vollständigen Antwortschema finden Sie in der [Web Search API-Referenz](/api-reference/endpoint/augment/search).

## Eine Seite scrapen

Nachdem Sie ein Ergebnis ausgewählt haben, senden Sie dessen öffentliche URL an die Scrape API:

```bash
curl https://api.venice.ai/api/v1/augment/scrape \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/article"}'
```

Der Endpunkt gibt den Seiteninhalt als Markdown zurück. Er fordert zunächst die native Markdown-Darstellung der Seite an, sofern verfügbar, und nutzt andernfalls eine browserbasierte Extraktion.

## Search-to-Scrape-Workflow

<Steps>
  <Step title="Suchen">
    Reichen Sie eine gezielte Anfrage ein und sammeln Sie die strukturierten Ergebnisse.
  </Step>
  <Step title="Quellen auswählen">
    Filtern Sie Ergebnisse nach Relevanz, Domain und Datum, bevor Sie Seiteninhalte abrufen.
  </Step>
  <Step title="Scrapen">
    Extrahieren Sie Markdown nur von den Seiten, die Ihre Anwendung tatsächlich verwenden möchte.
  </Step>
  <Step title="Inhalte verwenden oder speichern">
    Fügen Sie das Markdown in einen Prompt ein, erstellen Sie Embeddings, cachen Sie es oder bewahren Sie Quell-URLs für Zitate auf.
  </Step>
</Steps>

## Eigenständige Suche oder Modell-Websuche?

Verwenden Sie die eigenständigen APIs, wenn Ihre Anwendung Kontrolle über die Quellenauswahl benötigt oder abgerufene Inhalte wiederverwenden möchte. Aktivieren Sie die Websuche bei einer Modellanfrage, wenn Venice das Retrieval als Teil einer einzelnen Inferenzanfrage übernehmen soll.

| Bedarf | Ansatz |
|---|---|
| Ergebnisse vor der Inferenz prüfen oder filtern | Eigenständige Web Search |
| Eine bestimmte bekannte URL extrahieren | Eigenständiges Web Scrape |
| Abgerufene Inhalte cachen oder wiederverwenden | Eigenständige APIs |
| Das Modell aktuellen Kontext automatisch abrufen lassen | Modell-Websuche |

<Warning>
Web Search und Web Scrape sind experimentelle APIs. Ihre Anfrage- und Antwortformate können sich ohne Vorankündigung ändern. Web Scrape akzeptiert nur öffentlich zugängliche Seiten, und Websites, die automatisierten Zugriff blockieren, können abgelehnt werden.
</Warning>

[Aider](https://aider.chat/) ist ein KI-Pair-Programming-Tool, das dein lokales git-Repository vom Terminal aus bearbeitet. Venice funktioniert über die [OpenAI-kompatible API](https://aider.chat/docs/llms/openai-compat.html)-Unterstützung von Aider — setze die Venice-Basis-URL und den API-Schlüssel und stelle der Modell-ID `openai/` voran.

## Voraussetzungen

- Ein Venice API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api)
- Python 3.9+ auf deinem Rechner verfügbar
- Ein git-Repository, in dem du arbeitest

## Einrichtung

<Steps>
  <Step title="Aider installieren">
    Installiere Aider mit dem offiziellen Installer:

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    Siehe die [Aider-Installationsdokumentation](https://aider.chat/docs/install.html), falls du eine andere Installationsmethode bevorzugst.
  </Step>

  <Step title="Venice als OpenAI-Endpoint konfigurieren">
    Richte Aider auf Venice aus, indem du die OpenAI-kompatible Basis-URL und deinen Venice API-Schlüssel setzt.

    **Mac / Linux:**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell):**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (Eingabeaufforderung, persistiert):**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    Starte das Terminal nach `setx` neu, damit die neuen Variablen verfügbar sind.
  </Step>

  <Step title="Aider mit einem Venice-Modell starten">
    Stelle im Projektverzeichnis der Venice-Modell-ID `openai/` voran:

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    Weitere gute Einstiegsmodelle:

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    Alle Textmodell-IDs findest du im [Modell-Katalog](/models/text).
  </Step>
</Steps>

## Konfiguration mit `.env` persistieren

Für ein projektlokales Setup erstelle eine `.env`-Datei im Root deines git-Repositorys:

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

Starte Aider dann wie gewohnt:

```bash
aider
```

Aider liest `.env` auch aus deinem Home-Verzeichnis. Werte im Repo-Root haben Vorrang vor Werten aus dem Home-Verzeichnis. Halte diese Datei aus git heraus — füge `.env` zur `.gitignore` hinzu, falls sie noch nicht ignoriert wird.

<Info>
Du kannst auch `AIDER_OPENAI_API_BASE` und `AIDER_OPENAI_API_KEY` setzen. Die offizielle Aider-Dokumentation nennt für OpenAI-kompatible Anbieter weiterhin `OPENAI_API_BASE` und `OPENAI_API_KEY`.
</Info>

---

## Konfiguration mit YAML persistieren

Du kannst dieselben Einstellungen in `.aider.conf.yml` im Root deines Repositorys oder in deinem Home-Verzeichnis ablegen:

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

Führe dann aus:

```bash
aider
```

## Setup verifizieren

Frage Aider nach dem Start etwas Kleines, das belegt, dass das Modell erreichbar ist:

```txt
Reply with one sentence confirming you are running through Venice.
```

Wenn Endpoint und Schlüssel korrekt sind, streamt Aider eine Antwort vom ausgewählten Venice-Modell.

Du kannst den API-Schlüssel auch außerhalb von Aider bestätigen:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## Modelle auswählen

| Anwendungsfall              | Beispiel-Modell-ID  | Aider-Flag                         |
| --------------------------- | ------------------- | ---------------------------------- |
| Starkes Coding / Refactoring | `zai-org-glm-5-1`   | `--model openai/zai-org-glm-5-1`   |
| Schnelle Iteration          | `minimax-m25`       | `--model openai/minimax-m25`       |
| Coding mit langem Kontext   | `kimi-k2-5`         | `--model openai/kimi-k2-5`         |
| Unzensierter Chat           | `venice-uncensored` | `--model openai/venice-uncensored` |

Stelle den Venice-Modell-IDs beim Start von Aider immer `openai/` voran. Ohne dieses Präfix leitet Aider die Anfrage nicht über den OpenAI-kompatiblen Provider-Pfad.

Aktuelle Modell-IDs zur Laufzeit ermitteln:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

Oder prüfe [GET /models/traits](/api-reference/endpoint/models/traits) für Defaults wie `text:default`.

## Konfigurationshinweise

- Halte `OPENAI_API_BASE` auf `https://api.venice.ai/api/v1` gesetzt. Hänge kein `/chat/completions` an; Aider fügt den Chat-Pfad selbst hinzu.
- Verwende einen Venice API-Schlüssel in `OPENAI_API_KEY`. Nutze keinen OpenAI-Schlüssel mit der Venice-Basis-URL.
- Aider warnt möglicherweise, dass das Modell ein unbekanntes Kontextfenster oder unbekannte Kosten hat. Diese Warnung ist für viele Venice-Modell-IDs zu erwarten und kann meist ignoriert werden. Siehe [Aider Model-Warnungen](https://aider.chat/docs/llms/warnings.html).
- Bevorzuge leistungsfähige Coding-/Reasoning-Modelle für Bearbeitungen über mehrere Dateien. Kleinere oder nicht auf Coding ausgelegte Modelle können mit den Bearbeitungsformaten von Aider Probleme haben.

<Warning>
Das Setzen von `OPENAI_API_BASE` und `OPENAI_API_KEY` in deiner Shell wirkt sich auch auf andere Tools aus, die diese Variablen lesen. Bevorzuge eine projektbezogene `.env` oder `.aider.conf.yml`, wenn du OpenAI anderweitig in derselben Umgebung nutzt.
</Warning>

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="401- oder Authentifizierungsfehler">
    Stelle sicher, dass `OPENAI_API_KEY` ein Venice-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api) ist, ohne zusätzliche Leerzeichen. Starte unter Windows nach `setx` das Terminal neu.
  </Accordion>
  <Accordion title="Modell nicht gefunden oder leere Antwort">
    Bestätige, dass die Modell-ID im [Textmodell-Katalog](/models/text) existiert und dass du Aider mit dem `openai/`-Präfix gestartet hast, zum Beispiel `openai/zai-org-glm-5-1`.
  </Accordion>
  <Accordion title="Anfragen gehen weiterhin an OpenAI">
    Prüfe, dass `OPENAI_API_BASE` in derselben Shell-Sitzung exakt `https://api.venice.ai/api/v1` lautet. Gib es mit `echo $OPENAI_API_BASE` aus (oder `$env:OPENAI_API_BASE` in PowerShell).
  </Accordion>
  <Accordion title="Warnung zu unbekanntem Kontextfenster / Kosten">
    Bei vielen Venice-Modellen zu erwarten. Aider fällt auf sinnvolle Defaults zurück. Um sie zu unterdrücken, füge Modell-Metadaten wie in den [Aider Advanced Model Settings](https://aider.chat/docs/config/adv-model-settings.html) beschrieben hinzu.
  </Accordion>
</AccordionGroup>

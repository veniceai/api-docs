[Claude Code](https://docs.anthropic.com/en/docs/claude-code) ist Anthropic's CLI-Tool für agentisches Coding. Dieser Leitfaden zeigt, wie Sie es über Venice AI nutzen, um Pay-per-Token-Zugriff auf Claude Opus 4.5/4.6 und Sonnet 4.5/4.6 zu erhalten.

## Warum Sie einen Router brauchen

Claude Code verbindet sich standardmäßig direkt mit Anthropic's API. Um es mit Venice zu nutzen, benötigen Sie [claude-code-router](https://github.com/musistudio/claude-code-router), einen Open-Source-Local-Proxy, der:

<Steps>
  <Step title="Abfängt" icon="hand">
    Fängt ausgehende Requests von Claude Code ab, bevor sie Anthropic erreichen
  </Step>
  <Step title="Transformiert" icon="arrows-rotate">
    Konvertiert das Request-Format und ordnet Modell-IDs zu (z. B. `claude-opus-4-5`)
  </Step>
  <Step title="Leitet weiter" icon="route">
    Leitet Requests an Venice an `api.venice.ai/api/v1/chat/completions` weiter
  </Step>
</Steps>

## Einrichtung

<Steps>
  <Step title="Claude Code installieren">
    Falls noch nicht geschehen, installieren Sie Anthropic's Claude-Code-CLI:

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="Router installieren">
    ```bash
    npm install -g @musistudio/claude-code-router
    ```
  </Step>

  <Step title="API-Schlüssel holen">
    Erzeugen Sie einen Schlüssel auf [venice.ai/settings/api](https://venice.ai/settings/api). Sie fügen ihn im nächsten Schritt direkt in die Konfigurationsdatei ein.
  </Step>

  <Step title="Konfiguration erstellen">
    Config-Verzeichnis anlegen:

    ```bash
    mkdir -p ~/.claude-code-router
    ```

    Dann mit dem bevorzugten Editor `~/.claude-code-router/config.json` erstellen:

    ```bash
    # Mit nano
    nano ~/.claude-code-router/config.json

    # Oder mit VS Code
    code ~/.claude-code-router/config.json
    ```

    Fügen Sie die folgende Konfiguration ein:

    ```json
    {
      "APIKEY": "",
      "LOG": true,
      "LOG_LEVEL": "info",
      "API_TIMEOUT_MS": 600000,
      "HOST": "127.0.0.1",
      "Providers": [
        {
          "name": "venice",
          "api_base_url": "https://api.venice.ai/api/v1/chat/completions",
          "api_key": "your-venice-api-key-here",
          "models": [
            "claude-opus-4-5",
            "claude-sonnet-4-5",
            "claude-opus-4-6",
            "claude-opus-4-6-fast",
            "claude-sonnet-4-6"
          ],
          "transformer": {
            "use": ["anthropic"]
          }
        }
      ],
      "Router": {
        "default": "venice,claude-opus-4-5",
        "think": "venice,claude-opus-4-5",
        "background": "venice,claude-opus-4-5",
        "longContext": "venice,claude-opus-4-5",
        "longContextThreshold": 100000
      }
    }
    ```

    <Note>
    Wenn Sie `config.json` ändern, während der Router läuft, starten Sie ihn mit `ccr restart` neu, um die Änderungen zu übernehmen.
    </Note>
  </Step>

  <Step title="Starten">
    Router starten, dann Claude Code:

    ```bash
    ccr start
    ccr code
    ```

    Oder die Aktivierungsmethode nutzen:

    ```bash
    eval "$(ccr activate)" && claude
    ```
  </Step>
</Steps>

## Unterstützte Modelle

| Modell | Venice-ID | Am besten für |
|-------|-----------|----------|
| Claude Opus 4.5 | `claude-opus-4-5` | Komplexes Reasoning, große Refactors |
| Claude Sonnet 4.5 | `claude-sonnet-4-5` | Schnelle Iteration, Alltags-Coding |
| Claude Opus 4.6 | `claude-opus-4-6` | Komplexes Reasoning, große Refactors |
| Claude Opus 4.6 Fast | `claude-opus-4-6-fast` | Komplexes Reasoning mit geringerer Latenz |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | Schnelle Iteration, Alltags-Coding |

<Info>
Claude Code ist auf Claude-Modelle optimiert. Andere über Venice verfügbare Modelle (GPT, DeepSeek, Grok usw.) können funktionieren, doch wir können keine gleichwertige Erfahrung garantieren, da Claude Code auf Claude-spezifischen Funktionen wie Extended Thinking aufbaut. Für andere Modelle ziehen Sie die [Standard-API](/de/api-reference/endpoint/chat/completions) von Venice in Betracht.
</Info>

## Router-Funktionen

Der Router bietet über das reine Routing hinaus mehrere nützliche Funktionen:

<AccordionGroup>
  <Accordion title="Modelle im laufenden Betrieb wechseln">
    Verwenden Sie den `/model`-Befehl innerhalb von Claude Code, um ohne Neustart Modelle zu wechseln:
    
    ```
    /model venice,claude-sonnet-4-5
    ```
    
    Nützlich, wenn Sie Opus für komplexe Aufgaben und Sonnet für schnelle Iterationen verwenden möchten.
  </Accordion>

  <Accordion title="Visuelle Konfiguration im UI-Modus">
    Lieber eine GUI? Den webbasierten Config-Editor starten:
    
    ```bash
    ccr ui
    ```
    
    Damit öffnen Sie eine Browser-Oberfläche zum Bearbeiten Ihrer `config.json`, ohne die Datei direkt anfassen zu müssen.
  </Accordion>

  <Accordion title="Router-Szenarien erklärt">
    Der `Router`-Konfigurationsabschnitt steuert, welches Modell verschiedene Aufgabentypen bearbeitet:
    
    | Szenario | Wann es verwendet wird |
    |----------|----------------|
    | `default` | Allgemeine Requests |
    | `think` | Reasoning-lastige Aufgaben (Plan-Modus) |
    | `background` | Hintergrundoperationen |
    | `longContext` | Wenn der Kontext `longContextThreshold` Tokens übersteigt |
    
    Sie können verschiedene Szenarien auf verschiedene Modelle routen. Verwenden Sie z. B. Sonnet für Hintergrundaufgaben, um Kosten zu sparen.
  </Accordion>

  <Accordion title="Debugging mit Logs">
    Wenn etwas nicht funktioniert, prüfen Sie die Logs:
    
    ```bash
    # Server-Logs (HTTP, API-Aufrufe)
    ~/.claude-code-router/logs/ccr-*.log
    
    # Anwendungs-Logs (Routing-Entscheidungen)
    ~/.claude-code-router/claude-code-router.log
    ```
    
    Setzen Sie in Ihrer Konfiguration `"LOG_LEVEL": "debug"` für ausführlichere Ausgabe.
  </Accordion>
</AccordionGroup>

## Caching-Verhalten

Venice's [Prompt-Caching](/de/guides/features/prompt-caching) funktioniert zusammen mit den nativen Cache-Markern von Claude Code. Venice erkennt automatisch, wenn Claude Code `cache_control`-Felder sendet, und passt seine Caching-Strategie entsprechend an.

| Szenario | Cache-TTL | Wer steuert |
|----------|-----------|--------------|
| Standard (empfohlen) | 5 Minuten | Claude Code + Venice |
| Mit `cleancache`-Transformer | 1 Stunde | Nur Venice |

<AccordionGroup>
  <Accordion title="Wann NICHT cleancache (die meisten Nutzer)">
    Die Standardkonfiguration lässt beide Systeme kooperieren:
    
    - Claude Code sendet seine nativen `cache_control`-Marker
    - Venice ergänzt Caching darum mit einer 5-Minuten-TTL
    - Beide Systeme teilen sich das 4-Block-Cache-Limit
    
    Das funktioniert gut für aktive Coding-Sessions mit häufigen Requests.
  </Accordion>

  <Accordion title="Wann cleancache verwenden">
    Fügen Sie `cleancache` dem Transformer hinzu, wenn Sie:
    
    - Die 4-Block-Cache-Limit-Fehler erreichen
    - Seltsames Caching-Verhalten erleben
    - Venice's 1-Stunden-TTL für längere Sessions bevorzugen
    
    ```json
    "transformer": {
      "use": ["anthropic", "cleancache"]
    }
    ```
    
    Damit werden die Cache-Marker von Claude Code entfernt und Venice erhält die volle Kontrolle mit längerer TTL.
  </Accordion>
</AccordionGroup>

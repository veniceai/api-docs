Der [Venice MCP Server](https://github.com/veniceai/venice-mcp-server) ist der offizielle [Model-Context-Protocol](https://modelcontextprotocol.io/)-Server für Venice. Er stellt die gesamte Venice-API (Chat, Bild, Video, Audio, Musik, Embeddings, Web-Augment und Charaktere) als **31 Tools** bereit, die jeder MCP-kompatible Agent aufrufen kann.

## Schnellstart

<Steps>
  <Step title="Venice-API-Schlüssel holen">
    Erzeugen Sie einen auf [venice.ai/settings/api](https://venice.ai/settings/api). Schritt-für-Schritt-Anleitung im [API-Schlüssel-Leitfaden](/de/guides/getting-started/generating-api-key).
  </Step>

  <Step title="Venice zur MCP-Host-Konfiguration hinzufügen">
    Fügen Sie dies in die Konfigurationsdatei Ihres MCP-Hosts ein:

    ```json
    {
      "mcpServers": {
        "venice": {
          "command": "npx",
          "args": ["-y", "@veniceai/mcp-server@0.2.0"],
          "env": { "VENICE_API_KEY": "<your-venice-api-key>" }
        }
      }
    }
    ```

    Übliche Konfigurationspfade:

    | Host | Pfad |
    |---|---|
    | Claude Desktop (macOS) | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Claude Desktop (Windows) | `%APPDATA%\Claude\claude_desktop_config.json` |
    | Cursor | `~/.cursor/mcp.json` |
    | LM Studio | `mcp.json` (aus den MCP-Einstellungen der App) |
  </Step>

  <Step title="MCP-Host neu starten">
    Ihr Agent hat nun Chat, Bild, Video, Musik, TTS, ASR und 25 weitere Venice-Tools zur Verfügung.
  </Step>
</Steps>

<Note>
Die meisten MCP-Hosts geben nur Umgebungsvariablen weiter, die **explizit** im `env`-Block aufgelistet sind. System-Umgebungsvariablen werden nicht vererbt. Wenn Sie trotz gesetztem API-Schlüssel 402-Fehler sehen, prüfen Sie, ob `VENICE_API_KEY` in Ihrer Konfiguration innerhalb von `env` steht.
</Note>

## Was Sie erhalten

**31 Tools** für jede Venice-Modalität, **3 Resources** (`venice://models`, `venice://styles`, `venice://voices`) und **3 Prompt-Templates**.

### Chat & Embeddings

| Tool | Beschreibung |
|---|---|
| `venice_chat` | OpenAI-kompatible Chat-Completion gegen Venice's gesamten LLM-Katalog. |
| `venice_responses` | OpenAI-kompatible Responses-API mit Single- oder Multi-Turn-Tool-Unterstützung. |
| `venice_embeddings` | Embeddings für Texteingaben berechnen. |
| `venice_chat_with_character` | Chat mit einem Venice-Charakter per Slug. |

### Bild

| Tool | Beschreibung |
|---|---|

| `venice_image_multi_edit` | Mehrere Bilder mit einem Prompt gemeinsam bearbeiten. |

## Konfiguration

Der Server wird vollständig über Umgebungsvariablen konfiguriert.

| Umgebungsvariable | Standard | Hinweise |
|---|---|---|
| `VENICE_API_KEY` | _(keiner)_ | Ihr Venice-API-Schlüssel. Die einfachste Einrichtung. |
| `VENICE_DEFAULT_CHAT_MODEL` | `venice-uncensored` | |
| `VENICE_DEFAULT_IMAGE_MODEL` | `flux-2-pro` | |
| `VENICE_DEFAULT_TTS_MODEL` | `tts-kokoro` | |
| `VENICE_DEFAULT_ASR_MODEL` | `openai/whisper-large-v3` | |
| `VENICE_DISABLE_NSFW` | `0` | Auf `1` setzen, um NSFW-Capability-Hinweise aus Tool-Beschreibungen zu entfernen. |
| `VENICE_HTTP_TIMEOUT_MS` | `60000` | |
| `VENICE_SIWX_TOKEN` | _(keiner)_ | x402-Wallet-Mode-Auth-Token. Siehe [x402 unten](#x402-wallet-mode). |

Wenn sowohl `VENICE_API_KEY` als auch `VENICE_SIWX_TOKEN` gesetzt sind, gewinnt der API-Schlüssel.

## x402-Wallet-Modus

Venice unterstützt zusätzlich zur normalen API-Schlüssel-Methode die Authentifizierung mit einem [Sign-In-With-X-Wallet-Token](/de/guides/integrations/x402-venice-api), der durch vorausbezahlte USDC-Credits auf Base oder Solana gedeckt ist. Keine E-Mail, kein Telefon, kein KYC erforderlich: Ihre Wallet ist die einzige Identität.

```json
{
  "mcpServers": {
    "venice": {
      "command": "npx",
      "args": ["-y", "@veniceai/mcp-server@0.2.0"],
      "env": { "VENICE_SIWX_TOKEN": "<base64 Sign-In-With-X payload>" }
    }
  }
}
```

Der MCP-Server gibt `VENICE_SIWX_TOKEN` als `X-Sign-In-With-X`-Header bei jedem Venice-API-Aufruf weiter. Der Server sieht Ihren Private Key nie. Wallet-Signaturen und USDC-Top-Up-Autorisierungen geschehen in Ihrer eigenen Wallet.

| Flow | Was passiert |
|---|---|
| **Einmalige Einrichtung** | Eine Sign-In-With-X-Nachricht in Ihrer Wallet signieren → erzeugt einen SIWX-Token (base64-JSON). |
| **Aufladen** | `POST /api/v1/x402/top-up` gibt 402 + Zahlungsanforderungen zurück. Signieren Sie eine USDC-Zahlung für eine der zurückgegebenen Base- oder Solana-Optionen, senden Sie erneut, und Venice schreibt Ihr Guthaben gut. |
| **Jeder Inference-Aufruf** | MCP-Server sendet `X-Sign-In-With-X: <SIWX>`; Venice bucht von Ihrem Prepaid-Guthaben ab. |

Mindest-Top-Up: **5 $**. Mindestguthaben, um Inference aufzurufen: **0,10 $**. Nach dem Aufladen sind Aufrufe unter 100 ms, da die Abrechnung off-chain über ein schnelles Credit-Konto erfolgt.

<Tip>
Wallets, die mit einem Venice-Konto mit gestaktem DIEM verknüpft sind, verbrauchen statt USDC-Credits vom Staking-Guthaben — daher ist kein Top-Up nötig.
</Tip>

## Selbst hosten (Streamable HTTP)

Für Team- oder Workspace-Deployments betreiben Sie den MCP-Server über HTTP statt über stdio:

```bash
docker run -p 3333:3333 \
  -e VENICE_API_KEY=<your-venice-api-key> \
  -e VENICE_MCP_AUTH_TOKEN=<choose-a-long-random-token> \
  ghcr.io/veniceai/venice-mcp-server:latest
```

Der Server ist nun unter `http://localhost:3333/mcp` erreichbar. HTTP-Clients müssen `Authorization: Bearer <VENICE_MCP_AUTH_TOKEN>` mitsenden.

<Warning>
`/mcp` ist ein credential-gestützter Tool-Execution-Endpoint: Aufrufer können den konfigurierten Venice-API-Schlüssel oder das x402-Guthaben ausgeben. Wenn der HTTP-Modus außerhalb von Loopback bindet, schlägt der Start fehl, sofern `VENICE_MCP_AUTH_TOKEN` nicht gesetzt ist. Für Produktion: Pinnen Sie die npm-Paketversion explizit, statt sich auf `latest` zu verlassen.
</Warning>

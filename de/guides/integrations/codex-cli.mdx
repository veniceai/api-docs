Dieser Leitfaden zeigt, wie Sie OpenAI Codex CLI mit Venice betreiben — über die offiziellen Codex-Konfigurationspfade: `~/.codex/config.toml` (Nutzerebene) oder `.codex/config.toml` (Projektebene).

## Voraussetzungen

- Ein Venice-API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api)
- Codex CLI installiert und auf Ihrem Rechner lauffähig

---

## Einrichtung

<Steps>
  <Step title="Projektkonfigurationspfad anlegen">
    Im Projekt-Root:

    ```bash
    mkdir -p .codex
    ```
  </Step>

  <Step title=".codex/config.toml erstellen">
    Datei anlegen und die folgende Konfiguration einfügen:

    ```toml
    #:schema https://developers.openai.com/codex/config-schema.json

    model = "openai-gpt-54" # ein beliebiges Venice-Modell verwenden
    model_provider = "venice"
    model_reasoning_effort = "high"
    personality = "pragmatic"
    sandbox_mode = "workspace-write"

    [model_providers.venice]
    name = "Venice"
    base_url = "https://api.venice.ai/api/v1/"
    experimental_bearer_token = "YOUR VENICE API KEY"
    wire_api = "responses"
    ```
  </Step>

  <Step title="Die beiden Platzhalter ersetzen">
    Aktualisieren:

    - `model` mit der gewünschten Venice-Modell-ID
    - `experimental_bearer_token` mit Ihrem echten Venice-API-Schlüssel

    Verfügbare Modell-IDs finden Sie im [Textmodell-Katalog](/de/models/text).
  </Step>

  <Step title="Codex CLI wie gewohnt ausführen">
    Starten Sie Codex CLI im selben Projekt. Es lädt `.codex/config.toml` (für vertrauenswürdige Projekte) und leitet Anfragen über Venice.
  </Step>
</Steps>

---

## Offizielle Codex-Konfigurations-Pfade

- **Nutzer-Defaults**: `~/.codex/config.toml`
- **Projekt-Overrides**: `.codex/config.toml` (nur für vertrauenswürdige Projekte geladen)

Wenn Venice-Einstellungen überall gelten sollen, legen Sie dieselbe Konfiguration in `~/.codex/config.toml` ab.

---

## Konfigurations-Vorrang (höchste zuerst)

1. CLI-Flags und `--config`-Overrides
2. Profilwerte (`--profile <name>`)
3. Projekt-Konfig-Schichten (`.codex/config.toml`, näheres Verzeichnis gewinnt)
4. Nutzer-Konfig (`~/.codex/config.toml`)
5. System-Konfig (`/etc/codex/config.toml`, Unix)
6. Eingebaute Standardwerte

---

## Hinweise

- Halten Sie Ihren API-Schlüssel privat und committen Sie niemals echte Schlüssel in Git.
- Codex ignoriert die `.codex/`-Projektkonfiguration, wenn ein Projekt als nicht vertrauenswürdig markiert ist.
- Beim Wechsel des Modells aktualisieren Sie nur das `model`-Feld.
- Die Einstellung `wire_api = "responses"` ist für dieses Provider-Setup erforderlich.

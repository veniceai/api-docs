[Brave Leo](https://brave.com/leo/) ist der KI-Assistent, der im Brave-Browser integriert ist. Leos Bring-Your-Own-Model-Funktion (BYOM) kann sich direkt mit Venices OpenAI-kompatiblem Chat-Completions-Endpoint verbinden, sodass du Venice-Modelle für Chat und seitenbewusste Unterstützung nutzen kannst, ohne eine Erweiterung zu installieren.

## Voraussetzungen

- Brave Browser 1.69 oder neuer unter Windows, macOS oder Linux
- Ein Venice API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api)
- Eine Modell-ID aus dem [Venice-Textmodell-Katalog](/models/text)

<Note>
Braves BYOM-Funktion ist auf dem Desktop verfügbar. Die Modell-ID muss exakt einer Venice-Modell-ID entsprechen.
</Note>

## Einrichtung

<Steps>
  <Step title="Leo-Einstellungen öffnen">
    Öffne `brave://settings/leo-ai`, oder öffne Leo über Braves Seitenleiste, wähle das Einstellungssymbol und navigiere zu **Leo**.
  </Step>

  <Step title="Ein benutzerdefiniertes Modell hinzufügen">
    Scrolle zu **Bring your own model** und wähle **Add new model**.
  </Step>

  <Step title="Venice-Endpoint konfigurieren">
    Gib die folgenden Werte ein:

    | Feld | Wert |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | Eine Modell-ID aus dem [Textmodell-Katalog](/models/text) |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | Dein Venice API-Schlüssel |

    Falls Brave **Context size** anzeigt, setze diesen Wert auf das vom gewählten Modell unterstützte Kontextfenster. Du kannst auch einen optionalen System-Prompt hinzufügen.

    Gib den API-Schlüssel selbst ohne `Bearer`-Präfix ein. Brave fügt den Authorization-Header bei jeder Anfrage hinzu.
  </Step>

  <Step title="Speichern und Venice auswählen">
    Wähle **Add model** und setze anschließend **Venice AI** als Standardmodell für neue Konversationen oder wähle es in Leos Modellauswahl aus.
  </Step>

  <Step title="Verbindung testen">
    Öffne Leo auf einer Webseite und sende einen Prompt wie:

    ```txt
    Summarize this page in five bullet points.
    ```

    Leo sendet die Konversation und relevanten Seiteninhalt direkt an den Venice-Endpoint.
  </Step>
</Steps>

## Venice-Websuche aktivieren

Brave stellt in BYOM keine Venice-spezifischen Request-Body-Parameter zur Verfügung. Um Venices Websuche zu aktivieren, hänge an die Modell-ID im Feld **Model request name** ein [Model-Feature-Suffix](/api-reference/endpoint/chat/model_feature_suffix) an:

```txt
<model-id>:enable_web_search=auto
```

Um die Suche stets zu aktivieren und Quellenangaben mit einzubeziehen, verwende:

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
Für die Websuche fallen zusätzliche nutzungsabhängige Kosten an. Aktuelle Preise findest du unter [API-Preise](/overview/pricing).
</Info>

## Privacy

Wenn du einen entfernten BYOM-Endpoint verwendest, verbindet sich Brave direkt mit diesem Anbieter. Anfragen umgehen Braves Server, sodass Braves Proxy- und serverseitige Retention-Schutzmaßnahmen für diese Anfragen nicht greifen.

Venice speichert oder loggt Prompt- und Antwortinhalte am eigenen Proxy nicht. Das gewählte Modell bestimmt die Privacy-Stufe zur Laufzeit:

- **Private**-Modelle nutzen Zero-Data-Retention-Infrastruktur.
- **Anonymized**-Modelle verbergen identifizierende Informationen vor dem Upstream-Anbieter, aber der Anbieter kann den Prompt verarbeiten.
- **TEE**- und **E2EE**-Modelle ergänzen hardwaregestützte Isolation oder clientseitige Verschlüsselung, wo unterstützt.

Prüfe das Privacy-Label des Modells im [Textmodell-Katalog](/models/text) und lies [Venice Privacy](/overview/privacy), bevor du sensible Informationen sendest.

Brave kann den Leo-Chatverlauf im lokalen Browserspeicher aufbewahren. Du kannst Leo-Daten löschen, einen temporären Chat verwenden oder den lokalen Chatverlauf in Braves Leo-Einstellungen deaktivieren.

## Konfigurationshinweise

- Brave erfordert den vollständigen Endpoint-Pfad: `https://api.venice.ai/api/v1/chat/completions`.
- Modell-IDs und Verfügbarkeit ändern sich mit der Zeit. Kopiere die aktuelle ID aus dem [Textmodell-Katalog](/models/text).
- Die Context size sollte das unterstützte Kontextfenster des gewählten Modells nicht überschreiten.
- Seitenzusammenfassungen können Seiteninhalte als Teil des Prompts an Venice senden.
- Entfernte Venice-Anfragen benötigen eine Internetverbindung und verbrauchen Venice-API-Guthaben oder dein verfügbares DIEM-Kontingent.

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="401- oder Authentifizierungsfehler">
    Stelle sicher, dass der Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api) stammt. Füge den Rohschlüssel ohne `Bearer` ein und entferne führende oder abschließende Leerzeichen.
  </Accordion>
  <Accordion title="Modell nicht gefunden">
    Kopiere die Modell-ID exakt aus dem [Textmodell-Katalog](/models/text). Wenn du ein Feature-Suffix hinzugefügt hast, teste das Modell zunächst ohne das Suffix.
  </Accordion>
  <Accordion title="Netzwerkfehler oder keine gestreamte Antwort">
    Stelle sicher, dass der Server-Endpoint `/api/v1/chat/completions` enthält. Brave Leo erwartet eine OpenAI-kompatible Streaming-Antwort.
  </Accordion>
  <Accordion title="Seitenzusammenfassungen enthalten keinen Seitenkontext">
    Prüfe Leos Berechtigung, Seiteninhalte zu verwenden, und stelle sicher, dass die aktuelle Seite für den Browser zugänglich ist. Interne Brave-Seiten und einige geschützte Seiten können nicht zusammengefasst werden.
  </Accordion>
</AccordionGroup>

[Venice Skills](https://github.com/veniceai/skills) ist die maßgebliche Sammlung von [Agent Skills](https://www.anthropic.com/news/agent-skills) für die Venice-API. Jeder Skill ist ein in sich geschlossener Ordner mit einer `SKILL.md`, die ein LLM-Agent bei Bedarf lädt, um auf einem bestimmten Bereich der API korrekt zu arbeiten.

## Warum Skills?

Ohne Skills muss Ihr Agent die Eigenheiten von Venice mühsam selbst entdecken: `venice_parameters`, Modell-Typ-Enums, 402-Payment-Required-Flows, Video-Queue/Retrieve-Lebenszyklus, Charakter-Slugs und so weiter. Skills bündeln dieses Wissen in fokussierten, bedarfsgesteuerten Dateien, sodass der Agent nur das lädt, was er für die aktuelle Aufgabe braucht.

Jede `SKILL.md` enthält:

- Die abgedeckten Endpoints
- Erforderliche Header, Parameter und Antwortformen
- Ein curl-Beispiel plus ein minimales SDK-Beispiel
- Einen Abschnitt "Stolperfallen" mit den Dingen, über die echte Integrierende stolpern

## Installation

Jeder Skill ist nur ein Ordner mit einer `SKILL.md`, die mit YAML-Frontmatter beginnt:

```yaml
---
name: venice-chat
description: Wann der Agent diesen Skill laden sollte und was darin steht
---
```

Legen Sie den Ordner `skills/` (oder eine beliebige Teilmenge) in den Pfad, den Ihre Runtime beobachtet.

<Tabs>
  <Tab title="Claude Code">
    Projektlokal:

    ```bash
    git clone https://github.com/veniceai/skills.git
    cp -r skills/skills/* .claude/skills/
    ```

    Oder global, für jedes Projekt auf Ihrem Rechner:

    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.claude/skills/venice
    ```
  </Tab>

  <Tab title="Cursor">
    Projektlokal:

    ```bash
    git clone https://github.com/veniceai/skills.git .cursor/skills-venice
    ```

    Oder einzelne Skills kopieren:

    ```bash
    cp -r skills/venice-chat .cursor/skills/
    ```
  </Tab>

  <Tab title="Codex">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.codex/skills/venice
    ```

    Für projektlokale Installationen verwenden Sie stattdessen `.codex/skills/`.
  </Tab>

  <Tab title="OpenCode">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.config/opencode/skills/venice
    ```

    OpenCode liest außerdem `.opencode/skills/`, `.claude/skills/` und `.agents/skills/` aus dem Projekt-Root.
  </Tab>

  <Tab title="Hermes Agent">
    Hermes hat einen eingebauten Installer für Venice-Skills:

    ```bash

    ```

    Oder direkt symlinken:

    ```bash

    ```
  </Tab>

  <Tab title="Cline">

### Pfad-Referenz

| Runtime | Projektlokal | Global |
|---|---|---|
| Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| Codex | `.codex/skills/` | `~/.codex/skills/` (oder `$CODEX_HOME/skills/`) |
| OpenCode | `.opencode/skills/` (auch `.claude/skills/`, `.agents/skills/`) | `~/.config/opencode/skills/` |
| Hermes Agent | `$HERMES_OPTIONAL_SKILLS_DIR` | `~/.hermes/skills/` |
| Cursor | `.cursor/skills/` | `~/.cursor/skills/` |
| Cline | `.clinerules/skills/` | n. v. |
| Andere Runtimes | `.agents/skills/` (Konvention) | `~/.agents/skills/` |

<Tip>
Runtimes, die zusätzliche Frontmatter-Felder definieren (`version`, `platforms`, `metadata.*`, `compatibility`, …), müssen laut Spec unbekannte Felder ignorieren — dieselbe Skill-Datei funktioniert daher überall ohne Forks.
</Tip>

### Als Git-Submodul

Wenn Sie in Ihrem eigenen Repo gepinnte Versionen wünschen:

```bash
git submodule add https://github.com/veniceai/skills.git vendor/venice-skills
```

Symlinken oder kopieren Sie dann die gewünschten Teilmengen in den Skill-Pfad Ihres Agenten.

## Wie Agenten sie laden

Der Agent findet jede `SKILL.md` über das Frontmatter `name` und `description`. Fragt die Nutzerin oder der Nutzer nach etwas, das zum Zweck eines Skills passt, lädt der Agent diese eine Datei in den Kontext (nicht den gesamten Katalog), sodass der Prompt klein und die Antwort präzise bleibt.

Ein Agent, der etwa Musik generieren soll, lädt `venice-audio-music` und weiß sofort:

- Dass Musik durch den queue/retrieve/complete-Lebenszyklus läuft, nicht über einen synchronen Endpoint
- Welche Modelle verfügbar sind und wie sie pro Minute abgerechnet werden
- Wie zunächst `/audio/quote` für eine Kostenschätzung aufgerufen wird
- Wie das Polling-Backoff aussehen sollte

Ohne den Skill würde der Agent womöglich versuchen, `/audio/speech` für Musik aufzurufen, und eine unbrauchbare Antwort erhalten.

## Einen neuen Skill verfassen

1. Kopieren Sie `template/` nach `skills/<your-skill>/`.
2. Füllen Sie Frontmatter und Body aus. Halten Sie `description` konkret, da der Agent damit entscheidet, wann der Skill geladen wird.
3. Verlinken Sie am Ende verwandte Skills zur Cross-Navigation.
4. Öffnen Sie einen PR gegen [`veniceai/skills`](https://github.com/veniceai/skills).

Siehe `CONTRIBUTING.md` des Repositories für Stilkonventionen (kurzer erster Absatz, explizite Endpoint-Tabellen, curl + ein SDK-Beispiel, "Stolperfallen"-Abschnitt, ≤ 500 Zeilen).

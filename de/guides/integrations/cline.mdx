[Cline](https://cline.bot/) ist ein KI-Coding-Assistent für VS Code, der Dateien lesen und bearbeiten, Befehle ausführen und mehrstufige Entwicklungsaufgaben durcharbeiten kann. Venice funktioniert über die OpenAI-kompatiblen Provider-Einstellungen von Cline.

## Voraussetzungen

- Ein Venice API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api)
- [Visual Studio Code](https://code.visualstudio.com/) installiert
- Die [Cline-Erweiterung](https://marketplace.visualstudio.com/items?itemName=saoudrizwan.claude-dev) in VS Code installiert

---

## Einrichtung

<Steps>
  <Step title="Cline-Einstellungen öffnen">
    Wähle das Cline-Symbol in der VS-Code-Aktivitätsleiste und anschließend das Zahnrad-Symbol, um Clines Einstellungen zu öffnen.
  </Step>

  <Step title="API-Provider auswählen">
    Setze **API Provider** auf **OpenAI Compatible**.
  </Step>

  <Step title="Venice konfigurieren">
    Gib die folgenden Werte ein:

    | Einstellung | Wert |
    | --- | --- |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API Key | Dein Venice API-Schlüssel |
    | Model ID | Eine Modell-ID aus dem [Textmodell-Katalog](/models/text) |

    Verwende zum Beispiel `zai-org-glm-5-1` als Model ID.

    Gib den API-Schlüssel selbst ohne `Bearer`-Präfix ein. Behalte die Base URL bei `/api/v1`; Cline hängt den Pfad `/chat/completions` selbst an.
  </Step>

  <Step title="Modellfähigkeiten konfigurieren">
    Klappe **Model Configuration** auf und setze das Kontextfenster sowie die maximalen Output-Tokens passend zum gewählten Modell. Aktiviere die Fähigkeiten, die das Modell unterstützt, einschließlich Bildeingabe oder Computer Use, sofern verfügbar.

    Prüfe das gewählte Modell im [Textmodell-Katalog](/models/text), bevor du optionale Fähigkeiten aktivierst.
  </Step>

  <Step title="Speichern und testen">
    Speichere die Provider-Einstellungen, öffne ein Projekt und gib Cline eine kleine Aufgabe wie:

    ```txt
    Review the current file and suggest one safe improvement.
    ```

    Prüfe jede Datei- oder Befehlsberechtigung, bevor du sie bestätigst.
  </Step>
</Steps>

## Venice-API verifizieren

Falls sich Cline nicht verbinden kann, bestätige den API-Schlüssel und die verfügbaren Modell-IDs unabhängig:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

Kopiere die `id` eines Modells exakt in das Feld **Model ID** von Cline.

---

## Konfigurationshinweise

- Verwende `https://api.venice.ai/api/v1` als Base URL, nicht den vollständigen `/chat/completions`-Endpoint.
- Modell-IDs können sich mit der Zeit ändern. Nutze den [Textmodell-Katalog](/models/text) oder `GET /models`, anstatt dich auf ein älteres Beispiel zu verlassen.
- Clines Einstellungen für Kontextfenster und Output-Tokens beeinflussen, wie Cline Anfragen budgetiert. Passe sie an das gewählte Modell an, um die besten Ergebnisse zu erzielen.
- Halte deinen Venice API-Schlüssel geheim. Füge ihn nicht in Quelldateien ein und committe ihn nicht in git.

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="401- oder Authentifizierungsfehler">
    Stelle sicher, dass der API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api) stammt. Füge den Rohschlüssel ohne `Bearer` und ohne zusätzliche Leerzeichen in Cline ein.
  </Accordion>
  <Accordion title="Modell nicht gefunden">
    Stelle sicher, dass die Model ID im [Textmodell-Katalog](/models/text) oder in der Antwort von `GET /models` erscheint, und kopiere sie dann exakt.
  </Accordion>
  <Accordion title="Verbindungs- oder Endpoint-Fehler">
    Stelle sicher, dass **API Provider** auf **OpenAI Compatible** gesetzt ist und die Base URL exakt `https://api.venice.ai/api/v1` lautet.
  </Accordion>
  <Accordion title="Cline kann keine Tools verwenden oder Dateien bearbeiten">
    Öffne **Model Configuration**, prüfe, dass das gewählte Modell die erforderlichen Fähigkeiten unterstützt, und aktiviere Computer Use, falls Cline diese Option für das Modell anbietet.
  </Accordion>
</AccordionGroup>

[NanoClaw](https://github.com/qwibitai/nanoclaw) ist ein schlanker, selbstgehosteter KI-Assistent, der auf WhatsApp und Telegram läuft. Dieser Fork ergänzt Venice-AI-Unterstützung, sodass alles privat — ohne Anthropic-Abo — läuft.

## Warum Venice AI?

[Venice](https://venice.ai) ist eine datenschutzorientierte KI-Plattform. Sie [speichern oder protokollieren keine Prompts oder Antworten](https://venice.ai/privacy) auf ihren Servern — Ihre Gespräche existieren nur auf Ihrem Gerät. Anfragen werden Ende-zu-Ende-verschlüsselt durch ihren Proxy an dezentrale GPU-Anbieter geleitet, mit Zero Data Retention. Damit bleiben Ihre KI-Assistenten-Gespräche privat — selbst gegenüber Venice.

Venice bietet anonymisierten Zugang zu Frontier-Modellen (Claude Opus, Claude Sonnet) und vollständig privaten Zugang zu Open-Source-Modellen (GLM, Qwen) über eine einzige API — jederzeit umschaltbar.

| | **Venice AI** | **Klassische KI-Anbieter** |
|---|---|---|
| **Datenspeicherung** | Keine — keine Logs | Ja |
| **Prompt-Datenschutz** | Verschlüsselt, nie gespeichert | Auf Anbieter-Servern gespeichert |
| **Open-Source-Modelle** | Ja (GLM, Qwen und weitere) | Nein |
| **Frontier-Modelle** | Claude, GPT und mehr — anonym | Nur über direkte Abos |
| **Preise** | Pay-per-Token, ohne Abo. Oder [DIEM](https://venice.ai/lp/diem) staken für täglich erneuerte Credits | 20–200 $/Monat Abos oder Pay-per-Token-API |
| **Unbeschränkte Inference** | Ja (Open-Source-Modelle) | Nein |

## Warum NanoClaw?

NanoClaw ist eine schlanke, minimalistische Alternative zu größeren Plattformen wie OpenClaw. Es ist für eine Person und einen Bot gedacht.

| | **NanoClaw (Venice)** | **OpenClaw** |
|---|---|---|
| **Codebasis** | ~2.000 Zeilen, eine Handvoll Dateien | ~500.000 Zeilen, 53 Konfigurationsdateien |
| **Abhängigkeiten** | ~15 Pakete | 70+ Pakete |
| **Sicherheitsmodell** | OS-Ebene-Docker-Container-Isolation | Anwendungs-Allowlists und Pairing-Codes |
| **Isolation pro Gruppe** | Jede Gruppe erhält eigenen Container, Dateisystem und Speicher | Geteilter Prozess, geteilter Speicher |
| **Setup** | Ein Assistent (`/setup`), ~10 Minuten | Manuelle Mehrschritt-Konfiguration |
| **KI-Anbieter** | Venice AI (privat, kein Abo) | Anthropic (API-Schlüssel oder Abo erforderlich) |
| **Anpassung** | Code direkt bearbeiten — klein genug zum Lesen | Konfigurationsdateien und Plugins |
| **Zielgruppe** | Eine Person, ein Bot | Mehr-Nutzer-Plattform |

## Was Sie erhalten

- Persönlicher KI-Assistent auf **Telegram** und/oder **WhatsApp**
- Betrieben mit **Venice AI** — kein Anthropic-Account nötig
- Bot läuft in einem **isolierten Docker-Container** (Sandbox, kein Zugriff auf Ihr System)
- **Modellumschaltung** — sagen Sie dem Bot "switch to zai-org-glm-5" oder "use opus" jederzeit
- **Geplante Aufgaben** — Erinnerungen, wiederkehrende Aufgaben
- **Websuche und -browsen** eingebaut
- **Markdown-Formatierung** in Telegram-Nachrichten

**Für Telegram** (empfohlen für den Erstkontakt):
1. Telegram öffnen und nach **@BotFather** suchen
2. `/newbot` senden und den Anweisungen folgen
3. Den Token speichern, den BotFather Ihnen gibt (sieht aus wie `123456789:ABCdef...`)

<Warning>
**Für WhatsApp — verwenden Sie eine virtuelle Nummer, NICHT Ihre persönliche:**

NanoClaw verbindet sich als verknüpftes Gerät mit Ihrer WhatsApp-Nummer. Das bedeutet: **Der Agent sieht jede ein- und ausgehende Nachricht** — alle persönlichen Gespräche, Gruppenchats, Fotos, alles. Ihr Telefon funktioniert weiterhin normal, doch der Bot hat vollen Einblick in Ihr gesamtes WhatsApp-Konto.

**Verwenden Sie stattdessen eine virtuelle Telefonnummer.** Diese Apps geben Ihnen eine zweite Nummer, die Sie vollständig dem Bot widmen können:

| App | Preis | Hinweise |
|-----|-------|-------|
| [Hushed](https://hushed.com) | ~5 $/Monat | Zuverlässig, funktioniert gut für die WhatsApp-Verifizierung |
| [Burner](https://www.burnerapp.com) | ~5 $/Monat | Ähnlich wie Hushed, Einweg-Nummern |
| [Google Voice](https://voice.google.com) | Kostenlos | Nur in den USA, eventuell nicht für jede WhatsApp-Verifizierung geeignet |

**So richten Sie es ein:**
1. Eine virtuelle Nummer von einer der obigen Apps besorgen
2. WhatsApp auf einem zweiten Gerät (altes Telefon, Tablet oder Emulator) mit dieser virtuellen Nummer installieren
3. Während des NanoClaw-Setups den QR-Code mit diesem zweiten Gerät scannen — nicht mit Ihrem persönlichen Telefon
</Warning>

## Einrichtung

Die Einrichtung dauert etwa 10 Minuten. Sie benötigen nur **ein Terminal-Fenster**.

<Steps>
  <Step title="Klonen und installieren">
    Terminal öffnen und ausführen:

    ```bash
    git clone https://github.com/lorenzovenice/nanoclaw-venice.git
    cd nanoclaw-venice
    npm install
    ```

    Warten, bis `npm install` fehlerfrei abschließt.
  </Step>

  <Step title="Claude Code mit Venice starten">
    Ersetzen Sie `your-key` durch Ihren Venice-API-Schlüssel und führen Sie aus:

    ```bash
    VENICE_API_KEY=your-key npm run venice
    ```

    Damit wird der Venice-Proxy gestartet und Claude Code in einem einzigen Befehl darüber gestartet.

    <Note>
    Claude Code verwendet standardmäßig **GLM 5** (`zai-org-glm-5`), um die Setup-Kosten niedrig zu halten. Nach dem Setup wechseln Sie mit `/model` innerhalb von Claude Code zu `claude-sonnet-4-6` oder `claude-opus-4-6` für beste Performance.
    </Note>

    Wenn gefragt "Do you want to use this API key?" — wählen Sie **Yes**.
  </Step>

  <Step title="Setup-Assistent ausführen">
    Im Claude-Code-Terminal eingeben:

    ```
    /setup
    ```

    Der Assistent führt Sie durch:

    1. **Bootstrap** — prüft Node.js und Abhängigkeiten
    2. **Venice-API-Schlüssel** — validiert und speichert Ihren Schlüssel
    3. **Kanal-Auswahl** — WhatsApp, Telegram oder beides wählen
    4. **Container-Build** — baut den Docker-Container (dauert beim ersten Mal einige Minuten)
    5. **WhatsApp-Authentifizierung** — QR-Code mit Ihrem Telefon scannen (falls zutreffend)
    6. **Telegram-Einrichtung** — eine Nachricht an Ihren Bot senden, damit er Ihren Chat erkennt
    7. **Trigger-Wort** — Präfix, das den Bot aktiviert (Standard: `@Andy`)
    8. **Mount-Verzeichnisse** — vorerst "No" wählen (Datei-Zugriff können Sie später ergänzen)
    9. **Dienste starten** — NanoClaw und der Venice-Proxy starten beide als Hintergrunddienste

    Der Setup-Assistent installiert zwei Hintergrunddienste:
    - **NanoClaw** — der eigentliche Bot
    - **Venice-Proxy** — ein kleiner lokaler Server (localhost:4001), der zwischen Claude Code und Venice AI übersetzt

    Beide starten beim Booten automatisch und starten sich nach Crashes neu.

    <Note>
    Sollte der Assistent zwischen Schritten stehenbleiben, geben Sie "continue" oder "next step" ein, um ihn weiterzuschubsen.
    </Note>
  </Step>

  <Step title="Loslegen">
    Sobald das Setup abgeschlossen ist, öffnen Sie Ihren Chat (Telegram oder WhatsApp) und senden:

    ```
    @Andy hello, are you there?
    ```

    Der Bot sollte innerhalb von Sekunden antworten. In Ihrem Hauptkanal können Sie normal tippen, ohne das Präfix `@Andy`.

    **Sie können das Terminal-Fenster nun schließen.** Alles läuft als Hintergrunddienste und startet automatisch, wenn Ihr Rechner bootet.
  </Step>
</Steps>

## So funktioniert es

Es gibt zwei Schichten in NanoClaw:

| Schicht | Was sie tut |
|-------|-------------|
| **Claude-Code-CLI** | Admin-Tool für Setup, Debugging und Anpassung |
| **Der Bot** | KI in Ihrem Chat, die in einem isolierten Docker-Container läuft |

Claude Code jederzeit öffnen:

```bash
cd nanoclaw-venice
ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
```

Verwenden Sie es, um `/setup`, `/debug`, `/customize` auszuführen oder das Verhalten des Bots zu ändern.

## Fehlersuche

<AccordionGroup>
  <Accordion title="Der Proxy läuft nicht">
    Der Venice-Proxy läuft als Hintergrunddienst und startet sich selbst neu. Wenn er nicht funktioniert:

    **macOS:**
    ```bash
    # Prüfen, ob er läuft
    launchctl list | grep venice-proxy

    # Neu starten
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Logs prüfen
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```

    **Linux:**
    ```bash
    # Prüfen, ob er läuft
    systemctl --user status nanoclaw-venice-proxy

    # Neu starten
    systemctl --user restart nanoclaw-venice-proxy

    # Logs prüfen
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```
  </Accordion>

  <Accordion title="Claude Code zeigt 403-Fehler oder 'Please run /login'">
    Das bedeutet, dass Claude Code keine Verbindung zum Venice-Proxy herstellen kann.

    1. **Prüfen, dass der Proxy läuft.** Siehe den vorherigen Schritt.
    2. **Sicherstellen, im richtigen Ordner zu sein.** Immer zuerst `cd nanoclaw-venice`.
    3. **Neu beginnen:** Alle Terminals schließen und ausführen:
       ```bash
       cd nanoclaw-venice
       ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
       ```
  </Accordion>

  <Accordion title="Modell-Fehler ('model does not exist')">
    Proxy und Bot neu starten:

    **macOS:**
    ```bash
    # Proxy neu starten
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Bot neu starten
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw
    ```

    **Linux:**
    ```bash
    # Proxy neu starten
    systemctl --user restart nanoclaw-venice-proxy

    # Bot neu starten
    systemctl --user restart nanoclaw
    ```

    Verfügbare Modelle finden Sie im [Modellkatalog](/de/models/text).
  </Accordion>

  <Accordion title="Bot antwortet nicht auf Nachrichten">
    Arbeiten Sie diese Schritte der Reihe nach ab:

    1. **Trigger-Wort prüfen.** Stellen Sie sicher, dass Sie das richtige Präfix verwenden (z. B. `@Andy hello`).
    2. **Prüfen, dass Docker läuft.** `docker info` ausführen — gibt es einen Fehler, Docker Desktop öffnen.
    3. **Prüfen, dass der Proxy läuft.** Siehe "Der Proxy läuft nicht" oben.
    4. **Logs prüfen:** `tail -f logs/nanoclaw.log` im Projektordner.
    5. **Container-Logs prüfen.** Den Ordner `nanoclaw-venice/groups/main/logs/` öffnen. Die jüngste Datei öffnen, die mit `container-` beginnt.
    6. **Alles neu starten:** Sowohl Proxy als auch Bot neu starten (siehe oben).
  </Accordion>

  <Accordion title="Container-Build schlägt beim Setup fehl">
    Stellen Sie sicher, dass Docker Desktop geöffnet und einsatzbereit ist. Warten Sie 10 Sekunden, bis Docker vollständig gestartet ist, und tippen Sie dann im Assistenten "continue", um es erneut zu versuchen.
  </Accordion>

  <Accordion title="WhatsApp getrennt">
    Ihre WhatsApp-Session kann ablaufen. Zum erneuten Verbinden:

    ```bash
    cd nanoclaw-venice
    npm run auth
    ```

    Scannen Sie den QR-Code mit WhatsApp (Einstellungen → Verknüpfte Geräte → Gerät verknüpfen), und starten Sie dann den Bot neu:
    - macOS: `launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
    - Linux: `systemctl --user restart nanoclaw`
  </Accordion>
</AccordionGroup>

## Fortgeschritten

<AccordionGroup>
  <Accordion title="Dem Bot Zugriff auf Dateien auf Ihrem Rechner geben">
    Standardmäßig ist der Bot vollständig von Ihrem Rechner abgeschottet — er sieht nur sein eigenes Memory und den Gesprächsverlauf.

    - **Während des Setups:** Wenn nach dem Verzeichniszugriff gefragt wird, wählen Sie "Yes"
    - **Nach dem Setup:** `/customize` in Claude Code ausführen
  </Accordion>

  <Accordion title="Bot manuell starten/stoppen">
    NanoClaw betreibt zwei Hintergrunddienste, die beim Boot automatisch starten.

    **macOS:**
    | Aktion | Befehl |
    |--------|---------|
    | Bot starten | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Bot stoppen | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Bot neu starten | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw` |
    | Proxy starten | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Proxy stoppen | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Proxy neu starten | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy` |

    **Linux:**
    | Aktion | Befehl |
    |--------|---------|
    | Bot starten | `systemctl --user start nanoclaw` |
    | Bot stoppen | `systemctl --user stop nanoclaw` |
    | Bot neu starten | `systemctl --user restart nanoclaw` |
    | Proxy starten | `systemctl --user start nanoclaw-venice-proxy` |
    | Proxy stoppen | `systemctl --user stop nanoclaw-venice-proxy` |
    | Proxy neu starten | `systemctl --user restart nanoclaw-venice-proxy` |
  </Accordion>

  <Accordion title="Claude Code über Venice nutzen (ohne Bot)">
    Wenn Sie nur Claude Code mit Venice möchten und WhatsApp/Telegram nicht brauchen, muss der Proxy-Dienst laufen. Wenn Sie bereits `/setup` ausgeführt haben, läuft er bereits als Hintergrunddienst.

    ```bash
    cd nanoclaw-venice
    ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
    ```

    **Tipp:** Fügen Sie dies in Ihre `~/.zshrc` (oder `~/.bashrc`) ein, um in jedem Terminal schnell auf Venice umzuschalten:

    ```bash
    alias venice='export ANTHROPIC_BASE_URL=http://localhost:4001 && export ANTHROPIC_API_KEY=venice-proxy && echo "Using Venice API"'
    alias anthropic='unset ANTHROPIC_BASE_URL && unset ANTHROPIC_API_KEY && echo "Using Anthropic API"'
    ```

    Tippen Sie dann in einem Terminal `venice`, bevor Sie `claude` ausführen, um Venice zu verwenden, oder `anthropic`, um zurückzuwechseln.
  </Accordion>

  <Accordion title="Mehrere Bots betreiben">
    Sie können mehrere NanoClaw-Bots auf demselben Rechner laufen lassen (z. B. einen für privat und einen für ein Team). Klonen Sie das Repo einfach in einen anderen Ordner und führen Sie das Setup erneut aus. Hinweis: Sie teilen sich dasselbe Docker-Image, ein Rebuild des einen wirkt sich also auf alle aus.
  </Accordion>

  <Accordion title="Entwickler-Befehle">
    Für Personen, die den NanoClaw-Code anpassen möchten:

    ```bash
    npm run dev          # Proxy + NanoClaw mit Hot Reload starten
    npm run proxy        # Nur den Venice-Proxy starten
    npm run build        # TypeScript kompilieren
    npm test             # Tests ausführen
    ./container/build.sh # Agent-Container neu bauen
    ```
  </Accordion>
</AccordionGroup>

## Architektur

```
Sie (WhatsApp/Telegram)
        ↓
   NanoClaw (Node.js)
        ↓
   Docker-Container (isolierte Sandbox)
        ↓
   Venice-Proxy (localhost:4001)
        ↓
   api.venice.ai (private Inference)
```

| Datei | Zweck |
|------|---------|
| `proxy/venice-proxy.ts` | Übersetzt Anthropic-Format in OpenAI-Format für Venice |
| `src/index.ts` | Haupt-Orchestrator — Nachrichtenschleife, Agenten-Aufruf |
| `src/channels/whatsapp.ts` | WhatsApp-Verbindung via baileys |
| `src/channels/telegram.ts` | Telegram-Bot via grammy |
| `src/container-runner.ts` | Startet isolierte Agenten-Container |

## FAQ

<AccordionGroup>
  <Accordion title="Warum brauche ich einen Proxy?">
    Das Claude-Agent-SDK spricht Anthropic's Message-Format. Venice spricht OpenAI's Format. Der Proxy übersetzt dazwischen, sodass alles ohne SDK-Änderungen funktioniert.
  </Accordion>

  <Accordion title="Kann ich Open-Source-Modelle nutzen?">
    Ja. Venice hostet viele Modelle. Sagen Sie dem Bot "switch to zai-org-glm-5" oder eine beliebige Venice-Modell-ID. Siehe den [Modellkatalog](/de/models/text).
  </Accordion>

  <Accordion title="Ist das sicher?">
    Agenten laufen in Docker-Containern mit echter OS-Ebene-Isolation. Der Venice-API-Schlüssel wird per stdin übergeben, niemals im Container auf die Platte geschrieben. Jede Gruppe erhält ihre eigene isolierte Umgebung.
  </Accordion>

  <Accordion title="Brauche ich ein Anthropic-Abo?">
    Nein. Alles läuft über Venice AI. Sie brauchen nur einen Venice-API-Schlüssel.
  </Accordion>

  <Accordion title="Kann ich das auf einem Server betreiben?">
    Ja. Es läuft auf jeder Linux-Maschine mit Docker. Verwenden Sie den systemd-Service für Autostart beim Booten.
  </Accordion>
</AccordionGroup>

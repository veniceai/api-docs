[OpenCode](https://opencode.ai/) ist ein KI-Coding-Agent für das Terminal. Venice funktioniert über die Custom-Provider-Konfiguration von OpenCode unter Verwendung des OpenAI-kompatiblen Adapters und einer eigenen Base-URL.

## Voraussetzungen

- Ein Venice-API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api)
- OpenCode installiert und auf Ihrem Rechner lauffähig

---

## Einrichtung

<Steps>
  <Step title="Venice-API-Schlüssel setzen">
    Exportieren Sie Ihren Venice-API-Schlüssel in der Shell, in der Sie OpenCode ausführen:

    ```bash
    export VENICE_API_KEY="your-api-key"
    ```
  </Step>

  <Step title="OpenCode-Konfigurationsdatei erstellen">
    Für eine projektspezifische Einrichtung erstellen Sie `opencode.json` im Projekt-Root.

    Für eine globale Einrichtung verwenden Sie stattdessen `~/.config/opencode/opencode.json`.
  </Step>

  <Step title="Venice-Provider hinzufügen">
    Fügen Sie diese Konfiguration in `opencode.json` ein:

    ```json
    {
      "$schema": "https://opencode.ai/config.json",
      "model": "venice/zai-org-glm-5-1",
      "small_model": "venice/zai-org-glm-5-1",
      "provider": {
        "venice": {
          "npm": "@ai-sdk/openai-compatible",
          "name": "Venice AI",
          "options": {
            "baseURL": "https://api.venice.ai/api/v1",
            "apiKey": "{env:VENICE_API_KEY}"
          },
          "models": {
            "zai-org-glm-5-1": {
              "name": "GLM 5.1"
            },
            "kimi-k2-5": {
              "name": "Kimi K2.5"
            },
            "minimax-m25": {
              "name": "MiniMax M2.5"
            }
          }
        }
      }
    }
    ```
  </Step>

  <Step title="OpenCode ausführen">
    Starten Sie OpenCode aus demselben Projektverzeichnis:

    ```bash
    opencode
    ```

    OpenCode lädt `opencode.json`, verwendet `VENICE_API_KEY` und leitet das ausgewählte Venice-Modell über `https://api.venice.ai/api/v1`.
  </Step>
</Steps>

---

## Einrichtung überprüfen

Führen Sie in OpenCode aus:

```txt
/models
```

Sie sollten die Venice-Modelle aus Ihrer Konfiguration sehen. Wählen Sie eines aus und senden Sie einen kleinen Test-Prompt.

Wenn Sie wie oben `"model": "venice/zai-org-glm-5-1"` gesetzt haben, sollte OpenCode dieses Venice-Modell standardmäßig verwenden. Sie können den Standard ändern, indem Sie `zai-org-glm-5-1` durch eine beliebige Modell-ID aus dem [Textmodell-Katalog](/de/models/text) ersetzen und in die `models`-Map aufnehmen.

---

## Hinweise zur Konfiguration

- Lassen Sie `baseURL` auf `https://api.venice.ai/api/v1` gesetzt. Der OpenAI-kompatible Adapter von OpenCode nutzt diese Base-URL für Chat Completions.
- Lassen Sie `npm` auf `@ai-sdk/openai-compatible` gesetzt für den OpenAI-kompatiblen Chat-Endpoint von Venice.
- Die Provider-ID ist `venice`, daher verwenden Modell-Referenzen das Format `venice/<model-id>`.
- Wenn Sie den `/connect`-Flow von OpenCode bevorzugen, wählen Sie **Other**, nutzen `venice` als Provider-ID, fügen Ihren Venice-API-Schlüssel ein und entfernen `options.apiKey` aus der Konfiguration.

<Info>
OpenCode unterstützt bereits eigene `baseURL`-Werte für Provider und OpenAI-kompatible Provider über `@ai-sdk/openai-compatible`. Es sind keine Core-Provider-Änderungen in OpenCode für Venice erforderlich.
</Info>

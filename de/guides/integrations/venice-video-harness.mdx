Das [Venice Video Harness](https://github.com/jordanurbs/venice-video-harness) ist ein agent-first, Venice-optimiertes Community-Toolkit für die **konsistenzorientierte Videoproduktion in jeder Länge**. Es verwandelt einen IDE-Agenten (Claude Code, Cursor, Codex usw.) in einen Operator eines wiederverwendbaren Venice-Produktionssystems mit über 50 Venice-Video-, -Bild-, -Audio- und -Musikmodellen.

## Worum es geht

Die meisten Venice-Integrationen sind schlanke Wrapper um API-Aufrufe. Das Venice Video Harness ist die **höhere Schicht** zwischen Ihrem Agenten und der Venice-API:

- **Orchestrierungs-Regeln** in `CLAUDE.md`
- **Wiederverwendbare Playbooks** in `.claude/commands/` (19 Workflow-Befehle)
- **Spezialisierte Agenten** in `.claude/agents/` (art-director, prompt-engineer, cut-qa und mehr)
- **Venice-Produktions-Skills** in `.claude/skills/` (kompatibel mit dem [Agent-Skills](/de/guides/integrations/venice-skills)-Format)
- **TypeScript-Ausführungs-Schicht** in `src/`
- **Umfassende Modell-Registry** mit über 50 Venice-Video-, -Bild-, -Audio- und -Musikmodellen

Gebaut für Kreative, die produzieren:

- Charakter-konsistente Videoprojekte (jedes Genre, jede Länge)
- Visuell-stil-gelockte Serien oder Kampagnen
- Storyboard-zu-Video-Workflows
- Kurze und lange narrative Inhalte
- Gebrandete cinematische Sequenzen, Trailer und Teaser
- Social-Series mit wiederkehrenden Charakteren

## Was Venice-optimiert ist

- **Bild-Prompts, abgestimmt auf Venice-Bildmodelle** wie `seedream-v5-lite`, `nano-banana-pro`, `flux-2-pro/max` und mehr
- **Zwei-Pass-Panel-Generierung** mit Venice-Multi-Edit-Verfeinerung zur Charakter-Korrektur
- **Modell-Routing-Logik** für Action-, Atmosphäre- und Charakter-Konsistenz-Stufen
- **Referenzbewusste Videogenerierung**, die `elements`, `reference_image_urls` und `scene_image_urls` pro Modell korrekt verwendet
- **Umgebungsbewusste Prompt-Anpassung** für Tag- vs. Nachtszenen
- **Venice-native Audio-Pfade** für TTS (Kokoro, Qwen3, ElevenLabs), SFX und Musik
- **Kostenschätzung** vor der Generierung über `/video/quote` und `/audio/quote`
- **Modellbewusster Parameter-Aufbau**, der Parameter automatisch überspringt, die das Zielmodell nicht unterstützt

## Modell-Routing-Standards

Die Standardwerte des Harnesses sind dezidiert, denn Konsistenz ist das Ziel. Das aktuelle Routing (April 2026):

**Seedance 2.0 R2V als Standard. Kling O3 R2V als Fallback für Szenen mit 3+ Charakteren. Seedance 2.0 i2v für Establishing-Shots.**

| Rolle | Standardmodell | Wann verwendet |
|---|---|---|
| Charakter-Shots (1–2 Charaktere) | `seedance-2-0-reference-to-video` | Standard-R2V mit flachen `reference_image_urls`, `@Image`-Tags, bis zu 15s, native Stereo-Audio |
| Charakter-Shots (3+ Charaktere) | `kling-o3-standard-reference-to-video` | Automatischer Fallback mit strukturierten `elements` für Multi-Charakter-Identität |
| Establishing / Mood / Action | `seedance-2-0-image-to-video` | Keine Charaktere; epische cineastische Qualität, bis zu 15s |

Diese sind pro Projekt über `series.json → videoDefaults` überschreibbar. Um eine Nicht-Seedance-Familie zu wählen (z. B. Accounts ohne Seedance-Zugriff), setzen Sie `videoDefaults` auf `kling-o3-standard-reference-to-video` und `veo3.1-fast-image-to-video`.

<Note>
**Seedance-Gesichts-Regel:** Seedance 2.0 blockiert gesichtshaltige Eingabebilder, die nicht von `seedream-v5-lite` oder `seedream-v5-lite-edit` erzeugt wurden. Das Harness handhabt dies automatisch, indem es charakterhaltige Bildaufgaben über Seedream routet und vor jedem Seedance-Aufruf einen Pre-Flight-Gate durchläuft.
</Note>

## Produktions-Pipelines

### Generierungs-Pipeline

End-to-End narratives Video (Skript → Storyboard → Video → Audio → Assembly):

```bash
npm run dev -- produce-episode -p output/my-series -e 1
```

Die Referenzimplementierung in `src/mini-drama/` deckt ab:

- Serien-/Charakter-/Episoden-Management
- LLM-gestütztes Skript-Workshopping
- Zwei-Pass-Storyboard-Generierung (generieren + Multi-Edit-Refine)
- Vision-basierte Panel-QA
- Videogenerierung mit Frame-Verkettung
- Geschichtete Audio-Postproduktion
- Untertitel-Burn-in und finale Assembly

### Editing-Pipeline

Bereits vorhandene Medien schneiden (Venice-generierte Shots oder echtes Rohmaterial). **Text-first**: Das LLM liest ein kompaktes `takes_packed.md` (~12 KB pro 40 Min. Audio), statt Frame-Dumps des Videos.

Die fünf Schritte:

<Steps>
  <Step title="Transkribieren">
    Lokales whisper.cpp erzeugt pro Quelle `*.words.json` + `takes_packed.md`.
  </Step>
  <Step title="Pack lesen">
    Das LLM bildet eine Schnitt-Strategie allein aus dem Text.
  </Step>
  <Step title="Bestätigen">
    Schlägt die Strategie vor und wartet auf "yes / revise / cancel".
  </Step>
  <Step title="EDL rendern">
    JSON-Cut-Liste → ffmpeg-Concat mit 30 ms Audio-Fades. Archive-first, sodass Originale nie überschrieben werden.
  </Step>
  <Step title="Self-Eval">
    Der `cut-qa`-Agent führt an jeder Schnittgrenze 6 programmatische Prüfungen durch; maximal 3 Fix-Iterationen.
  </Step>
</Steps>

Die `cut-qa`-Prüfungen erkennen Seitenverhältnis-Regressionen, Frame-Hash-Sprünge mitten im Wort, VO-Verstümmelungen, Lichtbrüche, Audio-Peaks über -6 dBFS und Untertitel-Überlappung mit Text im Bild.

<Tip>
Die Editing-Pipeline ist inspiriert von [browser-use/video-use](https://github.com/browser-use/video-use). Ihre Kern-Erkenntnis — *"das LLM schaut das Video nie an, es liest es"* — ist der Grund, warum agentengesteuertes Editing ohne Frame-Dump-Token-Flut funktioniert.
</Tip>

## NLE-Round-Trip

Nach dem Rendern können Sie die zusammengesetzte Timeline als XML exportieren, um sie in Ihrem bevorzugten Editor feinzuschleifen. Jedes Video-Segment, jeder Dialog-Clip, SFX-Clip und Musik-Cue landet auf einer eigenen Spur.

```bash
mini-drama export-timeline -p output/<project> -e 1 --format fcpxml      # Final Cut Pro X
mini-drama export-timeline -p output/<project> -e 1 --format premiere    # Premiere Pro
mini-drama export-timeline -p output/<project> -e 1 --format davinci     # DaVinci Resolve
```

## Programmatische Nutzung

Sie können die Module des Harnesses auch direkt aus eigenem TypeScript aufrufen:

```typescript
import { VeniceClient } from './src/venice/client.js';
import { generateVideo, quoteVideo } from './src/venice/video.js';
import { listVideoModels } from './src/venice/models.js';

const client = new VeniceClient();

const quote = await quoteVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  duration: '8s',
  audio: true,
});
console.log(`Estimated cost: $${quote.quote}`);

const result = await generateVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  prompt: 'A slow dolly shot pushes forward...',
  duration: '8s',
  imageUrl: 'data:image/png;base64,...',
  audio: true,
  outputPath: 'output/shot-001.mp4',
});

const longModels = listVideoModels({ minDurationSec: 20 });
```

[Jan](https://jan.ai/) ist ein Open-Source-KI-Assistent zum Betrieb lokaler und entfernter Modelle in einer Desktop-App. Venice funktioniert in Jan über die Unterstützung OpenAI-kompatibler Remote-Modellanbieter.

## Voraussetzungen

- Ein Venice-API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) installiert und auf Ihrem Rechner lauffähig
- Eine Venice-Textmodell-ID aus dem [Modellkatalog](/de/models/text)

---

## Einrichtung

<Steps>
  <Step title="Venice-API-Schlüssel holen">
    Erzeugen oder kopieren Sie einen API-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Jans Modell-Provider-Einstellungen öffnen">
    Öffnen Sie in Jan **Settings** und navigieren Sie zu **Model Providers**.

    Wählen Sie den OpenAI-kompatiblen Custom-Provider-Flow von Jan. Je nach Jan-Version erscheint dies möglicherweise als **OpenAI** mit aktiviertem **Custom Endpoint** oder als **Any OpenAI Compatible API**-Provider.
  </Step>

  <Step title="Venice-Endpoint konfigurieren">
    Geben Sie die Venice-Provider-Einstellungen ein:

    | Feld | Wert |
    |-------|-------|
    | Provider-Name | `Venice AI` |
    | Base-URL | `https://api.venice.ai/api/v1` |
    | API-Schlüssel | Ihr Venice-API-Schlüssel |
    | API-Typ | OpenAI-kompatibel |

    Speichern bzw. übernehmen Sie die Provider-Einstellungen, bevor Sie Modelle hinzufügen.
  </Step>

  <Step title="Venice-Modelle hinzufügen">
    Fügen Sie die Venice-Modell-IDs hinzu, die Sie verwenden möchten. Jan verlangt, dass die Modell-ID exakt zum Provider passt.

    Gute Einstiegsoptionen:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Sie können jedes unterstützte Textmodell aus dem [Modellkatalog](/de/models/text) hinzufügen.
  </Step>

  <Step title="Chat starten">
    Öffnen Sie einen neuen oder bestehenden Jan-Chat, wählen Sie eines Ihrer Venice-Modelle aus dem Modell-Selector und senden Sie eine Testnachricht.

    Jan sendet die Anfrage über Ihren API-Schlüssel und die OpenAI-kompatible Base-URL an Venice.
  </Step>
</Steps>

---

## Einrichtung überprüfen

Senden Sie aus Jan einen kurzen Prompt:

```txt
Reply with one sentence confirming you are running through Venice.
```

Wenn der Provider korrekt konfiguriert ist, sollte Jan eine Antwort vom ausgewählten Venice-Modell streamen.

Sie können Ihren API-Schlüssel auch außerhalb von Jan prüfen:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Hinweise zur Konfiguration

- Verwenden Sie `https://api.venice.ai/api/v1` als Base-URL. Hängen Sie nicht `/chat/completions` an; Jan ergänzt den Endpoint-Pfad für Chat-Requests selbst.
- Modell-IDs müssen exakt mit Venice übereinstimmen. Wenn ein Modell nicht im Selector von Jan erscheint, fügen Sie es manuell mit der Modell-ID aus dem [Modellkatalog](/de/models/text) hinzu.
- Konfigurieren Sie Venice weiterhin als OpenAI-kompatiblen Provider. Verwenden Sie für Venice-Claude-Modelle nicht den nativen Anthropic-Provider von Jan.
- Remote-Anfragen an Venice erfordern eine Internetverbindung und verbrauchen Venice-API-Guthaben.

<Info>
Lokale Jan-Modelle bleiben verfügbar, nachdem Sie Venice hinzugefügt haben. Sie können im Chat-Modell-Selector zwischen lokalen und Venice-Modellen wechseln.
</Info>

## Fehlersuche

<AccordionGroup>
  <Accordion title="401 oder Authentifizierungsfehler">
    Prüfen Sie, ob der API-Schlüssel ein Venice-Schlüssel von [venice.ai/settings/api](https://venice.ai/settings/api) ist, ohne überflüssige Leerzeichen davor oder danach.
  </Accordion>

  <Accordion title="404 oder Modell nicht gefunden">
    Bestätigen Sie, dass die Modell-ID in Jan exakt mit einer Venice-Modell-ID übereinstimmt. Prüfen Sie außerdem, ob die Base-URL `https://api.venice.ai/api/v1` lautet.
  </Accordion>

  <Accordion title="Keine Venice-Modelle im Selector">
    Fügen Sie das Modell in Jans Provider-Einstellungen manuell hinzu. Verwenden Sie die exakte Modell-ID, etwa `zai-org-glm-5-1`, und speichern Sie den Provider erneut.
  </Accordion>

  <Accordion title="Anfragen gehen weiterhin an einen anderen Provider">
    Stellen Sie sicher, dass der Chat ein Modell verwendet, das Ihrem Venice-Provider zugeordnet ist. Wenn Jan einen eingebauten OpenAI- oder Anthropic-Modelleintrag nutzt, wechseln Sie zu dem hinzugefügten Venice-Modell.
  </Accordion>
</AccordionGroup>

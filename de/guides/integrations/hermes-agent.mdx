[Hermes Agent](https://hermes-agent.nousresearch.com) ist ein Open-Source-, selbstgehosteter KI-Agent von [Nous Research](https://nousresearch.com). Er verfügt über persistentes Memory, autonome Skill-Erstellung und eine eingebaute Lernschleife, die mit zunehmender Laufzeit fähiger wird. Richten Sie ihn auf die Venice-API, und Ihr Agent erhält Zugriff auf über 230 Modelle und Tools für Text, Bild, Video, Audio, Embeddings und mehr.

## Warum Venice + Hermes Agent?

Die Venice-API gibt Ihrem Hermes Agent über einen einzigen OpenAI-kompatiblen Endpoint Zugriff auf die gesamte Venice-Plattform.

| Fähigkeit | Was Sie erhalten |
|---|---|
| **Text und Chat** | Private und anonymisierte Modelle (GLM, Qwen, Claude, GPT und mehr) |
| **Bild** | Generierung, Bearbeitung, Hochskalierung und Hintergrund entfernen |
| **Video** | Generierung und Transkription |
| **Audio** | Sprachsynthese (TTS), Musikgenerierung und Speech-to-Text |
| **Embeddings** | Vektor-Embeddings für RAG und semantische Suche |
| **Tools** | Web-Scraping, Web-Suche, Text-Parsing und Crypto-RPC |

## Einrichtung

### 1. Hermes Agent installieren

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>

  <Tab title="WSL2 (Windows)">
    Installieren Sie zuerst [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install) und führen Sie dann denselben Befehl aus:
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>
</Tabs>

Nach der Installation Shell neu laden:

```bash
source ~/.zshrc   # oder: source ~/.bashrc
```

### 2. Venice als Provider konfigurieren

Den Modell-Setup-Wizard ausführen:

```bash
hermes model
```

Wählen Sie **Custom endpoint (self-hosted / VLLM / etc.)** und geben Sie bei der Aufforderung Folgendes ein:

| Feld | Wert |
|---|---|
| **API base URL** | `https://api.venice.ai/api/v1` |
| **API key** | Ihr Venice-API-Schlüssel |
| **Model name** | Eine Venice-Modell-ID (siehe unten) |

Holen Sie sich einen API-Schlüssel auf [venice.ai/settings/api](https://venice.ai/settings/api), falls Sie noch keinen haben.

<Tip>
Sie können Venice auch direkt in `~/.hermes/config.yaml` konfigurieren:

```yaml
model:
  default: zai-org-glm-5
  provider: custom
  base_url: https://api.venice.ai/api/v1
  api_key: ${VENICE_API_KEY}
```

Und den Schlüssel in `~/.hermes/.env` setzen:

```bash
VENICE_API_KEY=your-key-here
```
</Tip>

### 3. Modell auswählen

Wenn der Wizard nach einem Modell fragt, wählen Sie eines passend zu Ihrem Anwendungsfall:

| Anwendungsfall | Modell | Privatsphäre |
|---|---|---|
| Allgemein | `zai-org-glm-5` | Privat |
| Reasoning | `kimi-k2-6` | Privat |
| Coding | `claude-opus-4-7` | Anonymisiert |
| Vision | `z-ai-glm-5v-turbo` | Anonymisiert |
| Unbeschränkt | `venice-uncensored-1-2` | Privat |

Modell jederzeit ohne Neustart ändern:

```bash
hermes model              # vollständiger Wizard
```

Oder mitten in einer Session wechseln:

```text
/model custom:claude-opus-4-7
```

### 4. Loslegen

Klassisches CLI oder modernes TUI öffnen:

```bash
hermes            # klassisches CLI
hermes --tui      # modernes TUI (empfohlen)
```

Versuchen Sie einen Prompt, um zu prüfen, dass alles funktioniert:

```text
Summarize this repo in 5 bullets and tell me what the main entrypoint is.
```

## Messaging-Plattformen verbinden

Sobald das CLI funktioniert, verbinden Sie Ihre Messaging-Apps über das Gateway:

```bash
hermes gateway setup
```

Damit werden Sie durch das Verbinden von Telegram, Discord, Slack, WhatsApp, Signal und anderen Plattformen geführt. Ihr Agent wird über jeden verbundenen Kanal erreichbar — alles betrieben von Venice.

```bash
hermes gateway      # Messaging-Gateway starten
```

## Datenschutz-Modi

Venice-Modelle im Hermes Agent folgen denselben [Privacy-Tiers](/de/overview/privacy) wie die Venice-API:

- **Private**-Modelle (GLM, Qwen, DeepSeek, Llama, Venice Uncensored) laufen auf Venice's GPU-Flotte. Prompts werden niemals gespeichert oder protokolliert.
- **Anonymized**-Modelle (Claude, GPT, Gemini, Grok) werden über Venice proxied; alle identifizierenden Informationen werden entfernt. Der Drittanbieter sieht Venice als Kunden, nicht Sie.

## Venice-API-Skills

Hermes Agent hat ein eingebautes Skills-System, das mit dem [Agent-Skills](https://github.com/veniceai/skills)-Format kompatibel ist. Venice veröffentlicht offizielle Skills, die Ihrem Agenten die Verwendung jedes Venice-Endpoints beibringen (Chat, Bildgenerierung, Video, Audio, Embeddings, Augment-Tools und mehr).

Venice-Skills direkt aus GitHub installieren:

```bash
hermes skills install veniceai/skills
```

Oder nach einzelnen Skills suchen:

```bash
hermes skills search venice
```

Hermes erkennt jeden Skill am `SKILL.md`-Frontmatter und lädt ihn bei Bedarf.

| `hermes update` | Auf die neueste Version aktualisieren |

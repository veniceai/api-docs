[Hermes Agent](https://hermes-agent.nousresearch.com) ist ein quelloffener, selbstgehosteter KI-Agent von [Nous Research](https://nousresearch.com). Er bietet persistentes Memory, autonomes Skill-Erstellen und einen eingebauten Lern-Loop, der mit der Laufzeit immer fähiger wird. Richte ihn auf die Venice-API aus, und dein Agent erhält Zugriff auf über 230 Modelle und Tools für Text, Bild, Video, Audio, Embeddings und mehr.

## Warum Venice + Hermes Agent?

Die Venice-API gibt deinem Hermes Agent über einen einzigen OpenAI-kompatiblen Endpoint Zugriff auf die gesamte Venice-Plattform.

| Fähigkeit | Was du bekommst |
|---|---|
| **Text und Chat** | Private und anonymized Modelle (GLM, Qwen, Claude, GPT und mehr) |
| **Image** | Generierung, Bearbeitung, Upscaling und Hintergrundentfernung |
| **Video** | Generierung und Transkription |
| **Audio** | Sprachsynthese (TTS), Musikgenerierung und Speech-to-Text |
| **Embeddings** | Vektor-Embeddings für RAG und semantische Suche |
| **Tools** | Web-Scraping, Web-Suche, Text-Parsing und Crypto RPC |

## Einrichtung

### 1. Hermes Agent installieren

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>

  <Tab title="WSL2 (Windows)">
    Installiere zuerst [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install) und führe dann denselben Befehl aus:
    ```bash
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
    ```
  </Tab>
</Tabs>

Lade deine Shell nach der Installation neu:

```bash
source ~/.zshrc   # oder: source ~/.bashrc
```

### 2. Venice als Provider konfigurieren

Starte den Modell-Setup-Wizard:

```bash
hermes model
```

Wähle **Custom endpoint (self-hosted / VLLM / etc.)** und gib bei Aufforderung Folgendes an:

| Feld | Wert |
|---|---|
| **API Base URL** | `https://api.venice.ai/api/v1` |
| **API-Schlüssel** | Dein Venice API-Schlüssel |
| **Modellname** | Eine Venice-Modell-ID (siehe unten) |

Hol dir einen API-Schlüssel unter [venice.ai/settings/api](https://venice.ai/settings/api), falls noch nicht vorhanden.

<Tip>
Du kannst Venice auch direkt in `~/.hermes/config.yaml` konfigurieren:

```yaml
model:
  default: zai-org-glm-5
  provider: custom
  base_url: https://api.venice.ai/api/v1
  api_key: ${VENICE_API_KEY}
```

Und den Schlüssel in `~/.hermes/.env` setzen:

```bash
VENICE_API_KEY=your-key-here
```
</Tip>

### 3. Modell auswählen

Wenn der Wizard nach einem Modell fragt, wähle eins passend zu deinem Anwendungsfall:

| Anwendungsfall | Modell | Privacy |
|---|---|---|
| Allgemein | `zai-org-glm-5` | Private |
| Reasoning | `kimi-k2-6` | Private |
| Coding | `claude-opus-4-7` | Anonymized |
| Vision | `z-ai-glm-5v-turbo` | Anonymized |
| Uncensored | `venice-uncensored-1-2` | Private |

Modell jederzeit ohne Neustart wechseln:

```bash
hermes model              # vollständiger Wizard
```

Oder mitten in der Session umschalten:

```text
/model custom:claude-opus-4-7
```

### 4. Chatten

Öffne die klassische CLI oder die moderne TUI:

```bash
hermes            # klassische CLI
hermes --tui      # moderne TUI (empfohlen)
```

Teste mit einem Prompt, ob alles funktioniert:

```text
Summarize this repo in 5 bullets and tell me what the main entrypoint is.
```

## Messaging-Plattformen verbinden

Sobald die CLI läuft, verbinde deine Messaging-Apps über das Gateway:

```bash
hermes gateway setup
```

Das führt dich durch die Verbindung mit Telegram, Discord, Slack, WhatsApp, Signal und weiteren Plattformen. Dein Agent wird über jeden verbundenen Kanal erreichbar – betrieben mit Venice.

```bash
hermes gateway      # Messaging-Gateway starten
```

## Privacy-Modi

Venice-Modelle in Hermes Agent folgen denselben [Privacy-Stufen](/overview/privacy) wie die Venice-API:

- **Private**-Modelle (GLM, Qwen, DeepSeek, Llama, Venice Uncensored) laufen auf Venices GPU-Flotte. Prompts werden niemals gespeichert oder geloggt.
- **Anonymized**-Modelle (Claude, GPT, Gemini, Grok) werden über Venice geproxied, wobei alle identifizierenden Informationen entfernt werden. Der Drittanbieter sieht Venice als Kunden, nicht dich.

## Venice API-Skills

Hermes Agent hat ein integriertes Skills-System, das mit dem [Agent Skills](https://github.com/veniceai/skills)-Format kompatibel ist. Venice veröffentlicht offizielle Skills, die deinem Agent beibringen, wie er jeden Venice-Endpoint (Chat, Image Generation, Video, Audio, Embeddings, Augment-Tools und mehr) verwendet.

Venice-Skills direkt aus GitHub installieren:

```bash
hermes skills install veniceai/skills
```

Oder nach einzelnen Skills suchen:

```bash
hermes skills search venice
```

Hermes erkennt jeden Skill anhand seines `SKILL.md`-Frontmatters und lädt ihn bei Bedarf.

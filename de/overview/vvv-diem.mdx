VVV und DIEM sind die On-Chain-Finanzierungsebene von Venice für die API. Staken Sie DIEM, um eine feste tägliche Inferenz-Zuteilung zu erhalten – **1 DIEM = $1 pro Tag** Venice-API-Guthaben – ohne für jede Anfrage vorausbezahlte USD-Credits kaufen zu müssen.

| Token | Rolle |
| --- | --- |
| **VVV** | Der grundlegende Token von Venice auf Base. Staken Sie ihn, um Rendite zu erzielen und DIEM zu prägen. |
| **sVVV** | Beleg für gestaktes VVV. Sperren Sie sVVV zum aktuellen [Mint Rate](https://diem-calculator.venice.ai/), um DIEM zu prägen. |
| **DIEM** | Tokenisierte Recheneinheit. Staken Sie DIEM, um ein dauerhaftes tägliches API-Guthaben in Höhe von $1 pro DIEM freizuschalten. |

DIEM ist ein ERC-20 auf Base. Sie können ihn übertragen, handeln oder staken. Anders als bei USD-Credits nach dem Pay-as-you-go-Prinzip ist gestaktes DIEM Kapazität, die Ihnen gehört: Die tägliche Zuteilung erneuert sich in jeder Epoche und verwässert nicht, wenn die Netzwerknutzung steigt.

<Info>
  Modellraten, die gegen diese Zuteilung abgerechnet werden, finden Sie unter [API-Preise](/overview/pricing). Für Wallet-Pay-per-Request ohne API-Schlüssel siehe [x402](/guides/integrations/x402-venice-api).
</Info>

## So funktioniert die Finanzierung

API-Schlüssel belasten das verknüpfte Venice-Konto in dieser Reihenfolge: **DIEM**, dann gebündelte Credits, dann USD.

Wenn DIEM die aktive Währung ist:

* Ihre tägliche Zuteilung entspricht der Menge an DIEM, die Sie gestakt haben (1 gestaktes DIEM → $1 API-Guthaben in dieser Epoche).
* Ungenutztes DIEM in einer Epoche wird nicht übertragen. Die Zuteilung erneuert sich um **00:00 UTC**.
* Konten benötigen mindestens **0,1** gestaktes DIEM, bevor ein DIEM-Guthaben verwendbar ist.
* Anfragekosten verwenden dieselbe USD-Preisliste wie Pay-as-you-go; DIEM ist die Abrechnungswährung zu einem Kurs von 1:1 zum Dollar für die Tageszuteilung.

Prüfen Sie Ihr aktuelles und das Guthaben der nächsten Epoche im Dashboard unter [venice.ai/settings/api](https://venice.ai/settings/api) oder über den [Billing Balance](/api-reference/endpoint/billing/balance)-Endpoint.

## DIEM beziehen und staken

Verwenden Sie das Token-Dashboard unter [venice.ai/token](https://venice.ai/token). Verbinden Sie ein Base-Wallet – senden Sie Tokens nicht direkt an eine Contract-Adresse.

<Steps>
  <Step title="VVV oder DIEM erwerben">
    Kaufen Sie VVV oder DIEM auf einem Base-DEX wie [Aerodrome](https://aerodrome.finance) oder [Uniswap](https://app.uniswap.org), oder erhalten Sie DIEM per Übertragung aus einem anderen Wallet.

    | Asset | Contract (Base) |
    | --- | --- |
    | VVV | [`0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf`](https://basescan.org/token/0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf) |
    | DIEM | [`0xF4d97F2da56e8c3098f3a8D538DB630A2606a024`](https://basescan.org/token/0xF4d97F2da56e8c3098f3a8D538DB630A2606a024) |
  </Step>

  <Step title="VVV staken (beim Prägen von DIEM)">
    Staken Sie auf [venice.ai/token](https://venice.ai/token) VVV, um sVVV zu erhalten. Gestaktes VVV erzielt Emissionen. Der Staking-Contract ist [`0x321b7ff75154472B18EDb199033fF4D116F340Ff`](https://basescan.org/address/0x321b7ff75154472B18EDb199033fF4D116F340Ff#code).
  </Step>

  <Step title="DIEM aus sVVV prägen (optional)">
    Sperren Sie sVVV zum aktuellen Mint Rate, um DIEM zu prägen. Der Mint Rate gibt an, wie viel sVVV pro DIEM erforderlich ist, und steigt mit wachsender DIEM-Versorgung – Vorschau im [DIEM-Rechner](https://diem-calculator.venice.ai/).

    Geprägtes DIEM erscheint als ERC-20 in Ihrem Wallet. Das gesperrte sVVV bleibt gesperrt, bis Sie die gleiche Menge DIEM verbrennen, um es freizugeben.
  </Step>

  <Step title="DIEM für API-Guthaben staken">
    Staken Sie DIEM im selben Token-Dashboard. Jedes gestakte DIEM gewährt **$1 pro Tag** Venice-API-Guthaben, solange es gestakt bleibt.

    Das Unstaking von DIEM hat eine **1-tägige** Abkühlphase. Das Freigeben von sVVV, das geprägtes DIEM besichert hat, erfordert das Verbrennen dieses DIEM; das anschließende Unstaking des freigegebenen sVVV hat dann eine **7-tägige** Abkühlphase.
  </Step>

  <Step title="Einen API-Schlüssel erstellen oder verwenden">
    Melden Sie sich bei Venice mit demselben Wallet an (Sign-In-With-Ethereum), damit das gestakte DIEM Ihrem Konto zugeordnet wird, und erstellen Sie dann einen Schlüssel über die [API-Einstellungen](https://venice.ai/settings/api) oder den [Leitfaden für API-Schlüssel](/guides/getting-started/generating-api-key).

    Autonome Agenten können einen Schlüssel aus einem Wallet prägen, das gestaktes VVV hält – siehe [Autonome Erstellung von Agenten-API-Schlüsseln](/guides/getting-started/generating-api-key-agent).
  </Step>
</Steps>

<Warning>
  Senden Sie VVV oder DIEM niemals direkt an eine Contract-Adresse. Auf diese Weise gesendete Tokens können nicht wiederhergestellt werden. Staken Sie immer über [venice.ai/token](https://venice.ai/token) oder einen bekannten Staking-Transaktionsablauf.
</Warning>

## DIEM mit der API verwenden

Sobald DIEM gestakt und mit Ihrem Konto verknüpft ist, rufen Sie die API mit einem normalen Bearer-Schlüssel auf. Es sind keine zusätzlichen Header oder `venice_parameters` erforderlich – Venice zieht automatisch von DIEM ab, wenn `consumptionCurrency` gleich `DIEM` ist.

```bash
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### DIEM-Guthaben prüfen

```bash
curl https://api.venice.ai/api/v1/billing/balance \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

Beispielhafte Antwortstruktur:

```json
{
  "canConsume": true,
  "consumptionCurrency": "DIEM",
  "balances": {
    "diem": 90.5,
    "usd": 25
  },
  "diemEpochAllocation": 100
}
```

| Feld | Bedeutung |
| --- | --- |
| `balances.diem` | Verbleibendes DIEM-Guthaben für die aktuelle Epoche (`null`, wenn Sie kein DIEM staken) |
| `diemEpochAllocation` | Insgesamt für diese Epoche zugeteiltes DIEM (gestakte Menge). Vergleichen Sie mit `balances.diem`, um die bisherige Nutzung zu sehen |
| `consumptionCurrency` | Währung, die als Nächstes belastet wird (`DIEM`, `USD`, `BUNDLED_CREDITS` oder das ältere `VCU`) |
| `canConsume` | Gibt an, ob das Konto derzeit eine Anfrage bezahlen kann |

Listen Sie Modelle oder Nutzungsanalysen auf, um die Kosten pro Anfrage sowohl in USD als auch in DIEM zu sehen – sie stimmen für die Tageszuteilung zum Kurs von $1 pro DIEM überein. Siehe [Billing Usage](/api-reference/endpoint/billing/usage) und [Usage Analytics](/api-reference/endpoint/billing/usage-analytics).

### Ausgaben pro Schlüssel begrenzen

Beim Erstellen eines Schlüssels können Sie ein Epoch-Verbrauchslimit in `diem` oder `usd` festlegen, damit eine einzelne Integration nicht die gesamte Tageszuteilung aufbrauchen kann. Dashboard-Schlüssel bieten dies als **Epoch Consumption Limits** an; Web3-geprägte Schlüssel akzeptieren `consumptionLimit` bei [`POST /api_keys/generate_web3_key`](/api-reference/endpoint/api_keys/generate_web3_key/post).

## DIEM vs. USD-Credits vs. x402

| Weg | Am besten für | Wie er sich erneuert |
| --- | --- | --- |
| **Gestaktes DIEM** | Vorhersehbare tägliche Kapazität, Agenten, feste KI-Kosten | Volle Zuteilung in jeder Epoche (00:00 UTC), solange DIEM gestakt bleibt |
| **USD-/Krypto-Credits** | Spitzen über Ihre DIEM-Zuteilung hinaus, einfache Prepaid-Ausgaben | Guthaben nimmt mit der Nutzung ab; Credits verfallen nie |
| **x402 (USDC)** | Headless Pay-per-Request ohne API-Schlüssel | Wallet lädt USDC über `POST /x402/top-up` auf; DIEM auf einem verknüpften Konto wird trotzdem zuerst verbraucht, sofern vorhanden |

Wenn eine Anfrage das verbleibende DIEM überschreiten würde und weder USD- noch gebündelte Credits sie abdecken, gibt die API `402` mit `INSUFFICIENT_BALANCE` zurück. Staken Sie mehr DIEM, warten Sie auf die nächste Epoche oder fügen Sie USD-Credits unter [Einstellungen → API](https://venice.ai/settings/api) hinzu.

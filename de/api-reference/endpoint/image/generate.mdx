## Größenoptionen

Bildmodelle verwenden modellspezifische Größenparameter:

- Pixelbasierte Modelle akzeptieren `width` und `height`, zum Beispiel `venice-sd35` und `qwen-image`.
- Seitenverhältnis-Modelle akzeptieren `aspect_ratio`, zum Beispiel `qwen-image-2`.
- Modelle mit Auflösungsstufen akzeptieren sowohl `aspect_ratio` als auch `resolution`.

`gpt-image-2`, `nano-banana-2` und `nano-banana-pro` unterstützen beispielsweise `resolution`-Werte von `1K`, `2K` und `4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

Dasselbe Muster für Auflösungsstufen gilt auch für die Nano-Banana-Modelle:

```json
{
  "model": "nano-banana-pro",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Für Modelle, die ein Seitenverhältnis akzeptieren, aber keine Auflösungsstufen anbieten, senden Sie `aspect_ratio` ohne `resolution`:

```json
{
  "model": "qwen-image-2",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9"
}
```

Für Modelle, die direkte Pixelabmessungen verwenden, senden Sie stattdessen `width` und `height`:

```json
{
  "model": "venice-sd35",
  "prompt": "a serene canal in venice at sunset",
  "width": 1024,
  "height": 1024
}
```

Prüfen Sie [Bildmodelle](/de/models/image) oder den [Models-Endpoint](/de/api-reference/endpoint/models/list) auf die unterstützten Größenoptionen jedes Modells, bevor Sie Größenfelder zwischen Modellen mischen.

## Qualitätsstufen

Einige Modelle akzeptieren einen optionalen `quality`-Parameter (`low`, `medium`, `high`), der visuelle Qualität gegen Kosten abwägt. Wird derzeit von `gpt-image-2` unterstützt; andere Modelle ignorieren den Parameter.

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K",
  "quality": "medium"
}
```

Wenn Sie `quality` weglassen, verwendet das Modell seine Standardstufe (`high` für `gpt-image-2`). Der Preis hängt von der Kombination aus `resolution` und `quality` ab — die Preise pro Stufe für `gpt-image-2` (1K/2K/4K × low/medium/high) finden Sie unter `model_spec.pricing.quality` im [Models-Endpoint](/de/api-reference/endpoint/models/list) sowie in der [Preisübersicht](/de/overview/pricing).

<Info>
Qualitätsabhängige Preise sind der Standard für alle API- und SDK-Aufrufer — kein Opt-in-Header erforderlich. Anfragen ohne `quality` werden zur Standardstufe des Modells abgerechnet (`high` für `gpt-image-2`).
</Info>

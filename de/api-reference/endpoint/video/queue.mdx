Rufen Sie `/video/quote` auf, um einen Preisvoranschlag zu erhalten, und fragen Sie dann `/video/retrieve` mit der zurückgegebenen `queue_id` ab, bis der Vorgang abgeschlossen ist.

Private Modelle geben außerdem eine `download_url` für das fertige Video zurück. Es handelt sich um eine kurzlebige Delivery-URL (einige Wiederholungen sind in Ordnung, wenn ein Download abbricht); Details und das optionale `DELETE` aus Datenschutzgründen finden Sie im [Video Generation Guide](/guides/media/video-generation#private-download-links).

### Seedance 2.0

Für `seedance-2-0-*`-Modelle (text-to-video, image-to-video, reference-to-video plus die `-fast-*`-Varianten) siehe den [Seedance-2.0-Leitfaden](/guides/media/seedance-2-0) für das Vier-Workflow-Modell (Reference / Edit / Extend / Stitch), kanonische Prompt-Muster, multimodale Eingabegrenzen und Preisdetails.

### Seedance-Einwilligung

Wenn eine Seedance-Image- oder Reference-to-Video-Anfrage ein menschliches Gesicht enthält, gibt die API ein `409 needs_consent` mit dem Richtlinientext zurück, und Sie reichen dieselbe Anfrage erneut mit einer `consents.seedance`-Bestätigung ein. Den vollständigen Ablauf, das Consent-Objekt, Dedup und Widerruf finden Sie im [Seedance-Face-Media-Consent-Leitfaden](/guides/media/seedance-face-consent).

### Video-Upscaling

Für das `topaz-video-upscale`-Modell verwenden Sie `upscale_factor` (1, 2 oder 4) anstelle von `resolution` und stellen eine `video_url` bereit. Dauer und FPS werden automatisch aus der Videodatei erkannt. Vollständige Details und Beispiele finden Sie im [Video-Upscaling-Leitfaden](/guides/media/video-upscaling).

Rufen Sie `/video/quote` auf, um eine Preisschätzung zu erhalten, und pollen Sie anschließend `/video/retrieve` mit der zurückgegebenen `queue_id`, bis der Vorgang abgeschlossen ist.

Private Modelle geben außerdem eine `download_url` für das fertige Video zurück. Es handelt sich um eine kurzlebige Auslieferungs-URL (ein paar Wiederholungsversuche sind in Ordnung, wenn ein Download abbricht); Details und das optionale `DELETE` zum Schutz der Privatsphäre finden Sie im [Leitfaden zur Videogenerierung](/de/guides/media/video-generation#private-download-links).

### Seedance 2.0

Für `seedance-2-0-*`-Modelle (text-to-video, image-to-video, reference-to-video sowie die `-fast-*`-Varianten) finden Sie im [Seedance-2.0-Leitfaden](/de/guides/media/seedance-2-0) das Vier-Workflow-Modell (Reference / Edit / Extend / Stitch), kanonische Prompt-Muster, multimodale Eingabegrenzen und Preisdetails.

### Seedance-Einwilligung

Wenn eine Seedance-image- oder -reference-to-video-Anfrage ein menschliches Gesicht enthält, gibt die API ein `409 needs_consent` mit dem Richtlinientext zurück, und Sie senden dieselbe Anfrage erneut mit einer `consents.seedance`-Attestierung. Den vollständigen Ablauf, das Consent-Objekt, die Deduplizierung und den Widerruf finden Sie im [Seedance-Face-Media-Einwilligungsleitfaden](/de/guides/media/seedance-face-consent).

### Video-Hochskalierung

Für das `topaz-video-upscale`-Modell verwenden Sie `upscale_factor` (1, 2 oder 4) anstelle von `resolution` und geben eine `video_url` an. Dauer und FPS werden automatisch aus der Videodatei erkannt. Vollständige Details und Beispiele finden Sie im [Video-Upscaling-Leitfaden](/de/guides/media/video-upscaling).

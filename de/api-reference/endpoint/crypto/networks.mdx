Gibt die alphabetisch sortierte Liste der Netzwerk-Slugs zurück, die der Venice-Crypto-RPC-Proxy unterstützt. Verwenden Sie diese, um zu ermitteln, welche `:network`-Werte für `POST /crypto/rpc/{network}` gültig sind.

Dieser Endpoint ist **öffentlich** — es ist keine API-Schlüssel- oder Wallet-Authentifizierung erforderlich.

## Form der Antwort

```json
{
  "networks": [
    "arbitrum-mainnet",
    "arbitrum-sepolia",
    "avalanche-fuji",
    "avalanche-mainnet",
    "base-mainnet",
    "base-sepolia",
    "bsc-mainnet",
    "bsc-testnet",
    "ethereum-holesky",
    "ethereum-mainnet",
    "ethereum-sepolia",
    "linea-mainnet",
    "linea-sepolia",
    "optimism-mainnet",
    "optimism-sepolia",
    "polygon-amoy",
    "polygon-mainnet",
    "starknet-mainnet",
    "starknet-sepolia",
    "zksync-mainnet",
    "zksync-sepolia"
  ]
}
```

Die Liste ist verbindlich — fehlt ein Slug hier, gibt der Proxy-Endpoint `400 Unsupported RPC network` zurück.

## Preise nach Qualitätsstufen

Für Bildmodelle, die den optionalen `quality`-Parameter akzeptieren (derzeit `gpt-image-2` und `gpt-image-2-edit`), enthält die Antwort unter `model_spec.pricing.quality` eine Preismatrix pro Qualität. Jeder Schlüssel der obersten Ebene ist eine Auflösungsstufe (`1K`, `2K`, `4K`), und jeder verschachtelte Schlüssel ist eine Qualitätsstufe (`low`, `medium`, `high`) mit eigenem `usd`- und `diem`-Preis:

```json
"pricing": {
  "resolutions": {
    "1K": { "usd": 0.27, "diem": 0.27 },
    "2K": { "usd": 0.51, "diem": 0.51 },
    "4K": { "usd": 0.84, "diem": 0.84 }
  },
  "quality": {
    "1K": {
      "low":    { "usd": 0.02, "diem": 0.02 },
      "medium": { "usd": 0.07, "diem": 0.07 },
      "high":   { "usd": 0.26, "diem": 0.26 }
    },
    "2K": {
      "low":    { "usd": 0.03, "diem": 0.03 },
      "medium": { "usd": 0.13, "diem": 0.13 },
      "high":   { "usd": 0.50, "diem": 0.50 }
    },
    "4K": {
      "low":    { "usd": 0.05, "diem": 0.05 },
      "medium": { "usd": 0.21, "diem": 0.21 },
      "high":   { "usd": 0.83, "diem": 0.83 }
    }
  }
}
```

`pricing.resolutions` ist die alte Preisstaffel pro Bild, die aus Gründen der Abwärtskompatibilität beibehalten wird. `pricing.quality` ist die Matrix pro (Auflösung, Qualität), die immer dann gilt, wenn der Parameter `quality` unterstützt wird. Beide Felder bleiben in der Antwort enthalten, damit Clients die Qualitätsunterstützung erkennen und die Matrix in eigenen UIs darstellen können.

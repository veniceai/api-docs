Die Venice-API bietet HTTP-basierte REST- und Streaming-Schnittstellen zum Bauen von KI-Anwendungen mit unbeschränkten Modellen und privater Inference. Sie können mit Textgenerierung, Bilderzeugung, Embeddings und mehr arbeiten — alles ohne restriktive Inhaltsrichtlinien. Integrationsbeispiele und SDKs sind in der [Dokumentation](/de/overview/getting-started) verfügbar. Unsere API-Referenz steht außerdem als [OpenAPI-YAML-Spezifikation](https://api.venice.ai/doc/api/swagger.yaml) zur Verfügung.

## Authentifizierung

Die Venice-API verwendet API-Schlüssel zur Authentifizierung. Erstellen und verwalten Sie Ihre API-Schlüssel in Ihren [API-Einstellungen](https://venice.ai/settings/api).


Alle API-Anfragen erfordern HTTP-Bearer-Authentifizierung:

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
Ihr API-Schlüssel ist ein Geheimnis. Geben Sie ihn nicht weiter und legen Sie ihn nicht in clientseitigem Code offen.
</Note>

## OpenAI-Kompatibilität

Die Venice-API implementiert die OpenAI-API-Spezifikation und stellt damit Kompatibilität mit bestehenden OpenAI-Clients und -Tools sicher. So können Sie sich mit Venice über die vertraute OpenAI-Schnittstelle integrieren und gleichzeitig auf Venice's einzigartige Funktionen und unbeschränkten Modelle zugreifen.

### Einrichtung

Konfigurieren Sie Ihren Client auf die Base-URL von Venice (`https://api.venice.ai/api/v1`) und stellen Sie Ihre erste Anfrage:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## Venice-spezifische Funktionen

### System-Prompts

Venice stellt Standard-System-Prompts bereit, die unbeschränkte und natürliche Modellantworten gewährleisten sollen. Sie haben zwei Optionen für den Umgang mit System-Prompts:

1. **Standardverhalten**: Ihre System-Prompts werden an die Venice-Standards angehängt
2. **Eigenes Verhalten**: Die Venice-System-Prompts vollständig deaktivieren

#### Venice-System-Prompts deaktivieren

Verwenden Sie die Option `venice_parameters`, um die Standard-System-Prompts von Venice zu entfernen:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### Venice-Parameter

Das `venice_parameters`-Objekt gibt Ihnen Zugriff auf Venice-spezifische Funktionen, die in der Standard-OpenAI-API nicht verfügbar sind:

| Parameter | Typ | Beschreibung | Standard |
|-----------|------|-------------|---------|
| `character_slug` | string | Der Charakter-Slug eines öffentlichen Venice-Charakters (auf der veröffentlichten Charakterseite als "Public ID" sichtbar) | - |
| `strip_thinking_response` | boolean | `<think></think>`-Blöcke aus der Antwort entfernen (Modelle mit altem `<think>`-Tag-Format). Siehe [Reasoning-Modelle](/de/guides/features/reasoning-models). | `false` |
| `disable_thinking` | boolean | Bei unterstützten Reasoning-Modellen Thinking deaktivieren und die `<think></think>`-Blöcke aus der Antwort entfernen | `false` |
| `enable_web_search` | string | Web-Suche für diesen Request aktivieren (`off`, `on`, `auto` — `auto` aktiviert je nach Modell-Ermessen)<br/>Es fallen zusätzliche, nutzungsabhängige Kosten an, siehe [Preise](/de/overview/pricing#web-search-and-scraping). | `off` |
| `enable_web_scraping` | boolean | Web-Scraping für bis zu 5 in der Nutzernachricht erkannte URLs aktivieren. Gescrapte Inhalte ergänzen Antworten und umgehen die Web-Suche. Es werden nur erfolgreich gescrapte URLs berechnet.<br/>Es fallen zusätzliche, nutzungsabhängige Kosten an, siehe [Preise](/de/overview/pricing#web-search-and-scraping). | `false` |
| `enable_x_search` | boolean | xAIs native Suche (Web + X/Twitter) für unterstützte Grok-Modelle (z. B. `grok-4-20-beta`) aktivieren. Liefert hochwertigere Suchergebnisse über xAIs Suchinfrastruktur. Wenn aktiviert, wird die Standard-Web-Suche von Venice umgangen.<br/>Es fallen zusätzliche, nutzungsabhängige Kosten an, siehe [Preise](/de/overview/pricing#web-search-and-scraping). | `false` |
| `enable_web_citations` | boolean | Wenn die Web-Suche aktiviert ist, das LLM bitten, seine Quellen im Format `[REF]0[/REF]` zu zitieren | `false` |
| `include_search_results_in_stream` | boolean | Experimentell: Suchergebnisse als ersten emittierten Chunk in den Stream aufnehmen | `false` |
| `return_search_results_as_documents` | boolean | Suchergebnisse als OpenAI-kompatiblen Tool-Call namens `venice_web_search_documents` für die LangChain-Integration bereitstellen | `false` |
| `include_venice_system_prompt` | boolean | Ob die Standard-System-Prompts von Venice zusammen mit den angegebenen System-Prompts eingebunden werden | `true` |

<Note>
Diese Parameter können auch als Modell-Suffixe an den Modellnamen angehängt werden (z. B. `zai-org-glm-5:enable_web_search=auto`). Details siehe [Model Feature Suffixes](/de/api-reference/endpoint/chat/model_feature_suffix).
</Note>

### Prompt-Caching

Venice unterstützt Prompt-Caching auf ausgewählten Modellen, um Latenz und Kosten für wiederkehrende Inhalte zu reduzieren. Für unterstützte Modelle cacht Venice System-Prompts automatisch — ohne Codeänderungen. Sie können Inhalte mit der `cache_control`-Eigenschaft am Message-Inhalt zusätzlich manuell zum Caching markieren.

| Parameter | Typ | Beschreibung |
|-----------|------|-------------|
| `prompt_cache_key` | string | Optionaler Routing-Hint zur Verbesserung der Cache-Trefferraten. Wenn angegeben, leitet Venice Requests an dieselbe Backend-Infrastruktur, was die Wahrscheinlichkeit von Cache-Treffern über mehrere Gesprächsrunden erhöht. |

Details zur Funktionsweise, Abrechnung und Best Practices finden Sie unter [Prompt-Caching](/de/guides/features/prompt-caching).

## Response-Header-Referenz

Alle Antworten der Venice-API enthalten HTTP-Header mit Metadaten zum Request, zu Rate Limits, Modellinformationen und Kontoguthaben. Zusätzlich zu Fehlercodes können Sie diese Header inspizieren, um die eindeutige ID einer bestimmten API-Anfrage zu erfahren, Rate Limits zu überwachen und Ihr Kontoguthaben zu verfolgen.

Venice empfiehlt, Request-IDs (Header `CF-RAY`) in Produktiv-Deployments zu protokollieren, um die Fehlersuche mit unserem Support-Team effizienter zu gestalten.

Die folgende Tabelle gibt eine umfassende Referenz aller Header, denen Sie begegnen können:

| Header | Typ | Zweck | Wann zurückgegeben |
|--------|------|---------|---------------|
| **Standard-HTTP-Header** ||||
| `Content-Type` | string | MIME-Typ des Antwort-Bodys (`application/json`, `text/csv`, `image/png` usw.) | Immer |
| `Content-Encoding` | string | Verwendete Komprimierung des Antwort-Bodys (`gzip`, `br`) | Wenn der Client den `Accept-Encoding`-Header sendet |
| `Content-Disposition` | string | Wie der Inhalt dargestellt werden soll (z. B. `attachment; filename=export.csv`) | Beim Download von Dateien oder Exporten |
| `Date` | string | RFC-7231-formatierter Zeitstempel der Antworterzeugung | Immer |
| **Request-Identifikation** ||||
| `CF-RAY` | string | Eindeutige Kennung dieses API-Requests, für Fehlersuche und Support-Anfragen | Immer |
| `x-venice-version` | string | Aktuelle Version/Revision des Venice-API-Dienstes (z. B. `20250828.222653`) | Immer |
| `x-venice-timestamp` | string | Server-Zeitstempel der Verarbeitung (ISO-8601-Format) | Wenn Timestamp-Tracking aktiviert ist |
| `x-venice-host-name` | string | Hostname des Servers, der den Request verarbeitet hat | Bei Fehlerantworten und Debugging-Szenarien |
| **Modellinformationen** ||||
| `x-venice-model-id` | string | Eindeutige Kennung des für den Request verwendeten KI-Modells (z. B. `venice-01-lite`) | Inference-Endpoints mit KI-Modellen |
| `x-venice-model-name` | string | Freundlicher/Anzeigename des verwendeten KI-Modells (z. B. `Venice Lite`) | Inference-Endpoints mit KI-Modellen |
| `x-venice-model-router` | string | Router/Backend-Dienst, der die Modell-Inference verarbeitet hat | Inference-Endpoints, wenn Routing-Informationen verfügbar sind |
| `x-venice-model-deprecation-warning` | string | Warnmeldung für zur Deprecation geplante Modelle | Wenn ein deprekiertes Modell verwendet wird |
| `x-venice-model-deprecation-date` | string | Datum, an dem das Modell deprekiert wird (ISO-8601-Datum) | Wenn ein deprekiertes Modell verwendet wird |
| **Rate-Limit-Informationen** ||||
| `x-ratelimit-limit-requests` | number | Maximal erlaubte Anzahl von Requests im aktuellen Zeitfenster | Alle authentifizierten Requests |
| `x-ratelimit-remaining-requests` | number | Verbleibende Requests im aktuellen Zeitfenster | Alle authentifizierten Requests |
| `x-ratelimit-reset-requests` | number | Unix-Zeitstempel, wann das Request-Rate-Limit zurückgesetzt wird | Alle authentifizierten Requests |
| `x-ratelimit-limit-tokens` | number | Maximal erlaubte Anzahl von Tokens (Prompt + Completion) im Zeitfenster | Alle authentifizierten Requests |
| `x-ratelimit-remaining-tokens` | number | Verbleibende Tokens im aktuellen Zeitfenster | Alle authentifizierten Requests |
| `x-ratelimit-reset-tokens` | number | Dauer in Sekunden, bis das Token-Rate-Limit zurückgesetzt wird | Alle authentifizierten Requests |
| `x-ratelimit-type` | string | Art des angewandten Rate Limits (`user`, `api_key`, `global`) | Wenn Rate Limiting durchgesetzt wird |
| **Paginierungs-Header** ||||
| `x-pagination-limit` | number | Anzahl Elemente pro Seite | Paginierte Endpoints |
| `x-pagination-page` | number | Aktuelle Seitennummer (1-basiert) | Paginierte Endpoints |
| `x-pagination-total` | number | Gesamtanzahl Elemente über alle Seiten | Paginierte Endpoints |
| `x-pagination-total-pages` | number | Gesamtanzahl Seiten | Paginierte Endpoints |
| **Konto-Guthaben-Informationen** ||||
| `x-venice-balance-diem` | string | Ihr DIEM-Token-Guthaben vor Verarbeitung des Requests | Alle authentifizierten Requests |
| `x-venice-balance-usd` | string | Ihr USD-Credit-Guthaben vor Verarbeitung des Requests | Alle authentifizierten Requests |
| **Header zur Inhaltssicherheit** ||||
| `x-venice-is-blurred` | string | Zeigt an, ob das erzeugte Bild aufgrund von Inhaltsrichtlinien unscharf gemacht wurde (`true`/`false`) | Bilderzeugung mit aktiviertem Safe Venice |
| `x-venice-is-content-violation` | string | Zeigt an, ob Inhalte gegen die Venice-Inhaltsrichtlinien verstoßen (`true`/`false`) | Content-Generierungs-Endpoints |
| `x-venice-is-adult-model-content-violation` | string | Zeigt an, ob Inhalte gegen die Inhaltsrichtlinien für Adult-Modelle verstoßen (`true`/`false`) | Bilderzeugungs-Endpoints |
| `x-venice-contains-minor` | string | Zeigt an, ob ein Bild Minderjährige enthält (`true`/`false`) | Bild-Analyse-Endpoints mit Altersbestimmung |
| **Client-Informationen** ||||
| `x-venice-middleface-version` | string | Version des Venice-Middleface-Clients | Anfragen aus Venice-Middleface-Clients |
| `x-venice-mobile-version` | string | Version des Venice-Mobile-App-Clients | Anfragen aus Mobile-Apps |
| `x-venice-request-timestamp-ms` | number | Vom Client gelieferter Request-Zeitstempel in Millisekunden | Wenn der Client einen Zeitstempel im Request liefert |
| `x-venice-control-instance` | string | Control-Instance-Kennung zum Debugging | Bilderzeugungs-Endpoints zum Debugging |
| **Authentifizierungs-Header** ||||
| `x-auth-refreshed` | string | Zeigt an, dass das Auth-Token während der Anfrage aufgefrischt wurde (`true`/`false`) | Wenn Auth-Token automatisch aufgefrischt werden |
| `x-retry-count` | number | Anzahl der Retry-Versuche für die Anfrage | Wenn Retries auftreten |

### Wichtige Hinweise

- **Header-Groß-/Kleinschreibung**: HTTP-Header sind case-insensitiv, Venice verwendet aus Konsistenzgründen Kleinbuchstaben mit Bindestrichen
- **String-Werte**: Boolesche Werte in Headern werden als Strings zurückgegeben (`"true"` oder `"false"`)
- **Zahlenwerte**: Große Zahlen und Saldowerte können als Strings zurückgegeben werden, um Genauigkeitsverluste zu vermeiden
- **Optionale Header**: Nicht alle Header werden in jeder Antwort zurückgegeben; das Vorhandensein hängt von Endpoint und Request-Kontext ab
- **Komprimierung**: Verwenden Sie `Accept-Encoding: gzip, br` in Anfragen, um — sofern unterstützt — komprimierte Antworten zu erhalten

### Beispiel: Auf Response-Header zugreifen

```javascript
// Nach einem API-Request auf die Header des Response-Objekts zugreifen
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// Auf Modell-Deprecation-Warnungen prüfen
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## Best Practices

1. **Rate Limiting**: Beobachten Sie die Header `x-ratelimit-remaining-requests` und `x-ratelimit-remaining-tokens` und implementieren Sie exponentielles Backoff
2. **Guthaben-Überwachung**: Verfolgen Sie die Header `x-venice-balance-usd` und `x-venice-balance-diem`, um Dienstunterbrechungen zu vermeiden
3. **System-Prompts**: Testen Sie mit und ohne Venice-System-Prompts, um die beste Variante für Ihren Anwendungsfall zu finden
4. **API-Schlüssel**: Halten Sie Ihre API-Schlüssel sicher und rotieren Sie diese regelmäßig
5. **Request-Logging**: Loggen Sie `CF-RAY`-Header-Werte für die Fehlersuche mit dem Support
6. **Modell-Deprecation**: Prüfen Sie bei der Nutzung von Modellen auf den Header `x-venice-model-deprecation-warning`

## Unterschiede zur OpenAI-API

Während Venice eine hohe Kompatibilität mit der OpenAI-API-Spezifikation pflegt, gibt es einige wichtige Unterschiede:

1. **venice_parameters**: Zusätzliche Konfigurationen wie `enable_web_search`, `character_slug` und `strip_thinking_response` für erweiterte Funktionalität
2. **System-Prompts**: Venice hängt Ihre System-Prompts an Standardwerte an, die für unbeschränkte Antworten optimiert sind (deaktivierbar mit `include_venice_system_prompt: false`)
3. **Modell-Ökosystem**: Venice bietet ein eigenes [Modellsortiment](/de/overview/models), inklusive unbeschränkter und Reasoning-Modelle — verwenden Sie Venice-Modell-IDs statt OpenAI-Mappings
4. **Response-Header**: Einzigartige Header für die Guthabenverfolgung (`x-venice-balance-usd`, `x-venice-balance-diem`), Modell-Deprecation-Warnungen und Inhaltssicherheits-Flags
5. **Inhaltsrichtlinien**: Erlaubendere Richtlinien mit dedizierten unbeschränkten Modellen und optionaler Inhaltsfilterung

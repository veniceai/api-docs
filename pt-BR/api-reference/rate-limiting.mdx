  -H "Authorization: Bearer $VENICE_API_KEY"

## Tratamento de erros

Requisições com falha (500, 503, 429) devem ser repetidas com exponential backoff.

Para erros 429 especificamente, verifique o header `x-ratelimit-reset-requests` para o timestamp Unix exato em que você pode tentar novamente. A maioria das bibliotecas HTTP tem mecanismos de retry embutidos que lidam com isto automaticamente.

### Proteção contra abuso

Se você gerar mais de 20 requisições com falha em 30 segundos, a API bloqueará novas requisições por 30 segundos:

```
Too many failed attempts (> 20) resulting in a non-success status code. Please wait 30s and try again.
```

## Headers de resposta

Toda resposta inclui estes headers:

| Header | Descrição |
|:-------|:------------|
| `x-ratelimit-limit-requests` | Máximo de requisições permitidas na janela atual |
| `x-ratelimit-remaining-requests` | Requisições restantes na janela atual |
| `x-ratelimit-reset-requests` | Timestamp Unix de quando a janela reinicia |
| `x-ratelimit-limit-tokens` | Máximo de tokens permitidos por minuto |
| `x-ratelimit-remaining-tokens` | Tokens restantes no minuto atual |
| `x-ratelimit-reset-tokens` | Segundos até que o limite de tokens reinicie |

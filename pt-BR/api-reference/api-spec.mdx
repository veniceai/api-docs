A API Venice oferece interfaces REST e de streaming baseadas em HTTP para construir aplicações de IA com modelos uncensored e inferência privada. Você pode criar com geração de texto, criação de imagens, embeddings e mais — tudo sem políticas restritivas de conteúdo. Exemplos de integração e SDKs estão disponíveis na [documentação](/pt-BR/overview/getting-started). Nossa referência de API também está disponível como uma [especificação OpenAPI em YAML.](https://api.venice.ai/doc/api/swagger.yaml)

## Autenticação

A API Venice usa chaves de API para autenticação. Crie e gerencie suas chaves de API nas suas [configurações de API](https://venice.ai/settings/api).


Todas as requisições da API exigem autenticação HTTP Bearer:

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
Sua chave de API é um segredo. Não a compartilhe nem a exponha em qualquer código do lado do cliente.
</Note>

## Compatibilidade com OpenAI

A API da Venice implementa a especificação da API OpenAI, garantindo compatibilidade com clientes e ferramentas OpenAI existentes. Isto permite que você integre com a Venice usando a interface familiar do OpenAI enquanto acessa os recursos únicos da Venice e modelos uncensored.

### Configuração

Configure seu cliente para usar a URL base da Venice (`https://api.venice.ai/api/v1`) e faça sua primeira requisição:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## Recursos específicos da Venice

### System prompts

A Venice fornece system prompts padrão projetados para garantir respostas uncensored e naturais dos modelos. Você tem duas opções para lidar com system prompts:

1. **Comportamento padrão**: Seus system prompts são acrescentados aos padrões da Venice
2. **Comportamento personalizado**: Desabilita totalmente os system prompts da Venice

#### Desabilitando os system prompts da Venice

Use a opção `venice_parameters` para remover os system prompts padrão da Venice:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### Venice Parameters

O objeto `venice_parameters` permite acessar recursos específicos da Venice não disponíveis na API padrão da OpenAI:

| Parâmetro | Tipo | Descrição | Padrão |
|-----------|------|-------------|---------|
| `character_slug` | string | O slug de personagem de um personagem público da Venice (visível como "Public ID" na página do personagem publicado) | - |
| `strip_thinking_response` | boolean | Remove blocos `<think></think>` da resposta (modelos que usam o formato legado de tag `<think>`). Veja [Modelos de Reasoning](/pt-BR/guides/features/reasoning-models). | `false` |
| `disable_thinking` | boolean | Em modelos de reasoning compatíveis, desabilita o thinking e remove os blocos `<think></think>` da resposta | `false` |
| `enable_web_search` | string | Habilita web search para esta requisição (`off`, `on`, `auto` — auto habilita a critério do modelo)<br/>Aplica-se preço adicional por uso, veja [preços](/pt-BR/overview/pricing#web-search-and-scraping). | `off` |
| `enable_web_scraping` | boolean | Habilita web scraping de até 5 URLs detectadas na mensagem do usuário. O conteúdo extraído enriquece as respostas e contorna o web search. Apenas URLs extraídas com sucesso são cobradas.<br/>Aplica-se preço adicional por uso, veja [preços](/pt-BR/overview/pricing#web-search-and-scraping). | `false` |
| `enable_x_search` | boolean | Habilita a busca nativa do xAI (web + X/Twitter) para modelos Grok compatíveis (por exemplo, `grok-4-20-beta`). Fornece resultados de busca de maior qualidade usando a infraestrutura de busca do xAI. Quando habilitada, o web search padrão da Venice é ignorado.<br/>Aplica-se preço adicional por uso, veja [preços](/pt-BR/overview/pricing#web-search-and-scraping). | `false` |
| `enable_web_citations` | boolean | Quando web search está habilitado, solicita que o LLM cite suas fontes usando o formato `[REF]0[/REF]` | `false` |
| `include_search_results_in_stream` | boolean | Experimental: Inclui os resultados de busca no stream como o primeiro chunk emitido | `false` |
| `return_search_results_as_documents` | boolean | Expõe os resultados de busca em uma tool call compatível com OpenAI chamada `venice_web_search_documents` para integração com LangChain | `false` |
| `include_venice_system_prompt` | boolean | Se inclui os system prompts padrão da Venice junto com os system prompts especificados | `true` |

<Note>
Esses parâmetros também podem ser especificados como sufixos do modelo, anexados ao nome do modelo (por exemplo, `zai-org-glm-5:enable_web_search=auto`). Veja [Sufixos de Feature de Modelo](/pt-BR/api-reference/endpoint/chat/model_feature_suffix) para detalhes.
</Note>

### Prompt Caching

A Venice oferece suporte a prompt caching em modelos selecionados para reduzir latência e custos para conteúdo repetido. Para modelos compatíveis, a Venice cacheia os system prompts automaticamente — nenhuma alteração de código é necessária. Você também pode marcar conteúdo manualmente para caching usando a propriedade `cache_control` no conteúdo da mensagem.

| Parâmetro | Tipo | Descrição |
|-----------|------|-------------|
| `prompt_cache_key` | string | Dica opcional de roteamento para melhorar as taxas de cache hit. Quando fornecida, a Venice roteia as requisições para a mesma infraestrutura de backend, aumentando a probabilidade de cache hits em conversas multi-turn. |

Veja [Prompt Caching](/pt-BR/guides/features/prompt-caching) para detalhes sobre como o caching funciona, cobrança e melhores práticas.

## Referência de headers de resposta

Todas as respostas da API Venice incluem headers HTTP que fornecem metadados sobre a requisição, rate limits, informações do modelo e saldo da conta. Além dos códigos de erro retornados pelas respostas da API, você pode inspecionar esses headers para obter o ID único de uma requisição específica da API, monitorar rate limiting e acompanhar o saldo da sua conta.

A Venice recomenda registrar os IDs de requisição (header `CF-RAY`) em deployments de produção para uma resolução de problemas mais eficiente com nosso time de suporte, caso seja necessário.

A tabela abaixo fornece uma referência abrangente de todos os headers que você pode encontrar:

| Header | Tipo | Propósito | Quando é retornado |
|--------|------|---------|---------------|
| **Headers HTTP padrão** ||||
| `Content-Type` | string | Tipo MIME do corpo da resposta (`application/json`, `text/csv`, `image/png`, etc.) | Sempre |
| `Content-Encoding` | string | Codificação usada para comprimir o corpo da resposta (`gzip`, `br`) | Quando o cliente envia o header `Accept-Encoding` |
| `Content-Disposition` | string | Como o conteúdo deve ser exibido (por exemplo, `attachment; filename=export.csv`) | Ao baixar arquivos ou exports |
| `Date` | string | Timestamp formatado em RFC 7231 de quando a resposta foi gerada | Sempre |
| **Identificação da requisição** ||||
| `CF-RAY` | string | Identificador único desta requisição da API, usado para troubleshooting e solicitações de suporte | Sempre |
| `x-venice-version` | string | Versão/revisão atual do serviço da API Venice (por exemplo, `20250828.222653`) | Sempre |
| `x-venice-timestamp` | string | Timestamp do servidor quando a requisição foi processada (formato ISO 8601) | Quando o rastreamento de timestamp está habilitado |
| `x-venice-host-name` | string | Hostname do servidor que processou a requisição | Respostas de erro e cenários de debug |
| **Informações do modelo** ||||
| `x-venice-model-id` | string | Identificador único do modelo de IA usado na requisição (por exemplo, `venice-01-lite`) | Endpoints de inferência usando modelos de IA |
| `x-venice-model-name` | string | Nome amigável/de exibição do modelo de IA usado (por exemplo, `Venice Lite`) | Endpoints de inferência usando modelos de IA |
| `x-venice-model-router` | string | Serviço de router/backend que lidou com a inferência do modelo | Endpoints de inferência quando há informações de roteamento disponíveis |
| `x-venice-model-deprecation-warning` | string | Mensagem de aviso para modelos agendados para depreciação | Ao usar um modelo depreciado |
| `x-venice-model-deprecation-date` | string | Data em que o modelo será depreciado (data em ISO 8601) | Ao usar um modelo depreciado |
| **Informações de rate limiting** ||||
| `x-ratelimit-limit-requests` | number | Número máximo de requisições permitidas na janela de tempo atual | Todas as requisições autenticadas |
| `x-ratelimit-remaining-requests` | number | Número de requisições restantes na janela de tempo atual | Todas as requisições autenticadas |
| `x-ratelimit-reset-requests` | number | Timestamp Unix de quando o rate limit de requisições reinicia | Todas as requisições autenticadas |
| `x-ratelimit-limit-tokens` | number | Número máximo de tokens (prompt + completion) permitidos na janela de tempo | Todas as requisições autenticadas |
| `x-ratelimit-remaining-tokens` | number | Número de tokens restantes na janela de tempo atual | Todas as requisições autenticadas |
| `x-ratelimit-reset-tokens` | number | Duração em segundos até que o rate limit de tokens reinicie | Todas as requisições autenticadas |
| `x-ratelimit-type` | string | Tipo de rate limit aplicado (`user`, `api_key`, `global`) | Quando rate limiting é aplicado |
| **Headers de paginação** ||||
| `x-pagination-limit` | number | Número de itens por página | Endpoints paginados |
| `x-pagination-page` | number | Número da página atual (começando em 1) | Endpoints paginados |
| `x-pagination-total` | number | Número total de itens em todas as páginas | Endpoints paginados |
| `x-pagination-total-pages` | number | Número total de páginas | Endpoints paginados |
| **Informações de saldo da conta** ||||
| `x-venice-balance-diem` | string | Seu saldo de tokens DIEM antes do processamento da requisição | Todas as requisições autenticadas |
| `x-venice-balance-usd` | string | Seu saldo de crédito em USD antes do processamento da requisição | Todas as requisições autenticadas |
| **Headers de segurança de conteúdo** ||||
| `x-venice-is-blurred` | string | Indica se a imagem gerada foi desfocada devido a políticas de conteúdo (`true`/`false`) | Geração de imagem com Safe Venice habilitado |
| `x-venice-is-content-violation` | string | Indica se o conteúdo viola as políticas de conteúdo da Venice (`true`/`false`) | Endpoints de geração de conteúdo |
| `x-venice-is-adult-model-content-violation` | string | Indica se o conteúdo viola as políticas de conteúdo de modelos adultos (`true`/`false`) | Endpoints de geração de imagem |
| `x-venice-contains-minor` | string | Indica se a imagem contém menores de idade (`true`/`false`) | Endpoints de análise de imagem com detecção de idade |
| **Informações do cliente** ||||
| `x-venice-middleface-version` | string | Versão do cliente Venice middleface | Requisições de clientes Venice middleface |
| `x-venice-mobile-version` | string | Versão do cliente do app móvel Venice | Requisições de aplicações móveis |
| `x-venice-request-timestamp-ms` | number | Timestamp da requisição fornecido pelo cliente em milissegundos | Quando o cliente fornece timestamp na requisição |
| `x-venice-control-instance` | string | Identificador da instância de controle para debug | Endpoints de geração de imagem para debug |
| **Headers de autenticação** ||||
| `x-auth-refreshed` | string | Indica que o token de autenticação foi atualizado durante a requisição (`true`/`false`) | Quando tokens de autenticação são atualizados automaticamente |
| `x-retry-count` | number | Número de tentativas de retry para a requisição | Quando ocorrem retries da requisição |

### Notas importantes

- **Case dos nomes de header**: Headers HTTP são case-insensitive, mas a Venice usa minúsculas com hífens por consistência
- **Valores de string**: Valores boolean em headers são retornados como strings (`"true"` ou `"false"`)
- **Valores numéricos**: Números grandes e valores de saldo podem ser retornados como strings para evitar perda de precisão
- **Headers opcionais**: Nem todos os headers são retornados em toda resposta; a presença depende do endpoint e do contexto da requisição
- **Compressão**: Use `Accept-Encoding: gzip, br` nas requisições para receber respostas comprimidas onde houver suporte

### Exemplo: Acessando os headers de resposta

```javascript
// Após fazer uma requisição da API, acesse os headers a partir do objeto de resposta
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// Verifique avisos de depreciação de modelo
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## Melhores práticas

1. **Rate Limiting**: Monitore os headers `x-ratelimit-remaining-requests` e `x-ratelimit-remaining-tokens` e implemente exponential backoff
2. **Monitoramento de saldo**: Acompanhe os headers `x-venice-balance-usd` e `x-venice-balance-diem` para evitar interrupções de serviço
3. **System prompts**: Teste com e sem os system prompts da Venice para encontrar o melhor encaixe para seu caso de uso
4. **Chaves de API**: Mantenha suas chaves de API seguras e faça rotação delas regularmente
5. **Logging de requisições**: Registre os valores do header `CF-RAY` para troubleshooting com o suporte
6. **Depreciação de modelos**: Verifique os headers `x-venice-model-deprecation-warning` ao usar modelos

## Diferenças em relação à API da OpenAI

Embora a Venice mantenha alta compatibilidade com a especificação da API da OpenAI, há algumas diferenças importantes:

1. **venice_parameters**: Configurações adicionais como `enable_web_search`, `character_slug` e `strip_thinking_response` para funcionalidades estendidas
2. **System prompts**: A Venice acrescenta seus system prompts aos padrões que otimizam para respostas uncensored (desabilite com `include_venice_system_prompt: false`)
3. **Ecossistema de modelos**: A Venice oferece seu próprio [lineup de modelos](/pt-BR/overview/models) incluindo modelos uncensored e de reasoning — use os IDs de modelo da Venice em vez dos mapeamentos da OpenAI
4. **Headers de resposta**: Headers únicos para rastreamento de saldo (`x-venice-balance-usd`, `x-venice-balance-diem`), avisos de depreciação de modelo e flags de segurança de conteúdo
5. **Políticas de conteúdo**: Políticas mais permissivas com modelos uncensored dedicados e filtragem de conteúdo opcional

## Estabilidade da API

A Venice mantém compatibilidade retroativa para endpoints e parâmetros v1. Para a política de ciclo de vida de modelos, notificações de depreciação e orientação para migração, veja [Depreciações](/pt-BR/overview/deprecations).

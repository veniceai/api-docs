## Opções de dimensionamento

Os modelos de imagem usam parâmetros de dimensionamento específicos do modelo:

- Modelos baseados em pixels aceitam `width` e `height`, por exemplo `venice-sd35` e `qwen-image`.
- Modelos baseados em aspect ratio aceitam `aspect_ratio`, por exemplo `qwen-image-2`.
- Modelos baseados em tier de resolução aceitam tanto `aspect_ratio` quanto `resolution`.

Por exemplo, `gpt-image-2`, `nano-banana-2` e `nano-banana-pro` suportam valores de `resolution` `1K`, `2K` e `4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

O mesmo padrão de dimensionamento por tier de resolução também se aplica aos modelos Nano Banana:

```json
{
  "model": "nano-banana-pro",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Para modelos que aceitam aspect ratio mas não expõem tiers de resolução, envie `aspect_ratio` sem `resolution`:

```json
{
  "model": "qwen-image-2",
  "prompt": "a serene canal in venice at sunset",
  "aspect_ratio": "16:9"
}
```

Para modelos que usam dimensões diretas em pixels, envie `width` e `height`:

```json
{
  "model": "venice-sd35",
  "prompt": "a serene canal in venice at sunset",
  "width": 1024,
  "height": 1024
}
```

Consulte [Modelos de Imagem](/pt-BR/models/image) ou o [endpoint de Modelos](/pt-BR/api-reference/endpoint/models/list) para ver as opções de dimensionamento suportadas por cada modelo antes de misturar campos de dimensionamento entre modelos.

## Tiers de qualidade

Alguns modelos aceitam um parâmetro opcional `quality` (`low`, `medium`, `high`) que troca fidelidade visual por custo. Atualmente é suportado por `gpt-image-2`; outros modelos ignoram o parâmetro.

```json
{
  "model": "gpt-image-2",
  "prompt": "a cinematic wide shot of Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K",
  "quality": "medium"
}
```

Quando você omite `quality`, o modelo usa seu tier padrão (`high` para `gpt-image-2`). O preço depende da combinação de `resolution` e `quality` — os preços por tier para `gpt-image-2` (1K/2K/4K × low/medium/high) são expostos em `model_spec.pricing.quality` no [endpoint de Modelos](/pt-BR/api-reference/endpoint/models/list) e estão listados na [Visão geral de preços](/pt-BR/overview/pricing).

<Info>
A precificação por qualidade é padrão para todos os chamadores da API e do SDK — nenhum header de opt-in é necessário. Requisições que omitem `quality` são cobradas no tier padrão do modelo (`high` para `gpt-image-2`).
</Info>

VVV e DIEM formam a camada de financiamento on-chain da Venice para a API. Faça staking de DIEM para receber uma cota diária fixa de inferência — **1 DIEM = US$ 1 por dia** de crédito da API Venice — sem precisar comprar créditos pré-pagos em USD para cada requisição.

| Token | Função |
| --- | --- |
| **VVV** | Token fundacional da Venice na Base. Faça staking para gerar rendimento e para cunhar DIEM. |
| **sVVV** | Recibo do VVV em staking. Bloqueie sVVV à [Taxa de Cunhagem](https://diem-calculator.venice.ai/) atual para cunhar DIEM. |
| **DIEM** | Unidade de computação tokenizada. Faça staking de DIEM para desbloquear um crédito diário perpétuo da API equivalente a US$ 1 por DIEM. |

DIEM é um ERC-20 na Base. Você pode transferi-lo, negociá-lo ou fazer staking. Ao contrário dos créditos em USD no modelo pay-as-you-go, o DIEM em staking é uma capacidade que pertence a você: a cota diária é renovada a cada época e não dilui conforme o uso da rede cresce.

<Info>
  Para as tarifas por modelo cobradas contra essa cota, consulte [Preços da API](/overview/pricing). Para pagamento por requisição via carteira sem uma chave de API, consulte [x402](/guides/integrations/x402-venice-api).
</Info>

## Como funciona o financiamento

As chaves de API gastam da conta Venice vinculada nesta ordem: **DIEM**, depois [créditos empacotados](/overview/pricing#bundled-credits), depois USD.

Quando DIEM é a moeda ativa:

* Sua alocação diária é igual à quantidade de DIEM em staking (1 DIEM em staking → US$ 1 de crédito da API naquela época).
* DIEM não utilizado em uma época não é acumulado para a próxima. A cota é renovada às **00:00 UTC**.
* As contas precisam ter pelo menos **0,1** DIEM em staking antes que qualquer saldo de DIEM possa ser gasto.
* Os custos das requisições usam a mesma tabela de preços em USD do pay-as-you-go; DIEM é a moeda de liquidação a uma taxa 1:1 com o dólar para a alocação daquele dia.

Verifique seu saldo atual e o da próxima época no painel em [venice.ai/settings/api](https://venice.ai/settings/api), ou pelo endpoint [Billing Balance](/api-reference/endpoint/billing/balance).

## Obter DIEM e fazer staking

Use o painel de tokens em [venice.ai/token](https://venice.ai/token). Conecte uma carteira Base — não envie tokens diretamente para um endereço de contrato.

<Steps>
  <Step title="Adquira VVV ou DIEM">
    Compre VVV ou DIEM em uma DEX da Base como [Aerodrome](https://aerodrome.finance) ou [Uniswap](https://app.uniswap.org), ou receba DIEM transferido de outra carteira.

    | Ativo | Contrato (Base) |
    | --- | --- |
    | VVV | [`0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf`](https://basescan.org/token/0xacfE6019Ed1A7Dc6f7B508C02d1b04ec88cC21bf) |
    | DIEM | [`0xF4d97F2da56e8c3098f3a8D538DB630A2606a024`](https://basescan.org/token/0xF4d97F2da56e8c3098f3a8D538DB630A2606a024) |
  </Step>

  <Step title="Faça staking de VVV (se for cunhar DIEM)">
    Em [venice.ai/token](https://venice.ai/token), faça staking de VVV para receber sVVV. VVV em staking gera emissões. O contrato de staking é [`0x321b7ff75154472B18EDb199033fF4D116F340Ff`](https://basescan.org/address/0x321b7ff75154472B18EDb199033fF4D116F340Ff#code).
  </Step>

  <Step title="Cunhe DIEM a partir de sVVV (opcional)">
    Bloqueie sVVV à Taxa de Cunhagem atual para cunhar DIEM. A Taxa de Cunhagem é a quantidade de sVVV necessária por DIEM e aumenta conforme a oferta de DIEM cresce — visualize-a na [calculadora DIEM](https://diem-calculator.venice.ai/).

    O DIEM cunhado aparece na sua carteira como um ERC-20. O sVVV bloqueado permanece bloqueado até que você queime a mesma quantidade de DIEM para desbloqueá-lo.
  </Step>

  <Step title="Faça staking de DIEM para obter crédito de API">
    Faça staking de DIEM no mesmo painel de tokens. Cada DIEM em staking concede **US$ 1 por dia** de crédito da API Venice enquanto permanecer em staking.

    Retirar DIEM do staking tem um período de espera (cooldown) de **1 dia**. Desbloquear o sVVV que lastreou o DIEM cunhado exige queimar esse DIEM; retirar o sVVV liberado do staking tem então um cooldown de **7 dias**.
  </Step>

  <Step title="Crie ou use uma chave de API">
    Faça login na Venice com a mesma carteira (Sign-In-With-Ethereum) para que o DIEM em staking seja vinculado à sua conta e, em seguida, crie uma chave em [Configurações da API](https://venice.ai/settings/api) ou pelo [guia de chaves de API](/guides/getting-started/generating-api-key).

    Agentes autônomos podem cunhar uma chave a partir de uma carteira que possui VVV em staking — consulte [Criação de Chave de API para Agente Autônomo](/guides/getting-started/generating-api-key-agent).
  </Step>
</Steps>

<Warning>
  Nunca envie VVV ou DIEM diretamente para um endereço de contrato. Tokens enviados dessa forma não podem ser recuperados. Sempre faça staking através de [venice.ai/token](https://venice.ai/token) ou de um fluxo de transação de staking conhecido.
</Warning>

## Usar DIEM com a API

Depois que o DIEM estiver em staking e vinculado à sua conta, chame a API com uma chave Bearer normal. Nenhum cabeçalho extra ou `venice_parameters` é necessário — a Venice debita do DIEM automaticamente quando `consumptionCurrency` é `DIEM`.

```bash
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### Consultar o saldo de DIEM

```bash
curl https://api.venice.ai/api/v1/billing/balance \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

Exemplo do formato da resposta:

```json
{
  "canConsume": true,
  "consumptionCurrency": "DIEM",
  "balances": {
    "diem": 90.5,
    "usd": 25
  },
  "diemEpochAllocation": 100
}
```

| Campo | Significado |
| --- | --- |
| `balances.diem` | Crédito de DIEM restante para a época atual (`null` se você não estiver fazendo staking de DIEM) |
| `diemEpochAllocation` | Total de DIEM alocado para esta época (quantidade em staking). Compare com `balances.diem` para ver o uso até o momento |
| `consumptionCurrency` | Moeda que será cobrada em seguida (`DIEM`, `USD`, `BUNDLED_CREDITS` ou o legado `VCU`) |
| `canConsume` | Se a conta pode pagar por uma requisição agora |

Liste modelos ou analytics de uso para ver os custos por requisição tanto em USD quanto em DIEM — eles são equivalentes à taxa de US$ 1 por DIEM para a cota do dia. Consulte [Uso de Faturamento](/api-reference/endpoint/billing/usage) e [Analytics de Uso](/api-reference/endpoint/billing/usage-analytics).

### Limitar gastos por chave

Ao criar uma chave, você pode definir um limite de consumo por época em `diem` ou `usd`, para que uma única integração não esgote toda a alocação diária. As chaves do painel expõem isso como **Limites de Consumo por Época**; chaves cunhadas via Web3 aceitam `consumptionLimit` em [`POST /api_keys/generate_web3_key`](/api-reference/endpoint/api_keys/generate_web3_key/post).

## DIEM vs créditos em USD vs x402

| Caminho | Ideal para | Como é renovado |
| --- | --- | --- |
| **DIEM em staking** | Capacidade diária previsível, agentes, custo fixo de IA | Alocação completa a cada época (00:00 UTC) enquanto o DIEM permanecer em staking |
| **Créditos em USD / cripto** | Picos acima da sua alocação de DIEM, gasto pré-pago simples | O saldo diminui com o uso; os créditos nunca expiram |
| **x402 (USDC)** | Pagamento por requisição headless sem chave de API | A carteira recarrega USDC via `POST /x402/top-up`; DIEM em uma conta vinculada continua sendo gasto primeiro, quando presente |

Se uma requisição exceder o DIEM restante e não houver créditos em USD ou empacotados para cobri-la, a API retorna `402` com `INSUFFICIENT_BALANCE`. Faça staking de mais DIEM, aguarde a próxima época ou adicione créditos em USD em [Configurações → API](https://venice.ai/settings/api).

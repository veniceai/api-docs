Às vezes lançamos modelos em beta para coletar feedback e confirmar seu desempenho antes de um rollout completo de produção. Os modelos beta estão disponíveis para todos os usuários, mas **não são recomendados para uso em produção**.

O status beta não garante a promoção para produção. Um modelo beta pode ser removido se for muito custoso operar, tiver desempenho ruim em escala ou levantar preocupações de segurança. Modelos beta podem mudar sem aviso prévio e podem ter documentação ou suporte limitados. Modelos que se mostram estáveis, amplamente úteis e alinhados aos nossos padrões são promovidos para disponibilidade geral.

## Considerações importantes

Ao usar modelos beta, tenha em mente:

- Podem ser alterados ou removidos a qualquer momento sem o período padrão de aviso de depreciação
- Não são adequados para aplicações de produção ou fluxos de trabalho críticos
- Podem ter desempenho, disponibilidade ou comportamento inconsistentes
- Suporte limitado ou inexistente para migração se forem removidos
- Mais adequados para testes, avaliação e projetos experimentais

Para aplicações de produção, recomendamos usar os modelos estáveis do nosso [lineup principal de modelos](/pt-BR/models/overview).

### Verificando o status beta via API

Você pode verificar se um modelo está em beta chamando o endpoint [List Models](/pt-BR/api-reference/endpoint/models/list). Os modelos beta incluem um campo `betaModel` definido como `true` em seu `model_spec`:

```json
{
  "id": "some-beta-model",
  "model_spec": {
    "name": "Some Beta Model",
    "betaModel": true,
    "privacy": "private"
  },
  "type": "text",
  "object": "model",
  "owned_by": "venice.ai"
}
```

Você pode verificar com `if (model.model_spec.betaModel)` para identificar modelos beta e avisar os usuários ou tratá-los de forma diferente na sua aplicação.

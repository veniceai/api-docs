A Venice oferece APIs separadas para descobrir páginas web e extrair seu conteúdo:

- **Web Search** retorna resultados estruturados com títulos, URLs, trechos e datas.
- **Web Scrape** converte uma página pública selecionada em Markdown.

Use-as em conjunto quando sua aplicação precisar de material de origem atualizado e inspecionável antes da inferência.

## Pesquisar na web

Envie uma consulta para a API Search:

```bash
curl https://api.venice.ai/api/v1/augment/search \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query":"latest developments in decentralized AI"}'
```

O Brave é o provedor padrão e oferece Zero Data Retention. O Google também está disponível, com consultas anonimizadas encaminhadas via proxy pela Venice. Consulte a [referência da API Web Search](/api-reference/endpoint/augment/search) para a seleção de provedor e o esquema completo da resposta.

## Extrair uma página

Depois de selecionar um resultado, envie sua URL pública para a API Scrape:

```bash
curl https://api.venice.ai/api/v1/augment/scrape \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/article"}'
```

O endpoint retorna o conteúdo da página em Markdown. Primeiro ele solicita a representação nativa em Markdown do site, quando disponível, e caso contrário utiliza extração baseada em navegador.

## Fluxo de pesquisa para extração

<Steps>
  <Step title="Pesquise">
    Envie uma consulta focada e colete os resultados estruturados.
  </Step>
  <Step title="Selecione as fontes">
    Filtre os resultados por relevância, domínio e data antes de obter o conteúdo das páginas.
  </Step>
  <Step title="Extraia">
    Extraia o Markdown apenas das páginas que sua aplicação pretende usar.
  </Step>
  <Step title="Use ou armazene o conteúdo">
    Inclua o Markdown em um prompt, crie embeddings, coloque em cache ou preserve as URLs de origem para citações.
  </Step>
</Steps>

## APIs autônomas ou busca web do modelo?

Use as APIs autônomas quando sua aplicação precisa de controle sobre a seleção das fontes ou pretende reutilizar o conteúdo recuperado. Ative a busca web em uma requisição de modelo quando quiser que a Venice cuide da recuperação como parte de uma única requisição de inferência.

| Necessidade | Abordagem |
|---|---|
| Inspecionar ou filtrar resultados antes da inferência | Web Search autônomo |
| Extrair uma URL específica conhecida | Web Scrape autônomo |
| Colocar em cache ou reutilizar o conteúdo recuperado | APIs autônomas |
| Deixar o modelo recuperar contexto atualizado automaticamente | Busca web do modelo |

<Warning>
Web Search e Web Scrape são APIs experimentais. Seus formatos de requisição e resposta podem mudar sem aviso prévio. O Web Scrape aceita apenas páginas acessíveis publicamente, e sites que bloqueiam acesso automatizado podem ser rejeitados.
</Warning>

O [Brave Leo](https://brave.com/leo/) é o assistente de IA integrado ao navegador Brave. O recurso Bring Your Own Model (BYOM) do Leo pode se conectar diretamente ao endpoint de chat completions compatível com OpenAI da Venice, permitindo que você use modelos Venice para chat e assistência com reconhecimento de página sem instalar uma extensão.

## Pré-requisitos

- Brave Browser 1.69 ou posterior no Windows, macOS ou Linux
- Uma chave de API Venice em [venice.ai/settings/api](https://venice.ai/settings/api)
- Um ID de modelo do [catálogo de modelos de texto da Venice](/models/text)

<Note>
O recurso BYOM do Brave está disponível no desktop. O ID do modelo deve corresponder exatamente a um ID de modelo Venice.
</Note>

## Configuração

<Steps>
  <Step title="Abra as configurações do Leo">
    Abra `brave://settings/leo-ai`, ou abra o Leo pela barra lateral do Brave, selecione o ícone de configurações e navegue até **Leo**.
  </Step>

  <Step title="Adicione um modelo personalizado">
    Role até **Bring your own model** e selecione **Add new model**.
  </Step>

  <Step title="Configure o endpoint da Venice">
    Insira os seguintes valores:

    | Campo | Valor |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | Um ID de modelo do [catálogo de modelos de texto](/models/text) |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | Sua chave de API Venice |

    Se o Brave exibir **Context size**, defina-o para a janela de contexto suportada pelo modelo selecionado. Você também pode adicionar um prompt de sistema opcional.

    Insira a chave de API em si, sem prefixo `Bearer`. O Brave adiciona o cabeçalho de autorização a cada requisição.
  </Step>

  <Step title="Salve e selecione a Venice">
    Selecione **Add model** e, em seguida, defina **Venice AI** como o modelo padrão para novas conversas ou escolha-o no seletor de modelos do Leo.
  </Step>

  <Step title="Teste a conexão">
    Abra o Leo em uma página da web e envie um prompt como:

    ```txt
    Summarize this page in five bullet points.
    ```

    O Leo envia a conversa e o conteúdo relevante da página diretamente para o endpoint da Venice.
  </Step>
</Steps>

## Habilite a busca na web da Venice

O Brave não expõe parâmetros do corpo da requisição específicos da Venice no BYOM. Para habilitar a busca na web da Venice, adicione um [sufixo de recurso de modelo](/api-reference/endpoint/chat/model_feature_suffix) ao ID do modelo em **Model request name**:

```txt
<model-id>:enable_web_search=auto
```

Para habilitar sempre a busca e incluir citações, use:

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
A busca na web tem preços adicionais baseados no uso. Veja [preços da API](/overview/pricing) para tarifas atuais.
</Info>

## Privacidade

Quando você usa um endpoint BYOM remoto, o Brave se conecta diretamente a esse provedor. As requisições contornam os servidores do Brave, portanto as proteções de proxy e retenção do lado do servidor do Brave não se aplicam a essas requisições.

A Venice não armazena nem registra o conteúdo de prompts e respostas em seu proxy. O modelo selecionado determina o nível de privacidade em tempo de execução:

- Modelos **Private** usam infraestrutura de retenção zero de dados.
- Modelos **Anonymized** ocultam informações identificáveis do provedor upstream, mas o provedor pode processar o prompt.
- Modelos **TEE** e **E2EE** adicionam isolamento com respaldo de hardware ou criptografia do lado do cliente, quando suportados.

Verifique o rótulo de privacidade do modelo no [catálogo de modelos de texto](/models/text) e leia [Privacidade da Venice](/overview/privacy) antes de enviar informações sensíveis.

O Brave pode manter o histórico de chat do Leo no armazenamento local do navegador. Você pode limpar os dados do Leo, usar um chat temporário ou desativar o histórico de chat local nas configurações do Leo no Brave.

## Notas de configuração

- O Brave exige o caminho completo do endpoint: `https://api.venice.ai/api/v1/chat/completions`.
- IDs de modelos e disponibilidade mudam com o tempo. Copie o ID atual do [catálogo de modelos de texto](/models/text).
- O tamanho de contexto não deve exceder a janela de contexto suportada pelo modelo selecionado.
- Resumos de página podem enviar o conteúdo da página para a Venice como parte do prompt.
- Requisições remotas à Venice exigem conexão com a internet e consomem créditos de API Venice ou seu limite de DIEM disponível.

## Solução de problemas

<AccordionGroup>
  <Accordion title="Erro 401 ou de autenticação">
    Confirme que a chave veio de [venice.ai/settings/api](https://venice.ai/settings/api). Cole a chave bruta sem `Bearer` e remova quaisquer espaços no início ou no fim.
  </Accordion>
  <Accordion title="Modelo não encontrado">
    Copie o ID do modelo exatamente do [catálogo de modelos de texto](/models/text). Se você adicionou um sufixo de recurso, primeiro teste o modelo sem o sufixo.
  </Accordion>
  <Accordion title="Erro de rede ou resposta sem streaming">
    Confirme que o endpoint do servidor inclui `/api/v1/chat/completions`. O Brave Leo espera uma resposta com streaming compatível com OpenAI.
  </Accordion>
  <Accordion title="Resumos de página sem contexto da página">
    Verifique a permissão do Leo para usar o conteúdo da página e garanta que a página atual esteja acessível ao navegador. Páginas internas do Brave e algumas páginas protegidas não podem ser resumidas.
  </Accordion>
</AccordionGroup>

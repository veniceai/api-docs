O [Jan](https://jan.ai/) é um assistente de IA de código aberto para executar modelos locais e remotos em um único aplicativo de desktop. A Venice funciona no Jan por meio de seu suporte a provedores de modelo remoto compatíveis com OpenAI.

## Pré-requisitos

- Uma chave de API Venice em [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) instalado e rodando na sua máquina
- Um ID de modelo de texto Venice do [catálogo de modelos](/models/text)

---

## Configuração

<Steps>
  <Step title="Obtenha sua chave de API Venice">
    Gere ou copie uma chave de API em [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Abra as configurações de provedor de modelo do Jan">
    No Jan, abra **Settings** e vá em **Model Providers**.

    Escolha o fluxo de provedor personalizado compatível com OpenAI do Jan. Dependendo da sua versão do Jan, isso pode aparecer como **OpenAI** com **Custom Endpoint** habilitado, ou como um provedor **Any OpenAI Compatible API**.
  </Step>

  <Step title="Configure o endpoint Venice">
    Insira as configurações do provedor Venice:

    | Campo | Valor |
    |-------|-------|
    | Nome do provedor | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | Sua chave de API Venice |
    | Tipo de API | Compatível com OpenAI |

    Salve ou aplique as configurações do provedor antes de adicionar modelos.
  </Step>

  <Step title="Adicione modelos Venice">
    Adicione os IDs dos modelos Venice que você deseja usar. O Jan exige que o ID do modelo corresponda exatamente ao provedor.

    Boas opções iniciais:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Você pode adicionar qualquer modelo de texto suportado do [catálogo de modelos](/models/text).
  </Step>

  <Step title="Inicie um chat">
    Abra um chat novo ou existente do Jan, selecione um dos seus modelos Venice no seletor de modelos e envie uma mensagem de teste.

    O Jan enviará a requisição à Venice usando sua chave de API e a base URL compatível com OpenAI.
  </Step>
</Steps>

---

## Verifique a configuração

Envie um prompt curto pelo Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

Se o provedor estiver configurado corretamente, o Jan deve receber em streaming uma resposta do modelo Venice selecionado.

Você também pode verificar sua chave de API fora do Jan com:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Notas de configuração

- Use `https://api.venice.ai/api/v1` como base URL. Não inclua `/chat/completions`; o Jan acrescenta o caminho do endpoint para requisições de chat.
- Os IDs de modelo devem corresponder à Venice exatamente. Se um modelo não aparecer no seletor do Jan, adicione-o manualmente usando o ID do modelo do [catálogo de modelos](/models/text).
- Mantenha a Venice configurada como um provedor compatível com OpenAI. Não use o provedor nativo Anthropic do Jan para modelos Claude da Venice.
- Requisições remotas à Venice exigem uma conexão de internet e consomem créditos de API Venice.

<Info>
Os modelos locais do Jan permanecem disponíveis depois que você adiciona a Venice. Você pode alternar entre modelos locais e Venice no seletor de modelos do chat.
</Info>

## Solução de problemas

<AccordionGroup>
  <Accordion title="Erro 401 ou de autenticação">
    Verifique se a chave de API é uma chave Venice de [venice.ai/settings/api](https://venice.ai/settings/api), sem espaços extras antes ou depois.
  </Accordion>

  <Accordion title="Erro 404 ou modelo não encontrado">
    Confirme que o ID do modelo no Jan corresponde exatamente a um ID de modelo Venice. Também verifique se a base URL é `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="Sem modelos Venice no seletor">
    Adicione o modelo manualmente nas configurações de provedor do Jan. Use o ID exato do modelo, como `zai-org-glm-5-1`, e salve o provedor novamente.
  </Accordion>

  <Accordion title="As requisições ainda vão para outro provedor">
    Certifique-se de que o chat está usando um modelo vinculado ao seu provedor Venice. Se o Jan estiver usando uma entrada de modelo OpenAI ou Anthropic integrada, troque para o modelo Venice que você adicionou.
  </Accordion>
</AccordionGroup>

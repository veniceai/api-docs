O [Aider](https://aider.chat/) é uma ferramenta de programação em par com IA que edita seu repositório git local a partir do terminal. A Venice funciona por meio do suporte à [API compatível com OpenAI](https://aider.chat/docs/llms/openai-compat.html) do Aider — defina a URL base e a chave de API da Venice, e então adicione o prefixo `openai/` ao ID do modelo.

## Pré-requisitos

- Uma chave de API Venice em [venice.ai/settings/api](https://venice.ai/settings/api)
- Python 3.9+ disponível na sua máquina
- Um repositório git para trabalhar

---

## Configuração

<Steps>
  <Step title="Instale o Aider">
    Instale o Aider com o instalador oficial:

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    Consulte a [documentação de instalação do Aider](https://aider.chat/docs/install.html) se preferir outro método de instalação.
  </Step>

  <Step title="Configure a Venice como endpoint OpenAI">
    Aponte o Aider para a Venice definindo a URL base compatível com OpenAI e sua chave de API Venice.

    **Mac / Linux:**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell):**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (Prompt de Comando, persistido):**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    Reinicie o terminal após o `setx` para que as novas variáveis fiquem disponíveis.
  </Step>

  <Step title="Inicie o Aider com um modelo Venice">
    A partir do diretório do seu projeto, adicione o prefixo `openai/` ao ID do modelo Venice:

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    Outros bons modelos para começar:

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    Navegue por todos os IDs de modelos de texto no [catálogo de modelos](/models/text).
  </Step>
</Steps>

## Persista a configuração com `.env`

Para uma configuração local do projeto, crie um arquivo `.env` na raiz do seu repositório git:

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

Depois inicie o Aider normalmente:

```bash
aider
```

O Aider também lê o `.env` do seu diretório home. Os valores da raiz do repositório têm prioridade sobre os do diretório home. Mantenha este arquivo fora do git — adicione `.env` ao `.gitignore` se ele ainda não estiver ignorado.

<Info>
Você também pode definir `AIDER_OPENAI_API_BASE` e `AIDER_OPENAI_API_KEY`. A documentação oficial do Aider ainda documenta `OPENAI_API_BASE` e `OPENAI_API_KEY` para provedores compatíveis com OpenAI.
</Info>

---

## Persista a configuração com YAML

Você pode armazenar as mesmas configurações em `.aider.conf.yml` na raiz do seu repositório ou no diretório home:

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

Depois execute:

```bash
aider
```

## Verifique a configuração

Depois que o Aider iniciar, peça algo simples que comprove que o modelo está acessível:

```txt
Reply with one sentence confirming you are running through Venice.
```

Se o endpoint e a chave estiverem corretos, o Aider fará streaming de uma resposta do modelo Venice selecionado.

Você também pode confirmar a chave de API fora do Aider:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## Escolhendo modelos

| Caso de uso                    | Exemplo de ID de modelo | Flag do Aider                      |
| ------------------------------ | ----------------------- | ---------------------------------- |
| Codificação/refatoração forte  | `zai-org-glm-5-1`       | `--model openai/zai-org-glm-5-1`   |
| Iteração rápida                | `minimax-m25`           | `--model openai/minimax-m25`       |
| Codificação de contexto longo  | `kimi-k2-5`             | `--model openai/kimi-k2-5`         |
| Chat sem censura               | `venice-uncensored`     | `--model openai/venice-uncensored` |

Sempre adicione o prefixo `openai/` aos IDs de modelo Venice ao iniciar o Aider. Sem esse prefixo, o Aider não roteará a requisição pelo caminho do provedor compatível com OpenAI.

Descubra os IDs de modelo atuais em tempo de execução:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

Ou verifique [GET /models/traits](/api-reference/endpoint/models/traits) para padrões como `text:default`.

## Notas de configuração

- Mantenha `OPENAI_API_BASE` definido como `https://api.venice.ai/api/v1`. Não anexe `/chat/completions`; o Aider adiciona o caminho do chat automaticamente.
- Use uma chave de API Venice em `OPENAI_API_KEY`. Não use uma chave OpenAI com a URL base da Venice.
- O Aider pode avisar que o modelo tem janela de contexto ou custo desconhecidos. Esse aviso é esperado para muitos IDs de modelo Venice e geralmente pode ser ignorado. Veja [Avisos de modelo do Aider](https://aider.chat/docs/llms/warnings.html).
- Prefira modelos capazes de codificação/raciocínio para edições em múltiplos arquivos. Modelos menores ou não voltados a código podem ter dificuldade com os formatos de edição do Aider.

<Warning>
Definir `OPENAI_API_BASE` e `OPENAI_API_KEY` no seu shell também afeta outras ferramentas que leem essas variáveis. Prefira um `.env` do projeto ou `.aider.conf.yml` se você usa OpenAI em outros lugares no mesmo ambiente.
</Warning>

## Solução de problemas

<AccordionGroup>
  <Accordion title="Erro 401 ou de autenticação">
    Confirme que `OPENAI_API_KEY` é uma chave Venice de [venice.ai/settings/api](https://venice.ai/settings/api), sem espaços extras. No Windows, após `setx`, reinicie o terminal.
  </Accordion>
  <Accordion title="Modelo não encontrado ou resposta vazia">
    Confirme que o ID do modelo existe no [catálogo de modelos de texto](/models/text) e que você iniciou o Aider com o prefixo `openai/`, por exemplo `openai/zai-org-glm-5-1`.
  </Accordion>
  <Accordion title="As requisições ainda vão para a OpenAI">
    Verifique se `OPENAI_API_BASE` está exatamente `https://api.venice.ai/api/v1` na mesma sessão de shell. Imprima com `echo $OPENAI_API_BASE` (ou `$env:OPENAI_API_BASE` no PowerShell).
  </Accordion>
  <Accordion title="Aviso de janela de contexto / custos desconhecidos">
    Esperado para muitos modelos Venice. O Aider recorre a padrões razoáveis. Para silenciá-lo, adicione metadados do modelo conforme descrito em [Configurações avançadas de modelo do Aider](https://aider.chat/docs/config/adv-model-settings.html).
  </Accordion>
</AccordionGroup>

O [Cline](https://cline.bot/) é um assistente de codificação com IA para o VS Code que pode ler e editar arquivos, executar comandos e trabalhar em tarefas de desenvolvimento com várias etapas. A Venice funciona por meio das configurações de provedor compatível com OpenAI do Cline.

## Pré-requisitos

- Uma chave de API Venice em [venice.ai/settings/api](https://venice.ai/settings/api)
- [Visual Studio Code](https://code.visualstudio.com/) instalado
- A [extensão Cline](https://marketplace.visualstudio.com/items?itemName=saoudrizwan.claude-dev) instalada no VS Code

---

## Configuração

<Steps>
  <Step title="Abra as configurações do Cline">
    Selecione o ícone do Cline na barra de atividades do VS Code e, em seguida, selecione o ícone de engrenagem para abrir as configurações do Cline.
  </Step>

  <Step title="Escolha o provedor de API">
    Defina **API Provider** como **OpenAI Compatible**.
  </Step>

  <Step title="Configure a Venice">
    Insira os seguintes valores:

    | Configuração | Valor |
    | --- | --- |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API Key | Sua chave de API Venice |
    | Model ID | Um ID de modelo do [catálogo de modelos de texto](/models/text) |

    Por exemplo, use `zai-org-glm-5-1` como Model ID.

    Insira a chave de API em si, sem prefixo `Bearer`. Mantenha a Base URL em `/api/v1`; o Cline adiciona o caminho `/chat/completions`.
  </Step>

  <Step title="Configure as capacidades do modelo">
    Expanda **Model Configuration** e defina a janela de contexto e o máximo de tokens de saída para corresponder ao modelo selecionado. Habilite as capacidades que o modelo suporta, incluindo entrada de imagem ou uso do computador, quando aplicável.

    Verifique o modelo selecionado no [catálogo de modelos de texto](/models/text) antes de habilitar capacidades opcionais.
  </Step>

  <Step title="Salve e teste">
    Salve as configurações do provedor, abra um projeto e dê ao Cline uma pequena tarefa como:

    ```txt
    Review the current file and suggest one safe improvement.
    ```

    Revise cada permissão de arquivo ou comando antes de aprová-la.
  </Step>
</Steps>

## Verifique a API Venice

Se o Cline não conseguir se conectar, confirme a chave de API e os IDs de modelo disponíveis de forma independente:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

Copie o `id` de um modelo exatamente para o campo **Model ID** do Cline.

---

## Notas de configuração

- Use `https://api.venice.ai/api/v1` como Base URL, e não o endpoint completo `/chat/completions`.
- IDs de modelos podem mudar com o tempo. Use o [catálogo de modelos de texto](/models/text) ou `GET /models` em vez de depender de um exemplo antigo.
- As configurações de janela de contexto e tokens de saída do Cline afetam como ele orça as requisições. Ajuste-as ao modelo selecionado para melhores resultados.
- Mantenha sua chave de API Venice privada. Não a cole em arquivos-fonte nem faça commit dela no git.

## Solução de problemas

<AccordionGroup>
  <Accordion title="Erro 401 ou de autenticação">
    Confirme que a chave de API veio de [venice.ai/settings/api](https://venice.ai/settings/api). Cole a chave bruta no Cline sem `Bearer` ou espaços em branco extras.
  </Accordion>
  <Accordion title="Modelo não encontrado">
    Confirme que o Model ID aparece no [catálogo de modelos de texto](/models/text) ou na resposta de `GET /models` e, em seguida, copie-o exatamente.
  </Accordion>
  <Accordion title="Erro de conexão ou endpoint">
    Confirme que **API Provider** é **OpenAI Compatible** e que a Base URL é exatamente `https://api.venice.ai/api/v1`.
  </Accordion>
  <Accordion title="O Cline não consegue usar ferramentas ou editar arquivos">
    Abra **Model Configuration**, confirme que o modelo selecionado suporta as capacidades necessárias e habilite o uso do computador se o Cline expuser essa opção para o modelo.
  </Accordion>
</AccordionGroup>

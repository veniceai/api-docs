A API da Venice é protegida por chaves de API. Para começar a usar a API Venice, primeiro você precisa gerar uma nova chave. Siga estes passos para começar.

<Steps>
  <Step title="Acesse a página de configurações da API">
    Para chegar à página de configurações da API, visite https://venice.ai/settings/api. Essa página é acessível clicando em "API" na barra de ferramentas à esquerda ou clicando em "API" nas configurações do seu usuário.

    Dentro desse dashboard, você pode visualizar seus saldos em Diem e em USD, seu tier de API, seu uso da API e suas chaves de API.

    <Frame>
      <img src="/images/guides/API-Overview.png" alt="Visão geral da API" />
    </Frame>
  </Step>

  <Step title="Clique em Gerar nova chave de API">
    Role o dashboard para baixo e selecione "Generate New API Key". Você verá uma lista de opções.

    * **Description:** Usada para nomear sua chave de API

    * **API Key Type:**

      * Chaves "Admin" têm a capacidade de excluir ou gerar chaves de API adicionais programaticamente.

      * Chaves "Inference Only" só têm permissão para executar inferência.

    * **Expires at:** Você pode optar por definir uma data de expiração para a chave de API, após a qual ela deixará de funcionar. Por padrão, uma data não é definida e a chave funcionará indefinidamente.

    * **Epoch Consumption Limits:** Isso permite criar limites de uso da API para a chave individual. Você pode optar por limitar a quantidade em Diem ou em USD permitida dentro de uma determinada época (24h).

    <Frame>
      <img src="/images/guides/api-keys/create-key.png" alt="Gerar nova chave de API" />
    </Frame>
  </Step>

  <Step title="Gere a chave">
    Clicar em Generate mostrará a chave de API.

    <Warning>
      **Importante:** Esta chave só é exibida uma vez. Certifique-se de copiá-la e armazená-la em um local seguro. Se você a perder, terá que excluí-la e criar uma nova.
    </Warning>

    <Frame>
      <img src="/images/guides/api-keys/result.png" alt="Sua chave de API" />
    </Frame>
  </Step>
</Steps>

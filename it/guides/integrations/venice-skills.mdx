[Venice Skills](https://github.com/veniceai/skills) è la collezione canonica di [Agent Skills](https://www.anthropic.com/news/agent-skills) per l'API Venice. Ogni skill è una cartella autoconsistente con un `SKILL.md` che un agente LLM carica on demand per funzionare correttamente su una specifica area dell'API.

## Perché le skill?

Senza skill, il tuo agente deve scoprire le particolarità di Venice nel modo più difficile: `venice_parameters`, enum dei tipi di modello, flussi di pagamento 402, ciclo di vita queue/retrieve per i video, slug dei character e così via. Le skill raggruppano queste conoscenze in file focalizzati e on demand, in modo che l'agente carichi solo ciò che gli serve per il task corrente.

Ogni `SKILL.md` include:

- Gli endpoint che copre
- Header, parametri e strutture di risposta richiesti
- Un esempio curl più un esempio minimo con SDK
- Una sezione "gotcha" con le cose su cui gli integratori reali inciampano

## Installazione

Ogni skill è semplicemente una cartella con un `SKILL.md` che inizia con il frontmatter YAML:

```yaml
---
name: venice-chat
description: When the agent should load this skill and what's in it
---
```

Sposta la cartella `skills/` (o qualsiasi sottoinsieme) in qualunque percorso il tuo runtime sorveglia.

<Tabs>
  <Tab title="Claude Code">
    A livello di progetto:

    ```bash
    git clone https://github.com/veniceai/skills.git
    cp -r skills/skills/* .claude/skills/
    ```

    Oppure globale, per ogni progetto sulla tua macchina:

    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.claude/skills/venice
    ```
  </Tab>

  <Tab title="Cursor">
    A livello di progetto:

    ```bash
    git clone https://github.com/veniceai/skills.git .cursor/skills-venice
    ```

    Oppure copia skill singole:

    ```bash
    cp -r skills/venice-chat .cursor/skills/
    ```
  </Tab>

  <Tab title="Codex">
    ```bash
    git clone https://github.com/veniceai/skills.git ~/src/venice-skills
    ln -s ~/src/venice-skills/skills ~/.codex/skills/venice
    ```

    Per installazioni a livello di progetto, punta a `.codex/skills/`.
  </Tab>

  <Tab title="OpenCode">

### Riferimento percorsi

| Runtime | A livello di progetto | Globale |
|---|---|---|
| Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| Codex | `.codex/skills/` | `~/.codex/skills/` (o `$CODEX_HOME/skills/`) |
| OpenCode | `.opencode/skills/` (anche `.claude/skills/`, `.agents/skills/`) | `~/.config/opencode/skills/` |
| Hermes Agent | `$HERMES_OPTIONAL_SKILLS_DIR` | `~/.hermes/skills/` |
| Cursor | `.cursor/skills/` | `~/.cursor/skills/` |
| Cline | `.clinerules/skills/` | n/d |
| Altri runtime | `.agents/skills/` (convenzione) | `~/.agents/skills/` |

<Tip>
I runtime che definiscono campi extra del frontmatter (`version`, `platforms`, `metadata.*`, `compatibility`, …) sono tenuti dalla spec a ignorare i campi sconosciuti, quindi lo stesso file di skill funziona ovunque senza fork.
</Tip>

### Come git submodule

Se vuoi versioni pinnate nel tuo repo:

```bash
git submodule add https://github.com/veniceai/skills.git vendor/venice-skills
```

Poi crea symlink o copia i sottoinsiemi che vuoi nel percorso skill del tuo agente.

## Come gli agenti le caricano

L'agente scopre ogni `SKILL.md` dal suo frontmatter `name` e `description`. Quando l'utente chiede qualcosa che corrisponde allo scopo di una skill, l'agente carica quel singolo file nel contesto (non l'intero catalogo), così il prompt resta piccolo e la risposta resta accurata.

Ad esempio, un agente che deve generare musica caricherà `venice-audio-music` e saprà subito:

- Che la musica passa attraverso il ciclo queue/retrieve/complete, non un endpoint sincrono
- Quali modelli sono disponibili e il loro pricing al minuto
- Come chiamare prima `/audio/quote` per la stima del costo
- Come dovrebbe essere il backoff del polling

Senza la skill, l'agente potrebbe provare a chiamare `/audio/speech` per la musica e ottenere una risposta inutile.

## Scrivere una nuova skill

1. Copia `template/` in `skills/<your-skill>/`.
2. Riempi il frontmatter e il body. Mantieni `description` concreto, dato che è ciò che un agente usa per decidere quando caricare la skill.
3. Collega le skill correlate in fondo per la navigazione incrociata.
4. Apri una PR su [`veniceai/skills`](https://github.com/veniceai/skills).

Consulta il `CONTRIBUTING.md` del repository per le convenzioni di stile (primo paragrafo breve, tabelle esplicite degli endpoint, esempio curl + uno SDK, sezione "gotcha", ≤ 500 righe).

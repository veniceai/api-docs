Il [Venice MCP Server](https://github.com/veniceai/venice-mcp-server) è il server ufficiale [Model Context Protocol](https://modelcontextprotocol.io/) per Venice. Espone l'intera API Venice (chat, immagini, video, audio, musica, embedding, web augment e character) come **31 tool** che qualsiasi agente compatibile MCP può chiamare.

## Quick start

<Steps>
  <Step title="Ottieni una API key Venice">
    Generane una da [venice.ai/settings/api](https://venice.ai/settings/api). Consulta la [guida API key](/it/guides/getting-started/generating-api-key) per istruzioni passo passo.
  </Step>

  <Step title="Aggiungi Venice alla configurazione del tuo host MCP">
    Incolla questo nel file di configurazione del tuo host MCP:

    ```json
    {
      "mcpServers": {
        "venice": {
          "command": "npx",
          "args": ["-y", "@veniceai/mcp-server@0.2.0"],
          "env": { "VENICE_API_KEY": "<your-venice-api-key>" }
        }
      }
    }
    ```

    Percorsi comuni di configurazione:

    | Host | Percorso |
    |---|---|
    | Claude Desktop (macOS) | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | Claude Desktop (Windows) | `%APPDATA%\Claude\claude_desktop_config.json` |
    | Cursor | `~/.cursor/mcp.json` |
    | LM Studio | `mcp.json` (dalle impostazioni MCP dell'app) |
  </Step>

  <Step title="Riavvia il tuo host MCP">
    Il tuo agente ora ha disponibili chat, immagini, video, musica, TTS, ASR e altri 25 tool Venice.
  </Step>
</Steps>

<Note>
La maggior parte degli host MCP passa solo le variabili d'ambiente **esplicitamente elencate** nel blocco `env`. Le variabili d'ambiente di sistema non vengono ereditate. Se vedi errori 402 con una API key impostata, controlla che `VENICE_API_KEY` sia all'interno di `env` nella tua configurazione.
</Note>

## Cosa ottieni

**31 tool** che coprono ogni modalità Venice, **3 risorse** (`venice://models`, `venice://styles`, `venice://voices`) e **3 template di prompt**.

### Chat e embedding

| Tool | Descrizione |
|---|---|
| `venice_chat` | Chat completion compatibile OpenAI sull'intero catalogo LLM di Venice. |
| `venice_responses` | API Responses compatibile OpenAI con supporto a tool single- o multi-turn. |
| `venice_embeddings` | Calcola embedding per input testuale. |
| `venice_chat_with_character` | Chatta con un character Venice tramite slug. |

### Immagini

| Tool | Descrizione |
|---|---|
| `venice_image_generate` | Genera un'immagine (Flux 2, Lustify SDXL, Anime/WAI, Qwen Image, GPT Image, Nano Banana Pro e altri). |
| `venice_image_edit` | Modifica un'immagine con un prompt. |
| `venice_image_multi_edit` | Modifica più immagini insieme con un singolo prompt. |
| `venice_image_upscale` | Esegue upscale di un'immagine fino a 4×. |
| `venice_image_remove_bg` | Rimuove lo sfondo di un'immagine. |
| `venice_image_styles` | Elenca i preset di stile immagine. |

### Video

| Tool | Descrizione |
|---|---|
| `venice_video_generate` | Mette in coda una generazione video (Sora 2, Veo 3.1, Kling, Wan, LTX 2, Seedance, Runway Gen-4 e altri). |
| `venice_video_status` | Controlla lo stato di un job video in coda. |
| `venice_video_complete` | Marca un video completato come scaricato; elimina i media lato server. |
| `venice_video_transcriptions` | Trascrive un URL video di YouTube. |
| `venice_video_quote` | Ottieni un preventivo prima di mettere in coda. |

### Audio (TTS / ASR)

| Tool | Descrizione |
|---|---|
| `venice_tts` | Text-to-speech con voci clonate e tag emotivi. |
| `venice_asr` | Trascrivi audio da un URL. |
| `venice_voice_clone` | Elenca le voci integrate o clona una voce da un campione. |
| `venice_audio_quote` | Ottieni un preventivo per la generazione musicale. |

### Musica

| Tool | Descrizione |
|---|---|
| `venice_music_generate` | Mette in coda la generazione musicale (`ace-step-15`, `elevenlabs-music`, `minimax-music-v2/v25/v26`, `stable-audio-25`, `mmaudio-v2`, `elevenlabs-sound-effects-v2`). |
| `venice_music_status` | Controlla lo stato di un job musicale in coda. |
| `venice_music_complete` | Marca un job musicale completato come scaricato. |

### Web augment, catalogo e crypto

| Tool | Descrizione |
|---|---|
| `venice_web_search` | Cerca sul web (basato su Firecrawl). |
| `venice_web_scrape` | Scrapa un URL in markdown. |
| `venice_text_parser` | Estrae testo da PDF/DOCX/EPUB/PPTX/XLSX. |
| `venice_list_models` | Elenca il catalogo modelli live con i prezzi. |
| `venice_list_characters` | Elenca i character pubblici di Venice. |
| `venice_crypto_rpc` | Inoltra chiamate JSON-RPC verso Base, Ethereum, Polygon, Arbitrum o Optimism. |

### Helper wallet x402

Rilevanti solo se ti autentichi con un wallet tramite [x402](/it/guides/integrations/x402-venice-api) invece di una API key.

| Tool | Descrizione |
|---|---|
| `venice_x402_balance` | Verifica il saldo crediti x402 prepagato per un indirizzo wallet EVM o Solana. |
| `venice_x402_top_up_info` | Recupera i requisiti per il top-up (network, token USDC, receiver, importo minimo). |
| `venice_x402_transactions` | Elenca le recenti transazioni di top-up e debito x402 per un indirizzo wallet EVM o Solana. |

## Configurazione

Il server è configurato interamente tramite variabili d'ambiente.

| Variabile d'ambiente | Default | Note |
|---|---|---|
| `VENICE_API_KEY` | _(nessuno)_ | La tua API key Venice. La configurazione più semplice. |
| `VENICE_DEFAULT_CHAT_MODEL` | `venice-uncensored` | |
| `VENICE_DEFAULT_IMAGE_MODEL` | `flux-2-pro` | |
| `VENICE_DEFAULT_TTS_MODEL` | `tts-kokoro` | |
| `VENICE_DEFAULT_ASR_MODEL` | `openai/whisper-large-v3` | |
| `VENICE_DISABLE_NSFW` | `0` | Imposta a `1` per rimuovere le note di capacità NSFW dalle descrizioni dei tool. |
| `VENICE_HTTP_TIMEOUT_MS` | `60000` | |
| `VENICE_SIWX_TOKEN` | _(nessuno)_ | Token di autenticazione wallet-mode x402. Vedi [x402 sotto](#x402-wallet-mode). |

Se sono impostati sia `VENICE_API_KEY` sia `VENICE_SIWX_TOKEN`, vince l'API key.

## Modalità wallet x402

Venice supporta l'autenticazione con un [token wallet Sign-In-With-X](/it/guides/integrations/x402-venice-api) sostenuto da credito USDC prepagato su Base o Solana, oltre al normale flusso con API key. Niente email, telefono o KYC: il tuo wallet è l'unica identità.

```json
{
  "mcpServers": {
    "venice": {
      "command": "npx",
      "args": ["-y", "@veniceai/mcp-server@0.2.0"],
      "env": { "VENICE_SIWX_TOKEN": "<base64 Sign-In-With-X payload>" }
    }
  }
}
```

Il server MCP inoltra `VENICE_SIWX_TOKEN` come header `X-Sign-In-With-X` su ogni chiamata API Venice. Il server non vede mai la tua chiave privata. La firma del wallet e le autorizzazioni di top-up USDC avvengono nel tuo wallet.

| Flusso | Cosa succede |
|---|---|
| **Configurazione una tantum** | Firma un messaggio Sign-In-With-X nel tuo wallet → produce un token SIWX (JSON base64). |
| **Top up** | `POST /api/v1/x402/top-up` restituisce 402 + requisiti di pagamento. Firma un pagamento USDC per una delle opzioni Base o Solana restituite, reinvia, e Venice accredita il tuo saldo. |
| **Ogni chiamata di inferenza** | Il server MCP invia `X-Sign-In-With-X: <SIWX>`; Venice scala dal tuo saldo prepagato. |

Il top-up minimo è **\$5 USD**. Il saldo minimo per chiamare l'inferenza è **\$0.10**. Una volta ricaricato, le chiamate sono sub-100ms perché il settlement avviene off-chain su un veloce account di credito.

<Tip>
I wallet collegati a un account Venice con DIEM in staking consumano dal saldo di staking invece dei crediti USDC, quindi non serve top-up.
</Tip>

## Self-hosting (HTTP Streamable)

Per deployment di team o workspace, esegui il server MCP su HTTP invece di stdio:

```bash
docker run -p 3333:3333 \
  -e VENICE_API_KEY=<your-venice-api-key> \
  -e VENICE_MCP_AUTH_TOKEN=<choose-a-long-random-token> \
  ghcr.io/veniceai/venice-mcp-server:latest
```

Il server è ora disponibile su `http://localhost:3333/mcp`. I client HTTP devono inviare `Authorization: Bearer <VENICE_MCP_AUTH_TOKEN>`.

<Warning>
`/mcp` è un endpoint di esecuzione tool con credenziali: i chiamanti possono spendere l'API key Venice o il saldo x402 configurato. Quando la modalità HTTP si lega a un'interfaccia fuori da loopback, l'avvio fallisce se `VENICE_MCP_AUTH_TOKEN` non è impostato. Per la produzione, fissa esplicitamente la versione del pacchetto npm invece di affidarti a `latest`.
</Warning>

[NanoClaw](https://github.com/qwibitai/nanoclaw) è un assistente AI leggero e self-hosted che gira su WhatsApp e Telegram. Questo fork aggiunge il supporto a Venice AI in modo che tutto giri privatamente senza un abbonamento Anthropic.

## Perché Venice AI?

[Venice](https://venice.ai) è una piattaforma AI privacy-first. [Non memorizzano né registrano prompt o risposte](https://venice.ai/privacy) sui loro server — le tue conversazioni esistono solo sul tuo dispositivo. Le richieste sono criptate end-to-end attraverso il loro proxy verso provider GPU decentralizzati, con zero data retention. Questo significa che le conversazioni del tuo assistente AI restano private, anche da Venice stessa.

Venice fornisce accesso anonimizzato ai modelli frontier (Claude Opus, Claude Sonnet) e accesso completamente privato ai modelli open-source (GLM, Qwen) tramite un'unica API — passa tra loro in qualsiasi momento.

| | **Venice AI** | **Provider AI tradizionali** |
|---|---|---|
| **Data retention** | Nessuna — zero log | Sì |
| **Privacy dei prompt** | Criptati, mai memorizzati | Memorizzati sui server del provider |
| **Modelli open-source** | Sì (GLM, Qwen e altri) | No |
| **Modelli frontier** | Claude, GPT e altri — anonimamente | Solo tramite abbonamenti diretti |
| **Pricing** | Pay-per-token, niente abbonamento. Oppure stake [DIEM](https://venice.ai/lp/diem) per crediti che si rinnovano giornalmente | Abbonamenti $20–200/mese o API pay-per-token |
| **Inferenza senza censura** | Sì (modelli open-source) | No |

## Perché NanoClaw?

NanoClaw è un'alternativa pulita e minimale a piattaforme più grandi come OpenClaw. È progettato per una persona che gestisce un bot.

| | **NanoClaw (Venice)** | **OpenClaw** |
|---|---|---|
| **Codebase** | ~2.000 righe, una manciata di file | ~500.000 righe, 53 file di config |
| **Dipendenze** | ~15 pacchetti | 70+ pacchetti |
| **Modello di sicurezza** | Isolamento container Docker a livello OS | Allowlist a livello applicazione e codici di pairing |
| **Isolamento per gruppo** | Ogni gruppo ottiene il proprio container, filesystem e memoria | Processo condiviso, memoria condivisa |
| **Setup** | Una procedura guidata (`/setup`), ~10 minuti | Configurazione manuale multi-step |
| **Provider AI** | Venice AI (privato, niente abbonamento) | Anthropic (richiede API key o abbonamento) |
| **Personalizzazione** | Modifica direttamente il codice — è abbastanza piccolo da leggere | File di config e plugin |
| **Utente target** | Una persona, un bot | Piattaforma multi-utente |

## Cosa ottieni

- Assistente AI personale su **Telegram** e/o **WhatsApp**
- Alimentato da **Venice AI** — niente account Anthropic
- Il bot gira in un **container Docker isolato** (sandbox, non può accedere al tuo sistema)
- **Cambio modello** — dì al bot "switch to zai-org-glm-5" o "use opus" in qualsiasi momento
- **Task pianificati** — imposta promemoria, task ricorrenti
- **Ricerca web e browsing** integrati
- **Formattazione Markdown** nei messaggi Telegram

**Per Telegram** (raccomandato per utenti alle prime armi):
1. Apri Telegram e cerca **@BotFather**
2. Invia `/newbot` e segui le istruzioni
3. Salva il token che BotFather ti dà (assomiglia a `123456789:ABCdef...`)

<Warning>
**Per WhatsApp — usa un numero virtuale, NON il tuo personale:**

NanoClaw si connette come dispositivo collegato sul tuo numero WhatsApp. Questo significa che **l'agente può vedere ogni messaggio in entrata e in uscita** — tutte le tue conversazioni personali, chat di gruppo, foto, tutto. Il tuo telefono funziona ancora normalmente, ma il bot ha piena visibilità sul tuo intero account WhatsApp.

**Usa invece un numero di telefono virtuale.** Queste app ti danno un secondo numero che puoi dedicare interamente al bot:

| App | Prezzo | Note |
|-----|-------|-------|
| [Hushed](https://hushed.com) | ~$5/mese | Affidabile, funziona bene per la verifica WhatsApp |
| [Burner](https://www.burnerapp.com) | ~$5/mese | Simile a Hushed, numeri usa e getta |
| [Google Voice](https://voice.google.com) | Gratis | Solo USA, potrebbe non funzionare per la verifica WhatsApp in tutti i casi |

**Come configurarlo:**
1. Ottieni un numero virtuale da una delle app sopra
2. Installa WhatsApp su un secondo dispositivo (vecchio telefono, tablet o emulatore) usando quel numero virtuale
3. Durante la configurazione di NanoClaw, scansiona il QR code con quel secondo dispositivo — non con il tuo telefono personale
</Warning>

## Configurazione

Il setup richiede circa 10 minuti. Ti serve solo **una finestra di Terminale**.

<Steps>
  <Step title="Clona e installa">
    Apri Terminale ed esegui:

    ```bash
    git clone https://github.com/lorenzovenice/nanoclaw-venice.git
    cd nanoclaw-venice
    npm install
    ```

    Aspetta che `npm install` finisca senza errori.
  </Step>

  <Step title="Avvia Claude Code con Venice">
    Sostituisci `your-key` con la tua API key Venice ed esegui:

    ```bash
    VENICE_API_KEY=your-key npm run venice
    ```

    Questo avvia il proxy Venice e lancia Claude Code attraverso di esso in un unico comando.

    <Note>
    Claude Code passa di default a **GLM 5** (`zai-org-glm-5`) per mantenere bassi i costi di setup. Dopo il setup, digita `/model` dentro Claude Code per passare a `claude-sonnet-4-6` o `claude-opus-4-6` per le migliori prestazioni.
    </Note>

    Se richiesto "Do you want to use this API key?" — seleziona **Yes**.
  </Step>

  <Step title="Esegui la procedura guidata di setup">
    Nel tuo terminale Claude Code, digita:

    ```
    /setup
    ```

    La procedura guidata ti accompagna attraverso:

    1. **Bootstrap** — controlla Node.js e dipendenze
    2. **API key Venice** — valida e salva la tua key
    3. **Scelta canale** — scegli WhatsApp, Telegram o entrambi
    4. **Build container** — costruisce il container Docker (impiega qualche minuto la prima volta)
    5. **Auth WhatsApp** — scansiona il QR code con il telefono (se applicabile)
    6. **Setup Telegram** — invia un messaggio al tuo bot in modo che rilevi la tua chat
    7. **Trigger word** — prefisso che attiva il bot (default: `@Andy`)
    8. **Directory di mount** — scegli "No" per ora (puoi aggiungere l'accesso ai file più tardi)
    9. **Avvia servizi** — NanoClaw e il proxy Venice si avviano entrambi come servizi in background

    La procedura guidata installa due servizi in background:
    - **NanoClaw** — il bot stesso
    - **Proxy Venice** — un piccolo server locale (localhost:4001) che traduce tra Claude Code e Venice AI

    Entrambi si avviano automaticamente al boot e si riavviano da soli se crashano.

    <Note>
    Se la procedura guidata si ferma tra i passi, digita "continue" o "next step" per spingerla avanti.
    </Note>
  </Step>

  <Step title="Inizia a chattare">
    Una volta completato il setup, apri la tua chat (Telegram o WhatsApp) e invia:

    ```
    @Andy hello, are you there?
    ```

    Il bot dovrebbe rispondere entro pochi secondi. Nel tuo canale principale, puoi digitare normalmente senza il prefisso `@Andy`.

    **Ora puoi chiudere la finestra del terminale.** Tutto gira come servizi in background e si avvia automaticamente quando il tuo computer si avvia.
  </Step>
</Steps>

## Come funziona

Ci sono due livelli in NanoClaw:

| Livello | Cosa fa |
|-------|-------------|
| **Claude Code CLI** | Strumento admin per setup, debugging e personalizzazione |
| **Il bot** | AI nella tua chat, in esecuzione dentro un container Docker isolato |

Per aprire Claude Code in qualsiasi momento:

```bash
cd nanoclaw-venice
ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
```

Usalo per eseguire `/setup`, `/debug`, `/customize` o fare modifiche al comportamento del bot.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Il proxy non è in esecuzione">
    Il proxy Venice gira come servizio in background e si riavvia automaticamente. Se non funziona:

    **macOS:**
    ```bash
    # Check if it's running
    launchctl list | grep venice-proxy

    # Restart it
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Check logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```

    **Linux:**
    ```bash
    # Check if it's running
    systemctl --user status nanoclaw-venice-proxy

    # Restart it
    systemctl --user restart nanoclaw-venice-proxy

    # Check logs
    tail -f ~/nanoclaw-venice/logs/venice-proxy.log
    ```
  </Accordion>

  <Accordion title="Claude Code mostra errore 403 o 'Please run /login'">
    Significa che Claude Code non può connettersi al proxy Venice.

    1. **Controlla che il proxy sia in esecuzione.** Vedi il passo di troubleshooting sopra.
    2. **Assicurati di essere nella cartella giusta.** Sempre `cd nanoclaw-venice` prima.
    3. **Riparti da zero:** Chiudi tutti i terminali ed esegui:
       ```bash
       cd nanoclaw-venice
       ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
       ```
  </Accordion>

  <Accordion title="Errori di modello ('model does not exist')">
    Riavvia il proxy e il bot:

    **macOS:**
    ```bash
    # Restart proxy
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy

    # Restart bot
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw
    ```

    **Linux:**
    ```bash
    # Restart proxy
    systemctl --user restart nanoclaw-venice-proxy

    # Restart bot
    systemctl --user restart nanoclaw
    ```

    Controlla i modelli disponibili nel [catalogo modelli](/it/models/text).
  </Accordion>

  <Accordion title="Il bot non risponde ai messaggi">
    Lavora attraverso questi passi in ordine:

    1. **Controlla la tua trigger word.** Assicurati di usare il prefisso giusto (es. `@Andy hello`).
    2. **Controlla che Docker sia in esecuzione.** Esegui `docker info` — se va in errore, apri Docker Desktop.
    3. **Controlla che il proxy sia in esecuzione.** Vedi "Il proxy non è in esecuzione" sopra.
    4. **Controlla i log:** `tail -f logs/nanoclaw.log` nella cartella del progetto.
    5. **Controlla i log del container.** Apri la cartella `nanoclaw-venice/groups/main/logs/`. Apri il file più recente che inizia con `container-`.
    6. **Riavvia tutto:** Riavvia sia proxy sia bot (vedi sopra).
  </Accordion>

  <Accordion title="Il build del container fallisce durante il setup">
    Assicurati che Docker Desktop sia aperto e in esecuzione. Aspetta 10 secondi che Docker si avvii completamente, poi digita `continue` nella procedura guidata per ritentare.
  </Accordion>

  <Accordion title="WhatsApp disconnesso">
    La tua sessione WhatsApp può scadere. Per riconnetterti:

    ```bash
    cd nanoclaw-venice
    npm run auth
    ```

    Scansiona il QR code con WhatsApp (Settings → Linked Devices → Link a Device), poi riavvia il bot:
    - macOS: `launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
    - Linux: `systemctl --user restart nanoclaw`
  </Accordion>
</AccordionGroup>

## Avanzato

<AccordionGroup>
  <Accordion title="Dai al bot accesso ai file sul tuo computer">
    Di default, il bot è completamente isolato dal tuo computer — può vedere solo la propria memoria e cronologia conversazionale.

    - **Durante il setup:** Quando ti viene chiesto sull'accesso alle directory, scegli "Yes"
    - **Dopo il setup:** Esegui `/customize` in Claude Code
  </Accordion>

  <Accordion title="Avvia/ferma manualmente il bot">
    NanoClaw esegue due servizi in background che si avviano automaticamente al boot.

    **macOS:**
    | Azione | Comando |
    |--------|---------|
    | Avvia bot | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Ferma bot | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist` |
    | Riavvia bot | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw` |
    | Avvia proxy | `launchctl load ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Ferma proxy | `launchctl unload ~/Library/LaunchAgents/com.nanoclaw.venice-proxy.plist` |
    | Riavvia proxy | `launchctl kickstart -k gui/$(id -u)/com.nanoclaw.venice-proxy` |

    **Linux:**
    | Azione | Comando |
    |--------|---------|
    | Avvia bot | `systemctl --user start nanoclaw` |
    | Ferma bot | `systemctl --user stop nanoclaw` |
    | Riavvia bot | `systemctl --user restart nanoclaw` |
    | Avvia proxy | `systemctl --user start nanoclaw-venice-proxy` |
    | Ferma proxy | `systemctl --user stop nanoclaw-venice-proxy` |
    | Riavvia proxy | `systemctl --user restart nanoclaw-venice-proxy` |
  </Accordion>

  <Accordion title="Usare Claude Code tramite Venice (senza bot)">
    Se vuoi solo Claude Code con Venice e non hai bisogno di WhatsApp/Telegram, il servizio proxy deve essere in esecuzione. Se hai già eseguito `/setup`, sta già girando come servizio in background.

    ```bash
    cd nanoclaw-venice
    ANTHROPIC_BASE_URL=http://localhost:4001 ANTHROPIC_API_KEY=venice-proxy claude
    ```

    **Consiglio:** Aggiungi questo al tuo `~/.zshrc` (o `~/.bashrc`) così puoi velocemente passare qualsiasi terminale a Venice:

    ```bash
    alias venice='export ANTHROPIC_BASE_URL=http://localhost:4001 && export ANTHROPIC_API_KEY=venice-proxy && echo "Using Venice API"'
    alias anthropic='unset ANTHROPIC_BASE_URL && unset ANTHROPIC_API_KEY && echo "Using Anthropic API"'
    ```

    Poi basta digitare `venice` in qualsiasi terminale prima di eseguire `claude` per usare Venice, o `anthropic` per tornare indietro.
  </Accordion>

  <Accordion title="Eseguire più bot">
    Puoi eseguire più bot NanoClaw sulla stessa macchina (ad esempio, uno per uso personale e uno per un team). Basta clonare il repo in una cartella diversa ed eseguire di nuovo il setup. Nota: condividono la stessa immagine Docker, quindi ricostruirne uno influisce su tutti loro.
  </Accordion>

  <Accordion title="Comandi per sviluppatori">
    Per chi vuole modificare il codice di NanoClaw:

    ```bash
    npm run dev          # Start proxy + NanoClaw with hot reload
    npm run proxy        # Start just the Venice proxy
    npm run build        # Compile TypeScript
    npm test             # Run tests
    ./container/build.sh # Rebuild agent container
    ```
  </Accordion>
</AccordionGroup>

## Architettura

```
You (WhatsApp/Telegram)
        ↓
   NanoClaw (Node.js)
        ↓
   Docker Container (isolated sandbox)
        ↓
   Venice Proxy (localhost:4001)
        ↓
   api.venice.ai (private inference)
```

| File | Scopo |
|------|---------|
| `proxy/venice-proxy.ts` | Traduce il formato Anthropic in formato OpenAI per Venice |
| `src/index.ts` | Orchestrator principale — loop dei messaggi, invocazione agente |
| `src/channels/whatsapp.ts` | Connessione WhatsApp via baileys |
| `src/channels/telegram.ts` | Bot Telegram via grammy |
| `src/container-runner.ts` | Genera container agente isolati |

## FAQ

<AccordionGroup>
  <Accordion title="Perché ho bisogno di un proxy?">
    Il Claude Agent SDK parla il formato messaggi di Anthropic. Venice parla il formato di OpenAI. Il proxy traduce tra loro così tutto funziona senza modificare l'SDK.
  </Accordion>

  <Accordion title="Posso usare modelli open-source?">
    Sì. Venice ospita molti modelli. Dì al bot "switch to zai-org-glm-5" o qualsiasi ID modello Venice. Vedi il [catalogo modelli](/it/models/text).
  </Accordion>

  <Accordion title="È sicuro?">
    Gli agenti girano in container Docker con vero isolamento a livello OS. L'API key Venice viene passata via stdin, mai scritta su disco dentro i container. Ogni gruppo ottiene il proprio ambiente isolato.
  </Accordion>

  <Accordion title="Ho bisogno di un abbonamento Anthropic?">
    No. Tutto gira attraverso Venice AI. Hai bisogno solo di una API key Venice.
  </Accordion>

  <Accordion title="Posso usarlo su un server?">
    Sì. Funziona su qualsiasi macchina Linux con Docker. Usa il servizio systemd per auto-start al boot.
  </Accordion>
</AccordionGroup>

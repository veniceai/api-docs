## Perché ti serve un router

Claude Code si connette di default direttamente all'API di Anthropic. Per usarlo con Venice, hai bisogno di [claude-code-router](https://github.com/musistudio/claude-code-router), un proxy locale open-source che:

<Steps>
  <Step title="Intercetta" icon="hand">
    Cattura le richieste in uscita di Claude Code prima che raggiungano Anthropic
  </Step>
  <Step title="Trasforma" icon="arrows-rotate">
    Converte il formato della richiesta e mappa gli ID dei modelli (es. `claude-opus-4-5`)
  </Step>
  <Step title="Reindirizza" icon="route">
    Inoltra le richieste a Venice su `api.venice.ai/api/v1/chat/completions`
  </Step>
</Steps>

## Configurazione

<Steps>
  <Step title="Installa Claude Code">
    Se non l'hai ancora fatto, installa il CLI Claude Code di Anthropic:

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="Installa il router">
    ```bash
    npm install -g @musistudio/claude-code-router
    ```
  </Step>

  <Step title="Ottieni la tua API key">
    Genera una key da [venice.ai/settings/api](https://venice.ai/settings/api). La incollerai direttamente nel file di configurazione nel prossimo passo.
  </Step>

  <Step title="Crea la configurazione">
    Crea la directory di configurazione:

    ```bash
    mkdir -p ~/.claude-code-router
    ```

    Poi crea `~/.claude-code-router/config.json` con il tuo editor preferito:

    ```bash
    # Using nano
    nano ~/.claude-code-router/config.json

    # Or using VS Code
    code ~/.claude-code-router/config.json
    ```

    Incolla la seguente configurazione:

    ```json
    {
      "APIKEY": "",
      "LOG": true,
      "LOG_LEVEL": "info",
      "API_TIMEOUT_MS": 600000,
      "HOST": "127.0.0.1",
      "Providers": [
        {
          "name": "venice",
          "api_base_url": "https://api.venice.ai/api/v1/chat/completions",
          "api_key": "your-venice-api-key-here",
          "models": [
            "claude-opus-4-5",
            "claude-sonnet-4-5",
            "claude-opus-4-6",
            "claude-opus-4-6-fast",
            "claude-sonnet-4-6"
          ],
          "transformer": {
            "use": ["anthropic"]
          }
        }
      ],
      "Router": {
        "default": "venice,claude-opus-4-5",
        "think": "venice,claude-opus-4-5",
        "background": "venice,claude-opus-4-5",
        "longContext": "venice,claude-opus-4-5",
        "longContextThreshold": 100000
      }
    }
    ```

    <Note>
    Se modifichi `config.json` mentre il router è in esecuzione, riavvialo con `ccr restart` per applicare le modifiche.
    </Note>
  </Step>

  <Step title="Avvia">
    Avvia il router, poi Claude Code:

    ```bash
    ccr start
    ccr code
    ```

    Oppure usa il metodo di attivazione:

    ```bash
    eval "$(ccr activate)" && claude
    ```
  </Step>
</Steps>

## Modelli supportati

| Modello | ID Venice | Ideale per |
|-------|-----------|----------|
| Claude Opus 4.5 | `claude-opus-4-5` | Ragionamento complesso, refactor di grandi dimensioni |
| Claude Sonnet 4.5 | `claude-sonnet-4-5` | Iterazione veloce, coding quotidiano |
| Claude Opus 4.6 | `claude-opus-4-6` | Ragionamento complesso, refactor di grandi dimensioni |
| Claude Opus 4.6 Fast | `claude-opus-4-6-fast` | Ragionamento complesso con latenza più bassa |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | Iterazione veloce, coding quotidiano |

<Info>
Claude Code è ottimizzato per i modelli Claude. Sebbene gli altri modelli disponibili tramite Venice (GPT, DeepSeek, Grok, ecc.) possano funzionare, non possiamo garantire un'esperienza equivalente dato che Claude Code si basa su funzionalità specifiche di Claude come extended thinking. Per altri modelli, considera l'uso dell'[API standard](/it/api-reference/endpoint/chat/completions) di Venice.
</Info>

## Funzionalità del router

Il router fornisce diverse funzionalità utili oltre al routing di base:

<AccordionGroup>
  <Accordion title="Cambia modello al volo">
    Usa il comando `/model` all'interno di Claude Code per cambiare modello senza riavviare:
    
    ```
    /model venice,claude-sonnet-4-5
    ```
    
    Utile quando vuoi Opus per task complessi e Sonnet per iterazioni veloci.
  </Accordion>

  <Accordion title="Configurazione visiva con modalità UI">
    Preferisci una GUI? Avvia l'editor di configurazione basato sul web:
    
    ```bash
    ccr ui
    ```
    
    Questo apre un'interfaccia browser per modificare il tuo `config.json` senza toccare direttamente il file.
  </Accordion>

  <Accordion title="Scenari del router spiegati">
    La sezione `Router` della configurazione controlla quale modello gestisce i diversi tipi di task:
    
    | Scenario | Quando viene usato |
    |----------|----------------|
    | `default` | Richieste generali |
    | `think` | Task ad alta intensità di ragionamento (Plan Mode) |
    | `background` | Operazioni in background |
    | `longContext` | Quando il contesto supera i token di `longContextThreshold` |
    
    Puoi instradare diversi scenari verso diversi modelli. Ad esempio, usa Sonnet per task in background per risparmiare costi.
  </Accordion>

  <Accordion title="Debugging con i log">
    Se qualcosa non funziona, controlla i log:
    
    ```bash
    # Server logs (HTTP, API calls)
    ~/.claude-code-router/logs/ccr-*.log
    
    # Application logs (routing decisions)
    ~/.claude-code-router/claude-code-router.log
    ```
    
    Imposta `"LOG_LEVEL": "debug"` nella tua configurazione per un output più verboso.
  </Accordion>
</AccordionGroup>

## Comportamento del caching

Il [prompt caching](/it/guides/features/prompt-caching) di Venice funziona insieme ai marcatori cache nativi di Claude Code. Venice rileva automaticamente quando Claude Code invia campi `cache_control` e adatta di conseguenza la sua strategia di caching.

| Scenario | TTL cache | Chi controlla |
|----------|-----------|--------------|
| Default (raccomandato) | 5 minuti | Claude Code + Venice |
| Con il transformer `cleancache` | 1 ora | Solo Venice |

<AccordionGroup>
  <Accordion title="Quando NON usare cleancache (la maggior parte degli utenti)">
    La configurazione predefinita permette ai due sistemi di cooperare:
    
    - Claude Code invia i suoi marcatori `cache_control` nativi
    - Venice aggiunge caching attorno a essi con un TTL di 5 minuti
    - Entrambi i sistemi condividono il limite di cache di 4 blocchi
    
    Funziona bene per sessioni di coding attive in cui fai richieste frequenti.
  </Accordion>

  <Accordion title="Quando usare cleancache">
    Aggiungi `cleancache` al transformer se:
    
    - Stai incontrando errori di limite di 4 blocchi cache
    - Sperimenti comportamenti strani della cache
    - Preferisci il TTL di 1 ora di Venice per sessioni più lunghe
    
    ```json
    "transformer": {
      "use": ["anthropic", "cleancache"]
    }
    ```
    
    Questo rimuove i marcatori cache di Claude Code, dando a Venice il pieno controllo con un TTL più lungo.
  </Accordion>
</AccordionGroup>

[Jan](https://jan.ai/) è un assistente AI open-source per eseguire modelli locali e remoti da un'unica app desktop. Venice funziona in Jan grazie al suo supporto al provider di modello remoto compatibile OpenAI.

## Prerequisiti

- Una API key Venice da [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) installato e in esecuzione sulla tua macchina
- Un ID modello di testo Venice dal [catalogo modelli](/it/models/text)

---

## Configurazione

<Steps>
  <Step title="Ottieni la tua API key Venice">
    Genera o copia una API key da [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Apri le impostazioni dei provider di modelli di Jan">
    In Jan, apri **Settings** e vai su **Model Providers**.

    Scegli il flusso di provider personalizzato compatibile OpenAI di Jan. A seconda della tua versione di Jan, potrebbe apparire come **OpenAI** con **Custom Endpoint** abilitato, o come provider **Any OpenAI Compatible API**.
  </Step>

  <Step title="Configura l'endpoint Venice">
    Inserisci le impostazioni del provider Venice:

    | Campo | Valore |
    |-------|-------|
    | Nome provider | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | La tua API key Venice |
    | Tipo API | OpenAI-compatible |

    Salva o applica le impostazioni del provider prima di aggiungere i modelli.
  </Step>

  <Step title="Aggiungi i modelli Venice">
    Aggiungi gli ID modello Venice che vuoi usare. Jan richiede che l'ID modello corrisponda esattamente al provider.

    Buone opzioni di partenza:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Puoi aggiungere qualsiasi modello di testo supportato dal [catalogo modelli](/it/models/text).
  </Step>

  <Step title="Avvia una chat">
    Apri una chat Jan nuova o esistente, seleziona uno dei tuoi modelli Venice dal selettore di modello e invia un messaggio di test.

    Jan invierà la richiesta a Venice usando la tua API key e la base URL compatibile OpenAI.
  </Step>
</Steps>

---

## Verifica la configurazione

Invia un prompt breve da Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

Se il provider è configurato correttamente, Jan dovrebbe trasmettere in streaming una risposta dal modello Venice selezionato.

Puoi anche verificare la tua API key al di fuori di Jan con:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Note di configurazione

- Usa `https://api.venice.ai/api/v1` come base URL. Non includere `/chat/completions`; Jan aggiunge il path dell'endpoint per le richieste di chat.
- Gli ID modello devono corrispondere esattamente a Venice. Se un modello non appare nel selettore di Jan, aggiungilo manualmente usando l'ID modello dal [catalogo modelli](/it/models/text).
- Mantieni Venice configurato come provider compatibile OpenAI. Non usare il provider Anthropic nativo di Jan per i modelli Claude di Venice.
- Le richieste remote a Venice richiedono una connessione internet e consumano crediti API Venice.

<Info>
I modelli locali di Jan restano disponibili dopo aver aggiunto Venice. Puoi passare tra modelli locali e Venice dal selettore di modello nella chat.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Errore 401 o di autenticazione">
    Controlla che la API key sia una key Venice da [venice.ai/settings/api](https://venice.ai/settings/api), senza spazi prima o dopo.
  </Accordion>

  <Accordion title="404 o modello non trovato">
    Conferma che l'ID modello in Jan corrisponda esattamente a un ID modello Venice. Verifica anche che la base URL sia `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="Nessun modello Venice nel selettore">
    Aggiungi il modello manualmente nelle impostazioni del provider in Jan. Usa l'ID modello esatto, ad esempio `zai-org-glm-5-1`, e salva di nuovo il provider.
  </Accordion>

  <Accordion title="Le richieste finiscono ancora su un altro provider">
    Assicurati che la chat stia usando un modello collegato al tuo provider Venice. Se Jan sta usando una voce di modello OpenAI o Anthropic integrata, passa al modello Venice che hai aggiunto.
  </Accordion>
</AccordionGroup>

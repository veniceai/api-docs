[Jan](https://jan.ai/) è un assistente AI open source per eseguire modelli locali e remoti da un'unica app desktop. Venice funziona in Jan grazie al supporto per provider di modelli remoti compatibili con OpenAI.

## Prerequisiti

- Una API key Venice da [venice.ai/settings/api](https://venice.ai/settings/api)
- [Jan](https://jan.ai/) installato e in esecuzione sulla tua macchina
- Un ID di modello di testo Venice dal [catalogo modelli](/models/text)

---

## Configurazione

<Steps>
  <Step title="Ottieni la tua API key Venice">
    Genera o copia una API key da [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Apri le impostazioni del provider di modelli di Jan">
    In Jan, apri **Settings** e vai su **Model Providers**.

    Scegli il flusso del provider personalizzato compatibile con OpenAI di Jan. A seconda della tua versione di Jan, potrebbe apparire come **OpenAI** con **Custom Endpoint** abilitato, o come provider **Any OpenAI Compatible API**.
  </Step>

  <Step title="Configura l'endpoint Venice">
    Inserisci le impostazioni del provider Venice:

    | Campo | Valore |
    |-------|-------|
    | Provider name | `Venice AI` |
    | Base URL | `https://api.venice.ai/api/v1` |
    | API key | La tua API key Venice |
    | API type | OpenAI-compatible |

    Salva o applica le impostazioni del provider prima di aggiungere i modelli.
  </Step>

  <Step title="Aggiungi i modelli Venice">
    Aggiungi gli ID dei modelli Venice che vuoi usare. Jan richiede che l'ID del modello corrisponda esattamente al provider.

    Buone opzioni di partenza:

    - `zai-org-glm-5-1`
    - `kimi-k2-5`
    - `minimax-m25`

    Puoi aggiungere qualsiasi modello di testo supportato dal [catalogo modelli](/models/text).
  </Step>

  <Step title="Avvia una chat">
    Apri una chat Jan nuova o esistente, seleziona uno dei tuoi modelli Venice dal selettore di modelli e invia un messaggio di prova.

    Jan invierà la richiesta a Venice usando la tua API key e il base URL compatibile con OpenAI.
  </Step>
</Steps>

---

## Verifica la configurazione

Invia un breve prompt da Jan:

```txt
Reply with one sentence confirming you are running through Venice.
```

Se il provider è configurato correttamente, Jan dovrebbe trasmettere in streaming una risposta dal modello Venice selezionato.

Puoi anche verificare la tua API key al di fuori di Jan con:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

---

## Note di configurazione

- Usa `https://api.venice.ai/api/v1` come base URL. Non includere `/chat/completions`; Jan aggiunge il path dell'endpoint per le richieste chat.
- Gli ID dei modelli devono corrispondere esattamente a Venice. Se un modello non appare nel selettore di Jan, aggiungilo manualmente usando l'ID del modello dal [catalogo modelli](/models/text).
- Mantieni Venice configurato come provider compatibile con OpenAI. Non usare il provider nativo Anthropic di Jan per i modelli Claude di Venice.
- Le richieste remote a Venice richiedono una connessione a internet e consumano crediti API Venice.

<Info>
I modelli locali di Jan rimangono disponibili dopo aver aggiunto Venice. Puoi passare tra modelli locali e Venice dal selettore di modelli della chat.
</Info>

## Risoluzione problemi

<AccordionGroup>
  <Accordion title="Errore 401 o di autenticazione">
    Controlla che la API key sia una chiave Venice da [venice.ai/settings/api](https://venice.ai/settings/api), senza spazi extra prima o dopo.
  </Accordion>

  <Accordion title="404 o modello non trovato">
    Conferma che l'ID del modello in Jan corrisponda esattamente a un ID di modello Venice. Verifica anche che il base URL sia `https://api.venice.ai/api/v1`.
  </Accordion>

  <Accordion title="Nessun modello Venice nel selettore">
    Aggiungi il modello manualmente nelle impostazioni del provider di Jan. Usa l'ID esatto del modello, come `zai-org-glm-5-1`, e salva di nuovo il provider.
  </Accordion>

  <Accordion title="Le richieste vanno ancora a un altro provider">
    Assicurati che la chat stia usando un modello collegato al tuo provider Venice. Se Jan sta usando una voce di modello integrata di OpenAI o Anthropic, passa al modello Venice che hai aggiunto.
  </Accordion>
</AccordionGroup>

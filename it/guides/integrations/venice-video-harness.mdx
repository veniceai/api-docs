[Venice Video Harness](https://github.com/jordanurbs/venice-video-harness) è un toolkit community, agent-first, ottimizzato per Venice per la **creazione video consistency-first di qualsiasi lunghezza**. Trasforma un agente IDE (Claude Code, Cursor, Codex, ecc.) in un operatore di un sistema di produzione Venice riutilizzabile che copre 50+ modelli video, immagini, audio e musica di Venice.

## Cos'è questo

La maggior parte delle integrazioni Venice sono sottili wrapper attorno alle chiamate API. Venice Video Harness è il **layer di livello superiore** che si frappone tra il tuo agente e l'API Venice:

- **Regole di orchestrazione** in `CLAUDE.md`
- **Playbook riutilizzabili** in `.claude/commands/` (19 comandi di workflow)
- **Agenti specializzati** in `.claude/agents/` (art-director, prompt-engineer, cut-qa e altri)
- **Skill di produzione Venice** in `.claude/skills/` (compatibili con il formato [Agent Skills](/it/guides/integrations/venice-skills))
- **Layer di esecuzione TypeScript** in `src/`
- **Registry modelli completo** che copre 50+ modelli video, immagini, audio e musica Venice

Costruito per creator che producono:

- Progetti video character-consistent (qualsiasi genere, qualsiasi lunghezza)
- Serie o campagne con stile visivo bloccato
- Workflow storyboard-to-video
- Contenuti narrativi short-form e long-form
- Sequenze cinematiche brandizzate, trailer e teaser
- Serie social con personaggi ricorrenti

## Cosa è Venice-ottimizzato

- **Prompt immagine sintonizzati per i modelli immagine Venice** come `seedream-v5-lite`, `nano-banana-pro`, `flux-2-pro/max` e altri
- **Generazione panel a due passaggi** con raffinamento multi-edit Venice per la correzione dei personaggi
- **Logica di model routing** per fasce di azione, atmosfera e character-consistency
- **Generazione video reference-aware** che usa correttamente `elements`, `reference_image_urls` e `scene_image_urls` per modello
- **Adattamento prompt environment-aware** per la gestione di scene diurne vs notturne
- **Percorsi audio nativi Venice** per TTS (Kokoro, Qwen3, ElevenLabs), SFX e musica
- **Stima costi** prima della generazione tramite `/video/quote` e `/audio/quote`
- **Costruzione parametri model-aware** che salta automaticamente i parametri non supportati dal modello di destinazione

## Default del model routing

I default dell'harness sono opinionati perché la consistenza è il punto. Il routing corrente (aprile 2026):

**Seedance 2.0 R2V di default. Fallback Kling O3 R2V per scene con 3+ personaggi. Seedance 2.0 i2v per inquadrature di stabilimento.**

| Ruolo | Modello predefinito | Quando usato |
|---|---|---|
| Inquadrature di personaggio (1-2 personaggi) | `seedance-2-0-reference-to-video` | R2V di default con `reference_image_urls` flat, tag `@Image`, fino a 15s, audio stereo nativo |
| Inquadrature di personaggio (3+ personaggi) | `kling-o3-standard-reference-to-video` | Auto-fallback con `elements` strutturati per identità multi-personaggio |
| Stabilimento / mood / azione | `seedance-2-0-image-to-video` | Nessun personaggio; qualità cinematica epica, fino a 15s |

Sono sovrascrivibili per progetto via `series.json → videoDefaults`. Per mirare a una famiglia non-Seedance (es. account che non hanno accesso a Seedance), imposta `videoDefaults` su `kling-o3-standard-reference-to-video` e `veo3.1-fast-image-to-video`.

<Note>
**Regola volti Seedance:** Seedance 2.0 blocca le immagini di input contenenti volti che non sono state prodotte da `seedream-v5-lite` o `seedream-v5-lite-edit`. L'harness gestisce questo automaticamente instradando il lavoro su immagini con personaggi tramite Seedream ed eseguendo un gate pre-flight prima di ogni chiamata Seedance.
</Note>

## Pipeline di produzione

### Pipeline di generazione

Video narrativo end-to-end (script → storyboard → video → audio → assembly):

```bash
npm run dev -- produce-episode -p output/my-series -e 1
```

L'implementazione di riferimento in `src/mini-drama/` copre:

- Gestione di serie / personaggi / episodi
- Workshopping di script con LLM
- Generazione storyboard a due passaggi (generate + multi-edit refine)
- QA dei panel basato su vision
- Generazione video con frame chaining
- Post-produzione audio a livelli
- Burn-in dei sottotitoli e assembly finale

### Pipeline di editing

Taglia media già esistenti (inquadrature generate con Venice o footage raw reale). **Text-first**: l'LLM legge un compatto `takes_packed.md` (~12KB per 40 min di audio) invece di fare frame-dump del video.

I cinque passi:

<Steps>
  <Step title="Trascrivi">
    whisper.cpp locale produce per sorgente `*.words.json` + `takes_packed.md`.
  </Step>
  <Step title="Leggi il pack">
    L'LLM forma una strategia di taglio dal solo testo.
  </Step>
  <Step title="Conferma">
    Propone la strategia e aspetta "yes / revise / cancel".
  </Step>
  <Step title="Rendering dell'EDL">
    Cut list JSON → ffmpeg concat con audio fade di 30ms. Archive-first, così gli originali non vengono mai sovrascritti.
  </Step>
  <Step title="Self-eval">
    L'agente `cut-qa` esegue 6 controlli programmatici a ogni boundary di taglio; massimo 3 iterazioni di correzione.
  </Step>
</Steps>

I controlli `cut-qa` colgono regressioni di aspect ratio, salti di frame-hash dentro una parola, troncamento VO, discontinuità di illuminazione, picchi audio sopra -6 dBFS e sovrapposizione di didascalia con testo in-frame.

<Tip>
La pipeline di editing è ispirata a [browser-use/video-use](https://github.com/browser-use/video-use). La loro intuizione chiave, *"l'LLM non guarda mai il video, lo legge"*, è ciò che fa funzionare l'editing pilotato da agenti senza annegare in token di frame-dump.
</Tip>

## Comandi, agenti e skill

L'harness espone 19 comandi di workflow, 10 agenti specializzati e 7 skill di produzione. In evidenza:

| Comando di workflow | Scopo |
|---|---|
| `new-series` | Crea una nuova serie con estetiche bloccate |
| `add-character` / `lock-character` | Blocco personaggio + voce |
| `workshop-episode` | Scripting collaborativo di episodio |
| `storyboard-episode` | Storyboard di un episodio |
| `produce-episode` | Pipeline completa in un solo comando |
| `generate-trailer` | Pipeline completa per trailer |
| `edit-footage` | Pipeline di editing text-first per media esistenti |
| `ingest-screenplay` | Ingerisce una sceneggiatura Fountain o PDF |

| Agente specializzato | Ruolo |
|---|---|
| `art-director` | Decisioni di estetica, palette, illuminazione, composizione |
| `prompt-engineer` | Prompt immagine Venice, consistenza dei personaggi |
| `storyboard-qa` | QA dei panel per continuità e controlli dei personaggi |
| `cut-qa` | Gate di qualità post-render (6 controlli per taglio, max 3 iterazioni) |
| `overlay-designer` | Motion graphics brandizzate, sub-agenti paralleli |
| `trailer-curator` | Selezione inquadrature trailer e regole anti-spoiler |

| Skill di produzione | Scopo |
|---|---|
| `venice-api` | Utilizzo dell'API REST Venice e default |
| `venice-video-model-routing` | Routing R2V-first, decision tree |
| `character-consistency` | Guida alla consistenza dei personaggi multi-shot |
| `shot-composition` | Composizione dell'inquadratura e guida alla camera |
| `screenplay-parsing` | Workflow di parsing sceneggiatura |
| `video-editing` | Filosofia di editing text-first, formato EDL, loop cut-qa |

## NLE round-trip

Dopo il rendering, esporta la timeline assemblata come XML per il fine-tuning nel tuo editor di scelta. Ogni segmento video, clip di dialogo, clip SFX e cue musicale finisce sulla propria traccia.

```bash
mini-drama export-timeline -p output/<project> -e 1 --format fcpxml      # Final Cut Pro X
mini-drama export-timeline -p output/<project> -e 1 --format premiere    # Premiere Pro
mini-drama export-timeline -p output/<project> -e 1 --format davinci     # DaVinci Resolve
```

## Utilizzo programmatico

Puoi anche chiamare i moduli dell'harness direttamente dal tuo TypeScript:

```typescript
import { VeniceClient } from './src/venice/client.js';
import { generateVideo, quoteVideo } from './src/venice/video.js';
import { listVideoModels } from './src/venice/models.js';

const client = new VeniceClient();

const quote = await quoteVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  duration: '8s',
  audio: true,
});
console.log(`Estimated cost: $${quote.quote}`);

const result = await generateVideo(client, {
  model: 'kling-v3-pro-image-to-video',
  prompt: 'A slow dolly shot pushes forward...',
  duration: '8s',
  imageUrl: 'data:image/png;base64,...',
  audio: true,
  outputPath: 'output/shot-001.mp4',
});

const longModels = listVideoModels({ minDurationSec: 20 });
```

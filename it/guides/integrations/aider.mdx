## Prerequisiti

- Una API key Venice da [venice.ai/settings/api](https://venice.ai/settings/api)
- Python 3.9+ disponibile sulla tua macchina
- Un repository git in cui lavorare

## Configurazione

<Steps>
  <Step title="Installa Aider">
    Installa Aider con l'installer ufficiale:

    ```bash
    python -m pip install aider-install
    aider-install
    ```

    Consulta la [documentazione di installazione di Aider](https://aider.chat/docs/install.html) se preferisci un altro metodo di installazione.
  </Step>

  <Step title="Configura Venice come endpoint OpenAI">
    Punta Aider a Venice impostando il base URL compatibile con OpenAI e la tua API key Venice.

    **Mac / Linux:**

    ```bash
    export OPENAI_API_BASE=https://api.venice.ai/api/v1
    export OPENAI_API_KEY=your-venice-api-key
    ```

    **Windows (PowerShell):**

    ```powershell
    $env:OPENAI_API_BASE = "https://api.venice.ai/api/v1"
    $env:OPENAI_API_KEY = "your-venice-api-key"
    ```

    **Windows (Prompt dei comandi, persistente):**

    ```bat
    setx OPENAI_API_BASE https://api.venice.ai/api/v1
    setx OPENAI_API_KEY your-venice-api-key
    ```

    Riavvia il terminale dopo `setx` in modo che le nuove variabili siano disponibili.
  </Step>

  <Step title="Avvia Aider con un modello Venice">
    Dalla directory del tuo progetto, anteponi `openai/` all'ID del modello Venice:

    ```bash
    cd /to/your/project

    aider --model openai/zai-org-glm-5-1
    ```

    Altri buoni modelli per iniziare:

    ```bash
    aider --model openai/kimi-k2-5
    aider --model openai/minimax-m25
    aider --model openai/venice-uncensored
    ```

    Sfoglia tutti gli ID dei modelli di testo nel [catalogo dei modelli](/models/text).
  </Step>
</Steps>

## Persistere la configurazione con `.env`

Per una configurazione locale al progetto, crea un file `.env` nella root del tuo repo git:

```bash
OPENAI_API_BASE=https://api.venice.ai/api/v1
OPENAI_API_KEY=your-venice-api-key
AIDER_MODEL=openai/zai-org-glm-5-1
```

Poi avvia Aider normalmente:

```bash
aider
```

Aider legge anche `.env` dalla tua home directory. I valori nella root del repo hanno prioritГ  su quelli della home directory. Tieni questo file fuori da git вЂ” aggiungi `.env` a `.gitignore` se non ГЁ giГ  ignorato.

<Info>
Puoi anche impostare `AIDER_OPENAI_API_BASE` e `AIDER_OPENAI_API_KEY`. La documentazione ufficiale di Aider continua a documentare `OPENAI_API_BASE` e `OPENAI_API_KEY` per i provider compatibili con OpenAI.
</Info>

## Persistere la configurazione con YAML

Puoi memorizzare le stesse impostazioni in `.aider.conf.yml` nella root del repo o nella home directory:

```yaml
openai-api-base: https://api.venice.ai/api/v1
openai-api-key: your-venice-api-key
model: openai/zai-org-glm-5-1
```

Poi esegui:

```bash
aider
```

## Verifica la configurazione

Dopo l'avvio di Aider, chiedigli qualcosa di semplice che dimostri che il modello ГЁ raggiungibile:

```txt
Reply with one sentence confirming you are running through Venice.
```

Se l'endpoint e la chiave sono corretti, Aider trasmette in streaming una risposta dal modello Venice selezionato.

Puoi anche confermare la API key al di fuori di Aider:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

## Scelta dei modelli

| Caso d'uso                       | ID modello di esempio | Flag Aider                         |
| -------------------------------- | --------------------- | ---------------------------------- |
| Coding / refactoring performante | `zai-org-glm-5-1`     | `--model openai/zai-org-glm-5-1`   |
| Iterazione veloce                | `minimax-m25`         | `--model openai/minimax-m25`       |
| Coding con contesto lungo        | `kimi-k2-5`           | `--model openai/kimi-k2-5`         |
| Chat uncensored                  | `venice-uncensored`   | `--model openai/venice-uncensored` |

Anteponi sempre `openai/` agli ID dei modelli Venice quando avvii Aider. Senza quel prefisso, Aider non instraderГ  la richiesta attraverso il path del provider compatibile con OpenAI.

Scopri gli ID dei modelli attuali a runtime:

```bash
curl https://api.venice.ai/api/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

Oppure consulta [GET /models/traits](/api-reference/endpoint/models/traits) per i default come `text:default`.

## Note di configurazione

- Mantieni `OPENAI_API_BASE` impostato su `https://api.venice.ai/api/v1`. Non aggiungere `/chat/completions`; Aider aggiunge da sГ© il path della chat.
- Usa una API key Venice in `OPENAI_API_KEY`. Non usare una chiave OpenAI con il base URL di Venice.
- Aider potrebbe avvisare che il modello ha una context window o un costo sconosciuti. Questo avviso ГЁ previsto per molti ID di modelli Venice e di solito puГІ essere ignorato senza problemi. Consulta [gli avvisi sui modelli di Aider](https://aider.chat/docs/llms/warnings.html).
- Preferisci modelli capaci di coding/reasoning per modifiche multi-file. Modelli piГ№ piccoli o non orientati al coding possono avere difficoltГ  con i formati di edit di Aider.

<Warning>
Impostare `OPENAI_API_BASE` e `OPENAI_API_KEY` nella tua shell influisce anche su altri strumenti che leggono queste variabili. Preferisci un `.env` di progetto o un `.aider.conf.yml` se usi OpenAI altrove nello stesso ambiente.
</Warning>

## Risoluzione dei problemi

<AccordionGroup>
  <Accordion title="Errore 401 o di autenticazione">
    Verifica che `OPENAI_API_KEY` sia una chiave Venice da [venice.ai/settings/api](https://venice.ai/settings/api), senza spazi extra. Su Windows, dopo `setx`, riavvia il terminale.
  </Accordion>
  <Accordion title="Modello non trovato o risposta vuota">
    Verifica che l'ID del modello esista nel [catalogo dei modelli di testo](/models/text) e che tu abbia avviato Aider con il prefisso `openai/`, per esempio `openai/zai-org-glm-5-1`.
  </Accordion>
  <Accordion title="Le richieste vanno ancora a OpenAI">
    Controlla che `OPENAI_API_BASE` sia esattamente `https://api.venice.ai/api/v1` nella stessa sessione di shell. Stampalo con `echo $OPENAI_API_BASE` (o `$env:OPENAI_API_BASE` in PowerShell).
  </Accordion>
  <Accordion title="Avviso su context window / costi sconosciuti">
    Г€ previsto per molti modelli Venice. Aider ricade su default ragionevoli. Per silenziarlo, aggiungi i metadata del modello come descritto in [Aider advanced model settings](https://aider.chat/docs/config/adv-model-settings.html).
  </Accordion>
</AccordionGroup>

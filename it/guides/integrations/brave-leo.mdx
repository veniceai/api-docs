[Brave Leo](https://brave.com/leo/) è l'assistente AI integrato in Brave Browser. La funzione Bring Your Own Model (BYOM) di Leo può connettersi direttamente all'endpoint delle chat completions compatibile con OpenAI di Venice, permettendoti di usare i modelli Venice per la chat e per l'assistenza consapevole della pagina senza installare un'estensione.

## Prerequisiti

- Brave Browser 1.69 o versione successiva su Windows, macOS o Linux
- Una API key Venice da [venice.ai/settings/api](https://venice.ai/settings/api)
- Un ID modello dal [catalogo dei modelli di testo Venice](/models/text)

<Note>
La funzione BYOM di Brave è disponibile su desktop. L'ID del modello deve corrispondere esattamente a un ID modello Venice.
</Note>

## Configurazione

<Steps>
  <Step title="Apri le impostazioni di Leo">
    Apri `brave://settings/leo-ai`, oppure apri Leo dalla barra laterale di Brave, seleziona l'icona delle impostazioni e vai su **Leo**.
  </Step>

  <Step title="Aggiungi un modello personalizzato">
    Scorri fino a **Bring your own model** e seleziona **Add new model**.
  </Step>

  <Step title="Configura l'endpoint Venice">
    Inserisci i seguenti valori:

    | Campo | Valore |
    | --- | --- |
    | Label | `Venice AI` |
    | Model request name | Un ID modello dal [catalogo dei modelli di testo](/models/text) |
    | Server endpoint | `https://api.venice.ai/api/v1/chat/completions` |
    | API key | La tua API key Venice |

    Se Brave mostra il campo **Context size**, impostalo sulla finestra di contesto supportata dal modello selezionato. Puoi anche aggiungere un prompt di sistema opzionale.

    Inserisci la sola API key senza il prefisso `Bearer`. Brave aggiunge l'header di autorizzazione a ciascuna richiesta.
  </Step>

  <Step title="Salva e seleziona Venice">
    Seleziona **Add model**, poi imposta **Venice AI** come modello predefinito per le nuove conversazioni oppure sceglilo dal selettore di modelli di Leo.
  </Step>

  <Step title="Testa la connessione">
    Apri Leo su una pagina web e invia un prompt come:

    ```txt
    Summarize this page in five bullet points.
    ```

    Leo invia la conversazione e il contenuto rilevante della pagina direttamente all'endpoint Venice.
  </Step>
</Steps>

## Abilitare la ricerca web di Venice

Brave non espone parametri del corpo delle richieste specifici di Venice in BYOM. Per abilitare la ricerca web di Venice, aggiungi un [suffisso di feature del modello](/api-reference/endpoint/chat/model_feature_suffix) all'ID del modello nel campo **Model request name**:

```txt
<model-id>:enable_web_search=auto
```

Per abilitare sempre la ricerca e includere le citazioni, usa:

```txt
<model-id>:enable_web_search=on&enable_web_citations=true
```

<Info>
La ricerca web ha una tariffazione aggiuntiva basata sull'utilizzo. Consulta [Prezzi API](/overview/pricing) per le tariffe correnti.
</Info>

## Privacy

Quando usi un endpoint BYOM remoto, Brave si connette direttamente a quel provider. Le richieste bypassano i server di Brave, quindi il proxy di Brave e le protezioni di retention lato server non si applicano a tali richieste.

Venice non memorizza né registra il contenuto di prompt e risposte al suo proxy. Il modello selezionato determina il livello di privacy a runtime:

- I modelli **Private** utilizzano un'infrastruttura a retention zero dei dati.
- I modelli **Anonymized** nascondono le informazioni identificative al provider upstream, ma il provider può elaborare il prompt.
- I modelli **TEE** ed **E2EE** aggiungono l'isolamento supportato dall'hardware o la cifratura lato client dove supportato.

Controlla l'etichetta di privacy del modello nel [catalogo dei modelli di testo](/models/text) e leggi [Venice privacy](/overview/privacy) prima di inviare informazioni sensibili.

Brave potrebbe conservare la cronologia delle chat di Leo nello storage locale del browser. Puoi cancellare i dati di Leo, usare una chat temporanea o disabilitare la cronologia chat locale nelle impostazioni di Leo in Brave.

## Note di configurazione

- Brave richiede il percorso completo dell'endpoint: `https://api.venice.ai/api/v1/chat/completions`.
- Gli ID dei modelli e la loro disponibilità cambiano nel tempo. Copia l'ID attuale dal [catalogo dei modelli di testo](/models/text).
- La dimensione del contesto non deve superare la finestra di contesto supportata dal modello selezionato.
- I riassunti di pagina possono inviare il contenuto della pagina a Venice come parte del prompt.
- Le richieste remote a Venice richiedono una connessione internet e consumano crediti API Venice o la tua quota DIEM disponibile.

## Risoluzione dei problemi

<AccordionGroup>
  <Accordion title="Errore 401 o di autenticazione">
    Verifica che la chiave provenga da [venice.ai/settings/api](https://venice.ai/settings/api). Incolla la chiave grezza senza `Bearer` e rimuovi eventuali spazi iniziali o finali.
  </Accordion>
  <Accordion title="Modello non trovato">
    Copia l'ID del modello esattamente dal [catalogo dei modelli di testo](/models/text). Se hai aggiunto un suffisso di feature, prova prima il modello senza il suffisso.
  </Accordion>
  <Accordion title="Errore di rete o nessuna risposta in streaming">
    Verifica che l'endpoint del server includa `/api/v1/chat/completions`. Brave Leo si aspetta una risposta in streaming compatibile con OpenAI.
  </Accordion>
  <Accordion title="Ai riassunti di pagina manca il contesto della pagina">
    Controlla il permesso di Leo per usare il contenuto della pagina e assicurati che la pagina corrente sia accessibile al browser. Le pagine interne di Brave e alcune pagine protette non possono essere riassunte.
  </Accordion>
</AccordionGroup>

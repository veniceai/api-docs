Reference to Video ti permette di bloccare l'aspetto di personaggi, oggetti e scene in modo che i tuoi video generati con AI restino visivamente consistenti. Invece di sperare che il modello interpreti correttamente il tuo prompt, fornisci ancore visive — immagini di riferimento che dicono al modello esattamente come appare il tuo soggetto.

Questa funzionalità è disponibile sui modelli **Kling O3** e **Grok Imagine R2V** nel [Venice Video Studio](https://venice.ai/video). Ciascuna famiglia di modelli usa un approccio diverso alle immagini di riferimento — vedi le sezioni specifiche del modello sotto.

## Quando usare Reference to Video

Usa Reference to Video quando hai bisogno di:

- **Consistenza dei personaggi** — la stessa persona o personaggio in più inquadrature
- **Accuratezza del prodotto** — un prodotto reale che deve sembrare identico all'originale
- **Continuità di scena** — un ambiente o sfondo specifico tra le generazioni
- **Scene multi-personaggio** — più personaggi distinti che interagiscono senza fondersi

Per text-to-video o image-to-video semplici dove la consistenza non è critica, i modelli standard funzionano bene senza riferimenti.

## Modelli disponibili

| Modello | Approccio | Ideale per |
|---|---|---|
| **Kling O3 Pro R2V** | Elementi + scene image | Scene complesse multi-personaggio con controllo preciso dell'identità |
| **Kling O3 Standard R2V** | Elementi + scene image | Iterazione più veloce su scene basate su elementi |
| **Grok Imagine R2V** | Immagini di riferimento flat | Generazione veloce guidata da riferimenti con fino a 7 immagini |

**Kling O3** usa un approccio strutturato con Elementi (ancore di identità del personaggio con immagini frontali + di riferimento) e Scene Image. **Grok Imagine R2V** prende un approccio più semplice — carichi immagini di riferimento direttamente e le referenzia nel prompt con `@Image1`, `@Image2`, ecc.

## Kling O3 Reference to Video

### Concetti chiave

Kling O3 Reference to Video usa tre tipi di input visivo che lavorano insieme:

| Input | Obbligatorio | Scopo | Come referenziare nel prompt |
|---|---|---|---|
| **Elementi** | Almeno un input visivo* | Blocca l'identità di un personaggio o oggetto | `@Element1`, `@Element2`, ecc. |
| **Scene Reference Image** | Almeno un input visivo* | Imposta l'ambiente, stile e mood | `@Image1`, `@Image2`, ecc. |
| **Start Frame** | Almeno un input visivo* | Controlla il primo frame del video | N/D (impostato via upload) |
| **End Frame** | No | Controlla l'ultimo frame del video | N/D (impostato via upload) |

*È richiesto almeno uno tra: start frame, elementi o scene reference image.

### Elementi

Un Elemento è un personaggio o oggetto che vuoi mantenere visivamente stabile per tutto il video. Ogni elemento consiste di:

- **Frontal Image** (obbligatorio per elemento) — una foto chiara, frontale del soggetto. È l'ancora primaria di identità. Pensala come la "foto del passaporto" del tuo personaggio o prodotto.
- **Reference Images** (1–3, opzionale) — angoli aggiuntivi dello stesso soggetto (vista laterale, angolo a 45 gradi, retro). Aiutano il modello a capire il soggetto nello spazio 3D. Se non forniti, l'immagine frontale viene automaticamente usata come riferimento.

Puoi aggiungere fino a **7 elementi** per generazione (limitato dal totale combinato). Referenziali nel prompt usando `@Element1`, `@Element2`, ecc.

### Scene Reference Image

I riferimenti di scena definiscono il "palco" dove avviene l'azione. Influenzano:

- Illuminazione e palette colori
- Architettura e dettagli ambientali
- Stile visivo e mood complessivi

Puoi aggiungere fino a **4 scene image**. Referenziale come `@Image1`, `@Image2`, ecc. nel tuo prompt.

### Limitazioni

Il numero totale di immagini su tutti i tipi di input è limitato:

| Limite | Valore |
|---|---|
| **Minimo richiesto** | Almeno 1 input visivo (start frame, elemento o scene image) |
| **Totale combinato** (first frame + last frame + elementi + scene image) | **7 massimo** |
| Elementi (senza start/end frame) | 7 massimo |
| Elementi (con start o end frame) | 3 massimo |
| Scene reference image | 4 massimo |
| Reference image per elemento | 1–3 |

**Scenari di esempio:**

- 7 elementi + 0 scene image = 7 ✓ (nessun frame)
- 5 elementi + 2 scene image = 7 ✓ (nessun frame)
- First frame (1) + 3 elementi + 3 scene image = 7 ✓
- First frame (1) + last frame (1) + 3 elementi + 2 scene image = 7 ✓
- First frame (1) + 4 elementi = ✗ (max 3 elementi con frame)
- First frame (1) + last frame (1) + 4 elementi = ✗ (max 3 elementi con frame)

<Note>
Ogni elemento richiede una **frontal image**. Se non fornisci reference image per un elemento, la frontal image viene automaticamente usata come riferimento.
</Note>

### Modalità multi-shot

Multi-shot ti permette di dividere una singola generazione in più scene, ciascuna con il proprio prompt e durata. Elementi e riferimenti di scena si propagano tra tutti gli shot, mantenendo la consistenza. La durata totale tra tutti gli shot non può superare **15 secondi**.

### Guida passo-passo (Video Studio)

#### 1. Apri Video Studio e seleziona il modello

Vai su [venice.ai/video](https://venice.ai/video). Nel Model Browser a sinistra, seleziona uno dei modelli **Kling O3 Reference to Video**:

- **Kling O3 Pro R2V** — qualità superiore, tempo di generazione più lungo (~6 min)
- **Kling O3 Standard R2V** — più veloce, più conveniente per l'iterazione

#### 2. Aggiungi Visual Input (almeno uno richiesto)

Devi fornire **almeno un input visivo** per generare un video: uno start frame, un elemento o una scene reference image. Nel pannello di input, vedrai la sezione **Elements**. Clicca **Add Element** per creare un elemento per i personaggi o oggetti che vuoi mantenere visivamente consistenti.

Per ogni elemento:

1. Clicca il tile **Frontal** per caricare un'immagine chiara, frontale del tuo personaggio o oggetto
2. Opzionalmente clicca **Add** sotto **Reference Images** per caricare angoli aggiuntivi (1–3)

Ripeti per personaggi o oggetti aggiuntivi (fino a 7 elementi totali, o 3 se usi start/end frame).

<Warning>
Il totale combinato di first frame, last frame, elementi e scene image non può superare **7**. Vedi [Limitazioni](#limitazioni) per i dettagli.
</Warning>

<Tip>
**Migliori reference image:** Usa foto ben illuminate con sfondo pulito. Fornisci viste frontale, laterale e a 45 gradi per il lock di identità più forte. Assicurati che tutte le reference image condividano lo stesso stile visivo (non mescolare fotorealistico e anime).
</Tip>

#### 3. Aggiungi Scene Reference Image (opzionale)

Sotto la sezione Elements, vedrai **Scene Reference Images**. Carica immagini che definiscono l'ambiente che vuoi — una location specifica, un setup di illuminazione o uno stile artistico.

Sono taggate automaticamente come `@Image1`, `@Image2`, ecc.

#### 4. Carica uno Start Frame (opzionale)

Se vuoi controllare l'esatto primo frame del tuo video, passa al tipo di input **Image** e carica uno start frame. Puoi opzionalmente impostare anche un end frame.

#### 5. Scrivi il tuo prompt

Nel campo del prompt, descrivi l'azione che vuoi mentre referenzi i tuoi elementi e scene image usando i tag `@`:

```
@Element1 walks through the streets of @Image1, looking up at the buildings.
The camera slowly tracks from behind, revealing the city skyline.
```

Per **scene multi-personaggio**:

```
@Element1 and @Element2 enter the cafe in @Image1 from opposite sides.
@Element1 waves and walks toward @Element2, who is sitting at a corner table.
```

#### 6. Configura le impostazioni

Apri **Video Settings** per regolare:

| Impostazione | Opzioni | Default |
|---|---|---|
| Durata | 3s – 15s | 5s |
| Aspect Ratio | 16:9, 9:16, 1:1 | 16:9 |
| Generate Audio | On/Off | Off |

<Note>
La generazione audio aggiunge effetti sonori nativi, dialoghi e audio ambientale sincronizzato al video. Aumenta il costo del ~25%.
</Note>

#### 7. Genera

Clicca **Generate Video**. Kling O3 tipicamente impiega 4–6 minuti a seconda del tier del modello e della durata. Puoi mettere in coda più generazioni e sfogliare i risultati nel Video Gallery.

### Storyboarding multi-shot

Per sequenze narrative, usa la modalità multi-shot per definire scene separate all'interno di una singola generazione.

1. Nell'area del prompt, clicca **Add Shot** per creare shot aggiuntivi
2. Scrivi un prompt separato per ogni shot
3. Imposta la durata per ogni shot (3–15s ciascuno, totale ≤ 15s)

Elementi e riferimenti di scena persistono tra tutti gli shot automaticamente:

```
Shot 1 (5s): @Element1 stands at the edge of @Image1, looking out at the horizon.
Slow camera push forward.

Shot 2 (5s): Close-up of @Element1's face as they turn toward the camera.
Soft natural lighting, shallow depth of field.

Shot 3 (5s): @Element1 walks away from camera into the distance.
Wide cinematic shot, golden hour lighting.
```

<Warning>
La durata totale multi-shot non può superare 15 secondi. Ad esempio, tre shot da 5 secondi = 15s massimo.
</Warning>

### Consigli per il prompting

#### Struttura il tuo prompt

Segui questo schema per risultati affidabili:

```
[subject with @Element tag] + [action] + [environment with @Image tag] + [camera movement] + [lighting/style]
```

**Esempio:**

```
@Element1 hops happily across the candy ground of @Image1, stops to look at a
giant lollipop, tilts its head curiously. Cinematic tracking shot, soft warm lighting.
```

#### Mantieni i prompt 50–150 parole

I prompt più corti mancano di dettaglio. I prompt più lunghi introducono contraddizioni. Punta al punto giusto.

#### Usa un linguaggio camera semplice

Il modello risponde meglio a direzioni camera dirette:

| Usa | Evita |
|---|---|
| `slow camera push forward` | `dolly zoom with rack focus transition` |
| `tracking shot from behind` | `complex handheld parallax movement` |
| `close-up` | `extreme macro with tilt-shift bokeh` |
| `wide cinematic shot` | `anamorphic ultra-wide establishing crane shot` |

#### Usa vocabolario consistente

Se descrivi un personaggio che indossa "a red jacket" in un prompt, non passare a "crimson coat" nel prossimo. Il modello tratta parole diverse come intenti diversi.

#### Metti le istruzioni della camera all'inizio

Metti la direzione camera vicino all'inizio del prompt per risultati più affidabili:

```
Cinematic tracking shot of @Element1 walking through @Image1, leaves
blowing in the wind, golden afternoon light.
```

### Troubleshooting Kling O3

| Problema | Causa probabile | Correzione |
|---|---|---|
| Il pulsante Generate è disabilitato | Nessun input visivo fornito | Aggiungi almeno un input visivo: start frame, elemento o scene reference image |
| Errore "Number of images exceeds the limit" | Troppi input combinati | Il totale di first frame + last frame + elementi + scene image deve essere ≤ 7 |
| Il volto del personaggio cambia tra gli shot | Frontal image diversa o mancante | Usa la stessa frontal image consistentemente, mantieni la descrizione identica |
| Il movimento di camera sembra casuale | Istruzioni di camera multiple o conflittuali | Usa una singola istruzione di camera, mettila all'inizio del prompt |
| Lo stile cambia tra le generazioni | Riferimenti di scena inconsistenti o stili misti | Riusa le stesse scene image, mantieni le parole chiave dello stile consistenti |
| Gli elementi si fondono in scene multi-personaggio | Istruzioni spaziali vaghe | Sii esplicito sulla posizione di ogni elemento: "foreground left", "entering from right" |
| Lo sfondo sembra distorto | Scene reference image disordinata o complessa | Usa scene reference image pulite e di alta qualità |
| Il movimento sembra innaturale | Troppe azioni in un solo prompt | Semplifica l'azione, usa durata più breve, una azione per shot |

<Tip>
Testa con una clip di 3–5 secondi prima di impegnarti su durate più lunghe. Le clip più brevi mantengono una consistenza migliore e ti permettono di iterare più velocemente.
</Tip>

## Grok Imagine Reference to Video

Grok Imagine R2V prende un approccio più semplice di Kling O3. Invece di Elementi strutturati con separazione frontal/reference, carichi **immagini di riferimento flat** e le referenzia direttamente nel tuo prompt usando `@Image1`, `@Image2`, ecc. Il modello incorpora quei soggetti nel video generato.

### Come funziona

1. Carica **1–7 reference image** — foto di personaggi, oggetti o scene che vuoi nel video
2. Scrivi un prompt che descrive il video, usando `@Image1`, `@Image2`, ecc. per referenziare immagini specifiche
3. Il modello genera un video che incorpora quei riferimenti

Se non includi tag `@Image` nel tuo prompt, tutte le immagini caricate vengono referenziate automaticamente.

### Impostazioni

| Impostazione | Opzioni | Default |
|---|---|---|
| Aspect Ratio | 16:9, 4:3, 3:2, 1:1, 2:3, 3:4, 9:16 | 16:9 |
| Risoluzione | 480p, 720p | 480p |
| Durata | 5s, 8s, 10s | 8s |

<Note>
Grok Imagine R2V non supporta la generazione audio, la modalità multi-shot o gli Elementi. Per quelle funzionalità, usa Kling O3 R2V.
</Note>

### Guida passo-passo (Video Studio)

#### 1. Seleziona il modello

Vai su [venice.ai/video](https://venice.ai/video). Nel Model Browser, seleziona **Grok Imagine R2V**.

#### 2. Carica reference image

Clicca **References** nella toolbar di input (o usa il menu +) per aprire il pannello delle reference image. Carica 1–7 immagini dei personaggi, oggetti o scene che vuoi nel video.

Ogni immagine è automaticamente taggata come `@Image1`, `@Image2`, ecc. nell'ordine in cui le carichi (da sinistra a destra).

#### 3. Scrivi il tuo prompt

Descrivi il video che vuoi. Usa i tag `@Image` per referenziare immagini specifiche:

```
@Image1 and @Image2 walking together through a sunlit park,
camera slowly tracking alongside them, warm afternoon light.
```

Digita `@` nel campo prompt per vedere un menu di autocompletamento dei riferimenti immagine disponibili.

<Tip>
Se ometti del tutto i tag `@Image`, il backend prepende automaticamente i riferimenti a tutte le immagini caricate. È utile quando vuoi che tutte le immagini siano usate senza specificare quale.
</Tip>

#### 4. Configura le impostazioni e genera

Apri **Video Settings** per regolare aspect ratio, risoluzione e durata. Clicca **Generate Video**.

### Prezzi Grok Imagine R2V

Grok Imagine R2V usa prezzi basati su durata e risoluzione:

| Risoluzione | Per secondo |
|---|---|
| 480p | ~$0.063 |
| 720p | ~$0.088 |

**Esempio:** Un video di 8 secondi a 480p = 8 × $0.063 = **~$0.50**

<Note>
Grok Imagine addebita una tariffa di moderazione contenuto per i video generati, anche se il video viene rifiutato. Questo si riflette nel costo in crediti mostrato prima della generazione.
</Note>

### Troubleshooting Grok Imagine R2V

| Problema | Causa probabile | Correzione |
|---|---|---|
| Il pulsante Generate è disabilitato | Nessuna reference image caricata | Carica almeno 1 reference image |
| Errore "At least one reference image is required" | `referenceImageUrls` è vuoto o mancante | Fornisci almeno un URL immagine in `referenceImageUrls` |
| Immagine sbagliata associata al tag `@Image` | L'ordine delle immagini non corrisponde ai tag | `@Image1` corrisponde alla prima immagine nel tuo ordine di upload (da sinistra a destra). Riordina gli upload se necessario. |
| Il soggetto non appare nel video | Troppi riferimenti senza tag espliciti | Usa i tag `@Image` nel tuo prompt per essere esplicito su quali immagini usare |
| Output di bassa qualità | Usando risoluzione 480p | Prova 720p per qualità superiore (costa di più) |
| Video troppo corto | Durata predefinita 8s | Imposta durata su `"10"` per video più lunghi |

La generazione di immagini su Venice è sincrona. Invia un prompt a `/image/generate` e ricevi la tua immagine nella stessa risposta, sia come base64 dentro JSON sia come binario raw quando `return_binary` è `true`.

## Endpoint

| Endpoint | Scopo | Quando usarlo |
|---|---|---|
| `POST /image/generate` | API di generazione immagini nativa di Venice | Usalo per il pieno supporto delle funzionalità |
| `GET /image/styles` | Elenca i preset di stile disponibili | Usalo prima di inviare `style_preset` |
| `POST /images/generations` | API di generazione immagini compatibile OpenAI | Usalo quando migri client OpenAI di immagini esistenti |

## Passo 1: Invia una richiesta di generazione

Il dimensionamento è specifico per modello. Alcuni modelli accettano `width` e `height` espliciti; alcuni espongono `aspect_ratio`; e i modelli a fasce di risoluzione espongono `aspect_ratio` più valori di `resolution` come `1K`, `2K` o `4K`.

**Esempio di dimensionamento basato su pixel:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "venice-sd35",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
  "negative_prompt": "blurry, low quality, distorted anatomy, text, watermark",
  "width": 1024,
  "height": 1024,
  "format": "webp"
}
```

**Esempio di dimensionamento per aspect ratio:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-image-2",
  "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "format": "webp"
}
```

**Esempio di dimensionamento a fasce di risoluzione:**
```bash
POST https://api.venice.ai/api/v1/image/generate
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "gpt-image-2",
  "prompt": "A cinematic wide shot of a gondola passing through a narrow Venice canal at blue hour",
  "aspect_ratio": "16:9",
  "resolution": "4K",
  "format": "png"
}
```

Lo stesso schema si applica ad altri modelli a fasce di risoluzione:

```json
{
  "model": "nano-banana-pro",
  "prompt": "A serene canal in Venice at sunset",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Usa [Modelli di immagine](/it/models/image) o l'[API Models](/it/api-reference/endpoint/models/list) per confermare quali campi di dimensionamento accetta ciascun modello.

**Risposta (200):**
```json
{
  "id": "generate-image-1234567890",
  "images": [
    "UklGRiIAAABXRUJQVlA4IBYAAAAwAQCdASoQABAAPm..."
  ],
  "timing": {
    "inferenceDuration": 1840,
    "inferencePreprocessingTime": 22,
    "inferenceQueueTime": 31,
    "total": 1893
  }
}
```

L'array `images` contiene dati immagine codificati in base64. Decodifica il primo elemento per salvarlo o visualizzarlo. `timing.total` è la durata completa della richiesta in millisecondi.

## Passo 2: Decodifica e salva l'immagine

<CodeGroup>
```python Python
import base64
import os
import requests

response = requests.post(
    "https://api.venice.ai/api/v1/image/generate",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "venice-sd35",
        "prompt": "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
        "width": 1024,
        "height": 1024,
        "format": "webp",
    },
)

data = response.json()
image_bytes = base64.b64decode(data["images"][0])

with open("output.webp", "wb") as f:
    f.write(image_bytes)

print(f"Saved image from request {data['id']}")
```

```javascript Node.js
import fs from "fs";

const response = await fetch("https://api.venice.ai/api/v1/image/generate", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "venice-sd35",
    prompt: "A cinematic photo of a gondola passing through a narrow Venice canal at blue hour, warm window lights reflecting on the water",
    width: 1024,
    height: 1024,
    format: "webp",
  }),
});

const data = await response.json();
const imageBuffer = Buffer.from(data.images[0], "base64");
fs.writeFileSync("output.webp", imageBuffer);

console.log(`Saved image from request ${data.id}`);
```
</CodeGroup>

## Passo 3: Restituisci binario invece di JSON (opzionale)

Se vuoi che il body della risposta sia il file immagine stesso, imposta `return_binary: true`. Utile quando vuoi fare streaming o salvare l'immagine direttamente senza decodifica base64.

```bash
curl https://api.venice.ai/api/v1/image/generate \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o output.png \
  -d '{
    "model": "qwen-image-2",
    "prompt": "Minimalist poster of a moonlit Venetian bridge in deep blue tones",
    "format": "png",
    "return_binary": true
  }'
```

Quando `return_binary` è `true`, il body della risposta è raw `image/jpeg`, `image/png` o `image/webp` in base al `format` richiesto.

<Note>
`variants` è supportato solo quando `return_binary` è `false`.
</Note>

## Passo 4: Elenca gli stili immagine disponibili (opzionale)

Se vuoi usare `style_preset`, recupera prima gli stili disponibili da `/image/styles`:

```bash
curl https://api.venice.ai/api/v1/image/styles \
  -H "Authorization: Bearer $VENICE_API_KEY"
```

**Risposta (200):**
```json
[
  "3D Model",
  "Analog Film",
  "Anime",
  "Cinematic",
  "Digital Art"
]
```

Poi passa uno di quei valori nella tua richiesta di generazione:

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "Cinematic"
}
```

Usa l'endpoint styles quando vuoi nomi di preset esatti invece di indovinarli.

## Parametri di richiesta

| Parametro | Tipo | Obbligatorio | Default | Descrizione |
|---|---|---|---|---|
| `model` | string | Sì | - | ID del modello da usare per la generazione |
| `prompt` | string | Sì | - | Cosa generare |
| `negative_prompt` | string | No | - | Cosa evitare nell'immagine |
| `width` | integer | No | `1024` | Larghezza di output in pixel per modelli basati su pixel come `venice-sd35` e `qwen-image` |
| `height` | integer | No | `1024` | Altezza di output in pixel per modelli basati su pixel come `venice-sd35` e `qwen-image` |
| `format` | string | No | `webp` | Formato di output: `jpeg`, `png` o `webp` |
| `variants` | integer | No | `1` | Numero di immagini da generare (`1`-`4`), solo quando `return_binary` è `false` |
| `return_binary` | boolean | No | `false` | Restituisce i byte raw dell'immagine invece di JSON base64 |
| `safe_mode` | boolean | No | `true` | Sfoca contenuto per adulti quando abilitato |
| `seed` | integer | No | random | Riusa lo stesso seed per iterazioni più consistenti |
| `cfg_scale` | number | No | dipende dal modello | Valori più alti spingono il modello a seguire il prompt più strettamente |
| `style_preset` | string | No | - | Applica uno stile preset da [Image Styles](/it/api-reference/endpoint/image/styles) |
| `aspect_ratio` | string | Condizionale | - | Usato da modelli che supportano dimensionamento basato sul rapporto, come `qwen-image-2`, `gpt-image-2`, `nano-banana-2` e `nano-banana-pro` |
| `resolution` | string | Condizionale | - | Usato da modelli che supportano fasce di risoluzione come `1K`, `2K` o `4K` |
| `enable_web_search` | boolean | Condizionale | `false` | Permette ai modelli supportati di usare informazioni web attuali; aggiunge costo extra |

La validazione è specifica per modello. Controlla [Modelli di immagine](/it/models/image) e l'[API Models](/it/api-reference/endpoint/models/list) prima di affidarti a un parametro tra più modelli.

## Opzioni specifiche del modello

### Generazione ad alta risoluzione

Alcuni modelli di immagine supportano `aspect_ratio` senza una fascia di `resolution` selezionabile. Ad esempio, `qwen-image-2` accetta aspect ratio e lo mappa a dimensioni di output specifiche del modello:

```json
{
  "model": "qwen-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9"
}
```

Altri modelli di immagine supportano `aspect_ratio` più una fascia di `resolution`. Ad esempio, `gpt-image-2`, `nano-banana-2` e `nano-banana-pro` supportano `1K`, `2K` e `4K`:

```json
{
  "model": "gpt-image-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "4K"
}
```

```json
{
  "model": "nano-banana-2",
  "prompt": "Editorial product photo of a luxury watch on black marble, dramatic studio lighting",
  "aspect_ratio": "16:9",
  "resolution": "2K"
}
```

Usa [Modelli di immagine](/it/models/image) per vedere quali modelli supportano risoluzioni più alte e come sono prezzati.

### Stili preset

Se il modello selezionato lo supporta, `style_preset` ti permette di guidare l'output senza riscrivere l'intero prompt. Puoi recuperare i nomi dei preset validi da [Image Styles](/it/api-reference/endpoint/image/styles):

```json
{
  "model": "qwen-image-2",
  "prompt": "A futuristic Venice skyline at sunrise",
  "style_preset": "3D Model"
}
```

Vedi [Image Styles](/it/api-reference/endpoint/image/styles) per l'elenco corrente degli stili.

## Endpoint compatibile OpenAI

Se stai già usando SDK OpenAI di immagini o integrazioni DALL-E esistenti, Venice supporta anche `POST /images/generations`. Offre un formato di richiesta più semplice, ma meno funzionalità rispetto all'endpoint nativo di Venice.

**Richiesta:**
```json
{
  "model": "qwen-image-2",
  "prompt": "A clean isometric illustration of an AI control room",
  "size": "1024x1024",
  "response_format": "b64_json"
}
```

Usa la route compatibile OpenAI per migrazioni più veloci. Usa `/image/generate` quando hai bisogno di opzioni specifiche Venice come `cfg_scale`, `style_preset`, `variants` o risposte binarie.

## Consigli per il prompting

1. Inizia con il soggetto, poi aggiungi medium, illuminazione, composizione e mood.
2. Metti i dettagli da evitare in `negative_prompt` invece di sovraccaricare il prompt principale.
3. Riusa `seed` quando iteri così puoi confrontare modifiche al prompt senza cambiare completamente la composizione.
4. Mantieni il dimensionamento consapevole del modello. Alcuni usano `width`/`height`, alcuni `aspect_ratio`, e i modelli a fasce di risoluzione usano `aspect_ratio` più `resolution`.
5. Usa `variants` durante l'esplorazione, poi torna a un singolo output una volta scelta la direzione.

## Errori

| Stato | Significato | Azione |
|---|---|---|
| `400` | Parametri di richiesta non validi | Controlla nomi dei campi, tipi e vincoli specifici del modello |
| `401` | Autenticazione fallita o il modello richiede un tier di accesso superiore | Controlla la tua API key e l'accesso al modello |
| `402` | Saldo insufficiente | Aggiungi crediti su [venice.ai/settings/api](https://venice.ai/settings/api) |
| `415` | Content type non valido | Invia JSON con `Content-Type: application/json` |
| `429` | Rate limit superato o modello sovraccarico | Ritenta con backoff; controlla l'header `Retry-After` |
| `500` | Elaborazione di inferenza fallita | Ritenta la richiesta |
| `503` | Modello a capacità | Ritenta dopo un breve ritardo |

<Note>
Quando Safe Venice è abilitato, ispeziona header di risposta come `x-venice-is-blurred` e `x-venice-is-content-violation` se devi rilevare programmaticamente gli esiti di moderazione.
</Note>

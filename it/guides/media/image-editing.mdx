La modifica di immagini su Venice è sincrona. Invia la tua immagine sorgente a `/image/edit` o `/image/multi-edit` e il risultato modificato torna nella stessa risposta come file PNG. Per i ritagli, `/image/background-remove` restituisce un PNG trasparente.

<Warning>
Gli endpoint di image edit sono sperimentali e il comportamento specifico del modello potrebbe cambiare nel tempo.
</Warning>

## Endpoint

| Endpoint | Scopo | Ideale per |
|---|---|---|
| `POST /image/edit` | Modifica una singola immagine con un prompt | Modifiche generali e inpainting guidato da prompt |
| `POST /image/multi-edit` | Modifica usando 1-3 immagini a livelli | Modifiche più controllate con mask o overlay |
| `POST /image/background-remove` | Rimuovi lo sfondo da un'immagine | Ritagli trasparenti per prodotti, ritratti e asset |

## Quando usare quale endpoint

- Usa `/image/edit` quando hai una sola immagine sorgente e vuoi cambiare, rimuovere o ristilizzare parte di essa con un prompt.
- Usa `/image/multi-edit` quando hai bisogno di controllo extra da mask, overlay o livelli di riferimento.
- Usa `/image/background-remove` quando vuoi solo un soggetto in primo piano pulito con trasparenza.

<Note>
Per l'inpainting, usa `/image/edit` o `/image/multi-edit`. Il vecchio parametro `inpaint` su `/image/generate` è deprecato.
</Note>

## Passo 1: Modificare una singola immagine

L'edit a singola immagine è il flusso di inpainting più semplice. Invia un'immagine più un prompt breve come "remove the sign", "change the sky to sunrise" o "replace the background with a studio backdrop".

**Richiesta:**
```bash
POST https://api.venice.ai/api/v1/image/edit
Authorization: Bearer $VENICE_API_KEY
Content-Type: application/json

{
  "model": "qwen-edit",
  "prompt": "Replace the cloudy sky with a warm sunrise while preserving the buildings and canal",
  "image": "https://example.com/venice-canal.jpg"
}
```

**Risposta (200):**
Il body della risposta è un dato binario raw `image/png`. Salvalo direttamente su file.

<CodeGroup>
```python Python
import base64
import os
import requests

with open("input.jpg", "rb") as f:
    image_base64 = base64.b64encode(f.read()).decode("utf-8")

response = requests.post(
    "https://api.venice.ai/api/v1/image/edit",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json",
    },
    json={
        "model": "qwen-edit",
        "prompt": "Remove the tourist crowd from the square and keep the architecture intact",
        "image": image_base64,
    },
)

with open("edited.png", "wb") as f:
    f.write(response.content)
```

```javascript Node.js
import fs from "fs";

const imageBase64 = fs.readFileSync("input.jpg").toString("base64");

const response = await fetch("https://api.venice.ai/api/v1/image/edit", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "qwen-edit",
    prompt: "Remove the tourist crowd from the square and keep the architecture intact",
    image: imageBase64,
  }),
});

const editedImage = Buffer.from(await response.arrayBuffer());
fs.writeFileSync("edited.png", editedImage);
```

```bash cURL
curl https://api.venice.ai/api/v1/image/edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o edited.png \
  -d '{
    "model": "qwen-edit",
    "prompt": "Colorize this black and white portrait naturally",
    "image": "https://example.com/portrait-bw.jpg"
  }'
```

## Passo 2: Usa multi-edit per mask o inpainting a livelli

`/image/multi-edit` accetta fino a tre immagini. La prima immagine è l'immagine base. Le immagini rimanenti sono trattate come livelli di edit o mask, il che ti dà più controllo rispetto al solo editing tramite prompt.

È la scelta migliore quando vuoi:

- mirare a una regione specifica con una mask
- combinare una composizione esistente con un overlay
- vincolare la modifica più strettamente di quanto possa fare un prompt a singola immagine

**Richiesta JSON:**
```json
{
  "modelId": "qwen-edit",
  "prompt": "Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective",
  "images": [
    "https://example.com/street-scene.png",
    "https://example.com/billboard-mask.png"
  ]
}
```

**Richiesta multipart:**
```bash
curl https://api.venice.ai/api/v1/image/multi-edit \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "modelId=qwen-edit" \
  -F "prompt=Replace the blank billboard area with a glowing Venice film festival poster while preserving lighting and perspective" \
  -F "images=@street-scene.png" \
  -F "images=@billboard-mask.png" \
  -o multi-edited.png
```

Come `/image/edit`, il body della risposta è raw `image/png`.

<Note>
`/image/multi-edit` attualmente usa il campo `modelId` invece di `model` nello schema della richiesta.
</Note>

## Consigli per l'inpainting

L'inpainting basato su prompt funziona meglio quando l'istruzione è breve e locale:

- `remove the tree`
- `change the sky to sunset`
- `replace the logo with a blank sign`
- `restore the torn corner of the photo`

Per modifiche di scena più ampie, descrivi cosa deve restare invariato:

```text
Replace the background with a modern photo studio backdrop while preserving the subject pose, facial features, and clothing.
```

Se la modifica continua a colpire la zona sbagliata, passa da `/image/edit` a `/image/multi-edit` e fornisci un livello di mask o overlay.

## Passo 3: Rimuovi lo sfondo

Usa `/image/background-remove` quando vuoi il soggetto in primo piano isolato su uno sfondo trasparente. Questo endpoint restituisce un PNG con trasparenza alpha.

**Usando un URL immagine:**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -o cutout.png \
  -d '{
    "image_url": "https://example.com/product-photo.jpg"
  }'
```

**Usando un upload locale:**
```bash
curl https://api.venice.ai/api/v1/image/background-remove \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -F "image=@product-photo.jpg" \
  -o cutout.png
```

Usa la rimozione sfondo per:

- foto prodotto ecommerce
- foto profilo e ritratti
- asset che intendi posizionare su un nuovo sfondo

## Parametri di richiesta

### `/image/edit`

| Parametro | Tipo | Obbligatorio | Default | Descrizione |
|---|---|---|---|---|
| `image` | file, base64 o URL | Sì | - | Immagine sorgente da modificare |
| `prompt` | string | Sì | - | Istruzioni testuali per la modifica |
| `model` | string | No | `qwen-edit` | ID del modello di edit |
| `aspect_ratio` | string | No | default del modello | Rapporto di output per i modelli che lo supportano |
| `modelId` | string | Deprecato | - | Alias deprecato per `model` |

### `/image/multi-edit`

| Parametro | Tipo | Obbligatorio | Default | Descrizione |
|---|---|---|---|---|
| `images` | array di 1-3 file, base64 o URL | Sì | - | La prima immagine è l'immagine base; le altre sono livelli di edit o mask |
| `prompt` | string | Sì | - | Istruzioni testuali per come combinare o modificare i livelli |
| `modelId` | string | No | `qwen-edit` | ID del modello di edit |

### `/image/background-remove`

| Parametro | Tipo | Obbligatorio | Descrizione |
|---|---|---|---|
| `image` | file o base64 | Uno tra `image` e `image_url` | Immagine sorgente da ritagliare |
| `image_url` | string | Uno tra `image` e `image_url` | URL pubblico dell'immagine da ritagliare |

## Formati di input supportati

| Endpoint | Input JSON | Input multipart | Output |
|---|---|---|---|
| `/image/edit` | Stringa base64 o URL | Upload file | `image/png` |
| `/image/multi-edit` | Stringhe base64 o URL | Upload file | `image/png` |
| `/image/background-remove` | Stringa base64 o URL | Upload file | `image/png` |

Per gli endpoint di edit, le dimensioni dell'immagine devono essere almeno `65536` pixel e non più di `33177600` pixel. I file caricati devono essere sotto `25MB`.

## Errori

| Stato | Significato | Azione |
|---|---|---|
| `400` | Parametri di richiesta non validi | Controlla il numero di immagini, i nomi dei campi e il formato di input |
| `401` | Autenticazione fallita | Controlla la tua API key |
| `402` | Saldo insufficiente | Aggiungi crediti su [venice.ai/settings/api](https://venice.ai/settings/api) |
| `415` | Content type non valido | Usa correttamente JSON o multipart form-data |
| `429` | Rate limit superato o modello sovraccarico | Ritenta con backoff; controlla l'header `Retry-After` |
| `500` | Elaborazione di inferenza fallita | Ritenta la richiesta |
| `503` | Modello a capacità | Ritenta dopo un breve ritardo |

<Note>
Alcuni modelli di edit hanno policy di contenuto più stringenti rispetto ai modelli di generazione immagini. Ad esempio, `qwen-edit` blocca le richieste che coinvolgono immagini sessuali esplicite, sessualizzazione di minori o violenza nel mondo reale.
</Note>

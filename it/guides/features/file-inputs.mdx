I file input ti permettono di allegare documenti e file sorgente direttamente a una richiesta `/chat/completions`. Venice estrae il file in testo prima di inviarlo al modello selezionato, in modo che tu possa fare domande, riassumere, confrontare o trasformare il contenuto del file senza dover prima costruire un tuo parser.

Usa i file input quando il tuo prompt dipende dal contenuto di un documento, di un foglio di calcolo, di un file markdown, di un file JSON o di un file di codice. Sono input legati alla richiesta, non uno storage file persistente, quindi includi il file in ogni richiesta che ne ha bisogno.

<Info>
I file input usano l'array di content della chat compatibile OpenAI. Aggiungi un blocco di content con `type: "file"` e fornisci il contenuto del file in `file.file_data`.
</Info>

## Tipi di file supportati

L'API chat accetta i file input come URL data in base64 o come URL accessibili pubblicamente.

La dimensione massima del file è **25MB per file**, misurata dopo aver decodificato un data URL base64 o dopo aver scaricato un URL.

| Categoria | Formati |
|----------|---------|
| Documenti | PDF, DOCX, PPTX |
| Fogli di calcolo | XLSX, XLS, CSV |
| Testo e dati | TXT, Markdown, JSON |
| Codice sorgente | I file di codice più comuni, inclusi `.py`, `.js`, `.ts`, `.c`, `.cpp`, `.java`, `.go`, `.rs`, `.ps1`, `.sh`, `.yaml` e `.sql` |

<Note>
I file vengono estratti in testo prima dell'inferenza. Il testo estratto conta verso il contesto di input del modello, quindi scegli un modello con `availableContextTokens` sufficienti per il file più le tue istruzioni e la risposta attesa.
</Note>

## Utilizzo base

Invia un array `messages` dove il `content` del messaggio user è un array di blocchi di testo e file:

<CodeGroup>
```python Python
import base64
import os
from pathlib import Path

from openai import OpenAI

client = OpenAI(
    api_key=os.environ["VENICE_API_KEY"],
    base_url="https://api.venice.ai/api/v1",
)

path = Path("q3-report.pdf")
file_data = "data:application/pdf;base64," + base64.b64encode(path.read_bytes()).decode("utf-8")

response = client.chat.completions.create(
    model="openai-gpt-55",
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "text",
                    "text": "Summarize this report in five bullets and list the main risks.",
                },
                {
                    "type": "file",
                    "file": {
                        "file_data": file_data,
                        "filename": "q3-report.pdf",
                    },
                },
            ],
        }
    ],
)

print(response.choices[0].message.content)
```

```javascript Node.js
import OpenAI from "openai";
import { readFile } from "node:fs/promises";

  apiKey: process.env.VENICE_API_KEY,

      role: "user",

        {
          type: "text",
          text: "Summarize this report in five bullets and list the main risks.",
        },
        {
          type: "file",
          file: {
            file_data: fileData,
            filename: "q3-report.pdf",
          },
        },
      ],
    },
  ],
});

console.log(response.choices[0].message.content);
```

## URL dei file

Se il file è già ospitato su un URL HTTP o HTTPS pubblico, passa l'URL in `file_data` invece di codificarlo in base64:

```json
{
  "model": "openai-gpt-55",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "Identify the governing law, renewal terms, and termination rights in this agreement."
        },
        {
          "type": "file",
          "file": {
            "file_data": "https://example.com/contracts/vendor-agreement.pdf",
            "filename": "vendor-agreement.pdf"
          }
        }
      ]
    }
  ]
}
```

<Warning>
Usa solo URL pubblici che Venice può scaricare senza autenticazione. Per i file privati, invia un data URL base64.
</Warning>

## File multipli

Puoi includere più di un blocco file nello stesso messaggio. Metti una breve istruzione testuale prima dei file in modo che il modello sappia come usarli.

```json
{
  "model": "openai-gpt-55",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "Compare these two policy drafts. Return the material differences and recommend which version is clearer."
        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,JVBERi0xLjQK...",
            "filename": "policy-v1.pdf"
          }
        },
        {
          "type": "file",
          "file": {
            "file_data": "data:application/pdf;base64,JVBERi0xLjQK...",
            "filename": "policy-v2.pdf"
          }
        }
      ]
    }
  ]
}
```

Per risultati migliori, dai a ogni file un nome chiaro e fai riferimento a quei nomi nel tuo prompt.

## Data URL

Per i file locali, codifica i byte del file in base64 e prefissali con il MIME type corretto:

| Tipo di file | Prefisso data URL |
|-----------|-----------------|
| PDF | `data:application/pdf;base64,` |
| DOCX | `data:application/vnd.openxmlformats-officedocument.wordprocessingml.document;base64,` |
| PPTX | `data:application/vnd.openxmlformats-officedocument.presentationml.presentation;base64,` |
| XLSX | `data:application/vnd.openxmlformats-officedocument.spreadsheetml.sheet;base64,` |
| CSV | `data:text/csv;base64,` |
| Markdown | `data:text/markdown;base64,` |
| Testo semplice | `data:text/plain;base64,` |
| JSON | `data:application/json;base64,` |

Se non conosci il MIME type esatto, usa `application/octet-stream`. Includere un `filename` accurato aiuta comunque Venice a identificare e mostrare il file.

## Lavorare con file di grandi dimensioni

Poiché i file diventano testo del prompt, file grandi possono aumentare latenza, utilizzo di token e costo. Tieni a mente la context window del modello.

Il file raw deve essere di 25MB o meno. La codifica base64 aumenta la dimensione della richiesta di circa il 33%, quindi un file vicino al limite di 25MB produrrà un body JSON della richiesta più grande.

Buoni pattern per file grandi:

- Chiedi un task specifico invece di un prompt generico "analizza tutto".
- Includi solo i documenti necessari per la risposta corrente.
- Usa modelli con `availableContextTokens` maggiori per report o codebase lunghi.
- Metti documenti stabili e ripetuti prima delle domande dinamiche dell'utente se stai usando anche il [prompt caching](/it/guides/features/prompt-caching).
- Usa `stream: true` quando ti aspetti una risposta lunga.

## File input vs. Text Parser

Usa i file input chat quando vuoi che il modello ragioni sul file immediatamente.

Usa l'[API Text Parser](/it/api-reference/endpoint/augment/text-parser) quando vuoi prima estrarre il testo, ispezionare il conteggio dei token, memorizzare il testo estratto nel tuo sistema, o inviare lo stesso testo estratto a più richieste.

| Necessità | Usa |
|------|-----|
| Chiedere a un modello informazioni su un documento in una sola richiesta | File input chat |
| Estrarre testo senza inferenza del modello | API Text Parser |
| Verificare il conteggio dei token estratti prima del prompt | API Text Parser |
| Riutilizzare il testo estratto in molte richieste | API Text Parser, poi includi il testo nei prompt |

## Best practice

- Includi `filename` ogni volta che è possibile, specialmente quando invii più file.
- Metti l'istruzione prima dei blocchi file in modo che il modello conosca il task prima di leggere il contenuto estratto.
- Usa URL pubblici solo per file che possono essere scaricati senza cookie, header o stato di sessione firmato.
- Preferisci data URL base64 per file privati o file generati all'interno della tua applicazione.
- Fai domande mirate e specifica il formato di output che vuoi.
- Per estrazioni strutturate, combina i file input con le [risposte strutturate](/it/guides/features/structured-responses).

## Troubleshooting

<AccordionGroup>
  <Accordion title="Il modello dice di non poter accedere al file">
    Assicurati che il content del messaggio usi un array e includa un blocco `type: "file"`. Se hai usato un URL, verifica che sia pubblicamente raggiungibile senza autenticazione.
  </Accordion>

  <Accordion title="La richiesta è lenta o costosa">
    Il file potrebbe estrarre una grande quantità di testo. Usa un modello con contesto più ampio, restringi il task, invia meno file, o pre-estrai e taglia il testo con l'API Text Parser.
  </Accordion>

  <Accordion title="La risposta ignora uno dei miei file">
    Dai a ogni file un `filename` descrittivo e fai riferimento direttamente ai filename nel tuo prompt. Ad esempio: "Compare `policy-v1.pdf` against `policy-v2.pdf`."
  </Accordion>

  <Accordion title="Un modello rifiuta il contenuto del file">
    I file input sono disponibili sui modelli di chat compatibili. Controlla la [pagina Modelli](/it/models/overview) per le capacità correnti dei modelli e i limiti di contesto, o prova un modello di testo a contesto ampio attuale.
  </Accordion>
</AccordionGroup>

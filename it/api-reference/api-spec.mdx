L'API Venice offre interfacce REST e di streaming basate su HTTP per costruire applicazioni AI con modelli senza censura e inferenza privata. Puoi creare con generazione di testo, creazione di immagini, embeddings e altro ancora, il tutto senza policy di contenuto restrittive. Esempi di integrazione e SDK sono disponibili nella [documentazione](/it/overview/getting-started). Il nostro riferimento API è disponibile anche come [spec OpenAPI YAML.](https://api.venice.ai/doc/api/swagger.yaml)

## Autenticazione

L'API Venice usa API key per l'autenticazione. Crea e gestisci le tue API key nelle [impostazioni API](https://venice.ai/settings/api).


Tutte le richieste API richiedono l'autenticazione HTTP Bearer:

```
Authorization: Bearer VENICE_API_KEY
```

<Note>
La tua API key è un segreto. Non condividerla né esporla in nessun codice lato client.
</Note>

## Compatibilità OpenAI

L'API di Venice implementa la specifica dell'API OpenAI, garantendo la compatibilità con i client e gli strumenti OpenAI esistenti. Questo ti permette di integrare Venice usando la familiare interfaccia OpenAI mentre accedi alle funzionalità uniche e ai modelli senza censura di Venice.

### Configurazione

Configura il tuo client per usare il base URL di Venice (`https://api.venice.ai/api/v1`) e fai la tua prima richiesta:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```javascript JavaScript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.VENICE_API_KEY,
  baseURL: "https://api.venice.ai/api/v1",
});

const response = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [{ role: "user", content: "Hello!" }]
});

console.log(response.choices[0].message.content);
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("VENICE_API_KEY"),
    base_url="https://api.venice.ai/api/v1"
)

response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```
</CodeGroup>

## Funzionalità specifiche di Venice

### System Prompts

Venice fornisce dei system prompt predefiniti progettati per garantire risposte del modello senza censura e naturali. Hai due opzioni per gestire i system prompt:

1. **Comportamento predefinito**: I tuoi system prompt vengono accodati a quelli di default di Venice
2. **Comportamento personalizzato**: Disabilita completamente i system prompt di Venice

#### Disabilitare i system prompt di Venice

Usa l'opzione `venice_parameters` per rimuovere i system prompt predefiniti di Venice:

<CodeGroup>
```bash curl
curl https://api.venice.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "venice-uncensored",
    "messages": [
      {"role": "system", "content": "Your custom system prompt"},
      {"role": "user", "content": "Why is the sky blue?"}
    ],
    "venice_parameters": {
      "include_venice_system_prompt": false
    }
  }'
```

```javascript JavaScript
const completion = await client.chat.completions.create({
  model: "venice-uncensored",
  messages: [
    {
      role: "system",
      content: "Your custom system prompt",
    },
    {
      role: "user",
      content: "Why is the sky blue?",
    },
  ],
  venice_parameters: {
    include_venice_system_prompt: false,
  },
});
```

```python Python
response = client.chat.completions.create(
    model="venice-uncensored",
    messages=[
        {"role": "system", "content": "Your custom system prompt"},
        {"role": "user", "content": "Why is the sky blue?"}
    ],
    extra_body={
        "venice_parameters": {
            "include_venice_system_prompt": False
        }
    }
)
```
</CodeGroup>

### Venice Parameters

L'oggetto `venice_parameters` ti permette di accedere a funzionalità specifiche di Venice non disponibili nell'API OpenAI standard:

| Parametro | Tipo | Descrizione | Default |
|-----------|------|-------------|---------|
| `character_slug` | string | Lo slug di un character pubblico di Venice (visibile come "Public ID" nella pagina del character pubblicato) | - |
| `strip_thinking_response` | boolean | Rimuove i blocchi `<think></think>` dalla risposta (modelli che usano il formato tag `<think>` legacy). Vedi [Reasoning Models](/it/guides/features/reasoning-models). | `false` |
| `disable_thinking` | boolean | Sui modelli di ragionamento supportati, disabilita il thinking e rimuove i blocchi `<think></think>` dalla risposta | `false` |
| `enable_web_search` | string | Abilita la web search per questa richiesta (`off`, `on`, `auto` - auto la abilita a discrezione del modello)<br/>Si applicano costi aggiuntivi basati sull'utilizzo, vedi [pricing](/it/overview/pricing#web-search-and-scraping). | `off` |
| `enable_web_scraping` | boolean | Abilita lo scraping di un massimo di 5 URL rilevati nel messaggio utente. Il contenuto scrapato arricchisce le risposte e bypassa la web search. Solo gli URL scrapati con successo vengono fatturati.<br/>Si applicano costi aggiuntivi basati sull'utilizzo, vedi [pricing](/it/overview/pricing#web-search-and-scraping). | `false` |
| `enable_x_search` | boolean | Abilita la ricerca nativa di xAI (web + X/Twitter) per i modelli Grok supportati (ad esempio `grok-4-20-beta`). Fornisce risultati di ricerca di qualità superiore usando l'infrastruttura di ricerca di xAI. Quando abilitata, la web search standard di Venice viene bypassata.<br/>Si applicano costi aggiuntivi basati sull'utilizzo, vedi [pricing](/it/overview/pricing#web-search-and-scraping). | `false` |
| `enable_web_citations` | boolean | Quando la web search è abilitata, richiede all'LLM di citare le sue fonti usando il formato `[REF]0[/REF]` | `false` |
| `include_search_results_in_stream` | boolean | Sperimentale: Include i risultati della ricerca nello stream come primo chunk emesso | `false` |
| `return_search_results_as_documents` | boolean | Espone i risultati della ricerca in una tool call compatibile OpenAI chiamata `venice_web_search_documents` per l'integrazione con LangChain | `false` |
| `include_venice_system_prompt` | boolean | Se includere i system prompt predefiniti di Venice insieme a quelli specificati | `true` |

<Note>
Questi parametri possono anche essere specificati come suffissi del modello accodati al nome del modello (ad esempio `zai-org-glm-5:enable_web_search=auto`). Vedi [Model Feature Suffix](/it/api-reference/endpoint/chat/model_feature_suffix) per i dettagli.
</Note>

### Prompt Caching

Venice supporta il prompt caching su modelli selezionati per ridurre la latenza e i costi per contenuti ripetuti. Per i modelli supportati, Venice mette automaticamente in cache i system prompt — non sono richieste modifiche al codice. Puoi anche contrassegnare manualmente il contenuto per la cache usando la proprietà `cache_control` sul contenuto del messaggio.

| Parametro | Tipo | Descrizione |
|-----------|------|-------------|
| `prompt_cache_key` | string | Hint di routing opzionale per migliorare il tasso di cache hit. Quando fornito, Venice instrada le richieste verso la stessa infrastruttura backend, aumentando la probabilità di cache hit nelle conversazioni multi-turno. |

Vedi [Prompt Caching](/it/guides/features/prompt-caching) per i dettagli sul funzionamento della cache, la fatturazione e le best practice.

## Riferimento degli header di risposta

Tutte le risposte dell'API Venice includono header HTTP che forniscono metadati sulla richiesta, sui rate limit, sulle informazioni del modello e sul saldo dell'account. Oltre ai codici di errore restituiti dalle risposte API, puoi ispezionare questi header per ottenere l'ID univoco di una particolare richiesta API, monitorare il rate limiting e tracciare il saldo del tuo account.

Venice raccomanda di registrare gli ID di richiesta (header `CF-RAY`) nei deployment di produzione per un troubleshooting più efficiente con il nostro team di supporto, qualora necessario.

La tabella sotto fornisce un riferimento completo di tutti gli header che potresti incontrare:

| Header | Tipo | Scopo | Quando restituito |
|--------|------|---------|---------------|
| **Header HTTP standard** ||||
| `Content-Type` | string | Tipo MIME del body della risposta (`application/json`, `text/csv`, `image/png`, ecc.) | Sempre |
| `Content-Encoding` | string | Codifica usata per comprimere il body della risposta (`gzip`, `br`) | Quando il client invia l'header `Accept-Encoding` |
| `Content-Disposition` | string | Come dovrebbe essere visualizzato il contenuto (ad esempio `attachment; filename=export.csv`) | Durante il download di file o export |
| `Date` | string | Timestamp formattato RFC 7231 di quando la risposta è stata generata | Sempre |
| **Identificazione della richiesta** ||||
| `CF-RAY` | string | Identificatore univoco per questa richiesta API, usato per troubleshooting e richieste di supporto | Sempre |
| `x-venice-version` | string | Versione/revisione corrente del servizio API Venice (ad esempio `20250828.222653`) | Sempre |
| `x-venice-timestamp` | string | Timestamp del server quando la richiesta è stata elaborata (formato ISO 8601) | Quando il tracking dei timestamp è abilitato |
| `x-venice-host-name` | string | Hostname del server che ha elaborato la richiesta | Risposte di errore e scenari di debug |
| **Informazioni sul modello** ||||
| `x-venice-model-id` | string | Identificatore univoco del modello AI utilizzato per la richiesta (ad esempio `venice-01-lite`) | Endpoint di inferenza che usano modelli AI |
| `x-venice-model-name` | string | Nome friendly/visualizzato del modello AI utilizzato (ad esempio `Venice Lite`) | Endpoint di inferenza che usano modelli AI |
| `x-venice-model-router` | string | Servizio di router/backend che ha gestito l'inferenza del modello | Endpoint di inferenza quando le info di routing sono disponibili |
| `x-venice-model-deprecation-warning` | string | Messaggio di avviso per modelli pianificati per la deprecazione | Quando si usa un modello deprecato |
| `x-venice-model-deprecation-date` | string | Data in cui il modello sarà deprecato (data ISO 8601) | Quando si usa un modello deprecato |
| **Informazioni sui rate limit** ||||
| `x-ratelimit-limit-requests` | number | Numero massimo di richieste consentite nella finestra temporale corrente | Tutte le richieste autenticate |
| `x-ratelimit-remaining-requests` | number | Numero di richieste rimanenti nella finestra temporale corrente | Tutte le richieste autenticate |
| `x-ratelimit-reset-requests` | number | Timestamp Unix di quando il rate limit di richieste si resetta | Tutte le richieste autenticate |
| `x-ratelimit-limit-tokens` | number | Numero massimo di token (prompt + completion) consentiti nella finestra temporale | Tutte le richieste autenticate |
| `x-ratelimit-remaining-tokens` | number | Numero di token rimanenti nella finestra temporale corrente | Tutte le richieste autenticate |
| `x-ratelimit-reset-tokens` | number | Durata in secondi fino al reset del rate limit di token | Tutte le richieste autenticate |
| `x-ratelimit-type` | string | Tipo di rate limit applicato (`user`, `api_key`, `global`) | Quando il rate limiting è applicato |
| **Header di paginazione** ||||
| `x-pagination-limit` | number | Numero di elementi per pagina | Endpoint paginati |
| `x-pagination-page` | number | Numero della pagina corrente (1-based) | Endpoint paginati |
| `x-pagination-total` | number | Numero totale di elementi su tutte le pagine | Endpoint paginati |
| `x-pagination-total-pages` | number | Numero totale di pagine | Endpoint paginati |
| **Informazioni sul saldo dell'account** ||||
| `x-venice-balance-diem` | string | Il tuo saldo di token DIEM prima che la richiesta venisse elaborata | Tutte le richieste autenticate |
| `x-venice-balance-usd` | string | Il tuo saldo di crediti USD prima che la richiesta venisse elaborata | Tutte le richieste autenticate |
| **Header di sicurezza del contenuto** ||||
| `x-venice-is-blurred` | string | Indica se l'immagine generata è stata sfocata a causa delle policy di contenuto (`true`/`false`) | Generazione immagini con Safe Venice abilitato |
| `x-venice-is-content-violation` | string | Indica se il contenuto viola le policy di contenuto di Venice (`true`/`false`) | Endpoint di generazione contenuti |
| `x-venice-is-adult-model-content-violation` | string | Indica se il contenuto viola le policy di contenuto dei modelli per adulti (`true`/`false`) | Endpoint di generazione immagini |
| `x-venice-contains-minor` | string | Indica se l'immagine contiene minori (`true`/`false`) | Endpoint di analisi immagini con rilevamento età |
| **Informazioni sul client** ||||
| `x-venice-middleface-version` | string | Versione del client middleface di Venice | Richieste dai client middleface di Venice |
| `x-venice-mobile-version` | string | Versione del client dell'app mobile Venice | Richieste dalle applicazioni mobile |
| `x-venice-request-timestamp-ms` | number | Timestamp della richiesta fornito dal client in millisecondi | Quando il client fornisce un timestamp nella richiesta |
| `x-venice-control-instance` | string | Identificatore dell'istanza di controllo per il debug | Endpoint di generazione immagini per il debug |
| **Header di autenticazione** ||||
| `x-auth-refreshed` | string | Indica che il token di autenticazione è stato aggiornato durante la richiesta (`true`/`false`) | Quando i token di autenticazione vengono aggiornati automaticamente |
| `x-retry-count` | number | Numero di tentativi di retry per la richiesta | Quando si verificano retry della richiesta |

### Note importanti

- **Maiuscole/minuscole nei nomi degli header**: Gli header HTTP non distinguono maiuscole/minuscole, ma Venice usa lettere minuscole con trattini per coerenza
- **Valori stringa**: I valori booleani negli header sono restituiti come stringhe (`"true"` o `"false"`)
- **Valori numerici**: Numeri grandi e valori di saldo possono essere restituiti come stringhe per evitare la perdita di precisione
- **Header opzionali**: Non tutti gli header sono restituiti in ogni risposta; la presenza dipende dall'endpoint e dal contesto della richiesta
- **Compressione**: Usa `Accept-Encoding: gzip, br` nelle richieste per ricevere risposte compresse dove supportato

### Esempio: accesso agli header di risposta

```javascript
// Dopo aver fatto una richiesta API, accedi agli header dall'oggetto risposta
const requestId = response.headers.get('CF-RAY');
const remainingRequests = response.headers.get('x-ratelimit-remaining-requests');
const remainingTokens = response.headers.get('x-ratelimit-remaining-tokens');
const usdBalance = response.headers.get('x-venice-balance-usd');

// Verifica gli avvisi di deprecazione del modello
const deprecationWarning = response.headers.get('x-venice-model-deprecation-warning');
if (deprecationWarning) {
  console.warn(`Model Deprecation: ${deprecationWarning}`);
}
```

## Best practice

1. **Rate limiting**: Monitora gli header `x-ratelimit-remaining-requests` e `x-ratelimit-remaining-tokens` e implementa un backoff esponenziale
2. **Monitoraggio del saldo**: Traccia gli header `x-venice-balance-usd` e `x-venice-balance-diem` per evitare interruzioni del servizio
3. **System prompt**: Testa con e senza i system prompt di Venice per trovare la soluzione migliore per il tuo caso d'uso
4. **API key**: Mantieni sicure le tue API key e ruotale regolarmente
5. **Logging delle richieste**: Registra i valori dell'header `CF-RAY` per il troubleshooting con il supporto
6. **Deprecazione dei modelli**: Verifica gli header `x-venice-model-deprecation-warning` quando usi i modelli

## Differenze rispetto all'API di OpenAI

Mentre Venice mantiene un'alta compatibilità con la specifica dell'API OpenAI, ci sono alcune differenze chiave:

1. **venice_parameters**: Configurazioni aggiuntive come `enable_web_search`, `character_slug` e `strip_thinking_response` per funzionalità estese
2. **System prompt**: Venice accoda i tuoi system prompt a quelli predefiniti che ottimizzano per risposte senza censura (disabilita con `include_venice_system_prompt: false`)
3. **Ecosistema dei modelli**: Venice offre la sua [lineup di modelli](/it/overview/models) inclusi modelli senza censura e di ragionamento — usa gli ID modello Venice anziché i mapping OpenAI
4. **Header di risposta**: Header unici per il tracking del saldo (`x-venice-balance-usd`, `x-venice-balance-diem`), avvisi di deprecazione dei modelli e flag di sicurezza dei contenuti
5. **Policy di contenuto**: Policy più permissive con modelli dedicati senza censura e filtraggio dei contenuti opzionale

## Stabilità dell'API

Venice mantiene la retrocompatibilità per gli endpoint e i parametri v1. Per la policy del ciclo di vita dei modelli, gli avvisi di deprecazione e le linee guida per la migrazione, vedi [Deprecazioni](/it/overview/deprecations).

## Pricing per fascia di qualità

Per i modelli di immagine che accettano il parametro opzionale `quality` (attualmente `gpt-image-2` e `gpt-image-2-edit`), la risposta espone una matrice di prezzi per qualità sotto `model_spec.pricing.quality`. Ogni chiave di primo livello è una fascia di risoluzione (`1K`, `2K`, `4K`) e ogni chiave nidificata è un livello di qualità (`low`, `medium`, `high`) con il proprio prezzo `usd` e `diem`:

```json
"pricing": {
  "resolutions": {
    "1K": { "usd": 0.27, "diem": 0.27 },
    "2K": { "usd": 0.51, "diem": 0.51 },
    "4K": { "usd": 0.84, "diem": 0.84 }
  },
  "quality": {
    "1K": {
      "low":    { "usd": 0.02, "diem": 0.02 },
      "medium": { "usd": 0.07, "diem": 0.07 },
      "high":   { "usd": 0.26, "diem": 0.26 }
    },
    "2K": {
      "low":    { "usd": 0.03, "diem": 0.03 },
      "medium": { "usd": 0.13, "diem": 0.13 },
      "high":   { "usd": 0.50, "diem": 0.50 }
    },
    "4K": {
      "low":    { "usd": 0.05, "diem": 0.05 },
      "medium": { "usd": 0.21, "diem": 0.21 },
      "high":   { "usd": 0.83, "diem": 0.83 }
    }
  }
}
```

`pricing.resolutions` è la tariffa per immagine legacy mantenuta per retrocompatibilità. `pricing.quality` è la matrice per (risoluzione, qualità) che si applica ogni volta che il parametro `quality` è supportato. Entrambi i campi sono mantenuti nella risposta in modo che i client possano rilevare il supporto della qualità e mostrare la matrice nelle proprie UI.

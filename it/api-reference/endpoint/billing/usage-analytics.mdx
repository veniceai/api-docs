Questo è un endpoint beta e può essere instabile o cambiare senza preavviso.
</Warning>

Ottieni analisi aggregate sull'utilizzo per l'utente autenticato, con suddivisioni per data, modello e API key. Questo endpoint fornisce viste sintetiche dei tuoi dati di utilizzo dell'API per costruire dashboard e monitorare il consumo. I dati vengono memorizzati in cache per 10 minuti.

## Parametri di query

Puoi specificare il periodo di tempo per le analisi usando uno dei seguenti:
- **lookback**: Un periodo relativo come "7d" (7 giorni), "30d" (30 giorni), fino a "90d" (90 giorni)
- **startDate** e **endDate**: Un intervallo di date personalizzato nel formato `YYYY-MM-DD`. Sono entrambi obbligatori se uno dei due è fornito.

Se non viene specificato alcun parametro, il periodo di lookback predefinito è di 7 giorni.

## Campi della risposta

### lookback
Il periodo di lookback usato per la query. In formato "Nd" (ad esempio "7d") o "startDate:endDate".

### byDate
Totali di utilizzo giornaliero per il periodo richiesto.

- **date**: La data nel formato `YYYY-MM-DD`
- **USD**: Utilizzo totale in USD per quel giorno
- **DIEM**: Utilizzo totale in DIEM per quel giorno

### byModel
Suddivisione dell'utilizzo per modello, ordinata per spesa totale (dalla più alta).

- **modelName**: Nome visualizzato del modello (ad esempio "GLM 5")
- **unitType**: Tipo di unità consumate (tokens, images, chars, minutes, seconds)
- **modelType**: Tipo di modello (LLM, IMAGE, TTS, ASR, VIDEO), o null
- **totalUsd**: Totale USD spesi su questo modello
- **totalDiem**: Totale DIEM spesi su questo modello
- **totalUnits**: Totale unità consumate per questo modello
- **breakdown**: Array di suddivisioni di utilizzo per tipo (presente solo se ci sono più tipi). Ogni voce contiene:
  - **type**: Tipo di token (ad esempio "Input", "Output", "Cache Read", "Cache Write")
  - **usd**: Importo in USD per questa suddivisione
  - **diem**: Importo in DIEM per questa suddivisione
  - **units**: Numero di unità per questa suddivisione

### byModelDaily
Dati grafico giornalieri per i primi 8 modelli. Ogni voce contiene una "date" (timestamp) più i nomi dei modelli come chiavi con i valori di utilizzo in DIEM.

### topModels
Array dei primi 8 nomi di modelli per utilizzo, per le legende dei grafici.

### byKey
Suddivisione dell'utilizzo per API key, ordinata per spesa totale (dalla più alta).

- **apiKeyId**: L'ID dell'API key, o null se l'utilizzo proveniva dall'app web
- **description**: Descrizione dell'API key o "Web App"
- **totalUsd**: Totale USD spesi tramite questa key
- **totalDiem**: Totale DIEM spesi tramite questa key
- **totalUnits**: Totale unità consumate tramite questa key

### byKeyDaily
Dati grafico giornalieri per le prime 8 API key. Ogni voce contiene una "date" (timestamp) più le descrizioni delle key come chiavi con i valori di utilizzo in DIEM.

### topKeyNames
Array delle prime 8 descrizioni di API key per utilizzo, per le legende dei grafici.

## Esempio di utilizzo

```bash
# Ottieni le analisi di utilizzo degli ultimi 7 giorni (predefinito)
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Ottieni le analisi di utilizzo degli ultimi 30 giorni
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics?lookback=30d" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Ottieni le analisi di utilizzo per un intervallo di date specifico
curl -X GET "https://api.venice.ai/api/v1/billing/usage-analytics?startDate=2024-01-01&endDate=2024-01-31" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Venice is built for private, zero-retention inference. We do not retain API prompts or generations for moderation review. Moderation decisions usually happen at the model or upstream provider layer, or through automated safety checks required for specific endpoints.

Some models and upstream providers still apply their own content policies. If they block a request, Venice returns that moderation result to the API user so clients can handle it directly.

These responses do not change your obligation to use the API in compliance with Venice's Terms of Service and applicable law.

## When moderation can happen

Moderation can happen when a provider, model host, or model implementation rejects a prompt, input, or output. Some endpoints may also have workflow-specific safety handling that affects the response. Some models are marked as moderated to signal known provider-side or model-side checks, but the label is not exhaustive. A request may still be blocked even when a model is not marked as moderated.

If provider-side moderation behavior is not suitable for your application, choose a model whose moderation characteristics better match your requirements, or handle moderation responses in your client.

## Chat completions

For non-streaming chat completions, moderated requests return an error response instead of a completion.

```json
{
  "error": "Your prompt violates the content policy of Venice.ai or the model provider"
}
```

The response uses HTTP `422`.

For streaming chat completions, if a provider blocks content mid-stream, the stream emits an OpenAI-compatible error event and then ends.

```json
{
  "error": {
    "message": "Content blocked by provider",
    "type": "content_filter_error",
    "code": "content_blocked_by_provider",
    "param": null
  }
}
```

## Images

Image behavior depends on the endpoint and model.

Some image generation requests return HTTP `200` with moderation headers, even when the output was blocked or replaced.

```text
x-venice-is-content-violation: true
x-venice-is-blurred: true
```

In these cases, the returned image may be blurred, blacked out, or replaced with a content-policy placeholder.

Other image and image-editing flows may return HTTP `422` with a content-policy error when the provider or required safety checks reject the request.

## Videos

Video generation is asynchronous. A moderation block is usually surfaced when retrieving the video result.

If a video is blocked and credits are refunded, the retrieve endpoint returns HTTP `422`.

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits have been refunded.",
  "credits_refunded": true
}
```

For models where the upstream provider charges for moderated requests, the response may be:

```json
{
  "error": "Your generation was blocked due to the model provider's content policies. Credits were not refunded as this model charges for content-moderated requests.",
  "credits_refunded": false
}
```

Some flows may also suggest an alternative model or a provider-approved prompt when one is available.

## What to do when content is moderated

If your request is blocked:

- Review the selected model. Moderated models may reject content according to provider-side or model-side policies.
- If the response suggests an alternative model or prompt, use it only if it fits your application and compliance requirements.
- Revise the prompt or input media if the request unintentionally triggered the provider's policy filters.
- For video requests, check `credits_refunded` to know whether the request was refunded.

Venice exposes these responses so API clients can handle moderation clearly instead of treating it as an unknown failure.

Reference to Video (R2V) models let you generate videos where characters and objects maintain consistent identity across frames. Define elements with reference images, and the model keeps them recognizable throughout the video.

## Supported Models

| Model ID | Tier | Best For |
|----------|------|----------|
| `kling-o3-pro-reference-to-video` | Pro | Highest quality and strongest character consistency |
| `kling-o3-standard-reference-to-video` | Standard | Balanced quality, speed, and cost |

Both models share the same API format. Examples below use the Pro model; swap the `model` value to use Standard.

---

## Quick Start

Queue a video generation request, then retrieve the result when ready.

### 1. Queue the Video

<CodeGroup>
```bash cURL
curl -X POST "https://api.venice.ai/api/v1/video/queue" \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-o3-pro-reference-to-video",
    "prompt": "A person walks through a sunny park",
    "duration": "5s",
    "image_url": "https://example.com/person.png"
  }'
```

```python Python
import requests
import os

response = requests.post(
    "https://api.venice.ai/api/v1/video/queue",
    headers={
        "Authorization": f"Bearer {os.environ['VENICE_API_KEY']}",
        "Content-Type": "application/json"
    },
    json={
        "model": "kling-o3-pro-reference-to-video",
        "prompt": "A person walks through a sunny park",
        "duration": "5s",
        "image_url": "https://example.com/person.png"
    }
)

result = response.json()
print(f"Queue ID: {result['queue_id']}")
```

```javascript Node.js
const response = await fetch("https://api.venice.ai/api/v1/video/queue", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.VENICE_API_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    model: "kling-o3-pro-reference-to-video",
    prompt: "A person walks through a sunny park",
    duration: "5s",
    image_url: "https://example.com/person.png"
  })
});

const result = await response.json();
console.log("Queue ID:", result.queue_id);
```
</CodeGroup>

**Response:**

```json
{
  "model": "kling-o3-pro-reference-to-video",
  "queue_id": "abc123-def456-..."
}
```

### 2. Retrieve the Result

Poll the retrieve endpoint with the `queue_id` and `model` from the queue response:

```bash
curl -X POST "https://api.venice.ai/api/v1/video/retrieve" \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-o3-pro-reference-to-video",
    "queue_id": "abc123-def456-..."
  }'
```

## Elements

Elements define characters or objects that maintain consistent identity throughout the video. Reference them in your prompt as `@Element1`, `@Element2`, etc.

Each element can be **image-based** or **video-based** (not both):

**Image-based element:**

```json
{
  "frontal_image_url": "https://example.com/character-front.png",
  "reference_image_urls": [
    "https://example.com/character-side.png",
    "https://example.com/character-back.png"
  ]
}
```

**Video-based element:**

```json
{
  "video_url": "https://example.com/character-reference.mp4"
}
```

<Warning>
Each image-based element **must include `frontal_image_url`**. You cannot provide only `reference_image_urls`.
</Warning>

### Example: Two Characters Interacting

```bash
curl -X POST "https://api.venice.ai/api/v1/video/queue" \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-o3-pro-reference-to-video",
    "prompt": "@Element1 and @Element2 enter the scene from opposite sides. They meet in the center and shake hands.",
    "duration": "8s",
    "image_url": "https://example.com/scene-start.png",
    "elements": [
      {
        "frontal_image_url": "https://example.com/person-a-front.png",
        "reference_image_urls": ["https://example.com/person-a-side.png"]
      },
      {
        "frontal_image_url": "https://example.com/person-b-front.png"
      }
    ]
  }'
```

## Scene Images

Scene images provide style, environment, or appearance references. Reference them in your prompt as `@Image1`, `@Image2`, etc.

### Example: Environment Reference

```bash
curl -X POST "https://api.venice.ai/api/v1/video/queue" \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-o3-pro-reference-to-video",
    "prompt": "@Element1 explores the mystical forest shown in @Image1, then transitions to the beach in @Image2",
    "duration": "10s",
    "image_url": "https://example.com/start-frame.png",
    "elements": [
      { "frontal_image_url": "https://example.com/explorer.png" }
    ],
    "scene_image_urls": [
      "https://example.com/mystical-forest.png",
      "https://example.com/sunset-beach.png"
    ]
  }'
```

## Start and End Frames

Control the first and last frames for smooth transitions or specific poses using `image_url` and `end_image_url`.

### Example: Pose Transition

```bash
curl -X POST "https://api.venice.ai/api/v1/video/queue" \
  -H "Authorization: Bearer $VENICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-o3-pro-reference-to-video",
    "prompt": "@Element1 smoothly transitions from sitting to standing",
    "duration": "5s",
    "image_url": "https://example.com/person-sitting.png",
    "end_image_url": "https://example.com/person-standing.png",
    "elements": [
      { "frontal_image_url": "https://example.com/person-front.png" }
    ]
  }'
```

## Audio

Audio generation is enabled by default. The model generates native audio based on your prompt content.

<Tip>
Include dialogue directly in your prompt: `@Element1 says "Hello, welcome to Venice AI!"`
</Tip>

To disable audio:

```json
{
  "audio": false
}
```

## Base64 Images

You can send images as base64 data URLs instead of HTTP URLs:

```
data:image/<format>;base64,<base64-encoded-data>
```

Supported formats: `png`, `jpeg`, `jpg`, `webp`, `gif`

<CodeGroup>
```python Python
import base64

def encode_image(path):
    with open(path, 'rb') as f:
        encoded = base64.b64encode(f.read()).decode()
        ext = path.split('.')[-1]
        mime = 'jpeg' if ext in ['jpg', 'jpeg'] else ext
        return f'data:image/{mime};base64,{encoded}'

# Use in request
"image_url": encode_image('start-frame.png')
```

```javascript Node.js
const fs = require('fs');

const imageBuffer = fs.readFileSync('image.png');
const base64Image = `data:image/png;base64,${imageBuffer.toString('base64')}`;

// Use in request
image_url: base64Image
```

```bash Bash
BASE64_IMAGE="data:image/png;base64,$(base64 -i image.png | tr -d '\n')"
```
</CodeGroup>

<Note>
Large base64 strings increase request size. For images over 5MB, consider using HTTP URLs instead.
</Note>

## Parameters

### Required

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | string | `kling-o3-pro-reference-to-video` or `kling-o3-standard-reference-to-video` |
| `prompt` | string | What should happen in the video (max 2500 chars). Use `@Element1`, `@Image1`, etc. to reference elements and scene images. |
| `duration` | string | Video length: `3s`, `4s`, `5s`, `6s`, `7s`, `8s`, `9s`, `10s`, `11s`, `12s`, `13s`, `14s`, `15s` |

### Visual Input (at least one required)

| Parameter | Type | Description |
|-----------|------|-------------|
| `image_url` | string | Start frame image URL or data URL |
| `elements` | array | Characters/objects to include (max 4) |
| `scene_image_urls` | array | Reference images for style/appearance (max 4) |

### Optional

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `aspect_ratio` | string | `16:9` | `16:9`, `9:16`, or `1:1` |
| `audio` | boolean | `true` | Generate native audio |
| `end_image_url` | string | — | End frame image URL |

---

## Limits

- Maximum **4 elements** per request
- Maximum **3 reference images** per element
- Maximum **4 scene images**
- Each element must be **image-based OR video-based** (not both)
- Maximum prompt length: **2500 characters**
- Video duration: **3-15 seconds**

---

## Best Practices

1. **Clear element references** — Always use `@Element1`, `@Element2` syntax when you want specific characters to appear

2. **Quality reference images** — Use clear, well-lit frontal images. Include side/back references for complex movements

3. **Be descriptive** — `"@Element1 slowly walks toward the camera, smiling"` works better than `"Person walking"`

4. **Match duration to complexity** — Simple actions (wave, nod): 3-5s. Multi-element interactions: 10-15s

5. **Use scene images for consistency** — Reference environment/style images to maintain visual coherence across shots

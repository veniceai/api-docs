## Quick Start

The fastest way to use x402 is with the official SDK:

```bash
npm install @venice-ai/x402-client
```

```typescript
import { VeniceClient } from '@venice-ai/x402-client'

// Initialize with your wallet's private key
const venice = new VeniceClient(process.env.WALLET_KEY)

// Top up your balance (minimum $5)
await venice.topUp(10) // $10 USDC on Base

// Make requests - charges deducted automatically
const response = await venice.chat({
  model: 'llama-3.3-70b',
  messages: [{ role: 'user', content: 'Hello!' }]
})

console.log(response.choices[0].message.content)
```

That's it. Five lines to authenticated AI inference.

<Info>
**Prefer API keys?** If you want traditional authentication, see [Generating an API Key](/overview/guides/generating-api-key). x402 is best for agents, privacy-focused use cases, or when you want to avoid account creation.
</Info>

## Why x402?

| Feature | API Key | x402 Wallet |
|---------|---------|-------------|
| Account required | Yes | No |
| KYC required | For some payment methods | No |
| Setup time | Minutes | Seconds |
| Key management | You manage API keys | Sign with wallet |
| Payment | Credit card, crypto | USDC on Base |
| Agent-friendly | Requires secure key storage | Native wallet signing |
| Privacy | Linked to account | Pseudonymous |

**Use x402 when:**
- Building autonomous AI agents that need to pay for their own compute
- You want privacy and don't want to create an account
- You're already in the crypto ecosystem and have a wallet
- You need programmatic, permissionless access

**Use API keys when:**
- You're building a traditional web app
- You want simpler integration without crypto dependencies
- Your team isn't familiar with wallets/signing

## Prerequisites

Before using x402, you need:

1. **An EVM wallet on Base** - Create one with [Coinbase Wallet](https://www.coinbase.com/wallet), [MetaMask](https://metamask.io/), [Phantom](https://phantom.app/), or any EVM-compatible wallet

2. **USDC on Base** - For topping up credits. Bridge from other chains or buy directly on Base

3. **Small amount of ETH on Base** - For gas fees (~$0.01 per transaction)

<Warning>
**Security best practice:** Use a dedicated wallet for x402, not your main treasury. This limits exposure if your agent's private key is compromised.
</Warning>

## How It Works

x402 uses two authentication mechanisms:

1. **`X-Sign-In-With-X`** - For balance checks and inference requests. A signed [SIWE (Sign-In With Ethereum)](https://eips.ethereum.org/EIPS/eip-4361) message that proves wallet ownership.

2. **`X-402-Payment`** - For topping up credits. A signed USDC transfer authorization following the [x402 protocol](https://x402.org).

```text
┌─────────────────────────────────────────────────────────────────┐
│                        x402 Flow                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Check Balance (optional)                                    │
│     GET /x402/balance/{address}                                 │
│     Header: X-Sign-In-With-X: <signed SIWE>                     │
│                                                                 │
│  2. Top Up (if needed)                                          │
│     POST /x402/top-up (no header) → get payment requirements    │
│     POST /x402/top-up + X-402-Payment: <signed payment>         │
│                                                                 │
│  3. Make Requests                                               │
│     POST /chat/completions                                      │
│     Header: X-Sign-In-With-X: <signed SIWE>                     │
│     → Credits deducted automatically                            │
│                                                                 │
│  4. Monitor (optional)                                          │
│     GET /x402/transactions/{address}                            │
│     Header: X-Sign-In-With-X: <signed SIWE>                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Integration Guide

### Option 1: SDK (Recommended)

The `@venice-ai/x402-client` package handles all signing automatically.

#### Basic Chat

```typescript
import { VeniceClient } from '@venice-ai/x402-client'

const venice = new VeniceClient(process.env.WALLET_KEY)

const response = await venice.chat({
  model: 'llama-3.3-70b',
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user', content: 'Explain quantum computing in simple terms.' }
  ],
  max_tokens: 1000
})
```

#### Streaming

```typescript
const stream = await venice.chat({
  model: 'llama-3.3-70b',
  messages: [{ role: 'user', content: 'Write a haiku about code.' }],
  stream: true
})

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '')
}
```

#### With OpenAI SDK

If you're using a tool that accepts a custom fetch function:

```typescript
import OpenAI from 'openai'
import { createAuthFetch } from '@venice-ai/x402-client'

const authFetch = createAuthFetch(process.env.WALLET_KEY)

const openai = new OpenAI({
  baseURL: 'https://api.venice.ai/api/v1',
  apiKey: 'not-needed', // x402 handles auth
  fetch: authFetch
})

const response = await openai.chat.completions.create({
  model: 'llama-3.3-70b',
  messages: [{ role: 'user', content: 'Hello!' }]
})
```

#### Check Balance

```typescript
const balance = await venice.getBalance()
console.log(`Balance: $${balance.balanceUsd}`)
console.log(`Can consume: ${balance.canConsume}`)
console.log(`DIEM credits: $${balance.diemBalanceUsd || 0}`)
```

### Option 2: Direct HTTP (Advanced)

If you need full control or are using a language without SDK support, you can call the API directly.

#### Step 1: Generate SIWE Auth Header

The `X-Sign-In-With-X` header is a base64-encoded JSON object containing a signed SIWE message.

<CodeGroup>
```typescript TypeScript
import { Wallet } from 'ethers'
import { SiweMessage, generateNonce } from 'siwe'

async function createAuthHeader(privateKey: string, uri: string): Promise<string> {
  const wallet = new Wallet(privateKey)
  const address = wallet.address
  const issuedAt = new Date().toISOString()
  
  const siwe = new SiweMessage({
    domain: 'outerface.venice.ai',
    address: address,
    statement: 'Sign in to Venice AI',
    uri: uri,
    version: '1',
    chainId: 8453, // Base
    nonce: generateNonce(),
    issuedAt: issuedAt,
    expirationTime: new Date(Date.now() + 10 * 60 * 1000).toISOString() // 10 min
  })
  
  const message = siwe.prepareMessage()
  const signature = await wallet.signMessage(message)
  
  return Buffer.from(JSON.stringify({
    address: address.toLowerCase(),
    message: message,
    signature: signature,
    chainId: 8453,
    timestamp: Date.parse(issuedAt)
  })).toString('base64')
}

// Usage
const authHeader = await createAuthHeader(
  process.env.WALLET_KEY,
  'https://api.venice.ai/api/v1/chat/completions'
)
```

```python Python
from eth_account import Account
from eth_account.messages import encode_defunct
from siwe import SiweMessage
import json
import base64
from datetime import datetime, timedelta
import secrets

def create_auth_header(private_key: str, uri: str) -> str:
    account = Account.from_key(private_key)
    address = account.address
    issued_at = datetime.utcnow()
    
    siwe = SiweMessage(
        domain='outerface.venice.ai',
        address=address,
        statement='Sign in to Venice AI',
        uri=uri,
        version='1',
        chain_id=8453,
        nonce=secrets.token_hex(8),
        issued_at=issued_at.isoformat() + 'Z',
        expiration_time=(issued_at + timedelta(minutes=10)).isoformat() + 'Z'
    )
    
    message = siwe.prepare_message()
    signable = encode_defunct(text=message)
    signature = account.sign_message(signable)
    
    payload = {
        'address': address.lower(),
        'message': message,
        'signature': signature.signature.hex(),
        'chainId': 8453,
        'timestamp': int(issued_at.timestamp() * 1000)
    }
    
    return base64.b64encode(json.dumps(payload).encode()).decode()

# Usage
auth_header = create_auth_header(
    os.environ['WALLET_KEY'],
    'https://api.venice.ai/api/v1/chat/completions'
)
```
</CodeGroup>

<Warning>
**Important:** The `uri` in the SIWE message must match the endpoint you're calling. Generate a fresh header for each different endpoint.
</Warning>

#### Step 2: Check Balance

```bash
export WALLET_ADDRESS=0xYourWalletAddress
export AUTH_HEADER=$(node generate-auth.js "https://api.venice.ai/api/v1/x402/balance/$WALLET_ADDRESS")

curl -X GET "https://api.venice.ai/api/v1/x402/balance/$WALLET_ADDRESS" \
  -H "X-Sign-In-With-X: $AUTH_HEADER"
```

Response:
```json
{
  "success": true,
  "data": {
    "walletAddress": "0x...",
    "balanceUsd": 12.50,
    "canConsume": true,
    "minimumTopUpUsd": 5,
    "suggestedTopUpUsd": 10,
    "diemBalanceUsd": 5.25
  }
}
```

| Field | Description |
|-------|-------------|
| `balanceUsd` | Current USDC credit balance |
| `canConsume` | Whether you can make paid requests |
| `diemBalanceUsd` | Credits from staked DIEM (used first) |
| `minimumTopUpUsd` | Minimum top-up amount ($5) |

#### Step 3: Top Up (if needed)

First, request payment requirements:

```bash
curl -X POST "https://api.venice.ai/api/v1/x402/top-up"
```

Response (402 Payment Required):
```json
{
  "error": "PAYMENT_REQUIRED",
  "message": "Send x402 payment via X-402-Payment header",
  "paymentInfo": {
    "receiverWallet": "0x2670B922ef37C7Df47158725C0CC407b5382293F",
    "network": "eip155:8453",
    "token": "USDC",
    "tokenAddress": "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
    "minimumAmountUsd": 5,
    "suggestedAmountUsd": 10
  }
}
```

Then create and send the payment:

```bash
# Install x402 SDK
npm install x402 viem

# Generate payment header and top up
export X402_PAYMENT=$(node -e "
const { createPaymentHeader } = require('x402/client');
const { privateKeyToAccount } = require('viem/accounts');

(async () => {
  const signer = privateKeyToAccount(process.env.WALLET_KEY);
  const amountUsd = 10;
  const header = await createPaymentHeader(signer, 2, {
    scheme: 'exact',
    network: 'base',
    maxAmountRequired: String(amountUsd * 1e6),
    resource: 'https://api.venice.ai/api/v1/x402/top-up',
    description: 'Venice x402 top-up',
    mimeType: 'application/json',
    payTo: '0x2670B922ef37C7Df47158725C0CC407b5382293F',
    maxTimeoutSeconds: 300,
    asset: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913',
    extra: { name: 'USD Coin', version: '2' }
  });
  process.stdout.write(header);
})();
")

curl -X POST "https://api.venice.ai/api/v1/x402/top-up" \
  -H "X-402-Payment: $X402_PAYMENT"
```

Response:
```json
{
  "success": true,
  "data": {
    "walletAddress": "0x...",
    "amountCredited": 10,
    "newBalance": 22.50,
    "paymentId": "payment_01HZY8M4W4Y6QZ8B6Q4P0V3J2K"
  }
}
```

#### Step 4: Make Inference Requests

```bash
export AUTH_HEADER=$(node generate-auth.js "https://api.venice.ai/api/v1/chat/completions")

curl -X POST "https://api.venice.ai/api/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "X-Sign-In-With-X: $AUTH_HEADER" \
  -d '{
    "model": "llama-3.3-70b",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

#### Step 5: View Transaction History

```bash
export AUTH_HEADER=$(node generate-auth.js "https://api.venice.ai/api/v1/x402/transactions/$WALLET_ADDRESS")

curl -X GET "https://api.venice.ai/api/v1/x402/transactions/$WALLET_ADDRESS?limit=10" \
  -H "X-Sign-In-With-X: $AUTH_HEADER"
```

Response includes `TOP_UP`, `CHARGE`, and `REFUND` entries with timestamps, amounts, and request IDs.

## Supported Endpoints

x402 wallet auth works with these Venice endpoints:

| Category | Endpoint | Auth Header |
|----------|----------|-------------|
| **Chat** | `POST /chat/completions` | `X-Sign-In-With-X` |
| **Balance** | `GET /x402/balance/{address}` | `X-Sign-In-With-X` |
| **Transactions** | `GET /x402/transactions/{address}` | `X-Sign-In-With-X` |
| **Top-Up** | `POST /x402/top-up` | `X-402-Payment` |

<Info>
Currently only `chat/completions` supports x402 for inference. Image, audio, video, and embeddings endpoints require API key authentication. Want x402 support for other endpoints? [Request it on Featurebase](https://featurebase.venice.ai/).
</Info>

## For AI Agents

### Machine-Readable Prompt

Copy this prompt to give your agent x402 capabilities:

```text
VENICE x402 WALLET AUTHENTICATION

You can use the Venice API with your Ethereum wallet. No API key needed.

AUTHENTICATION:
1. Build a SIWE message with these fields:
   - domain: outerface.venice.ai
   - address: your wallet address
   - statement: "Sign in to Venice AI"
   - uri: the exact Venice endpoint URL you're calling
   - version: "1"
   - chainId: 8453 (Base)
   - nonce: random 16-char hex string
   - issuedAt: current ISO timestamp
   - expirationTime: current time + 10 minutes (ISO format)

2. Sign the message with your private key (EIP-191)

3. Create JSON payload:
   {"address":"<lowercase>","message":"<siwe>","signature":"0x...","chainId":8453,"timestamp":<unix_ms>}

4. Base64-encode the JSON and send as X-Sign-In-With-X header

ENDPOINTS:
- Chat: POST https://api.venice.ai/api/v1/chat/completions
- Balance: GET https://api.venice.ai/api/v1/x402/balance/<address>
- Transactions: GET https://api.venice.ai/api/v1/x402/transactions/<address>
- Top-up: POST https://api.venice.ai/api/v1/x402/top-up (uses X-402-Payment, not X-Sign-In-With-X)

PAYMENT:
- DIEM staked to wallet is used first (free daily credits)
- Otherwise, USDC balance on Base is charged
- Minimum top-up: $5 USD
- Top-up uses X-402-Payment header (different from auth)

IMPORTANT:
- Generate a NEW X-Sign-In-With-X header for each different endpoint
- Headers expire after 10 minutes
- The uri in SIWE must match the endpoint being called

SDK: npm install @venice-ai/x402-client
Docs: https://docs.venice.ai/overview/guides/x402-venice-api
```

## Pricing & Costs

### How Credits Work

1. **DIEM credits used first** - If you've staked DIEM tokens, those daily credits are consumed before USDC
2. **USDC credits used second** - Your topped-up balance is charged per request
3. **Cost varies by model** - Check the [pricing page](/overview/pricing) for per-token costs

### DIEM Staking

Stake DIEM tokens for perpetual daily credits:
- **1 DIEM staked = $1/day** in inference credits
- Credits reset daily
- No top-ups needed while staked
- Stake at [venice.ai/stake](https://venice.ai/stake)

## Troubleshooting

### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `401 Authentication failed` | Invalid or expired SIWE signature | Regenerate `X-Sign-In-With-X` header. Check that nonce is random, timestamps are correct, and signature matches wallet. |
| `403 Unauthorized` | Wallet address mismatch | The address in `X-Sign-In-With-X` must match the `{walletAddress}` in the URL path. |
| `402 Insufficient balance` | Not enough credits | Top up with `POST /x402/top-up` or stake more DIEM. |
| `402 PAYMENT_REQUIRED` (on top-up) | Expected - need to provide payment | This is the first step of top-up. Use the returned `paymentInfo` to create `X-402-Payment` header. |
| `400 Invalid request` | Malformed header or payload | Check JSON structure, ensure base64 encoding is correct, verify all required fields are present. |
| `429 Rate limit exceeded` | Too many requests | Implement exponential backoff. Check [rate limits](/api-reference/rate-limiting). |

### Common Mistakes

1. **Wrong `uri` in SIWE message** - The most common error. The `uri` must exactly match the endpoint you're calling:
   - Balance check: `https://api.venice.ai/api/v1/x402/balance/0x...`
   - Chat: `https://api.venice.ai/api/v1/chat/completions`

2. **Reusing expired headers** - SIWE headers expire after 10 minutes. Generate fresh headers for each request or batch of requests.

3. **Using `X-Sign-In-With-X` for top-up** - The top-up endpoint uses `X-402-Payment`, not SIWE auth.

4. **Incorrect address format** - Use lowercase addresses in the JSON payload.

5. **Missing gas for top-up** - The USDC transfer requires a small amount of ETH on Base for gas (~$0.01).

## Security Best Practices

1. **Use a dedicated wallet** - Create a separate wallet for x402 with limited funds. Don't use your main treasury.

2. **Rotate wallets periodically** - If a private key is exposed, create a new wallet and transfer remaining balance.

3. **Set spending limits** - Only top up what you need. Keep the wallet balance low.

4. **Secure private key storage**:
   - Use environment variables, not hardcoded strings
   - Use secret managers in production (AWS Secrets Manager, HashiCorp Vault, etc.)
   - Never log or print private keys

5. **Monitor transactions** - Regularly check `/x402/transactions` for unexpected charges.

6. **Short header expiration** - The 10-minute SIWE expiration limits the window for replay attacks.

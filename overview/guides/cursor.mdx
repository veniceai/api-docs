## Setup

<Steps>
  <Step title="Get Your API Key">
    Generate a key from [venice.ai/settings/api](https://venice.ai/settings/api).
  </Step>

  <Step title="Open Cursor Settings">
    Go to **Cursor Settings → Models → OpenAI API Key** and enter:
    - **API Key**: Your Venice API key
    - **Base URL**: `https://api.venice.ai/api/v1`
  </Step>

  <Step title="Add Models">
    Add the Venice model IDs you want to use. See the [model catalog](/models/text) for available models.
  </Step>
</Steps>

## The `venice-` Model Prefix

<Warning>
**Claude model IDs must be prefixed with `venice-` in Cursor.**

Cursor auto-detects model IDs starting with `claude-` and rewrites the request into Anthropic's native format instead of OpenAI-compatible format. This breaks the Venice API since Venice expects OpenAI-compatible requests.
</Warning>

The fix is simple: prefix the model ID with `venice-`. Venice strips this prefix during model resolution, so the request works correctly while bypassing Cursor's detection logic.

| Model | ❌ Don't use | ✅ Use in Cursor |
|-------|-------------|-----------------|
| Claude Opus 4.6 | `claude-opus-4-6` | `venice-claude-opus-4-6` |
| Claude Sonnet 4.6 | `claude-sonnet-4-6` | `venice-claude-sonnet-4-6` |
| Claude Opus 4.5 | `claude-opus-45` | `venice-claude-opus-45` |
| Claude Sonnet 4.5 | `claude-sonnet-45` | `venice-claude-sonnet-45` |

<Info>
Non-Claude models (e.g. `deepseek-r1`, `llama-3.3-70b`, `gpt-4o`) are **not affected** and work without the prefix. The `venice-` prefix is safe to use on any model — Venice will always strip it — but it's only required for Claude models in Cursor.
</Info>

### Why This Happens

Cursor intercepts requests for Claude models and converts them to Anthropic's native API format. Since Venice uses the OpenAI-compatible format exclusively, this rewriting causes requests to fail. The `venice-` prefix prevents Cursor from recognizing the model as a Claude model, so the request passes through in the correct OpenAI format.

## Using Model Feature Suffix

You can combine the `venice-` prefix with [Model Feature Suffix](/api-reference/endpoint/chat/model_feature_suffix) for additional capabilities:

```
venice-claude-opus-4-6:enable_web_search=auto
```

```
venice-claude-sonnet-4-6:enable_web_search=on&enable_web_scraping=true
```
